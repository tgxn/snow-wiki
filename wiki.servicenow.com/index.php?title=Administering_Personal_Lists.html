1 Overview

Users can customize the layout for any list view by creating a personal list. Administrators can manage this function using the following options:

  • Activate or deactivate it globally
  • Control user access to it based on roles
  • Manage users' personal lists

The personal list function requires the list mechanic UI Macro.

Note: Administrators may also create global views for various roles or tasks. Personal lists modify a specific view according to individual user preferences.

2 Activating the List Mechanic

To activate the list mechanic:

  1. In the application navigator, select System UI > UI Macros.
  2. Activate (or deactivate) the list mechanic UI Macro:
    • list_mechanic, List original
    • list_mechanic2, List v2 plugin

3 Controlling Access to the List Mechanic

To control who can create personal lists:

  1. Navigate to System Properties > UI Properties.
  2. Locate the field labelled List of roles (comma-separated) that can use Personal Lists. If blank, all can use.
  3. Enter the roles for which personal lists are available, separated by commas, or leave the field blank to allow access for all users (default).
    This field modifies the glide.ui.list_mechanic.roles property.

4 Managing Personal Lists

To manage personal lists:

  1. In the application navigator, select System UI > Lists.
  2. Personal lists have a user value in the Sys user field.
