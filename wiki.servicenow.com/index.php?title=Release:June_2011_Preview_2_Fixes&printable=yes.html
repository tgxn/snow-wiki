1 June 2011 Preview 2 Fixes

1.1 PRB560386 (Data): Insert and Insert and Stay UI actions led to duplicate numbers

Using the Insert or Insert and Stay UI Actions will now blank the number field for a record, so when the new record is inserted it will get a new number.

1.2 PRB560547 (Views): An admin might delete the Default view

ACLs and UI Policy now prevent an admin from deleting the Default view.

1.3 PRB560364 (Filters): After running a filter with a date field and the "between" operator, a backslash might improperly be appended to the date specified

1.4 PRB560495, PRB560496 (i18n): Field and List context menus might not display the expected language

A cacheing issue has been fixed to resolve this issue.

1.5 PRB560265 (Currency): A custom currency with an invalid (non-ISO standard) currency code could not be saved

Custom currencies are now supported.

1.6 PRB043577 (UI Policy): Top-level OR conditions were not suppoted in UI Policy

UI Policy has been enhanced to support top-level OR conditions. e.g., (x=1 and y=2) OR x=2

1.7 PRB560356 (i18n): The "translate and learn" property did not work as expected for translated_fields on catalog items

Catalog items now write out translated field entries as expected when "translate and learn" is enabled.

1.8 PRB560152 (Reports): When printing a homepage with a report with grid enabled, the grid improperly wasn't included

1.9 PRB560511 (Rota): A rota could possibly skip a 24-hour period if the period contained a Daylight Saving Time shift

1.10 PRB560337 (i18n): With a comma used as the decimal separator, submitting a form with a read-only decimal field could cause an Invalid Number error

1.11 PRB559586 (SLA): Task SLA Pause Duration might not calculate properly

Now when a SLA goes directly from Paused to Stopped without the Pause Condition ever evaluating to false, the pause duration is correctly calculated.==

1.12 PRB560215 (SLA): SLA Reset Condition was not supported, even though the field existed

With the new June 2011 SLA engine, the SLA Reset Condition is supported. Support will not be added to legacy SLA engine.

1.13 PRB559605 (SLA): Relative durations with Pause conditions are not supported for SLA's, but give no indication is given that this is so

Now the Pause Condition will be removed from the form when a relative duration is selected, since a pause condition doesn't make sense in conjunction with a relative duration.

1.14 PRB560471 (i18n): Many-to-many Related List labels were not properly translated

1.15 PRB560350 (UI): The Back button when clicked in a reference field popup did not redirect correctly

1.16 PRB560373 (Reports): "Display grid" and "Grid size" were improperly hidden for Line Chart reports

1.17 PRB560497 (Database Views): CSV export of a database view that contained only a single table did not return data

1.18 PRB560494 (Reports): Click through from a Control Chart used an invalid filter, improperly displaying all records in the list

1.19 PRB560442 (Search): A global text search containing square brackets (e.g.: "[INC0224690]") generated an error in the system log

1.20 PRB560357 (Service Catalog): Deleting a variable from a catalog item would not properly delete it from Requested Items

1.21 PRB560472 (Approvals): Approval Summary for a catalog Request might not properly display variables in the summarizer

1.22 PRB560348 (Service Catalog): When cloning a Request for another user, the other user's addresses was not used as the delivery address

1.23 PRB560460 (Auditing): Since sys_ fields are not audited, changes to the sys_domain field were also not audited

The sys_domain field is now audited as an exception to the rule.

1.24 PRB560454 (History): The History Calendar might report a user as the user's email address, rather than their name

1.25 PRB559226 (Import Sets): A HTTP Post bulk-loaded CSV file could not be transformed

1.26 PRB560419 (Subscriptions): As of Fall 2010, SMS users could no longer subscribe to notifications that were not subscribable

Apparently, SMS users had been relying on the previous behavior, so the restricting reference qualifier has been removed from the "Notification Message" field until the subscription logic can be enhanced.

1.27 PRB560270 (i18n): List V2 context menus (sys_ui_context_menu) were not translatable

sys_ui_context_menu.name is now a translated field.

1.28 PRB560396 (List Editing): List Editing required write access to all dependent fields in order to set a reference value

e.g., if Configuration Item is dependent on Company, user required write access to both fields to list edit Configuration Item. This limitation has been removed, and only access to the edited field is required.

1.29 PRB560304 (History): Duplicate entries could be created in History Sets for the same update

1.30 PRB560359 (i18n): sys_translated entries were not captured in Update Sets on insert, though they are on update

sys_translated entries are now captured on both insert and update.

1.31 PRB560301 (Service Catalog): A catalog reference variable that referenced another sc_cat_item record improperly caused the referenced item's price to be included in the total cost

1.32 PRB560312 (Time Cards): Time Worked information could update the wrong day in time cards (introduced Winter 2011)

1.33 PRB560298 (Reports): Chart Colors might not be properly applied for reports using aggregation

1.34 PRB560219 (System): An instance could be made unavailable if an admin set glide.ui.homepage.parallelism to 0 for some reason

Protection is now in place against this very thing.

1.35 PRB560286 (Reports): Unauthenticated users could not see a published report if the report was designated for a specific user or group

Unauthenticated users can see reports with the public role even if the reports are designated for a specific user or group.

1.36 PRB560202 (Rota): Daily-rotation rotas could fail if rotation cycle length approached 90 days

1.37 PRB560185 (Database Views): Domain Separation could cause a Database View to return unexpected results if there was more than one top level domain and the database view used a left join
