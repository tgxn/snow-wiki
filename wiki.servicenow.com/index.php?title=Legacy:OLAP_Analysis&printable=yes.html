1 Overview

OLAP (Online analytical processing) reports use cubes to allow for fast analysis of data. The Cubes that are defined for a ServiceNow instance can not be customized by the customer at the current time. They are defined and set-up when the instance is installed and modification is not possible. Below are the cubes and the dimensions that are defined in the system.

Since Spring 2010 Stable 1, the OLAP Analysis functionality has been deprecated in favor of the Pivot Table Report. Pivot Tables are two-dimensional OLAP Cubes, and are easier to configure and use.

2 Defining an OLAP Cube

An OLAP Cube is defined by the following properties:


Field Input Value
Cube The table on which the OLAP Cube will report. Only certain tables are available for OLAP cube analysis.
Dimensions The list of available fields from the table which can be reported on.
Column The dimension that will define the columns of the OLAP cube. Only one dimension can be used.
Rows Any number of dimensions will define the rows of the OLAP cube.
Show empty cells This determines how empty cells will be displayed.
Order
  • Rows and Columns displays in alphabetical or numerical order by row/column names.
  • Rank Rows arranges the rows based on the values within the fields of the row, in descending rank.
  • Rank Columns arranges the columns based on the values within the fields of the column, in descending rank.
  • Top 10/Top 20 truncates the table to only display the 10 highest values in each row/column.
Show the ALL member Toggles display of a column or row which contains all of the other values for its dimension.
Report type The type of report display.

2.2 OLAP Examples

The basic idea of the OLAP reporting is to pick one or more dimensions that form the rows of a table and one dimension that form the columns for a table and intersection (or cells) will contain the number of items that match the dimensions chosen.  From the left navigation pane, select  Reports -> OLAP Analysis and click the Incident report for Priority by Category. You will see the image below:



If you wanted to see the relationship between callers and priorities then you would replace the "Category" dimension with the "Caller" dimension and then click "Run Query" and you would see the following:



You can have more than one dimension in the Rows so if you wanted to see the relationship between Callers, categories and priorities you could add Category back as a row dimension and click "Run Query" to see the following:



You can click on the little yellow arrows to collapse and expand various areas of the table. You can click on the number in the cell to be taken to the list of the matching records. Once you have the report the way you want it, you can name the report and save it and then it will be available as an option on the OLAP Analysis page.
