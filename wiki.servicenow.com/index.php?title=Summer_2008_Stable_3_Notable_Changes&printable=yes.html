1 Discovery enhancements

  • Added "chunky ping", automatically breaking up pings of large IP address ranges into smaller chunks. This increases overall Discovery performance plus provides faster feedback.
  • SNMP probes now have a related list for OID specifications, instead of one large text field
  • Added a simple way to use SNMP System OIDs for classification
  • Added events that are thrown when a MID server goes up or down, and a sample email notifier on the MID server down event
  • Added CI Consolidation, useful when Discovery is combined with either imported or manually entered CMDB data
  • We now capture ALL serial numbers present in a device (in a new Serial Number table)
  • Added new probe report, useful for watching Discoveries in progress, analyzing their performance, or troubleshooting problems
  • Added support for OS/X servers (Mac); they are now classified as servers, not workstations
  • Added a MID server UI action (list context/form link) to get the latest stats for any MID server with a single click
  • Enhancement to Export all capability to improve subsequent import success
  • Network adapters and IP addresses are now discovered for UPSs

2 Administration links added to Reports page

"Administration" and "Scheduled Reports" links have been added to the top of the View/Run Reports page for admins who want an easy way to get to those lists. Administration takes you to the sys_report list where reports are defined, and Scheduled Reports takes you to the list of reports scheduled to be emailed. Users with the report_admin role will see these links. If this role does not exist in your instance, you can create it.

3 Default Boilerplate on Common Administrative Forms

If you create a new Script Include, UI Page, or UI Macro, the system will pre-populate the script or Jelly field with appropriate boilerplate for your item.

For example, a script include would default to: 

gs.include("PrototypeServer");

var NewClass = Class.create();

NewClass.prototype = {
  initialize : function() {
  }
}

Whereas a UI Macro would default to: 

<?xml version="1.0" encoding="utf-8" ?>
<j:jelly trim="false" xmlns:j="jelly:core" xmlns:g="glide" xmlns:j2="null" xmlns:g2="null">

</j:jelly>

4 .form and .list navigation in nav filter

Using the Navigator filter to search for the name of a table followed by .list or .form will take you to the list or form for that table. This is the same capability that already existed if a user typed tablename_list.do or tablename.do into the browser's address bar.

e.g., incident.list will take you to the incident list, incident.form will take you to a blank incident form.

5 "since" operator changed to "since baseline"

The "since" operator has been changed to "since baseline", since that is what it is.

6 Label height consistency

The label heights of some field types have been adjusted to make them consistent across the board and across browsers. (e.g., in IE, the field label height for a choice list field was 1 pixel less than the same label for an input field.)

7 Catalog Category Loops

You are no longer allowed to created a Service Catalog category loop with parentage.

8 Domain Multiple Updates

For multiple update the domain field will now be empty rather than defaulting to "global"

9 "Reply to" override for Email Client

You can now override the Reply to value for the email client. Previously you could only override the From value.

10 Delegate notifications

A distinction is now made for delegate notifications between Email Notifications and Meeting Invitations. Another checkbox has been added to the Delegate form for Meeting Invitations.

11 Perspective information

If a user has a perspective other than "All", we provide a visual indication in the form of a subtle "Show all applications" link just beneath the navigator controls.

12 Catalog and Wizard client scripts converted to UI Policies

Catalog and Wizard client scripts have been converted to UI Policies instead.

13 Link URL moved to bottom of notification

Due to the fact that handheld devices show the complete "Link" URL for email notifications, it has been placed at the bottom all notifications to ensure legibility both in email clients and handhelds.

14 Parent exposed on related list UI Actions

The parent record is now exposed on related list UI Actions. For example, if you wanted to disable Affected CIs New and Edit buttons for closed changes you could copy the global m2m UI Actions to the task_ci table and add a condition of parent.active.
