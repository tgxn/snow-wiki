1 Overview

The IBM Tivoli® Netcool®/OMNIbus integration allows ServiceNow to send and receive event data to a Netcool/OMNIbus ObjectServer. By default, the integration transforms Netcool events into incidents, but you have the option to transform the events to another task record type.

1.1 Inbound Workflow

When you set up the integration, you choose how it receives incoming Netcool events.

  • You can configure a socket monitor to listen for events from the Netcool Socket Writer Gateway and automatically send them to the ECC queue.
  • You can configure a SOAP insert web service to automatically send events directly to the Netcool Alerts Status web service.

If you use a socket monitor to listen for inbound events, the business rule NetcoolEvent Sensor monitors the ECC queue for Netcool events and adds them to the Netcool Alerts Status import set.

If you use a SOAP insert web service, the integration adds events directly to the Netcool Alerts Status import set.

Since the Netcool Alerts Status import set is a synchronous import set, the integration uses the Netcool Alerts Status transformation map to immediately convert the import set. By default, the transformation map adds inbound events to the incident table.

1.2 Outbound Workflow

The integration always sends outgoing events to the Netcool/OMNIbus ObjectServer through the ECC queue, which forwards them to a JDBC probe running on a MID Server.

1.3 Parts of the Application

The Netcool integration application consists of these sections and modules:

  • Setup - This module allows you to configure how the MID Server sends outbound messages using JDBC connections, as well as manage how the socket monitor forwards inbound messages if you choose to use it.
  • Inbound - These modules allow you to configure the Netcool Alerts Status web service, the transform map, and the socket monitor.
  • Outbound - These modules allow you to configure the JDBC connections, the connection tester, the updater script, and monitor JDBC probes during updates.

2 Integration Requirements

Enabling an integration between your ServiceNow instance and a Netcool/OMNIbus ObjectServer requires these actions:

  • Identify a local system with network access to your Netcool/OMNIbus ObjectServer. Install a MID Server on this local system.
  • Provide port level access for the JDBC probe to communicate with the Netcool database.

2.1 Socket Monitor Requirements

Managing inbound events with the socket monitor requires these additional actions.

  • Install and configure a Netcool Socket Writer Gateway on the Netcool/OMNIbus ObjectServer.
  • Provide the MID Server with port level access to the Socket Writer Gateway communications port. The Socket Monitor listens for inbound events on the Socket Writer Gateway communication port.
  • Provide the MID Server with command line execute rights if you want the MID server to automatically start the Netcool Socket Writer Gateway.

2.2 Direct Web Services Requirements

Managing inbound events with a direct web service request requires these additional actions:

  • Create a custom web service client on the Netcool/OMNIbus ObjectServer if you want to add events directly to the Alerts Status web service.

3 Integration Setup

These steps enable the transfer of events between a Netcool/OMNIbus ObjectServer and a ServiceNow instance.

  1. Activate the Integration – IBM Netcool/OMNIbus plugin.
  2. Install a MID Server on a local system that can access the NetCool/Omnibus ObjectServer. Install the MID Server on a host machine that has direct JDBC connectivity (usually port 4100) to your Netcool/OMNIbus ObjectServer.
  3. Create a Netcool Integration Setup record.
  4. (Optional) Add custom fields to the integration.

4 Netcool Integration Setup Record

A Netcool Integration Setup record allows you to:

  • Specify the MID Server that manages the integration
  • See the current status of the integration MID Server
  • Enable or disable error notifications
  • Create one or more JDBC connections to send events to the Netcool/OMNIbus ObjectServer
  • (Optional) Create one or more socket monitors to receive events from the Netcool/OMNIbus ObjectServer

4.1 Creating a Netcool Integration Setup Record

An integration setup record defines the connection and mapping properties neccessary to transfer events between an ObjectServer and a ServiceNow instance.

  1. Click Integration - Netcool/OMNIbus > Setup.
  2. Click the New button.
  3. Select the MID Server that manages the integration. You can only specify a MID server that you have previously installed and registered with the instance.
  4. Right-click the Netcool Integration Setup banner and click Save.
  5. Add one or more JDBC connections. You can add as many JDBC connection as necessary, but the integration only maintains one active connection at a time.
  6. If the ObjectServer uses a SOAP web service to send incoming events, go to step 8.
  7. If the ObjectServer uses a Netcool Socket Writer gateway to send incoming events, create a socket monitor record.
  8. Click Save.

4.2 Viewing MID Server Status

The integration requires an active MID Server connection to transfer events.

  1. Click Integration - Netcool/OMNIbus > Setup.
  2. Click on the creation date link of an integration setup record.

    The Status field lists the current state of the integration MID Server. The status is either Up or Down.

4.3 Enabling Notifications

The integration can send email notifications to a specific user when there are connection or socket monitor errors.

  1. Click Integration - Netcool/OMNIbus > Setup.
  2. Click on the creation date link of an integration setup record.
  3. Click the Notify on error check box.
  4. In the Notify user field, select a user to receive notifications about the MID server.

The integration can send an e-mail notification when these conditions occur.

  • When a JDBC connection changes to an Error state
  • When there are no more JDBC connections available
  • When a Socket Monitor changes to an Error state
  • When a Socket Monitor stops

To edit the integration notifications, go to System Policy > Notifications.

5 Socket Monitor Record

A socket monitor record allows you to:

  • Specify the port on which it listens for incoming events from the Netcool Socket Writer Gateway
  • (Optional) Automatically start the Socket Writer Gateway
    • Specify the path to the Socket Writer Gateway
    • Specify the ObjectServer fields mapped to the integration
    • Specify the properties of the connection between the ObjectServer and ServiceNow
  • Start the socket monitor

5.1 Creating a Socket Monitor Record

A socket monitor record allows the integration to listen for events from the Socket Writer Gateway on the ObjectServer.

  1. Click Integration - Netcool/OMNIbus > Socket Monitors.
  2. Click the New button.
  3. Specify the communications port the monitor listens for incoming events from the Socket Writer Gateway.
  4. Click Submit.

5.2 Automatically Starting the Socket Writer Gateway

The socket monitor can automatically start the Socket Writer Gateway whenever the integration is active.

  1. Click Integration - Netcool/OMNIbus > Socket Monitors.
  2. Click on the Port link on a socket monitor record.
  3. Select the Start socket writer checkbox. The form displays the Socket writer gateway, Map, and Properties fields.
  4. Type or select the path to the Socket writer gateway executable. Click the light bulb icon to select to correct path for your operating system and installation.
  5. Update the Map field to match the fields you want to include in the integration. Add any custom fields to the map definition file.
  6. Update the Properties field to match the connection properties you want to use for the Socket Writer Gateway.

5.3 Starting the Socket Monitor

The integration requires an active socket monitor in order to listen for events from the Socket Writer Gateway.

  1. Click Integration - Netcool/OMNIbus > Socket Monitors.
  2. Click on the Port link on a socket monitor record.
  3. Click the Start Related Link.

6 Netcool Alerts Status Record

A Netcool alerts status record allows you to:

  • View the name of the integration's SOAP web service
  • View the name of the integration's import set table
  • View the URL to the integration's web service WSDL
  • (Optional) Customize the integration's list of web service fields
  • Link to the integration's transform map

7 Logging Integration Events

The integration tracks integration events in these ways:

  • The integration maintains a historical list of all incoming events
  • The integration logs socket monitor activities

7.1 Viewing the History of all Incoming Events

The integration maintains a history of all incoming events.

  1. Click Integration - Netcool/OMNIbus > Netcool Events.
  2. Click on the created date link to view details of the incoming event.
  3. Click the XML button from the Payload field to view the event record as an XML file.

7.2 Logging Socket Monitor Activity

By default, the integration uses the Socket Monitor Script MID Server script include to log all socket monitor activity in the the imp_notification table. In a baseline instance, the Notification web service then transforms these events into incident records.
