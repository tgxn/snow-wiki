2 Tables

The tables used by GRC are divided up between two plugins: Governance, Risk and Compliance (GRC) and Governance, Risk, and Compliance Core.

2.1 Governance, Risk and Compliance (GRC)

Name Description
Activity [grc_activity] Stores the task-based activities used in GRC audits.
Audit [grc_audit] Stores all audit instances.
Audit Control Test Definition [m2m_audit_control_test_def] Stores the relationship between an audit and a control test definition.
Audit Control Test Instances [m2m_audit_control_test] Stores the relationship between an audit and a control test instance.
Audit Definition [grc_audit_definition] Stores all audit definitions.
Audit Definition Control Test Definition [m2m_audit_def_control_test_def] Stores the relationship between an audit definition and a control test definition.
Audit Definition Entity [m2m_audit_definition_entity] Stores the relationship between an audit definition and an entity (scope).
Audit Definition Requirement [m2m_audit_def_requirement] Stores the relationship between an audit definition and a requirement and a control test definition.
Audit Entity [m2m_audit_entity] Stores the relationship between an audit and an entity (scope).
Authority Document [grc_authoritative_source] Stores the primary authority documents.
Citation [grc_authoritative_src_content] Stores the citations that are the individual components of authority documents.
Citation Mapping [m2m_auth_src_con_auth_src_con] Stores the mapping between two citation entries and their relationship.
Citation Relationship types [auth_content_rel_type] Stores the definitions for the available relationships between different parts of an authority document.
Condition [grc_condition] Stores the conditions used by advanced control tests.
Condition Collection [grc_condition_collection] Stores condition collection for advanced control tests.
Condition Collection Condition [m2m_condition_coll_condition] Stores the relationship between a condition collection and a condition.
Control [grc_control] Stores the definitions of how a policy is enforced.
Control Authority Document [m2m_control_authoritative_source] Stores the relationship between an authority document and a control.
Control Citation [m2m_control_auth_src_content] Stores the relationship between a citation and a control.
Control Test [grc_control_test] Stores the instances of a control being used in practice.
Control Test Definition [grc_control_test_definition] Stores the definitions of how controls are verified.
Control Test Sample Data [grc_control_test_sample_data] Stores the sample data generated by control tests
GRC UCF Authority Document [grc_ucf_authority_document] Stores the latest authority documents downloaded from the UCF site. The data in this staging table is overwritten each time UCF authority documents are downloaded.
GRC UCF Citation [grc_ucf_citation] Stores the latest citation downloaded from the UCF site. The data in this staging table is overwritten each time UCF authority documents are downloaded.
GRC UCF Control [grc_ucf_control] Stores the latest controls downloaded from the UCF site. The data in this staging table is overwritten each time UCF authority documents are downloaded.
GRC UCF Difference [grc_ucf_difference] Stores the differences between the existing GRC entities and the available UCF updates. The data in this table is rebuilt each time UCF authority documents are downloaded or updated in GRC.
GRC UCF filter [grc_ucf_filter] Stores all the information GRC uses to manage document counts in the document filter. These are the values displayed for each group in the filter screen. The data in this table is rebuilt each time UCF authority documents are downloaded or updated in GRC.
GRC Update Status [grc_ucf_update_status] Stores all the information used to manage the update of UCF Authority Documents into GRC. These values are used to keep track of approval states of requested updates.
Observation [grc_observation] Stores tasks which record information uncovered during the audit process.
Observation Affected Items [grc_observation_affected_item] Stores affected items related to observations.
Policy Authority Document [m2m_policy_authoritative_source] Stores the relationship between an authority document and a policy.
Policy Citation [m2m_policy_auth_src_content] Stores the relationship between a citation and a policy.
Policy Control [m2m_control_policy] Stores the relationship between a control and a policy.
Requirements [grc_audit_requirement] Stores the relationship between an audit and a requirement.
Risk Authority Document [m2m_risk_authoritative_source] Stores the relationship between an authority document and a risk.
Risk Control [m2m_risk_control] Stores the relationship between a control and a risk.
UCF Download Status [grc_ucf_download_status Stores the results of the latest UCF download, such as the file version downloaded and the duration. The data in this table is overwritten each time UCF authority documents are downloaded.

2.2 Governance, Risk, and Compliance Core

Name Description
Policy [grc_policy] Stores the definitions of standard procedures or controls.
Remediation [grc_remediation] Stores the tasks by which problems revealed by control tests can be addressed.
Risk [grc_risk] Stores the definitions of specific harm or consequences that can occur if policies are violated.
Risk Approach Rules [grc_risk_approach_rules] Stores the definitions of risk responses.
Risk Criteria [grc_risk_criteria] Stores the criteria by which risks are assessed.
Standard [grc_standard] Stores standards, which are policies specific to a company. This table extends the Policy [grc_policy] table.
Standard Operating Procedure [grc_standard_operating_procedure] Stores policies that are standard operating procedure (SOP) for a company's line of business. This table extends the Policy [grc_policy] table.
Policy Entity [m2m_policy_entity] Stores the relationship between a policy and an entity (scope).
Risk Policy [m2m_risk_policy] Stores the relationship between a policy and a risk.
Scope [grc_entity] Stores the named scope that can be applied to other GRC objects to create a name-space separation.

2.3 Fields for Report Processing

The fields listed here are added to their respective tables for GRC reporting, starting with the Fuji release. These fields are not used for any other processing and do not appear on forms by default.

2.3.1 Authoritative Source [grc_authoritative_source]

Field Description
Total controls (total_controls) Total number of controls for each authority document.
Controls covered (controls_covered) Number of controls that are covered by a control test definition.
Coverage (coverage) Percentage of controls that are covered by at least one control test definition.
Passing tests (pass) Number of control test instances that were successful for the last run.
Failing tests (fail) Number of control test instances that failed for the last run.
Total tests (total_tests) Total number of control test instances for each authority document.
Non compliance (percent_fail) Percentage of failing controls.
Compliance (percent_pass) Percentage of passing controls.
Complete (percent_complete) Percentage of passing and failing controls.

2.3.2 Control [grc_control]

Field Description
Passing tests (pass) Number of control test instances that were successful for the last run.
Failing tests (fail) Number of control test instances that failed for the last run.
Total tests (total_tests) Total number of control test instances for each control.
Non compliance (percent_fail) Percentage of failing controls.
Compliance (percent_pass) Percentage of passing controls.
Complete (percent_complete) Percentage of passing and failing controls.

2.3.3 Policy [grc_policy]

Field Description
Passing tests (pass) Number of control test instances that were successful for the last run.
Failing tests (fail) Number of control test instances that failed for the last run.
Total tests (total_tests) Total number of control test instances for each policy linked with a control.
Non compliance (percent_fail) Percentage of failing controls.
Compliance (percent_pass) Percentage of passing controls.
Complete (percent_complete) Percentage of passing and failing controls.

2.3.4 Risk [grc_risk]

Field Description
Passing tests (pass) Number of control test instances that were successful for the last run.
Failing tests (fail) Number of control test instances that failed for the last run.
Total tests (total_tests) Total number of control test instances for each risk linked with a control.
Non compliance (percent_fail) Percentage of failing controls.
Compliance (percent_pass) Percentage of passing controls.
Complete (percent_complete) Percentage of passing and failing controls.

2.3.5 Control test definition [grc_control_test_definition]

Field Description
Latest test result (test_status) State of the latest run of the control test instance for each control test definition.

3 Plugins

Governance, Risk, and Compliance activates the following plugins, if they are not already active.

Plugin Name Plugin ID Description
Governance, Risk, and Compliance Core com.snc.governance_core Provides the core risk and policy functionality used by GRC. This plugin is available starting with the Fuji release.
Lists V2 com.glide.list_v2 Allows users to search, sort, filter, and edit data in lists. Lists also can be embedded in forms and can be hierarchical (have sublists).
Certification Core com.snc.certification_core Provides all core compliance functionality, including certification templates used by GRC to create control test definitions. This plugin is available starting with the Eureka release.
Assessments com.snc.assessment_core Loads the ServiceNow assessment engine, which powers GRC attestations. Attestations created in GRC use assessment tables and can be customized using the Assessment application. This plugin is available starting with the Eureka release.
Planned Task com.snc.planned_task Provides additional fields for tasks pertaining to time and effort as part of a planned, multi-stage process. This plugin is available for versions prior to the Eureka release.

8 User Roles

These roles provide access to GRC starting with the Dublin release. Each role can perform all the activities of the roles it contains. If you are using an earlier release, see the previous version information at the end of this section.

8.1 Functional Roles

For versions starting with the Fuji release, GRC provides four functional roles that describe general compliance responsibilities in the system. This is a best practice approach to granting auditors, approvers, and users who manage control tests the technical roles they need to perform their jobs. These roles can be modified to suit an organization's needs.



Role Description Contains Roles
GRC Control Test Processor Responsible for ensuring that a control test is executed, run, and managed properly. grc_compliance_reader

grc_control_test_owner

GRC Executive Approver Responsible for approving any changes to authority documents, citations, and controls imported from UCF authority documents. Users with this role have access to their list of UCF document requests in the My Approvals module. grc_compliance_approver
GRC External Auditor Responsible for reviewing the control tests and the observations generated from them. These users have read-only access to all records involved in an audit and are external to the organization. grc_audit_reviewer
GRC Internal Auditor Responsible for reviewing and auditing control test results, and managing observations for those results. These users have the following access:
  • Read-only access to authority documents, citations, controls, risks, and policies.
  • Read-only access to audit tables.
