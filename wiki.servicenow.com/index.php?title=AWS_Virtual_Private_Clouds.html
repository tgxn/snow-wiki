1 Virtual Private Clouds

A Virtual Private Cloud (VPC) is a logically isolated section of the Amazon Web Services (AWS) cloud in which you can launch AWS resources such as EC2 instances within a virtual network. Each VPC is dedicated to an AWS account and region. Each VPC also specifies a set of IP addresses to be used for a subnet associated with the VPC. VPCs enable you to build a virtual network in the AWS cloud. VPNs, hardware, or physical datacenters are not required. You can define your own network space and control how your network, and the Amazon EC2 resources inside your network, are exposed to the Internet.

You can select the VPC and subnet your EC2 belongs to when provisioning your instance. For more information, see requesting an instance.

1.1 Viewing VPC Details

To view the details of a VPC:

  1. Navigate to Amazon Web Services > VPCs > VPCs.
  2. Click on a VPC ID.

1.2 Discovering Subnets

The list of subnets is automatically discovered after you complete the configuration tasks for setting up an EC2 account. To see the list of available subnets for each VPC, navigate to Amazon EC2 > Configuration > Subnets. You can also click on Update Subnets under the related links for each EC2 account.

1.3 Viewing Subnet Details

To view the details of a subnet:

  1. Navigate to Amazon Web Services > VPCs > Subnets.
  2. Click on a subnet.

2 Security Groups for VPCs

A security group acts as a virtual firewall for your instance to control inbound and outbound traffic. When you launch an EC2 instance in a VPC, up to five security groups can be assigned to the instance. Security groups act at the instance level, not the subnet level. Therefore, each instance in a subnet in your VPC could be assigned to a different set of security groups. Your instance is automatically assigned to the default security group for the VPC.

Each security group has a set of rules that control the inbound traffic to instances, and a separate set of rules that control the outbound traffic.

Note: Amazon EC2-Classic is not supported.


2.1 Discovering VPC Security Groups

To view a list of the VPC security groups, you must first provision an EC2 instance. Once the instance has been provisioned, click Update VPC in the Related Links, then navigate to Amazon Web Services > VPCs > VPC Security Groups. All of the security groups that are discovered are based on the region set in the EC2 account.

Field Description
Group name Name of the VPC security group.
Group ID ID number of the VPC security group.
VPC VPC of the security group.
Description Short description of the security group.

2.1.1 Viewing Security Group Details

Navigate to Amazon Web Services > VPCs > VPC Security Groups and click on a record.

Field Description
Name Name of the security group.
Group ID ID of the group.
Group Name Name of the group.
Account EC2 account name.
Region settings Region of the EC2 account.
Description Description of the security group.

2.2 Discovering VPC Security Group Rules

Rules are associated to individual security groups. A rule applies either to inbound traffic or outbound traffic. To view the VPC security group rules for all of the security groups within the VPC, you must first provision an EC2 instance. Once the instance has been provisioned, click Update VPC in the Related Links, then navigate to Amazon Web Services > VPCs > VPC Security Group Rules.

Field Description
Direction The direction of traffic the rule controls, either inbound or outbound.
Type The type of rule.
Protocol The networking protocol used by the rule.
Port Range The range of ports covered by the rule.
Target The instance source (outbound) or destination (inbound) for the rule.

2.2.1 Viewing VPC Security Traffic Types

To view a list of the traffic types allowed by the security group, navigate to Amazon Web Services > VPCs > Traffic Types.

2.3 Setting EC2 Provisioning Rules

You can set up provisioning rules to follow when provisioning EC2 instances. If the rule conditions are met, then the EC2 instance will automatically be provisioned into a VPC and VPC security group.

  1. Navigate to Amazon Web Services > VPCs > Provisioning Rules.
  2. Click New.
  3. Use the Provision request filter to set conditions that must be met in order for the EC2 instance to be assigned to a specific VPC and VPC security group.

3 Viewing ICMP Translated Messages

Internet Control Message Protocol (ICMP) messages are used to send error messages and perform TCP/IP troubleshooting. To view your ICMP translated messages, navigate to Amazon Web Services > Translation > ICMP Messages.

3.1 Managing ICMP Message Translations

In addition to being able to view ICMP message translations, you can make message translation extensible starting with the Fuji release. This allows you to either change the default ICMP message translations or create new message translations for any internet protocol. In order to create a new message translation, you must create a script include that contains your translation logic and extends from the base IPProtocolTranslator class.

  1. Navigate to Amazon Web Services >Translation > Translator Mapping.
  2. Click New.
  3. In Protocol number enter the number of the protocol to which you want to provide a mapping script.
    You can find the list of protocol numbers and names by navigating to Amazon Web Services > Translation > Protocols.
  4. In Translator class name enter the name of the translator script.
