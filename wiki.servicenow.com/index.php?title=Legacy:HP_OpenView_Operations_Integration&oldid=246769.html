1 Overview

The out-of-box integration with HP Openview Operations currently consists of uni-directional creation and updates of incidents on ServiceNow. This integration can be customized through the use of maps and business rules.

This integration installs an application module under the System Imports application called OVO Notification.

  • Maps - configure maps that are used to process incoming OVO notifications
  • ECC Queue - the ECC queue records that are created vy OVO integration Perl script
  • Business Rules - all business rules that are related to this integration

1.1 System Requirements

  • HP Openview Operations 8.X
  • Perl 5.8
    • SOAP::Lite (prerequisites http://soaplite.com/prereqs.html)
    • XML::Writer
    • XML::Writer::String
    • Crypt::SSLeay
    • IO::Socket::SSL
  • HTTPS POST access from OVO to ServiceNow instance

1.2 Installing SOAP Packages

To install Perl packages, you can specify the following command line 

perl -MCPAN -e 'install SOAP::Lite'

The command above will install the SOAP::Lite package from CPAN. This is an interactive installation process, when prompted for input, just accept the suggested or default response.

1.3 Installing ServiceNow Perl Script

The integration requires the download, configuration, and installation of a ServiceNow provided Perl script into the OVO scripts directory.

1.3.1 Download or Create Perl script

You can download a copy of the required ServiceNow OVO integration Perl script here: snc_notify.pl or a copy of the script may be copied and pasted from the following: 

#!/usr/bin/perl -w

#use SOAP::Lite ( +trace => all, maptype => {} );
use SOAP::Lite;
use XML::Writer;
use XML::Writer::String;


#
# Get parameters passed by OVO notification
#
$OVMSG{id}              = $ARGV[0];
$OVMSG{node_name}       = $ARGV[1];
$OVMSG{node_type}       = $ARGV[2];
$OVMSG{date_created}    = $ARGV[3];
$OVMSG{time_created}    = $ARGV[4];
$OVMSG{date_received}   = $ARGV[5];
$OVMSG{time_received}   = $ARGV[6];
$OVMSG{application}     = $ARGV[7];
$OVMSG{msg_group}       = $ARGV[8];
$OVMSG{object}          = $ARGV[9];
$OVMSG{severity}        = $ARGV[10];
$OVMSG{operator_list}   = $ARGV[11];
$OVMSG{msg_text}        = $ARGV[12];
$OVMSG{instruction}     = $ARGV[13];

sub SOAP::Transport::HTTP::Client::get_basic_credentials {
   return 'itil' => 'itil';
}

#
# Change the following proxy URL to match the URL to your ServiceNow
# instance
#
my $soap = SOAP::Lite
    -> proxy('https://www.service-now.com/demo/ecc_queue.do?SOAP');

my $method = SOAP::Data->name('insert')
    ->attr({xmlns => 'http://www.service-now.com/'});

# get all incidents with category Network
my @params = ( SOAP::Data->name(agent => 'OVO_Notification') );
push(@params, SOAP::Data->name(queue => 'input') );
push(@params, SOAP::Data->name(name => 'HP Openview OVO Notification') );
push(@params, SOAP::Data->name(source => $OVMSG{id}) );

my $s = XML::Writer::String->new();
my $writer = new XML::Writer(OUTPUT => $s);

#$writer->xmlDecl();
$writer->startTag('notification');

write_element('id');
write_element('node_name');
write_element('node_type');
write_element('date_created');
write_element('time_created');
write_element('date_received');
write_element('time_received');
write_element('application');
write_element('msg_group');
write_element('object');
write_element('severity');
write_element('operator_list');
write_element('msg_text');
write_element('instruction');

$writer->endTag('notification');

$writer->end;

sub write_element {
 my $label = shift;
 my $value = $OVMSG{$label};

 $writer->startTag($label);

 if($value) {
        $writer->characters($value);
 }

 $writer->endTag($label);
}


push(@params, SOAP::Data->name(payload => $s->value()) );

print $soap->call($method => @params)->result;
==== Modify Perl Script for your ServiceNow Instance ====

note After downloading the script, you must modify the line that specify the proxy URL 

#
# Change the following proxy URL to match the URL to your ServiceNow
# instance
#
my $soap = SOAP::Lite
    -> proxy('https://www.service-now.com/demo/ecc_queue.do?SOAP');

To match with your ServiceNow instance's URL for example 

#
# Change the following proxy URL to match the URL to your ServiceNow
# instance
#
my $soap = SOAP::Lite
    -> proxy('https://www.service-now.com/my_instance_name/ecc_queue.do?SOAP');

If your company uses a proxy firewall for restricting HTTP access, you can specify your proxy settings on the same line as well (http://cookbook.soaplite.com/#specifying%20proxy). For example: 

 my $soap = SOAP::Lite->proxy('https://www.service-now.com/demo/ecc_queue.do?SOAP', 
                              proxy => ['http' => 'http://my.proxy.server/']);

 # -- OR --

 my $soap = SOAP::Lite->proxy('https://www.service-now.com/demo/ecc_queue.do?SOAP', 
                              proxy => 'http://my.proxy.server/');

 # -- OR --

 my $soap = SOAP::Lite->proxy('https://www.service-now.com/demo/ecc_queue.do?SOAP');
 $soap->transport->proxy('http' => 'http://my.proxy.server/');

For configuring authenticating proxy servers, please see this link http://cookbook.soaplite.com/#accessing%20service%20with%20proxy%20authentication

1.3.2 Install Perl Script

After you have downloaded and modified the script, you can upload or place it in the OVO default directory for scripts and programs calling external interfaces 

/opt/OV/bin/OpC/extern_intf

note: If you place your scripts and programs in this directory, they will be erased when you de-install OVO.

You can refer to Chapter 7 About Notifications Services and Trouble Ticket Systems of the HP Openview Operations Administrators Guide for more information about installing scripts and configuring OVO to invoke them.

OVO sends its own message parameters to the external interface. You may not use a command that requires additional parameters. The ServiceNow Perl script (snc_notify.pl) will accept these parameters exactly the way they are, and in the right order. The following is a table of the default parameters and their acceptable values.


Parameter Description and Example
1 Unique message number.

Example: c1c79228-ae12-71d6-1a8f-0f887ebe0000

2 Message node name.

Example: hpbbxyz3.bbn.hp.com

3 Node type. For a list of supported managed nodes, see the Add Node window in the OVO administrator GUI or the OVO Installation Guide for the Management Server.

Example: HP 9000 PA-RISC

4 Date (mm/dd/yyyy) on which the message was received on the managed node in the time zone (system-specific TZ variable) of the management server.

Example: 08/02/2002

5 Time (hh:mm:ss) at which the message was received on the managed node. This time uses a 24-hour clock in the time zone (system-specific TZ variable) of the management server.

Example: 16:22:04

6 Date (mm/dd/yyyy) on which the message was received on the management server in the time zone (system-specific TZ variable) of the management server.

Example: 08/02/2002

7 Time (hh:mm:ss) at which the message was received on the management server. This time uses a 24-hour clock in the time zone (system-specific TZ variable) of the management server.

Example: 16:22:05

8 Application name.

Example: /bin/su(1) Switch User

9 Message group.

Example: Security

10 Object name.

Example: root

11 Message severity (unknown, normal, warning, minor, major or critical).

Example: normal

12 List of responsible OVO operators. Names are separated with one space.

Example: opc_op Bill John

13 Message text. Text is not enclosed in quotation marks ("").

Example: Succeeded switch user to root by charlie

14 Instructions (empty string if not available). The instructions are passed without quotation marks (""), backslashes (\), or other characters that might be interpreted by a UNIX shell.

Example: This is the instruction text for the appropriate message condition. It is available for the operator when a message matching this condition displays in the Message Browser.

15 Custom message attributes (empty string if not available). Multiple name=value pairs are separated with two semi-colons (;;).

Example: Customer=Hewlett-Packard;;Country=United States of America

1.4 Mapping OVO fields to ServiceNow Incidents

The result of the integration (the Perl script being invoked by OVO) creates an ECC input queue record in ServiceNow with an XML payload for the OVO notification. This queue record will be picked up by a queue processor that will utilize a set of maps to create a record. In this case the map will create or update an incident record.

An example of an input queue record's XML payload looks as follows:

The out-of-box map record looks as follows:

  1. Coalesce - this value specifies what value in the input XML should match with what field value in an incident in order to determine that it is the same incident and therefore should be updated instead of creating a new incident.
  2. Condition script - an XPath (http://www.w3.org/TR/xpath) description that resolves the input XML payload in order to execute the indicated map. For this case, since there is only one map, it will always run due to the specification of the Agent field.
  3. Field Maps - this is where you specify an XPath resolution (External name) into the XML payload for the value to be mapped to a field value (Mapped field) in the resulting incident record.
    1. In the case of a referenc field eg. cmdb_ci, if the value does not already exist, it will be created, and then referenced.
    2. In the case when additional mapping logic needs to be specified, eg. //severity maps to priority, javascript can be specified on a per-mapped field basis to fine tune the mapped value.

In the javascript, use the global variable answer for the returning result, and value as the variable containing the externally resolved value as it is entering the script.
