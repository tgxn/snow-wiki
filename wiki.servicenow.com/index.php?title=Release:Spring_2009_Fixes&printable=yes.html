Release:Spring 2009 Fixes

From ServiceNow Wiki
Home > Release Notes > Legacy Versions > Spring 2009 Fixes
Jump to: navigation, search
  • The following issues were fixed after the Spring 2009 release in the Spring 2009 release patch, available on an as-needed basis since June 8, 2009:
    • PRB43595 -- Behavior change: Mandatory derived fields were enforced when not desired. Fix reverts the change.
    • PRB43593 -- Empty display values caused the tree picker to render empty
    • PRB43594 -- Scheduled reports (chart type) were not emailed if the Custom Charts plugin was not installed
    • PRB43588 -- Meter and Thermometer gauges lost their color
  • Fixes for the Spring 2009 release:
    • PRB43467 -- CMDB -- Relationship editor always returned a list of CIs even when the relationship was with users/groups
    • PRB43417 -- Core -- Replication did not support basic auth, so requiring it would prevent replication from working
    • PRB43527 -- Homepage -- Refreshing a gauge sometimes added an extra edit icon
    • PRB43525 -- Homepage -- (Introduced Winter 2009 Stable 3) Homepage performance caching sometimes led to inconsistent data in My Groups Summary Count gauge
    • PRB42766 -- I18n -- Suggestion text fields did not correctly filter on language
    • PRB43520 -- Import Sets -- Adding / modifying a web service import set did not produce update set XML
    • PRB43461 -- Import Sets -- Import set fields could be created too short if the sample data set was not representative, resulting in truncation of imported data
    • PRB43462 -- LDAP -- LDAP Group imports should be disabled by default
    • PRB43489 -- Reports -- Report field on scheduled reports form incorrectly displayed Calendar reports, even though they could not be successfully emailed
    • PRB43251 -- Service Catalog -- A multiple-choice variable might show up on the Requested Item with a selected value if one was not set during checkout
    • PRB42740 -- Service Catalog -- Mandatory fields on opening page of an Order Guide were not enforced
    • PRB43372 -- Service Catalog -- Hitting "back" after checkout in catalog might show incorrect cart items
    • PRB43156 -- UI -- Read only URL fields could not be hidden with UI Policy
    • PRB43466 -- UI -- Multiple record update form did not display properly for multi-section forms
    • PRB43457 -- UI -- Workflow field type in a related list viewed using tabs UI generated a "nonsecure items" warning
    • PRB43460 -- UI -- In Firefox, tabbing from the Hours input of the date/time picker did not correctly advance to the Minutes input
    • PRB43459 -- UI -- Empty user_image field type displayed an inoperable "Click to add..." link in the list view
    • PRB42754 -- UI -- UI Policy on Wizard Variable form was incorrect
    • PRB43553 -- UI -- (Introduced Winter 2009 Stable 3) Text searched against root table (task, cmdb_ci) returned zero matches
    • PRB43464 -- Web Services -- Asynchornous SOAPClient via ECC queue did not support basic auth
  • Fixes for the Winter 2009 Stable 4 build:
    • PRB43319 -- CMDB -- BSM map did not properly display icon when a CI had an active Problem
    • PRB43246 -- Database -- Some instances did not have the right index on sys_ui_macro
    • PRB43350 -- Database -- Clearing the Parent field on a sys_audit_relation record (e.g., when deleting a task related to another task) sometimes took excessively long
    • PRB43323 -- Homepages -- IE JavaScript error when rendering home pages where a <script> tag was the first thing in the section
    • PRB43382 -- Integrations -- Web service import sets were not returning error and status message
    • PRB43316 -- Knowledge -- Knowledge base Short Description was not a translated field OOB
    • PRB43340 -- Knowledge -- Feedback might not be properly submitted if it contained a linebreak
    • PRB43335 -- List Editing -- In some cases with list editing, a NULL value might be improperly stored as a "NULL" string
    • PRB43370 -- List Editing -- Fix issue with list editing a glide_list field
    • PRB43309 -- Notifications -- (Winter 2009 Stable 3) SMS notifications were not properly sent
    • PRB43079 -- Notifications -- Automatic SMS address construction did not have sufficient flexibility
    • PRB43199 -- Reports -- (Winter 2009 Stable 2) Some bar charts were improperly displayed
    • PRB43250 -- Reports -- Could not scroll through months on published Calendar report
    • PRB43383 -- Service Catalog -- Current rate loading logic might not return all rates if not all currencies were updated, resulting in a UI error
    • PRB43385 -- Service Catalog -- (Winter 2009 Stable 3) "Set Filter" catalog client script did not work properly after enhancement to allow multiple list collectors
    • PRB43262 -- Service Catalog -- Could not remove a condition from an Order Guide rule base
    • PRB43089 -- UI -- composite_name elements incorrectly rendered both UI modes when hidden/displayed with UI Policy
    • PRB43407 -- UI -- Refreshing the World Clocks gadget in a homepage would incorrectly display the Edit icon
    • PRB43399 -- UI -- JavaScript error when rendering a read-only price field
    • PRB43405 -- UI -- Error when a list contained a column which was not in the table
    • PRB41870 -- UI -- (Chrome/Safari) refreshing browser after login returned you to login prompt
    • PRB43279 -- UI -- (Winter 2009 Stable 1) Date/time picker did not default to today's date when current value was invalid
    • PRB43292 -- UI -- Global Text Search might return duplicate results if an "after insert" business rule improperly updated the current record
    • PRB43150 -- Workflow -- Workflow Timer activity did not correctly pause activities with an excessively long name
  • Fixes for the Winter 2009 Stable 3 build:
    • PRB42379 -- Catalog -- Multiple list collector variables on the same catalog item did not work properly.
    • PRB43118 -- Core -- Exporting a form containing a multi-line text field to a PDF could result in HTML tags appearing in the output in place of line breaks.
    • PRB43159 -- Core -- After session timeout, the language picker was still visible, allowing modification of the language for the "guest" user.
    • PRB42680 -- Discovery -- Help the Help Desk failed to work properly if basic authorization was enabled for SOAP requests.
    • PRB43114 -- Email -- Approval emails from a blackberry, sometimes did not work.
    • PRB43198 -- Email -- Using subscription notifications, email addresses were limited to 40 characters, which was a problem if an email address exceeded 40 characters.
    • PRB42243 -- Localization -- French translations of 'save' and 'update' were both translated to the same term 'enregistrer', but they have different contextual meanings, leading to confusion.
    • PRB42485 -- Localization -- The time zone picker always showed the system default timezone in English, even if a different language was used. e.g., "System (America/Los Angeles)" instead of "System (Amerique/Los Angeles)"
    • PRB43151 -- Localization -- Using a locale with a comma as the decimal separator resulted in catalog prices being off by a factor of 10 in the UI.
    • PRB43173 -- Localization -- Email subjects containing international characters (like Greek, Chinese, etc.) were not output properly in UTF-8, and the resulting email showed ??? in their place. Email bodies were not affected.
    • PRB43220 -- Localization -- Locale formatting code did not validate the specified locale. As a result, a user with an incorrect country code (according to ISO) could not properly use the service catalog.
    • PRB43210 -- Mobile -- Clicking the link to open ServiceNow in its mobile version resulted in the mobile interface appearing inside of a frame, rather than replacing the current frameset.
    • PRB43221 -- Mobile -- When rotating an iPhone, the mobile menu font improperly changed.
    • PRB43064 -- Reporting -- Grouping a trend report by date, then drilling through into one of the columns, improperly returned no results.
    • PRB42649 -- UI -- Inconsistency between Firefox and IE with field label text wrapping. One wrapped, the other did not.
    • PRB42708 -- UI -- In IE 7, long text in the application banner improperly wrapped.
    • PRB43015 -- UI -- Field styles were not applied consistently across all views. e.g., the style might be applied to the ess view but not the default view.
    • PRB43049 -- UI -- glide_list fields (such as the watch list on the incident form) could not be set to read-only using UI policy.
    • PRB43126 -- UI -- The "drill through" icon on the watch list was improperly enabled if an email address was selected (as opposed to a user) on the watch list. Clicking it led to a blank screen.
    • PRB43133 -- UI -- If a new view was created with a mixed case name like "Incident Management" from the Personalize Form UI, the refresh of that view in the Personalize Form UI did not work.
    • PRB43140 -- UI -- "New Page" option improperly removed from homepage "Switch Page Menu".
    • PRB43169 -- UI -- As of Winter 2009 Stable 2, dependent choice list entries e.g. category -> subcategory, did not work properly.
    • PRB43196 -- UI -- Email CC line on the activity formatter was not properly omitted when empty.
  • Fixes for the Winter 2009 Stable 2 build:
    • PRB43019 -- Application -- Rendering performance of the Gantt chart inside the project management application was very poor with large numbers of tasks spanning long time durations.
    • PRB42943 -- Service Catalog -- As of Winter 2009 Stable 1, variable sets were not properly handling multiple column layouts.
    • PRB42928 -- Service Catalog -- If using the workflow engine in conjunction with the service catalog, the delivery dates on the catalog checkout screen were wrong.
    • PRB43013 -- Service Catalog -- Mousing over a subcategory on the catalog home page didn't display the subcategory's description.
    • PRB42875 -- Service Catalog -- If using a record producer with a template containing child records e.g. a change_request that also created change_tasks, duplicates of the topmost record (the change_request) would be generated.
    • PRB43001 -- Core -- Use of a tree picker on a reference field whose *target* had a display column other than "name" would result in an empty tree being rendered.
    • PRB42387 -- Core -- List controls (omit filter, omit new, etc) are not being honored on grouped lists.
    • PRB43096 -- Core -- If a form used an annotation, attempting to export that form to PDF resulted in an NPE. NPE has been removed, but the annotation still will not render in the PDF (it shows up blank).
    • PRB42816 -- Core -- If using a QBE type search form, and fill some data into a multi-line text field, the system was doing an equals query, while it should have been doing a STARTSWITH query.
    • PRB42940 -- Core -- As of Winter 2009 Stable 1, a process guide with multiple approval steps could produce duplicate approvers for any step after the first step.
    • PRB42967 -- Core -- A query for sys_user for roles != admin showed duplicate entries in the resulting list
    • PRB43085 -- Email -- Content type in outbound notifications was not being honored; the system always sent multi-part mixed emails containing both HTML and plaintext parts, even if you specifically requested that only text/plain be sent out.
    • PRB43023 -- Email -- If using the subscription based notification plugin, weighting rules for emails were not being applied. As a result, multiple emails resulting from the same event could be sent to the same recipient, regardless of their relative weights.
    • PRB43010 -- Email -- Certain patterns in the subject line of an email (the leading prefix of an incident/problem/change number), followed by a space would cause an inbound email to fail processing if the user also had a "blank" entry in their sys_number table
    • PRB42944 -- Import Sets -- Null pointer exception (and a failed transformation) if you tried to transform to a target field that didn’t exist.
    • PRB42358 -- Import Sets -- When transforming a date/time field, the "copy empty fields" flag in the transform map was not being honored.
    • PRB43058 -- Import Sets -- If you used the mapping assist feature to create a transform map, the resulting map entries were not properly recorded in an update set and hence wouldn't promote when you promoted the update set.
    • PRB42990 -- Import Sets -- As of Winter 2009 Stable 1, canceling an import set wouldn't cancel it.
    • PRB42989 -- Import Sets -- As of Winter 2009 Stable 1, ignoring an import set row via a before import script could cause the import to loop and process rows forever, or at least until you killed it, whichever came first.
    • PRB43100 -- Knowledge Base -- On the kb_submission table (knowledge base -> open submissions), the number field reset to 10000 on each upgrade, resulting in duplicate submission numbers.
    • PRB43076 -- Reporting -- Grouping a report by a field with international characters in it (like DE München) grouped properly, but you couldn't drill down into the DE München bar (or wedge) properly from the chart.
    • PRB42901 -- Reporting -- On a report, if you grouped by a "dot walked" field e.g. incident.caller_id.location, the grouping would fail with an invalid SQL error.
    • PRB42997 -- UI -- The task activity formatter displayed the timestamps for emails in GMT, rather than the user's timezone.
    • PRB43039 -- UI -- If you placed an HTML area (the built-in HTML editor used on knowledge base articles, catalog items, etc) on a tabbed form, and then switched to and from the tab containing the html area, you received JavaScript errors on IE (and the tab didn't come back properly).
    • PRB42351 -- UI -- g_form.setReadOnly() didn't work properly for condition type fields
    • PRB42894 -- UI -- On Safari, condition type field had all three selects overlay one another.
    • Graphical Workflow fixes:
      • The "Set wait_for to workflow" business rule was changed to only run on insert
      • When publishing a workflow version, all currently published versions are unpublished via a business rule, in case there is more than one
      • Enhanced logic to determine which workflow version is used, either the version checked out to the current user or the most recently created published version (in case there is more than one published version)
      • Misspecified audit and update_synch attributes were fixed
      • Workflow versions get synched with all of their child data in a single update entry
      • Added 'Mark as Requested' to the list of convenience actions in the Approval Actions activity
      • Fixed Manual Approvals to properly handle user approvals when there are no group approvals.
      • Also added support for requesting any manual user and group approvals that are in a 'not requested' state when the Manual Approval activity is first started
      • Fixed Workflow Rollback activity to properly take into account the child activities of an Approval Coordinator
      • Fixed a misleading example in the default value for Task activity advanced scripts
      • Fixed Join activity to not create two Join activities in a race condition in which the first Join completes before the second Join has time to realize it is a duplicate
      • Prevent looping in business rules on sc_req_item
      • Adjustments to the approval business rules that run the workflows on parent tasks so that workflows are only run once
      • Fixes for workflow restart
  • Fixes for the Winter 2009 Stable 1 build:
    • PRB42697 -- Application -- Pause conditions on advanced SLAs do not work properly on an instance with a non-standard date/time format
    • PRB42693 -- Application -- SLA percentage timers do not work properly in Winter release
    • PRB42712 -- Application -- If you install the project management plugin, the project management home page becomes everybody's default homepage if they don't already have a preference
    • PRB42720 -- Application -- An approval task which was set to auto close on approval would not auto-close on some circumstances until it was manually updated at least once.
    • PRB42819 -- Service Catalog -- Two step order guides insisted on presenting all three steps
    • PRB42824 -- Service Catalog -- Clone functionality on checkout screen wasn't cloning price properly
    • PRB42863 -- Service Catalog -- Show quantity option on order guides would only save the last quantity you chose (on the last tab prior to placing your order)
    • PRB42714 -- Domain -- Personalizing the knowledge list as a domain user (not a global user) could, under some circumstances, result in extra columns being displayed on the list
    • PRB42687 -- Email -- Outlook calendar integration does not function properly (the invites go to outlook, but are malformed)
    • PRB42689 -- Email -- Email text sometimes is not stripped of HTML tags
    • PRB42860 -- Email -- A malformed watermark could cause the email reader to stop processing email until manually restarted
    • PRB42660 -- Email -- Inbound emails being ignored if they were forwarded through the outbound email address
    • PRB42861 -- Internationalization -- After installation of a language plugin, the set of categories inside the knowledge base was incorrect
    • PRB42685 -- Imports -- Import set cleanup deletes the history log for all import sets, not just the one you are cleaning up
    • PRB42794 -- Imports -- Scheduled Import jobs with a type of "periodically" didn't display the time field properly
    • PRB42852 -- Imports -- Import set cleanup could take an extremely long time due to a large number of cache flushes involved.
    • PRB42655 -- Imports -- After Winter release, import of a file looks like it is still running when it is not
    • PRB42691 -- Mobile -- Using a reference picker, then hitting a VCR navigation button on the reference list, then picking an entry off the list, did not return to the proper form.
    • PRB42723 -- Mobile -- A user who had chosen a navigator perspective and then logged into mobile was shown an empty mobile menu
    • PRB42652 -- Mobile -- Mobile browser properties added to out-of-box properties
    • PRB42664 -- Reports -- Reporting issues introduced with the Winter release
    • PRB42758 -- SOAP -- WSDL for getElement API call not properly exposing sys_id field
    • PRB42811 -- SOAP -- WSDL generation off of the ecc_queue in the form ecc_queue.do?payload=incident&WSDL was missing name attribute in WSDL output
    • PRB42804 -- System -- Text indexing can fail if a reference field points to an invalid table
    • PRB42848 -- System -- Text indexing was re-indexing rows whenever they changed, even if the change didn't include and relevant fields
    • PRB42667 -- System -- Cannot update journal fields through script if the GlideRecord is against the base table, e.g., cannot update an incident's journal fields via a GlideRecord against task
    • PRB42668 -- System -- text_index events may not get processed if instance has a custom date format
    • PRB42716 -- System -- Relationship auditor does not properly audit CI changes (auditing of CI's unaffected, error only affected auditing of relationships between Cis)
    • PRB42416 -- System -- Delete Table button might not delete the table under some circumstances
    • PRB42725 -- UI -- Client scripts which hide or reveal entire sections did not interact properly with tab code, resulting in rendering artifacts
    • PRB42654 -- UI -- Behavior change with multiple form sections
    • PRB42591 -- UI -- Special characters not properly displayed in Welcome Page Content HTML text area
    • PRB42617 -- UI -- Show matching and filter out do not work correctly on defined related lists
    • PRB42676 -- UI -- Legacy glide.ui.tabsall property breaks forms after Winter release (patch removes the property)
    • PRB42631 -- UI -- Mandatory fields warning dialog had extra whitespace before and after label names in Firefox
    • PRB42633 -- UI -- A grouped list gauge still showed the navigator (vcr buttons) even if the list was empty
    • PRB42817 -- UI -- Toggle groups and filters icon sometimes did not work IE
    • PRB42863 -- UI -- Time fields did not honor g_form.setReadOnly()
    • PRB42679 -- UI -- Reference pop-up view broken for multiple-section forms
    • PRB42662 -- UI -- Form resolution logic errors (wrong view being displayed under some circumstances)
    • PRB42624 -- UI -- Doing a query with a large IN operator e.g. assignment_groupINjavascript:getMyGroups() rendered improperly in the breadcrumbs
    • PRB42665 -- UI -- Grouping by a dot walked reference field, e.g., incident.caller_id.company might fail if the length of the dot walked field name exceeded 30 characters
