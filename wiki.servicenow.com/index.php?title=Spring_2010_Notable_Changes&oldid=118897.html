1 Changes for Spring 2010

1.1 attachment.read event

A new event, attachment.read, is fired when an attachment is read. Parm1 has the name of the attachment, Instance holds the sys_id of the attachment, Parm2 has the table of the record to which the attachment belongs.

1.2 Ctrl+M changed to Ctrl+. (period)

The Ctrl+M command to maximize the main frame has been changed to Ctrl+. (period) to work better across all browsers.

1.3 sys_properties.description translated

System Properties (sys_properties) Description field is now a translated_text field.

1.4 Blank passwords not allowed

Users with blank passwords are not allowed to login by default. This requirement can be customized by creating and setting the "glide.login.allow_blank_password" property to true. This property does not exist by default.

1.5 XML field type filter operators

The XML field type has been given a more useful set of operators in a filter (e.g., "contains").

1.6 no_audit for cmdb_ci.start_date

The cmdb_ci.start_date field has been given the no_audit field attribute to prevent changes to it from being audited. This can be a high-traffic field, and auditing it was of little value.

1.7 Priority scheduled jobs always queued

"Priority" scheduled jobs are now always queued, even if the schedule queue is near capacity. Non-priority scheduled jobs will not be queued if the schedule queue is more than 70% full. They will be picked up in the next query (in 10 seconds).

1.8 GoTo change for number matches

If an exact number match is typed into the Goto input for the Number field on a task table (incident, problem, etc.), navigation will be to the form for that record. This behavior is controlled by the glide.ui.task.number.match property (default true). If this property is changed to false, an "equals" query will be issued on the list instead, returning exactly one record in the resultant list.

1.9 Drag and drop attachments and HTML field images for Firefox 3.6

For Firefox 3.6 and up, attachments may be easily added to a form by dragging them from the desktop to the form. Images may be added to HTML fields in the same manner. Other browsers will be added as they begin to implement this portion of the HTML specification.

1.10 "Load demo data" when activating a plugin

When activating a plugin, a new "Load demo data" checkbox is available if the plugin has demo data available.

1.11 JavaScript editor is a user preference

The JavaScript editor was a system-wide property. It is now a user preference that may be set individually per user. An icon on a script field's label can be toggled to turn on and off the JavaScript editor.

1.12 glide.ui.report_expand_header honored in Report portal

The glide.ui.report_expand_header property is intended to prevent a user from seeing the report header. This worked fine coming from a list, but didn't through the Report portal. Now it does.

1.13 Single Currency Mode

There is a new option to run an instance in "single currency mode". If this mode is used, the currency to use must be selected (like USD or GBP). If running in single currency mode: 1) Users will always see prices/currencies in the system currency 2) When a currency/price is saved, a currency cannot be specified - it will use the system currency

Properties controlling this behavior have been added to System Localization.

1.14 glide.pop3readerjob.create_caller Defaults to False

The glide.pop3readerjob.create_caller property determines whether an inbound email from an unknown address will create a new user record for that email. Previously, the default of this property was true, and now the default is false.

1.15 HR and Facilities Record Producer

Both Human Resources and Facilites Management Plugins have had changes made to their out-of-box record producer to simplify the form for end users.

1.16 glide.smtp.auth Defaults to False

Connections to SMTP servers no longer require a user name and password for authentication. This property should only be set to true when an instance must connect to an external SMTP server.

1.17 Forms Scroll to Field-Level Info Messages

When a field-level information message is displayed on a form on load, the form will now scroll down so that the info message is visible.


The following are notable changes you may encounter with our Winter 2010 Stable 3 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before.


2 Winter 2010 Stable 3

2.1 Dragging support for iPad and iPhone

Homepages now recognize touch devices, so homepage gauges can be moved around on devices like the iPad and iPhone.

2.2 Notification device activation

With Subscription Based Notifications, when the Notification field on a user record is changed to "Email", the user's primary email device is set to Active.

2.3 CMS site copying

Create a copy of a CMS site or page by clicking the "Copy" UI Action when looking at a CMS Site or Page record.

2.4 Empty values converted to NULL

Empty string values are automatically converted to NULL. This was the case before, but there were still ways an empty value could be stored. This change fixes that.

2.5 update_synch attribute not allowed

The "update_synch" dictionary attribute may no longer be added to Dictionary record.

2.6 Unpublished articles not shown by default

For new instances, the "Show unpublished articles" knowledge property defaults to False so behavior does not conflict with Global Text Search results.

2.7 SFTP data source support

Secure FTP is now supported as an Import Set data source file retrieval method.

2.8 Delegates can be specified with date/time

Service Delegation can now be specified using date/times instead of just dates.

2.9 syslog table cleaner

For new instances, syslog records are purged after 90 days.

2.10 New catalog item options

"No quantity" and "No cart" fields have been added to catalog items. "No quantity" hides the quantity widget. "No cart" removes the Add to Cart button and the shopping cart when viewing an item.

2.11 List Control supports domain processing

List Control records can now be specified per domain.

2.12 Faster field creation

Adding a new field to a table no longer has to flush the instance cache, resulting in much faster field creation.

2.13 Choice List option creation

When personalizing choices for a choice list field, the UI to add a new choice will no longer appear if Create ACLs don't let the user create sys_choice records.

2.14 Choice List option hints

A value in the "Hint" field for a sys_choice record will appear as a tooltip when selecting a choice and hovering over that option.

2.15 Knowledge audited

The kb_knowledge table is now automatically audited for new instances. Existing instances can simply check the "audit" checkbox on the Dictionary record for that table.

2.16 New web_service_admin role

A new web_service_admin role has been created. The System Web Services application is protected with this role, as are tables related to Web Services. Previously, the admin role was required.

2.17 sys_m2m fields are read-only

After a Many-to-many relationship has been created using a sys_m2m record, the fields are made read-only to prevent changes. The following are notable changes you may encounter with our Winter 2010 Stable 2 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before.


3 Winter 2010 Stable 2

3.1 Client script GlideRecord query limit

Client scripts can now query only 500 records, defined in the com.glide.max_query_size property, unless a higher limit is set with setLimit(int).

3.2 Variable help text Close link removed

Removed the [x]Close link from the bottom of variables' Help expand/collapse text blocks. It was redundant, and people understand toggles.

3.3 sys_ref_list view of form cannot be overridden

There were multiple ways the sys_ref_list view of a form could be overridden. Those ways have been removed. When the sys_ref_list view is required, it is required.

3.4 Email client icon ID

An ID of "email_client_open" has been placed on the email client icon anchor element at the top of a form, so it can be easily used in client script if desired.

3.5 glide.email.smtp.max_send property

The new glide.email.smtp.max_send property specifies how many emails can be sent in a single SMTP connection. Default value is 100. Multiple connections will be made each time the SMTP Sender scheduled job runs to accommodate all outgoing emails.

3.6 sys_audit_role table audits role granting/removal

With Contextual Security, the new sys_audit_role table audits the granting and removal of user roles. The System Security --> Role Audit module has been added to view the contents of this table.

3.7 "automatic renumber" business rule removed

The "automatic renumber" business rule on task has been removed for new instances. It has not been removed from existing instances.

3.8 Email client To: block mandatory

The To: block in the email client is now mandatory, since emails without a To: recipient are not supported.

3.9 no_filter attribute for list collector variable

Adding the "no_filter" attribute to a list collector variable will hide the filter. Until now, a catalog client script had to be written to hide it.

3.10 "contains" query for glide_list

glide_list fields now support a "contains" filter condition.

3.11 Updating multiple properties is faster

Updating multiple properties on a properties page now flushes the cache just once at the end, instead of once for each property. This makes it faster to change multiple properties.

3.12 SMTP username sets mail.smtp.auth to true

Providing an SMTP username in Email Properties now automatically sets mail.smtp.auth property to true, causing authentication to be used for SMTP.

3.13 Filter operators for "number" fields

"Number" fields now have the same filter operators as "decimal" fields.

3.14 Keyboard shortcuts for navigation filter

When using the navigation filter atop the list of applications and modules, Up and Down arrow keys can now be used to select the desired module, and the Enter key can be used to launch the module.

3.15 Subscription Based Notifications support domain processing

Subscription Based Notifications now support domain processing, meaning different domains can set up their own notifications and override global notifications.

3.16 Multiple Form Splits property now out-of-box

The glide.ui.form_multiple_splits property for enabling Multiple Form Splits now exists out-of-box. Previously, an admin would have to create it in the instance in order to use it.

3.17 New report_admin role

The new report_admin role can create reports for any group, edit reports from the sys_report list, and includes report_publisher, report_scheduler, report_group, and report_global roles.

3.18 New sys_user_role.grantable field

A Grantable field has been added to the Role (sys_user_role) table. If true, the role can be delegated. If false, the role cannot be delegated.

3.19 Roles added to System Definition and System UI

To allow new roles (e.g., business_rule_admin, ui_page_admin, etc.) to see what they should, these roles have been given access to the necessary applications. Where appropriate, modules within these applications have also had their roles modified to either 1) allow access to these new roles, or 2) restrict access to admin. Instances that have customized these modules/applications will not see any change.

3.20 New roles for import sets

Four new roles have been created to control access to imports:

  • import_set_loader - can load data into an import set table from an external source
  • import_transformer - can manage transform maps and run a transform on an import set table
  • import_scheduler - can schedule an import
  • import_set_admin - includes all three of these roles and can do everything associated with imports

The following are notable changes you may encounter with our Winter 2010 Stable 1 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before.

4 Winter 2010 Stable 1

4.1 "Guest" user cannot submit SOAP requests

The Guest user is disallowed from being used to submit SOAP requests, because it is a system user that has no password.

4.2 Empty date/time field filter operations

"is empty" and "is not empty" filter operations were added for glide_date and glide_date_time fields.

4.3 Field attributes affecting slushbucket behavior

Introduced two new attributes for controlling how List Layout and Form Layout slushbuckets handle elements:

  • list_layout_ignore : do not show this element in the List Layout slushbucket
  • slushbucket_ref_no_expand : do not allow this element to be expanded (like a reference element) in a Form or List slushbucket

4.4 Manage Relationships BSM Map Action change

The Manage Relationships BSM Map Action no longer appears for users who do not have 1) write access to the CI and 2) edit_ci_relations access to the CI, as defined in Contextual Security

4.5 Dictionary protection against field type change

The "Internal type" choice list on the Dictionary form will not allow a field's internal type to be changed to a type that doesn't have the same underlying database column type. e.g., a glide_date field cannot be changed to glide_date_time field.

4.6 Domain reference picker

A new domain picker UI Macro named domain_reference_picker has been added to Domain Separation, allowing the domain picker to function like a reference field, useful for instances with a large number of domains. The UI Macro is separate from the domain_select UI Macro, and either may be used.

4.7 Suggestion icon no longer clickable in a reference popup

Since multiple levels of popups are not supported, the Suggestion "lightbulb" icon is no longer clickable in a reference popup. e.g., When clicking the magnifying glass for a reference field, and then clicking New to create a new record, the lightbulb icon will be disabled.

4.8 HTML field toolbar cleanup and reorganization

The toolbar for HTML fields has been given a facelift with new button images. The buttons have been reorganized, with unnecessary controls removed (cut, copy, paste).

4.9 Clickthrough property support for reference variables

glide.ui.clickthrough.popup and glide.ui.clickthrough.replace properties are now honored for service catalog reference variables. They have always been honored for standard reference fields.

4.10 New spellcheck dictionaries

New spellcheck dictionaries have been added.

4.11 New {URI_REF} token available in Email Notifications

The {URI} token has always added a link to an email notification with the URL of the current record, and display text of "LINK". The new {URI_REF} token will also add a link to the email notification with the URL of the current record, but the link text is the display value of the record (e.g., "INC11201").

4.12 Changes to pickers in the welcome stripe

The various pickers in the welcome stripe (Time zone picker, Domain picker, Language picker, etc.) have been standardized and given better tooltips.

4.13 Catalog and Knowledge search limit properties

Two new properties were added to control the number of records returned from Catalog and Knowledge searches. The default number of search results returned for each is 30:

  • glide.knowman.search.rowcount: Number of Knowledge search matches returned.
  • glide.sc.search.rowcount: Number of Service Catalog search matches returned.

4.14 CSS property to control width of reference variables

Since field styles do not apply to variables, a new property css.questionset.reference.width can be created in an instance to specify the width of all reference variables in an instance. e.g., 240px

4.15 Service Catalog performance enhancement

Optimizations were made to the loading sequence of the Service Catalog, dramatically reducing load time for customers with a large number of items in their Catalog portal.

4.16 Global domain not affected by non-global scripts

Users in the Global domain are no longer affected by Business Rules, Client Scripts, UI Policy, and UI Actions in non-global domains.

4.17 Domain picker inactive by default

The domain_select and domain_reference_picker UI Macros are inactive by default. If this release deactivates an active picker, simply activate the appropriate UI Macro.

4.18 SOAP field element names lowercased

Incoming SOAP field element names are converted to lowercase during insert/update to be consistent with query interfaces and WSDL.

4.19 Role picker width dynamically adjusts to display contents

The user_roles role picker widget on forms will dynamically adjust to display its contents.

4.20 Go to functionality for mobile lists

Provide "Go to" functionality for mobile lists.

4.21 glide_list honors ref_auto_completer attributes

glide_list fields now honor ref_auto_completer, ref_ac_order_by, and ref_ac_columns attributes, for consistency with reference fields.

4.22 BSM Map Save button removed

The non-functional Save button was removed from BSM maps.

4.23 Graphical Workflow change

The current stage is marked as 'active' even if there are no activities associated with that stage. A cancelled request is treated the same as rejected - that is, the stages are marked as rejected instead of complete.


5 Changes in Discovery for Winter 2010 Stable I

5.1 Sensor errors during Discovery

Sensor errors for an empty payload now show up as an issue in the device history list in the Details link.

5.2 New logging messages

The wording in Discovery logging messages has been improved.

5.3 New MID Server configuration parameters

The following MID Server configuration parameters were added:

  • mid.ssh.use_keyboard_interactive: Forces the keyboard interactive SSH authentication mode
  • mid.ssh.suppress_history: Suppresses the generation of the SSH history file.

5.4 New information in the discovery_device_history table

Added the Issues, Completed activity, and Current activity fields to the Discovery Devices Related List in the Discovery Status form. The Issues field contains a link for Details of the error.

5.5 UI Actions displayed for specific CI groups

The UI Actions Show discovery events and Test CI consolidation are now displayed only for the following major CI groups:

  • Computer
  • Printer
  • UPS
  • Netgear
