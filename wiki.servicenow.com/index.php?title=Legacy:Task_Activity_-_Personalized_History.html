1 Overview

For all tasks (incidents, changes, problems, etc.), there is a formatter called "Task activity" that will display selected activity or history for the task directly on the form. You can add this to a form by configuring the form. Formatters appear at the bottom of the list of fields. 

 Note: For Task Activity to work as expected, it requires that the table auditing (history) for a table is activated.

5 Personalize Activities

The fields that are displayed can be customized. Right-click within the activity header, and you will see an option to "Personalize Activities" as shown below. You will be taken to an Available list to choose the fields you would like included.


5.1 Sent and Received Emails

Emails will not be displayed in the Activity formatter unless specifically requested. Sent/Received Emails appears as an option at the bottom of the Available side of the list. Use caution with this, however, because there is no way to determine whether a given user SHOULD see a particular email. For example, you might have an email with work notes, sent to the assigned_to individual. If you include emails in the Activity formatter on the ESS view of the incident form, ESS users will see the emails with the work notes.

(Available from the Winter 2009 Stable 3 build and Spring 2009 release) The glide.ui.activity.email_roles property specifies who can see emails when included in the Activity formatter. The property description is, "List of roles (comma separated) that can view emails in the Activity formatter when 'Sent/Received Emails' are included. If blank, all can view."


6 Activity Formatters for other tables

Any table that is audited can have an activity formatter. The formatter needs to be defined under System UI --> Formatters (use Task Activity as a model and just change the Table - do NOT change the Formatter value of incident_activity.xml). Once defined it will show up when configuring the form. You can then use Personalize Activity to choose the fields of interest and as long as the table is audited, it will work.

7 Task Activity Security

Although you can personalise which fields show up in Task Activity section, a user will only see entries for fields that they have read access to.

Example: A Self service user might see the task activity on their self service view of the incident form but they will not see work notes (unless out of the box security rules have been changed) or any other fields they don't have read access to.

8 Troubleshooting

Task Activity requires that auditing for the table is activated.
