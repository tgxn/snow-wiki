Release:October2011 Preview 2 Fixes

From ServiceNow Wiki
October2011 Preview 2 Fixes
Jump to: navigation, search

The following is a list of issues fixed in the October2011 Preview 2 release. To sort by descending or ascending order, click the sorting icon in the column heading.

Number Category Details Steps to Reproduce
PRB561454 Attachments The "'Attachments'" related list did not get refreshed automatically after adding an attachment on the form
  1. Ensure List v2 is activated.
  2. Navigate to Change Request.
  3. Open or create a change record.
  4. Ensure you have the Attachments related list added on the change request form layout.
  5. Add any attachment.
  6. Check the Attachments related list. It does not include the attachment you just added.
    Note that reloading the form displays the attachment correctly.
PRB561562 Calendar In the date and time picker, the first day of the month could render incorrectly if the first day of the week was changed.
PRB560128 CMS Some CMS IFrames were not fully visible in Chrome and Safari.
  1. Navigate to a demo instance.
  2. Access the instance via the ESS portal (/ess/main.do)
  3. Click on Something's Broken.
    Result in Firefox: The form looks fine.
    Result in Chrome and Safari: The form will be cut off around the Impact field.
PRB561450 Database Views A database view involving an extension table pulled sys_choice options for all tables extending the same base table.
  1. Create a database view joining the incident[u_incident] table to the task[task] table on incident.parent = task.sys_id.
  2. Ensure there are choice list entries on this incident for which the value is not being used by any other table extending task as well as entries for which the value is being used.
  3. Show a list of records with the incident state field visible.
PRB559817 Domain Separation Domain separation was not applied to global text search results if there was no other query restriction.
  1. Note incidents on two different domains.
  2. Impersonate a user on one of the domains who can view global text search.
  3. Search for an incident on a different domain.
    Note that the incident shows up in the list of search results but only displays a blank frame when clicked on.
PRB561612 Field Service Management A new Service Order record's default state might have been improperly be set to (-5).
  1. Create a new service order.
    The default state is -5.
    The state should be open.
PRB562032 Filters Daylight Saving Time not taken into account with Show After and Show Before options on context menus for lists.
  1. Open a list.
  2. Ensure the Updated field is displayed in the list view.
  3. Right click on an entry in the Updated field.
  4. Select Show After or Show Before.
    Note that the breadcrumb at the top shows an hour earlier and the filter itself is empty.
PRB561720 I18N Field were not properly translated with a language plugin activated.
  1. Enable I18N: Internationalization plugin.
  2. Change the display language.
  3. Mouse over a field hint.
    Note that the hint is displayed in English instead of the selected language.
PRB561558 Importing JDBC data load via MID server could drop data.
PRB222917 Importing Binary characters could be present and display wrong in imported records.
  1. Create a new MySQL table and populate varchar columns with binary chars, for example:
    mysql> insert into mytable values( 1, 'one', char(31,31,31,10,11,12,13,14,15,31,31,31));
    Query OK, 1 row affected (0.00 sec)
    mysql> insert into mytable values( 2, 'two', char(32,32,32,0,1,2,3,4,5,32,32,32));
    Query OK, 1 row affected (0.00 sec)
  2. Set up a JDBC data source to load the data.
  3. Using a script or Java debugger, observe that the binary characters have been imported into ordinary columns.
PRB562102 Importing XML documents with nesting greater than two levels did not handle the "expand node children" feature correctly.
PRB561354 Knowledge Base An error "html value empty" could be generated when submitting feedback to a knowledge base article.
  1. Hide the comments on knowledge base articles. (Knowledge Base > Properties and set the value of Show user comments on knowledge articles to Never.)
  2. Navigate to Knowledge Base > View.
  3. Open an article.
  4. Submit feedback.
    Note the "html value empty" error that occurs.
PRB561835 LDAP LDAP authentication could fail if more than one sys_user OU was defined.
PRB560833 Lists List options would appear from incorrect task types when editing an extended list.
  1. Install Release v2.
  2. Add enhancement as an extension of features as well as state.
  3. Enter task.list in the navigation filter.
    Note that the choices for a change request show all choices from all tables that extend from task.
PRB561997 Lists Adding choices to a dictionary form threw a JavaScript error on update when an embedded list had no records and "omit if empty" was set to true in List Control.
  1. Navigate to a dictionary form.
  2. Personalize the layout.
  3. Add the red Choices field to the form.
  4. Click update.
    Note the onSubmit error. This error remains even the choices are removed.
PRB561698 Lists List v2 dropped support in a grouped list to display the total number of records in the grouped list
  1. Navigate to Reports > View / Run.
  2. Select All Incidents by Assignment
  3. Change the Type to List.
  4. Run the report.
PRB560510 Navigation Certain parameters from the uri= parameter were trimmed by nav_to.do, potentially affecting SSO redirect scripts.
  1. Open a closed incident record.
  2. Copy the URL of the inner frame.
  3. Paste the URL into a new window.
  4. Insert nav_to.do?uri= before incident_list.do
  5. Press enter.
  6. Compare the results against the original incident record.
PRB562478 Notifications If the List field field of an Email Notification had a space between names, the specified list of users would not receive the notification.
  1. Open a notification record.
  2. Add a space between names in the List field field.
  3. Send out the notification.
    Note that the listed users would not receive the notification.
PRB561383 On-Call Rotation On-call notifications were not being sent.
  1. Create a new Rota for an existing group with members.
  2. Execute the following script.
    This will erroneously print "false".
var a = new OnCallRotation(); 
a.who("<sys_id of the group>") 
gs.print(a.next())
PRB561899 Platform Administration - Change Management Proposed changes could not empty a field.
  1. Create a new change request.
  2. Add affected CIs to the change request.
  3. Right click the Affected CIs related list.
  4. Select Propose Changes.
  5. Make an alteration to a field and empty another field.
    Note that the alteration takes effect but emptying a field does not.
PRB561389 Platform Administration - UI Switching to a different form view could throw a JavaScript error if a related list on the form was empty.
  1. Create or open a change form with an empty related list.
  2. Click a field.
  3. Press Tab key multiple times.
    Firebug throws an error.
PRB562347 Platform Administration - UI If the glide.ui.buttons_bottom was set to false, showQuickForm would no longer display any buttons.
PRB561914 Platform Administration - UI Navigating using form paging buttons (blue arrows in upper right) were no longer saving modified field values when the system property was set to do so.
  1. Open an incident.
  2. Make a change.
  3. Page to the next record using the down arrow.
  4. Return to the page you were first on.
    Note that the change was not saved.
PRB561218 Platform Administration - UI Values in the glide_time field were not properly saved on forms when a custom time format omitted seconds.
  1. Create a glide_time field on an incident form.
  2. Navigate to System Properties > System
  3. Change the Time Format setting to HH:mm.
  4. Clear your cache, log out and log back in.
  5. Enter a value in the glide_time field created in step 1.
  6. Save the incident.
    Note that the value entered disappears.
PRB562430 Platform Administration - UI Fields in the user_image table might improperly show escaped HTML in the sys_ref_list view with the glide.ui.escape_text property set to true.
  1. Verify that the propery glide.ui.escape_text is true. High security is the default on newer systems.
  2. Open an incident.
  3. Click the magnifying glass next to the Assigned To field.
  4. At the top of sys_ref_list, right-click the header and choose Personalize > List Layout.
  5. Add the Photo column.
  6. Save your changes.
    Note the escaped HTML displaying in the Photo column.
  7. Set glide.ui.escape_text to false and repeat these steps.
    The photo is displayed instead.
PRB562031 Platform Administration - Zing Text Search Enabling "Did You Mean" support for global text search could generate an error.
PRB560151 Project Management Project tasks imported from Microsoft Project could improperly be created as inactive.
  1. Activate Project Management v2 plugin.
  2. Navigate to Project Portfolio Management > Administration > Import Project.
  3. Import a project with more than one sub task.
  4. Navigate to the project just created and start the project.
  5. Close a sub task.
    Note that the project is closed even though there are project tasks that are not closed.
PRB561189 Service Catalog An error could be generated if a script tried to delete an invalid entry from the Service Catalog shopping cart.
  1. Create a business rule to use as a reference qualifier.
  2. Create a catalog item with a reference field.
  3. Set this field's reference qualifier to call a method defined in the business rule.
  4. View the item and click on the magnifying glass next to the reference field.
  5. A blank window will display and a no pointed exception will be thrown.
PRB561784 SLA Specifying an invalid Script Include name in the "SLA default Condition class" property generated a JavaScript error.
  1. Add an invalid SLA condition script name to the SLA Condition[sla_condition_class] table.
    Note that an error occurs when creating an incident.
PRB561883 SLA The SLA "has_breached" flag might not get set when it should (over 100% elapsed time).
  1. Ensure a SLA definition exists with a small duration for testing purposes.
  2. Create a record that meets the SLA conditions.
  3. Ensure a task SLA is created and a sys_trigger record is created for that task SLA.
  4. Wait for the incident to breach the SLA time.
    Note that the task SLA in question is successfully calculated with a percentage above 100% but is still listed as not having been breached.
    If you update the parent task record such that the SLA is completed, it will appropriately display as breached.
PRB560011 System A Java deadlock related to the DBConnectionPool and GlideProperties could consume UI semaphores.
PRB561311 UI11 Fixed queries in list breadcrumbs could not be properly be dragged to UI11 bookmarks.
  1. Create a fixed query for a module (&sysparm_fixed_query=active=true for example).
  2. Click the link.
  3. Drag the fixed query to the bookmark bar to create a bookmark.
  4. Open the bookmark.
PRB559964 Variables Read-only date or date/time variables could lose their values on subsequent updates.
  1. Create a data type variable on a catalog item.
  2. Create a ClientScript or a UI policy that sets this variable to ReadOnly with the g_form.
  3. Order that item through the Service Catalog.
  4. Set the time during ordering.
  5. Open the Requested Item list.
  6. Save the form.
    Note that the variable field is now blank.
PRB562564 Web Services Scripted web services that return a DOM element result generated an exception in java.io.FileOutputStream.writeBytes(Native Method) when responses were larger than 256KB.
  1. Create a scripted web service which returns a large DOM element as its response. The serialized length of the DOM element must exceed 256KB.
  2. The scripted web service must set the global variable response.soapResponseElement to the element to be returned.
  3. Turn the debug for SOAPProcessor on by setting glide.processor.debug.SOAPProcessor.
  4. Invoke the web service from SOAP UI.
PRB560626 Web Services SOAP session timeout did not work properly.
  1. Use a tool such as soapUI to send web services requests to an instance both with and without Basic Authorization headers.
  2. View the logged-in users page to see what sessions exist.
    Note that the sessions persist far longer than the time limit set in glide.soap.invalidate_session_timeout.
PRB562066 Workflow The Approval Action workflow activity might not properly set pending approvals.
PRB561950 Workflow Stages declared in a Workflow might not properly show up in the workflow icon column of Request Items.
  1. Create a workflow
  2. Add stages and use them in the workflow.
  3. Assign the workflow to a service catalog item.
  4. Order the item.
    Note that in the Requested Items menu the workflow stages are correct.
  5. Add new stages to the workflow but do not use them.
  6. Publish the workflow.
  7. Modify the workflow to use the previously unused stages.
  8. Assign the workflow to a service catalog item.
    Note that the previously unused stages are not represented in the icon list.
