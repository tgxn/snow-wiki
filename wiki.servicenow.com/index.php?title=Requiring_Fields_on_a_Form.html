Requiring Fields on a Form

Functionality described here requires the Required Form Fields plugin.
Functionality described here requires the Admin role.

1 Overview

With this plugin, an administrator can designate specific fields on a form as required (protected) fields. If present on a form, required fields can only be removed by an administrator. Required fields are defined in the sys_ui_element_required table.

The expected (although certainly not the only) use case for this feature is as part of a delegated administration scheme. One might, for example, grant branch offices the rights to modify forms (by granting the personalize_form role), but not allow them to remove certain fields which are critical to overall business processing.

2 Defining Required Fields

  1. Install the Required Form Fields plugin.
  2. Navigate to System Properties > All Properties.
  3. Locate the property named glide.ui.form.enforce_required_fields and make sure it is enabled (Value is Yes.)
  4. Go to System UI > Required Form Fields, and click New.

If the table specified has extension tables, then the Required Form Field record applies to forms of all extended tables. For example, if an administrator specifies that the Short description field is required for the Task table, then this configuration applies to the Incident form, Change Request form, Problem form, etc.

An extended table can override its base table's Required Form Field rule. For example, if the Short description' field is required for the Task table, but not required for the Incident table, it will be required for all Task tables except Incident.

3 User Interface

When a delegated administrator configures a form, required fields appear in a gray color and have a tooltip indicating they are required. Delegated administrators users can move their fields within a form but cannot remove them from the form.
