Legacy:Domain Numbering

From ServiceNow Wiki
Home > Administer > User Administration > Users and Groups > Legacy:Domain Numbering
Jump to: navigation, search
 Functionality described here is obsolete since the Aspen release and has been replaced by Domain Paths.

1 Overview

ServiceNow provides domain separation for customers who have numerous domains and want to control access to those domains or provide different policies for each domain. For some customers, such as managed service providers (MSP) whose networks contain hundreds of domains and thousands of subdomains, simple domain separation might result in decreased performance when opening and saving records. Queries made against such a deep domain structure require extra processing to parse long names and can slow down database input and output.

When performance in such networks becomes an issue, customers can use domain numbering to speed up processing. Domain numbering automatically assigns reference numbers to each domain. This numbering system is created and managed automatically, and uses easy-to-process decimals to maintain the proper parent/child domain relationships. By using numbers, the database can retrieve records with a simple range query against a numeric value.

Domain numbering requires the Domain Number Support plugin, which can be activated by a user with the admin role.

2 How the Numbering System Works

Domain numbering uses a decimal hierarchy that expresses domains, sub-domains, and sub-sub-domains in groups of 10, 100, 1000, 10000, etc. Domains are numbered from 1-9 initially. The main domain is always number 1. Sub domains of each parent are identified with the parent number plus a decimal value starting at .00 for the first level, with a limit of 100 domains. Second level children are expressed as .0000, with a limit of 1000, and so on. The children of the main domain have numbers such as 1.00, 1.01, and 1.02. Children of those sub-domains have numbers such as 1.0000 and 1.0001. When the threshold is reached at each level, the affected domains are renumbered accordingly. This numbering system can grow to accommodate networks with tens of thousands of sub-sub-domains. This diagram illustrates how domain numbering is used to replace domain names.


3 Enabling Domain Numbering

When you activate the Domain Number Support plugin, it automatically numbers all domains, but does not apply the numbering scheme immediately. The plugin installs a property called glide.sys.domain.numbering with a value of false, which prevents domain numbering from being enabled until you have an opportunity to inspect the numbering scheme. After determining that domain numbering is set up as expected, change the value of the property to true. To disable domain numbering at any time, switch the property back to false. To access the list of ServiceNow properties, type sys_properties.list in the navigation filter in the Application Navigator.
