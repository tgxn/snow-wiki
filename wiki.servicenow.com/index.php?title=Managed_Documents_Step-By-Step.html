1 Overview

Use the Managed Documents application to control electronic documents within your ServiceNow instance. After parameters have been set by the administrator and the Knowledge Document plugin installed, documents can move through the managed document cycle. This page explains the steps to create, review, revise, and publish a managed document.

Note: Step through the process below on your test (or any non-production) instance.

2 1. Create a New Managed Document Record

The first step is to create a new managed document record.

  1. Navigate to Managed Documents > Create New.
  2. In the Name field, type Policy.
  3. In the Requested by field, add your name.
  4. In the Owner field, add your name.
  5. In the Reviewer field, add your name.
  6. In the Type field, select Policy.
  7. In the Classification field, select Confidential.
  8. In the Audience field, select Internal.
  9. Under Revision Settings, select the Auto increment revision option.
    Revision settings

  10. Click Submit.

3 2. Upload a Document

After creating a document record, add the document to the record as an attachment.

  1. Open the document record.
  2. Select the Upload/Check in Revision related link.
    Upload a document

  3. Click Choose File.
  4. Select a simple document, such as a text file, and click Open.
  5. Check that the name reads Policy_POL_0.1.
  6. Click OK. The document is listed under Document Revisions.


4 3. Check Out a Document

After a document is added to a managed document record, check the file out to make changes. Checking documents in and out keeps a record of document changes and ensures that only one person at a time is editing the document.

  1. Open the document record.
  2. Select the Check Out Document related link.
    Check out a document

  3. Select Download file.
  4. Click Check Out.
  5. Open the file and make a small change.
  6. Save the document with the same name, but a new revision number. For example, Policy_POL_02.txt.


5 4. Check in a Document

After making changes to a document, check the revised document into Managed Documents.

  1. Open the document record.
  2. Click the Upload/Check in Revision related link.
  3. Click Choose File, select the revised file, and click Open.
  4. Check that the Name and Revision Number contain the updated number.
  5. Ensure that the Check in option is selected.
  6. Click OK. The Document Revisions list updates to contain the most recent revision.
    Document Revisions


6 5. Send a Document Out For Review and Approval

When a document is ready, send it out for review and approval. Note that if a reviewer or approver is not identified in the document record, the document is moved directly to the publishing stage.

  1. In the Document Revisions list, right-click on the revised document and select Submit Revision. Because you identified yourself as the reviewer, you receive an approve request email message.
    Submit Revision

  2. In the email message, click the link next to Click here to view Approval Request.
  3. (Optional) View the document by clicking the attachment name under Document Revision.
  4. Click Approve. The Approvals page displays and lists the document you just approved.
  5. Open the document record.
  6. In the Document Revisions list, the revision you approved is now in the Ready for Publishing stage.


7 6. Publish a Document in Managed Documents

After the document is reviewed and approved, publish the document within Managed Documents.

  1. Open the document record.
  2. In the Document Revisions list, click the name of the revision that was approved and is listed in the Ready for Publishing stage.
  3. In the header bar, click Publish Revision.
    Publish

  4. In the Publish revision dialog box, check that the name and revision number are correct and click Publish. In the Document Revisions list, the revision you just published is now in the Published stage.


8 7. Publish a Document to the Knowledge Base

To make the document available to readers, select knowledge settings and publish the document to your Knowledge Base.

  1. Open the document record.
  2. In the Knowledge Settings list, set the Behavior to Always create record.
  3. Set the Topic to Policies.
  4. Select a Published date.
  5. Select a Valid to date.
  6. Type a few words into Short Description.
    Knowledge Settings

  7. Click Update.
  8. Open the document record.
  9. Click the Link to Knowledge related link. A sentence displays at the top of the document that reads Knowledge created successfully. Click here to open it.
    Click Here to Open

  10. Click the here link. The Knowledge record displays, showing the correct revision as an attachment, the automatically generated KB number, and the valid dates.
  11. (Optional) If you want to publish the document to the Knowledge Base on your instance, click Mark Public in the header bar and then click Update. View the Knowledge Base to see the published document.
    Mark Public
