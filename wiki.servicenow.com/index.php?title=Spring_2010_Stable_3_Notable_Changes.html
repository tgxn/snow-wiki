Spring 2010 Stable 3 Notable Changes

From ServiceNow Wiki
Home > Release Notes > Legacy Versions > Spring 2010 Stable 3 Notable Changes
Jump to: navigation, search

The following are notable changes you may encounter with our Spring 2010 Stable 3 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before. For additional changes, see Spring 2010 Stable 3 Fixes.

1 Changes for Spring 2010 Stable 3

1.1 Updated Client Side Libraries

The version of the Prototype javascript library used by the product has been updated to version 1.7. The product now includes with the version 1.4.2 of the JQuery client side library.

1.2 Percent Complete field type

A new field type called percent_complete displays a progress bar in lists that tracks task or project completion rates. Add display attributes to this field to compare the actual percentage completion of a task with a predicted target value.

1.3 Removed AJAX Calls

All AJAX calls (some of which were synchronous) previously run when the navpage framework was loaded have been removed. This change improves the rendering speed of the framework.

1.4 New fields added to planned_task

Two new fields, Remaining duration and Remaining effort, have been added to the planned_task table.

1.5 Property to deactivate legacy SLA engine

A new property, com.snc.sla.run_old_sla_engine, deactivates the legacy SLA engine if the new SLA plugin is installed, preventing both engines from running and conflicting. The property can be viewed in the sys_properties list.

1.6 Changes to SLA Plugin Calculations

Three changes have been made to SLA Plugin calculations:

  1. The multiple business rules that power the SLA plugin have been consolidated into a single business rule, Process SLAs, which is now asynchronous. The SLA will no longer need to be calculated fully after the user submits the form; the user will be returned to the form immediately and the calculation will occur in the background. This means that the SLA may not appear immediately after the task is inserted; the form may need to refreshed.
  2. A single calculation script now calculates all of the duration fields in the SLA, including a new field named Business Pause Duration.
  3. The scheduled job that processes the SLAs every two minutes has been broken up into 6 smaller scheduled jobs, which will calculate more frequently as the SLA approaches breached, reducing the load on the system.

1.7 Increased use of Asynchronous AJAX

Throughout the system, many synchronous AJAX calls have been converted to asynchronous AJAX calls. This improves the end user experience because the browser no longer has to wait for the AJAX call to complete before the user can continue working.

1.8 Category field added to UI Page (sys_ui_page) table

A Category field to UI Pages, such as the one that exists for UI Macros, has been added to track the functional part of the system to which a UI Page belongs. This allows for easy grouping of all Service Catalog pages, CMS pages, Knowledge pages, etc.

1.9 New operators for Discovery Classification

Two new negative operators (does not equal and does not start with) have been added to Discovery Classification criteria to make classification conditions more flexible.

1.10 ESX added as an OS choice

ESX has been added as an operating system choice for computers.

1.11 New Mobile modules default to mobile view

If a new Mobile module is created, the view name will default to mobile on the sys_app_module record. This is done with the default value field on the sys_app_module.view_name Dictionary record.

1.12 Property specifying SNMP probe use of public

A new Discovery property, mid.snmp.enable_auto_public, controls whether SNMP probes automatically use the community string public. The default value is true.

1.13 Default value of a field can be overridden

A field's default value in the Dictionary can now be overridden with Dictionary Overrides.

1.14 CMDB schema change for hardware classes

Added an intermediate hardware class to the CMDB schema for various types of hardware. For example: cmdb_ci > cmdb_ci_computer is changed to: cmdb_ci > cmdb_ci_hardware > cmdb_ci_computer. The affected classes are:

  • cmdb_ci_computer
  • cmdb_ci_printer
  • cmdb_ci_netgear
  • cmdb_ci_ups
  • cmdb_ci_pdu.

1.15 addJoinQuery(joinTable, [primaryField], [joinTableField]) method added to GlideRecord

A new method, addJoinQuery(), has been added to GlideRecord. This method makes it possible to query records via relationships in other tables. For example, find all the users that are in the database group (users via sys_user_grmember table). Or find all problems that have an assigned incident (problems via the incident.problem_id relationship).

1.16 Help the Help Desk now requires soap_ecc role

Help the Help Desk now requires a particular ACL to access the instance through a script: soap_ecc.
