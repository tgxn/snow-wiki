SNC Regex API

From ServiceNow Wiki

Jump to: navigation, search

Related Topics

1 Overview

The SNC.Regex API exposes regular expressions to JavaScript.

For versions prior to Eureka Patch 5, native JavaScript regex performance can be slow when evaluating long strings. It is recommended that you use the SNC.Regex API, which provides faster performance. If you have created scripts using native JavaScript regular expressions, you should modify your code to use the SNC.Regex API.

Starting with the Eureka Patch 5 release, an enhanced regex engine replaces the native regex engine. JavaScript regular expressions automatically use the new regex engine, which has better performance. You should not use the SNC Regex API in in Eureka Patch 5 and later releases, see Using_Regular_Expressions_in_Scripts. The SNC.Regex API is not available for scoped applications. See the Fuji release notes for more information on what is available with scoped applications.

2 Method Summary

Method Summary Description
replaceAll(String str, String replacement) Replaces within the given string anything that matches the pattern passed into the constructor with a replacement string.
match(String str) Attempts to match the given input against the regular expression.
reset() Resets the current Regex instance, making it ready for a new match.
toString() Returns the regular expression passed into the constructor as a string representation of a native JavaScript regular expression.


3 Supported Flags

The following are the flags supported by the SNC Regex API and can be included as input to the SNC.Regex constructor. For example: 

var pattern = new SNC.Regex('/[a-z]/i');
Flag Description
c Canonical Equivalence: Enables canonical equivalence while matching.
d UNIX Lines: Only considers the Unix new line character as a new line character.
i Case Insensitive: Indicates that matching should be case-insensitive.
l Literal: Indicates that meta characters (such as ^, $, {3,4}) and escape sequences (such as d, w, \) will not be considered as meta characters and escape sequences. This flag is available starting with the Fuji release.
m Multiline: Includes ^(Start of String) or $(End of String) characters in matches.
s Dot-all: Includes new line (n) and carriage return (r) characters in matches.
u Unicode Case: Enables matching on unicode strings when performing case-sensitive matches.
x Comments: Allows comments to be included that are ignored in the match.


4 SNC Regex constructor

The constructor creates a new Regex object with a regular expression as an argument. The following example instantiates a Regex object: 

var rgx = new SNC.Regex('/\[0-9]{3}-\[0-9]{2}-\[0-9]{4}/');

If no argument is supplied, then no pattern is initialized. If an argument is supplied, it is used to initialize the pattern. The supplied argument is treated as a string, which allows flags to be included as part of the string.

5 Method Detail

5.1 replaceAll(String str, String replacement)

Replaces within the given string anything that matches the pattern passed into the constructor with a replacement string.

5.1.1 Input Fields

Parameters:

  • str - (String) The string to be replaced.
  • replacement - (String) The replacement string.

5.1.2 Output Fields

Returns: (String) the new string containing the replacement strings.

5.1.3 Example

var rgx = new SNC.Regex('/\[0-9]{3}-\[0-9]{2}-\[0-9]{4}/');
var result = '234-56-7979 more text 448-33-3331 more text 555-66-3333 ';
gs.print(result);
gs.print(rgx.replaceAll(result,"***-**-***"));

Output: 

*** Script: 234-56-7979 more text 448-33-3331 more text 555-66-3333 
*** Script: ***-**-*** more text ***-**-*** more text ***-**-***


5.2 match(String str)

Attempts to match the given string against the regular expression. If the given string contains at least one match to the regular expression, this method returns an array of the results. Element 0 of the result contains the entire matched sequence. This method may be called repeatedly to find all the matches in a given string. The second and subsequent calls are not required to provide the argument with the text to match.

5.2.1 Input Fields

Parameters:

  • str - (String) The string to match.

5.2.2 Output Fields

Returns: an array of the matching results. Element 0 of the result contains the entire matched sequence. Elements 1 through n of the array contain the last match in each capture group. These elements are returned only if the regluar expression contains those capture groups. In other words, if this instance is initialized with a regular expression that contains two capture groups, and a match is found, then the returned array will contain elements 0 (for the entire matched sequence), 1 (for the first matched capture group), and 2 (for the second matched capture group). If there is no match, this method returns null.

5.2.3 Example 1

var pattern = new SNC.Regex('/[a-z]/i');

gs.print(pattern + ' '+ (pattern.match('UPPERCASEWORD') !== null));
pattern.reset();
gs.print(pattern + ' '+ (pattern.match('lowercase') !== null));

var pattern = new SNC.Regex('/(?i)[a-z]/');

gs.print(pattern +' '+ (pattern.match('UPPERCASE')!==null));
pattern.reset();
gs.print(pattern + ' '+ (pattern.match('lowercase')!== null));

Output: 

*** Script: /[a-z]/i true
*** Script: /[a-z]/i true
*** Script: /(?i)[a-z]/ true
*** Script: /(?i)[a-z]/ true


5.2.4 Example 2

var sampleString = 'abc_abc def ab3c abcd abc'; 
var reg = new SNC.Regex('/abc/i'); 
var match = reg.match(sampleString); 

var count =0;
var matches = []; 

while (match != null) { 
  matches.push(match); 
  match = reg.match(); 
  gs.log('match[‘ +count+ ‘] is ' + matches[count]); 
  count++;
} 
gs.log(‘count is ‘ + count);

Output: 

*** Script: match[0] is abc
*** Script: match[1] is abc
*** Script: match[2] is abc
*** Script: match[3] is abc
*** Script: count is 4


5.3 reset()

Resets the current SNC Regex instance, making it ready for a new match.

5.3.1 Input Fields

Parameters:

  • None

5.3.2 Output Fields

Returns: void

5.3.3 Example

var patt = new SNC.Regex("/[a-z]/i");
gs.print(patt+' '+ (patt.match('auth-1') !== null));
gs.print(patt+' '+ (patt.match('@') !== null));

var patt = new SNC.Regex("/[a-z]/i");
gs.print(patt+' '+ (patt.match('auth-1') !== null));
patt.reset();
gs.print(patt+' '+ (patt.match('@') !== null));

Output: 

*** Script: /[a-z]/i true
*** Script: /[a-z]/i true
*** Script: /[a-z]/i true
*** Script: /[a-z]/i false


5.4 toString()

Returns the regular expression passed into the constructor as a string representation of a native JavaScript regular expression.

5.4.1 Input Fields

Parameters:

  • None

5.4.2 Output Fields

Returns: (String) the native JavaScript representation of the regular expression.

5.4.3 Example

var patt = new SNC.Regex("/[a-z]/i");
gs.print(patt.toString());

Output: 

*** Script: /[a-z]/i
