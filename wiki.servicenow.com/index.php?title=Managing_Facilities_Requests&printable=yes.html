1 Overview

Facilities staff members need to regularly access request records as they resolve the request and correspond with the submitter. Staff members can also access built-in reports to see information like the number of unassigned requests or the number of active requests for each building.

2 Creating Facilities Requests

A facilities request is a record in the system that tracks a proposed change to an organization's facilities. Typical facilities requests are to report something broken, such as a malfunctioning thermostat, or report in issue like a beeping smoke alarm that needs a new battery.

Any user can submit a facilities request through the Facilities catalog, which is a type of service catalog available to all users in self-service. Users with the Facilities staff role can also create and update facilities requests using the Facilities Request form directly.

2.1 Creating Requests Through the Facilities Catalog

Users without a role can use the Facilities catalog to submit requests. The catalog provides several different categories so users can choose the one that closely relates to their request.

To use the facilities catalog:

  1. Navigate to Self-Service > Facilities Catalog.
  2. Choose from the follow categories:
    • Maintenance and Repair: submit general requests like janitorial requests and carpet cleaning.
    • Security Requests: request a security escort or report an issue with your security badge.
    • Other Inquiries: submit a question to the facilities team.
  3. Select a sub-category, if necessary.
  4. Fill in the fields (see table).
  5. Click Submit.
The facilities request catalog record producer

Field Description
Priority The priority that describes the importance of this request.
Category The type of request you are creating.
Location The location where this request pertains.
Opened for The name of the person submitting this request. Select a new name if you are opening this request on behalf of another user.
Short Description A brief summary of the request.
Detailed Description A detailed description of the request.

2.2 Creating Requests with the Floor Plan Viewer

Facilities staff members can access the floor plan viewer to create a request for a specific location on the plan.

Note: Creating incidents from the floor plan requires that the floor plan image fit within the browser window. The X, Y coordinates cannot be properly calculated if the scroll bars are visible. Press control + or control - to resize the map.


  1. Navigate to Facilities > View Floor Plans.
  2. Select the building and floor from the choice lists to display the correct floor plan.
  3. Select the Show labels check box to display all of the room markers on the floor plan.
  4. Drag the new request icon () to the closest () room marker where you want to make the facilities request. You can only locate the request on an existing room marker. If you drop the request on an empty space, the request is automatically relocated to the nearest room marker.
    The Create New Facilities Request form opens.
    The Create New Facilities Request form

  5. Fill in the fields, as appropriate (see table).
  6. Click Submit.
 Note: Room markers can have multiple requests associated with them. If you drag the request icon to a room marker that already has a request icon, another request is created for that location. The number of requests for each room marker is displayed within the request icon.

2.3 Creating Requests with the Facilities Request Form

Facilities staff members can create requests through the Facilities Request form as an alternative to using the floor plan viewer. Use this form if you want to associate the request with a configuration item, such as a printer or projector, which is affected by the request. Associating a CI to a request helps your configuration management team understand which services could be negatively impacted by a facilities issue. You can also use this form if you want to include additional comments and work notes for the request.

  1. Navigate to Facilities > Requests > Create New.
  2. Fill in the fields, as appropriate.
  3. Click Submit.

3 Facilities Request States

Facilities requests follow a specific life cycle. Additionally, progress through the request life cycle is shown visually with a process flow formatter above the facilities request record. Completed tasks appear in green.

The states that display depend on your settings on the Facilities Management Configuration screen. For example, if you select Approval is not required for new requests, the states relating to approval are not displayed in the process flow formatter, and the request moves to the next state.

Note: When a request initiator adds information about work to be done, the request begins in the Draft state.
Note: During a upgrade from Eureka to Fuji, all facilities requests in the Pending state in Eureka will be moved to the Draft state in Fuji.


4 Approving Requests

Approving a facilities request means that the request has been reviewed and is ready to be qualified for task creation and assignment. When a request is sent to a user with the facilities_approver_user role, the approver has several choices (see table).

If you select New requests will need approval in the Facilities Management Configuration screen, a newly-created request automatically moves to the Awaiting Approval state. Otherwise, the request moves to the next configured state.

Approval Choice Description
Approved The facilities request is approved.
Rejected The facilities request is not qualified and it will be moved to the cancelled state. Additionally, the following work note is added to the request:

The facilities request is rejected.

More information required The request does not contain enough information. It reverts back to the Draft state and the following work note is added to the request:

The facilities request needs more information for further approval.

Duplicate The request is no longer required, because the work was already performed by another request. The request is moved to the Cancelled state and the following work note is added to the request:

This is a duplicate facilities request.

5 Assigning Active Requests

  1. Navigate to one of the following:
    • Facilities > Open - Unassigned for a list of requests that no one is assigned to.
    • Facilities > Open for a list of all open incidents regardless of their current assignment.
  2. Open the request you want to assign.
  3. Click the reference lookup icon () beside the Assignment group field.
  4. Select the group that handles this kind of request. If no groups are available, leave this field blank. You do not have to select an assignment group, but doing so limits the number of users you can assign the request to.
  5. Click the reference lookup icon beside Assigned to.
  6. Select the staff member to handle the request.
    The users available in this field are limited to the users who are in the Assignment group, if one was selected.
  7. Click Update. An email notification is automatically sent to the assigned staff member if email notifications are set up for the instance.

6 Collaborating on Requests

  1. Navigate to Facilities > Open.
  2. Open the request you want to collaborate on.
  3. In the Additional comments (Customer visible) field, enter the comments that you want the person who submitted the request to see. The submitter can see the comments in this field and add additional comments as necessary.
    Update this field as many times as necessary to correspond with the submitter.
  4. To correspond with other facilities staff members, enter content that you do not want the submitter to see in the Work notes field.

7 Changing the Location of a Request

  1. Navigate to one of the following:
    • Facilities > Open and open the request you want to modify.
    • Facilities > View Floor Plans, click the request icon () on the map, and click the request number on the list that appears.
  2. On the Facilities Request form, click the reference lookup icon beside Room.
    A list of locations defined for your organization appears. A location could be a room or any point on a floor plan.
  3. Select the correct location. If you do not see it, contact the facilities administrator to add the location to the floor plan.

8 Closing Requests

  1. Navigate to Facilities > Assigned to me.
  2. Click the request number.
  3. In the Additional comments field, enter any final notes or comments that you want the submitter to be aware of.
  4. Change the State field to the appropriate closed state.
  5. Click Update.

8.1 Closing/Completing Requests

After a request has been assigned to an agent, that agent can complete and close the request under two conditions:

  • When the Request lifecycle is task driven option is set in Facilities Service Management Configuration, all states of the request are driven by the task. The agent can click the Close Complete button on the Task form to close any tasks that need to be manually closed. After all of the request's tasks are closed, the request is automatically closed.
  • When the Request lifecycle is request driven option and all of the request's tasks are closed, the agent can click the Close Complete button on the Request form to close and complete the request.
