1 Overview

The Calgary CA 6 Hotfix 3 release provides fixes for the Calgary release. This release is no longer recommended for upgrades. If you want to upgrade an instance, consider a current version. For more information about how to upgrade your instance, see Requesting an Upgrade.

For the Calgary CA 6 Hotfix 3 release:

Build date: 08-07-2013_1709
Build tag: glide-calgary-02-15-2013__cca6-hotfix3-08-06-2013
 Note: See how to determine the version on your instance.


2 New in This Release

Category Problem Description
Text Search PRB584653 When a Microsoft Excel file is attached to a record in a table and text search indexes the attachment, formulas on rows that reference other rows within the Excel file may cause the instance to run out of memory.

To enable this fix, create the glide property glide.ts.index.poi.use_event_extractor and set it to true. This will cause the indexer to use a document parser that uses less memory. It will also skip the metadata and some calculated cells' content during indexing.

Calgary CA 6 Hotfix 1

The Calgary CA 6 Hotfix 1 release provides fixes for the Calgary release.

Category Problem Description
Persistence PRB573877 Exception in bigger, but still reasonably sized batch on SQLserver.
Click the plus for steps to reproduce

Re-install domain numbers on fresh zboot SQLServer or run the automated test StatementBatcherIT on SQLServer.

Persistence PRB585408 Disable use of snapshot isolation on SQLServer.
Click the plus for steps to reproduce
  1. Turn on snapshot isolation on your SQLServer database
    ALTER DATABASE {database name}
    SET READ_COMMITTED_SNAPSHOT ON;
    ALTER DATABASE {database name}
    SET ALLOW_SNAPSHOT_ISOLATION ON;
  2. (Re)Start the instance.
  3. Execute following statement in sys.scripts.do:
    DBCC USEROPTIONS
  4. Confirm that last line read 'isolation level, read committed snapshot' and NOT 'isolation level, snapshot'.
Platform Miscellaneous PRB586216 glide.sm.default_mode property changed to Deny during Calgary upgrade even though High Security was not activated.
