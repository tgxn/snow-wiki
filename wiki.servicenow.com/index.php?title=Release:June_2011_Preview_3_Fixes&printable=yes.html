1 June 2011 Preview 3 Fixes

1.1 Multiple chat windows on chat desktop only

Users can now open multiple chat windows on the chat desktop only. In all other locations (such as the ESS Portal), only one window can be open at a time.

1.2 PRB560765 (On-call Rotation): On-call calendar no longer allowed a user to provide coverage for another user (introduced June 2011 Preview 2)

Providing coverage for another user once again works as designed.

1.3 PRB560070 (UI): Field styles were not applied to multi-line text fields in forms

Field styles are now applied to multi-line text fields in forms.

1.4 PRB560892 (UI): Field Style for project.end_date did not work properly with a custom date format

That field style has been modified to work properly independent of date format.

1.5 PRB560881 (Client Scripts): g_form.setValue() no longer worked for duration fields (introduced June 2011 Preview 2)

g_form.setValue() was not working as expected for duration fields and was returning the following error:

left is undefined [Break On This Error] var daysEnd = left.indexOf(' ');

1.6 PRB560869 (Release): Could not move Feature to Releases in the Release Planning Board without SDLC plugin activation

This planning board functionality has now been decoupled from the SDLC plugin, so only Release v2 is required.

1.7 PRB560872 (Variables): Variables did not render the "more info" link if a previous reference variable was read only

The "more info" link is now visible where appropriate.

1.8 PRB560873 (Client Scripts): Error when serializing a Knowledge article for the client using GlideRecord

Knowledge articles can now be properly serialized without error using client side GlideRecord.

1.9 PRB560760 (Workflow): After a workflow was defined, there was no way to associate existing or newly created Stages affiliated with the workflow's table

Stages are now correctly associated with the workflow on update in addition to on insert.

1.10 PRB560822 (Reset Password): The Reset Password plugin no longer correctly enabled the reset_password public page

The Reset Password plugin once again correctly enables the reset_password public page.

1.11 PRB060470 (UI): Multi-line text fields made read only with UI Policy either could not be scrolled (IE) or could not be copied (Firefox)

Behavior is now consistent with multi-line fields made read only with UI Policy - they can be scrolled and copied.

1.12 PRB560810 (Workflow): The Show Workflow UI Action for the Change Request table did not work for the change_request_imac extension table

That UI Action now works as expected for the change_request_imac table.

1.13 PRB560693 (Service Catalog): Variable pricing implications not reflected in shopping cart when original item price was zero

Variable pricing implications are now reflected in the shopping cart even if the original item price is zero.

1.14 PRB560636 (i18n): Service Catalog delivery time units (e.g., "days") were not properly translated

Service Catalog delivery time units can now be translated.

1.15 PRB560544 (Update Sets): An update synch'd record could cause a Unique Key Violation if its class was changed and it was subsequently pulled in an Update Set

Update Sets can now properly handle record class changes for customizations.

1.16 PRB560745 (Discovery): The "active" field for Discovery Range Sets was not being honored

The "active" field for Discovery Range Sets now behaves as expected, preventing the range set from being included in the discovery.

1.17 PRB560656 (UI): Sorting on a field whose values were all equal and then paging through the list did not work properly in Oracle

Paging now works as expected in Oracle when sorted on such a field.

1.18 PRB560434 (Searching): Before query business rules were not applied to Global Text Searches

This has been fixed with the new text search engine included with this release.

1.19 PRB560508 (Variables): Label variables could not be hidden using a Catalog Client Script

Issues were fixed with doing the following with Catalog Client Scripts:

  • hiding a label variable
  • hiding a checkbox container label
  • hiding a container
  • hiding a checkbox having the container automatically hide when last checkbox has been removed from it

1.20 PRB559564 (Help the Help Desk): Help the Help Desk did not find 32-bit software in a 64-bit operating system

Help the Help Desk can now find 32-bit software in a 64-bit OS. Note that a 64-bit browser MUST be used for Help the Help Desk to work properly in a 64-bit OS.

1.21 PRB426355 (Discovery): IP Default Gateway was not populated by a network probe

It is now.

1.22 PRB560688 (Release): The Release v2 Planning Board generated a JavaScript error in IE related to the use of "removeClassName"

The cause of the error has been removed.

1.23 PRB560458 (Conditions): Assignment Rules might not work if there was an apostrophe or open brace in the condition

Special characters in CONTAINS expressions could potentially generate an error in a condition field. The source of this error has been removed.

1.24 PRB560509 (Help the Help Desk): Help the Help Desk ECC queue entries might not get processed

The business rule that processes these entries required a modification due to another code change.

1.25 PRB559912 (Discovery): The sysName tag of SNMP devices was not being properly captured when a dotted notation was used in the device name (e.g., abc.xyz)

This situation is now handled correctly.

1.26 PRB560449 (Discovery): Rediscovery of an existing software package could improperly reset install count, license count, and licenses available numbers

These numbers are no longer reset by a rediscovery. They are also no longer populated on creation, since a nightly scheduled job does the right calculations.

1.27 PRB042264 (Wizards): Advanced Reference Qualifiers did not work in Wizard reference variables

Advanced Reference Qualifiers now work as expected for Wizard reference variables.

1.28 PRB559545 (Discovery): Values set in the SNMP or Windows classifier might not get executed

The "run classification" script has been fixed to correct this issue.

1.29 PRB560279 (Update Sets): Views might not get created soon enough when included in an Update Set

sys_ui_view records are now created before anything else except Dictionary entries when an update set is applied.

1.30 PRB560843 (Security): Newly added attachments on new records cannot be viewed by non-admins

"read" ACL fixed for sys_attachment. If record doesn't exist yet (i.e., it's a new record form), let users see the attachment they just uploaded.
