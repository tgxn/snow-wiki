1 Overview

The Calgary Patch 6 release provides fixes for the Calgary release. This release is in a state of controlled availability (see release terminology). For more information about how to upgrade your instance, see Requesting an Upgrade.

For the Calgary Patch 6 release:

Build date: 05-27-2014_1557
Build tag: glide-calgary-02-15-2013__patch6-05-20-2014
 Note: See how to determine the version on your instance.


2 Before Upgrading to Calgary Patch 6

Review this information before upgrading an instance to Calgary Patch 6:

  • Customers who use Amazon EC2 or VMware for cloud provisioning must perform additional actions before upgrading to Calgary Patch 6.

3 Fixed in This Release

Category Problem Description
Authentication PRB594205 OpenID redirection does not preserve the original target URI properly.
Click the plus for steps to reproduce
  1. Enable OpenID plugin and enable the installation exit script.
  2. Clear out all cookies (or open a new incognito window in Chrome).
  3. Click a nav_to link.
  4. Sign on with a valid account to pass OpenID authentication.
  5. Trace the network calls and you will notice that /navigator.do is redirecting the session.
  6. Click the same link again and it should open the sys id record because session already has a valid OpenID cookie.
Chat PRB598937 Chat sessions between a chat agent and ESS user are lost if there is a network outage.
Click the plus for steps to reproduce
  1. Login to an instance.
  2. Enable the chat plugin.
    You will need to use two different browsers.
  3. In one browser (Chrome) impersonate David Loo (chat agent) and in another browser (Firefox) impersonate Joy employee as ESS user.
  4. In Firefox while impersonating as "Joy employee", navigate to ESS portal and click on Service Desk Chat and initiate a chat with help desk.
  5. In Chrome while impersonating as David Loo (chat agent), answer the chat in queue initiated by "Joy employee" in step 4.
  6. Type few messages from both end and notice that message flow is fine in chat window for both the users.
  7. Now disable the network connection for 2 minutes and enable again. Do not close the chat windows in between.
  8. After enabling the internet connection, try communicating using both chat windows.
    Notice that messages from David Loo (Chat agent) are reaching to Joe Employee (ESS user) but at the same time messages typed from Joe Employee (ESS) are not reaching to David loo (Chat agent). Also on the chat window for Joe Employee (ESS user) messages are duplicating like they are echoing but not reaching to David Loo (Chat agent) end.
Content Management System PRB588603 There is a cache issue with CMS page title translation. If a Title field on a CMS page is translated, the language setting of the first user to visit that page following the translation sets the language of the Title field for subsequent users.
Click the plus for steps to reproduce
  1. Open two instances in two different browsers.
  2. Log in to the first instance in English and to the second instance in French.
  3. In the English session, navigate to the Portal - Ask a Question page (https://xxx.service-now.com/nav_to.do?uri=content_page.do?sys_id=66e5e90fc0a8016b0059f2bc533ee8c8).
    The Title field should be empty.
  4. Enter the following text in the Title field: ESS Ask a Question.
  5. Save the form.
  6. Navigate to the page in the CMS (https://xxx.service-now.com/ess/ask_question.do ) and verify that the new title is displayed correctly.
  7. In the French session, navigate to the Portal - Ask a Question page (https://xxx.service-now.com/nav_to.do?uri=content_page.do?sys_id=66e5e90fc0a8016b0059f2bc533ee8c8).
  8. Enter the following translated text in the Title field: ESS Poser une Question.
  9. Save the form.
  10. Navigate to the page in the CMS (https://xxx.service-now.com/ess/ask_question.do ) and verify that the French title is displayed correctly.
  11. In the English session, reload the page (https://xxx.service-now.com/ess/ask_question.do).
    Note that the title appears in French.
  12. Clear the cache by going to /cache.do with either session.
  13. In the English session, reload the page (https://xxx.service-now.com/ess/ask_question.do).
    Note that the title now appears in English.
  14. In the French session, without clearing the cache again, reload the page (https://xxx.service-now.com/ess/ask_question.do).
    Note that the title appears in English.
Contract Management PRB591724 In Contract Management, if you were a contract manager and you specified a personal date format that was different than the system default, date fields in the Renew and Extend pop-up windows did not work. The pop-up windows only displayed the dates in the system default format (yyyy-MM-dd). When you submitted these dates, conflicts arose in the underlying scripts because the date format did not match your personal user format. The system generated error messages about the dates because it did not correctly interpret the dates.
Discovery PRB597426 J2ssh does not support AES-encrypted private keys.
IT Governance, Risk and Compliance PRB598435 IT GRC Control Tests have no default method of deactivating records.
Click the plus for steps to reproduce
  1. Go to System Definition > Plugins.
  2. Search for and activate IT Governance, Risk and Compliance (ITGRC) with Demo Data.
  3. Go to IT GRC > Control Tests > All.
  4. Select a Pending or Work in Progress record.
  5. Set the State to Success or Failure.
  6. Save the record.
    Active will still be set to True.
Localization PRB588808 Setting the property glide.sys.language to uk without any language packs installed can cause conflicts with the Ukranian standard language code.
Click the plus for steps to reproduce
  1. Navigate to System Properties > System Localization, change Default language for the system to uk.
  2. Log out then log back in.
  3. Open a problem record.
  4. Click the link Post Knowledge.
  5. Record the new article number, for example, "KB0010002".
  6. Navigate to "Knowledge Base" > Edit.
  7. Open the newly created knowledge article.
  8. Notice the Text field is empty.
Notifications PRB560708 Email contains the same Message-ID as a previously sent email, preventing loop by setting it to the Ignored state.
Notifications PRB570164 Customers running Gmail or Outlook 2011 on Macintosh cannot accept calendar invitations from ServiceNow.
Click the plus for steps to reproduce
  1. Configure email for the instance.
  2. Navigate to User Administration > Users.
  3. Select a user and set the Calendar Integration field to Outlook.
  4. Set the Email field to a valid address on an Outlook 2011 Mac client.
  5. Create a change request and assign it to the user selected in step 3.
  6. Specify a planned start and end date.
    The instance sends a calendar invitation notification to the email client. The email displays as plain text rather than an invitation.
Notifications PRB576131 When a translated HTML field is referenced inside an email notification, the sent email displays the HTML as plain text.
Click the plus for steps to reproduce
  1. Create a translated HTML field.
  2. Reference the field in an email notification.
  3. Send the email notification with some text in the translated HTML field.
Notifications PRB577336 When opening then closing an email client without sending the email, an email appears in the Inbox with type "received."
Click the plus for steps to reproduce
  1. Click on the email icon when viewing an incident or change.
  2. After the pop-up window appears, close the window.
  3. Check System Log > Emails and you will find the matching email with "Type" set to 'received'.
Notifications PRB582027 Inbound emails with foreign characters do not show properly in activity logs.
Click the plus for steps to reproduce
  1. Send email to the instance with the following body text:
    ----------------------
    Letter 1: é
    Letter 2: î
    Letter 3: á
    é , î , á
    ---------------------
  2. View the created incident and the email record as displayed in the Activity Log.
Notifications PRB582891 The glide.email.reply_subject_prefix and glide.email.forward_subject_prefix properties do not work when processing emails whose reply or forward case does not match exactly. In instances running versions prior to the Dublin release, the property names must appear in all lower case.
Click the plus for steps to reproduce
  1. Change the glide.email.forward_subject_prefix property to a random value, such as the following: Blue?
  2. Change the glide.email.reply_subject_prefix property to the following value: fw:, fwd:, re:, r:
  3. Forward an email to the instance with the normal FW: subject.
    The email will be treated as a forward.
  4. Forward the same email again and change the subject to Blue? instead of FW:
    The email will be processed as a new email.
Notifications PRB582908 An invalid watermark on an inbound email can prevent updates to a record, and produce unclear log statements.
Click the plus for steps to reproduce
  1. Trigger a notification generated from a record, such as commenting on an incident.
  2. Reply to the email.
  3. Notice the logs generated for the Inbound Email actions.
Notifications PRB583280 Since Berlin Patch 3, JavaScript no longer works in the email client template.
Notifications PRB584727 Emails with attachments that do not have file names are not delivered. This commonly occurs with emails from Lotus Notes.
Notifications PRB586226 When the UI macro email_reply is enabled and you reply to a received email using this feature, the To field is populated with a sys_id as opposed to the actual email address.
Click the plus for steps to reproduce
  1. Enable the UI macro email_reply.
  2. Send an email to the instance to generate a received email.
  3. Select either Reply or Reply All.
    Notice that the To field is populated with a sys_id instead of an email address.
Notifications PRB597407 Notifications may not be sent as expected. This can happen when the notification is assigned to a domain and configured to be sent when a record is inserted or updated.
Click the plus for steps to reproduce
  1. Turn on domains.
  2. Set a notification to a domain.
  3. Clear the cache.
  4. Log in as a user in the same domain to which you set the notification.
  5. Trigger the notification by updating the incident.
    Note that the notification does not create an email.
Platform PRB590634 Upgrading to Calgary with UK as the system language creates sys_translated_text records with incorrect values.
Click the plus for steps to reproduce
  1. Log into a non-Calgary instance.
  2. Set the glide.sys.language property to UK.
  3. Upgrade the instance to Calgary.
  4. Go to the demo CMS site <instance url>/ess.
    The menu items will all say "Using Dynamic Catalog Categories".
Platform PRB597356 The TextFieldQuestion class cannot be used as a scriptable object.
Platform PRB599946 Instance is not recoverable from encryption error caused by bad user input.
Reporting PRB585007 When using a pivot table and clicking on a cell, the expanded breadcrumb is missing a condition.
Reporting PRB594176 Trend reports do not display correctly. For example, the filter produces data from Monday to Sunday but the rendered chart displays data from Sunday to Saturday. Changing the glide.ui.date_format.first_day_of_week attribute has no effect on the selection.
Click the plus for steps to reproduce
  1. Navigate to Reports > View / Run.
  2. Select Type = Trend Report.
  3. Select Trend Field = date_field per = Day.
  4. Add filter and Order = date_field on Last week.
Reporting PRB599847 For group reports, the auto-complete feature for the Groups and Users fields does not work.
Click the plus for steps to reproduce
  1. Open a group report.
  2. In the Groups field, type the first few letters of a valid group name.
    The auto-complete feature should display a choice list of group names that begin with the same letters but does not.
  3. In the Users field, type the first few letters of a valid user name.
    Note that the same behavior occurs.
Service Catalog PRB583281 You cannot preview a record producer that is flagged as inactive.
Click the plus for steps to reproduce
  1. Log in to an instance.
  2. Navigate to Service Catalog > Record Producers > Emergency Change.
  3. If the Emergency Change record producer is active, clear the check box to make it inactive.
  4. Click Preview Item on the record producer form.
    Note that you will get a "Not Authorized" message and a white form even if you are an admin.
Service Catalog PRB595996 When an order guide with an "Ignore Mandatory Evaluation" item is visited, items *without* this field set become non-mandatory until /cache.do is run.
Service Catalog PRB597077 In Portal view, the shopping cart is missing.
Click the plus for steps to reproduce
  1. Log in to a demo instance/ess.
  2. Select Services. Notice there is no shopping cart.
  3. Log in to the demo instance, then navigate to Sites and then open Employee Self Service.
  4. Notice Use external cart is not checked (if it is not checked it should have a shopping cart), then check use external cart.
  5. Log in to a demo instance/ess.
  6. Select Services. Notice there is still no shopping cart: checking or not checking external cart makes no difference.
Service Catalog PRB598530 Order guides with a large number of items can take a long time to open the choose options screen when the cache has been cleared.
Service Catalog PRB599714 The wizard client script is not working if the variable type is set to UI Page.
Service Level Agreements PRB582222 Use of "HistorySet" class in TaskSLAController script include can change the state of "current".
Click the plus for steps to reproduce
  1. Ensure you have the 2011 SLA engine enabled and that the property "Compute prior SLA pause time for new, retroactive SLAs (2011 SLA engine only)" is ticked
  2. Create 2 SLAs with start conditions that match with assignment group being set to different values and that have a pause condition of "Assigned to" is not empty
  3. Create an Incident and set the Assignment group to match with one of your SLA Definitions, enter a value in assigned to and save.
  4. You should find that your Incident has a single Task SLA record which is paused - this is correct as it has matched with the assignment group but then immediately paused because assigned to is not empty
  5. Update the Incident to change assignment group to the group specified in the start condition of your second SLA definition and also clear the Assigned to field and then save.
  6. You should find that the original Task SLA is now cancelled - this is correct as the start conditions no longer match. You'll also see that your second SLA definition is now attached to the Incident as a task sla record (which is correct because of the assignment group change), however, it will be paused which is incorrect as the assigned to field is empty.
Social IT PRB584439 The glide.live_feed.company_feed_exclude_groups property is not honored when there are replies.
Click the plus for steps to reproduce
  1. Set the glide.live_feed.company_feed_exclude_groups property to true.
  2. Create a live public group.
  3. Add a message to the group.
  4. Go to the company feed and notice the message is not displayed.
  5. Go to the group and add a reply to the group message added earlier.
  6. Go to the company feed and notice that the message and the reply are now displayed.
Software Asset Management PRB598627 When many software counters are updated using the Rebuild SAM Cache UI action at or near the same time, the count is incorrect for some of them.
Click the plus for steps to reproduce
  1. Activate the Software Asset Management plugin.
  2. Execute the daily SAM License Counters scheduled job.
  3. Confirm counters are rebuilt with correct results.
  4. Confirm there are not any records in the Software Counter Detail [sam_sw_counter_detail]] table where the Software counter summary field is empty.
  5. On the Software Counter table, modify the Rebuild SAM Cache UI action to display on list view. Also comment out the line to redirect to current while testing from list view.
  6. Navigate to the list of software counters.
  7. Select all counters in the list and select Rebuild SAM Cache from the list context menu.
  8. When all counters are rebuilt, confirm the results are different and that there are now records in the Software Counter Detail table that have an empty Software counter summary field.
Surveys PRB598518 After you upgrade from an instance that did not use high security settings, users with the survey_admin role cannot see the question text in the Survey Response [survey_response] table.
Text Search PRB587551 Text search queries with multiple search terms using relevancy sorting can return documents in an inconsistent order that can change for each execution of the query. This only affects phrase searches, since word position information is not returned to the document scorer.
Click the plus for steps to reproduce
  1. Navigate to a table without the text_index_attachment_displayed property (for example, the Task table).
  2. Enter a multi-word search.
    Documents will appear in an unexpected order, as the sequence scoring algorithm is not provided with position information for the documents' words.
UI Policy PRB593038 GlideForm methods are not working in the wizard checkout panel.
Click the plus for steps to reproduce
  1. In a demo instance, set the active flag to true on the sys_app_application system wizards.
  2. Open the wizard: Catalog Ordering.
  3. Edit the last panel and then check out.
  4. Edit variables and add the two available variables (bb and desktop).
  5. Test the wizard.
  6. On the checkout panel, open the JavaScript executor and try the following code: alert(g_form.getValue('bb'));
  7. Note that this returns an empty value.
