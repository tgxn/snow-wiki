1 Overview

The simplest single sign-on authentication method is to pass user credentials within an unencrypted HTTP header. ServiceNow reads the HTTP header value and searches for a matching value in the User table. If there is a matching value, the instance considers the user pre-authenticated and logs the user in.

2 Integration Requirements

An Unencrypted HTTP header integration requires:

  • A web server
  • SiteMinder or another single sign-on application to pre-authenticate the user on the local network
  • A web page or portal that passes user credentials to the target ServiceNow instance in one of these formats
    • HTTP Header
    • URL parameter
    • Cookie

3 Unencrypted HTTP Header Setup

Perform these steps to enable single sign-on using an unencrypted HTTP header:

  1. Generate an unencrypted HTTP header.
  2. Enter single sign-on system properties.
  3. Test the integration.

3.1 Step 1. Generate an Unencrypted HTTP Header

ServiceNow only accepts one HTTP header as the source of pre-authentication. Choose an HTTP header that matches data from the User (sys_user) table. Configure your single sign-on software or authentication portal to generate the desired HTTP header. For example, here are some sample SiteMinder HTTP headers.

User credential Sample HTTP header Matching sys_user field
User name SM_USER user_name
Email address EMAIL email

See Sample Implementations for other ways of generating the HTTP header.

3.2 Step 2. Enter Single Sign-on System Properties

Set these properties to enable single sign-on with an unencrypted HTTP header.

  1. Navigate to System Properties > Single Sign-on.
  2. For Enable external authentication, select Yes.
  3. For HTTP header name to look for an externally authorized user (e.g. for SiteMinder), enter the HTTP header you generated in Step 1. For example, SM_USER.
  4. For Service-now.com field name to match against the incoming header, enter the sys_user field that contains matching data. For example, user_name.
  5. Click Save.

3.3 Step 3. Test the Integration

This single sign-on method allows pre-authentication from URL parameters, HTTP headers, or cookies. Verify users can sign-on from all methods. For example:

  • URL Format: http://company.service-now.com/navpage.do?SM_USER=joe.employee
  • HTTP Header Format: SM_USER=joe.employee
  • Cookie Format: 'SM_USER=joe.employee; expires=Fri, 27 Jul 2012 02:47:11 UTC; path=/'
