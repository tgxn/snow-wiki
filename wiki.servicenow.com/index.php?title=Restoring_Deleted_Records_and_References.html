1 Overview

If you inadvertently delete a record, you can restore it; however, you cannot restore references to the record. For example, if you delete a user record, then restore it, incidents that referenced the user in the Caller ID or Assigned To fields display those fields as blank in the incident form.

2 Restoring Deleted Records

ServiceNow can track deletions on any table except in these circumstances:

  • Record deletions are not tracked on tables with the no_audit_delete=true dictionary attribute.
  • File attachments are not restored when the record is restored.
  • Deletions from tables with a sys prefix are not tracked. To track deletions from these tables, add the table name to the glide.ui.audit_deleted_tables property.

To restore delete records:

  1. Navigate to System Definition > Deleted Records.
    The Deleted Records module is not active by default. You may need to enable the module.
  2. Select the check box for records you want to restore.
  3. Select Undelete Records from the Actions on selected rows menu.
