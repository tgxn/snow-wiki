1 Overview

Administrators typically enable a Netcool integration to track and manage events from an ObjectServer with a ServiceNow instance. The procedures on this page guide you through the process of setting up a Netcool integration.

2 Integration Setup

These steps enable the transfer of events between a Netcool/OMNIbus ObjectServer and a ServiceNow instance.

  1. Activate the Integration - IBM Netcool/OMNIbus plugin.
  2. Identify a local system that has direct JDBC connectivity to the Netcool/OMNIbus ObjectServer, usually over port 4100. Install a MID Server on this local system.
  3. Set integration properties.
  4. Create a Netcool Integration Setup record.
  5. Create one or more JDBC connections to the MID Server.
  6. Choose the inbound workflow method.
    • Configure a Socket Monitor to listen for events from the Netcool Socket Writer Gateway and automatically send them to the ECC queue.
    • Configure a SOAP insert web service to automatically send events directly to the Netcool Alerts Status web service.
  7. [Optional] Add custom fields to the integration.

3 Selecting the Target Table

By default, the integration transforms incoming events into records in the Incident table. Administrators can instead import incoming events directly into the Event Management table to take advantage of improved event management and alert creation features (starting with the Fuji release).

  1. Navigate to Integration - Netcool/OMNIbus > Properties.
  2. From Choose where Netcool events should be redirected to, select the target table for storing incoming events.
  3. Click Save.

4 Creating a Netcool Integration Setup Record

An integration setup record defines the connection and mapping properties necessary to transfer events between an ObjectServer and a ServiceNow instance. A Netcool Integration Setup record allows you to:

  • Specify the MID Server that manages the integration.
  • Enable or disable error notifications.
  • See the current status of the integration MID Server.
  • Create one or more JDBC connections to send events to the Netcool/OMNIbus ObjectServer.
  • [Optional] Create one or more Socket Monitors to receive events from the Netcool/OMNIbus ObjectServer.
Sample Netcool Integration Setup record

To create an Netcool Integration Record:

  1. Navigate to Integration - Netcool/OMNIbus > Setup.
  2. Click New.
  3. Fill in the fields on the form, as appropriate (see table).
  4. Click Save.
Field Description
MID Server The MID Server managing communications between the ObjectServer and the instance. For more information, see MID Server Installation.
Notify on error Option to have the integration send email notifications when errors occur.
Notify user User who receives notifications. This field is only visible when Notify on error is selected.
Status Indicator of whether the integration MID Server is up or down.
Netcool JDBC Connections Related list of available connections to the ObjectServer. The integration uses JDBC connections to update events on the ObjectServer. You can add as many JDBC connection as necessary, but the integration only maintains one active connection at a time. See Adding JDBC Connections
Socket Monitors Related list of available Socket Monitors listening to Netcool events from a Netcool Socket Writer Gateway on the ObjectServer. See Creating a Socket Monitor Record.

4.1 Enabling Notifications

The integration can send email notifications to a specific user when there are connection or Socket Monitor errors.

  1. Navigate to Integration - Netcool/OMNIbus > Setup.
  2. Click the creation date link for an integration setup record.
  3. Select the Notify on error check box.
  4. In the Notify user field, select a user to receive notifications about the MID Server.

The integration can send an email notification when these conditions occur.

  • When a JDBC connection changes to an Error state.
  • When there are no more JDBC connections available.
  • When a Socket Monitor changes to an Error state.
  • When a Socket Monitor stops.

To edit the integration notifications, go to System Policy > Notifications.

Netcool Integration notification records

4.2 Viewing Integration Status

The integration requires an active MID Server connection to transfer events.

  1. Navigate to Integration - Netcool/OMNIbus > Setup.
  2. Click the creation date link for an integration setup record.
    The Status field shows the current state of the integration MID Server. The status is either Up or Down.

5 Creating a Socket Monitor Record

A Socket Monitor record allows the integration to listen for events from the Socket Writer Gateway on the ObjectServer. A Socket Monitor record allows you to:

  • Specify the port on which it listens for incoming events from the Netcool Socket Writer Gateway.
  • [Optional] Automatically start the Socket Writer Gateway.
    • Specify the path to the Socket Writer Gateway.
    • Specify the ObjectServer fields mapped to the integration.
    • Specify the integration connection properties.
  • Automatically start the Socket Monitor.
Sample Socket Monitor record

To create a Socket Monitor:

  1. Navigate to Integration - Netcool/OMNIbus > Socket Monitors.
  2. Click New.
  3. Fill in the fields, as appropriate (see table).
  4. Click Submit.
Field Description
Port Communications port number that the Socket Writer Gateway writes to.
Start socket writer [Optional] Option that allows the Socket Monitor to automatically start the Socket Writer Gateway on the ObjectServer. The Map and Properties fields are visible only if you select this option.
Socket writer gateway Path to the Socket Writer Gateway executable. Click the suggestion icon () to select the correct path for your operating system and installation.
Name Automatically created name to identify this Socket Monitor.
State Current state of the monitor. The possible states are:
  • Ready: ready to start.
  • Running: already started and currently running.
  • Stopped: stopped. If you manually stop the monitor, then you must also manually stop the Socket Writer Gateway.
  • Error: could not start.
Map Socket Writer Gateway map. The integration uses this map to create an ECC queue message. Typically, you only edit this field to add a custom ObjectServer field to the integration or update a previously added field. See Updating the Socket Writer Gateway Map.

This field is available only if you select the Start socket writer check box.

Properties Socket Writer Gateway properties such as event update/insert data prefixes and connection behavior. Typically, you only edit this field to change properties of the connection between the ObjectServer and the ServiceNow instance. See Updating Socket Writer Gateway Properties.

This field is available only if you select the Start socket writer check box.

5.1 Automatically Starting the Socket Writer Gateway

The Socket Monitor can automatically start the Socket Writer Gateway whenever the integration is active.

  1. Navigate to Integration - Netcool/OMNIbus > Socket Monitors.
  2. Click the Port link for a Socket Monitor record.
  3. Select the Start socket writer check box.
    The form displays the Socket writer gateway, Map, and Properties fields.
  4. Enter or select the path to the Socket writer gateway executable. Click the suggestion icon to select to correct path for your operating system and installation.
  5. Update the Map field to match the fields you want to include in the integration. Add any custom fields to the map definition.
  6. Update the Properties field to match the connection properties you want to use for the Socket Writer Gateway.

5.2 Updating the Socket Writer Gateway Map

The Socket Writer Gateway uses a map to define the event data sent through the Socket Monitor port. By default, the integration includes mappings for all the default fields in the ObjectServer alerts.status table. There is a one-for-one correspondence between the fields listed in the Socket Writer Gateway map and the web service fields listed in the Netcool Alerts Status web service. Typically, you need to modify the Map field only to include custom fields available from the ObjectServer.

5.4 Manually Configuring the Socket Writer Gateway

If you choose to run the Socket Writer Gateway manually or allow a Netcool process agent to control it, you must first configure the Socket Writer Gateway map definition and properties files on the ObjectServer.

  1. Log in to the ObjectServer operating system with an administrative account.
  2. Open the Socket Writer Gateway map definition file in a text editor.
    By default, the map definition file is available here:
    • UNIX: $OMNIHOME/gates/socket/socket.map
    • Windows: %OMNIHOME%\gates\socket\socket.map
  3. Update the map definition file to match the fields you want to include in the integration. Add any custom fields to the map definition file.
  4. Save the map definition file.
  5. Open the Socket Writer Gateway properties file in a text editor.
    By default, the properties file is available here:
    • UNIX: $OMNIHOME/gates/socket/NCO_GATE.props
    • Windows: %OMNIHOME%\gates\socket\NCO_GATE.props
  6. Update the properties file to match the connection properties you want to use for the Socket Writer Gateway.
  7. Save the properties file.
  8. Start the Socket Writer Gateway on the ObjectServer.

5.5 Manually Starting the Socket Monitor

The integration requires an active Socket Monitor in order to listen for events from the Socket Writer Gateway.

  1. Log in to the ServiceNow instance.
  2. Navigate to Integration - Netcool/OMNIbus > Socket Monitors.
  3. Click the Port link for a Socket Monitor record.
  4. Click the Start related link.

6 Adding JDBC Connections

The integration uses one or more JDBC connections to send outbound events to the ObjectServer. The MID Server caches successful JDBC connections for as long as the MID Server is running. This means that the integration only has to use a single login/connect command per connection during the lifetime of the MID Server or until the connection fails. You can add as many JDBC connections as necessary, but the integration only maintains one active connection at a time.

  1. Navigate to Integration - Netcool/OMNIbus > JDBC Connections.
  2. In the Netcool JDBC Connections section, click New.
  3. Fill in the fields, as appropriate (see table).
  4. Click Submit.
Sample JDBC connection

The Netcool JDBC Connection form contains the following fields.

Field Description
State The state the JDBC connection is in.
  • Ready: ready for use.
  • Active: currently connected or have successfully connected at least once to update the ObjectServer.
  • Error: did not connect to the ObjectServer. See the Error Message field for details.
Host The host name or IP address of the MID Server managing the JDBC connection.
Port The TCP port for the JDBC connection, typically 4100.
Database The Netcool/OMNIbus ObjectServer name (database name).
User The user ID that authenticates the JDBC connection.
Password The password that authenticates the JDBC user ID.
Query timeout The timeout period, in seconds, before the integration considers the JDBC connection inaccessible and places it in an error state.
Order The order or priority of the JDBC connection. The integration attempts to connect with the JDBC connection that has the lowest order value. If that connection fails, the integration retries the update on the next connection in order that is in the Ready state. If you specified a notify user in the Netcool Integration Setup record, the instance also sends email messages whenever a JDBC connection fails.
Driver The JDBC driver used for the connection (typically, the Sybase JConnect driver).
Driver Jar The Java JAR file containing the JDBC driver class (typically, the Sybase JConnect JAR file installed with Netcool/OMNIbus). You must specify a JAR file that is appropriate for the operating system on which the MID Server runs. Click the suggestion icon to select the correct JAR file.

7 Editing the Integration Web Service

If you choose to send ObjectServer events to the ServiceNow instance from a web service, you must create a web service client on the ObjectServer to consume the Netcool Alerts Status web service. A Netcool alerts status record allows you to:

  • View the name of the SOAP web service for the integration.
  • View the name of the import set table for the integration.
  • View the endpoint URL to the web service WSDL for the integration.
  • [Optional] Customize the list of web service fields for the integration.
  • Link to the transform map for the integration.
Edit Web Service form

Administrators typically only need to edit the web service to add custom fields to the integration.

7.1 Adding a Custom Field to the Web Service

To add a custom field to the list of web service fields in the NetCool Alerts Status web service record:

  1. Navigate to Integration - Netcool/OmniBus > Netcool Alerts Status.
  2. At the bottom of the Web Service Fields section, double-click Insert a new row .
  3. Enter the name of the web service label. For example MyCustomField.
  4. Click the green check mark. The integration automatically converts the label to a lower case field name. For example, u_mycustomfield.
    The system automatically creates the web service field name based on the label you type. It appends the prefix u_ to the beginning of the label and converts the string to all lowercase. Do not include a u_ prefix in the label name or the web service field name will start with a duplicate u_u_ prefix.

  5. Click the save icon.
  6. Click Update.

7.2 Adding a Custom Field to the Transform Map

To add the custom field to the Netcool Alerts Status transformation map:

  1. Navigate to Integration - Netcool/OmniBus > Transform Map.
  2. From the Field Maps related list, click New.
  3. From the Source field, select the label of the custom field you added to the web service field list. For example, MyCustomField.
  4. From the Target field, select the field where you want the integration to store the matching event data. For example My custom field.

  5. Click Submit.
