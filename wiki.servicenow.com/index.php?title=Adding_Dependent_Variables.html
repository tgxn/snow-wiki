1 Overview

This article presents an example of scripting field dependencies - in this case in the service catalog, creating Category\Subcategory dependency from the incident table.

Note: For variables of type Reference, see Adding Dependent Reference Variables.


Follow these steps to extend the Record Producer for submitting a new incident.

1.1 Create Two Variables for Category\Subcategory

  1. Navigate to the catalog item to be updated. 
  2. Right click to add new variable for category

1.1.4 Right click to add new client script

1.1.5 Create script as follows(you can copy script below image)

Note: When items are removed from a choice list by a Client Script, Google Chrome and Apple Safari will still display those "removed" items. When one of these web browsers is used, the "removed" items will be displayed as read only in the choice list and they cannot be chosen. 

 function onChange(control, oldValue, newValue, isLoading) {
if(newValue == oldValue){
 return;
 }
 //remove all items from subcat drop down to start
// Used the g_form.clearOptions() function instead of g_form.removeOption() function 
g_form.clearOptions('subcat');
 
 //build a new list of dependent options
 var gp = new GlideRecord('sys_choice');
 gp.addQuery('dependent_value', newValue);
 gp.addQuery('element', 'subcategory');
 gp.query();
 while(gp.next()){
  g_form.addOption('subcat', gp.value, gp.label);
 }
}

1.1.6 Additional OnLoad script to initially clear the Subcategory

  1. Create new onLoad script.
  2. Give it a name.
  3. Paste the following script in the Script field.
function onLoad (control, oldValue, newValue, isLoading) {
// Used the g_form.clearOptions() function instead of g_form.removeOption() function 
g_form.clearOptions('subcat');
}
