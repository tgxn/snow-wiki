1 Overview

ServiceNow provider users can request changes to a Linux computer configuration item (CI) that is assigned to them. ServiceNow provider change management ensures that requested changes go through a controlled change process that includes approvals. After a change is approved and the requesting user proceeds with the change, ServiceNow automatically modifies the computer to bring it to the desired state.

2 Configuration Workflows

Configuration workflows represent a configuration change to be made to a CI, such as installing a piece of software. Configuration workflows are mapped to configuration templates. Configuration workflows point to a workflow that defines how the configuration will be made.

2.1 Creating a Configuration Workflow

Configuration workflows are made up of an existing workflow in your system, configuration templates, and workflow attributes. For more information on working with workflows, see Workflows. Each configuration workflow is used to configure a CI or bring it to a desired state.

  1. Navigate to Configuration Automation > Configuration Workflows.
  2. Click New.
  3. Fill in the Configuration Workflow form as described in the table.
  4. Click Submit

For a workflow to be associated to a configuration workflow, it must have a special workflow input u_ci_id. This workflow input will contain the sys_id of the CI record being processed during workflow execution.

When the workflow is associated to a configuration workflow, the workflow inputs are copied as configuration workflow attributes. Optionally, a default value for each workflow attribute may be defined.

Field Description
Name [Required] Enter a name that identifies the attribute.
Workflow [Required] The selected workflow associated with the configuration.
Description Enter a textual description of the attribute.

2.1.1 Viewing the Graphical Workflow

Select a configuration workflow, and under Related Links, click Show Workflow to see the graphical workflow.

2.2 Editing Configuration Workflow Attributes

Configuration workflow attributes are copied from the associated workflow's inputs. Workflow attributes contain data that the workflows process. These attributes are typically made up of key value pairs. The attribute values may be edited as follows:

  1. Navigate to Configuration Automation > Configuration Workflows.
  2. Select a configuration workflow.
  3. Under Configuration Workflow Attributes, select an attribute record by clicking on the attribute name.
  4. Fill in the Configuration Workflow Attribute form as described in the table.
  5. Click Update to save the changes.


Field Description
Name [Required] Enter a name for the configuration workflow attribute.
Value Enter the value of the configuration workflow attribute
Description Enter a textual description of the configuration workflow attribute.

3 Configuration Templates

ServiceNow provider configuration templates can be composed of configuration workflows, attributes, and managed nodes. Configuration templates can be applied to single, or multiple node groups.

Note: A group of configuration items is called a configuration template in ServiceNow Provider and a node definition in Chef and Puppet.

3.1 Creating a Configuration Template

  1. Navigate to Configuration Automation > Configuration Templates.
  2. Click New Draft.
  3. Fill in the form as described in the table.
  4. Click Submit.
A new configuration template draft record


Field Description
Name Enter a name for the ServiceNow provider configuration template.
Provider [Read-only] The provider, in this case ServiceNow.
State [Read-only] The state of the configuration template: Draft or Published.
Replaces [Read-only] For configuration template draft records, the original configuration template record that the draft replaces when published.
Change Request [Read-only] The current state of the change request: Pending or Approved.

3.2 Editing Configuration Template Attributes

Attributes are added to the configuration templates in your ServiceNow instance when you add a configuration workflow. The attributes are typically made up of key value pairs. After a configuration workflow is added, the workflow-level attributes are inherited at the configuration template level.

  1. Navigate to Configuration Automation > Configuration Templates.
  2. Select a configuration template.
  3. Click Configuration Template Attributes.
  4. Double-click the Value field to edit the configuration workflow-level imported values.
    Edited values changed at the configuration template level will not change the corresponding values at the configuration workflow level.
Field Description
Name [Required] A name to identify the configuration template attribute
Value Enter the value of the configuration template attribute

4 Managing ServiceNow Provider Nodes

An administrator can configure the configuration templates and attributes for a specific node directly from the CI form. An individual node can have multiple configuration templates associated with it. To view a list of CIs that use ServiceNow configuration automation and are assigned to the logged-in user, navigate to Configuration Automation > Managed Nodes. By default, a ServiceNow user cannot directly modify the ServiceNow fields on the CI form, and must request changes through a controlled change process.

Note: Updates to a CI reset the Node group assignment state to Ready. The Ready state ensures that assignment rules are applied and that the CIs are re-assigned based on the new values. For example, updating fqdn or IP address might reassign a CI to a different node group.


To update a node:

  1. Navigate to Configuration Automation > Managed Nodes or Configuration > Servers > Linux.
  2. Open the node to update.
  3. Edit the fields under the Configuration Automation (see table).
  4. Click Update.
Field Description
Fully qualified domain name The fully qualified domain name that Chef uses to access this CI.
Provider The configuration automation provider managing this CI.
Node group assignment state The state of automatic assignment of this CI to a node group, as described in the following table.
State Description
Ready Default state for newly discovered CIs. Assignment rules are applied only to CIs in the Ready state.
In Progress CI is currently experiencing a configuration change.
Assigned CI is part of one or more node groups.
Processed Assignment rules have been applied to the CI. There may or may not be a match.
Excluded Configuration admin role can exclude a CI from being processed by the assignment rules.

4.1 Adding Additional Configuration Templates to a CI

An individual CI can have multiple configuration templates associated to it. Any attributes associated with the configuration template are inherited by the CI.

  1. Navigate to Configuration Automation > Managed Nodes or Configuration > Servers > Linux.
  2. Open the CI to update.
  3. Under Related Links, click Create New Configuration Change (or Modify the configuration change draft if a configuration change is already started).
  4. Under Requested Configuration, click New
  5. Edit the fields in the Requested Configuration form (see table).
  6. Right-click on the form header to select Save (or press Ctrl-S).
  7. Modify any attribute for the configuration template.
    Any attributes specified at the configuration template level will not populate to any configuration workflows that are associated with it.
  8. Click Update.
  9. Click Request Change.

These fields appear on the Requested Configuration form:

Field Description
Name Enter the name of the configuration template.
Configuration Template Select the configuration template.
Order Enter the listing order number for the configuration template.
