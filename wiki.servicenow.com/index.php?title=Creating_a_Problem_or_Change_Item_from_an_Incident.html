1 Overview

In certain situations, it is useful to be able to create a record on one table based on the information from a record on another table. In such a case, a UI Action can be configured that allows a user to create a record based on the current record simply by right-clicking in the form header bar.

Three records UI actions are available out-of-box:

  • Incident forms have Create Problem
  • Incident forms have Create Change
  • Problem forms have Create Change

2 Using a UI Action to Create a Record

To use a UI Action to create a record:

  1. Navigate to a record to be created from
  2. Right click on the header bar
  3. Click the appropriate UI Action.

The new record should appear on the appropriate table.

3 Configuring a Custom UI Action to Create a Record on Another Table

The out-of-box UI actions can be used as a template for implementing similar UI Actions on other pages. For instance, using the Problem UI Action, it is possible to create a Create Change UI Action on any Release forms. Because the Problem UI Action is simpler than the Incident UI Action, it serves as a better template.

Note: This functionality requires a knowledge of Javascript.


To configure a Custom UI Action:

  1. Navigate to System Definition > UI Actions
  2. Select the Create Change record that has Problem in the table field.
  3. Right click the header bar and select Insert.
  4. Update the condition and script as appropriate.
  5. Change the table to the desired new table. This is the table from which new records will be created.

Once the table is changed, the new record will automatically be saved and the record view will display.

4 Adding Related Records

It is common to create a related list of records on the new record in order to keep track of records on the original table. For instance, the Problem form has a list of related incidents, which might include the incident the problem was created from (if the problem was created from an incident).

If the related list is already present on the form, the record should already appear on the related list. More can be added to the related list by clicking the edit button.

If the related list does not already appear, add the appropriate related list.

5 Autoclosing Related Records

Task records on the related list will not close by default when the current task record is closed. This can be achieved by scripting a custom business rule.
