1 Overview

You can use the Test Management application to perform the following tasks if you have the tester (tm_tester) role:

  • Perform the tests in a test case and record the results.
  • Update the test status.
  • Report a defect if a test fails.
  • Report a blocking issue if a test is blocked.
  • Retest as necessary.
  • Update the test case status.

Any user can be assigned to a test case and perform the tests within the test case using the guided test execution feature.

2 Performing Tests

You can perform tests in one of these ways:

  • Using the Test Case form
  • Using the visual task board
  • Using guided test execution

2.1 Using the Test Case Form or the Visual Task Board

  1. Access the Test Case form or the visual task board:
    • Test Case form: open the test plan and click the assigned test case in the Test Cases related list.
    • Visual task board: open the test plan and click the Show Visual Task Board related link. Then open the assigned test case by clicking the test case number on the card.
  2. Set the Execution Status field on the Test Case form to In progress.
  3. Click Save.
  4. In the Tests related list, open the first test.
    The test order is determined by the number in the Order field.
  5. Set the Status field for the test to In progress.
  6. Right-click the form header and click Save.
  7. Perform the steps outlined in the Detailed description field.
  8. Record the result in the Actual result field.
  9. Compare the actual result to the expected result and update the Status field:
    • Passed
    • Failed
    • Blocked
  10. Click Update.
  11. Repeat steps 4-10 for the remaining tests in the test case.

2.2 Using Guided Test Execution

When the test manager initiates guided testing, the testers assigned to the test cases in the test plan are notified by email. Once you receive this email, you can begin testing.

  1. Navigate to Self-Service > My Tests or Self-Service > My Assessments.
    A card with the test plan name and assignment information appears on the page.
  2. Click the Start Testing button.
    This displays a list of assigned test cases and the tests included in each case.
  3. Perform each of the tests in the test case by following the individual test steps.
  4. Record a result for each of the tests: Failed, Passed, or Blocked.
  5. Record an overall result for the test case at the bottom of the list: Failed, Passed, or Blocked.
  6. Click Submit or Save.

3 Updating the Test Status

After performing a test and recording the result, set the Status field on the Test form to one of the following values.

  • Passed: the actual result of the test is the same as the expected result.
    • On the Test Case form, the Status field for the test is updated to Passed.
    • On the Test Plan form, the result is updated in the Tests Passed column.
  • Failed: the actual result of the test is not the same as the expected result.
    • A Report Defect link appears beside the Status field. If necessary, click this link to report a defect.
    • On the Test Case form, the Status field for the test is updated to Failed.
    • On the Test Plan form, the result is updated in the Tests Failed column.
  • Blocked: the test cannot be completed for one or more reasons.
    • A Blocked Reason field appears on the test form. This is a mandatory field; you must enter the blocking reason in this field before updating or saving the test.
    • On the Test Case form, the Status field for the test is updated to Blocked.

4 Reporting a Defect from a Failed Test

If necessary, you can report a defect related to a Failed test status.

  1. From the Test form:
    • Click the Report Defect link next to the Status field.
    • Click the Report Defect related link.
  2. Fill in the fields, as appropriate (see table).
  3. Click Submit.
Report Defect form

Field Description
Number Automatically assigned defect number. For example, DFCT0010002.
Configuration item Associated configuration item, if applicable to this defect.
Assignment group Group assigned to fix the defect.
Assigned to Individual assigned to fix the defect.
Priority Priority of the defect, from 1 (critical) to 5 (planning).
State The current state of the defect:
  • Draft
  • Scoping
  • Awaiting Approval
  • Work in Progress
  • Testing/QA
  • Deploy/Launch
  • Closed/Complete
  • On hold
  • Cancelled
Short description Brief explanation of the defect.
Description More detailed explanation of the defect.
Work notes Any comments, notes, or other information that would be helpful to the individuals fixing the defect.

5 Assigning a Defect to a Test Case

After reporting a defect from a failed test, you can assign the defect to a test case.

  1. From the Test form, click the Assign Defect to Test Case related link.
  2. Select a defect from the Assign Defect to Test Case dialog box.
  3. Click OK.
    The defect appears on the Test Plan form in the Test Plan Defects related list.

6 Updating the Test Case Status

After performing all of the tests in a test case, set the Execution Status on the Test Case form to one of the following values.

  • Passed: all of the tests assigned to this test case have a status of Passed.
    • The result is updated on the Test Plan form in the Test Cases related list.
  • Failed: one or more of the tests assigned to this test case have a status of Failed.
    • The result is updated on the Test Plan form in the Test Cases related list.
    • The Report Defect link appears beside the Execution Status field on the Test Case form. If necessary, click this link to create a defect. For more information, see Reporting a Defect from a Failed Test.
    • To link an existing defect to the test case, click the Assign Defect related link. For more information, see Assigning a Defect to a Test Case. When you link a defect to a test case, a reference to the test case is stored in the defect record. The Test Case form displays a related list of defects and the defect record displays a related list of test cases.
  • Blocked: one or more of the tests assigned to this test case have a status of Blocked.
    • The Blocked Reason field is displayed on the Test Case form. You must enter a reason why the test case is blocked before saving or updating the form.
  • Retest: a defect resulting from a failed test has been closed or cancelled and the test is ready to be run again.

The status of a test does not affect the status of the test case. Setting the status of the test case is a manual operation.
