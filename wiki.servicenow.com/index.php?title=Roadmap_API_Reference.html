Roadmap API Reference

From ServiceNow Wiki

Timeline Visualizations

1 Overview

The Roadmap API allows you to get view data for Timeline Visualizations. Using RoadmapUtil and AjaxRoadmapUtil you can get lanes and panels data for a three-dimensional view while moving either forward or backward on the road. Using Roadmap2DUtil and AjaxRoadmap2DUtil you can get lanes and panels data for a two-dimensional view while moving either forward or backward on the road.

1.1 Where To Use

Use these methods for customizing Timeline Visualization data.


2 Method Summary

Method Summary Description
getRoadmapLanes(key, keyVal) Gets data for all the lanes in JSON format.
getRoadmapItems(key, keyVal, fetchStartDate, laneIds, forward) Gets the data for three-dimensional lanes and panels.
getRoadmap2DItems(key, keyVal, fetchStartDate, laneIds, forward) Gets the data for two-dimensional lanes and panels.


3 Method Detail

3.1 getRoadmapLanes(key, keyVal)

Gets the data for all the lanes in JSON format.

3.1.1 Input Fields

Parameters:

  • key - (String) The key for the roadmap_config table (for example, name) to access the configuration record.
  • keyVal - (String) The key value for the roadmap_config table (for example, CIO Roadmap) to access the configuration record.

3.1.2 Output Fields

Returns: the data for all the lanes in JSON format

3.1.3 Example

var roadmapUtil = new RoadmapUtil();
var key = "name";
var keyVal = "CIO Roadmap";
var response = roadmapUtil.getRoadmapLanes(key, keyVal);

3.2 getRoadmapItems(key, keyVal, fetchStartDate, laneIds, forward)

Gets the data for three-dimensional lanes and panels (for example, items for each lane) in JSON format. The lanes or items for any given query result are numbered 0..(n-1), where there are n lanes or items.

3.2.1 Input Fields

Parameters:

  • key - (String) The key for the roadmap_config table (for example, name) to access configuration record.
  • keyVal - (String) The key value for the roadmap_config table (for example, CIO Roadmap) to access configuration record.
  • fetchStartDate - (GlideDateTime) [optional] The fetchStartDate from which to get the items.
  • laneIds - (String) [optional] The lanes for which data will be returned.
  • forward - (boolean) [optional] Determines whether data is fetched forward or backward. If set to false, data is fetched in the backward direction for the duration of ROADMAP_FETCH_DURATION (for example, 2 years).

3.2.2 Output Fields

Returns: the data for lanes and panels in JSON format

3.2.3 Example

var roadmapUtil = new RoadmapUtil();
var key = "name";
var keyVal = "CIO Roadmap";
var itemQuery = "gs.monthsAgoStart(9)^EQ^ORDERBYend_date";
var fetchStartDate = gs.now();
var forward = true; 
var selectedLanes = ["0d563434bf30110058c9beba3f0739d5","32a67434bf30110058c9beba3f07392c","02b67434bf30110058c9beba3f07393b"];
var laneIds = new JSON().encode(selectedLanes);
var response = roadmapUtil.getRoadmapItems(key, keyVal, itemQuery, fetchStartDate, laneIds, forward);

3.3 getRoadmap2DItems(key, keyVal, fetchStartDate, laneIds, forward)

Gets the data for two-dimensional lanes and panels (for example, items for each lane) in JSON format. The lanes or items for any given query result are numbered 0..(n-1), where there are n lanes or items.

3.3.1 Input Fields

Parameters:

  • key - (String) The key for the roadmap_config table (for example, name) to access configuration record.
  • keyVal - (String) The key value for the roadmap_config table (for example, CIO Roadmap) to access configuration record.
  • fetchStartDate - (GlideDateTime) [optional] The fetchStartDate from which to get the items.
  • laneIds - (String) [optional] The lanes for which data will be returned.
  • forward - (boolean) [optional] Determines whether data is fetched forward or backward. If set to false, data is fetched in the backward direction for the duration of ROADMAP_FETCH_DURATION (for example, 2 years).

3.3.2 Output Fields

Returns: the data for lanes and panels in JSON format.
