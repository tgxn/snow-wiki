1 Overview

A project template defines the basic structure of a project. Project templates can include project tasks and sub-tasks, attachments, and other project information. Because projects often get repeated, templates enable the you to create, save, and reuse project structure. You can also modify existing templates, create projects from templates, and apply templates to empty projects. A project is considered empty when it does not contain any sub-tasks. The project template feature is available starting with the Fuji release.

2 Creating Project Templates

You can create project templates if you have the project manager role.

Navigate to Project > Projects > Templates to display the Project Templates list. Click a template number to open the Project Template form.

The top section of the Project Template form includes basic template information, such as the template name and description. The middle section of the project template form defines the specific data included in the template. The information in this section is built from fields on the project [pm_project] table that the user selects and the values for those fields that the user defines.

Project Template form

Field Description
Name The name of the project template.
Table Display only. Templates are based on Project table.
Description A brief description of the project template.
Template Includes columns of fields and field values to be included in the template. Click a field in the left column and select the desired field name, then click the field in the right column to enter the field value. If additional information for the field is required, a field appear in the third column. For example, to identify the currency in a money field.
  • Click the 'X' to the right of a field to remove that field from the template.
  • Use the blank field at the bottom of the list to add new fields.
Related Links

Copy Template Creates a copy of this project template.
Related Lists

There are three ways to create a new template:

  • From an existing template
  • From a project
  • From the Project Workbench

2.1 Creating a New Template from an Existing Template

  1. Navigate to Project > Projects > Templates.
  2. Open the desired project template.
  3. Click the Copy Template related link.
  4. Fill in the fields on the Copy Template dialog box:
    • Template name: enter a unique name for the new template.
    • Description: enter a brief description of the new template.
    The Template field is auto-filled with the current template name.
  5. Click OK.
    The project template form opens for the newly created template.
  6. Make any desired changes to the fields on the project template form.
  7. Click Update.

2.2 Creating a New Template from a Project

When you create a template from a project, all of the project attachments are copied to the template. You can add or remove attachments from the template using the Project Template form.

  1. Navigate to Project > Projects > All.
  2. Open the desired project.
  3. Click the Save as New Template related link.
  4. Fill in the fields on the Create Template dialog box:
    • Template name: enter a unique name for the new template.
    • Description: enter a brief description of the new template.
    The Project Name field is auto-filled with the current project name.
  5. Click OK.
    The Project Template form opens for the newly created template.
  6. Make any desired changes to the new template.
  7. Click Update.

2.3 Creating a New Template from the Project Workbench

  1. Navigate to Project > Projects > All.
  2. Open the desired project.
  3. Click the Project Workbench related link.
  4. Click Project Detail () in the workbench header.
    This opens the project form in a dialog box.
  5. From the project dialog box, click the Save as New Template related link.
  6. Fill in the fields on the Create Template dialog box:
    • Template name: enter a unique name for the new template.
    • Description: enter a brief description of the new template.
    The Project Name field is auto-filled with the current project name.
  7. Click OK.
    This opens the Project Template form for the new template.
  8. Make any desired changes to the new template.
  9. Click Update.

2.4 Adding an Attachment to a Template

  1. Navigate to Project > Projects > Templates.
  2. Open the desired template.
  3. Click the Attachments tab to open the related list.
  4. Click New.
  5. Click Browse on the Attachments dialog box and select a file.
  6. If desired, click the Add Another Attachment button to select another file.
  7. Click Attach.
    The selected file appears in the Attachments related list.

3 Using Project Templates

The project template feature allows for easy creation of projects that follow a similar structure. You can create a project from a template if you have the project manager role. Projects can be created in the following ways:

  • From a project template form
  • From the project workbench
  • From the Create New module

3.1 Creating a Project from a Template

  1. Navigate to Project > Projects > Templates.
  2. Open the desired template.
  3. Click the Create Project related link.
  4. Fill in the fields on the Create Project From Template dialog box:
    • Project name: enter a name for the new project.
    • Start date: enter a start date for the new project.
    The Project template field is auto-filled with the current template name.
  5. Click OK.
    Template tasks and attachments are copied to the new project and displayed on the project form.
  6. Make any desired changes to the fields on the project form.
  7. Click Update.
Create Project From Template dialog box

3.2 Creating a Project from the Project Workbench

When you create a project from the project workbench, you have the option of creating the project using a template. Choose this option by clicking the link in the banner at the top of the project form.

Creating a project from the project workbench

  1. Navigate to Project > Projects > Workbench.
  2. Click the New Project button in the workbench header.
    This opens a new project form in a dialog box.
  3. Click the template link in the blue banner at the top of the form.
  4. Fill in the fields on the Apply Template dialog box:
    • Project name: enter a name for the new project.
    • Start date: enter a start date for the new project.
    • Project template: select the template to use for this project.
  5. Click OK.
    Template tasks and attachments are copied to the new project and displayed in the project workbench. For more information, see Using Project Workbench.

3.3 Applying a Template to an Empty Project

A project is considered empty when it does not contain any sub-tasks. When an empty project is opened in the project workbench, the Apply Template button appears in the workbench header.

  1. Navigate to Project > Projects > Workbench.
  2. Open the desired project.
  3. Click the Apply Template button in the workbench header.
  4. Fill in the fields on the Apply Template dialog box:
    • Start date: enter the start date of the project.
    • Project template: select a project template to be applied to this project.
  5. Click Apply.
    Template tasks and attachments are copied to the new project and displayed in the Project Workbench. For more information, see Using Project Workbench.
Apply Template dialog box

4 Project Template Configuration

The Project Template Configuration list defines the items that are included in a template. Navigate to Project > Settings > Template Config to display the Project Template Configuration list. Each item in this list has:

  • A defined parent table. The exception is pm_project, which is the root or top-level object in the template.
  • A link element. This is a field that links the parent table and the child table.
  • A list of fields to include in the template. The fields defined in this column are the fields that are copied to a project template.

The Project Template Configuration list contains three default template configuration items: project, project task, and project sub-task. You can modify these default items or create additional items from this list.

Item Definition Parent Link Element
pm_project Project object None. This object does not have a parent because it is the root level. None. This object does not have a link element because it is at the root level.
pm_project_task Project task object pm_project Parent. Because this task object is one level below the root level, it uses the parent table as a link element.
pm_project_task Project sub-task object pm_project_task Parent. Because this sub-task object is two levels below the root level, it uses the parent table as a link element.

Click on an item in the Project Template Configuration list to open the Project Template Configuration form.

Project Template Configuration form

Field Description
Table The entity to be parameterized in the template.
Parent table The parent table for this item.
Link element The field that links the table with the parent table.
Active A check box that indicates whether or not this item is included in a template.
Elements A comma-separated list of fields from the table selected in the Table field that are included in the template.

4.1 Adding a New Project Template Configuration Item

  1. Navigate to Project > Settings > Template Config.
  2. Click New.
  3. Fill in the following fields:
    • Table
    • Parent table
    • Link element
    • Active
    • Elements
  4. Click Submit.

For example, to add a requirement object to the template configuration:

  1. Navigate to Project > Settings > Template Config.
  2. Click New.
  3. Fill in the fields as follows:
    • Table: Requirement [dmn_requirement]
    • Parent table: Project [pm_project]
    • Link element: Task
    • Active: Enable this check box.
    • Elements:short_description,description,priority,type,state
  4. Click Submit.

4.2 Modifying a Project Template Configuration Item

  1. Navigate to Project > Settings > Template Config.
  2. Select a configuration item from the list.
  3. Make any desired changes to the fields on the form.
  4. Click Update.

4.3 Deleting a Project Template Configuration Item

  1. Navigate to Project > Settings > Template Config.
  2. Select a configuration item from the list.
  3. Click Delete.
 Note: Do not delete any of the default template configuration items: project, project task, or project sub-task. Doing so affects the ability to create new project templates.
