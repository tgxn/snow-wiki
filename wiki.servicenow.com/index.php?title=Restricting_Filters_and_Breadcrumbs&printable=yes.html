1 Overview

The record list view allows users to navigate to different subsets of a table using breadcrumbs and filters. Administrators, however, may want to limit access to parts of the table by restricting active links in breadcrumbs or by suppressing breadcrumbs and filters altogether for specific roles.

2 Restricting Breadcrumbs with Fixed Queries

A breadcrumb option enables an administrator to control the base view of a record list presented to users. By adding a fixed query to the argument for a module, an administrator can prevent users from expanding their view past a specified starting point. The argument for this fixed query is written as &sysparm_fixed_query=active=true. A use case for this query would be to prevent users from switching the view of a list of Open incidents to a list of All incidents using the breadcrumbs. When users select Incident > Open, they are limited to viewing and filtering a list of open (active=true) incidents only.

To configure a fixed query:

  1. Right-click the application in the navigation pane.
  2. Select Edit Application from the pop-up menu.
  3. Select the module to edit.
    For example, select Open.
  4. In the record for the Open module, select List of Records for the Link type.
  5. Delete the Active is true filter, if present.
  6. Add &sysparm_fixed_query=active=true to the Arguments field and update the record.


  7. Navigate to Incidents > Open and examine the breadcrumbs.
    The Active=true link is the widest view permitted in the list of Open incidents. Notice that the breadcrumb for All is not linked.


3 Suppressing Filters and Breadcrumbs

Suppress filters and breadcrumbs on defined tables for users with specific roles using list controls. For more information, see Configuring List Controls.

3.1 Script Includes

 Note: This functionality requires a knowledge of Javascript.

Filters and breadcrumbs can be restricted via script on a per-table or global basis to specific roles. This can more easily be done with List Control, but scripting in script includes offers additional flexibility.

Create a new script include with the name <tablename>DisplayFilter. The script section contains one function with the same name as the script include. Have your function set the global variable "answer" to either true (show the filters and breadcrumbs) or false (hide them.) The following example restricts filters and breadcrumbs on the "incident" table to users with a role (any role); that is, if a user has a role, then the user will be able to use filters and breadcrumbs. Be sure that the of the script matches the function name exactly (including case.)



If you want to exclude a specific role from having access to filters/breadcrumbs, you would make the following change:



Here the above text is duplicated in a form you can cut and paste: 


 function incidentDisplayFilter() {

   if (gs.getUser().hasRoles() && !gs.hasRole("newrole")) {
      answer = true;
   } else {
      answer = false;
   }
 
   if (gs.hasRole("admin")) {
      answer = true;
   }
 
   return answer;
 }


If you want to let all users use filters and breadcrumbs on the Incident table, your script might look like the following: 


 function incidentDisplayFilter() {
   answer = "true";
   return answer;
 }
