Domain Support PRB645230
KB0552436 		Database views are not honoring domain separation when domain paths is enabled This issue affects instances with Domain Separation configured to use Domain Paths as the query method. When the domain plugin is enabled and the domain_paths query method is used, the database views are not honoring domain separation. Database views that join two tables with both configured for domain separation are affected. As a result, a user in domain B may be able to access information that is restricted to domain A when accessing the view. Views on instances configured with all of the following are at risk: Domain Separation, Domain Paths, and Views joining two domain separated tables. For more information, see Database Views in the product documentation (http://wiki.servicenow.com/index.php?title=Database_Views).
Click the plus for steps to reproduce
  1. Ensure the domain plugin is active.
  2. Create new domains named DOMAIN_A and DOMAIN_B.
  3. Create table A with columns col1 and col2.
  4. Domain separate the table.
  5. Create table B with columns col3 and col4.
  6. Domain separate the table.
  7. Add a column refcolb on table A and make it a reference to table B.
  8. Add a column refcola on table B and make it a reference to table A.
  9. Create a database view named AB on tables A and B.
  10. Create a left join on table B and b.refcola=a.sys_id as the where clause on table B.
  11. Create several records on each of the tables and add references. When under DOMAIN_A on the DB view, a user can see entries under DOMAIN_B as well as the ones under DOMAIN_A and GLOBAL domains. This behavior is incorrect. When a user is under DOMAIN_A on the DB view, they should be able to see entries under DOMAIN_A and global domains and not the ones the ones under DOMAIN_B domain.
