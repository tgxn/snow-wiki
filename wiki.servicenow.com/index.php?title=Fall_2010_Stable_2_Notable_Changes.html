Fall 2010 Stable 2 Notable Changes

Fall 2010 Stable 2 Notable Changes
Jump to: navigation, search

The following are notable changes you may encounter with our Fall 2010 Stable 2 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before. For additional changes, see Fall 2010 Stable 2 Fixes.

1 Changes for Fall 2010 Stable 2

1.1 Personalizing Lists

  • If a user personalizes a list, an indicator () appears in the upper left corner (requires List v2 plugin).
  • If a list administrator has personalized a list and then changes the default columns for the list view (right-click the header and select Personalize > List Layout), a message appears above the slushbucket:
    You have personalized this list with the gear: () You will not see changes made here until your personal list is reset to column defaults.

1.2 Personalizing Forms

A new user preference (owned_by_indicator.form) allows administrators to configure an indicator () that appears on the header of forms with customer updates. Clicking the customer updates indicator opens the update set records for the item.

1.3 New Content Management Navigation Menu Block and UI

Two changes have been made to Navigation Menu Blocks:

  • When creating a new Navigation Menu block, a new graphical interface has been included to preview the Navigation Menu blocks.
  • A new Navigation Menu called Tab Menu for Headers has been created to integrate the existing Tab Menu block into a header block.

For more information, see Creating a Navigation Menu Block.

1.4 Time Card Role

A new role called timecard_admin enables users to approve, modify, and delete the timecards of other users.

1.5 Performance improvements on Knowledge portal for large topics

Performance has been improved in the Knowledge portal for very large topics of which very few articles are viewable. The roles part of the query was pushed to the database to prefilter what a user can see. An index was added to kb_knowledge.topic. Optimizations were made to attachment queries.

1.6 Wrapping tabs

Form Section and Related List tabs now wrap to a new row rather than scroll horizontally when there are too many tabs to render in one row onscreen.

1.7 New Monthly Schedule Options

Two new options have been added when a Schedule Entry is set to repeat Monthly:

  • Last Day of Month (runs on the last day of each month, e.g., 31 March)
  • Last Week Day of Month (runs on the last week day of each month, e.g., if 31 March is a Sunday, runs on 29 March)

1.8 CSS Theme Support Plugin Published

The CSS Theme Support Plugin, available for some time, no longer requires Technical Support to install it. An admin can now install this plugin.

1.9 Scheduled Chart Reports can be sent in PDF with Grid Support

Scheduled Bar Chart reports could previously be sent only as .png images, with no grid support. Bar Chart reports, when scheduled to be emailed, now have two additional Output Type options: PDF and PDF-landscape. These both will include the chart's grid data if "Include grid" is checked on the report.

1.10 ArrayUtil Script Include additions

Four methods have been added to the ArrayUtil Script Include: diff, intersect, union, and unique. They can be used as follows to perform common tasks with Arrays, and each returns an Array: 

var a = ["a", "b"];
var b = ["a", "c"];
var c = ["a", "b", "a"];
gs.print(new ArrayUtil().diff(a, b)); // prints b
gs.print(new ArrayUtil().intersect(a, b)); // prints a
gs.print(new ArrayUtil().union(a, b)); // prints b,a,c
gs.print(new ArrayUtil().unique(c)); // prints b,a

1.11 Attachments Ordered by Creation Date

File attachments to a record are now displayed in the order in which they were added. Previously they were displayed alphabetically, making it difficult to easily determine which was the most recently added attachment.
