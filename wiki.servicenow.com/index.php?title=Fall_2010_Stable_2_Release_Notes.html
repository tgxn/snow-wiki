The Fall 2010 Stable 2 release was released December 3rd and includes the following enhancements and new functionality.

A manual detailing new functionality can be found here.

This release is no longer recommended for upgrades. If you want to upgrade an instance, consider a current version. For the Fall 2010 Stable 2 release:

Build date: 12-03-2010_1413
Build tag: glide-fall2010-stable2-12-01-2010
 Note: See how to determine the version on your instance.

2 Description of Functionality

Our Fall 2010 Stable 2 release contains the following new or improved functionality:

  • Group On-Call Rotation Plugin Enhancements: New roles allow easier delegation of Rota management, notifications and reminders now have more configuration options, and coverage is now required for all active periods.
  • MID Server Autofinder for Runbook: MID Servers can be associated with IP address ranges, enabling Runbook to select the correct MID Server to use for an Orchestration activity based on the IP address of the target machine.
  • Survey Wizards: Survey Wizards create surveys using wizards. Advantages versus other survey functionality include capabilities to ask different questions based on responses (dynamic surveys), create multiple page surveys, and record answers for partially completed surveys.
  • Wizards enhancements:
    • Basic Panel Flow for Wizards: This feature allows wizards to moves through panels in order, without the need to define logic for each transition. The Basic panel flow field was added to the Wizard [expert] table and the order field was added to the Wizard Panel [expert_panel] table.
    • Role access control for Wizards: Administrators can now restrict user access to wizards by role.
    • UI Policy for Wizards: UI Policies can now be applied to Wizards.
    • Public Wizards: Administrators can now make wizards available to users that are not logged in (public).
    • Logged in users may now have more than one wizard in progress at a time. The expert field was added to the Wizard Instances [expert_instance] table.
  • Display Business Rules: Business Rules can now be set to run onDisplay to populate a form's scratchpad, accessible from client scripts.
  • Form Scratchpad: Client Scripts can now access data populated in the scratchpad by Business Rules.
  • Runbook PowerShell Activities for Workflows: Three new activities are added to Workflows by the ServiceNow Runbook Automation plugin that enable MID Servers to run commands on Windows machines using PowerShell.
  • Project Management Improvements: Project Management now incorporates Skills Management and Task Resources.
  • Cost Management Improvements: Cost Management now use Time Cards to calculate labor costs, add group rates to the group resources, and CI Rate Cards can use the Condition Builder to associate with conditions.
