1 Overview

Enhancements are available to customers in the quarterly release following the "Since" date for a given enhancement.

2 Summer 2008 release

2.1 Limit the number of History entries displayed

By default, a maximum of 1000 history entries will be displayed. You can customize this limit by navigating to System Properties > System, and then finding the property with the following description: "Maximum number of entries displayed in record history".

Summer 2008

3 Spring 2008 release

3.1 Add gauge to homepages

Add any gauge to your homepage!

Since Spring 2008

3.2 Default values for journal fields

Text entered in the default-value setting for a journal field now appears in the text area when creating a new record. Note that default values only apply for new records and don't get populated except for the creation of a new record. Because the text is populated into the form prior to submission it is possible to remove default text when desired

Since Spring 2008

3.3 Collapse a related list

Using a +/- icon, it is now possible to expand or collapse a related list on a record:

Since Spring 2008

3.4 Edit Application context menu

To edit an application and its modules, you can right-click the application's label in the navigation bar:

Since Spring 2008

3.5 Navigator Filter

Quickly find applications and modules you are working in with a keystroke command.  While your screen focus is in anywhere within a ServiceNow browser session you can press CTRL + F to access the left hand navigation bar's search filter.  Of course you can also click in the gray navigation bar header to access the search filter as well.

           


Since Spring 2008

3.6 Compact context menus

We have streamlined our list and form context menus:

Since Spring 2008

3.7 Workflow widget orientation toggle

You can click the [+] icon in a list workflow widget to toggle its display from horizontal to vertical. This allows for a more detailed display view of the Workflow status. Toggling the workflow of a record will toggle all records below it in the list as well.

Since Spring 2008

3.8 Knowledge Base article roles enforced on direct URL access

If a Knowledge Base article has roles on it, those roles are enforced when a user tries to access the article via direct URL link (e.g., http://www.service-now.com/demo/kb_view.do?sysparm_article=KB00001)

Since Spring 2008

4 Fall 2007 release

4.1 Menu Lists Sorted and Orderable

Customize the order in which Perspectives appear when a user switches menu perspectives. An "order" field has been added to the Menu List (sys_perspective) table for this purpose. If you give your Menu Lists numerical order values by navigating to System UI > Menu Lists, they will appear in the order defined. Menu Lists without an order will be displayed alphabetically. These changes make Menu List display consistent with that of Applications and Modules.


Since August 23, 2007

4.2 Smart Mandatory Field Enforcement

4.2.1 Next and Previous in Lists

The next and previous arrows will allow scrolling through the records from the list and will not require that mandatory data be provided if no fields on the form have been changed.

4.2.2 Attachments

A request to add an attachment to a record can be made and will not require that mandatory data be provided if no fields on the form have been changed.

Since August 23, 2007

4.3 Presentation of dependent choice lists in filters

The list of choices for a field that has a dependent choice list will be presented in alphabetical order. For example, the sub_category field is dependent on the category field. When choices for a sub_category are presented in a filter, the list will be alphabetical to make it easier to locate the sub_category of interest.

Since August 23, 2007

4.4 List select box behavior modification

If you select all items in a list using the Select All checkbox at the bottom, then subsequently uncheck an item in the list, the Select All box will be unchecked to reflect that not all items are selected.

Since August 23, 2007

4.5 Roles, glide_list, and watch list UI enhancement

The Roles, glide_list, and watch list UI has been enhanced to conserve display space. They are compressed when not in an editing state and expanded when being edited.

Since September 02, 2007

4.6 Increased import format options

Imports can be done against both Excel and CSV formats from the same UI, and we support many versions of the Excel spreadsheet format.

Since August 29, 2007

4.7 Filter UI enhancement

Filters have been enhanced to make them easier to use.

Since August 27, 2007

4.8 Basic Auth options

Any of our export processors can require basic authorization as required. A new property "glide.basicauth.enforced" has a value which is a comma-delimited list of processors enforcing basic auth. This property may be created in an instance if needed and not already present. The existing properties for requiring basic auth for SOAP, WSDL, and RSS still work the same.

For exporting, the following processors are available:

  • WSDL
  • SOAP
  • RSS
  • CSV
  • EXCEL
  • PDF
  • XML


Since August 31, 2007

4.9 no_attachment table attribute

The no_attachment attribute, when added to a table record, removes the attachment icon for that table's forms.

Since July 5, 2007

4.10 no_separation table attribute

The no_separation attribute, when added to a table record, disables company separation for that table, assuming company separation has been enabled for the instance. If added to an extended table, like "incident", separation will still be in effect on the parent table (e.g., "task"). So a user looking at a list of tasks would not see company separated incidents, even though the incident table has the "no_separation" attribute.

Since October 31, 2007

4.11 Internal type read-only unless user-created

The Internal type of a field is only editable for custom fields (where the Element begins with "u_")

Since September 4, 2007

4.12 IP Address Access Control usability enhancements

Field labels have been added to the IP Address Access Control plugin for discoverability. Client scripts to smooth usage are also included.

4.15 Default filters for slush buckets (Edit... button)

Personalize Control for a list will now allow the user to establish a default filter when the Edit... button is used to bring up a slush bucket for adding new entries to the list.

Since Oct 30, 2007

4.16 Dirty Form navigation support

 

4.17 Make a report public

You can make a report visible to anyone - without requiring a login. While looking at the report, click the Publish button which will make the report public and respond with the URL that can be used to access the report.
