1 Changes for Fall 2009 release

1.1 Discovery MID server SSH throughput improved 3X

Improvements come from two changes: first, the number of MID server threads needed by the SSH probes is much lower now; second, the MID server now caches and reuses SSH connections, instead of making a new connection on every SSH probe.

1.2 Discovery maximum configured threads increased at least 10x for Windows MID Server

Windows MID Server can now run with as many as 300 threads (or more), where previously 30 threads could generate "too many open files" exceptions.

1.3 SLA 'Maximum percentage' property added

A 'Maximum percentage' property has been added to the 'SLA Preferences' page. It serves to set the maximum 'Actual elapsed percentage' after which we'll stop regularly (default every two minutes) updating the SLA. This serves to cut down on unnecessary SLA updates, as SLAs that have long since breached don't continually need their time fields updated.

1.4 Notification Preferences page simplified

The 'Notification Preferences' page has been simplified, providing a simpler 'On/Off' toggle interface to manage notifications.

1.5 IP Address Access Control module moved

The IP Address Access Control module was moved from System Definition to System Security.

1.6 Popup view will respect tablenameGetViewName() function

The popup view of a form will now take into account any Global business rule functions, e.g. incidentGetViewName(), that specify which view a user should get.

1.7 Module to let admins run script

A new module, System Definition --> Scripts - Background, gives admins a text input where they can write and run arbitrary script to be executed on the instance.

1.8 Author field added to Knowledge articles

A new reference field, author, has been added to the kb_knowledge table.

1.9 Security checks for attachments

New security checks for attachments will prevent a user from modifying attachments to that record if they cannot modify the record. Security checks are also performed on attachment delete, which will check the delete role of the parent record.

1.10 Descriptions and help links added to many Plugin activation dialogs

1.11 journal_list and journal_input fields removed from filters

To be consistent with journal fields, journal_list and journal_input fields have been removed from filters. To search on these fields, use a Keyword or "For text" search.

1.12 Field deletion deletes related Client Scripts and UI Policy Actions

When a field is deleted, onChange() Client Scripts for that field and UI Policy Actions acting on that field are also deleted with a business rule on sys_dictionary.

1.13 POP3 SSL port override

The default POP3 SSL port can be overridden by the glide.pop3.secure.port property.

1.14 save_as_template ACL operation

If "save_as_template" operation row-level ACLs return false for a record, the "Save as Template" and "Save All as Template" context menu options will not appear

1.15 translated_text fields in filters get additional string operators

1.16 timer fields in filters get additional date/time operators

1.17 Personalizing provides additional information

When personalizing lists, forms, related lists, etc., additional information at the top of the slushbucket form now specifies what is being personalized, e.g. "Personalizing Change Request list" or "Personalizing Incidents related list on Problem form".

1.18 sys_security_acl and sys_security_acl_role tables are now audited

1.19 Knowledge Management with KCS plugin changes

  • User comments/feedback is now displayed when viewing the knowledge article. In Knowledge --> Properties, a property specifies whether comments are shown onload, onclick of a "show comments" link, or never. A property specifies what roles are required to see comments on articles. Default is no roles, so all authenticated users will see comments. A property specifies the max number of comments to show, reverse sorted by date.
  • Average star rating is displayed when viewing an article.
  • After giving a star rating, users may change their mind and re-rate the article without reloading it.

1.20 System UI modules given order values so they sort the same for MySQL and Oracle

1.21 no_truncate attribute added to the question field for Surveys, so viewing responses is easier

1.22 favicon added, can be overridden in System Properties --> System

1.23 Reference fields displayed as a dropdown

Reference fields can be displayed as a dropdown. Designed behavior is for those fields to default back to the standard referenced lookup UI if the dropdown would have more than 25 entries. A bug prevented this from happening. The bug has been fixed which may result in a behavior change for customers displaying a reference field with more than 25 entries as a dropdown.

1.24 Number of records returned with ?XML unloads limited

How records are returned with ?XML unloads is controlled by the following properties that can be created if they do not exist:

  • com.glide.processors.XMLProcessor.stream: specifies whether response is streamed if no sysparm_stream is specified, default is true
  • com.glide.processors.XMLProcessor.record_count: number of records returned if no sysparm_record_count is specified, default is 10000
  • com.glide.processors.XMLProcessor.max_record_count: max number of records that can be requested with sysparm_record_count, default is 50000

1.25 Discovery

  • Add support for discovering Vista in discovery and wmi scripts (OS names, product ID and etc).
  • Add name finders for computers, printers, netgears and UPSs available out of the box (defaults to inactive) in conslidation.
  • Add support for collecting active running processes for HPUX (if a patch is installed on HPUX).
  • Add support for running SSH and HTTP in different ports.
  • Add support for rounding the values of CPU and RAM.

2 Changes for Spring 2009 Stable 3

2.1 Properties to Enable or Disable SMTP and POP3

Enables or disables SMTP or POP3 without throwing warning messages in the log.

2.2 BSM views can be role restricted

BSM views can now be restricted by user role.

2.3 Enhanced BSM views

BSM views have been enhanced to include

  • a list of specific CI types
  • filters on CI tables
  • a list of specific relationship types

2.4 BSM map node limit

The total nodes displayed on a BSM map is now limited based on the property glide.bsm.max_nodes, defaulting to 1000.

2.5 BSM map collapsed nodes

Collapsed BSM nodes now expand in sets of a limited size based on the property glide.bsm.max_to_expand, defaulting to 100.

2.6 Report page live filter

On the Report portal, the search box at the top of the page filters automatically as text is typed, displaying only reports that match the typed text.

2.7 InfoMessage and ErrorMessage blocks can be closed

InfoMessage and ErrorMessage blocks at the top of the page can now be closed by clicking the X in the upper right corner.

2.8 Report type tooltip

On the Report portal, a report's type will be displayed as a tooltip when hovering over the report name.

2.9 Edit Homepage UI Action

An "Edit Homepage" UI Action has been added to Homepage records to facilitate easy editing.

2.10 Attachment icon for Knowledge articles

If a custom icon is not specified for an article, and the article links directly to the attachment from the knowledge portal, an attachment icon is displayed in the portal for that article.

2.11 Attachment name for Knowledge articles

If an article links directly to an attachment from the knowledge portal, the attachment filename is provided as a tooltip when hovering over the article link.

2.12 Article preview popup

On the knowledge portal, hovering over the article's icon will display a preview popup of the article. This behavior is customizable with a property (Knowledge Base --> Properties).

2.13 Once option when scheduling work

When scheduling an automated task (System Definition --> Scheduled Jobs), a "Once" option has been added to the Run field to allow for a single execution of a script, import, scheduled report, etc.

2.14 Incident reopen business rule change

The "Incident reopen" business rule's logic has been changed. Previously, when when incident_state value was less than 7, the Active field was set to true. This did not accommodate custom incident states created by customers. Now, when the incident_state value is not equal to 7, the Active field is set to true.

2.15 Indexed attachments

Attachments can now be text indexed. By default, attachments to the kb_knowledge table are text indexed for searches.

2.16 CMS link to full frame set

For admin users, a link has been added to the login widget in Content Management to switch to the full frame set from the CMS site.

2.17 Email properties to enable/disable SMTP/POP

Two email properties have been added (System Properties --> Email) to enable/disable SMTP and POP email. Previously, the only way to disable email was to remove authentication information.

2.18 Go to feature enhancement

When using the "Go to" input at the top of a list, prefixing text with an asterisk (*) will do a CONTAINS query on that text for the selected field. e.g., Doing a Go to on Short description with *network as the input will return records where the Short description contains the word "network". Default behavior with the asterisk is still to sort the list on that field and navigate to the position in the list with records where selected field starts with the input value.

  • %searchterm% will similarly add a CONTAINS filter for "searchterm"
  • %searchterm will add an ENDSWITH filter for "searchterm"
  • searchterm% will add a STARTSWITH filter for "searchterm"

3 Changes for Spring 2009 Stable 2

3.1 Domain visibility performance improvement

Performance was improved when determining domain visibility for a user.

3.2 Property to prevent record updates when using blue arrows

A property, glide.ui.update_on_iterate, can be created and set to false to prevent records from being updated when the blue navigation arrows are clicked.

3.3 Property to specify width of main content frame

A property, css.nav.menu.width.right, can be created to specify the width of the main content frame. Value should be a valid CSS for width (e.g., 82% or 980px).

3.4 The Database Images plugin is installed

All instances receive the Database Images plugin to store images in the database.

3.5 Department dropdown not visible by default for Knowledge Management Internationalization plugin

The Department dropdown can be set visible by the glide.knowman.search.show_department property with the following description: 

Display department input in Knowledge search header

3.6 Condition and Roles fields added to kb_navons table

Navigation Add-ons can now be restricted by conditions and roles, to control when they are displayed.

3.7 Navigation Add-ons moved to Advanced Search options

Knowledge Base Navigation Add-ons have been moved to the Advanced Search options of the Knowledge search header.

3.8 report_home.do converted to a UI Page

The report portal, report_home.do, has been converted to a UI Page and can thus be secured with an ACL.

3.9 Property to specify valid attachment file types

The glide.attachment.extensions property specifies valid attachment file types. If empty, all file types can be uploaded as attachments.

3.10 Property to allow extended fields in base lists

The glide.ui.list.allow_extended_fields property allows fields from extended tables to be added to (and filtered on) base table lists. e.g., it will allow the caller_id field from the incident table to be added to a task list, like the My Work list. 

Allow base table lists (task, cmdb_ci, etc.) to include extended table fields (incident_state, os_version, etc.), and allow filtering on extended table fields

3.11 Table cleaner entry added for ecc_agent_metrics table

A table cleaner entry added for ecc_agent_metrics table, with a default value of 30 days.

3.12 Property to allow URL images to be added to an HTML field

The glide.ui.html.image.allow_url property, with a default value of false, specifies whether the URL option appears when adding an image to an HTML field. 

Show URL option for HTML field images

3.13 Improved look of Project Management Gantt charts

As part of an ongoing effort to improve the look and feel of Project Management Gantt charts, Gantt chart labels were moved from the left side of the chart to under each bar, resulting in less horizontal scrolling.

4 Changes for Spring 2009 Stable 1

4.1 Associating a task with a survey

A task reference field has been added to the survey_instance table. When you build the survey URL you can add &sysparm_survey_task=task_sys_id to populate this field. You will then be able to report on survey responses based on task assignment, categories, or any other task related information. Survey functionality is now more field customizable using the SurveyProcessor Script Include and UI Pages for survey_take.do and survey_thanks.do.

4.2 Font Standardization

System fonts have been standardized throughout the instance to facilitate personal customization. Fonts in some areas that previously had hardcoded font sizes or font families may appear different due to this standardization.

4.3 Ongoing feedback on large catalog requests

When a catalog request is copied for multiple additional users, a progress indicator will provide updates as the catalog request is copied. At completion, the progress indicator will present options to return to the catalog request (and copy it again if desired), return to the catalog home page, or view the newly created requests.

4.4 Manager not auto-mapped in LDAP

When importing users from LDAP, the manager field is no longer automatically mapped when using the Mapping Assist capability. Manager information is handled in a separate part of the LDAP import, so the automatic mapping was unnecessary.

4.5 Plugins for newly created instances

View Definition and Metrics plugins are now installed automatically for newly created instances.

4.6 Refresh homepages

All users can use the "Refresh" functionality on homepages.

4.7 PDF output contains aggregations

PDF exports now include "Personalize Calculations" aggregations.

4.8 Cascade delete for SLAs

SLAs are now cascade deleted when their associated tasks are deleted.

4.9 Application and Module UI Actions restricted

Activate and Deactivate UI Actions for sys_app_application and sys_app_module tables are restricted to the admin role.

4.10 Mandatory derived field behavior

Previous behavior of mandatory derived fields was restored. They cannot be cleared out, but if they start out empty, they can remain empty.

4.11 Unavailable attachment message

If a user is not logged in, but tries to view an attachment, they now get an informative message.

4.12 Image picker UI cleanup

The image picker UI (e.g., when selecting an icon for a module) was improved.

4.13 AJAX scripts converted to Server Script Includes

Many AJAX Scripts (in System Definition) were converted to Server Script Includes.

4.14 UI Pages can be secured with ACLs

A new ACL type of "ui_page" was added. An ACL can be created with a Type of "ui_page", an Operation of "read", and a Name value equal to the name of a UI Page, to restrict access to a specific UI Page. The "governor" UI Page has been restricted to admin users.

4.15 Length of chart URLs reduced

Chart URL length has been dramatically reduced to accommodate IE's "get" string length restriction.

4.16 New ACLs in Security Jump Start plugin

New ACLs have been added to the Security Jump Start plugin to restrict system tables out-of-box.

4.17 Script input for admins

Admins can now use the script input in the sys.scripts.do page, and access uploaded Customer Scripts from that page.

4.18 Manager can see groups' Search Groups

If a user is a group manager (but not a member of the group), that group's Search Groups are now available.

4.19 Knowledge searches tracked

Knowledge Base searches are now tracked in the ts_query_kb table.

4.20 Custom charts can be emailed

Custom charts can now be emailed as scheduled reports.

4.21 All text search results tracked

All text search queries from all tables are tracked in the text_search table.

4.22 Brazilian-Portuguese i18n plugin

The Brazilian-Portuguese language plugin is not currently admin-installable.

4.23 Report button titles

Descriptive tooltips have been added to Report buttons.

4.24 Preview HTML Body UI Action

A new UI Action, "Preview HTML Body", has been added to the sys_email table.

4.25 Language picker added to login page

A language picker has been added to the login page so users can select a language before logging in.

4.26 View Article UI Action

A new UI Action, "View Article", has been added to the kb_knowledge table.

4.27 M2M table between Knowledge and Task

A new table, m2m_kb_task, has been added to relate articles to tasks in a many-to-many relationship.

4.28 Default filter on Event Log and Mailbox modules

A default filter has been added to Event Log and Mailbox modules to show only records created today.

4.29 System Localization Properties module

A new module, System Localization, has been added to the System Properties application.

4.30 CSS property for navigation frame width

A new property, css.nav.menu.width, can be created to specify the width of the navigation frame. Default value is 18% if the property doesn't exist in an instance.

4.31 Conditions and Roles fields added to Knowledge Add-ons

Knowledge Add-ons are used to add additional search targets and Knowledge Base links. They now have Conditions and Roles fields to flexibly restrict when and for whom they appear.

4.32 Import Sets "Run Transform" can run multiple maps in order

When running a transform on an import set, multiple transform maps can now be run in order. A slushbucket is shown where multiple maps can be specified and ordered.

4.33 Domain hierarchy loop detection

If a domain hierarchy loop is inadvertently set up (e.g., domain A is the parent of domain B which is the parent of domain A), the loop will be detected when building the domain display value.

4.34 Emails marked processed when processing skipped

When skipping email processing because the condition for the inbound email action doesn't match, the email is now marked as processed to prevent confusion.

4.35 max_length field mandatory on Dictionary form

The max_length field is now mandatory for sys_dictionary records.
