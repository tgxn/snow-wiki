 Functionality described here requires the Sendmail Support plugin.
Note: The following integration is not part of the supported platform. The integration may require additional customization, typically performed by an experienced administrator or consultant, prior to operating correctly.


Note: This functionality is intended for the very rare case where a POP server configuration is not possible. If you are not sure whether you need this functionality, you likely do not. It currently only supports plain text emails and does not support attachments.

1 Overview

Sendmail support lets you push email directly to ServiceNow (without the need for a POP server configuration) via SOAP into the ECC Queue, where they are processed and converted into sys_email records that Inbound Email Actions can act upon as usual. This functionality is intended for the very rare case where a POP server configuration is not possible.

2 What you need

  • UNIX, Linux, etc. server
  • Sendmail running on the server
  • Local mailbox on the server
  • Ability to run perl on the server
  • .forward configuration file
  • email_create.pl perl file

2.1 Perl Requirements

    • SOAP::Lite (prerequisites http://soaplite.com/prereqs.html)
    • XML::Writer
    • XML::Writer::String
    • Crypt::SSLeay
    • IO::Socket::SSL

Since the script may be doing an HTTPS POST, you will also need port 443 or 80 (HTTP) access from your client to your ServiceNow instance.

2.2 Installing the Perl modules

To install Perl modules, you can specify the following command line (you would want to run this for each module) 

perl -MCPAN -e 'install SOAP::Lite'

The command above will install the SOAP::Lite package from CPAN. This is an interactive installation process, when prompted for input, just accept the suggested or default response.

3 System Mailboxes

The Sendmail Support Plugin creates modules in the System Mailboxes application.

4 .forward file contents

Include the quotes. Replace filepath with the right one for your server, and the perl script name with the script name you're using. 

 "| /Users/davidloo/email_create.pl"
== email_create.pl perl script contents ==

Put the script in the folder specified in the .forward file. 

  #!/usr/bin/perl
  #
  use strict;
  use SOAP::Lite;
  use XML::Writer;
  use XML::Writer::String;
  
  my $SNC_HOST = "https://instance_name.service-now.com";
  
  email_create();
  
  sub email_create {
    my $soap = SOAP::Lite->proxy("$SNC_HOST/ecc_queue.do?SOAP");
    my $method = SOAP::Data->name('insert')->attr({xmlns =>   'http://www.service-now.com/'});  
  
    # set ECC Queue Agent so that the InboundEmail processor will be
    # executed
    my @params = ( SOAP::Data->name(agent => 'InboundEmail') );
    push(@params, SOAP::Data->name(queue => 'input') );
    push(@params, SOAP::Data->name(name => 'Inbound Email Processing') );
  
    # read stdin line by line into the email variable
    my $email;
    my $lcount = 0;
    while (<STDIN>) {
      if ($lcount == 0) {
        push(@params, SOAP::Data->name(source => $_) );
      } else {
        $email .= $_;
      }
      $lcount++;
    }
  
    # set the entire content of the email into the payload field
    push(@params, SOAP::Data->name(payload => $email) );
  
    # invoke the SOAP request
    my $result = $soap->call($method => @params);
  
    # capture the errors that occur and echo to stdout
    if ($result->fault) {
      print "faultcode=" . $result->fault->{'faultcode'} . "\n";
      print "faultstring=" . $result->fault->{'faultstring'} . "\n";
      print "detail=" . $result->fault->{'detail'} . "\n";
    }
  
    # capture the response and echo it to stdout
    if ($result->body && $result->body->{'insertResponse'}) {
      my %keyHash = %{ $result->body->{'insertResponse'} };
      foreach my $k (keys %keyHash) {
          print "name=$k   value=$keyHash{$k}\n"; 
      }
    }
  }

  # add the following subroutine if basic authentication for SOAP is enabled
  # enter the username, password that you will use
  sub SOAP::Transport::HTTP::Client::get_basic_credentials {
   return 'username' => 'password';
  }
