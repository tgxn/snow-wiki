1 Overview

Condition Check Definitions for Software License Management enable an administrator to define logical conditions based on the value of a specific field in the current software license record, and when the conditions match, an event is generated to drive notifications. Use the condition checking feature to evaluate license compliance and expiration levels. These conditions are evaluated daily by a scheduled job called Software License Compliance Check. This functionality requires the Software License Management plugin and is available in Aspen and earlier releases of ServiceNow.

Note: A significantly improved and updated application named Software Asset Management was included in the Berlin release. For information about upgrading from Software License Management to Software Asset Management, see Upgrading to Software Asset Management.


2 Defining Condition Checks

  1. Navigate to Software License Management > Condition Check Definitions.
  2. Configure the list to display the Condition field field.
    This helps you identify the field to which a condition is being applied.


  3. Click New in the Condition Checks list.
  4. Complete the form using the following fields.
    Refer to the table below for a list of the software license tables and fields to select.
    • Table: Select a table to which this condition applies. For example, Usage Software License [slm_license_usage] or Complex Software License [slm_license_complex].
    • Condition field: Select the appropriate field for the table: Expiration level or Compliance level. The other fields in the list are not used for condition checking.
    • Event name: Create an event using this syntax:
    <table_name>.<condition_field>
    • Category: The category for Software License Management defaults to License. Do not change this setting This form is used also for Asset Contract condition checks, which require a category of Asset.
  5. Click Submit.
    The Conditions Related List appears, enabling you to assign logical conditions to the Condition field field that trigger an event.
  6. Click New in the Conditions Related List.
  7. Complete the form using the following fields:
    • Condition check: The name of the table. This field is read-only.
    • Table: The name of the table. This field is read-only.
    • Name: Value that the field displays when the condition is met.
    • Order: Assign an order to the condition that determines the order in which the conditions are evaluated.
    • Event name: Name an event triggered by this condition. Use this event to perform tasks such as sending an email notification. This event must be registered before it will function.
    • Expiration Condition: Create the condition that populates this value. The name of the condition reflects the Condition field selected for this record, such as Expiration Condition. These conditions are evaluated in order until the a condition is met.
