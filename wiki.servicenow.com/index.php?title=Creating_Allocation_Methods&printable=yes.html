1 Overview

Allocation methods tell the allocation rule how to calculate the breakdown of the expense. Methods also specify which segment and account should receive the allocation. Create as many allocation methods as necessary until all of the expenses that match the rule are fully allocated. Methods can only be associated with one rule. If you delete the rule, associated methods are also deleted.

Note: When you use the workbench to assign expenses to accounts and segments, it creates all the rules, methods, conditions, and so on automatically. Using the workbench is the preferred method of setting up allocations, rather than using lists and forms.


2 Creating Cost Allocation Methods

  1. Open an allocation rule.
  2. In the Cost Allocation Methods related list, click New.
  3. Fill out the fields on the form as appropriate (see table).
  4. Click Submit.
An example allocation method

Field Description
Number Auto-generated identification number for the for the cost allocation method.
Allocation rule The allocation rule to which this method applies.
Allocation group The cost allocation group associated with this method.
Allocation metric The cost allocation metrics that this method uses. Click the lookup icon to select the metric. Each metric is associated with an allocation group.

If the Allocation group field already has a value, the metrics available for selection are restricted to those that are associated with the allocation group. If the Allocation group field does not have a value, you can select any metric, which then populates the allocation group that is associated with that metric. See cost allocation metrics form field descriptions for more information on metrics.

Allocation type The way the expense amount is broken down. Choose one of the following:
  • Percentage: Break down the allocation by percentage. The percentage amounts for all methods used by this rule must add up to 100.
  • Fixed: Break down the allocation by a fixed amount.
Amount The percentage or the fixed amount that applies to this method. If you enter a value that is more than the actual amount in the general ledger record, the system allocates the full general ledger amount to the current method.
Unallocated If the specified amount is allocated to segments or left unallocated. This option is not available if the method uses a metric.
Primary segment The segment from the IT Chart of Accounts to which the final expense applies. Select the segment from the choice list. If an allocation metric is selected, this field is hidden because the allocation metric determines the primary segment.
Order Numeric value that determines the order of the method, relative to other methods. This field appears only if you select the fixed allocation type. Methods are processed in order from the lowest order number to the highest. When you create a new method, the default order value is 10 points higher than the maximum order value of the existing methods for this allocation rule. The order value must be unique. No other cost allocation rule on the same table can have the same order value.
Split bucket The split bucket that the method applies to. You can split buckets on the workbench in the Allocation Setup stage.
[Dimensions] The segments that are defined in the IT chart of accounts appear on the right-side of the form. Click the lookup icon next for each segment and select the relevant record from the dimension table. You must select a record for the Primary segment. If there are no records in the dimension table, you must populate the dimension table.
