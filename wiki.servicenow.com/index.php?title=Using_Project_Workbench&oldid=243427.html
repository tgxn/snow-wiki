1 Overview

The project workbench allows project managers to manage all aspects of a project:

  • Create new projects or manage existing projects
  • Create and update project phases and milestones
  • Assign a team to an agile phase
  • Create and update project tasks, stories, and test cases
  • Attach test plans to a test phase
  • View and manage project details from a list view or visual task board

2 Accessing the Project Workbench

The project manager can access the project workbench in one of the following ways:

  • Click the Project Workbench related link on the Project form.
  • Navigate to Project > Projects > Workbench.
    Initially, the project workbench opens to a blank page. From this blank page, you can open an existing project or create a new project. If a project was already open in the project workbench in the current user session, it opens to this last selected project.
 Note: If the project workbench opens with no associated sys_id or an incorrect sys_id, an error message appears. Click Select Project in the project workbench header and then select a project from the choice list.


2.1 Opening a Project in the Project Workbench

  1. In the project workbench header, click Select Project.
  2. Select a project from the choice list.
    The selected project opens in the project workbench.

2.2 Creating a New Project from the Project Workbench

To create a project from a project template, see Creating a Project from a Template.

To create a project from project workbench without using a template:

  1. In the project workbench header, click New Project.
  2. Fill out the fields on the Project form.
    For more information, see Creating a Project Manually.
  3. Click one of the following buttons:
    • Create Project: creates the project and opens it in the project workbench.
    • Start Project: creates the project, opens it in the project workbench, and sets the project state to Work in progress.

3 Using the Header

Use the project workbench header to:

  • View project details
  • View live feed
  • View collaboration
  • Add phases to the timeline
  • Manage stories
  • Apply a template
  • Create a new project

3.1 Viewing Project Details

View project details in one of the following ways:

  • Click the back icon () to go to the Project form.
  • Click the project detail icon () to display project information in a pop-up window.

3.2 Viewing Live Feed

Display the project live feed stream in a pop-up window by clicking the live feed icon (). Live feed shows task state changes and any messages that users post about the project.

3.3 Viewing Collaboration

If Collaboration is active, display the project collaboration overlay by clicking the collaboration icon (). This overlay displays recent conversations about the project with people, groups, and records. These conversations appear under the respective sections of the overlay sidebar.

3.4 Managing Project Stories

The Manage Stories button appears only if Project Portfolio Suite is activated. Use this button to manage the stories for this project.

  1. In the project workbench header, click Manage Stories.
    A pop-up window lists the stories included in the project.
  2. Click a story to open the Story form in the pop-up window, or click New to create a new story.
  3. Fill in the fields on the Story form, as appropriate.
  4. Click Update or Submit.

For more information, see Managing Stories.

3.5 Applying a Template

The project manager can apply a template to a new project.

  1. In the project workbench header, click New Project.
  2. Enter a name in the Project Name field.
  3. Click Create Project to open the new project in the project workbench.
  4. Click Apply Template in the header.
  5. Enter a date in the Start date field.
  6. Select a template in the Project Template field.
  7. Click Apply.

For more information, see Using Project Templates.

4 Using the Timeline

The project manager can create and edit phases and milestones in the timeline. If Project Portfolio Suite is activated, the project manager can create these phases:

  • Waterfall: By adding a waterfall phase to the timeline, the project manager can manage project tasks and subtasks. All of the project task information is displayed below the timeline in the list view or visual task board (VTB).
  • Agile: By adding an agile phase to the timeline, the project manager can see all of the stories included in that phase. The story information is displayed below the timeline in the list view or VTB. Link the stories to the project by populating the Project field in the story record. One agile phase can be added to each project. In addition to the planned start and end dates for the phase, the project manager can also assign a team to an agile phase.
  • Test: The project manager can also add a test phase to the timeline. In addition to the planned start and end dates for the phase, a test plan is also assigned. The timeline gets updated with the new test phase, and the test cases associated with the test plan are displayed below the timeline in the list view or VTB. This view is automatically updated with any new test cases added to the test plan. As testing progresses, both the list view or VTB and the phase in the timeline are updated with the testing details. One test phase can be added to each project.

4.1 Creating a Phase in the Timeline

  1. In the project workbench header, click the add phase icon ().
  2. If necessary, click the Phase button at the top of the Add Phase pop-up window.
  3. Select the Phase Type to create: Waterfall, Agile, or Test.
    One agile phase and one test phase can be added to a project. If these phases have already been created, the Agile and Test options are disabled.
  4. Fill in the fields, as appropriate (see table).
  5. Click Submit.
    The project workbench is updated, and the newly created phase is displayed in the timeline.
Field Description
Short description The name of the phase. This name is displayed above the phase in the timeline.
Planned start date The intended start date for the phase. This field does not show minutes and seconds.
Planned end date The intended end date for the phase. This field does not show minutes and seconds.
Planned duration The expected duration of the phase. When the Planned start date and Planned end date fields are filled in, the value for this field is automatically calculated. This field does not show minutes and seconds.
Team The team assigned to an agile phase.
Start Sprint The first sprint selected for the agile phase. This field appears only after a Team has been selected. After you select a sprint, the Planned start date field is updated with this sprint start date.
End Sprint The last sprint selected for the agile phase. This field only appears after a Team has been selected. After you select a sprint, the Planned end date field is updated with this sprint end date.
Test Plan The name of the test plan associated with a test phase. Specifying a test plan is optional when creating a test phase. However, before you add test cases to a test phase, you must specify a test plan.

4.2 Editing a Phase in the Timeline

A project manager can edit a phase from the project workbench timeline using the inline form. After the phase has been edited, it is updated in the timeline and the phase information is updated in the list view and visual task board.

To edit a phase:

  1. In the project workbench timeline, click the edit phase icon () beside the phase name.
    The inline form opens and displays the phase information.
  2. Make any necessary changes to the fields on the inline form.
    You cannot change the Phase type.
  3. Click Update.

4.3 Deleting a Phase from the Timeline

A project manager can delete a phase from the project workbench timeline using the inline form. After the phase has been deleted, the project percentage complete field is updated and the phase is removed from the timeline. Depending on the phase type, all of the associated project tasks, stories, and test cases are also removed from the list view and visual task board.

To delete a phase:

  1. In the project workbench timeline, click the edit phase icon () beside the phase name.
    The inline form opens and displays the information for the selected phase.
  2. Click Delete.
  3. Click Delete again to confirm.

4.4 Assigning a Team to an Agile Phase

A project manager can assign a development team to an agile phase or update the currently assigned team. Assigning a team updates the project and all associated stories. This action sets the Team field in the following tables:

  • Project [pm_project] in the Project application
  • Story [rm_story] in the SDLC application

Changing the assigned team updates all associated stories which are not set to Complete or Cancelled. For stories with assigned sprints, changing the assigned team clears the Team, Sprint, and Assigned to fields.

Note: A project can have one team assignment but a team can be assigned to multiple projects.


To assign a team to an agile phase:

  1. In the project workbench timeline, click the edit phase icon ().
  2. Click the reference icon beside the Team field to display a list of available teams.
  3. Click the desired team.
  4. Click Update.
    The selected team is added to the phase and the team name is added to the project in the timeline. All of the stories in the project are also assigned to the selected team.

4.5 Selecting Sprints for an Agile Phase

A project manager can select sprints for an agile phase after assigning a team. All current sprints belonging to the assigned team are available for selection.

When the Team field is populated, the Start Sprint and End Sprint fields appear. Selecting sprints in these fields automatically updates the Planned start date and Planned end date fields with the start date from the Start Sprint and the end date from the End Sprint. After selecting sprints, the Planned start date and Planned end date fields become read-only.

Any change to the sprint dates is not reflected in the phase dates.

Note: Sprint selection is optional for creating an agile phase.


To select sprints for an agile phase:

  1. In the project workbench timeline, click the edit phase icon ().
  2. Ensure that a team has been selected in the Team field.
  3. Select a Start Sprint.
  4. Select an End Sprint.
    If you select a start sprint, you must select an end sprint.
    The Planned start date, Planned end date, and Planned duration fields are automatically updated.
  5. Click Submit.

4.6 Assigning a Test Plan to a Test Phase

A project manager can assign a test plan to a test phase or update the currently assigned test plan.

  1. In the project workbench timeline, click the edit phase icon ().
  2. Click the reference icon beside the Test Plan field to display a list of available plans.
  3. Click the desired plan.
  4. Click Update.

4.7 Creating a Milestone in the Timeline

To create a milestone in the project workbench timeline:

  1. In the project workbench timeline, click the add phase icon () in the upper right corner.
  2. Click the Milestone button in the pop-up window.
  3. Fill in the fields, as appropriate (see table).
  4. Click Submit.
    The project workbench is updated and the newly created milestone is displayed in the timeline.

For more information, see Milestones.

4.8 Editing a Milestone in the Timeline

  1. In the project workbench timeline, click a milestone to open the inline form.
    Dragging a milestone along the milestone timeline also opens the inline form for editing.
  2. Make any necessary changes to the milestone fields.
  3. Click Update.

5 Using the Detail View

The detail area at the bottom of the project workbench displays additional information based on the phase selected in the timeline.

  • Waterfall: displays the project tasks associated with the phase.
  • Agile: displays the stories assigned to the phase.
  • Test: displays the test cases in the test plan assigned to the phase.

This information can be displayed in either list view or a visual task board. The List and VTB buttons in the project workbench header control how this information is displayed.

5.1 List View

Click a phase in the timeline to display the phase details in a list view. The type of phase you click determines what can be created or edited:

  • Waterfall: Project tasks
  • Agile:' Stories
  • Test: Test cases

5.1.1 Creating a Project Task

  1. In the project workbench timeline, click a waterfall phase.
  2. Click New in the detail view.
  3. Fill out the Project Task form.
    For more information, see Creating Tasks.
  4. Click Submit.

5.1.2 Creating a New Story

  1. In the project workbench timeline, click a an agile phase.
  2. Click New in the detail view.
  3. Fill out the Story form.
    For more information, see Creating a Story.
  4. Click Submit.

5.1.3 Creating a New Test Case

A project manager can add a new test case to a test phase.

  1. In the project workbench timeline, click a test phase.
  2. Click New in the detail view
  3. Fill out the Test Case form.
    For more information, see Creating a New Test Case.
  4. Click Submit.
 Note: Specifying a test plan is optional when creating a test phase. However, before you add test cases to a test phase, you must specify a test plan.


5.1.4 Project Task Form Views

A view defines the elements that appear when a user opens a form or a list. The following views are available for the project task form and display the fields related to the specific project phase or milestone.

  • Waterfall
  • Milestone
  • Agile
  • Test

To select a project task form view, open a project task, right-click the form header and select View > [Desired View].

5.2 Visual Task Board

Click on a phase in the timeline to display the phase details in a visual task board. The type of phase you click determines what cards are displayed in the task board.

  • Waterfall: Project tasks
  • Agile: Stories
  • Test: Test cases

The project manager can add new cards to the board or edit existing cards.
