Specifying a Login Landing Page

From ServiceNow Wiki

Jump to: navigation, search
 Note: This article applies to Fuji and earlier releases. For more current information, see Specify a Login Landing Page at http://docs.servicenow.com

The ServiceNow Wiki is no longer being updated. Visit http://docs.servicenow.com for the latest product documentation.


Security

1 Overview

By default, ServiceNow users see their homepage upon login. Administrators can specify a different login landing page by using a system property or the content management system.

2 Using a System Property to Specify the Login Landing Page

To specify a login landing page for all users, change the property value on the sys_properties table:

  1. Type sys_properties.list in the navigation filter.
  2. Locate the glide.login.home system property.
  3. In the Value field, enter the name of the page that all users will see upon login. Use <page name>.do; you may omit the http://''instance''.service-now.com/ portion of the URL. To determine the page name or the URL of a page in the system, you can point to a link. Some possible pages are:
    • welcome.do - the introductory Welcome page
    • incident.do - the new incident form
 Note: This property is system-wide, so setting it affects all users.


3 Using Content Management to Specify the Login Landing Page

Administrators can also specify a login landing page with the content management system. Use the login dialog or login rules (legacy).
