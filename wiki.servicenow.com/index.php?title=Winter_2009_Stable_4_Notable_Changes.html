1 glide.ui.secure_cookies property

A glide.ui.secure_cookies property has been added to all instances, with a default value of true. If true:

  • Unused browser cookies expire after 30 days, requiring a user to reauthenticate if they have not logged in for 30 days.
  • An additional token is used in browser cookies and validated against the database to provide more strict user validation.

The property can be found by navigating to System Properties > Security. It will also appear in the UI Properties page. The property description is, "Enable additional cookie security. If checked, an additional token is used for more strict validation, and unused cookies expire after 30 days."

2 Context menu UI

The look and feel of context menus has been improved.

3 URL field link and label

(This Winter 2009 Stable 4 change was reverted for Spring 2009 due to issues found during testing, and it will likely be restored in a future stable build or release)

URL fields have been enhanced to allow for both a URL and a text label. When expanded on a form, the link and label can be entered separately. In a list or when collapsed on a form, the label is displayed. If no label is provided, the URL is displayed.

4 256-bit AES encryption for fields/attachments

Encryption support has been enhanced to use AES 256-bit encryption on instances that can support it. If supported, the option is available in the Type choice list on the Encryption Context form.

5 Hardcoded wiki links changed

In places where the wiki organizational structure has been modified, wiki links in ServiceNow have been modified to access the renamed articles or an appropriate portal page.

6 Tree picker node limit

A limit of 1000 has been placed on the number of nodes returned to the tree picker. This limit is configurable with the glide.ui.group_heirarchy.max_nodes property.

7 Impacted Services related list

A new related list, Impacted Services, has been added to the task table. It is a many-to-many relationship between task and cmdb_ci_service, displaying affected business services. The related list can be added to a task form (Change, Problem, etc.). It may be populated manually if desired, and a new form context menu UI Action labeled "Refresh Impacted Services" will auto-populate the list based on the Configuration Item on the task record's form. The UI Action will not remove manually-added services from the related list, but will add business services related to the CI.

8 Module form UI Policy

Module form (sys_app_module) client scripts have been converted to UI Policy to hide/display fields or make them mandatory as appropriate.

9 Text index checkbox on table dictionary records

The "Create or Remove Text Index" business rule has been modified to automatically and quickly set the text_index checkbox on related dictionary records as appropriate based on a change to a table's text_index checkbox. For example, if the text_index checkbox is checked for the Incident table's dictionary record, it is automatically checked for all other tables that extend task.

10 Notification Preferences UI

An improved Notification Preferences UI, accessible from a form link on the My Profile form, is available for instances using the Subscription Based Notifications plugin. Preferences are easily configured for each notification a user receives.

11 Field labels and splits for mobile lists

Field labels can now be added to mobile lists, and multiple fields can be displayed on one line by using splits when personalizing the Mobile view of a list.

12 New "sys_email" global variable available to Inbound Email Actions

In an Inbound Email Action, a new global variable "sys_email" is now available. It is the received sys_email record that triggered the Inbound Email Action. It can be used to reference fields on the email record, like uid, sys_id, content_type, etc.

13 AJAX calls reduced

The number of AJAX calls from the browser to the server has been reduced, improving performance.

14 Knowledge attachments display

Display Attachments option added to knowledge articles to enable the display of attachment links on the article view (kb_view) without having to use the Edit Article button.

15 List Collector variables in Client Scripts

It is now possible to add more than one list collector to a form. To accommodate this, the name of the global "g_filter" variable used to set the list collector filter in client script has been changed to be unique per variable. The new global variable used in client scripts is the name of the variable followed by "g_filter". For example, with the "Schedule a Move" catalog item, the name of the list collector variable is "items_to_move_", so to access it in client script the following must be used: "items_to_move_g_filter".

Impact: any existing catalog client scripts that reference a list collector filter with "g_filter" will need to be changed to match the new global variable naming convention.

16 Discovery

16.1 Windows services probe and sensor available now

The probe and sensor pair for discovering windows services are now available out of the box. It could be enabled through editing the windows classification.

16.2 Custom related list creation

An API function has been created for populating data in a custom related list.

16.3 CI hostnames can be stored in uppercase

A discovery property has been created for allowing hostnames to be stored in uppercase.

16.4 CMDB coalesce can be turned on or off

A discovery property has been created for turning CMDB coalesce functionality on or off

16.5 SNMP discovery fixes

Discovery now correctly populates base table information for SNMP devices (ie. printers, switches, UPS and etc...) as well as network adapters.
