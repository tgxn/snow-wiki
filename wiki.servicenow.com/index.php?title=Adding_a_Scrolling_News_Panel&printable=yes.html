1 Overview

 Note: This article applies to Fuji. For more current information, see RSS Feed Reader at http://docs.servicenow.com

The Wiki page is no longer being updated. Please refer to http://docs.servicenow.com for the latest product documentation.


Scrollable news panels offer a way for you to display high impact news items on your users home pages in an eye catching scrolling news display. Scrolling is always from the bottom up inside of a home page panel.

2 Adding a Scrolling News Panel to your Homepage

  1. Go to your home page by clicking on the home option in the top right corner of the screen (the one that looks like a house).
  2. Click on Add Content. You'll get a list of possible content types.
  3. Click on "Gadgets" in the leftmost box of the add content dialog. You'll get a list of all gadgets in the system.
  4. Click on the "News" gadget. You'll get a preview of it (quite likely empty of actual news). Then click "Add" to add it on your page.

3 Adding News to the Scroller

The news scroller gets its news list by going to the knowledge base and querying the "short descriptions" of any items there with a topic of "News". So to add news, just go into the knowledge base and add new entries with a topic of "News". It's that simple.

3.1 Style Control

If you want to make a headline standout in the news scroller use HTML tags on the short description. For example, a short description of 

<b>My Bold Test</b>

will be displayed in bold font in the scroller. Note that the property glide.ui.escape_text must be false to use HTML formatting.

4 Scrollable Areas

 Advanced customization: Functionality described in this page or section requires advanced scripting and may not be appropriate or practical for all implementations.

A scrollable area is basically a div whose contents scroll from the bottom up over time. It's a way to make something like a news ticket (which is the use case that motivated me to make it). You can scroll anything you want to, and anything inside the scroller is perfectly operational HTML e.g. links still work, images show up, etc.

To make one, all you have to do is wrap what you want to scroll inside of a scrollable_area tag like this (this would likely be done in a UI Page): 

    <g:scrollable_area height="100px">
        <g2:evaluate var="jvar_temp" expression="var kb = new GlideRecord('kb_knowledge');"/>
        <g:inline template="kb_section.xml"/>
    </g:scrollable_area>

The system will then create a 100 pixel high div and the contents will automatically scroll from bottom to top. If you have a 1000 pixel high block of text, for example, you'll see the top 100 pixels and then pixels 2-101, then 3-102, etc. Once it reaches the top it'll wrap back around to the bottom.

This sample code will create a scroller with a list of priority 1 incidents. 

<?xml version="1.0" encoding="utf-8" ?>
<j:jelly trim="false" xmlns:j="jelly:core" xmlns:g="glide" xmlns:j2="null" xmlns:g2="null">
  <g2:evaluate var="jvar_inc">
    var inc = new GlideRecord('incident');
    inc.addActiveQuery();
    inc.addQuery('priority',1);
    inc.query();
  </g2:evaluate>

  <g2:scrollable_area height="100px"> 
    <table border="0" cellspacing="2" cellpadding="0" width="100%">
      <j2:while test="$[inc.next()]">
        <j2:set var="jvar_inc_link" value="incident.do?sys_id=$[inc.sys_id]"/>
        <j2:set var="jvar_inc_list_link" value="incident_list.do?sysparm_query=active=true"/>
	<tr>
	  <td>
            <a href="$[jvar_inc_link]">
              <IMG SRC="images/services.png" style="padding-right:10px"></IMG>
            </a>
            <a href="$[jvar_inc_link]" style="padding-right:10px; color:blue">$[inc.number]</a>
          </td>
          <td>$[inc.short_description]</td>
        </tr>
      </j2:while>
      <tr>
        <td align="center" colspan="2"><a href="$[jvar_inc_list_link]" style="color:blue">View all active incidents</a></td>
      </tr>
    </table>
  </g2:scrollable_area> 
</j:jelly>

4.1 Scrollable Elements in Forms

Similar code can be used to add scrollable elements to forms.

To add a scrollable element to a form:

  1. Create a UI Macro with the script.
  2. Create a Formatter to reference the script.

4.1.1 Priority 1 Incidents Example

First, Create a UI Macro with the script. Use the following example code: 

<?xml version="1.0" encoding="utf-8" ?>
<j:jelly trim="false" xmlns:j="jelly:core" xmlns:g="glide" xmlns:j2="null" xmlns:g2="null">
  <g2:evaluate var="jvar_inc">
    var inc = new GlideRecord('incident');
    inc.addActiveQuery();
    inc.addQuery('priority',1);
    inc.query();
  </g2:evaluate>

  <div style="background-color:DDDDDD; padding-left:10px; line-height:19px; border:2px white solid" width="100%" nowrap="true">
    Priority 1 Incidents:
    <input id="make_spacing_ok" style="visibility:hidden;width:0px:" title=""/>
  </div>
  <g2:scrollable_area height="100px" width="100%"> 

      <j2:while test="$[inc.next()]">
        <j2:set var="jvar_inc_link" value="incident.do?sys_id=$[inc.sys_id]"/>
        <j2:set var="jvar_inc_list_link" value="incident_list.do?sysparm_query=active=true"/>
        <span style="line-height: 10px; padding-left:10px">    
          <a href="$[jvar_inc_link]">
            <IMG SRC="images/services.png" style="padding-right:10px"></IMG>
          </a>
          <a href="$[jvar_inc_link]" style="padding-right:10px; color:blue">$[inc.number]</a>
            $[inc.short_description]
        </span>
        <br style="line-height:5px"/>
      </j2:while>
      <span>
        <a href="$[jvar_inc_list_link]" style="color:blue; padding-left:10px">View all active incidents</a>
      </span>

  </g2:scrollable_area> 
</j:jelly>

Then Navigate to System UI > Formatters and create a Formatter that refers to the UI Macro above.

4.2 Notes & Limitations

  1. Scrolling is currently always from the bottom to the top.
  2. Scrolling is 1 pixel a at a time every 100 ms.
  3. You can have as many scrollable areas on a screen as you want (they get system generated unique names).
  4. Works in IE and Firefox.
  5. If the object you are scrolling is shorter than the scrollable area, it still scrolls.
  6. If you mouse over a scrollable area, it stops scrolling (so you can click on something without chasing it as it scrolls).
