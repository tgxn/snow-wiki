1 Fall 2010 Stable 3 Fixes

1.1 PRB559185, PRB559354 (Global Search): Clicking through to a record from Global Search results could give error "Request-URI Too Large"

Global Search results page has been modified to prevent this from happening.

1.2 PRB559321 (Project): When manually creating a predecessor/successor relationship from the predecessor related list on a project task, two entries would appear in the list

"Prevent duplicate/recursive relationship" Business Rule was modified to prevent the duplication.

1.3 PRB559273 (Suggestion Fields): Inactive choices might be presented for a Suggestion field that was dependent on another field

Inactive choices are now filtered out.

1.4 PRB559309 (On-call Rotation): An On-call Rota with a single person roster did not get reminders

A single person roster will now generate reminders as expected.

1.5 PRB559591 (Database Views): Label overrides (e.g., incident.short_description having a different label than task.short_description) were not being honored in a Database View

Label overrides are now honored in Database Views.

1.6 PRB559362 (UI Actions): In List v2, list UI Actions that worked with "current" improperly used the last record in the list as current

Behavior was modified to be consistent with List v1 behavior. The following is now true: 

- If a UI action has a name that starts with sysverb_, then the action can be run without selecting any records in the list
- If a UI action has a condition that contains 'current', the condition is not evaluated until after the action is clicked (by making an AJAX call to the server)
- If a UI action does not contain 'current' in the condition, the condition is evaluated on the server when building the list, so this controls the visibility of the action

1.7 PRB559522 (Filters): Show Matching on a document_id field produced an error

Show Matching now works as expected with document_id fields.

1.8 PRB559292 (Filters): List v2 did not support restricting filters and breadcrumbs with Business Rules

List v2 now supports restricting filters and breadcrumbs with Business Rules.

1.9 PRB559432 (Service Catalog): Multi-line variables with many lines of text could scroll to be fully viewed in IE8 without expanding the input

Multi-line variables with many lines of text now function as expected in IE8.

1.10 PRB555843 (Project): A project imported from MS Project might close too early

Imported tasks were being set to inactive by default. This no longer happens.

1.11 PRB559069 (History): Emails might not show in record History

Emails now show as expected in record History.

1.12 PRB426199 (BSM Map): BSM Map might rarely display relationships incorrectly

Issue was conflicting sys_id in a cmdb_rel_type record. This has been fixed, and relationships now display correctly in the BSM Map.

1.13 PRB559582 (i18n): Currency symbols could improperly appear as nonsense characters (introduced Fall 2010 Stable 2)

Currency symbols once again appear as expected.

1.14 PRB559569 (Homepage): The daily view of a Calendar Report did not render properly in a Homepage

The daily view of a Calendar Report now renders properly in a Homepage.

1.15 PRB559543 (Dictionary): Dictionary Overrides for an extension table more than one level down were not applied (e.g., cmdb_ci_server which extends cmdb_ci_computer which extends cmdb_ci)

Dictionary Overrides now apply for extension tables more than one level down.

1.16 PRB443751 (Reports): Printing a grouped, stacked report might improperly switch the group and stack fields in IE8

Printing a grouped, stacked report now works as expected in IE8.

1.17 PRB559576 (Release): If Project Management v2 was not installed, the Recalculate business rule on the rm_product table would generate an error

The dependency has been removed and the error no longer manifests.

1.18 PRB559298 (SDLC): Cannot move SCRUM stories out of releases through the planning board

SCRUM stories can now be removed from releases through the planning board.

1.19 PRB559217 (i18n): Insert and Stay did not carry forward translated field values

Insert and Stay now carries forward translated field values.

1.20 PRB559560 (Reports): A grouped List Report might resist expanding a group if grouped field value contained an apostrophe

Groups now expand as expected.

1.21 PRB559551 (History): The Activity Formatter could generate an error trying to display an audit entry for a field that no longer existed

This is now handled gracefully.

1.22 PRB559389 (Scheduler): The Autoclose Incident scheduled job, if set to run a future date, would get reset to current date in an upgrade

This scheduled job will no longer get reset in an upgrade.

1.23 PRB559374 (Printing): Print icon could be clicked before saving a new record

Clicking the Print icon before saving a new record now alerts the user to save the record first.

1.24 PRB559424 (UI): Pressing Enter when in the "Type filter text" input might create a new record from a list in IE

The Enter key now works as expected in the "Type filter text" input.

1.25 PRB559409 (Project): Adding a predecessor relationship to a Project Task could rarely consume an instance's memory if the Top Task was a Release

This is no longer a problem.

1.26 PRB559511 (Exports): The "Export to XML" UI Action on pm_project improperly exported all planned_task_rel_planned_task records

This UI Action now works as expected.

1.27 PRB559517 (Table Rotation): Deleting a record in a rotated table from a list failed

Deleting records from rotated tables now works as expected.

1.28 PRB559480, PRB559463 (i18n): Choice list options for non-default languages might display out of sequence

Choice list options for non-default languages once again display in expected order.

1.29 PRB559455 (Service Catalog): Global variables might not preserve their value when a new Catalog Task is manually created

Global variable values are now preserved as expected when new Catalog Tasks are manually created.

1.30 PRB559509 (Variables): variable_conditions fields might rarely render twice on a form

variable_conditions fields now render only once as expected.

1.31 PRB559502 (Email Client): The Reply button shown in the Activity Formatter by the email_reply UI Macro did not populate the Email Client with expected information

The Reply button once again works as expected.

1.32 PRB559500 (UI): A reference popup might fail to display in Chrome when the CI Relations formatter was in the view

The reference popup now works as expected in this case.

1.33 PRB559482 (Web Services): The sys_id field might be included twice in the WSDL insertResponse if the table had no display field

sys_id can no longer appear twice in the insertResponse.

1.34 PRB559479 (Reports): A new report might export a list of columns different that what was specified in the report

The columns exported are now as expected.

1.35 PRB559243 (Global Search): The exact match link after a Global Search did not update when a second search is performed for a different record

The link is now updated to reflect the correct matched record.

1.36 PRB559281 (Knowledge): Searching knowledge from a field did not escape ampersands properly

Searching knowledge from a field now handles ampersands as expected.

1.37 PRB559391 (Knowledge): New knowledge topics might not be shown in the Knowledge Base until the cache was flushed

New, populated knowledge topics now appear as expected.

1.38 PRB559477 (Integrations): SCCM Integration creates duplicate records in cmdb_software_instance table

The query at fault has been fixed and no longer creates duplicates.

1.39 PRB559112 (Knowledge): A hash character in an Incident's short description led to knowledge searches from the incident not working properly

This is no longer the case.

1.40 PRB559441 (Reports): A top-level OR condition in a bar chart report led to an incorrect filter when clicking a bar

The correct filter is now applied.

1.41 PRB542459 (Availability): A rare database lock during a zboot could result in instances becoming unavailable for a short time

The potential cause of the database lock has been eliminated.

1.42 PRB559293 (CMS): In Chrome, a CMS list frame did not resize when displaying fewer rows

The frame now resizes properly.

1.43 PRB559382 (Project): Planned effort did not roll up for a Project like actual effort did

Planned effort is now rolled up by a business rule.

1.44 PRB559066 (CMS): CMS IFrame resizing issue when a list was grouped and a group was expanded

The IFrame now resizes as expected

1.45 PRB559444 (UI): Spell check for an input element did not cover the field properly

Spell check now works as expected.

1.46 PRB559406 (Scripts): A long-running script could fails if the "Clean Temp Files" scheduled job ran during execution

Spooled data blocks are now cleaned after 24 hours, not 1 hour, to prevent this problem.

1.47 PRB559439 (Database Views): Could not run problem_metric database view against an Oracle database

An invalid database view definition was fixed for problem_metric.

1.48 PRB559425 (Database Views): Database View reference fields with same column name (from same base table) might display the wrong value

Correct values are now displayed for references when view has the same table twice (task for example when joining change_task with change_request).

1.49 PRB559426 (CMS): CMS search might not return results

CMS search now returns expected results

1.50 PRB559340 (Reports): Two out-of-box Problem reports have no filters

"Problems opened this month by Priority" and "Problems opened last month by Priority" have appropriate filters now.

1.51 PRB559295 (Reports): Group reference field might not display properly in report UI

Group reference field now appears as expected when Group is checked.

1.52 PRB559359 (UI): A space in a mandatory field passed validation

Whitespace no longer counts as an entry for mandatory fields.

1.53 PRB285108 (UI): HTML field input might not be usable if glide.ui.textarea_initial_rows was set to low

HTML field inputs now appear as expected.

1.54 PRB559394 (Reports): Special chars < or > in short description could render poorly the Change Calendar

The Change Calendar now renders as expected when brackets are used.

1.55 PRB559379 (Service Catalog): Setting Requested For on the Service Catalog portal could throw a JavaScript error

The potential cause of the JavaScript error has been removed.

1.56 PRB559414 (Service Catalog): Mandatory field message in Service Catalog was not translated

The message can now be translated.

1.57 PRB559395 (UI): List views on a homepage might be double grouped, causing a rendering issue

The cause of the potential double grouping has been removed.

1.58 PRB559390 (Database Views): Database Views were not being separated when domain numbering was turned on

Domain numbering separation now works as expected for Database Views.

1.59 PRB559396 (Filters): List calculations were incorrect when a filter involved an OR or AND condition on a date field

List calculations now work as expected in this case.

1.60 PRB559333 (Properties): Private properties (e.g., email properties) were not being written out as customer updates, so they could be overwritten in an upgrade

Private properties are now written out as customer updates to prevent this problem. They are appropriately not transferred in update sets.

1.61 PRB559392 (UI): fields of type "variable_conditions" did not populate on form load

This field type once again populates as expected on form load.

1.62 PRB559323 (UI): Could not close attachment dialog in IE if there were a large number of attachments

The attachment dialog can now be closed as expected.

1.63 PRB559376 (UI): A dot-walked document_id field did not display properly in a related list

This now displays properly.

1.64 PRB559375 (Import Sets): Imported datetime values were improperly modified according to daylight saving time

Transform now uses getDisplayValue rather than getValue to prevent this.

1.65 PRB559366 (Database): An invalid many-to-many table entry could cause a cascade delete to generate an error and stop

The cause of this problem has been fixed.

1.66 PRB559186 (On-call Rotation): Managers for groups that have on-call rotations might not see 'New' and 'Edit' buttons when managing a group rotation

Buttons once again appear when appropriate.

1.67 PRB538996 (FSM): Some modules from the Field Service Management Plugin gave a "form not found" message

Plugin dependency was fixed to prevent this problem.


1.68 PRB559423: The PDF Generator failed to generate a chart if the grid was not included.

Now when a PDF report is scheduled for email, the PDF is generated, even if the grid in hot included.

1.69 PRB559429: The SOAPEnvelope script include was not properly loaded on zbooted instances.

The SOAP protocol loads properly throughout the platform.

1.70 PRB559369: A NullPointerException occurred on the server when executing wizards.

Wizards now advance to the next panel without producing an error.

1.71 PRB559392: The variable_conditions field (used in Catalog UI Policy) was not rendering properly.

The value in the variable_conditions field is not deleted when the field is updated.

1.72 PRB559434: A NullPointerException occurred on Oracle when viewing tables with at least one Roles column.=

Record lists (using List v1) display all available rows.
