1 Overview

Tags allow the grouping and organizing of records. Tags can be visible to any user (global), visible only to specific groups or users, or visible to a single user.

Note: If you are using a version of the ServiceNow platform earlier than Eureka, see previous version information in Creating and Using Labels - Versions Prior to Eureka.


2 Assigning Tags

Users can assign tags:

  • From a list using inline field editing
  • To multiple records directly from the list view
  • From a list using the action menu
  • From a list using the context menu
  • From a form record using the Edit Tags icon
 Note: Tag names are case-insensitive and must not include punctuation.


2.1 Assigning a Tag from the List View Using Inline Field Editing

Users can assign tags to records directly from the list view (starting with the Fuji release).

  1. Navigate to a list.
  2. Personalize the list to display the Tags column.
  3. Click in the Add Tag field.
  4. Perform one of the following options:
    • To assign an existing tag, begin typing the tag name, select the tag from the drop-down, and press Enter.
    • To create a new tag, enter a new tag name and press Enter.
    The tag is added to the form and a new Add Tag field appears. You can assign more than one tag to a record.


2.2 Assigning a Tag to Multiple Records Directly from the List View

Users can assign tags to multiple records directly from the list view (starting with the Fuji release).

  1. Navigate to a list.
  2. Personalize the list to display the Tags column.
  3. Select one or more records using the Shift or Control/Command key.

  4. Double-click the highlighted border of the field.
  5. Click in the Add Tag field.
  6. Perform one of the following options:
    • To assign an existing tag, begin typing the tag name, select the tag from the drop-down, and press Enter.
    • To create a new tag, enter a new tag name and press Enter.
    The tag is assigned to all selected records. You can assign more than one tag to a record.

2.3 Assigning a Tag from a List using the Action Menu

Users can assign tags using the action menu.

  1. Navigate to a list.
  2. Select one or more records using the check box.

  3. Click Actions on selected rows.
  4. Perform one of the following options:
    • To assign an existing tag, begin typing the tag name and select the tag.
    • To create a new tag, enter a new tag name and press Enter.
    The tag is added to the form. You can assign more than one tag to a record.

2.4 Assigning a Tag Using the Edit Tags Icon

Users can assign tags using the edit tags icon () in the form header.

  1. Navigate to a form.
  2. Click the edit tags icon to expand the form header and display the Add Tag field.
  3. Click in the Add Tag field.
  4. Perform one of the following options:
    • To assign an existing tag, begin typing the tag name and select the tag.
    • To create a new tag, enter a new tag name and press Enter.
    The tag is added to the form. You can assign more than one tag to a document.

2.5 Assigning a Tag from the List Context Menu

Users can assign tags from the list context menu.

  1. Navigate to a list.
  2. Right-click a record.
  3. Perform one of the following options:
    • To assign an existing tag, select Assign Tag > [Tag Name].
    • To create a new tag, perform the appropriate option for your version:
      • Fuji: Select Assign Tag > New Tag, enter a new tag name, select the appropriate sharing level, and click Save.
      • Eureka: Select Assign Tag > New, enter a new tag name, and click OK.

3 Editing and Sharing Tags

3.1 Editing Tags from the List View

Users can edit tags that they have created from the list view (starting with the Fuji release).

Note: Tag names are case-insensitive and must not include punctuation.


  1. Navigate to a list with tags that you have previously assigned.
  2. Ensure the Tags column is visible.
  3. Click the Edit tag audience icon ( or ) by the tag name.
    This enables editing for the tag.
  4. Click in the Name field and make the necessary changes.
  5. Select the sharing level by selecting the appropriate Viewable by option.
  6. Click Save.
    All records with that tag are updated to reflect the change.

3.2 Editing Tags from the Form View

Users can edit tags that they have created from the form view (starting with the Fuji release).

  1. Navigate to a record with tags that you have previously assigned.
  2. Click the Edit Tags icon () in the form header.
  3. Click the Edit tag audience icon ( or ) by the tag name.
    This enables editing for the tag.
  4. Click in the Name field and make the necessary changes.
  5. Select the sharing level by selecting the appropriate Viewable by option.
  6. Click Save.
    All records with that tag are updated to reflect the change.

3.3 Editing Tags from the My Tags Module

Users can edit tags that they have created from the My Tags module (starting with the Fuji release).

  1. Navigate to Self-Service > My Tags.
  2. Open a tag.
  3. Make the necessary changes.
  4. Click Update.
    All records with that tag are updated to reflect the change.

3.4 Editing Tags from the Tags Module

Users with the admin role or, starting with the Fuji release, the tags_admin role can edit all tags by navigating to System Definition > Tags.

  1. Navigate to System Definition > Tags.
  2. Open a tag.
  3. Make the necessary changes.
  4. Click Update.

3.5 Editing Tags from the Tagged Documents Page

Perform these tasks to edit tags from the Tagged Documents page, starting with the Fuji release. If you are using an earlier release, see the previous version information below.

  1. In the edge, click Tagged Documents.
  2. Click Edit Tags.
    A list of all tags you have created appears.
  3. Open a tag.
  4. Make the necessary changes.
  5. Click Update.
    All records with that tag are updated to reflect the change.
Click the plus to expand previous version information
  1. In the edge, click Tagged Documents.
  2. Click Edit Tags.
    This enables editing for all of the tags on the Tagged Documents page header.
  3. Click a tag and make the necessary changes.
  4. Click Done to save the changes.
    All documents with that tag are updated to reflect the change.

3.6 Sharing Tags

Users can share tags with other users by setting Viewable by (starting with the Fuji release). The following sharing levels are available:

  • Me: Private tag, visible only to the owner.
  • Groups and Users: Shared tag, visible to the owner and specific groups/users.
  • Everyone: Shared tag, global tag visible to everyone. This option is available to users with the admin or tags_admin role.

3.6.1 Name Conflict: Private and Shared Tag with the Same Name

If a private tag and a shared tag have the same name (for example, apple), the private tag is renamed to name [private] (for example, apple [private]).

4 Filtering Tags

Any user can filter against any tags that are visible to that user (starting with the Fuji release).

A user can filter by tags using one of the following methods:

  • From the form view, click the edit tags icon, then click the tag name.
  • From the list view, with the Tags column visible, click the tag name.
    Clicking on two or more tags filters the tags together with the AND operator and shows records containing all of the selected tags.
  • From the list view, click the show/hide filter icon (), select Tags from the Choose field drop-down and enter the tag name.
    In order to filter on either of two tags, you must use two filter conditions with the OR operator.

5 Viewing Tagged Documents

The Tagged Documents page displays recently viewed documents or user-tagged documents in the content frame. Access this page by clicking the Tagged Documents button on the Edge.

UI14 Tagged Documents Page

By default, there is a label for recently viewed documents and for each user-created tag. You can click Show Global Tags to display a label for each global tag (starting with the Fuji release).

  • Click Most Recent to display the most recently viewed documents.
  • Click any of the tags to display the documents associated with that tag.

Individual preview areas display selected information for each document.

  • If necessary, point to a document to expand the preview (Eureka only).
  • Click the document title to open the form for that record.
  • Click the remove tag icon (X) in the upper right corner to remove the tag from that record.

6 Removing Tags from Records

Tags can be removed from records using any of the following methods:

  • From the list view, with the Tags column visible, click the remove tag icon (X) beside the tag name (starting with the Fuji release).
  • From the list view, with the Tags column visible, select multiple records using the Shift or Control/Command key, double-click the highlighted border of the field, then click the remove tag icon (X) beside the tag name to remove the tag from all selected records (starting with the Fuji release).
  • From the list view, select the check boxes for one or more records, then click Actions on selected rows > Remove Tag: > [Tag name].
  • From the form view, click the edit tags icon, then click the remove tag icon (X) beside the tag name.
  • From the Tagged Documents page, click the remove tag icon (X) in the upper right corner of the record preview.
 Note: If a tag was automatically applied to a record based on specific conditions, the tag is automatically removed when those conditions no longer apply.


7 Merging Tags

You can merge tags into other tags (starting with the Fuji release). For example, when you merge Tag A with Tag B, Tag A is deleted and all associated records are reassigned Tag B.

A user with the admin or tags_admin role can merge any tag into any other tag. All other users can merge tags they created into tags that are visible to them.

To merge a tag:

  1. Navigate to Self-Service > My Tags.
  2. Select the check boxes for the tags you want to merge into another tag.
  3. Select Merge Tags from the Actions on selected rows drop-down.
  4. Enter the tag to merge other tags into.
  5. Click OK.

8 Creating a Tag from the Tags List

Users with the admin role or, starting with the Fuji release, the tags_admin role can create tags directly from the Tags list.

To create a tag from the Tags list:

  1. Navigate to System Definition > Tags or, starting with the Fuji release, Self-Service > My Tags.
  2. Enter a name for the tag in the Name field.
  3. [Optional] Modify tag settings (see table).
Field Description
Owner Specifies the owner of the tag. Users are allowed to change the ownership of the tag to someone else.
Type Specifies the type of tag. For more information about tag types, see Types of Tags.
Viewable by Specifies the sharing level. By default tags are visible only to the user (Me), but you can share tags with Groups and Users. Users with the tags_admin role are the only users who can create global tags shared with Everyone. This field is available starting with the Fuji release.
Active Specifies whether the tag is enabled or disabled.

9 Zing Tag Indexing for Text Search

Text indexing is available for tags on records (starting with the Fuji release).

9.1 Configuring Zing Tag Indexing for Text Search

No tags are indexed by default. Only users with the admin role can enable text indexing of tags on records on a table-by-table basis.

Note: Private tags are never indexed.


To turn on text indexing for tags:

  1. Navigate to the Dictionary Entry for the table for which you want to turn on indexing.
  2. Click New under the Attribute section.
  3. Click the magnifying glass to see available attributes.
  4. Select the Text Index Tags attribute (text_index_tags).
  5. For attribute value, enter one of the following options:
    • everyone_only: Only tags shared with everyone will be indexed.
    • all_shared: All shared tags (Everyone, Groups and Users) will be indexed.
  6. Click Submit.
  7. Click Generate Text Index under Related Links to reindex the table.

9.2 Reindexing of Tags

Once text indexing of tags on records is turned on, records get reindexed upon:

  • Adding or removing tag associations.
  • Modifying a tag, provided fewer than 100 records are affected.
  • When triggering a reindex of an entire table (required after initial tag indexing setup).

If there are many records, then the administrator must manually trigger a text indexing.

10 Assigning Tags Automatically

A user with the admin role or, starting with the Fuji release, the tags_admin role can configure the system to automatically assign a tag to records based on criteria defined in the tag record. For example, you can create tags that group:

  • High urgency incidents
  • Incidents flagged as overdue
  • Canceled changes
  • Non-operational business services
  • Users whose last name starts with the letter 'P'

To assign a tag automatically:

  1. Navigate to System Definition > Tags.
  2. Open the tag that you want to automatically assign.
  3. Configure the form to add the Conditions for Labels embedded list.
  4. Create a new row in the Conditions for Labels list.
  5. Click the reference icon in the Table column.
    The Label Table list opens in a pop-up window.
  6. Click New.
  7. Enter a Name, select a Table, and add conditions for the automatic application of the tag.
  8. Click Submit.
  9. Click the check mark icon to save the new row.
  10. Click Update.

The system automatically assigns the tag to records that match the criteria and removes the tag from records that no longer match the criteria.

11 Configuring Tag UI Properties

A user with the admin role can set the refresh period for tags.

To change how often tags refresh:

  1. Navigate to System Properties > UI Properties.
  2. Enter the number of seconds in the Label refresh period (seconds) field.
  3. Click Save.

12 Notifications Using Tags

It is possible to configure a tag entry to send a notification when the record is changed.

To customize a tag entry:

  1. Navigate to System Definition > Tags.
  2. Open the desired tag.
  3. In the Label Entries related list, open the record on which you want notifications.
  4. From the label entry form, customize the entry for notifications:
    • Select the Notify onchange check box to be notified any time this record is modified.
    • Specify the conditions when you want to be notified with the Notify when condition builder.
    • Enter the name of a business rule in the Notify script field

13 Sorting Tags

Sorting is not available on tags.

14 Types of Tags

There are several types of tags:

  • Standard: is controlled by the user who created the tag. This user can add or remove records from this tag.
  • Most Active: displays the most often accessed modules and is automatically maintained by the system. There is no maximum duration a module can remain as Most Active.
  • Most Recent: displays the most recently used modules and is automatically maintained by the system.
  • Most Active Record: displays the most often viewed records and is automatically maintained by the system. There is no maximum duration a record can remain as Most Active.
  • Most Recent Record: displays the most recently viewed records and is automatically maintained by the system.

Most Active and Most Recent tags are maintained both at the system level and individually for each user. Modules or records tagged with the Most Recent tag cannot be removed by clicking the X icon. The Most Recent tag always shows the most recent modules or records. A Most Active tag contains the most active modules or records for the user who created the tag. A Most Active global tag contains the most active modules or records across all users.
