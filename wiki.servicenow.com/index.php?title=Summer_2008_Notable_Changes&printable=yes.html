Summer 2008 Notable Changes

From ServiceNow Wiki

The following are notable changes you may encounter with our Summer 2008 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before.

1 Display field behavior enhancement for extended tables

We have developed a new algorithm to determine the display name field for a table. Previously, we took the first field we came across that had the display flag set without regard to any table extensions.

Here is an example describing both algorithms. Say you had the display flag set on both of the following fields:

  • task.number display=true (NOTE: You probably do NOT want to change the display value on the Task table)
  • change_request.change_plan display=true

If you had the incident.rfc field, which references the change_request table, added to your incident form, then the display value for the change request record may have been task.number OR it may have been change_request.change_plan. It depended on which display=true value we found first.

Now we determine the display field by going through the following logic:

  1. Search the current table's fields for a display flag set to true. If one is found, use it.
  2. If no display field found in the current table (e.g., change_request), search fields in extended tables, going up the hierarchy until one is found (e.g., if change_request has no display field, next look at the task table for one). If one is found, use it.
  3. If no display field found in any table in the hierarchy, use NAME (if it exists)
  4. Use SYS_ID if all else fails

So, if you were used to seeing the number but now you aren't seeing anything, or you are seeing something different as the display value for a reference field, then you previously had multiple display fields designated, we were displaying the right one by chance, and this new algorithm is now coming into play.



2 Critical Incidents module and other Map Pages

The Incident Management --> Critical Incidents module is an example of our new Map Pages functionality that allows you to graphically display data on a map. You can deactivate this module if you wish.


3 LDAP OU Definitions possibly affecting login

Regarding our new simplified LDAP setup, if your LDAP users are locked out and you are seeing the message "No information found in LDAP for users XXXX" then there may be a problem with your LDAP OU definitions.  If you had spaces in your LDAP OU definition property in the old LDAP Properties page, then the automatic creation of LDAP OU definitions based on that property may have created OU's with a leading space character (' ou=myou').  The application validates that users are in mapped OU's, and the leading space causes the validation issue.  To resolve this issue go to System LDAP - LDAP Servers.  Choose your LDAP Server and Navigate down to LDAP OU Definitions.  Open each OU definition and remove the leading space. In the next release (or any future patch to the Summer release), leading spaces in OU definitions will be ignored.


4 New Email Notifications: Incident Resolved and Incident Closed

Two out-of-box email notifications, Incident Resolved and Incident Closed, are now active by default. If you do not want these email notifications (e.g., if their functionality duplicates custom notifications in your instance), they can be deactivated.


5 Use of JavaScript Object() vs. JavaScript Array() for an associative array

To support Discovery, a change was made to our extensions to JavaScript Array that adds two additional elements: "indexOf" and "contains". This elements are visible if you iterate the array with a "for(var in array)" instead of a "for(var i =0; i < array.length;i++)" loop. If this change negatively impacts your use of JavaScript Array, you can use JavaScript Object() instead. The only change you would need to make would be to change "new Array()" to "new Object()" in your script.


6 Duplicate sys_dictionary entries deleted

If an instance had duplicate sys_dictionary entries, meaning more than one record with the same name/element values, duplicates were deleted and only the oldest record retained. Then a unique index was placed on the name/element fields to prevent future duplicates.


7 World Clocks

IE Related Bug - The World Clocks will initially display with just the headers/labels visible.  To see the actual clocks in World Clocks then refresh that section of your Home Page.  We have an open problem ticket regarding this behavior.


8 TRENDBY Condition

A new TRENDBY condition was added in error to stacked charts.  This new condition does not affect the report results, and this has been fixed in recent patch builds of the Summer release.


9 Potential module changes

Several modules were inadvertently modified for instances created before Fall 2007. The modification would only be deployed to an instance where each module had not been customized, so most likely only a few modules would be affected on any given instance. A fix has been added to the next Stable release, and will also be added to a patch to the Summer release. In the meantime, if any of these modified modules causes concern or inconvenience, they can simply be manually fixed. In most cases, a module was simply either activated or deactivated when it should not have been. In some cases roles on a module were modified, though impact may be minor if the application itself is protected.

The following modules were modified:

  • Service Catalog
    • Variable Sets - roles changed
    • Open Requests - order, title changed
    • Maintain Packages - deactivated
    • All Variables - roles changed
    • Item Variables - roles changed
    • Plan Variables - roles changed
    • My Groups Work - deactivated
    • My Approvals - deactivated
    • Assignment Rules (Service Catalog) - deactivated
    • Approval Rules (Service Catalog) - deactivated
    • Fulfillment Groups - roles changed
    • Catalog Client Scripts - order, roles changed
    • Execution Plans - roles changed
    • Requested Items - title, order, roles changed
    • Service Catalog - order changed
    • Catalog - order changed
    • Variable Categories - deactivated
  • Self Service
    • Request Password Reset - deactivated
    • Create New Incident - order changed
    • About - deactivated, roles changed
    • Incidents - order changed
  • Dashboard (System UI) - deactivated
    • Dashboard modules - changed to broken homepage links
  • Other
    • Number Maintenance - activated
    • Companies - activated
    • Welcome Page Content - activated
    • Scheduled Report Emails - title changed
    • Group Business Rules - deactivated
