1 Overview

This is the base class for all workflow activities. As such, this code always executes as part of an activity.

1.1 Where To Use

When developing your own workflow activities, create your script object by deriving from this class and overriding methods as necessary to get the functionality you want. Client code must override method onExecute() to perform meaningful activity processing.


2 Method Summary

Method Summary Description
execute() Overrides onExecute() to change the execute handling. This is an internal method that should not be directly called.
setResultFailed(reason) Sets the activity result as failed with an optional reason string. See Activity "SOAP Message".
setResultSucceeded() Sets the result of this activity as successful.
setActivityOutput(output) Convenience method to set the activity output property. See activity "SOAP Message".
handleEvent(eventName) Handles an event for this activity.
onExecute() Virtual method. Activity subclasses must override this method to perform work appropriate to the activity.
onCancel(eventName) Event handler for 'cancel' event.
runScript: function(script) Convenience method for activities to run script contained in an activity variable of type script.
js(str) If a string starts with javascript, evaluate it.
info(msg, args) Logs an info message.
warn(msg, args) Logs a warning message.
debug(msg, args) Logs a debug message.
generate(activityId, order, startAtDspValue, noCreateFlag) Handles pre-generation of tasks and approvals (and other things as needed). See activity "Generate" for example usage.

3.4 handleEvent(eventName)

Called by the Prototype JavaScript Framework when an event is sent to this activity, this method dispatches the event to an event handler defined by the subclass. It is unlikely an activity would call this method or override this method.

Handles an event for this activity, as follows:

  • If onAnyEvent is defined, it is called first
  • If on<eventName>Event is defined, it is called
  • If on<eventName>Event is not defined and onUnhandledEvent is defined, onUnhandledEvent is called

3.4.1 Input Fields

Parameters:

  • eventName - The event name to be handled.

3.4.2 Output Fields

Returns: void

3.6 onCancel(eventName)

Event handler for 'cancel' event. This class (base class for activity script) sets this activity to be cancelled. Derived classes (activities) can override this method if additional processing is required to cancel this activity. Activity "Manual Approvals" is an example of overriding this method to perform additional processing to cancel this activity.

3.6.1 Input Fields

Parameters:

  • eventName - Not used

3.6.2 Output Fields

Returns: void


3.7 runScript(script)

Convenience method for activities to run script contained in an activity variable of type script. Example activities that use this method include "If", "Wait for condition", and "Approval - User".

3.7.1 Input Fields

Parameters:

  • script - String containing valid JavaScript.

3.7.2 Output Fields

Returns: If the script set the workflow variable "answer" then this value is returned. If not, then the result of the script execution is returned.

3.12 generate(activityId, order, startAtDspValue, noCreateFlag)

Handles pre-generation of tasks and approvals (and other things as needed). See activity "Generate" for example usage.

This allows things to be generated at the start of a workflow in a 'pending' state and not requested until the workflow gets to the activity, thereby providing a preview of the work to be done in the future. To pre-generate things, an activity's _generate() method is called. This method typically returns a pre-generation object that is saved in the workflow scratchpad. (For example, the object might be an array of approval ids that were generated) When the activity runs (onExecute) it may request the pre-generation object and determine if the approvals have already been created in a pending state or if it needs to create the approvals in the onExecute method.

3.12.1 Input Fields

Parameters:

  • activityId - sys_id of this running activity (table wf_activity)
  • order - The order number associated with the task or approval
  • startAtDspValue' - The starting time for the task or approval
  • noCreateFlag -

3.12.2 Output Fields

Returns: Internal method "_generate" needs to be overridden and needs to return an object with information that it can retrieve later on when the activity's onExecute() method is called. The object must contain a 'duration' property that contains the duration of the task(s) or approval(s) that were generated by the activity. The 'duration' property of the _generate return value is returned by this method.
