1 Overview

The detailed billing reports of the Amazon Web Services Billing application allow you to monitor costs and visualize your AWS spending. AWS Billing offers a set of billing reports on your AWS usage based on hourly usage. You can view graphs of these reports from the Cost reports page. The report details list tags associated with the resources. Tags allow you to categorize EC2 instances. For more information on tags, see Amazon Web Services Tagging.

Note: To receive the reports, you must have an Amazon S3 bucket available in your AWS account. See this documentation on the Amazon Web Services site


AWS billing is a feature of Amazon Web Services.

2 Required Roles

Users with the aws_admin role can configure, execute and view billing reports in ServiceNow. Users with the aws_user role can only view reports.

3 Viewing Billing Cost and Usage Reports

You can view a dashboard displaying graphs breaking down your costs by the following report types:

  • Product operation type - displays costs broken down by the operation type ran in the AWS account.
  • Total cost by resource - displays the total costs incurred by each AWS resource in the account.
  • Total cost by product name - displays the total costs incurred by each AWS product.
  • Total cost by linked account - displays the total costs incurred by each linked AWS account.

Navigate to Amazon Web Services > Billing > Usage Reports.

For usage reports, the y-axis is the aggregate usage quantity from the auto-generated Amazon billing report that was downloaded by the billing schedule job.

Navigate to Amazon Web Services > Billing > Cost Reports.

For cost reports, the y-axis is the aggregate blended costs.

3.1 Managing Billing Reports

You can configure jobs to process billings on your Amazon S3.

  1. Navigate to Amazon Web Services > Billing > Report Schedule.
  2. Click New.
  3. Enter a Job Name for the report. Job names do not have to be unique.
  4. Select the AWS account.
  5. Select the AWS Credential.
  6. Enter the Bucket.
  7. Click Submit.
An AWS billing report schedule‎

Field Description
Job Name The name of the report.
AWS Account Name of the AWS account on which you want to run the billing report.
Credential The AWS account credentials.
Bucket The S3 bucket of the AWS account containing the data.
Scheduler [read-only] The name of the reporting script.
Last Execution [read-only] Date and time the report was last completed.
Last Successful Execution [read-only] Date and time the report was last successfully completed.
Last Executed State [read-only] State of the last executed report.
Current Job Status Shows the status of the the report. This field appears when you select Execute Now for Current Job Status.
Note: If a billing job configuration is deleted, the associated scheduled job will be automatically deleted. The billing job configuration and scheduled job are mapped one-to-one.


3.2 Running a Billing Report

By default, the scheduled report will run on a daily basis. To run the report on demand, follow these steps:

  1. Navigate to Amazon Web Services > Billing > Report Schedule.
  2. Select a report.
  3. Click Execute Now.
 Note: This option is only available when the report is not currently running.


3.3 Amazon Billing Data Rollup

After a billing report is downloaded, billing data is rolled up and given a set of pivot, and aggregate fields. The rollup is done for both daily and monthly periods. Pivot fields can be provided in the aws_billing_rollup_pivot table, and aggregate fields can be provided in aws_billing_rollup_aggregate table. Data is stored in the following tables:

  • aws_billling_rollup_daily
  • aws_billling_rollup_monthly

The reporting on billing is driven from these tables. When the download of the billing report is complete, the billing data rollup automatically runs for the current month by default. Cost Reports reflect the rollup result.
