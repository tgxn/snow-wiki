1 Overview

Field styles allow administrators to declare individual CSS styles for a field in a list or form. The CSS can:

  • Change the color.
  • Change the font attributes (bold, italics, underline).
  • Change the padding and alignment of text.

2 Defining Field Styles

You can define field styles for tables and database views that are in the same scope as the field style and for other tables that have at least one field in the same scope as the field style (starting with the Fuji release). Note that field styles defined for a table will not apply to database views that include the table. You must create separate field styles for database views.

To define a field style, navigate to the Style form in one of the following ways and fill in the fields on the form, as appropriate (see table).

  • Right-click the field label on the form and select Configure Styles (Personalize Styles in versions prior to Fuji).
  • Navigate to System UI > Field Styles.
Field Description
Table Table that contains the field.

Note: The list shows only tables and database views that meet the scope protections for field styles (starting with the Fuji release).

Field name The field to which the style applies.
Value The matching value of the field for applying the style.

Note: Field styles based on value only work on lists. To apply field styles on both the field and the form, leave this field blank.

Style The CSS style to apply.

3 Advanced Style Selection

The Value field may contain JavaScript that is evaluated to determine if this style should be used for the field within a list. Set Value to javascript:<script> - the field value is available to the script as the variable value. If the evaluated script returns true, the style is used for the list item.

For example, this value may be used to set the style for overdue items in the list, where current.u_datefield is the name of a date field on your table: 

 javascript:gs.dateDiff(gs.now(), current.u_datefield.getDisplayValue(), true) < 0

The syntax javascript:<script> can only be used once in the Value field. If you want to enter multiple JavaScript conditions, they must be consolidated into one statement. For example, do not enter: 

javascript: current.state == "Completed" javascript: current.error_tables > 0

Instead, enter: 

javascript: current.state == "Completed" && current.error_tables > 0

4 Change Width of a Form Field

It is possible, through the use of field styles, to change the width of a form field in the UI11 and classic interfaces. Field width is set automatically in UI15 and UI14.

Note: The system prevents you from shortening system field widths to avoid potential data loss. See the description of the Max length field in the System Dictionary for more information.


To do this:

  1. Right-click the label of a field displayed on the form and select Configure Styles (Personalize Styles in versions prior to Fuji).
  2. From the list of Styles for that field, click New.
  3. In the Style field, to change the width of the field to 240 pixels, for example, use the CSS syntax width:240px.
    You may need to experiment a bit to find the best width for your need. Click Submit when you are done done.

5 Defining Reference Styles

Reference styles are similar to field styles, but apply to any reference to a table. For instance, a reference style on the User [sys_user] table would apply to any reference field on any table that refers to User.

To define a reference style, enter sys_ui_reference_style.list in the application navigator filter and click New.

Fill in the Reference Style form:

Field Input Value
Table The table to which all references will be styled.
Active If the check box is selected, the style will be applied where appropriate. If the check box is cleared, it will not.
Style A CSS style to apply.

6 Enhancements

6.1 Fuji

  • These changes support developing scoped applications:
    • The Table field shows only tables and database views that meet the scope protections for field styles.
    • For tables that are in a different scope than the field style record, the Field name field shows only fields in the same scope.
    • The Application field is added on the form view.
