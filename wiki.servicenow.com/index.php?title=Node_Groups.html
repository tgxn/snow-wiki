1 Overview

Node groups allow an administrator to group together a set of CIs and apply configuration templates to the entire group instead of individually applying configuration templates to each CI.

1.1 Creating a Node Group

  1. Navigate to Configuration Automation > Node Group.
  2. Click New.
  3. Enter a Name for the node group and then select the Provider.
  4. For Chef or Puppet, specify the Management server.
  5. Enter a description for the node group.
    The Assigned Configurations list shows the configuration that is currently assigned to this node group. ServiceNow provider supports multiple configurations. Chef supports only one configuration.
  6. Click Submit.
    Node group creation

1.2 Manually Adding CIs to a Node Group

  1. Navigate to Configuration Automation > Node Group.
  2. Select a node group.
  3. In the Configuration Items section, click Edit and then add the CIs to the node group.
  4. Click Save.
    Adding CIs to a node group

1.3 Using Assignment Rules to Add CIs to a Node Group

You can configure assignment rules that automatically map CIs to node groups when certain conditions are met.

Note: If the CI is in a change pending control state as part of another request, the CI will not be assigned to the node group. Pending node assignment will be created, but the Apply Assignments action will fail with the State of 'Error' and the reason: "Found that the node group has a change request open on it. Cannot proceed with the assignment because of the open change request."
  1. Navigate to Configuration Automation > Node Groups > Assignment Rules.
  2. Click New.
  3. Fill in the form as described in the table.
  4. Use the Node filter to set conditions that must to be met in order for the node to be assigned to the node group.
  5. Use the Node Filter Script to set additional custom variables that must be met. Use the 'current' variable to access cmdb_ci_computer fields in the node filter script.
  6. Click Update to save the rule, or click Execute Now to manually run the rule.
    On Execute Now, the rules are applied only to CIs with a Node group assignment state of Ready. Execution Information displays the last time that the rule was executed as well as the current state of the rule. All rules execute on a schedule and any newly onboarded or updated CIs are reassigned to node groups based on the assignment rules. In addition, when a new rule is added or an existing rule is updated, all CIs that are assigned are reset to a Node group assignment state of Ready.
Field Description
Name Name of the assignment rule.
Active Select the check-box to activate the rule.
Order The number used to order the rule.
Assign to Node Group [Required] Select the node group to which the rule applies to.
Auto Assign If Auto Assign is enabled for a rule, the CIs are automatically assigned to the corresponding node groups and change request will be opened to apply the currently applied configurations to the node group.

1.3.1 Pending Node Assignments

The Pending Node Assignments module lists the CIs that match any of the rules created in the assignment rules module. If the CI is not automatically assigned to the node group, you can assign it as follows:

  1. Navigate to Configuration Automation > Node Groups > Pending Node Assignments.
  2. Select the CIs that you want to assign on the list.
  3. Click 'Apply Assignments'.
Field Description
Configuration Item Name of CI.
Change Request Staus of the change request.
Node Group Name of the node group that the CI is assigned to.
Auto Assign Indicates whether the CI was automatically assigned to the node group.
Order Order in which the CI was assigned.
State Current state of the CI.

1.4 Modifying a Node Group

To modify a ServiceNow node group with change management enabled:

  1. Navigate to Configuration Automation > Node Groups.
  2. Select a node group.
  3. Click the Create New Configuration Change related link.
    The Node Group Change form appears.
  4. Edit the fields, as shown in the table.
  5. Click the Request Change related link.
    The Change Request form appears. The Description field displays the service provider, attributes, components, and affected CIs of the node group. You can use the information to view the change history for the node group.
Node group description field on the change request form.

Field Description
Phase Change request state of the node group.
Provider Configuration automation provider such as ServiceNow, Chef, or Puppet.
Management server Chef and Puppet only. Management server for the node group.
Change Request [Read only] Change request record number.

1.4.1 Viewing Node Group Change History

The change history feature allows you to view the details when modifying a node group and changing the configuration template. After you have requested a change, the Description field in the Change Request form displays the details of the attributes, components, and CIs that were deleted and those that were added as a result of the change.
