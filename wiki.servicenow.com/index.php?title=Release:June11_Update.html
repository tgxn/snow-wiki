Release:June11 Update

From ServiceNow Wiki
Home > Release Notes > Legacy Versions > June11 > June11 Update
Jump to: navigation, search

Overview

The following update fixes issues after the June11 release. To sort the entries in descending or ascending order, click the icon in the column heading.

This release is no longer recommended for upgrades. If you want to upgrade an instance, consider a current version. For the June11 Update release:

Build date: 06-17-2011
Build tag: glide-june2011-05-25-2011
 Note: See how to determine the version on your instance.



Number Category Details Steps to Reproduce
PRB561374 Approvals If using the legacy approval engine (not workflows, not process guides), upon approval submission the target (to be approved) document not properly updated to approved state.
PRB561392 Discovery If a Discovery could not resolve a Windows host name the platform produced a null pointer error. Discovery can now deal with a null value without producing an error.
PRB561307 Email Client Allow non-admin users to attach documents to outbound emails via the Email Client.
PRB561430 Email Client The incorrect user is being associated with newly created tasks from inbound email actions. The following error message appears in the log:

WARNING *** WARNING *** No access to impersonate


PRB561346 Knowledge Base Clicking on the Create Knowledge UI Action in Document Management displayed the Form not found error. UI Action was in the wrong plugin and was moved to the Knowledge Document plugin.
PRB561354 Knowledge Base If the comments section is disabled on Knowledge articles, users submitting feedback on an article are unexpectedly redirected to a page with the message: html value empty.
  1. Hide the comments from the Knowledge Base articles.
    a. Navigate to Knowledge Base > Properties.
    b. Set the value of Show user comments on knowledge articles to Never or set the sys_properties field to glide.knowman.show_user_feedback=onload.
  2. Go to Knowledge Base > View > select any article > submit feedback.
PRB500380 Lists Adding or creating a label in List v2 does not work.
  1. Use Spring 2010 Stable 3 and Internet Explorer 6 or 7.
  2. Navigate to any list.
  3. Check a record, and choose New... under Assign Label: from Actions on Selected Rows.
  4. Type in a label and click OK.
    Note that the label is not created. This also does not work for a label that has already been created.
PRB561344 Lists Restored ability of ESS users to use Filter Out and Show Matching sorting options in lists.
  1. Log in as an ESS user.
  2. Go to a list (My Incidents).
  3. Right-click on any element of the list.
Filter Out and Show Matching do not appear in the context menu.
PRB561209 Platform Currency conversion based on the user's locale is not accurate on Catalog Variables.
  1. Log into demo.service-now.com.
  2. Create two users, one with Country Code US and the other with UK.
  3. Create an item, such as a Blackberry or computer, and set the price to 1000.00 (just an example, but any price does the same thing).
  4. Open that item via the Service Catalog with both users and note the prices.
    Our conversion is inaccurate.
PRB561300 Platform A potential JVM Deadlock between TableRotationWatcher and EventScheduler threads was removed.
PRB561312 Platform The g:evaluate locals attribute was restored to the evaluate macro tag.
PRB561340 Platform Issue with auditing on journaled fields is causing problems with Work Notes.
  1. Log into demo.service-now.com.
  2. Go to the module Tasks under the Service Catalog application
  3. Go on any tasks
  4. Write some work notes and save.
  5. Write some work notes and save again.
    Each time you write a new work note and save it, the previous work notes appear an extra time.
PRB561425 Platform Older sys_update_xml records were being recreated with the same information after upgrading to the June 2011 release. Upgrade to June 2011 and look at the sys_update_xml table around the time that the upgrade started.
PRB561383 Rota Rota.next() now returns a correct value.
  1. Create a new Rota for an existing group with members (On-Call rotation > Create New Rota).
  2. Set schedule type to 24x7 and Rotation start date to the current date.
  3. Execute the following script from scripts background:
var a = new OnCallRotation(); 
a.who("<sys_id of the group>") 
gs.print(a.next())
This erroneously prints false. If you take the same steps with the Winter 2011 release, it will print true.
PRB561227 Security With High Security Settings plugin active and the Allow ESS users the option to cancel their requests from the checkout screen. property set to true in the Service Catalog, ESS users' attempts to cancel a request redirect to a blank page.
  1. Log into demo.service-now.com as an admin user
    The High Security Settings plugin should not be activated.
  2. Go to Service Catalog > Properties.
  3. Set the Allow ESS users the option to cancel their requests from the checkout screen property to true.
  4. Log out and log in again as an ESS user.
  5. Order an item.
  6. Click the Cancel button on the Checkout summary page.
    You are returned to the Service Catalog home page.
  7. Log out and log in again as an admin user.
  8. Enable the High Security Settings plugin.
  9. Log out and log in again as an ESS user.
  10. Order an item.
  11. Click the Cancel button on the Checkout summary page.
    A blank page appears.
PRB561360 Security During Encryption Wrapper conversion, continue conversion if we run into a password2 field that contains undecryptable (invalid) data. We will note which table, field, and sys_id has invalid data in the log. If we do not continue the conversion, we end up with a partially converted database which means it is only partially functional, and the instance must then be restored from backup.
PRB561315 SOAP SOAP requests which ran for > 2:40 could potentially consume and never release a semaphore, eventually locking the instance. Added additional functionality to support long running SOAP requests.
PRB561341 SOAP Restored the ability to request WSDL via the lowercase URL pattern: ?wsdl
PRB561376 SOAP
  • Fixed NPE in some .NET SOAP clients due to missing namespace prefixes in the SOAP response document.
  • Properly sort WSDL. Complex types were not being sorted.

PRB561198 UI Disabled fields are now being serialized properly for reference qualifiers. See demoamc.service-now.com.
  1. Click Incident > Create New
  2. Set a value for Required User field.
Note that the field becomes Read-Only.
  1. Click the reference lookup for the User Field.
Notice that no users are returned.
  1. Turn off the Turn ReadOnly User Field if Empty UI Policy.
  2. Repeat steps 1-4.
Notice that now only the user specified by Required User field is returned.
PRB561301 Zing, Indexing Reduce the likelihood of out-of-memory issues during indexing by:
  1. Limiting the number of tokens indexed per document to 250,000.
  2. No more than 4M of data per field.
  3. No more than 16M of data per attachment.
  1. Text index an instance that has abnormally large records/documents.
  2. Observe out-of-memory error.
PRB561316 Zing, Indexing When text indexing an index with a very small database connection pool, always leave at least 4 database connections for the UI.
PRB561321 Zing, Indexing Prevent users from trying to text index certain, massive system tables, which could impact performance:
  • sys_email
  • sys_audit
  • sysevent,syslog,sys_import_set_row
  • sys_replication_queue
PRB561372 Zing, Search Search string containing punctuation were not being properly parsed e.g. "alpha-beta". Search for:
  1. 4.133
  2. testing-the-dash
  3. dash<today
  4. and=dash
