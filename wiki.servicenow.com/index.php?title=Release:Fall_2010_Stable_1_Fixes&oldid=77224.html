1 Fall 2010 Stable 1 Fixes


1.1 PRB517230 — Issues with calls to GlideSession (query condition evaluation)

gs.getSession() now returns a stub object which supports isInteractive(). Other session level activities are still disabled within the script sandbox.

1.2 PRB513624 — Issues with onSubmit Client Script

Notification of failed client scripts have been improved. Errors in the Client Script will need to be fixed.

1.3 PRB511721 -- Dependent Fields not working properly in list editor

Dependent field checking has been enabled.

1.4 PRB513719 -- Print button on home pages doesn’t work

The print button now works as expected.

1.5 PRB516927 — SDLC, Release V2 and SCRUM are not installable by an admin

The Release Management Plugins are now available for installation by administrators.

1.6 PRB516927 — SMTP Sender Job throwing repeated OOM errors post upgrade

The SMTP sender job is no longer flagged debug.

1.7 PRB459183 (List v2): Blue nav arrows missing when opening record from list using preview icon

Blue record navigation arrows are now present when opening a record from a list using the preview icon for the row.

1.8 PRB492787 (Data Sources): Invalid ConnectionURL generated for DB2 JDBC data sources

In the connectionURL Script Include, the code to build the DB2 JDBC connection URL is now valid.

1.9 PRB456769 (Import Sets): Table name length not properly restricted

Import Set tables are now properly created with a maximum table name length of 30.

1.10 PRB482909 (Search): Text search query not correct when part of a top-level OR condition

Text search queries now return proper results when part of a top-level OR condition. e.g., (category=software and keywords=outlook) OR (category=hardware)

1.11 PRB491868 (i18n): Localized number format not processed correctly when creating a chart

Localized number format (e.g., 1.000,95 vs. 1,000.95) is now processed correctly when creating a chart.

1.12 PRB508015 (Knowledge): Email navigation add-on link did not work as expected

The Email navigation add-on link at the top of the knowledge base now passes correct URL parameters to populate the Email Client window with expected information.

1.13 PRB505973 (Domain): Homepages were not rendered in the domain selected in the domain picker

Homepages are now rendered in the domain selected in the domain picker, instead of the user's home domain.

1.14 PRB462893 (Database Views): Filters using a top-level OR condition on a Database View returned incorrect results

Filters using a top-level OR condition on a Database View now return correct results.

1.15 PRB100662 (IE6): Hidden fields' dropdowns overlayed into the next field

In the obsolete IE6 browser, a UI Policy that hides a field will no longer leave an artifact overlaying into the next field on the form.

1.16 PRB494482 (Reports): Scheduled report might send using the wrong timezone

Scheduled reports now correctly send using the specified system timezone.

1.17 PRB458165 (Templates): Filter Out might not work on inactive Templates

The active field on sys_template records is now set to false where the value was null, and Show Matching and Filter Out works as expected.

1.18 PRB516927 (SDLC): Plugin not published as indicated in the wiki

The SDLC plugin has now been published and available to admins.

1.19 PRB515297 (Email): Email properties overwritten during upgrade

glide.email.smtp.active and glide.email.read.active property values are no longer overwritten in an upgrade.

1.20 PRB519043 (Script): Reference fields could not be blanked out via script when updating a base class

Reference fields can now be set to empty when updating a base class. e.g., if a GlideRecord variable is for task, and an incident is being updated, gr.assigned_to = ""; now works as expected.

1.21 PRB446314 (IP Access): IP Address Access Control only checked first IP address when multiple ID addresses are present

Using the XFF standard, the format of the header is: X-Forwarded-For: client1, proxy1, proxy2. In this case the last entry proxy2 would be the most reliable IP address, connected to the last proxy. With IP Address Access Control enabled, now all addresses are checked to determine if the user should be allowed access.

1.22 PRB517064 (List v2): Personalize List Layout was not available for the sys_ref_list view

Personalize List Layout is once again available for the sys_ref_list view in List v2.

1.23 PRB521791 (Dependency): Reference field dependency might be ignored during list editing

Reference field dependency is now honored in all cases during list editing. e.g., if Assigned to is dependent on Assignment group, this is reflected when list editing the Assigned to field.

1.24 PRB526288 (UI): Using the magnifying glass and Searching for a unique task number broke the selection action

Using the magnifying glass to get a list of tasks, and subsequently searching for a unique task number will no longer take you directly to the form for the task, and reference field selection works as expected.

1.25 PRB517038 (Variables): "Variables on Task Form" in Catalog Task activity did not show any variables

"Variables on Task Form" in Catalog Task activity once again correctly show the expected variables.

1.26 PRB525920 (UI): Form with too many buttons would not probably display them with the new fixed form header

Forms with a large number of UI Action buttons now display all buttons properly, with horizontal scrolling available if necessary.

1.27 PRB516983 (Reports): Reports that grouped on a derived field and stacked on another no longer worked properly

Reports that are grouped on a derived field (like caller_id.notification) and stacked on another (like category) once again work as expected.

1.28 PRB453853 (Export): Customer proxy server configuration might break list exports

Polling job_id is now truncated in the case where proxy servers may add padding that would cause sys_id to be un-trimmable, allowing list exports to work as expected in these rare configurations.

1.29 PRB442048 (Database Views): Personalize Dictionary on a database view list showed an empty list

Personalize options are no longer available for Database Views, since they are not supported.

1.30 PRB543953 (Data): Duplicate record created on update of existing one

Duplicate record is no longer created when using the hover icon to drill through to a referenced record (Introduced Fall 2010).

1.31 PRB518718 (Global Search): List personalization "cog" changes not honored in Global Text Search result lists

List personalization "cog" has been removed from Global Text Search result lists, since it is not supported.

1.32 PRB523808 (Service Portfolio): Adding new commitment to Service Offering did not calculate historical availability

Adding a new Availability commitment to a Service Offering will now calculate the last year's worth of Availability intervals, for immediate display in gauges. This is done with a new Business Rule on service_offering_commitment.

1.33 PRB523813 (Service Portfolio): 100% availability might still appear red in the availability gauge due to rounding

A new field, Precision, has been added to the Commitment form to designate how many decimal places should be displayed in the homepage gauge.

1.34 PRB523509 (Welcome Page): HTML tags were no longer displayed in welcome page titles (introduced Fall 2010)

HTML tags are once again rendered in Welcome Page titles.

1.35 PRB441695 (Search Form): Search Forms could not correctly search for choice list labels with an ampersand

Search Form modules can now correctly search for choice list labels that have an ampersand in them (e.g., a category of "LAN & Network").

1.36 PRB518532 (Spell Check): Spell checking multiple fields generated an error

Multiple fields can now be spell checked one after another without issue.

1.37 PRB553140 (Oracle): A query helper business rule that returned more than 1000 values resulted in a query error on Oracle

A query helper business rule function that returns more than 1000 sys_id values in an array once again works properly on Oracle.

1.38 PRB521322 (List v2): Wrap column text personalization option did not stick in IE

The "wrap column text" list personalization option now works as expected in List v2 using IE.

1.39 PRB453825 (Workflow): Order of workflow activity conditions not preserved when a new one was added

Workflow activity condition order is now preserved when a new condition is added. The new condition is given an order greater than the highest existing condition order for the activity.

1.40 PRB503616 (System): Creating a "system" user record in the database could break functionality

Creating a user record with a user_name value of "system" will no longer prevent system functions from being able to perform their functions.

1.41 PRB201989 (UI): Activity formatter checkboxes duplicated in IE

Activity formatter checkboxes are no longer duplicated in IE.

1.42 PRB516854 (Reports): Links in Custom Charts did not work properly

Custom Chart links now have the correct URL to link to the correct list of records represented by the thing being clicked on.

1.43 PRB488696 (Login): Legacy /portal/login.htm page did not accept ampersand in password

Inputs on the legacy /portal/login.html page now work properly with ampersands.

1.44 PRB544787 (Reports): Some published Report URLs did not work in Oracle

A query going after an array field (roles) for a table that has a CLOB type field failed in Oracle. Such a query now returns proper results.

1.45 PRB480933 (Web Services): SOAP Message WSDL Parser did not properly namespace messages

The SOAP Message is now generated properly with proper namespacing.

1.46 PRB535594 (UI): Many-to-many related list might not be editable if m2m table had other references

Many-to-many related lists are now editable regardless of whether the m2m table has references other than the two required for the m2m relationship.

1.47 PRB539397 (Variables): tree_picker attribute did not work as expected for reference variables

The tree_picker attribute now works as expected for reference variables.

1.48 PRB517069 (UI): Ampersand in a form section caption (title) caused the form not to render

Ampersands are now permitted in Form Section titles.

1.49 PRB555241 (UI): Paging not working as expected with grouped lists

Paging now works as expected with grouped lists in both List v1 and List v2.

1.50 PRB453996 (Data): HR and Facilities records did not get their Active field correctly set after a state change

HR and Facilities records now get their Active field correctly set after a state change. New business rules take care of this for those two tables.

1.51 PRB557647 (Email): email.read event processing would generate an error if the email's insert was aborted

The email.read event is no longer created if insert of the inbound sys_email record is aborted.

1.52 PRB450829 (Report): Users could view non-global reports if they had the correct roles, despite not being in the "Visible to" group or the specified "User"

Users are no longer to view reports unless the report is global, the user is in the "Visible to" group, or the user is the "Visible to" user.

1.53 PRB544543 (Timeline Pages): Custom date format might not display properly in Timeline Pages

Custom date formats now display as expected in Timeline Pages.

1.54 PRB550158 (UI): Save with Form List Editing plugin was not saving records properly with the form

Save with Form List Editing plugin once again saves records properly with the form

1.55 PRB339761 (Timeline Pages): Changing the Table on the Timeline Page form did not properly update the list of available date/time fields

Changing the Table on the Timeline Page form now properly updates the list of available date/time fields

1.56 PRB432442 (UI): glide.xmlhttp.ac_wait_time property did not work properly

This property defines how long in milliseconds to wait before querying for reference field autocomplete results. This property now works as expected.

1.57 PRB557995 (Timeline Pages): New timeline pages did not support glide_date fields

New Timeline Pages now support both glide_date_time and glide_date fields.

1.58 PRB492444 (Service Catalog): Content Items set to open "Within Catalog" did not always do so

"External Content" Content Items that are set to open Within Catalog now always correctly open within the catalog. Previously they incorrectly opened in a new window if the items were in a subcategory.

1.59 PRB558998 (Query): Reference qualifier with a compound condition might return wrong results in autocomplete

A reference qualifier with a compound condition involving more than one table and including sys_id now returns the correct result in autocomplete. Example of one that might fail previously: companySTARTSWITHservice^sys_idIN[comma separated list of sys_id values]

1.60 PRB559004 (Discovery): Some software might not be discovered on Windows machines

Installed software that does not show up in the uninstall registries for whatever reason, but nevertheless shows up in Add/Remove Programs is now properly discovered.

1.61 PRB538592 (Browser): Excessive URL length caused problems in reference field lookup

The unwound query string is no longer sent out on reference lists. The server has the reference qualifier and evaluates it there. Passing it back and forth caused browser URL limit problems.

1.62 PRB459180 (Spell Check): Spell checking did not work

If no dictionary was selected for non-English instances, the platform did not default to the English dictionary and spell checking was unavailable.
