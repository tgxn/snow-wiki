Scripting in Scoped Applications

1 Overview

The scoped GlideTime class provides methods for performing operations on GlideTime objects, such as instantiating GlideTime objects or working with GlideTime fields.

The scoped GlideTime API is available starting with the Fuji release.

NEW: You can also find this class documented in the easy-to-read API reference on the Developer Program web site: GlideTime.

3 Constructors

3.1 GlideTime(Number ms)

Instantiates a GlideTime object with the specified datetime. If no parameter is specified, the current time is used.

Parameters:

  • ms - (Number) the datetime, in milliseconds.

Example 1: 

var gt = new GlideTime(10000); 
gs.info(gt.getDisplayValue());

Output: 

16:00:10

Example 2: No parameter 

//User's current time is 8:45:27
var gt = new GlideTime(); 
gs.info(gt.getDisplayValue());

Output: 

08:45:27

4 Method Details

4.1 getByFormat(String format)

Gets the time in the given time format.

Parameters:

  • format - (String) the time format.

Returns:

String - the time in the specified time format.

Example: 

var gt = new GlideTime(); 
gt.setValue('12:00:00');
gs.info(gt.getByFormat("HH:mm"));

Output: 

12:00

4.2 getDisplayValue()

Gets the time in the current user's display format and time zone.

Parameters:

  • None

Returns:

String - the time in the user's format and time zone. Keep in mind when designing business rules or script includes that this method may return values in different formats for different users.

Example: 

var gt = new GlideTime();
gt.setDisplayValue("12:00:00"); // User Time Zone
gs.info(gt.getDisplayValue()); // User Time Zone

Output: 

12:00:00

4.3 getDisplayValueInternal()

Gets the display value in the current user's time zone and the internal format (HH:mm:ss). Useful for date/time fields, but not for date fields.

Parameters:

  • None

Returns:

String - the time value for the GlideTime object in the current user's time zone and the internal time format of HH:mm:ss.

Example: 

var gt = new GlideTime();
gt.setValue("01:00:00"); //Internal Time Zone , UTC
gs.info(gt.getDisplayValueInternal()); //User Time Zone

Output: 

17:00:00

4.4 setDisplayValue(String asDisplayed)

Sets a time value using the current user's display format and time zone.

Parameters:

  • asDisplayed - (String) the time in the current user's display format and time zone. The parameter must be formatted using the current user's preferred display format, such as HH:mm:ss.

Returns:

void

Example: 

var gt = new GlideTime();
gt.setDisplayValue('01:00:00');   // User Time Zone
gs.info(gt.getDisplayValueInternal()); // User Time Zone

Output: 

01:00:00

4.5 getValue()

Gets the time value stored in the database by the GlideTime object in the internal format, HH:mm:ss, and the system time zone, UTC by default.

Parameters:

  • None

Returns:

String - the time value in the internal format and system time zone.

Example: 

var gt = new GlideTime();
gs.info(gt.getValue()); // Internal Time Zone, UTC

Output: 

2014-10-22 20:43:53

4.6 setValue(String o)

Sets the time of the GlideTime object in the internal time zone, which is UTC by default or the value of the glide.sys.internal.tz property, if set.

Parameters:

  • o - (String) the time in hh:mm:ss format.

Returns:

void

Example: 

var gt = new GlideTime();
gt.setValue('01:00:00');  //Internal Time Zone, UTC
gs.info("time is "+ gt.getByFormat('hh:mm:ss'));

Output: 

time is 01:00:00

4.7 subtract(GlideTime start, GlideTime end)

Gets the duration difference between two GlideTime values.

Parameters:

  • start - (GlideTime) the start value.
  • end - (GlideTime) the end value.

Returns:

GlideDuration - the duration between the two values.

Example: 

var gd1 = new GlideTime(); 
gd1.setDisplayValue("09:00:00"); 
var gd2 = new GlideTime(); 
gd2.setDisplayValue("09:10:00"); 

var dur = GlideDate.subtract(gd1, gd2); //the difference between gdt1 and gdt2 
gs.info(dur.getDisplayValue());

Output: 

10 Minutes
