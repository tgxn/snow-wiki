 The current recommended release is Berlin Patch 8 Hotfix 1. The current general availability release for the Aspen family is Aspen Patch 6.


This release includes the following minor enhancements or performance improvements. This page contains the accumulated Notable Changes for the Aspen release.


Feature Description
Common Practice
Forms Layout
  • Watch List and Work Notes fields are now located at the top of Notes on incident and change forms.
  • Knowledge field is now located under Closure Information on incident forms.
Incident Resolution Workflow If the caller closes an incident:
  • An infoMessage with a link to the incident is provided.
  • Close code and close notes are automatically set by an added business rule.
KPI Reports
  • Last 12 months date filter is now available on two KPI reports.
  • Best Practice - ITIL KPI Reports plugin is active for new instances.
Macro Renamed The show_related_tasks UI macro is now called show_related_records.
Discovery
Clustered Process Discovery Support for clustered process discovery has been generalized to extend beyond Microsoft SQL Server.
Microsoft SQL Server Instance Names Discovery correctly returns all Microsoft SQL Server instances (with their actual SQL names) that are running on the same machine and their corresponding catalogs (using the SQLDMO object). Also, for SQL 2000 servers, Discovery can match instance names by looking up the process ID.
MID Server
JAR Synchronization An extlib directory has been added by default to the MID Server for JAR files synchronized from the instance. Users can now upload JAR files and synchronize them to all MID Servers. Custom probes can be written to use the synchronized JAR files.
Upgrading and Testing The MID Server now:
  • Upgrades automatically upon instance upgrade.
  • Tests connectivity through a public scripted Web Service.
Integrations
MID Server The mid_server role has been added to the Read ACL for the sys_data_source table.
LDAP Server The Advanced view of the LDAP Server form is no longer created. Advanced fields are now available from the Default view by selecting Advanced view under Related Links.
Notifications
Inbound Email Actions A new reply Inbound Email Actions has been added for the problem, problem_task, change_request, change_task, and sc_req_item tables (available on instances created after June 3, 2011).
Email Notifications and Templates In several default email notifications and templates, ${URI} has been replaced with ${URI_REF} where appropriate. Customized notifications do not get the change.
New Email Notifications Nineteen new email notifications were added for comment/work note events and task assignments in the change_task, problem_task, problem, and change_request tables (available on instances created after June 3, 2011).
Platform Administration
Embedded HTML Images Images available for embedding in HTML fields now show a category, which can be viewed in the lookup list. Also, the lookup list displays a thumbnail view of the image.
Attachment Renaming Event A new event called attachment.renamed is triggered when an attachment on a record is renamed.
Field Maximum Length for Variables A new max_length attribute is available for Single Line Text and Wide Single Line Text variable types. This allows you to set a maximum length, limiting the number of characters in the field.
SNMP Synching SNMP MIB uses a new synching method. This change should not negatively affect existing custom MIBs. Remote attachments are no longer used.
Wizards - Edit Button An Edit Panel button is now available on wizard panels. This button appears if the user can write to the expert record.
Wizards - Redirect Panel A redirect wizard panel type is now available. The redirect panel specifies a URL to which the user is taken upon transition to the panel.
Workflow An example workflow is now available for resetting a user password.
Inbound Email Actions Two new fields are available in the Inbound Email Actions form:
  • Execution order: Enter a number to define the order in which this email action should be processed. Lower numbers are processed first.
  • Stop processing: Select this check box to stop processing of the email action when the rule produces a row in the table.
List Record Count This release adds list support to display the total record count for grouped lists. An example would be to show 57 total Incidents just below the paging controls in the list. This is shown only when the list is grouped on a field.
Upgrades Customer updates (sys_update_xml records) with no Update Set value were being ignored during an upgrade, so customizations could get overwritten. Such updates are no longer ignored.
Filter Labels In filters, the label Apply Filter was changed to Run and Save Filter label was changed to Save... for consistency.
High Security XML escaping behavior can be modified only by users with the elevated security_admin role.
Arranging Fields with the Slushbucket Moving a field from Available to Selected in the form/list slushbucket now places the new field just after the selected field instead of always at the bottom. This reduces the number of up clicks when creating a form.
Form Annotation Module Form Annotations are enabled by default for instances created after June 3, 2011. Two new tables have been added: Form Annotations [sys_ui_annotation] and Form Annotation Types [sys_ui_annotation_type]. A new module called Form Annotation Types has been added under System UI.
Enabling Form Annotation The form/list personalization slushbucket layout has been modified to accommodate enabling of Form Annotations. This change eliminates the "Create new field" button, which was present when annotations were enabled previously, and moves field creation back to the main slushbucket form for everyone. This change moves annotation customization options to the right of the Selected side of the slushbucket. This update is designed to fit everything without scrollbars on a 1280px width screen.
List Controls List Control for the New button was added for the Roles related list on the User and Group forms and on the UI Action Visibility related list on the UI Action form.
Enabling and Disabling List Highlighting All users can enable or disable list highlighting with a check box in the Personalize List Column slushbucket. List highlighting removes the static, alternate row highlighting and applies highlighting to a row as the cursor moves over that row. Click the gear icon in the upper left corner of a list to personalize the list.
Update Sets A UI Policy has been added to make sys_m2m fields mandatory.
User Interface Toolbar Display The HTML field toolbar is now in a DIV rather than a TABLE. This allows the toolbar buttons to wrap instead of pushing the form out to the right.
User Interface Toolbar Configuration HTML field toolbar buttons can now be configured with the glide.ui.html.toolbar property, found in System Properties > UI Properties with the following description: Configures the editing toolbar for HTML fields.
Application and Module Tables Show Complete Contents in Lists Application and Module tables' Roles and Hints fields have been given the no_truncate attribute, so they show their complete contents in a list.
Editing UI Actions UI Actions (buttons, links, etc.) now have an Edit UI Action context menu option that is available to users with the ui_action_admin or admin role.
Reference Record Pop-up Hovering over the diamond icon on a Glide List field now displays a pop-up of the selected referenced record, just like a Reference field pop-up.
Form Templates Lookup When applying form templates, users now select a template from a lookup list that references the Templates table. Instances with a large number of templates were experiencing a performance issue that rendered all their templates in the form context menu. This change includes a new version of the Read ACL for sys_template and the template query business rules to ensure that users can see applicable templates.
Customizing Banner Logo Two new properties control customization of the URL and target frame used when a user clicks the banner logo: glide.banner.image.url and glide.banner.image.url_target.
Schedules The cmn_schedule_blackout.description column has been moved to the Schedule [cmn_schedule] table.
Showing Schedules There is now a Show Schedule UI action link on the Schedule form.
Interceptors - UI Action The Try It UI action was added for interceptors (sys_wizard table). Access the UI action from a form button or a list context menu.
Interceptors - Edit Button An Edit Interceptor button is now available on interceptor (wizard) panels. This button appears if the user can write to the sys_wizard record.
Interceptors - Synchronize Roles Two new business rules, Sync CI answers with CI modules and Sync Interceptor Answers with CI Modules, synchronize roles and active fields for table-specific CMDB modules with their corresponding cmdb_ci.do interceptor answers. For example, if you deactivate the Tomcat module in the CMDB application or modify the roles, do the same to the Tomcat interceptor option for the cmdb_ci interceptor. This behavior is controlled by a new property, glide.ecmdb.synch_modules_with_interceptor, which is set to true by default. The business rules and property are available for instances created after the June 2011 release.
Show Related Tasks Decoration The show_related_tasks decoration is enabled on the task.cmdb_ci table by default for new instances. Existing instances can modify the ref_contributions attribute for the field to add the decoration. Ref contributions must be semicolon-separated: for example, ref_contributions=task_show_ci_map;show_related_tasks.
Show Related Records Decoration A UI macro was added to create a show_related_records decoration. When added to the cmdb_ci field (Incident form), the UI macro displays a decoration if the selected CI has active related tasks (related by the task_ci m2m table). Click the decoration to display a GlideDialogWindow showing a list of those tasks grouped by class.
Deactivate Multiple Modules Easily A new UI action called Change active state on the sys_app_module table adds a list choice action and an option in the list context menu to deactivate multiple modules at once.
Versions Related List Added to Several Forms The Compare to Previous link on Client Scripts, UI Actions, Business Rules, UI Pages and Script Includes has been replaced by the Versions related list. Users can use this list to compare specific versions of a record with the Compare option in the list choice menu.
Discovery Source Table An object source table called Source [sys_object_source] is now available. This table tags CIs with information identifying the source of a discovery (currently, only ServiceNow), the ID of that source, and the date/time of the last scan. This table is populated automatically.
Process Applications - Service Catalog
Order Guides A new Order to cart field is available for configuring order guides. This feature allows users to add order guides to their cart and then continue shopping. Previously, users had to check out immediately after completing an order guide.

Note: This only works when the Two step check box is also set to true.

Service Catalog Scripting Callback function support for ServiceCatalogForm.getReference is now available.
Preview Catalog Item The Preview item link is now disabled for inactive catalog items.
Process Applications - Change Management
Bulk CI Changes CI Class and Proposed Changes fields are no longer added to the Change Request form when the Bulk CI Changes plugin is activated. They can be added by personalizing the form after plugin activation.
Affected CIs Related List The Applied and Applied date fields have been removed from the Affected CIs related list (they are only relevant when working with proposed changes), and CI Class has been added. Customized related lists will not get the change.
Change Management Collision Detector - Field Requirements The start_date and end_date fields are no longer mandatory as part of the Change Management Collision Detector plugin.
Change Management Collision Detector - Change Date Tracking The last_checked field has been added to Conflict records for the Change Management Collision Detector plugin. This indicates when the change was last checked for conflicts. The default value is now.
Change Management Collision Detector - Field Label Change The name of the Collision Properties module has been changed to Conflict Properties. The role required to access Conflict Properties was changed from itil to itil_admin.
Change Management Collision Detector - Role Change The read ACL for the conflict table now require the itil role. This lets ITIL users see the conflict records when the High Security Settings plugin is activated. The itil role is required for users to see the Check Conflicts UI Action.
Change Management Collision Detector - ACL Changes Added write, create, and delete ACLs that require the admin role for the Conflict table, which is included with the Change Management Collision Detector plugin. Other users do not require direct access to create, write, or delete conflict records.
Process Applications - Incident Management
Reopening Incidents ESS users now see a Reopen Incident button on resolved incidents (available on instances created after June 3, 2011).
Inbound Email Actions Incidents created from Inbound Email Actions now set the Contact type field of the resulting Incident to Email.
Record Producer The Create a New Incident record producer has been modified to set the Contact type field of the resulting Incident to Self-service.
ESS User View The Watch list, Incident state, and Impact fields have been added to the ESS view of the Incident form and the variable formatter has been removed. ESS users have write access to the Watch list and Impact fields (available on instances created after June 3, 2011).
Best Practice - Incident Resolution Workflow plugin The Best Practice - Incident Resolution Workflow plugin can now be activated by a user with the admin role.
Process Applications - IT Governance Risk and Compliance
IT Governance Risk and Compliance Plugin The new Applies to field (type document_id) has been added to the grc_control table to allow for identification of the scope. The form and list layout are modified as a result. A property was also added to control tables available to the field.
IT Governance Risk and Compliance - Audits The following enhancements have been made to audits:
  • The new Audit Activity [grc_activity] table allows users performing an audit to assign and track tasks and activities within an audit.
  • The new Audit Requirements [grc_audit_requirement] table allows auditors to specify which Authoritative Source Content [grc_authortiative_source] sections are required for compliance during an audit instance. This audit requirement is associated with an audit definition. When an audit instance is created, the requirement is attached to the audit instance to allow auditors to mark compliance or non-compliance.
IT Governance Risk and Compliance - Audits Administrators now have the ability to track Authoritative Source requirements in audits. Requirements can be set up in the audit definition for reuse.
Process Applications - Project Management and Project Portfolio Management
Project Management Skills Related List Activating Project Management v2 no longer adds the Skills related list to the User form.
Using an Existing Project as a Template A new feature has been added to Project Management to use an existing project as a template/model to create a new base project. This UI action from a Project list or form makes a copy of the current project and all project tasks and prompts for a new project name and start date. All tasks are reset to Pending with no actual start/end date or percent complete values.
Project Management Tasks Project tasks can be created directly from the Project Task list with the list edit row insert option.
Project Management Permissions The project_manager and project_user roles now have project and project task read access by default.
Project Portfolio Management Permissions All PPM related tables now have security defined using PPM roles. The new project_portfolio_user role enables project manager and project portfolio user to view pm_portfolio and related records.
Process Applications - Other
BSM Maps Deleting a BSM map view now now deletes any related map filters.
BSM Maps Upstream relationships from clusters are now displayed in BSM maps.
Knowledge Management The Attach to Incident button on knowledge articles has been changed to reflect the relevant table. For example, when searching the knowledge base from a Change Request, the button is labeled Attach to Change Request. If the table cannot be determined due to customer modification of knowledge base UI pages or UI macros, the button is labeled Attach to record.
Managed Documents The Knowledge Document plugin now extends the Managed Documents application by providing the ability to publish managed documents to the knowledge base.
Managed Documents As of this release, the Managed Documents application can be activated by administrators.
Change and Problem Form Views This release provides revised versions of Change, Change Task, Problem, and Problem Task form views to make the field layout more consistent with other forms (available on instances created after June 3, 2011).
Certification Task preview link A new UI Action on the Certification Schedule form, called Preview Certification Tasks, saves and reloads the current record, simulates execution of the certification schedule, and displays an infoMessage about which certification tasks would have been created on execution.
Affected CIs Business Rules Two new business rules have been added for instances created after the June 2011 release:
  • Sync CI with Affected CIs: A rule on the Task table copies the CI on the form to the Affected CIs related list and keeps it synchronized. If the CI on the form changes, the old one is removed from the related list and the new one is added.
  • Prevent removal/update of primary CI: A rule on the task_ci table prevents removal of an Affected CI if it is for the CI on the task form, keeping things synchronized.
Reporting
Pivot Tables If a pivot table's aggregate is average, then Average is used instead of Total for the final column/row label.
KPI Reports
  • Added a grid to the KPI - Number of Incidents per Month by Category line chart report and filtered it on the last 12 months.
  • Changed KPI - Number of Incidents by Category report to a monthly trend chart filtered on the last 12 months.
  • Changed KPI - Incidents Assigned More than Once by Category report to a monthly trend chart filtered on the last 12 months.
  • Changed KPI - Average Incident Resolution Time by Category report to a pivot table.
  • Changed KPI - Number of Incidents per Problem to a list report on Problem that uses the new Related Incidents field. Filtered on active Problems with more than one related incident (available on instances created after June 3, 2011).
  • Changed KPI - Incident Resolution within SLA by Category to a bar chart stacked on SLA stage. Shows both achieved and breached, in green and red.
  • Changed KPI - Average Work Effort for Resolving Incidents by Category report to a bar chart.
  • Changed KPI - Number of Problems by State and CI creates a pivot table report, excluding where the CI is missing. Filtered on the last 12 months.
  • Changed KPI - Average Problem Resolution Time report type to trend chart, filtered on the last 12 months.
  • Changed KPI - Remotely Resolved Incidents report from a bar chart to a trend chart filtered on the last 12 months.
List Reports Fields from extension tables can now be added to list reports (for example, adding Caller ID to a report on task).
Scripting
Javascript JavascriptProbe derivatives are now capable of specifying script prefixes and suffixes.
XMLHelper Script Include
  • The toObject() method now returns an object whose properties are all JavaScript objects. This method now works properly whether the supplied parameter is an XML document or an XML string.
  • The methods toXMLDoc() and toXMLStr() are now available. These methods are the inverse of the existing toObject() method.
  • The toObject() method has been extended to take an optional parameter of the XML input to convert as an alternative to the (still present) mechanism of specifying the XML input in the constructor.
GlideRecord Query Performing a GlideRecord query() against a table with a field named query could generate an error. For these tables, a new _query() method can be used instead, for example, gr._query();. A similar method, _next(), has been added to address the same issue with tables that have a field named next.
GlideSystem Methods Added convenient GlideSystem methods for logging warnings and errors: gs.logWarning(message, [source]) and gs.logError(message, [source]).
GlideForm Function A GlideForm function has been added to return the decimal value of a field: g_form.getDecimalValue(fieldname).
Client Scripts Warnings Two Async Delete Warning client scripts now use showFieldMsg() instead of addInfoMessage() when warning against using current in an asynchronous business rule.
Client Scripts Field A new "Messages" multi-line field has been added to the Client Scripts form. Strings entered here, one per line, are made available to the client script as localized messages using the getMessage(msg) syntax, without the need for an AJAX call. This allows for the internationalization of messages displayed in alerts, for example, without a round trip to the server. As an example, if the Message field has the line Please populate the Reason field, then the client script could have the following line: alert(getMessage("Please populate the Reason field"));, and the user would see it in their language, assuming an entry exists in the sys_ui_message table for their language with that key.
Security
Viewing Watch Lists The incident query business rule has been modified to let ESS users on the watch list see Incidents.
Write ACLs Changed The incident write ACL in the Best Practice - Incident Resolution Workflow plugin has been modified to make Closed incidents read-only for non-administrators.
Templates A new property called glide.template.max_context specifies the maximum number of templates (default: 15) displayed in a form's context menu. If more than this number are available, click the Apply Template context menu option to open the reference list of templates.
Server-side null value handling Empty strings or null values for integer fields evaluate on the server as zero "0" rather than as an empty value " ".
