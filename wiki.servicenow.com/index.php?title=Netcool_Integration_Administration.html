1 Overview

After the integration is set up, administrators can manage it from the Integration - Netcool/OmniBus application.

2 Viewing Integration Status

The integration requires an active MID Server connection to transfer events.

  1. Navigate to Integration - Netcool/OmniBus > Setup.
  2. Click the creation date link of an integration setup record.
    The Status field shows the current state of the integration MID Server. The status is either Up or Down.

3 Viewing Events

The integration tracks integration events in these ways:

  • By maintaining a historical list of all inbound events.
  • By logging outbound events for the Socket Monitor.

3.1 Inbound Events

Inbound Netcool events all have a topic of Netcool.

  1. Navigate to Integration - Netcool/OmniBus > Netcool Events.
  2. Click the created date link to view details of the incoming event.
  3. Click the XML button from the Payload field to view the event record as an XML file.
Sample Netcool event record

3.2 Outbound Events

Outbound Netcool events all start with the string NetcoolUpdater in the name.

  1. Navigate to Integration - Netcool/OmniBus > JDBCProbes.
  2. Click the created date link to view details of the outgoing event.
  3. Click the XML button from the Payload field to view the event record as an XML file.

4 Testing JDBC Connections

  1. Navigate to Integration - Netcool/OMNIbus > JDBC Connections.
  2. Click a connection in the list.
  3. Click the Test Connection related link.
    This link sends a SELECT count(*) test command to count the number of rows in the Netcool Alerts.Status table.
Testing a JDBC connection

If the connection test fails, the integration sets the connection state to Error. The connection tester periodically tests all connections, including ones currently in an Error state, and resets the connection state to Ready if the test passes.

4.1 Scheduling JDBC Connection Tests

  1. Navigate to Integration - Netcool/OMNIbus > Connection Tester
  2. Select the Active check box.
  3. In the Repeat Interval field, set how often the script runs. By default, the script runs every 10 minutes.

The script tests all configured Netcool JDBC connections. If a test fails, the script sets the connection state to Error and includes the error message. If the test passes, the script sets the connection state to Ready and the integration can use this connection on the next update.
