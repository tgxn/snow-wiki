1 Overview

This object describes the default sensor object. All sensors should extend this object.

1.1 Where To Use

Use this object to define your own Discovery sensors.


2 Method Summary

Method Summary Description
initialize(definition) Called by the Prototype JavaScript Framework during object processing.
init(definition) Initializes the sensor.
main() Provides main processing sequence for the script.
run() Runs the sensor.
zeroResults() Override this in the sensor to change the behavior with no results.
process(result) Override this in the sensor to process each result.
prepare() Optionally override this in the sensor to prepare for processing results.
finish() Optionally override this in the sensor to prepare for processing results.
after() Optionally override this in the sensor to finish after processing and saving all results.
handleError(error) Optionally override this in the sensor to handle errors.
handleWarning(warn) Optionally override this in the sensor to handle warnings.
isDebugging() Checks to see if debugging mode is on.
debug(msg) If debugging is turned on, logs the message.
reclassify(ciClass) Reclassifies a device to the specified classification.
update(data) Updates the sensor data and related lists.
save() If there is a device history, saves the processed data from the probe.
logWarningPayload(result) Checks for warnings and logs them.
processWarning(msg) Processes a warning message by ignoring duplicate messages and calling handleWarning(warningMsg).
checkErrorPayload(result) Checks whether an error has occurred, and if so calls the function to process the error.
processError(msg) Processes an error by eliminating duplicates, writing the error to the error map, and sending the error to the error handler.
checkEmptyPayload(result) Checks if the device history is not yet created. If so, do not write to it.
logEmptyPayload() Creates a log entry if the device history is empty.
setCurrentFromJavaMap(javaMap) Sets the current sensor data from a Java map.
getAgent() Gets the agent name for the probe.
getCmdbCi() Gets the Configuration Management Database (CMDB) Configuration Item (CI) of the current device.
getCmdbRecord() Gets the current Configuration Management Database (CMDB) record.
getDeviceCi() Gets the current device's CI record.
getEccQueueId() Gets the queue ID of the current probe.
getEccQueueRecord(name) Gets the ECC Queue record for the current probe.
getParameter() Gets a specified parameter from the Discovery probe.
getSensorName() Gets the name of the current sensor.
getSource() Gets the probe's Source field value.
addToRelatedList(tableName, dataArray, refName, keyName) Adds the CI to the related list.
updateCounts() Updates the status count and the device count.
updateStatusCount() Updates the status count.
updateDeviceCount() Updates the count of completed devices.
setTriggerProbes(value) Sets the trigger value from the probe.
fireAdditionalClassifier() Fires additional classifiers.
addCIDataBaseValue(obj) Exposes the base values in CIData to the classifiers. Currently used by Windows, Unix, and SNMP.
runClassificationScript(dClasser, ciData) Adds variables to the ci_data variable after running the classification script and storing the variables in ci_data.


3 Method Detail

3.1 initialize(definition)

Called by the Prototype JavaScript Framework during object construction. Provide a value for the parameter, but do not call this method directly.

3.1.1 Input Fields

Parameters:

  • definition - The Discovery definition.

3.1.2 Output Fields

Returns: void


3.2 init(definition)

Initializes the sensor.

3.2.1 Input Fields

Parameters:

  • definition - The Discovery definition.

3.2.2 Output Fields

Returns: void


3.3 main()

Provides main processing sequence for the script. Loops recursively until a sys_id is found, then processes the CI.

3.3.1 Input Fields

Parameters:

  • None

3.3.2 Output Fields

Returns: void


3.4 run()

Runs the sensor, including preparation, checking for and logging any warnings and errors, and checking the payload to ensure that a valid sensor is available to process, and calling finish().

3.4.1 Input Fields

Parameters:

  • None

3.4.2 Output Fields

Returns: void


3.5 zeroResults()

Override this in the sensor to change the behavior with no results.

3.5.1 Input Fields

Parameters:

  • None

3.5.2 Output Fields

Returns: void


3.6 process(result)

Override this in the sensor to process each result.

3.6.1 Input Fields

Parameters:

  • result - The result from the probe.

3.6.2 Output Fields

Returns:


3.7 prepare()

Optionally override this in the sensor to prepare for processing results. This method is invoked once before any results have been processed.

3.7.1 Input Fields

Parameters:

  • None

3.7.2 Output Fields

Returns: void

3.8.1 Input Fields

Parameters:

  • None

3.8.2 Output Fields
