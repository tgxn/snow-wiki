1 Overview

Create one or more Amazon Web Services (AWS) accounts (http://aws.amazon.com ), and then configure the ServiceNow instance to communicate with the accounts. Be sure to copy the following information about your AWS accounts for use in the ServiceNow configuration procedures.

  • Account number
  • Access Key ID
  • Secret Access Key

2 Required Roles

Users who are members of the Virtual Provisioning Cloud Administrators group (cloud_admin role) can configure AWS accounts in ServiceNow.

3 Discover AWS Accounts and Resources

After creating your accounts in Amazon, you can discover the AWS accounts and associated resources through the ServiceNow instance. For more information on discovering AWS accounts and resources, see Discovering an Amazon Web Services Cloud.

4 Identify the AWS Account to the Instance

After you create an account in Amazon, you create an account record on the ServiceNow instance with information about the AWS account, including the AWS credentials.

  1. Navigate to Amazon Web Services > Configuration > Accounts.
  2. Click New.
  3. Enter a unique Name for the account.
  4. Enter the Account ID number.
  5. Click Submit.

4.1 Specify AWS credentials on the instance

AWS credentials are associated with AWS accounts and are made up of an access key ID and secret access key, which are used to sign requests that are made to AWS. They provide a layer of security for account access.

  1. Navigate to Amazon Web Services > Configuration > Accounts.
  2. Click on an account.
  3. Click the AWS Credentials tab.
  4. Click New.
  5. Enter a unique Name for the credentials.
  6. Select the Active checkbox to enable the credentials.
    The AWS Account is already selected.
  7. Enter the Access Key ID.
  8. Enter the Secret Access Key.
  9. Click Submit.


5 View AWS Account Details

To view account details, navigate to Amazon Web Services > Configuration > Accounts and click on an AWS account. You can view the following details from within an AWS account:

  • AWS Credentials
  • Discovery Schedules
  • AWS Billing Managements
  • EC2 Key Pairs
  • EC2 Images
  • EC2 Approved Image
  • Shared Image Accounts
  • EC2 Virtual Machine Instances
  • AWS VPCs
  • AWS VPCs Security Group
  • AWS Subnets
  • AWS CloudFormation Stack
  • AWS Elastic Load Balancers
  • AWS Auto Scaling Group
  • AWS Auto Scaling Group Launch Config
  • AWS Elastic Block Store Volumes
  • AWS Elastic Block Store Snapshots

6 Define AWS Policies

To assign permissions to an AWS account you must create a policy, which explicitly lists permissions. A Super Admin Policy is enabled by default.

  1. Navigate to Amazon Web Services > Configuration > Policies.
  2. Click New.
  3. Enter a unique Name for the policy.
  4. Enter the policy parameters using JSON format.


  5. Click Submit.

7 Viewing AWS Security Tokens

AWS security tokens allow you to provide trusted users with temporary, limited access to your AWS resources.

  1. Navigate to Amazon Web Services > Configuration > Security Tokens.
  2. Open a token and review the form fields (see table).
Field Description
Access key ID The access key ID that identifies the temporary security credentials.
Arn Amazon resource number that is used to identify the federated user associated with the credentials.
Account [read-only] The account used to generate a security token.
Expiration The date on which the current credentials expire.
Federated user ID The user ID for the federated user associated with the credentials.
Packed policy size A percentage of the allowed value indicating the size of the policy in packed form.
Secret access key The secret access key that can be used to sign requests.
Session token The token that users must pass to the service APU to use the temporary credentials.

7.1 Viewing Security Tokens Assigned to Users

  1. Navigate to Amazon Web Services > Configuration > User Security Tokens.
  2. Open a token and review the form fields (see table).
Field Description
Token link close time Date and time the link to use the security token will expire.
Token link open time Date and time the link to generate the security token will open.
Policy AWS policy associated with the token.
Purpose Description of the security token purpose.
Request item Indicates if the token is explicitly requested by a user or generated by other processes.
State State of the user token link. It can be scheduled, active, or closed.
Token AWS security token that is tied to the record.
User User to assign the security token to.

8 View Region Settings for the Account

Each region is a separate geographic area and is designed to be completely isolated from the other Amazon regions. This achieves the greatest possible fault tolerance and stability. Amazon provides multiple regions so that you can launch instances in locations that meet your requirements. For example, you might want to launch instances in Europe to be closer to your European customers or to meet legal requirements.

To view the available regions for your AWS account, click on Amazon Web Services > Configuration > Regions.
