1 SSL Certificate Testing

1.1 Instance Testing

An in-depth test can be done by using a host file hack to point to the early deployment IP which will allow full instance communication while using the new SSL certificate.

Warning: This is an advanced procedure. If done incorrectly or not properly cleaned up after use, it will lead to problems communicating with ServiceNow.

1.1.1 Determine the Early Deployment IP

Do a DNS lookup on the instance in question to find the current production IP: 

$ host hi.service-now.com
hi.service-now.com has address 199.91.136.100

Use this table to determine the early deployment IP:

37.98.232.100 37.98.232.102
37.98.233.100 37.98.233.102
37.98.234.100 37.98.234.102
37.98.235.100 37.98.235.102
37.98.236.100 37.98.236.102
103.23.64.100 103.23.64.102
103.23.65.100 103.23.65.102
103.23.66.100 103.23.66.102
103.23.67.100 103.23.67.102
149.96.1.100 149.96.1.102
199.91.136.100 199.91.136.102
199.91.137.100 199.91.137.102
199.91.138.100 199.91.138.102
199.91.139.100 199.91.139.102

199.91.141.100 199.91.141.102

If your production IP is not listed, please contact Customer Support for assistance.

1.1.2 Update Hosts File

Update the hosts file to point to the early deployment IP. This is platform dependent. On Unix-like environment, this is in the /etc/hosts file: 

$ grep 'hi.service-now.com' /etc/hosts
199.91.136.102 hi.service-now.com

On Windows environments, the equivalent file is in %SYSTEMROOT%\drivers\etc\hosts. It uses the same syntax.

Note: You may need to clear your DNS cache and browsing history to see the correct certificate after changing your hosts file.
Warning: When testing is complete, remove the line from the hosts file. It will cause problems if left in place.

1.1.3 Verify Communication to Early Deployment IP

Use either a web browser or a command line utility such as openssl s_client to connect to the instance. Verify the serial number (0x50D4334E) and the expiration date of the certificate (April 1, 2016).

1.1.4 Test Integration and Software

Perform whatever testing is required for the integration or proxy server to verify functionality.

1.1.5 Reset Hosts File

Be sure to remove the host file entry from the update hosts file step. It will cause problems if left in place.

Note: You may need to clear your DNS cache and browsing history to see the correct certificate after changing your hosts file.
