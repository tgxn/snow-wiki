1 Overview

When the homepage splash page is configured, a lightweight splash page loads before a homepage loads. The result is a more responsive experience for users. For example, after logging in, users can navigate directly to the information they need without waiting for the homepage to load.

Homepage splash page

2 Activating the Plugin

The plugin is automatically activated for new instances. For existing instances, activate the Homepage Splash Page plugin.

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.

3 Configuring Properties for Homepage Splash Pages

The Homepage Splash Page plugin installs the glide.home.page property and modifies the value of the glide.login.home property. To configure these properties:

  1. Enter sys_properties.list in the navigation filter.
  2. Locate the glide.home.page system property. This property defines which page to load when a user selects a homepage from the banner.
  3. Set the Value as follows:
    • To use the homepage splash page (default): home_splash.do?sysparm_direct=true
    • To use homepages without the splash page: home.do
  4. Locate the glide.login.home system property. This property defines which page to load when a user logs in.
  5. Set the Value as follows:
    • To use the homepage splash page (default): home_splash.do?sysparm_direct=true
    • To use homepages without the splash page: home.do
    • To specify a different login landing page, see Specifying a Login Landing Page.
