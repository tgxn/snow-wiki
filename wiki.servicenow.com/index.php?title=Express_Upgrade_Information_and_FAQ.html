1 Overview

The ServiceNow Express release cycle is designed to provide optimal stability and quality, with the flexibility to quickly address problems and deliver new features. ServiceNow releases are based on the concept of a family for a given feature set. For example, Eureka Patch 1 Hotfix 2 is in the Eureka family. A family contains:

  • Feature releases, which contain new functionality and fixes to existing functionality.
  • Patch releases and hotfixes, which provide problem fixes and are released as needed.

Customers are notified prior to the start of an upgrade rollout period. Release notes are available for all releases. Customer configurations are preserved throughout all upgrades.

2 Release types

Type Scope Upgrade Policy
Feature release
  • Introduces new features. A feature provides a complete solution that adds value to an organization. New features are available only as part of a feature release.
  • Includes all available fixes to existing functionality.
  • Is production-oriented; quality and stability are of the highest priority throughout the life cycle.
  • Applied automatically during the rollout period.
Patch release
  • Supports existing functionality with a collection of problem fixes.
  • Includes all previously issued hotfixes for a given release.
  • Does not include new features.
  • Applied as needed on a per customer basis.
  • Provided for the current and previous two feature releases.
Hotfix
  • Supports existing functionality with a specific problem fix for a feature release.
  • May or may not include any previous fixes for a given release.
  • Does not include new features.
  • Applied as needed on a per customer basis.
  • Provided for the current and previous two feature releases.

3 Schedule Upgrade FAQ

3.1 Why does ServiceNow schedule instance upgrades?

To ensure that customers benefit from the enhancements and fixes included in the most recent version of the software.

3.2 How should I prepare for a scheduled upgrade?

For best results:

  • Review the release notes and other release materials, such as webinars, from ServiceNow.com.
  • Notify users and internal stakeholders of the scheduled upgrade date and time.
  • Access the Express DemoNow site to become familiar with the new release

3.3 Will the upgrade break my configurations?

All Express configurations are preserved during the upgrade process. After the upgrade, be sure to validate all configurations.

3.4 What features will be turned on by default?

Review the version release notes to learn about new features that will be activated with the upgrade.

3.5 Will my system be operational while the upgrade is running?

Yes, however, there could be a slight performance impact. Scheduled upgrades run during slower times to minimize the performance impact.
