1 Overview

The following components are installed with Work Management:

  • Tables
  • Plugin Dependencies
  • Properties
  • User Roles
  • UI Policies
  • Script Includes
  • Client Scripts
  • Business Rules
  • Other

Demo data is available with Work Management. The demo data provides sample qualifiers, dispatchers, agents, work orders, work order tasks, stockrooms, and asset information.

Work Management is available starting with the Calgary release.

2 Tables

Work Management adds or modifies the following tables.

Table Description
Asset Usage [wm_asset_usage] Tracks assets used during the execution of a work order.
Group [sys_user_group] Stores group records.
Incidentals [wm_incidentals] Tracks incidental expenses for work order tasks (starting with the Eureka release).
Part Requirement [cmdb_model_part_requirement] Stores the Product Catalog part requirement model records (starting with the Eureka release).
Part Requirement [wm_part_requirement] Tracks the part sourcing requirements of a work order.
Transfer Order [alm_transfer_order] Adds a reference to a work order and the Delivered option to the Stage field.
Transfer Order Line [alm_transfer_order_line] Adds a reference to a part requirement and the Delivered option to the Stage field.
Work Order [wm_order] Stores work order records.
Work Order Affected CI [wm_m2m_order_affected_ci] Stores the relationship between a work order and the configuration items it affects.
Work Order Flow [sf_work_order] Stores the work order state flow records (starting with the Eureka release).
Work Order Groups Dependency [wm_m2m_group_dependency] Enables the dispatch group to manage scheduling for the assignment group.
Work Order Model [cmdb_workorder_product_model] Stores the Product Catalog work order model records (starting with the Eureka release).
Work Order Task [wm_task] Unit of work performed by one person, in one session (one location and one time).
Work Order Task Affected CI [wm_m2m_task_affected_ci] Stores the relationship between a work order task and the configuration items it affects.
Work Order Tasks Dependency [wm_m2m_task_dependency] Defines a dependency between two work order tasks. The downstream task cannot be started until the upstream task is completed.
Work Task Flow [sf_work_task] Stores the work task state flow records (starting with the Eureka release).
Work Task Model [cmdb_worktask_product_model] Stores the Product Catalog work task model records (starting with the Eureka release).
Work Order Task Model [wm_m2m_womodel_wtmodel] Stores the work order task model records (starting with the Eureka release).

3 Plugin Dependencies

Work Management activates the following plugins.

Plugin Name Plugin ID Description
Process Flow Formatter com.snc.process_flow_formatter Summarize multiple pieces of information about a process and display the stages graphically at the top of a form.
Skills Management com.snc.skills_management Assign configured competencies, called skills, to groups or individual users. These skills can then be used to determine who can be assigned to particular tasks.
State Flows com.snc.state_flows Customize the sequence in which records transition between states in Work Management. Control the behavior and visibility of fields during state transitions. The State flow plugin is available starting with the Eureka release.
Work Management Mobile com.snc.work_management_m Manage work orders and work order tasks from a mobile device. Enable geolocation to auto-route and auto-dispatch tasks using Google Maps from a mobile phone or tablet.
Territory Management com.snc.territory_management Assign geographical locations, called territories, to groups or individual users. Territories determine which users are best positioned to execute service calls.

4.1 Work Management Properties

Name Description
glide.ui.wm_order_activity.fields Determines which fields are tracked in the activities list for work orders. The system automatically synchronizes the values in this property and the selections made by personalizing activities.
  • Type: String
  • Default value: opened_by,work_notes
  • Location: System Properties [sys_properties] table
  • Learn more: Activity Formatter
glide.ui.wm_task_activity.fields Determines which fields are tracked in the activities list for work order tasks. The system automatically synchronizes the values in this property and the selections made by personalizing activities.
  • Type: String
  • Default value: work_notes,assigned_to
  • Location: System Properties [sys_properties] table
  • Learn more: Activity Formatter
work.management.default.end.time Default end time for all work agents when no schedule is set, formatted in a 24 hour clock. This property is available starting with the Eureka release.
  • Type: String
  • Default value: 17:00
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Default Start and End Times
work.management.default.start.time Default start time for all work agents when no schedule is set. This value sets the start time for a day other than the current day, when no scheduled task exists or is continued from the previous day. This property uses a 24 hour clock.
  • Type: String
  • Default value: 08:00
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Default Start and End Times
work.management.limit.location Work management should limit the dispatch groups available to choose by the location of the task. By default, ServiceNow filters the list of available dispatch groups by their proximity to the work order task location. When this property is set to false, the system displays all dispatch groups for selection, without any consideration of location. If only one dispatch group is available for a work order task in any location, ServiceNow automatically enters that group in the Dispatch group field. This property is available starting with the Eureka release.
  • Type: true/false
  • Default value: true
  • Location: Work Management > Administration > Work Management Properties > Dispatch Properties for Work Management
  • Learn more: Dispatch Groups
work.management.limit.skills Work management should only allow agents to be selected whose skills match all skills for the work order task. This property forces the system to consider skills when assigning an agent to a work task when auto-dispatch is used. When this property is set to false, ServiceNow selects agents for auto-dispatch assignments based on location only. This property is available starting with the Eureka release.
  • Type: true/false
  • Default value: true
  • Location: Work Management > Administration > Work Management Properties > Dispatch Properties for Work Management
  • Learn more: Matching Skills
work.management.max.agents.processed Sets the maximum number of agents that will be processed by auto-dispatch at a time. The system has an absolute limit of 300 agents and sets the value at that level if you specify more than 300. The system cannot auto-dispatch a task for a dispatch group that contains more agents than the value configured. This property is available starting with the Eureka release.
  • Type: Integer
  • Default value: 100
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Processing Count
work.management.qualification Include a qualification stage for work order tasks? This property controls whether or not qualification is required for work order tasks and is available starting with the Eureka release.
  • Type: true/false
  • Default value: true
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Automatic Qualification
work.management.work.spacing Amount of time (in minutes) to add between the end of a task and the travel start of the next. An example of a valid time value is 10.
  • Type: Integer
  • Default value: 0
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Work Spacing

4.2 Work Management Geolocation Properties

The system properties for Work Management Geolocation are available starting with the Eureka release.

Name Description
glide.geolocation.map.merge.task.agent.markers Merge the task and agent markers on the geolocation maps with a new purple marker. When this property is true, agent and task markers that are close together are merged into a purple cluster that displays the count of both marker types. The pop-up information window for the purple marker displays the items for all agents and tasks. If this property is false, agents and tasks will only be gathered into their own clusters and not into the single, purple cluster.
  • Type: true/false
  • Default value: false
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Clusters
work.management.allow.toll.roads Allow toll roads to be used. Allows the system to use toll roads when auto-routing an agent's tasks.
  • Type: True/false
  • Default value: True
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Toll Roads
work.management.evening.rush.hours Evening rush hour span, formatted as 14:30-16:00. This property uses a 24 hour clock. All times are expressed as a range, separated by a dash with no spaces. Times not using this format are ignored. This property must be used with the work.management.morning.rush.hours and work.management.rush.travel.buffer properties.
  • Type: String
  • Default: 0
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Rush Hour Allowances
work.management.morning.rush.hours Morning rush hour span, formatted as 06:30-08:00. This property uses a 24 hour clock. All times are expressed as a range, separated by a dash with no spaces. Times not using this format are ignored. This property must be used with the work.management.evening.rush.hours and work.management.rush.travel.buffer properties.
  • Type: String
  • Default: 0
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Rush Hour Allowances
work.management.rush.travel.buffer Percentage to add to all rush hour travel times. The system uses this percentage to calculate schedules when auto-routing an agent. Use this property when both morning and evening rush hour times are defined. An example of a valid time buffer percentage is 15.
  • Type: Integer
  • Default value: 0
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Rush Hour Allowances
work.management.travel.buffer Percentage to add to all travel times. An example of a valid percentage value is 15.
  • Type: Integer
  • Default value: 0
  • Location: Work Management > Administration > Work Management Properties
  • Learn more: Travel Time Buffer
