1 Architecture

When a ServiceNow incident is created, xMatters sends notification by voice, SMS, two-way pager, or email. When the notification recipient responds, xMatters updates the incident in ServiceNow. Processing occurs in real-time.

ServiceNow integrates with the following xMatters products:

  • xMatters (alarmpoint)
  • xMatters on demand (A hosted SaaS product)


2 Requirements

  • ServiceNow Incident Management
  • Integration - xMatters Relevance Engine


Additional requirements:

  • For xMatters on demand, an xMatters on demand account
  • For xMatters (alarmpoint):
  • xMatters (alarmpoint) engine version 4.1 (patch 003 or later)
  • xMatters integration agent version 4.1


2.1 Connectivity

The basic architecture for this integration is SOAP Web Services. Please note that in order for the integration to be able to successfully send data to xMatters, the instance will need to have proper connectivity channels set up. If your xMatters SOAP Endpoint is not accessible via the internet, then you will need to establish a VPN Tunnel between the ServiceNow instance and the xMatters SOAP endpoint.

For communication coming back from xMatters to ServiceNow, your xMatters server will need to be able to reach the public internet where the ServiceNow xMatters web service endpoints are accessible.

3 Integration Steps

  1. Request activation of the Integration - xMatters Relevance Engine plugin (prior to the Eureka release).
  2. Configure the integration, which is highly customizable to suit your deployment and needs. Use the xMatters Integration Guides to evaluate the configuration options.
Click the plus to expand instructions for requesting a plugin.
  1. Navigate to [HI].
  2. Click Service Catalog.
  3. Click Request Plugin Activation.
    • [Required] In Target Instance, select the instance on which to activate the plugin.
    • [Required] In Plugin Name, enter the name of the plugin to activate.
    • [Optional] In Date and time would you like the plugin to be enabled?, specify a date and time at least 12 hours in the future. Leave this field empty if you want the plugin activated as soon as possible.
    Note: Plugins are generally activated during business hours in the Pacific time zone, but can be scheduled for a different time with advance notice.
    • [Optional] In Reason/Comments, add any information that would be helpful for the ServiceNow technical support engineer activating the plugin.
  4. Click Submit.

4 Plugin Components

The Integration - xMatters Relevance Engine plugin installs a set of new Web Services, Business Rules, and properties into ServiceNow instances. Once activated, you can use these components to customize your deployment.

4.1 Modules

Name Description
Batch Load User and Group JavaScript module that performs an initial load of users and groups to xMatters. By default, this module is not enabled when the plugin is loaded.

4.2 Web Services

Field Description
xMatters SubmitAPXML xMatters web service used in the plugin script includes to send ServiceNow incident data to xMatters.
xMatters Incident Called by xMatters to update incident details such as incident assignee, incident state, or incident work notes.
xMatters (alarmpoint) xMatters web service used in the plugin script includes to send ServiceNow user and group data to xMatters.

4.3 Business Rules

Field Description
xMatters User Sync Updates ServiceNow user data to xMatters in real-time.
xMatters Incident Sends ServiceNow incident data to xMatters. By default, only incidents with a priority of Critical or High are sent to xMatters.
xMatters Group Sync Updates ServiceNow group data to xMatters in real-time.
xMatters Member Sync Updates ServiceNow group member data to xMatters in real-time.

4.4 Script Includes

Field Description
xMattersGroupSync Business logic for synchronizing ServiceNow group data to xMatters.
xMattersGroupMemberSync Business logic for synchronizing ServiceNow group member data to xMatters.
xMattersSOAPHelper Utility Java Script functions for interacting with xMatters web services.
xMattersUserSync Business logic for synchronizing ServiceNow user data to xMatters.
xMattersIncident Business logic for sending ServiceNow incident data to xMatters.
xMattersLogger Utility Java Script functions for logging xMatters plugin information to the ServiceNow System Log.

4.5 Other Includes

Name Description
Database View - Member List Database view of the ServiceNow group members used as part of ServiceNow group member synchronization.
Single Sign On Allows URL redirect from xMatters notifications to matching ServiceNow incident.
