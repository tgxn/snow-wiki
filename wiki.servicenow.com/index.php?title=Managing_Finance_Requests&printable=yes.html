1 Overview

Finance agents need to regularly access request records as they resolve requests and correspond with the submitter. Agents can also access built-in reports to see information like the number of active or unassigned requests.

2 Creating Finance Requests

You can submit a finance request through the Finance catalog. If you have the Finance agent role, you can also create and update finance requests using the Finance Request form directly.

2.1 Creating Requests Through the Finance Catalog

If you do not have a role, you can use the Finance catalog to submit requests. The catalog provides several different categories so you can choose the one that closely relates to your request.

To use the finance catalog:

  1. Navigate to Self-Service > Finance Catalog.
  2. Choose from one of the displayed categories.
  3. Select a sub-category, if necessary.
  4. Fill in the fields (see table).
  5. Click Submit.
Field Description
Opened for The name of the person submitting this request. Select a new name if you are opening this request on behalf of another user.
Location The location where this request pertains.
Priority The priority that describes the importance of this request.
Short Description A brief summary of the request.
Detailed Description A detailed description of the request.

2.2 Creating Requests with the Finance Request Form

You can create requests through the Finance Request form if you want to associate the request with a configuration item, which is affected by the request. Associating a CI to a request helps your configuration management team understand which services could be negatively impacted by a finance request issue. You can also use this form if you want to include additional comments and work notes for the request.

  1. Navigate to Finance > Requests > Create New.
  2. Fill in the fields (see table).
  3. Click Submit.
Field Description
Number An auto-generated number that begins with FR to identify the request record.
Company The name of the company from which the request initiated.
Caller The name of the requestor.
Template The template for creating this request (optional). Click the lookup icon and select a template. The description of the selected template will populate the Description field. For more information on templates, see Creating Service Order Templates.
Opened Auto-filled with the date and time the request was opened.
Priority The priority that describes the importance of this request. By default, all requests are set to 4-Low.
State The state that describes what work stage this request is in. By default, all requests are set to Open.
Assignment group The group from which a finance agent is assigned to the request. You can select only Finance type groups. See Creating Finance Groups for more information.

Note: If you selected the Finance will use the dispatch queue option in Finance Service Management Configuration, only users with the Dispatcher role can edit this field. If you selected the Finance will not use the dispatch queue option, all users except those with the Basic and Initiator roles can edit this field.

Assigned to The finance agent to assign the request to. If you already selected an assignment group, you can only select finance agents who belong to that group.

Notes:

  • If you selected the Finance will use the dispatch queue option in Finance Service Management Configuration, only users with the Dispatcher and Agent roles can edit this field. If you selected the Finance will not use the dispatch queue option, all users except those with the Basic and Initiator roles can edit this field.
  • If you selected an assignment group and want to assign the work to a new user, click the lookup icon (magnifying glass) next to Assigned to, click New, and create a new user. Be aware, however, that you must navigate to User Administration > Groups and add the user to the assignment group before the request can be assigned.
Initiated from The incident or task from which this request was created.
Billable The finance request is billable.
Short description [Required] A brief summary of the request. Optionally, you can click the search knowledge icon () to view articles in the knowledge base relating to this product model, plan, or CI. Doing so could provide a solution related to the reason you are submitting this request.
Description A detailed description of the request. The description is always visible to the submitter. Therefore, if you add or modify the description for a request that another user submitted, the user is able to see the changes.
Work notes Additional notes that you want to share between users who can access the Finance Request form. A user who submits the request through the service catalog cannot see the work notes.
Scheduling
Scheduled start Date and time when the earliest task is scheduled to start.
Estimated end Estimated work end date. This is the estimated date when the latest task will be completed.
Actual work start Date and time when the earliest task actually started.
Actual work end Date and time when the latest task actually ended.
Requested due by Estimated date when the latest task will be completed.

3 Finance Request States

Finance requests follow a specific life cycle. Additionally, progress through the request life cycle is shown visually with a process flow formatter above the finance request record. Completed tasks appear in green.

The states that display depend on your settings on the Finance Service Management Configuration screen. For example, if you select Approval is not required for new requests, the states relating to approval are not displayed in the process flow formatter, and the request moves to the next state. The State field on the Finance Request record is always read-only.

Note: For information on customizing state flows for Finance Service Management, as well as other Service Management (SM) applications, see State Flow Customization for SM Applications.


State Description
Draft The default state for new requests. The finance staff member adds information about the work to be done. When a request initiator adds information about work to be done, the request begins in the Draft state.
Awaiting Approval When the information is complete enough for review by an approver, the finance staff member marks it ready for approval.
Approved The appropriate finance person approves the request.
Work In Progress Work has started.
Closed Complete The finance request was completed to specification.
Closed Incomplete The finance request not be completed as specified.
Cancelled The finance request was cancelled.

3.1 Finance Request Task States

Like facilities requests, finance request tasks follow a specific life cycle. The State field on the task record is always read-only.

State Description
Draft Qualifier is not done describing the work.
Pending Dispatch Task is ready to be assigned. The parent work order state changes to Qualified if all associated tasks are in Pending Dispatch or a later state.
Assigned Task is pending acceptance from the assigned agent.
Accepted The task has been accepted by the agent and is ready to be done.
Work In Progress Work on the task has started. The parent finance request state changes to Work In Progress if no associated tasks are in Draft state.
Closed Complete Task was completed to specification.
Closed Incomplete Task could not be completed as specified.
Cancelled Task was cancelled.

4 Approving Requests

Approving a finance request means that the request is ready for task creation and assignment. When a request is sent to a user with the finance_approver_user role, the approver has several choices (see table).

If you select Approval is required for new requests in the Finance Configuration screen, a newly-created request automatically moves to the Awaiting Approval state. Otherwise, the request moves to the next configured state.

Approval Choice Description
Approved The finance request is approved.
Rejected The finance request is not qualified and it will be moved to the cancelled state. Additionally, the following work note is added to the request:

The finance request is rejected.

More information required The request does not contain enough information. It reverts back to the Draft state and the following work note is added to the request:

The finance request needs more information for further approval.

Duplicate The request is no longer required, because the work was already performed by another request. The request is moved to the Cancelled state and the following work note is added to the request:

This is a duplicate finance request.

5 Assigning Active Requests

  1. Navigate to one of the following:
    • Finance > Open - Unassigned for a list of requests that no one is assigned to.
    • Finance > All Finance Requests for a list of all open incidents regardless of their current assignment.
  2. Open the request you want to assign.
  3. Click the reference lookup icon () beside the Assignment group field.
  4. Select the group that handles this kind of request. If no groups are available, leave this field blank. You do not have to select an assignment group, but doing so limits the number of users you can assign the request to.
  5. Click the reference lookup icon beside Assigned to.
  6. Select the agent to handle the request.
    The users available in this field are limited to the users who are in the Assignment group, if one was selected.
  7. Click Update. An email notification is automatically sent to the assigned agent if email notifications are set up for the instance.

6 Collaborating on Requests

  1. Navigate to Finance > All Finance Requests.
  2. Open the request you want to collaborate on.
  3. In the Additional comments (Customer visible) field, enter the comments that you want the person who submitted the request to see. The submitter can see the comments in this field and add additional comments as necessary.
    Update this field as many times as necessary to correspond with the submitter.
  4. To correspond with other facilities agents, enter content that you do not want the submitter to see in the Work notes field.

7 Closing Requests

  1. Navigate to Finance > Assigned to me.
  2. Click the request number.
  3. In the Additional comments field, enter any final notes or comments that you want the submitter to be aware of.
  4. Change the State field to the appropriate closed state.
  5. Click Update.

7.1 Closing/Completing Requests

After a request has been assigned to an agent, that agent can complete and close the request when the Request lifecycle is request driven option is set in Finance Service Management Configuration and all of the request's tasks are completed.

A Close Complete button is visible on the Finance Request form to the agent assigned to the request. The agent must enter work notes before clicking Close Complete. When the button is clicked, the open task is automatically completed (if applicable) and the request transitions to the Complete state.
