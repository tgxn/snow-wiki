1 Overview

The look and feel of the user interface varies by version while the functionality remains similar. The following versions of the user interface are available:

  • UI15: replaces UI14 with an updated look, including redesigned icons and usability improvements. The UI15 interface is available starting with the Fuji release.
  • UI14: provides an updated look, including redesigned icons, buttons, and color themes. It incorporates the UI11 interface and provides a series of usability improvements. The UI14 interface is available starting with the Eureka release.
  • UI11: builds on the classic interface by adding the Edge and several features including split screen, bookmarks, and flyout windows.
  • Classic: is the original ServiceNow product interface.

Administrators can activate the user interface plugins and enable or disable a specific version of the user interface for an instance. Administrators can also enable a link that allows users to switch between UI11 and the classic interface.

2 Activating UI Plugins

  • Fuji: UI15 is the default user interface for new and upgraded instances.
  • Eureka:
    • UI14 is the default user interface for new and upgraded instances starting with the Eureka Patch 2 release.
    • Upgraded instances retain the existing user interface prior to the Eureka Patch 2 release. To activate UI14 for these upgraded instances, activate the UI 14 plugin.
  • Dublin and earlier:
    • UI11 is the default user interface for new instances.
    • Upgraded instances retain the existing user interface. To activate UI11 for upgraded instances, activate the UI11 - Navpage layout with multiple panes plugin.
Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.

3 Managing UI15 or UI14

An administrator can disable UI15 or UI14, which reverts the user interface to UI11 or classic, depending on configuration.

  1. Enter sys_properties.list in the navigation filter.
  2. Locate the glide.ui.doctype property in the System Properties list.
    The property is set to true by default when UI15 or UI14 is activated.
  3. Set the property to false.
  4. Log out and log back in.
    This disables UI15 or UI14 for the instance. The interface reverts to UI11 or classic in individual user sessions when each user logs out and logs back in. If the glide.ui11.use user preference is set to false, the classic interface displays.

To preview UI11 or classic without disabling UI15 or UI14, append /?sysparm_device=desktop to the instance URL. Depending on the glide.ui11.use user preference setting, UI11 or the classic interface displays.

4 Managing UI11

An administrator can add a link to the banner frame that allows users to disable UI11 and revert to the classic interface.

  1. Enter sys_properties.list in the navigation filter.
  2. Set the glide.ui11.show_switch_link property to true.
    This property shows the link in the banner frame that allows a user to switch between UI11 and the classic interface.
 Note: The glide.ui11.show_switch_link property is set to false by default starting with the Eureka release.


5 Browser Support

For more information about specific versions, see Supported Web Browsers.

5.1 Browser Support for UI15

The UI15 interface is supported on recent versions of most major browsers.

  • Firefox version 31 and up
  • Chrome version 39 and up
  • Safari version 6.1 and up
  • Internet Explorer version 9 and up
    • Edge mode is supported.
    • Compatibility mode is not supported.
 Note:
  • While UI15 is supported on Internet Explorer version 9, the browser may be slow when performing some operations. This does not necessarily indicate a performance degradation in the instance.
  • Internet Explorer 11 is susceptible to memory leaks, which may impact performance, especially in Windows 7.
  • For Safari version 5 and Internet Explorer versions 7 and 8, the user interface automatically redirects to UI11.


5.2 Browser Support for UI14

The UI14 interface is supported on recent versions of most major browsers.

  • Firefox version 26 and up
  • Chrome version 33 and up
  • Safari version 6.1 and up
  • Internet Explorer version 9 and up
    • Edge mode is supported.
    • Compatibility mode is not supported.
 Note: For Internet Explorer versions 7 and 8, the user interface automatically redirects to UI11.


6 Installed Components

The following components are added with the user interface plugins.

6.1 Tables

Table Description
Bookmarks [sys_ui_bookmark] Stores user bookmarks and preferences. To access, navigate to System Definition > Bookmarks.


6.2 Properties

Property Description
glide.ui11.show_switch_link Shows the link in the banner frame to allow switching between UI11 and the classic interface. If this property is true, the user can click the link to switch between interfaces. When UI11 is active, the user preference glide.ui11.use is set to true for that user.

The default value for this property is false and the link is not displayed starting with the Eureka release.

glide.ui.allow_ie_compatibility_mode In many cases, Internet Explorer is configured to use compatibility mode when viewing ServiceNow. However, UI15 and UI14 do not display when compatibility mode is turned on. When this property is set to false, the system does not allow use of compatibility mode on UI15 and UI14 pages. The default value for this property is false. Available starting with the Eureka release.
glide.ui.doctype Enables or disables the UI15 or UI14 interface. The default value for this property is true. Available starting with the Eureka release.

After enabling or disabling this property, the administrator must log out and log back in to effect the change in the instance. When users log out and log back in, they see the change in their individual user sessions.

glide.ui.doctype.ie_version Sets the minimum version of Internet Explorer required to use UI15 or UI14. The default value for this property is 9. It is recommended that you do not change this value. Available starting with the Eureka release.

6.3 User Preferences

User Preference Description
glide.ui11.use Indicates whether UI11 is active for this user. This record updates automatically when the user clicks the Switch to the new UI or the Switch to the old UI link in the banner frame. To turn on UI11 for all users by default, create the user preference with a value of true and select the System check box.
glide.ui.navpage_state Do NOT edit this preference; system record of a user's UI11 configuration.
ng.navigator_preference Stores a user's selected application navigator favorites. Available starting with the Eureka release.
glide.ui.field_style_circles Shows field status indicators on lists as circles instead of cell backgrounds. The system administrator can set this preference for new logins. Users can make this change by navigating to a list, clicking the personalize list icon (), and selecting the Modern cell coloring check box. Available starting with the Eureka release.
