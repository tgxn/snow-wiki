1 Overview

The Service Level Agreements (SLA) Plugin provides engines for tracking service levels of Catalog Fulfillment through two approaches:

  • Attaching an SLA to a Catalog Request.
  • Attaching an SLA to individual Catalog Tasks.

Attaching the SLA to the Catalog Request will track the total time it takes to fulfill a request, from beginning to end. Attaching the SLA to an individual Catalog Task tracks the time it takes to complete a specific stage within the catalog request. The Service Level Agreements (SLA) Plugin allows both approaches to be used simultaneously.

Below are a pair of examples of Service Levels for Catalog Fulfillment, demonstrating each of the approaches.

2 Defining an SLA for a Catalog Request

This example demonstrates how to define an SLA for any approved catalog request.

To define an SLA for a Catalog Task:

  1. Navigate to Service Level Management > Service Level Definitions and click New.
  2. Populate the form as follows:
    • Name - Request Fulfillment
    • Type - SLA
    • Table - Request [sc_request]
    • Duration Type - User Specified Duration
    • Duration - Days 10 Hours 00:00:00
    • Schedule - 8-5 Weekdays excluding Holidays
    • Start Condition - "Active is True" and "Approval is Approved"
    • Stop Condition - "State is Closed Complete"

This means that this SLA will take effect for any Catalog Request that is approved, and will track time until the Request reaches the state Closed Complete. The SLA is marked as Breached after 10 days.

3 Defining an OLA for a Catalog Task

This example demonstrates how to define an OLA for a department involved in the fulfillment of a particular catalog task. This OLA will hold the Procurement group to a 2 business day turnaround time on any catalog task assigned to them.

To define an OLA for a Catalog Task:

  1. Navigate to Service Level Management > Service Level Definitions and click New.
  2. Populate the form as follows:
    • Name - Procurement OLA
    • Type - OLA
    • Table - Catalog Task [sc_task]
    • Duration Type - 2 Business Days by 4 PM
    • Schedule - 8-5 Weekdays
    • Start Condition - "Active is True" and "Assignment Group is Procurement"
    • Stop Condition - "State is Closed Complete"

This means that this OLA will take effect any time that a catalog task is created and assigned to the Procurement group, and will continue until the catalog task is marked Closed Complete.
