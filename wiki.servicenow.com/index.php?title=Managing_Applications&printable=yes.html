1 Overview

The system offers several ways to manage applications. You must have the admin role to perform these procedures.

Managing applications changed starting with the Fuji release. If you are using an older version, see Managing Applications - Versions Prior to Fuji.

2 Select the Current Application

Application developers have have the following options to select the current application:

  • Select an application from the applications list
  • Select an application from the application picker

2.1 Applications List

Sample applications list

Use the Applications list to open a custom application record. If the contextual development environment detects that you are editing an application artifact in another application scope, it displays a warning message you can use to switch to another application.

  1. Navigate to System Applications > Applications.
  2. Click the button for the application type you want to display.
    • Develop: displays applications that have been created on this instance.
    • Downloads: displays applications that have been downloaded on this instance.
    • Updates: displays available updates for applications downloaded on this instance.
  3. Click the application name or the Edit button for the application you want to work on.
    If your current application does not match this application, the system displays a warning message.
    Switch applications from the scope warning message

  4. If needed, click the link to switch the other application.

2.2 Application Picker

Use the application picker to select the application to which any new records and updates apply.

  1. Click the gear Icon from the banner frame.
  2. From Application, select the application where you want to save changes.
    Application picker

3 Update an Application Record

You can update a custom application record to add new features or change application functionality.

  1. Navigate to System Applications > Applications.
  2. Click the button for the application type you want to update.
    • Develop: displays applications that have been created on this instance.
    • Downloads: displays applications that have been downloaded on this instance.
  3. Click the application name or the Edit button for the application you want to update.
  4. Fill in the fields, as appropriate (see table).
    Sample custom application record

  5. Click Update.
Field Description
Name [Required] Enter a label for the application. Changing the name does not change any other field value derived from the application name such as the Scope, Menu, or User role fields.
Scope [Read Only] Displays the unique application scope set during the creation process. You can change this value only by deleting and recreating the application with a new value. For more information about the protections offered, see Application Scope.
Menu Select the application menu where you want to display modules. For more information about menus and modules, see Administering Application Menus and Modules.
User role Select the user role required to access the application menu. For more information about user roles, see Creating Roles.
Short description Enter a description of the application purpose or usage.
Logo Select the image the system displays in the applications list and ServiceNow Store.
Version Enter any version information for the application. Both the ServiceNow application repository and the ServiceNow Store use this value to determine if updates are available to your application users.
Active Select this check box to make the application available to users. Clear the check box to prevent users from accessing the application. See Deactivate an Application.
Related lists
Tables View or create application data tables.
Roles View or create user roles for the application.
Application Menus View or create menus for the application.
Application Files View configuration records associated with this application in platform feature tables.
Dependencies View or add tables or applications on which this application depends. The system automatically adds records to this list when you extend tables or when another application creates application files for this application. You must manually add script-based dependencies.
Update Sets View or create update sets to store the application.

3.1 Add Tables

You can add new tables to store application data.

  1. Navigate to System Applications > Applications.
  2. Click the button for the application type you want to update.
    • Develop: displays applications that have been created on this instance.
    • Downloads: displays applications that have been downloaded on this instance.
  3. Click the application name or the Edit button for the application to which you want to add a table.
  4. From the Tables related list, click New.
  5. Complete the table form. See Creating a Custom Table.
    The system automatically associates the new table to the application.

3.2 Add Roles

You can add new user roles to control access to application menus and data.

  1. Navigate to System Applications > Applications.
  2. Click the button for the application type you want to update.
    • Develop: displays applications that have been created on this instance.
    • Downloads: displays applications that have been downloaded on this instance.
  3. Click the application name or the Edit button for the application to which you want to add a table.
  4. From the Roles related list, click New.
  5. Complete the User Role form. See Creating Roles.
    The system automatically associates the new user role to the application.

3.3 Add Application Menus

You can add new menus to organize application modules.

  1. Navigate to System Applications > Applications.
  2. Click the button for the application type you want to update.
    • Develop: displays applications that have been created on this instance.
    • Downloads: displays applications that have been downloaded on this instance.
  3. Click the application name or the Edit button for the application to which you want to add a table.
  4. From the Application Menus related list, click New.
  5. Complete the Application Menu form. See Creating Application Menus.
    The system automatically associates the new menu to the application.

3.4 Add Dependencies

The system can automatically identify and add dependencies when you:

  • Extend a table
  • Add a field to a table
  • Call a script include

However, the system cannot identify dependencies on client-side resources such as UI-based JavaScript libraries. In such cases, you must manually add the dependency.

  1. Navigate to System Applications > Applications.
  2. Click the button for the application type you want to update.
    • Develop: displays applications that have been created on this instance.
    • Downloads: displays applications that have been downloaded on this instance.
  3. Click the application name or the Edit button for the application to which you want to add a table.
  4. From the Dependencies related list, click Edit.
  5. Select the existing functionality on which the application depends.
  6. Click Save.

3.5 Add Update Sets

You can save a version of the application in an update set if you want to share it with other instances or need an external copy of the application. See Publish Applications to an Update Set.

4 Set the Application State

You can deactivate, activate, and delete applications from the custom application record.

4.1 Deactivate an Application

Deactivate applications that are no longer needed to remove user interface elements but preserve all application data.

Note: Administrators can still access application data tables from the application filter or direct URLs to the tables.


To deactivate an application:

  1. Navigate to System Applications > Applications.
  2. Click the button for the application type you want to update.
    • Develop: displays applications that have been created on this instance.
    • Downloads: displays applications that have been downloaded on this instance.
  3. Select the application you want to deactivate.
  4. Clear the Active check box.
  5. Click Update.
    • The system deactivates all associated menus, modules, and access control rules.
    • The system prevents non-admin users from accessing the application tables from the application navigator.
    • The system allows you to create new menus, modules, and access control rules for this application, but prevents you from making them active.

4.2 Activate an Application

Reactivate a deactivated application to restore it to normal service. You can selectively deactivate application menus, modules, and access control rules after you activate the application.

To activate an application:

  1. Navigate to System Applications > Applications.
  2. Click the button for the application type you want to update.
    • Develop: displays applications that have been created on this instance.
    • Downloads: displays applications that have been downloaded on this instance.
  3. Select the application you want to activate.
  4. Select the Active check box.
  5. Click Update.
    The system activates all associated menus, modules, and access control rules.

4.3 Delete an Application

You can delete applications that are no longer needed. Typically, you only delete applications that have never been shared with other instances. This ensures you can support any published applications.

By default, deleting an application also deletes all the records that are associated with the application, such as tables, business rules, and menus. Alternatively, you can deactivate an application that is no longer in use to preserve all application data.

Note: You can always reinstall an application from the ServiceNow Store or from a backup update set.


To delete an application:

  1. [Recommended] Back up the application by publishing it to the ServiceNow application repository or creating an update set. See Sharing Applications.
  2. Navigate to System Applications > Applications.
  3. Click the button for the application type you want to update.
    • Develop: displays applications that have been created on this instance.
    • Downloads: displays applications that have been downloaded on this instance.
  4. Open the application record.
  5. Click Delete.
    The system displays a confirmation dialog box indicating the number of application files in the application to be deleted.
    Confirm deletion

  6. [Optional] Click Show files to cancel deletion and navigate to a list of the application files in the application.
  7. Click OK.
  8. In the confirmation dialog box, enter delete and click OK.
    • The system deletes the custom application record unless the application extends other applications (starting with the Dublin release).

5 Move Application Files Between Global Applications

When you select an application in the application picker, new application files are automatically assigned to it. You cannot move an application file into or out of a private scope application (starting with the Fuji release). You can however move application files between global applications.

To move application files from one global application to another:

  1. Navigate to the application file in a list or form view. For example, navigate to System Definition > UI Policies.
  2. Locate the Move to Application UI action.
    • Lists: select the check box beside each appropriate row, and then select Move to Application in the Actions choice list.
    • Forms: right-click the header and select Move to Application.
  3. Select the global application to which you want to move the records.
    Move application files to a global application

  4. Click Move.
    The record and all of its descendant records are moved to the selected global application.
