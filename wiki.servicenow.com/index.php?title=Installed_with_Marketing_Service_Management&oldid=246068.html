1 Overview

Activating the Marketing Service Management plugin adds or modifies several tables, user roles, and other components.

Demo data is available with Marketing Service Management.

2 Tables

The Marketing application adds the following tables.

Table Description
Marketing Request [sn_sm_marketing_request] Stores all requests for changes to marketing.
Marketing Request Flow [sn_sm_marketing_sf_request] Stores the marketing request state flow records.
Marketing Request Task Flow [sn_sm_marketing_sf_task] Stores the marketing request task state flow records.
Marketing Request Template [sn_sm_marketing_request_template] Stores the Product Catalog marketing request template records.
Marketing Task [sn_sm_marketing_task] Stores all tasks for changes to marketing.
Marketing Task Template [sn_sm_marketing_task_template] Stores the Product Catalog marketing request task template records.

3 Plugins

The Marketing Service Management application uses the following plugins:

Plugin Description
Marketing Service Management [com.snc.marketing_service_automation] Activates the Marketing Service Management application.
Service Management Core [com.snc.service_management.core] Activates Asset Management, Process Flow Formatter, State Flows, Knowledge Management V3, Skills Management, Territory Management, Managed documents, and Time Card Management.
Marketing Service Management CMS Portal [com.snc.marketing.service.automation.cms] (Optional) Displays the Marketing Service Management application on the Service Management portal. Activation of this plugin on production instances may require a separate license. Contact ServiceNow for details.

4 Properties

The Marketing Service Management application uses the following properties:

Name Description
Properties for Marketing
marketing.default.start.time Default start time for all agents when no schedule is set. This value sets the start time for a day other than the current day, when no scheduled task exists or is continued from the previous day. This property uses a 24 hour clock.
  • Type: String
  • Default value: 08:00
  • Location: Marketing > Administration > Properties
  • Learn more: Default Start and End Times
marketing.default.end.time Default end time for all agents when no schedule is set, formatted in a 24 hour clock.
  • Type: String
  • Default value: 17:00
  • Location: Marketing > Administration > Properties
  • Learn more: Default Start and End Times
Dispatch Properties for Marketing
marketing.max.agents.processed Sets the maximum number of agents that will be processed by auto-dispatch at a time. The system has an absolute limit of 300 agents and sets the value at that level if you specify more than 300. The system cannot auto-dispatch a task for a dispatch group that contains more agents than the value configured.
  • Type: Integer
  • Default value: 100
  • Location: Marketing > Administration > Properties
  • Learn more: Processing Count
marketing.work.spacing Amount of time (in minutes) to add between the end of a task and the travel start of the next. An example of a valid time value is 10.
  • Type: Integer
  • Default value: 0
  • Location: Marketing > Administration > Properties
  • Learn more: Work Spacing
Geolocation Properties for Marketing
marketing.morning.rush.hours Morning rush hour span, formatted as 06:30-08:00. This property uses a 24 hour clock. All times are expressed as a range, separated by a dash with no spaces. Times not using this format are ignored. This property must be used with the marketing.evening.rush.hours and marketing.rush.travel.buffer properties.

Type: String Default: 0

marketing.evening.rush.hours Evening rush hour span, formatted as 14:30-16:00. All times are expressed as a range, separated by a dash with no spaces. Times not using this format are ignored. This property must be used with the marketing.morning.rush.hours and marketing.rush.travel.buffer properties and uses a 24 hour clock.
  • Type: String
  • Default: 0
marketing.rush.travel.buffer Percentage to add to all rush hour travel times. The system uses this percentage to calculate schedules when auto-routing an agent. Use this property when both morning and evening rush hour times are defined. An example of a valid time buffer percentage is 15.
  • Type: Integer
  • Default value: 0
marketing.allow.toll.roads Toll roads are used when auto-routing an agent's tasks.
  • Type: True/false
  • Default value: True
marketing.travel.buffer Percentage to add to all travel times. An example of a valid percentage value is 15.
  • Type: Integer
  • Default value: 0

5 User Roles

The Marketing Service Management application uses the following roles.

Role Description Contains Roles
catalog_admin Manages the Service Catalog application, including Catalog categories and items.
  • catalog
  • user_criteria_admin
inventory_user Provides access to stock information. none
knowledge_manager Administers knowledge bases and request new knowledge bases for the knowledge_admin to approve.
  • itil
  • knowledge
skill_admin Administers skills and relations to groups.
  • skill_user
skill_model_admin Defines and administers the relationship between skills and models.
  • skill_user
skill_user Views skills and relations. none
sn_sm_marketing_admin Provides full control over all Marketing Service Management data. Also administers Territories and Skills, as needed.
  • catalog_admin
  • knowledge_manager
  • skill_admin
  • skill_model_admin
  • sn_sm_marketing_agent
  • sn_sm_marketing_approver_user
  • sn_sm_marketing_dispatcher
  • template_admin
  • territory_admin
sn_sm_marketing_agent Provides the ability to accept or reject a task. It is the one who performs the work on the site.
  • sn_sm_marketing_basic
sn_sm_marketing_approver_user Allows users to approve orders/requests.
  • approver_user
sn_sm_marketing_basic Reads and creates service orders and follows up on those they created. none
sn_sm_marketing_dispatcher Schedules and assigns tasks to agents. They can be searched (that is, filtered by) the group they manage.
  • inventory_user
  • skill_model_user
  • sn_sm_marketing_basic
  • territory_user
sn_sm_marketing_initiator Reads and creates service orders and follows up on those they created. Also grants UI access.
  • sn_sm_marketing_basic
template_admin Grants the ability to create and administer template models. none
territory_admin Administers territories.
  • territory_user
territory_user Views territories. none
Note: All Marketing Service Management user roles provide access to all links in the Marketing Service Management CMS Portal. To use the Marketing Service Management CMS portal, you must first activate the Marketing Service Management CMS Portal plugin.


6 Catalogs

The Marketing Service Management application uses the following catalog.

Catalog Name Description
Marketing Catalog Provides a catalog and several categories for users to submit several types of marketing requests.
