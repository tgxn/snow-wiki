1 Overview

When inserting or updating a record, these GlideRecord methods also insert or update related records. Any related records are inserted/updated first, and then the record itself is inserted/updated.

In each related table, a record is inserted or updated.

  • If the reference field in the table has a value, then the record will be updated.
  • If the reference field in the table does not have a value, then a new record will be inserted.

Use these methods with care, as they can have a significant impact.

1.1 GlideRecord.updateWithReferences()

1.1.1 Does the Reference Field of the Current Record have a Value?

Yes. Update the current record with the information provided

For example, if processing a incident where the Caller ID is set to reference sys_user record 'David Loo,' then the following code would update David Loo's user record.
var inc = new GlideRecord(‘incident’);
inc.get(inc_sys_id);  // Looking up an existing incident record where 'inc_sys_id' represents the sys_id of a incident record
inc.caller_id.first_name = 'John';
inc.caller_id.last_name = 'Doe';
inc.updateWithReferences();
Insert/Update Sequence: Caller ID → Incident

No. Insert a new record with the information provided

For example, if processing a incident where there is no Caller ID specified, then the following code would create a new sys_user record with the provided information (first_name, last_name) and set the Caller ID value to the newly created sys_user record.
var inc = new GlideRecord(‘incident’);
inc.get(inc_sys_id);  // Looking up an existing incident record where 'inc_sys_id' represents the sys_id of a incident record
inc.caller_id.first_name = 'John';
inc.caller_id.last_name = 'Doe';
inc.updateWithReferences();
Insert/Update Sequence: Caller ID → Incident

1.1.2 Important Note

If you are setting the reference value and changing the reference via scripting, then you must set the reference using the display value first and then update any fields.
var inc = new GlideRecord(‘incident’);
inc.get(inc_sys_id);  // Looking up an existing incident record where 'inc_sys_id' represents the sys_id of a incident record
inc.caller_id.setDisplayValue('David Loo');
inc.caller_id.phone = '123456789';
inc.updateWithReferences();
Insert/Update Sequence: Caller ID → Incident
The user record is located with the display value of 'David Loo' and then the sys_user record is updated with the provided phone number.
If the user record is not located, a new user record is created with the provided phone number.

1.1.3 More Complicated Example

In the following, the references being inserted/updated are not first level fields (where gr.caller_id would be a first-level field because caller_id is a field that exists within incident).
var inc = new GlideRecord(‘incident’);
inc.get(inc_sys_id);  // Looking up an existing incident record where 'inc_sys_id' represents the sys_id of a incident record
inc.short_description = ‘Short description 1’;
inc.cmdb_ci.manufacturer.setDisplayValue('A Manufacturer');  // Where a core_company exists with the name of 'A Manufacturer'
inc.cmdb_ci.manufacturer.contact.first_name = "Jane";
inc.cmdb_ci.manufacturer.contact.last_name = "Doe";
inc.cmdb_ci.manufacturer.name = "Manufacturer 1";
inc.cmdb_ci.name = "CI 1";
inc.updateWithReferences();
Insert/Update Sequence: Contact → Manufacturer → CI → Incident


1. Inserts or Updates a sys_user record with the provided first_name, last_name. Does the Manufacturer record have a Contact value?
Yes. Updates the sys_user record with the provided first_name, last_name.
No. Inserts a new sys_user record with the provided first_name, last_name.


2. Inserts or Updates a core_company record with the provided name and contact. Does a core_company record with the name 'A Manufacturer' exist?
Yes. Updates the record with the provided name and contact (sys_user record inserted/updated in step 1).
No. Creates a new core_company record with the name and contact provided.


3. Inserts or Updates a cmdb_ci record with the provided name and manufacturer. Does the incident record have a Configuration Item value?
Yes. Updates the cmdb_ci record with the provided name (and manufacturer if it did not previously exist).
No. Inserts the cmdb_ci record with the provided name, manufacturer.


4. Updates the current incident record with the provided short_description (and cmdb_ci if it did not previously exist).

1.2 GlideRecord.insertWithReferences()

The same logic surrounding references applies to the Insert.

If a reference value is not specified (as below), then a new user record will be created with the provided first_name, last_name, and the caller_id value is set to this newly created sys_user record.
var inc = new GlideRecord(‘incident’);
inc.initialize();
inc.short_description = 'New incident 1';
inc.caller_id.first_name = 'John';
inc.caller_id.last_name = 'Doe';
inc.insertWithReferences();
The result is a new sys_user record with the provided first_name, last_name and a new incident record with the provided short_description and caller_id.


If a caller_id value is specified, then that caller_id is updated with the provided first_name, last_name.
var inc = new GlideRecord(‘incident’);
inc.initialize();
inc.short_description = 'New incident 1';
inc.caller_id.setDisplayValue('David Loo');
inc.caller_id.first_name = 'John';
inc.caller_id.last_name = 'Doe';
inc.insertWithReferences();
The result is a newly created incident record with values set for short_description and caller_id.
