1 Overview

Provides a set of miscellaneous functions used in Discovery.

1.1 Where To Use

Use this script include for general Discovery functions.

  • None

3.1.2 Output Fields

Returns: void

Gets the schedule record.

3.2.1 Input Fields

Parameters:

  • None

3.2.2 Output Fields

Returns: (GlideRecord) the schedule record.


3.3 getStatusRecord()

Gets the status record.

3.3.1 Input Fields

Parameters:

  • None

3.3.2 Output Fields

Returns: (GlideRecord) the Discovery status record.


3.4 getDiscoveryType(statusID)

Gets the discovery type for the given status ID.

3.4.1 Input Fields

Parameters:

  • statusID - The Discovery status ID.

3.4.2 Output Fields

Returns: (GlideRecord) the discovery type.


3.5 getService(port)

Gets the service assigned to a specified port.

3.5.1 Input Fields

Parameters:

  • port - The port to check.

3.5.2 Output Fields

Returns: (GlideRecord) the service assigned to the port.


3.6 isInRange(ip)

Determines if the given IP address is within the current IP range.

3.6.1 Input Fields

Parameters:

  • ip - The IP address to check.

3.6.2 Output Fields

Returns: true, if the IP address is in range; false, if the IP address is out of range.

3.6.3 Example

var df = new DiscoveryFunctions();
df.isInRange("71.191.290.131");


3.7 getXMLMemoryTable(tableName, doc, path)

Gets the XML memory table.

3.7.1 Input Fields

Parameters:

  • tableName - The table name to get.
  • doc - The document where the table resides.
  • path - The path to the table.

3.7.2 Output Fields

Returns: the XML memory table.


3.8 getFieldsThatChanged(gr)

Gets the fields that have changed for a given device.

3.8.1 Input Fields

Parameters:

  • gr - (GlideRecord) The device.

3.8.2 Output Fields

Returns: an array of fields that have changed.

3.8.3 Example

var df = new DiscoveryFunctions();
df.getFieldsThatChanged("device1");


3.9 updatedHistory(gr, sensor, eccID)

Updates the history of the current device.

3.9.1 Input Fields

Parameters:

  • gr - The device.
  • sensor - The sensor.
  • eccID - The External Communication Channel (ECC) Queue ID.

3.9.2 Output Fields

Returns: void


3.10 createRelationship(parent, child, pDescription, cDescription)

Creates a relationship between two virtual machines.

3.10.1 Input Fields

Parameters:

  • parent - The parent VM.
  • child - The child VM.
  • pDescription - A description of the parent.
  • cDescription - A description of the child.

3.10.2 Output Fields

Returns: (GlideRecord) the sys_id of the relationship.


3.11 createRelationshipIfNotExists(parent, child, descriptor)

Creates a relationship between a given parent and child if none already exists.

3.11.1 Input Fields

Parameters:

  • parent - The parent VM.
  • child - The child VM.
  • descriptor - The relationship descriptor.

3.11.2 Output Fields

Returns: the sys_id of the relationship if found or created; otherwise, returns null or undefined.


3.12 relationshipExists(parent, child, descriptor)

Determines whether a relationship already exists between the parent and child.

3.12.1 Input Fields

Parameters:

  • parent - The parent VM.
  • child - The child VM.
  • descriptor - The relationship descriptor.

3.12.2 Output Fields

Returns: (GlideRecord) the record of the relationship, if it exists.

3.13.1 Input Fields

Parameters:

  • reason - The reason for the reclassification.

3.13.2 Output Fields

Returns: (GlideRecord) the reclassified CI.


3.14 deleteCIBySysId(sysId, workflow)

Deletes a VM by its sys_id.

3.14.1 Input Fields

Parameters:

  • sysId - The sys_id of the VM to delete.
  • workflow - The workflow of the VM to delete.

3.14.2 Output Fields

Returns: void


3.15 deleteCI(gr, workflow)

Deletes a VM by record (cmdb_ci_computer) and workflow application (cmdb_ci_appl).

3.15.1 Input Fields

Parameters:

  • gr - (GlideRecord) The cmdb_ci_computer for the VM.
  • workflow - (GlideRecord) The workflow application cmdb_ci_appl.

3.15.2 Output Fields

Returns: void


3.16 deleteCIAndRelationshipsRecursive(appList)

Deletes VMs by traversing through the relationship tree. Traverses through the relationships that have RUNS_ON and HOSTED_ON relationships, looking for all the parent CIs that run on or are hosted on the child CI(s) and deletes them.

3.16.1 Input Fields

Parameters:

  • appList - The application list.

3.16.2 Output Fields

Returns: void

3.21 findRegistryNode(currNode, regName)

Finds the registry node for a given registry item.

3.21.1 Input Fields

Parameters:

  • currNode - The current node.
  • regName - The registry name to find.

3.21.2 Output Fields

Returns: the node, if found; otherwise, returns null.

3.21.3 Example

Example payload:
<results probe_time="6313">
  <result>
    <Registry>
       <entry key="HKEY_LOCAL_MACHINE">
          <entry key="Software">
             <entry key="Microsoft">
                <entry key="Windows">
                    <entry key="Name">
                        <value>Just a name</value>
                     </entry>
                  </entry>
               </entry>
            </entry>
         </entry>
      </Registry>
    </entry>
</results>

 // registry is a variable representing the payload
 var node = findRegistryNode(registry, "HKEY_LOCAL_MACHINE.Software.Microsoft");


3.22 findNodeWithAttribute(currNode, attrName)

Finds the node with a specified attribute.

3.22.1 Input Fields

Parameters:

  • currNode - The current node.
  • attrName - The attribute of the node to find.

3.22.2 Output Fields

Returns: the node array, if found; otherwise, returns null.


3.23 findNodeValueWithAttribute(currNode, attrName)

Finds the node value with a specified attribute.

3.23.1 Input Fields

Parameters:

  • currNode - The current node.
  • attrName - The attribute of the node to find.

3.23.2 Output Fields

Returns: the node array, if found; otherwise, returns null.

3.23.3 Example

Example payload:
<results probe_time="6313">
  <result>
    <Registry>
       <entry key="HKEY_LOCAL_MACHINE">
          <entry key="Software">
             <entry key="Microsoft">
                <entry key="Windows">
                    <entry key="Name">
                        <value>Just a name</value>
                     </entry>
                  </entry>
               </entry>
            </entry>
         </entry>
      </Registry>
    </entry>
</results>

 // To find the value of the key called "name", here's how these methods can be used.
 // registry is a variable representing the payload
 var node = findRegistryNode(registry, "HKEY_LOCAL_MACHINE.Software.Microsoft");
 var name = findNodeValueWithAttribute(node, "Name");
