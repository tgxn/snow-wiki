1 Overview

The Nested Interval Utilities API provides methods for managing the Planned Task tree structure for the Planned Task [planned_task] table. This API requires that the planned_task table has integer fields called a11, a12, a21, a22, level, and child_position. The table must also have three reference fields called parent, root_id, and top_task.

The Nested Interval Utilities API is available starting with the Dublin release.

1.1 Where to Use

Use these methods to manage the tree structure for the planned_task table in the Services interface.

3 Method Detail

3.1 NestedIntervalUtils(table)

This constructor takes the table name of the records to be inserted in the tree.

The table must have:

  • Integer fields called a11, a12, a21, a22, level, and child_position.
  • Reference fields called parent, sub_tree_root, and top_task.

3.1.1 Input Fields

Parameters:

  • table - (String) The name of the table to query.

3.1.2 Output Fields

Returns: void.


3.2 getTable()

Gets the name of the table to be queried.

3.2.1 Input Fields

Parameters:

  • None

3.2.2 Output Fields

Returns: The table name passed when constructing the utility.


3.3 initializeNode(newNode, parentID)

Initializes a new node in the tree under the parent. If no parent is defined, a new tree is created with the node as the root.

3.3.1 Input Fields

Parameters:

  • newNode - (GlideRecord) The node to add.
  • parentID - (String) The parent of the new node.

3.3.2 Output Fields

Returns: (boolean) true, if node values were set correctly; otherwise, false.


3.4 initializeSubTreeNode(newNode, parentID)

Initializes a new node in the tree, giving subtrees a separate sub_tree_root but with the same top_task as the main tree.

3.4.1 Input Fields

Parameters:

  • newNode - (GlideRecord) The new subtree root to be initialized.
  • parentID - (String) The parent of the new subtree root.

3.4.2 Output Fields

Returns: (boolean) true, if node values were set correctly; otherwise, false.


3.5 moveNode(node, position)

Moves a node to the specified child position [child_position] under the parent. For example, passing 3 as the position parameter moves the node to the third position in the list of children.

3.5.1 Input Fields

Parameters:

  • node - (GlideRecord) The node whose position is to be modified.
  • position - (int) The new child position [child_position] for the node under its parent.

3.5.2 Output Fields

Returns: (boolean) true, if the move was successful; otherwise, false.


3.6 changeParent(node, newParentID)

Changes the parent of the given node to the record identified by the sys_id specified in the newParentID parameter, and recalculates the tree values for the record and all of its descendants.

3.6.1 Input Fields

Parameters:

  • node - (GlideRecord) The record of the node whose parent is to be changed.
  • newParentID - (String) The sys_id of the new parent for the record.

3.6.2 Output Fields

Returns: (int) the status:

  • 1 = success
  • 2 = duplicate (relationship already exists)
  • 3 = recursive
  • 4 = invalid data
  • -1 = database error


3.7 checkNewParentValidity(itemTaskSysId, newParentSysId)

Checks the validity of a new parent node.

3.7.1 Input Fields

Parameters:

  • itemTaskSysId - (String) The item task sys_id.
  • newParentSysId - (String) The sys_id of the new parent.

3.7.2 Output Fields

Returns: (int) 1 = valid, any other value = invalid.


3.8 getChildren(parentID)

Gets all the children of a given parent, ordered by child position .

3.8.1 Input Fields

Parameters:

  • parentID - (Object) The record whose children are to be retrieved.

3.8.2 Output Fields

Returns: (GlideRecord) All of the children in order of child position [child_position].


3.9 getDescendants(node, inclusive, extraQuery)

Gets all of the descendants of the a node. Records are returned in order of rows descending the tree, then by parent, and then by child position.

3.9.1 Input Fields

Parameters:

  • node - (GlideRecord) The node whose descendants are to be retrieved.
  • inclusive - (boolean) Indicator for whether the passed node should be included in the returned records.
  • extraQuery - (String) An encoded query to further restrict the results of the function (similar to GlideRecord addEncodedQuery).

3.9.2 Output Fields

Returns: (GlideRecord) An ordered record of all the descendants, starting with the root and descending.


3.10 getDescendantsByLevel(node, inclusive, level)

Gets all of the descendants of a given node by the specified level.

3.10.1 Input Fields

Parameters:

  • node - (GlideRecord) The node whose descendants are to be retrieved.
  • inclusive - (boolean) Indicator for whether the passed node should be included in the returned records.
  • node - (int) The level of descendants to get.

3.10.2 Output Fields

Returns: (GlideRecord) An ordered record of all the descendants at the specified level, starting with the root and descending.


3.11 getAncestors(node, inclusive)

Gets all the ancestors of the given node.

3.11.1 Input Fields

Parameters:

  • node - (GlideRecord) The node whose ancestors are to be retrieved.
  • inclusive - (boolean) Indicator for whether the passed node should be included in the returned records.

3.11.2 Output Fields

Returns: (GlideRecord) The ancestors of the specified node, ordered by level ascending the tree.


3.12 getSiblings(node, inclusive)

Gets all of the siblings of the given node.

3.12.1 Input Fields

Parameters:

  • node - (GlideRecord) The node whose siblings are to be retrieved.
  • inclusive - (boolean) Indicator for whether to include the passed record in the returned records.

3.12.2 Output Fields

Returns: (GlideRecord) The list of siblings, ordered by child number.


3.13 copyPartialTree(node, newParentID, isSubTree)

Copies a complete or partial tree and inserts the given node and all of its descendants under the specified new parent. Any subtrees within the descendants are copied over as a subtree.

3.13.1 Input Fields

Parameters:

  • node - (GlideRecord) The record of the node to be copied.
  • newParentID - (String) The sys_id of the parent node to insert the node under.
  • isSubTree - (boolean) Indicator for whether the new tree is a subtree.

3.13.2 Output Fields

Returns: (String) The sys_id of the top node that was inserted.


3.14 copyPartialProject(node, newParentID, isSubTree, resetFieldsObj, setDefaultsObj)

Copies a complete or partial project by copying and inserting the given node and all of its descendants under the specified parent, and setting default fields and fields that need to be reset or removed.

3.14.1 Input Fields

Parameters:

  • node - (GlideRecord) The record of the node to be copied.
  • newParentID - (String) The sys_id of the parent node to insert the node under.
  • isSubTree - (boolean) Indicator for whether the new tree is a subtree.
  • resetFieldsObj - (Object) A string list of field names that need to reset or removed.
  • setDefaultsObj - (Object) A hash map of field names and values that need to be set.

3.14.2 Output Fields

Returns: (String) The sys_id of the top node that was inserted.


3.15 getOldIDs()

Gets the array of the old sys_ids of the records that were copied by the most recent copyPartialTree() call.

3.15.1 Input Fields

Parameters:

  • None

3.15.2 Output Fields

Returns: An array of Strings that contain the old sys_ids of the copied records.


3.16 getNewID(oldID)

Gets the new sys_id of the record that was copied by copyPartialProject() with the passed sys_id.

3.16.1 Input Fields

Parameters:

  • oldID - (String) The old sys_id of the copied record.

3.16.2 Output Fields

Returns: The new sys_id that corresponds to the old sys_id.


3.17 deleteSubTree(parentNode, inclusive)

Deletes a record and all of its descendants.

3.17.1 Input Fields

Parameters:

  • parentNode - (GlideRecord) The node whose descendants are being deleted.
  • inclusive - (boolean) Indicator for whether the passed record should also be deleted.

3.17.2 Output Fields

Returns: void.


3.18 getChildCount(parentID)

Gets the number of child nodes for a given parent.

3.18.1 Input Fields

Parameters:

  • parentID - (String) The node whose child count is to be retrieved.

3.18.2 Output Fields

Returns: (int) The number of child nodes.


3.19 getTopTask(node)

Gets the top task of the complete tree where the record resides.

3.19.1 Input Fields

Parameters:

  • node - (GlideRecord) The node whose top task is to be returned.

3.19.2 Output Fields

Returns: (GlideRecord) The record of the tree's top task.


3.20 getSubTreeRoot(node)

Gets the root of the direct subtree in which the record resides.

3.20.1 Input Fields

Parameters:

  • node - (GlideRecord) The node whose subtree root is to be returned.

3.20.2 Output Fields

Returns: (GlideRecord) The record for the subtree's root [sub_tree_root].


3.21 removeNode(node)

Removes the given node from the tree structure, but does not delete it.

3.21.1 Input Fields

Parameters:

  • node - (GlideRecord) The node to remove.

3.21.2 Output Fields

Returns: void.


3.22 getMaxLevel(parentID)

Gets the maximum level of the descendants of the given record.

3.22.1 Input Fields

Parameters:

  • parentID - (String) The record whose children are to be checked.

3.22.2 Output Fields

Returns: (int) The maximum level value.


3.23 getLeafNodesForStateChange(itemTaskSysId, leafOfSuccessors)

Gets the sys_ids of nodes that need to be activated. If the leafOfSuccessors parameter is set to true, this method first finds all active successors whose lag time is 0. From this subset of successors, it then gets the sys_ids of successors with:

  • The time constraint ASAP
  • No predecessor
  • All predecessors in the INACTIVE state

3.23.1 Input Fields

Parameters:

  • itemTaskSysId - (String) The sys_id of the task.
  • leafOfSuccessors - (boolean) If true, gets the sys_ids of all the successor nodes of itemTaskSysId that need to be activated; if false, gets the sys_ids of itemTaskSysId that need to be activated.

3.23.2 Output Fields

Returns: (String) the sys_ids of the nodes that need to be activated.


3.24 closeWholeProject(topTaskNode, taskKeyVals)

Closes all the tasks for a given root.

3.24.1 Input Fields

Parameters:

  • topTaskNode - (Object) The sys_id of top task, or root, of the project. It is the responsibility of the caller to get the topTask of a project and pass it here.
  • taskKeyVals - (Object) The key / value pairs to be set for each task under the project.

3.24.2 Output Fields

Returns: (int) The maximum level value.

3.24.3 Example

//Use in a business rule that is expecting to close all the tasks for project  
var tree = new SNC.NestedIntervalUtilities("planned_task");
var taskKeyVals = new Object();
var nowTime = gs.nowDateTime();
var taskKeyVals = {state:"3", work_end:nowTime};
tree.closeWholeProject(current, taskKeyVals);
