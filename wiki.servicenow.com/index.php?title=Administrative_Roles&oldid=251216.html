Administrative Roles

Administrative Roles

User Administration


Note: This article applies to Fuji. For more current information, see Special Administrative Roles at http://docs.servicenow.com

The Wiki page is no longer being updated. Please refer to http://docs.servicenow.com for the latest product documentation.


Overview

In previous versions of the product, a great many administrative tasks and rights were granted through the admin role. The rights to change business rules, client scripts, and UI policy, and to create script includes were all controlled by this role. In the current system, a family of more granular administrative roles allows the granting of more specific rights to individual users without granting the broader privileges of the admin role. For example, an administrator can now grant a user the rights to change UI policy, but not the rights to edit client scripts.

Note: The addition of these new roles does not change the existing behavior of the admin role, which still retains its general administrative privileges on the platform.
