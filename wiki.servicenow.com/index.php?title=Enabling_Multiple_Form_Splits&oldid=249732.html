Enabling Multiple Form Splits

 Functionality described here requires the Admin role.
Note: The latest release that this documentation applies to is Fuji. For documentation on the Geneva release, see Multiple form splits. Documentation for later releases is also on docs.servicenow.com.

Overview

It is possible to define more than one group of splits on a form with the property glide.ui.form_multiple_splits. If this property is missing or false, then things work as they always have. If this property is set to true, the following options appear in the Available column of the configure form slushbucket:

  • |- begin_split -|
  • |- split -|
  • |- end_split -|

When you move any of these options to the Selected column, the names remain in the Available column, so you can select them multiple times. Each of these split options makes up a form group that will contain two columns, enabling you to create elements that span the form at the top of the form and also multiple two-column split element groups within the form.

Note: Additional split sections beyond the default one must contain all three items above: the begin, the split, and the end.


The property to enable multiple form splits may not exist in some instances, and an admin at your organization may need to create it. If this is the case, create it as a true or false property, give it an appropriate description, and set it to true. An appropriate description might be "Enable multiple form splits". The property will be found in System Properties > UI Properties if it exists in your instance.
