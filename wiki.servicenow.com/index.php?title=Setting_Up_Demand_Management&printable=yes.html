1 Overview

Before using demand management, demand managers should perform these tasks:

  • Do preliminary planning.
  • Populate the stakeholder registry.
  • Create assessment categories.
  • View the bubble chart on the demand workbench (Fuji release) or create bubble charts (Eureka release).

2 Planning for Demand Management

  • Roles: Determine who should have the demand manager [demand_manager] and demand user [demand_user] roles.
  • Portfolios: Determine how you want to group or categorize demands, and then have a user with the project_manager role create portfolios based on those groupings. Demands can belong to multiple portfolios.
  • Stakeholders: Identify individuals who have the appropriate domain knowledge to evaluate demands related to each portfolio. Then, make them stakeholders for that portfolio and identify if they are to receive assessments or be demand approvers. Changes made to Approver or Receive Assessment in the demand record update the stakeholder register.
  • Assessments: Consider using assessments to facilitate the information gathering process. Define metric categories and assessment metrics needed to develop and distribute assessments to the appropriate audience. Assessments are automatically sent to the Receive Assessment portfolio stakeholders once the demand is moved into the screening state.
  • Bubble chart definitions: (for the Eureka release) Users can identify metrics that are important to stakeholders, departments, and the organization. Use these metrics to create bubble charts that visually compare demands. For example, when considering which projects to undertake, many organizations compare metrics such as risk rating, return on investment, and cost. For more information, see Administering Bubble Charts.
  • Resource plans: Consider developing resource plans to help the organization understand the time and resource costs associated with the demand. Resource plans require that the Resource Management application is active. Resource plans created for approved demands create duplicate resource plans in the project record.
  • Enhancement and defect requests: Determine if you need the ability to manage enhancement and defect requests as demands. This functionality requires that the SDLC Scrum Process Pack is active.

3 Populating the Stakeholder Registry

Add users to the stakeholder registry so that demand management can automatically populate the stakeholder list when a user creates a demand.

To add users to the stakeholder registry, navigate to one of the following and then click New to create a new record (see table for field descriptions).

  • Demand > Settings > Stakeholders (Fuji release).
  • Demand Management > Stakeholders > All (Eureka release).

You can also scroll to the Stakeholders related list on the Demand form and click New to create a record.

The User and Portfolio fields on the Stakeholder Register form are mandatory. When adding a user to the stakeholder registry, the combination of the user and the assigned portfolio must be unique. If you attempt to create a new stakeholder with a user name and portfolio that already exists, you will receive an error message.

Stakeholder Register form

Field Description
Number Unique identification number for the stakeholder. This number is automatically generated when a new entry is created.
User Name of user being added to the stakeholder registry. This is a required field starting with the Fuji release.
Portfolio A group of activities managed together to align to an organization's business or operational objectives. Typically, portfolios are named according to an organization's business units. This is a required field starting with the Fuji release.
Expectation Brief description of the user's expectations. Expectations can include a range of concepts such as how the demand is developed and what users get after the demand is completed.
Assessment recipient Indication of whether the user is authorized to receive assessment questionnaires.
Approver Indication of whether the user has authority to approve demands.
Influence Level of influence the user has over the group assessing the demand.
Level of interest Level of interest the user has in demands or demand outcomes. This is useful for determining levels of communication. For example, for users whose level of interest is Low, you may decide to communicate less frequently with them and only communicate high value information. This field is available in the Eureka release.
Engagement Indication of the user's support for demands. This is also useful for determining levels and channels of communication. For example, for users whose engagement type is Leader, you may want to communicate more often and in person so they can take appropriate action as issues arise over the course of the demand. This field is available in the Eureka release.

4 Creating Assessment Categories

Assessment categories are used in conjunction with assessment metric types and assessment metrics to generate bubble charts. The Demand Management application comes with the Demand assessment metric type, five assessment metric categories, and a number of assessment metrics.

Navigate to one of the following to modify existing assessment categories and create new categories:

  • Demand > Settings > Assessments (Fuji release)
  • Demand Management > Assessment Categories > All or Create New (Eureka release)

Default demand management assessment categories are as follows:

  • Size: Assesses demand size relative to the size of other demands.
  • Strategic Alignment: Assesses how closely the demand aligns with strategic goals of the organization compared to other demands.
  • Risk: Assesses demand risks compared to other demands.
  • ROI: Assesses demand return on investment compared to other demands.
  • Cost: Assesses demand cost compared to other demands.

For more information about creating assessment categories, see Creating Categories.

5 Viewing or Creating Bubble Charts

Bubble charts are useful for comparing demands based on common metrics such as risk rating, return on investment, and cost. Decision makers can use bubble charts when considering which demands to approve or reject.

5.1 Fuji Release

An interactive bubble chart for assessing business demands is available on the demand workbench. By default, this bubble chart displays all qualified demands based on three measurements: risk, value, and size. To display this bubble chart, navigate to Demand > Demands > Workbench.

If it is necessary to create a bubble chart, navigate to Assessments > Administration > Bubble Charts and create a new record. For more information, see Administering Bubble Charts.

5.2 Eureka Release

To create a bubble chart, navigate to Demand Management > Bubble Charts > Create New and create a new record. For more information about administering and creating bubble charts, see Administering Bubble Charts.
