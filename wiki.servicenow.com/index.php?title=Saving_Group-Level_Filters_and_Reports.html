1 Overview

Filters and reports can be stored with three levels of access control.

  • Me: The filter or report is visible only to the user who creates the report.
  • Everyone: The filter or report is visible to everyone in the system (may also include a role)
  • Group: The filter or report is visible to specific users or members of a specific group.

2 Saving Group Filters and Reports

Users can make a filter or report visible to all members of a group.

To save a group filter:

  1. Log in as a user with the admin role.
  2. Navigate to any list.
  3. Open the smart filter condition builder and create a filter.
  4. Click Save.
  5. In the Save as field, enter a name for the filter.
  6. In the Visible to field, select Group.
  7. Enter the group name or click the reference lookup icon () and select the group.
  8. Click Save.


To save a group report:

  1. Navigate to Reports.
  2. Select a report.
  3. Click the options menu next to Save.
  4. Click Sharing.
  5. In the Visible to field, select Groups and Users.
  6. From Groups, enter the group name or click the reference lookup icon and select the group.
  7. Click Close.
  8. Click Save.

To save a report with a role restriction:

  1. Navigate to Reports.
  2. Select a report.
  3. Click the options menu next to Save.
  4. Click Sharing.
  5. In the Visible to field, select Everyone.
  6. From Requires role, enter the role name or click the reference lookup icon and select the role.
  7. Click Close.
  8. Click Save.

3 Notes/Limitations

Security promotion moves a report, while a security demotion copies one.

If a user opens a global report and tries to save it as a group or personal report, the ServiceNow platform copies the report rather than moving it from one security state to another. This allows users who cannot create their own global reports to modify a global report, and then save their own personal version of the report. If a user opens a personal report and tries to save it as a group or global report, ServiceNow moves the report rather than copying it. The user must have the appropriate rights to do this.

4 Filters and Roles

The ability to create the various levels of filters is controlled by roles. All existing filters and reports retain their current visibility rights.

  • admin: Can create personal filters, global filters, and group filters for any group in the system
  • filter_global: Can create global filters
  • filter_group: Can create group filters for any group which the user belongs to
  • default: Can create personal filters as long as the user is assigned a role

Reports use a similar set of roles.

  • admin: Can create personal reports, global reports, and group reports for any group in the system
  • report_global: Can create global reports
  • report_group: Can create group reports for any group which the user belongs to
  • default(everyone else): Can create personal reports

Add roles to a user record at any time. If we granted Bob Smith the filter_global and filter_group roles, he could create the following filters:

  • Personal filters: Everyone with access to the filter widget has this right.
  • Group filters for his groups: Because of the filter_group role.
  • Public filters: Because of the filter_global role.
