1 Overview

VMware Provisioning is an Orchestration application that makes virtual servers available for request in the Service Catalog. When a virtual server is requested, Orchestration executes preconfigured approval and provisioning tasks. If the request is approved, Orchestration automatically creates a virtual server from a stored template, configures the virtual machine, and then starts the server.

Note: If you are using the ServiceNow Calgary release or later, see VMware for Cloud Provisioning.


2 How it Works

Orchestration in the ServiceNow platform uses a VMware Process Pack to integrate with the vCenter API. This Process Pack adds VMware Workflow activities to the existing Workflow application that enable Orchestration to clone new virtual machines from templates, configure virtual machines, and power virtual machines on and off.

3 Enabling VMware Provisioning

VMware provisioning is a feature of Orchestration, which is available as a separate subscription from the rest of the ServiceNow platform. To purchase an Orchestration subscription, contact your ServiceNow account manager.

4 Requirements

  • All virtual machine templates must contain VMware Tools.
  • For Windows VMs, click here to determine if Microsoft Sysprep is required on the vCenter instance.
  • The vCenter user must have proper credentials for cloning, customization, and for powering on the virtual machine.
  • On Windows 2003 templates, the password for an Administrator must be blank on the base image.

5 Tasks for Provisioning VMware Requests

The base ServiceNow system provides full functionality for ordering, approving, and provisioning virtual servers. Users must provide all VM templates, configuration information, group members, and IP addresses to allocate to virtual servers.

5.1 Task 1: Configure vCenter and the ESX Server

Install the vCenter management application. Create the Windows and Linux templates on your ESX Server that will be cloned and configured into virtual servers in Windows and Linux and load them on the ESX Server. Refer to VMware's product documentation for vCenter and the ESX Server for details about these procedures.

5.2 Task 2: Configure ServiceNow

  • Provide the Windows and Linux specific information for the virtual machines.
  • Create the pools of IP addresses available for allocation to virtual machines in each network.
  • Add users to the provisioner's and approval groups.
  • Provide credentials for the Discovery probes that log into vCenter to provision the requested virtual servers.
  • Configure the virtual server selections that are offered to the users in the Service Catalog.
  • Record the Resource Pools that were created on the ESX Server.
  • Enter the information on the VMware Templates into the platform.

5.3 Task 3: Approvals

Manually approve Service Catalog requests for virtual servers using a preconfigured task. This triggers the first provisioning task.

5.4 Task 4: Provisioning

Manually provision Service Catalog Requests using preconfigured tasks. This ultimately triggers a workflow that clones the virtual server from a template and powers up the machine on the network.

6 Installed with this Plugin

Installation of this plugin also installs the Orchestration plugin.

6.1 VMware Workflow Activities

The following Workflow activities are installed for VMware Support. For additional details about available variables, see VMware Activities.

Activity Table Description
Change State global Sends commands to vCenter to control the state of a given VMware virtual machine (powers the VM on or off).
Check VM Alive global Uses the VMware API to determine if a newly configured VM is alive. The virtual machine is alive if its state is powered on and if it uses the same IP address with which it was configured.
Clone global Sends commands to vCenter to clone a given VMware virtual machine.
Configure Linux global Sends commands to vCenter to set the identity and network information on a given VMware virtual Linux machine.
Configure Windows global Sends commands to vCenter to set the identity and network information on a given VMware virtual Windows machine.

6.2 MID Server Probes

Name Purpose
VMware - Clone Clones a virtual machine or virtual machine template
VMware - Change State Power on or power off a virtual machine
VMware - Configure Linux Sets the configuration specification for Linux
VMware - Configure Windows Sets the configuration specification for Windows
VMware - Get Guest Info Returns the state and IP address, used to determine if the virtual machine is up after configuration.

6.3 Workflows

Name Description
VMware - Provision Main workflow that clones, configures, and then powers on the virtual machine. This workflow uses the VMware - Wait for VM to start subflow, closes the task as complete or incomplete. Requestors receive an email if the provisioning was successful, and the IT provisioners group receives an email if the provisioning fails.
VMware - Wait for VM to start Subflow that queries the virtual machine using VMware Tools for its state and IP address to determine if it has powered up after its configuration changes.

6.4 Tables

Table Description
cmdb_ci_vm_instance Stores virtual machine instances.
cmdb_ci_vmware_instance Stores VMware instances
vm_allocatable_ip Stores IP addresses available for a given network.
vm_linux_config Contains specific information used to configure Linux servers.
vm_net_config Contains network configuration information used to configure new servers.
vm_win_config Contains specific information used to configure Windows servers.
