1 Overview

The Berlin release includes these minor enhancements and performance improvements.

2 Use

Enhancements for using the ServiceNow interface and performing self-service activities:

Feature Description Activated
Reports in UI11 Reports now open in a new frame when UI11 is active. Automatic for sessions running UI11.
Live Feed
  • Message time stamps are now indicated relative to the current time (for example, 1 minute ago). To see the exact post time, point to the time stamp.
  • A new property allows administrators to exclude public group messages from the Company Feed.
  • A new property allows administrators to define how frequently live feed is updated with new messages.
  • Notifications appear on the edge when new messages are posted in your My Feed (requires the UI11 plugin). When you open the bookmark, a blue circle appears beside new messages.
  • The PostToLiveFeed scripted web service, which allows external systems to post messages to live feed, now supports including a link in the post.
 Automatic for instances running Live Feed.
Date Filters The options Next quarter and Next 2 quarters are now available for date filters. Automatic for all instances.
CMS Homepage Cancellation Users now see a friendlier message when they cancel the loading of a homepage. Automatic for all instances.
Checking your release You can now see the feature release name when you check the version on an instance. The name appears on the Stats page and when you point to the logo in the header. Automatic for all instances.
Overlay Mask Enhancements to the overlay mask make resizing smoother for dialog boxes and backgrounds on a list. Automatic for all instances.

3 Administer

Enhancements for configuring and managing your ServiceNow instance:

3.1 Core Configuration

Feature Description Activated
Controls for List Buttons The List banner button field now appears on the UI Action form by default. The label for the existing List button check box now reads List bottom button to clarify where this option creates the button. Automatic for all instances.
Calendar Highlighting Administrators can now specify how calendar highlighting options are determined with the new property glide.ui.report.extend_calendar_choices. If the property is set to true, highlighting options are based on the current reporting table plus the base table. If the property is set to false, highlighting options are set by the styles in the Task [task] table only. Automatic for all instances.
Scroll Bars Setting the js_includes property to false no longer forces scroll bars to appear for pages that display less than one page of content. Automatic for all instances.
Condition Builder The condition builder now accepts the Changes, Changes to, and Changes from operators. These operators are only valid for conditions where an event is triggered, such as an email notification. Automatic for all instances.
UI Page A new UI page called not_found displays the error message when a user requests a page that does not exist (Form not found error). Administrators can customize the UI page.
not_found UI page

 Automatic for all instances.
Service Catalog Internationalization To improve usability and internationalization support, button images are replaced with HTML and CSS, and UI page code is streamlined. The affected UI pages are for these screens: Cart, Edit Cart, and Check Out (including workflows and approvals). If you have customized any of these elements, you must revert the customizations to receive the changes. Automatic for all instances.
US Holidays Schedule A new US Holidays schedule, which includes national holidays like Independence Day and Thanksgiving, has been added as a child of the 8-5 Weekdays schedule in the Schedule module. Automatic for new instances.
Annotation Fields An Omit spacer field is now available for the Form Annotation Type [sys_ui_annotation_type] table. When set to true, this option removes the spacer in the annotation's cell. Automatic for all instances.
Welcome Message Users can now click on their name in the welcome message to see their user profile. Administrators can control this behavior with the new glide.ui.welcome.profile_link system property. In new instances, this property is set to true. In existing instances, this property is set to false. Automatic for all instances.
Scheduled Job Improvements Scheduled jobs that previously had to run separately on multiple instance nodes now run on all nodes as a single job. Automatic for all instances.

3.2 Data Management

Feature Description Activated
Table Administration Administrators can now extend a table that is not already extended in the base system by configuring a system property. Automatic for all instances.
Display Value in Auto-complete Searches A new dictionary attribute, ref_ac_display_value, causes the reference field to hide the display value column so that auto-complete only matches text from the columns listed in the ref_ac_columns attribute. This feature requires the use of the AJAXTableCompleter class and the ref_ac_columns, ref_ac_columns_search, and ref_ac_display_value attributes. See Removing the Display Value Column. Automatic for all instances.
Table-level Attributes The attributes ref_auto_completer, ref_ac_columns, ref_ac_order_by, and ref_ac_columns_search for reference fields can now be defined at the table level. Element-level attributes with the same name override table level attributes. Automatic for all instances.
Data Lookup Data lookup definition, assignment, and priority for ITIL services now use data lookup tables. Automatic for all instances.
Glide List Dictionary Attribute The new dictionary attribute start_locked defines the initial locked or unlocked state for glide_list fields. Automatic for all instances.
Update Sets Two new business rules prevent users from renaming, deleting, or creating a second copy of the Default update set. Automatic for all instances.
Creating Users The User table user_name field [sys_user.user_name] enforces unique values. This guarantees that all users have a unique user ID. Automatic for new instances.

3.3 Discovery and Orchestration Automation

Feature Description Activated
vCenter Discovery Performance Discovery now identifies vCenter elements more efficiently. Automatic for instances running Discovery.
JavaScript Object The g_process object is now available for scripting use. The g_process object returns the value that Discovery sensors set for a process. Automatic for instances running Discovery.
VMware Orchestration Activity Variables Four new activity variables have been added in the VMware Clone activity and one existing variable has been deprecated. Automatic for instances running Orchestration Automation.
Selecting Templates to Provision Virtual Servers Orchestration now uses the same template for provisioning both custom and preconfigured virtual servers by replacing the specified components from the preconfigured server with those selected by the user. Automatic for instances running Orchestration Automation.
VMware Approval Group A new group called Virtual server networking is now available for selecting network information for virtual server requests made through the service catalog. Automatic for instances running Orchestration Automation.

3.4 Email Notifications

Feature Description Activated
Email Filtering
  • For inbound email filtering, administrators can now specify Content-Type headers to ignore.
  • Inbound email with a header of "Content-Type:multipart/report; report-type=delivery-status;" is now filtered by default.
Automatic for all instances.
Sending Email Notifications There are two new choices for triggering email notifications: when a named event calls the email notification or when a record is added or updated. The example notifications now use the added or updated trigger where appropriate. Automatic for all instances.
Email Properties New email properties allow administrators to:
  • Configure an instance to resend messages or stop delivery when a mail server returns specific SMTP error codes.
  • Toggle between stop and retry if the error codes are unknown (not configured in the other properties).
Automatic for all instances.
Maximum Size of Email Message Body A new property, com.glide.email.max_body_bytes, controls the maximum size a message body can have before the instance stops processing inbound email actions. The default value is set to match the data policy write limit of 16MB. You cannot set a higher value without conflicting with the write data policy limit. To set a lower limit, add this system property. Automatic for all instances.
Inbound Email Actions for Locked-out or Inactive Users Users who are locked out of an instance or whose accounts are inactive cannot update incidents via email. ServiceNow does not process inbound emails from these users. Automatic for all insts.

3.5 Other

Feature Description Activated
Role-Based Access to Emails in the Activity Formatter The property glide.ui.activity.email_roles controls which roles can see emails in the activity formatter. This property now has itil as the default role in new instances. Automatic for new instances.
On-call Reminder A new property, com.snc.on_call_rotation.reminders.showtz, lets administrators display a user's time zone along with the time period for an on-call rotation. This property is disabled by default. Automatic for instances running Group On-call Rotation.
Active Column for Database View Tables Administrators can deactivate a database view table for testing purposes. (Previously, you had to delete the database view table from the view if you wanted to test.) The Active column, with default value set to true, has been added to the Database View [sys_db_view] table. Automatic for all instances.

3.6 Performance

Feature Description Activated
System Diagnostic Display Value GlideServerStats now reports (headless) for MID buildstamp rather than using the Unix Epoch-0 timestamp. Automatic for all instances.
Transaction Cancellation Logging Additional information is now logged to differentiate whether a transaction is canceled by a user or by a defined quota. Automatic for all instances.


4 Deliver

Enhancements to process applications for delivering business services:

Feature Description Activated
Best Practice - ITIL KPI Reports Plugin Administrators can now directly activate the plugin. Activate the Best Practice - ITIL KPI Reports plugin.
Project Management Permissions Project users now have create and write access to the Expense Line [fm_expense_line] table. Automatic for instances running Project Management.
Service Catalog Property The new property glide.sc.checkout.task.display determines whether task records associated with a service catalog request record appear on the final checkout screen. Previously, this was only possible through UI page modification. The property is set to true by default. Automatic for all instances.
Additional Warning Messages The warning message for SLA Definition workflow conditions now includes additional detail. Automatic for all instances.
Expense Line Table Access Users with the itil role or the financial_mgmt_user role can now read records on the Expense Line [fm_expense_line] table. Automatic for instances running Cost Management.
Incident and Problem State Field The Incident and Problem tables now use the State field from the Task table instead of incident_state and problem_state. Automatic for new instances

5 Script

Enhancements for extending your instance with scripts:

Feature Description Activated
Improved Business Rule Debugging Record display values are now shown when administrators debug nested and recursive logic paths. Automatic for all instances.
Target Queues for New Events Administrators can select which event queue processes specific events without using scripts. There is now a Queue field on the event registry form. This does not affect existing event scripts. Automatic for all instances.
New Client Script Field The client script form now has a Description field to prevent confusion with the Messages field. Automatic for all instances.
Improved Script Include Functionality The XMLHelper script include now processes CDATA correctly. Automatic for all instances.

6 Integrate

Enhancements for integrating ServiceNow with third-party technologies:

Feature Description Activated
Web Service Consumer Support When consuming external web services, the SOAP message module can parse WSDL that contains imported XSD declarations and generate the correct SOAP Envelope templates. Automatic for all instances.
REST UI Policy Display messages for REST consumer errors now include parameter definitions and improved error reports when web services are unreachable. Automatic for all instances.
MID Server buildstamp The mid_buildstamp property replaces mid.version and identifies the MID Server version with an identifier based on the build date, expressed as yyyy-mm-dd-hhmm. Automatic for all instances.
MID Server upgrade error messages Two new messages can now be seen by users who encounter an error when upgrading a MID Server. Automatic for all instances.
