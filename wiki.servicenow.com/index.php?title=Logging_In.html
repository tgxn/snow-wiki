1 Overview

Each ServiceNow instance has a unique, secure web address. The base URL for each instance has the default format:

https://<instancename>.service-now.com

Users log in to the instance from a web browser.

Administrators can create a custom URL that redirects to the assigned URL, if needed.

2 Logging In

To access your company's ServiceNow instance:

  1. Enter the base URL in any web browser.
    • If your system uses external authentication, you are automatically logged in. For example, you may log in to company services when you log in to your computer.
    • If your system does not use external authentication, the Welcome page appears.
  2. Enter your user name and password.
  3. [Optional] Select the Remember Me check box to remain logged in until you manually log out.
    This option can be enabled or disabled by the administrator.
  4. Press Enter or click Login.
Welcome Page

3 Creating a Custom URL for an Instance

ServiceNow instances use the naming convention of instancename.service-now.com. Administrators can configure a custom URL, such as support.myurl.com, that redirects to instancename.service-now.com. While some customers choose to do this via mechanics such as a reverse proxy or other avenues discussed in the ServiceNow Community, ServiceNow does not currently support those methodologies.

If you set up a custom URL, ServiceNow, Inc. recommends also providing users with the default URL. This ensures that users can access the ServiceNow instance if any issues arise from the custom URL configuration.
