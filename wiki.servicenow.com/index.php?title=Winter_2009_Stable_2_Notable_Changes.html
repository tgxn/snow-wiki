1 Mobile Date/Time Picker

BlackBerry OS 4.6 or better (Bold or Storm) and iPhone users can click a button to the right of date or date/time fields to use a pop-up date/time picker.

2 Knowledge Base property to specify sort direction of sections

A new property, glide.knowman.section_sort_direction, lets an admin specify the direction to sort knowledge base sections. Default is descending, but an admin can now select "ascending". Navigate to Knowledge Base > Properties for the property with the description, "Knowledge Base section sort direction".

3 no_text_index field attribute

If a table is text indexed, the no_text_index attribute on a field will prevent it from being included in the text index. This is useful for fields that are updated continually, causing a large number of text_index events, but do not need to be searched. This can prevent the creation and processing of text_index events where only that field has been modified.

4 no_audit field attribute

If a table is audited, the no_audit attribute on a field will prevent it from being included in the audit log.

5 Script Change: getDisplayValue() on choice fields

The getDisplayValue() function on a choice type field displays the choice label for that choice entry. For example, the following logs "1 - Priority One". Previously "1" would have been logged: 

incident.priority = 1; 
gs.log(incident.priority.getDisplayValue())

6 Script Change: setValue() on choice fields

Calling g_form.setValue('choice_field_name', 'value', ['display_value']) from client script displays the display_value (or value) in the list as the current selection, even if the value is not defined as a choice for the list.

7 Nightly Refresh of Software License Counts - improved CMDB delete performance

The software license module (Asset Contracts -> Software Licenses) tracks licenses in use for software packages. Prior to Winter 2009 Stable 2, this count was implemented via business rules which incremented/decremented the count in response to activity on software instance records. This approach led to performance/concurrency issues on large instances. Now a scheduled job counts the number of licenses in use once a day. An admin can adjust the "Count Software Licenses" scheduled job as needed.

8 Manage MID server configuration from the instance

MID server configurations can now be viewed and edited from the instance.

9 Discovery: Microsoft clusters supported

Microsoft failover clusters are now detected and discovered. A CI is created for the cluster and its nodes, and the nodes are associated with the server that implements it. All cluster resources are enumerated and listed, and each cluster resource is associated with the node(s) that implement it.

10 Attachment form modified

The attachment form had two separate sections for selecting a file and then attaching it. These have been merged, with the Attach button moved to the right of the Browse... button:

11 Logout page refresh

Logging out now refreshes the entire page, instead of trying to correctly and individually set individual elements on the page.

12 Limit SOAP to 250 records

SOAP is limited to 250 rows, just like GetRecords.

13 glide.smtp.html property removed

The glide.smtp.html email property was unused and has been removed.

14 Licensed Users module

Navigate to User Administration > Licensed Users for a list of users with roles.

15 glide_secure internal type renamed to glide_encrypted

The glide_secure internal type has been renamed to glide_encrypted.

16 Deletions no longer audited in sys_audit_delete for some system tables

Deletes are no longer audited for several system tables, improving overall system performance. Tables include label entries, label history, workflow context.

17 glide.ui.list_mechanic.roles property

A new property in System Properties --> UI Properties can be used to specify who can use Personal Lists: "List of roles (comma-separated) that can use Personal Lists. If blank, all can use."

18 UI Policy builds faster

UI Policy building has been sped up approximately 100x on forms with a large number of UI Policies.

19 "Insert with Roles" UI Action for ACL rules

A new "Insert with Roles" UI Action duplicates an existing ACL rule including entries in the Requires Roles related list.

20 catalog_admin required to activate catalog item

A condition has been placed on the Activate and Deactivate UI Actions for catalog items so that only the catalog_admin role can use them.

21 JavaScript warning message for mobile user

Mobile users will see a warning message if JavaScript is turned off.

22 Property to set IMAP secure port

The IMAP secure port can now be customized with the glide.imap.secure.port property.

23 Email Content Types

The following email content types can be specified on an Email Notification:

  • HTML and plain text will send a multi-part mixed email with both an HTML and a Plain part. Most readers can, and will, read the HTML part
  • Plain text only will send a text/plain email
  • XML only will send a single part email with a content type of text/xml

24 'Mark as Requested' added to the list of convenience actions in the Approval Actions activity for Graphical Workflow

This allows a user to set the Approval state of a task to 'Requested' using the Approval Actions activity, instead of having to use a Set Values activity.

25 Help the Help Desk works with basic authentication activated

Help the Help Desk will now work even if basic authentication is enabled on an instance.
