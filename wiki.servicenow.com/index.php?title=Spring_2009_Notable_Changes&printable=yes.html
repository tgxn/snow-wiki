Spring 2009 Notable Changes

From ServiceNow Wiki

1 Changes for Spring 2009 release

1.1 URL field UI enhancement reverted

Winter 2009 Stable 4 included a URL field UI change. This change was removed for Spring 2009 and will most likely be restored in a future stable build or release.

1.2 Improved Activity Formatter

The improved activity formatter is active for all instances, replacing the legacy activity formatter.

1.3 Global Text Search header rearranged

The Global Text Search header was rearranged to improve readability.

1.4 Reference popup visibility for read-only fields

Reference popups/clickthroughs are hidden by default if a Client Script, UI Policy or ACL makes the field read-only. Being able to see or click through to the target record is unrelated to the reference field being writable. A UI property, glide.ui.reference.readonly.clickthrough, determines whether reference popups are present on read-only reference fields.

1.5 Click-through for mobile reference fields

Mobile reference field values can be clicked to see the referenced field.

1.6 update_synch attribute honored for sys_user_group table

The udpate_synch attribute is now honored for the sys_user_group table. By design, it was previously ignored.

1.7 Navigation frame icons combined

"collapse all" and "expand all" icon functionality in the navigation frame has been consolidated to one icon.

1.8 UI Action to reload from LDAP

A UI Action was added to the User table to reload the user data from LDAP.

1.9 Hint field added to UI Actions

A "Hint" field was added to UI Actions. The hint appears as a popup tooltip when hovering over a button, link or context action.

1.10 CMDB Relationship auditing audits updates

Previously, CMDB relationship auditing only tracked inserts/deletes, now updates are audited.

1.11 LDAP group population inactive by default for new instances

LDAP group population is now inactive by default for newly created instances.

1.12 Upload file access restricted to admin

Only admin users can upload files. This is unrelated to attachment capability. Creating the property, "glide.ui.security.upload.roles", can override this with a comma-separated list of roles.

1.13 Average aggregation added to reports

When creating a bar chart report, Average has been added to Count and Sum as choices for the "Aggregation" option.

1.14 Open nav filter forms and lists in new window

.list and .form can be added to a table name in the nav filter to see a list or form for that table (e.g., "incident.list" or "incident.form"). Now, the list or form can be opened in a new browser window by using .LIST and .FORM appended to the table name (e.g., "incident.LIST" or "incident.FORM").

1.15 Import Set rows with empty coalesce value are ignored

When the coalesced source field has no value, default behavior is to ignore the row. Customize this behavior by setting the property "com.glide.import_set.create_record_on_missing_coalesce" to true, which will result in a new record being created.

1.16 Empty URL fields locked by default

Empty URL fields are locked/collapsed by default. This behavior can be customized with the "glide.ui.unlock_empty_url" UI property.

1.17 Copy button removed from Record Producer

The Copy button was removed from the Record Producer form, since it did not apply to Record Producers.

1.18 "Integration - Microsoft SMS" plugin no longer available

The "Integration - Microsoft SMS" plugin is no longer available for installation by admin users, replaced with the "Integration - Microsoft SMS / SCCM 2.0" plugin.

1.19 Field hints added to form/list personalization slushbuckets

When personalizing a form or list, the Available and Selected fields will display their field hints as a popup tooltip. This does not apply when using the "cog" to personalize a list.

1.20 New reporting engine for new instances

New instances are created with a new reporting engine. Eventually, this reporting engine will be deployed to all instances to replace the existing one.

1.21 ID added to tabs toggle icon

An ID, "tabs_toggle_button", has been added to the tabs toggle icon, so it can be referenced via client script.

1.22 "Exclude manager" field added to Group table

A new field, "Exclude manager", has been added to the group table. If checked, email notifications sent to the group will not be sent to the group's manager. Note that the Subscription Based Notifications Plugin will override this if the manager is subscribed.

1.23 Double-clicking a date submits the date/time popup

Double-clicking on a date in the date/time picker now saves and dismisses the date/time popup.


2 Changes for Winter 2009 Stable 4

2.1 glide.ui.secure_cookies property

A glide.ui.secure_cookies property has been added to all instances, with a default value of true. If true:

  • Unused browser cookies expire after 30 days, requiring a user to reauthenticate if they have not logged in for 30 days.
  • An additional token is used in browser cookies and validated against the database to provide more strict user validation.

The property can be found by navigating to System Properties > Security. It will also appear in the UI Properties page. The property description is, "Enable additional cookie security. If checked, an additional token is used for more strict validation, and unused cookies expire after 30 days."

2.2 Context menu UI

The look and feel of context menus has been improved.

2.3 URL field link and label

(This Winter 2009 Stable 4 change was reverted for Spring 2009 due to issues found during testing, and it will likely be restored in a future stable build or release)

URL fields have been enhanced to allow for both a URL and a text label. When expanded on a form, the link and label can be entered separately. In a list or when collapsed on a form, the label is displayed. If no label is provided, the URL is displayed.

2.4 256-bit AES encryption for fields/attachments

Encryption support has been enhanced to use AES 256-bit encryption on instances that can support it. If supported, the option is available in the Type choice list on the Encryption Context form.

2.5 Hardcoded wiki links changed

In places where the wiki organizational structure has been modified, wiki links in ServiceNow have been modified to access the renamed articles or an appropriate portal page.

2.6 Tree picker node limit

A limit of 1000 has been placed on the number of nodes returned to the tree picker. This limit is configurable with the glide.ui.group_heirarchy.max_nodes property.

2.7 Impacted Services related list

A new related list, Impacted Services, has been added to the task table. It is a many-to-many relationship between task and cmdb_ci_service, displaying affected business services. The related list can be added to a task form (Change, Problem, etc.). It may be populated manually if desired, and a new form context menu UI Action labeled "Refresh Impacted Services" will auto-populate the list based on the Configuration Item on the task record's form. The UI Action will not remove manually-added services from the related list, but will add business services related to the CI.

2.8 Module form UI Policy

Module form (sys_app_module) client scripts have been converted to UI Policy to hide/display fields or make them mandatory as appropriate.

2.9 Text index checkbox on table dictionary records

The "Create or Remove Text Index" business rule has been modified to automatically and quickly set the text_index checkbox on related dictionary records as appropriate based on a change to a table's text_index checkbox. For example, if the text_index checkbox is checked for the Incident table's dictionary record, it is automatically checked for all other tables that extend task.

2.10 Notification Preferences UI

An improved Notification Preferences UI, accessible from a form link on the My Profile form, is available for instances using the Subscription Based Notifications plugin. Preferences are easily configured for each notification a user receives.

2.11 Field labels and splits for mobile lists

Field labels can now be added to mobile lists, and multiple fields can be displayed on one line by using splits when personalizing the Mobile view of a list.

2.12 New "sys_email" global variable available to Inbound Email Actions

In an Inbound Email Action, a new global variable "sys_email" is now available. It is the received sys_email record that triggered the Inbound Email Action. It can be used to reference fields on the email record, like uid, sys_id, content_type, etc.

2.13 AJAX calls reduced

The number of AJAX calls from the browser to the server has been reduced, improving performance.

2.14 Knowledge attachments display

Display Attachments option added to knowledge articles to enable the display of attachment links on the article view (kb_view) without having to use the Edit Article button.

2.15 List Collector variables in Client Scripts

It is now possible to add more than one list collector to a form. To accommodate this, the name of the global "g_filter" variable used to set the list collector filter in client script has been changed to be unique per variable. The new global variable used in client scripts is the name of the variable followed by "g_filter". For example, with the "Schedule a Move" catalog item, the name of the list collector variable is "items_to_move_", so to access it in client script the following must be used: "items_to_move_g_filter".

Impact: any existing catalog client scripts that reference a list collector filter with "g_filter" will need to be changed to match the new global variable naming convention.

2.16 Discovery

2.16.1 Windows services probe and sensor available now

The probe and sensor pair for discovering windows services are now available out of the box. It could be enabled through editing the windows classification.

2.16.2 Custom related list creation

An API function has been created for populating data in a custom related list.

2.16.3 CI hostnames can be stored in uppercase

A discovery property has been created for allowing hostnames to be stored in uppercase.

2.16.4 CMDB coalesce can be turned on or off

A discovery property has been created for turning CMDB coalesce functionality on or off

2.16.5 SNMP discovery fixes

Discovery now correctly populates base table information for SNMP devices (ie. printers, switches, UPS and etc...) as well as network adapters.


3 Changes for Winter 2009 Stable 3

3.1 Control who can see emails in Activity Formatter

A new property, glide.ui.activity.email_roles, specifies who can see emails in the activity formatter. End users may not be allowed to see emails, for example, in case they contain restricted work notes or other sensitive content. If this property does not exist or is blank, all users can see emails in the activity formatter if Sent/Received Emails are selected for display.

3.2 Improved attachment form

The attachment form has been simplified and made more intuitive.

3.3 Read only HTML rendered

If an HTML field is read only, rendered HTML is now displayed instead of the HTML markup.

3.4 Toggle Groups and Filters icon added for Firefox

The "Toggle Groups and Filters" icon has been added for Firefox. This icon hides and displays the "Group by", filters, and the rows per page controls at the top of lists.

3.5 New security properties

Two new security properties have been added to System Properties --> Security. One specifies whether [code] tags allow embedding HTML code in text fields, the other provides control over whether other websites can refer to URLs in the instance.

  • Allow support for embedding HTML code by using the [code] tag
  • Allow other websites to refer to URLs in your instance (Cross Site Requests)

3.6 SOAP requests impersonate the System user instead of Guest

When a SOAP request comes in and defaults to the Guest user, the System user is impersonated instead and the session is not interactive.

3.7 Display field for sys_email

The Subject field is now the display field for sys_email records. Previously references to sys_user would show the sys_id of the record, because it had no display field specified.

3.8 Email Watermark table exposed to admins

A module has been added, System Mailboxes --> Watermarks, to expose the email watermark table (sys_watermark) table to admins. An email watermark links an email to its source record (Incident, Problem, etc.) so replies can be properly associated.

3.9 LDAP Distinguished Name comparisons are case sensitive

Previously, LDAP distinguished name (DN) comparisons were case insensitive, like all database compares. They are now case sensitive.

3.10 IE and Firefox label wrapping consistent

IE and Firefox label wrapping is now consistent. Previously, IE would wrap long field labels, and Firefox would not. They are now consistent, and wrapping can be specified with the following property in System Properties --> CSS.

  • Form label white-space value. "normal" wraps long text in form labels, "nowrap" prevents wrapping.

3.11 last_login_device and ldap_server fields not text indexed

To improve text indexing performance on instances with heavy SOAP or LDAP update activity, the last_login_device and ldap_server fields are no longer text indexed. They can still be searched using a filter if necessary.

3.12 Faster UI Policy loading

Forms with a large number of UI Policies will load faster.

3.13 Email field change cmn_notif_device

The email field on the cmn_notif_device table (Notification Device) has been lengthened to 100 characters to be consistent with the email field on the sys_user table.

3.14 Homepage layout now a reference

Homepage layout was a text field, now it is a reference to a UI Macro to prevent invalid values.

3.15 View-specific UI Policy

UI Policies can now be made view-specific, like Client Scripts. Global and View fields have been added to the UI Policy table, and can be added to the form.

3.16 Session Timeout message displayed

If a user's session times out, they will see a message to this effect as they are being redirected to the login page.

3.17 User roles flagged as inherited

User roles are flagged as inherited unless explicitly added to user. All roles inherited from groups or contained in other roles will now be flagged as inherited.

3.18 Dictionary information context menu option

Form fields now have a context menu option for admins that displays a UI Page popup with relevant dictionary information about the field. This can be viewed without having to leave the original form.

3.19 Date/time picker saves when user clicks out

Clicking outside of the date/time picker after making a change will now save the change, not cancel it. This is consistent with list editing.

3.20 Global text search exact task number match

If a global text search query is an exact match for a task number, the task record will be returned instead of the normal search results. If this happens, an infoMessage provides a link to the full search results. This behavior can be disabled in the "Search tips and preferences" dropdown, and the system default can be changed in the ts.match system user preference in User Administration --> User Preferences.

3.21 Text index engine upgraded

We now use Lucene 2.4.1 for text indexing. Numerous performance improvements have also been made.

3.22 System UI --> Portal Pages module removed

Portal pages can still be accessed by navigating to Homepage Admin > Pages.

3.23 Portal Section has a new Summary field

A summary field is now included on a portal section record when looking at a Homepage (sys_portal_page) record. An admin who brings up a Homepage record (Homepage Admin --> Pages) can now see what is on the homepage by looking at the portal sections in the related list at the bottom instead of having to open each one to read the title.

4 Changes for Winter 2009 Stable 2

4.1 Mobile Date/Time Picker

BlackBerry OS 4.6 or better (Bold or Storm) and iPhone users can click a button to the right of date or date/time fields to use a pop-up date/time picker.

4.2 Knowledge Base property to specify sort direction of sections

A new property, glide.knowman.section_sort_direction, lets an admin specify the direction to sort knowledge base sections. Default is descending, but an admin can now select "ascending". Navigate to Knowledge Base > Properties for the property with the description, "Knowledge Base section sort direction".

4.3 no_text_index field attribute

If a table is text indexed, the no_text_index attribute on a field will prevent it from being included in the text index. This is useful for fields that are updated continually, causing a large number of text_index events, but do not need to be searched. This can prevent the creation and processing of text_index events where only that field has been modified.

4.4 no_audit field attribute

If a table is audited, the no_audit attribute on a field will prevent it from being included in the audit log.

4.5 Script Change: getDisplayValue() on choice fields

The getDisplayValue() function on a choice type field displays the choice label for that choice entry. For example, the following logs "1 - Priority One". Previously "1" would have been logged: 

incident.priority = 1; 
gs.log(incident.priority.getDisplayValue())

4.6 Script Change: setValue() on choice fields

Calling g_form.setValue('choice_field_name', 'value', ['display_value']) from client script displays the display_value (or value) in the list as the current selection, even if the value is not defined as a choice for the list.

4.7 Nightly Refresh of Software License Counts - improved CMDB delete performance

The software license module (Asset Contracts -> Software Licenses) tracks licenses in use for software packages. Prior to Winter 2009 Stable 2, this count was implemented via business rules which incremented/decremented the count in response to activity on software instance records. This approach led to performance/concurrency issues on large instances. Now a scheduled job counts the number of licenses in use once a day. An admin can adjust the "Count Software Licenses" scheduled job as needed.

4.8 Manage MID server configuration from the instance

MID server configurations can now be viewed and edited from the instance.

4.9 Discovery: Microsoft clusters supported

Microsoft failover clusters are now detected and discovered. A CI is created for the cluster and its nodes, and the nodes are associated with the server that implements it. All cluster resources are enumerated and listed, and each cluster resource is associated with the node(s) that implement it.

4.10 Attachment form modified

The attachment form had two separate sections for selecting a file and then attaching it. These have been merged, with the Attach button moved to the right of the Browse... button:

4.11 Logout page refresh

Logging out now refreshes the entire page, instead of trying to correctly and individually set individual elements on the page.

4.12 Limit SOAP to 250 records

SOAP is limited to 250 rows, just like GetRecords.

4.13 glide.smtp.html property removed

The glide.smtp.html email property was unused and has been removed.

4.14 Licensed Users module

Navigate to User Administration > Licensed Users for a list of users with roles.

4.15 glide_secure internal type renamed to glide_encrypted

The glide_secure internal type has been renamed to glide_encrypted.

4.16 Deletions no longer audited in sys_audit_delete for some system tables

Deletes are no longer audited for several system tables, improving overall system performance. Tables include label entries, label history, workflow context.

4.17 glide.ui.list_mechanic.roles property

A new property in System Properties --> UI Properties can be used to specify who can use Personal Lists: "List of roles (comma-separated) that can use Personal Lists. If blank, all can use."

4.18 UI Policy builds faster

UI Policy building has been sped up approximately 100x on forms with a large number of UI Policies.

4.19 "Insert with Roles" UI Action for ACL rules

A new "Insert with Roles" UI Action duplicates an existing ACL rule including entries in the Requires Roles related list.

4.20 catalog_admin required to activate catalog item

A condition has been placed on the Activate and Deactivate UI Actions for catalog items so that only the catalog_admin role can use them.

4.21 JavaScript warning message for mobile user

Mobile users will see a warning message if JavaScript is turned off.

4.22 Property to set IMAP secure port

The IMAP secure port can now be customized with the glide.imap.secure.port property.

4.23 Email Content Types

The following email content types can be specified on an Email Notification:

  • HTML and plain text will send a multi-part mixed email with both an HTML and a Plain part. Most readers can, and will, read the HTML part
  • Plain text only will send a text/plain email
  • XML only will send a single part email with a content type of text/xml

4.24 'Mark as Requested' added to the list of convenience actions in the Approval Actions activity for Graphical Workflow

This allows a user to set the Approval state of a task to 'Requested' using the Approval Actions activity, instead of having to use a Set Values activity.

4.25 Help the Help Desk works with basic authentication activated

Help the Help Desk will now work even if basic authentication is enabled on an instance.

5 Changes for Winter 2009 Stable 1

5.1 Format Phone Number Property

A property (glide.ui.format_phone) has been added which can be used to control the automatic formatting of 10-digit phone number fields in the U.S format of (999) 999-9999, for numbers that do not begin with "+".

If this property is ticked, include the "+" prefix to stop automatically formatting the phone number.

If the "+" prefix is not included, or the property is not ticked, the number is automatically formatted

To change the property, navigate to System Properties > UI Properties and look for the property with the description: Format the phone number fields if they do not begin with "+".

5.2 Security on a field is respected in the History

If a user does not have read-access to a field, that field's changes will be omitted when they view History of a record.

5.3 Customize view used for Global Text Search

In System Properties --> Global Text Search, there is a property to allow an admin to specify a different list/form view to be used for Global Text Search results. The default view is text_search.

5.4 Dependent Choice Fields Supported on Mobile Devices

Choice fields that depend on the value of another field are now supported on BlackBerry devices running 4.6 or better (such as the Bold and Storm) as well as iPhone.

5.5 Auto Pareto chart detection (focus field) has been removed

The capability caused confused consternation out of proportion to its usefulness.

5.6 g_form.setValue() now supports glide_list fields

The g_form.setValue() client script method now supports glide_list fields. See Set a field value in the Client Scripts article for additional detail.

5.7 Import Set Performance Improvements

The logic behind the transformation phase of an import set load has been revised for improved performance. In our test cases here, overall transformation time dropped by about 2/3 with the new logic. Individual customer results may vary, with the largest improvements being seen in transformations with extremely large numbers of reference or choice type fields.

5.8 Some client scripts converted to UI Policy

Many client scripts in the application were created before the advent of UI Policies. Client script on the following tables have been removed and UI Policies created in their stead:

  • expert_panel
  • sysauto
  • scheduled_import_set
  • scheduled_data_import
  • cmdb_ci_vpn

5.9 iPhone Multi-Line Text Inputs Expand When Edited

Multi-line text input fields expand to the size of their contents while being edited via iPhone to make the editing process easier.  When the "done" button is pressed to finish editing the field, it will return to the prior size.

5.10 Elimination of duplicate storage of journal fields

Prior to Winter 2009 Stable 1, the textual values of journal fields were stored twice. Once as rows in the sys_journal table, and additionally as a large text blob on the task table. The redundant copy on the task table has been removed.

5.11 SOAP changes with regards to journal fields

Prior to Winter 2009 Stable 1, a SOAP request against a table with journal fields (for example incident) would append all the available journal fields together (without formatting) and include that blob of text in the response.

Winter 2009 Stable 1 no longer includes journal fields in SOAP responses. To get the contents of a journal field, a second soap request should be made against the sys_journal table to pull the appropriate journal records back for the record in question.

5.12 Text index load reduction

Prior to Winter 2009 Stable 1, any change to a text-indexed table generated a text_index event that had to be processed to update the index for that record. This happened even if nothing changed that actually was part of the text index (like a Boolean or a date/time field).

Winter 2009 Stable 1 no longer creates the text_index event if nothing changed that the text index cares about, significantly reducing the text indexing workload for highly active tables.
