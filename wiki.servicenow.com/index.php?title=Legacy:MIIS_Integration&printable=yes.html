 This plugin has been retired and is no longer available for new activations (since Eureka).

This page applies only to existing users of the Integration - MIIS plugin.

1 Overview

The integration point between the two systems is a SQL database. MIIS already relies on a SQL server to store the information it needs. We leverage this server to host an additional database that will allow the two systems to exchange information.

Incidents in ServiceNow are created to request User Provisioning, the request is forwarded to MIIS via a MID Server issuing SQL commands to create records in the OBJECTS and ATTRIBUTES tables. Confirmation of the creation is communicated back to the instance, updating the incident with status.

Note: The following integration is not part of the supported platform. The integration may require additional customization, typically performed by an experienced administrator or consultant, prior to operating correctly.


2 Requirements

  • Install a MID Server that has access to the SQL Server database, it will be used to relay SQL commands to the database
  • Activate the Integration - MIIS plugin (com.glideapp.ecc.miis) by making a request to ServiceNow Technical Support (in versions prior to Eureka).

3 MID Server

Install a MID Server and configure it to connect back to your instance of ServiceNow.
