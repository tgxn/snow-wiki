1 Overview

In addition to being accessed through a supported browser, the platform can be accessed through email. The platform can generate email notifications to keep users informed about the relevant changes within their instance, and if the user responds the platform can process that email using scripts.

2 Email Notifications

An administrator can set an automatic rule to create an event once certain conditions have been met. The administrator can also set the event to trigger an email notification. The administrator defines what the conditions of the event and subsequent email notification will be, as well as what the email will say and who will receive it. By default, the users who have enabled email notification will be notified when an incident is opened on their behalf, assigned to them, or closed/resolved.

For more information, visit Email Notifications.

3 Inbound Email Actions

An administrator can also set inbound email actions to process received emails. Using a few lines of code, the administrator can specify numerous different responses to these emails. For instance, an Administrator can set a rule to create a new user from an email or approve a task or request. Because of the flexibility of the script, there are many possibilities for how to communicate with the platform via email.

For more information, visit Inbound Email Actions.
