1 Overview

Allocation rules contain the information that the Financial Management application needs to break expenses down and allocate them to segments. Rules use filter conditions, methods, and metrics.

Note: When you use the workbench to assign expenses to accounts and segments, it creates all the rules, methods, conditions, and so on automatically. Using the workbench is the preferred method of setting up allocations, rather than using lists and forms.


2 Creating Allocation Rules

  1. Navigate to IT Finance > Cost Model > Allocation Rules.
  2. Click New.
  3. Fill out the fields on the form as appropriate (see table).
  4. Click Submit.
Example allocation rule that allocates expenses by incident duration.

Field Description
Number Auto-generated identification number for the cost allocation rule.
Name Descriptive name for the rule.
Filter condition The allocation filter that applies to this rule. Select a filter that this rule can use.
Table The table that the rule runs on. The table is determined by the filter condition that the rule uses.
Order Numeric value that determines the order of the cost allocation rule, relative to other rules. Rules are processed in order from the lowest order number to the highest. When you create a new rule, the default order value is 10 points higher than the maximum order value of the existing rules. The order value must be unique. No other cost allocation rule on the same table can have the same order value.
Active If this rule is active and can be applied to expenses. Inactive rules are ignored during expense allocation. Rules are automatically activated when methods that belong to the rule add up to a full 100% allocation, and are deactivated if the methods do not add up to 100%.
Final If the allocation rule is the last rule that can apply to an expense during allocation (selected) or if the rule can be an intermediary rule that allows other rules to be run after it on the same allocations (not selected).
Capital cost If the cost is a capital expense (selected) or an operational expense (not selected).
Fixed cost If the cost is fixed (selected) or variable (not selected).
Bucket The bucket that you can use to categorize the expense. Select a bucket and a sub-bucket if necessary. If more than one rule matches a general ledger expense and each rule specifies different buckets, the allocation engine applies the bucket from the last rule that matched. See Putting Data Into Buckets for information.
Sub-bucket The sub-bucket that the expense belongs to.
Description A description of the rule.
Related Links
Preview Rule A preview of how the rule allocates the expenses that apply to the rule. Click this link to open the allocation viewer. This link appears only for active rules without processed allocations lines.
Related Lists
Cost Allocation Methods The allocation methods that this rule uses to break down expenses.
Cost Allocations The allocation lines that this rule created.
Note: You cannot delete a rule that is referenced by a locked allocation line.
