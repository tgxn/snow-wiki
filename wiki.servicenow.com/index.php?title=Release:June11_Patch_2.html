Overview

The June11 Patch 2 release provides fixes for the June11 release. To sort the entries in descending or ascending order, click the icon in the column heading.

This release is no longer recommended for upgrades. If you want to upgrade an instance, consider a current version. For the June11 Patch 2 release:

Build date: 10-21-2011_1430
Build tag: glide-june2011-05-25-2011__patch2
 Note: See how to determine the version on your instance.



Number Category Details Steps to Reproduce
PRB560020 Activity Formatter When set to false, the property glide.ui.escape_text made multi-line field values lose linebreaks in activity formatter.
  1. Set glide.ui.escape_text to false. You may need to create this property.
  2. Personalize the incident form and add a description field.
  3. Add this field to the activity formatter.
  4. Put a multi-line value in the field and save the form.
    • Note the slipped line breaks in the activity formatter.
PRB562833 Approvals The approval summarizer on the change request form was not rendering properly when the system property glide.ui.escape_text was set to true.
  1. Set the property glide.ui.escape_text to true.
  2. Navigate to a change request that has an approver assigned to it.
  3. Navigate to the approver record.
  4. View the approval summary.
    • XML displays instead of the rendered summary of the change.
PRB562359 Approvals Approval rules did not work as expected.
  1. Disable workflow service change request.
  2. Set the approval engine request to approval rules.
  3. Disable all out-of-box approval rules.
  4. Create an approval rule on the sc_request table.
  5. Order an item through the service catalog.
  6. Check that the sc_request record was created and the approval state is requested.
  7. Approve the approval request.
PRB561454 Attachments The Attachments related list did not get refreshed automatically after adding an attachment to the form.
  1. Ensure list v2 is activated.
  2. Open a change request.
  3. Ensure the Attachments related list is visible on the change request form.
  4. Add an attachment.
  5. Check the attachment related list. The new attachment is not displayed.
  6. Explicitly reload the form. The related list displays the attachment properly.
PRB564742 BSM Map The BSM map would go into an infinite loop when clusters contained circular references.
PRB564538 Change Risk Assessment Risk assessment survey answers were overwriting previous assessments when recalculating an assignment on an earlier change.

  2. Perform a risk assessment and set all values to Low.
    • The risk is low.
  3. Create a new change.
  4. Perform a risk assessment and set all values to High.
    • The risk is very high.
  5. Recalculate the originally low risk assessment, it displays as very high.

  1. Navigate to https://demo.service-now.com/ess/order_services.do in IE8.
  2. View VPN RSA token link in main content frame.
    • The bottom text is cut off.

PRB043138 Email Receiving an email that contains attached HTML with a content-disposition of "inline" and a message content-type of "multipart/alternative" were not handled correctly.
  1. In Mozilla Thunderbird, create a new message and explicitly specify a format of Plain and Rich (HTML) Text.
  2. Drag an a HTML file into the message body.
  3. View the message source.
  4. The source should specify the message as Content-Type: multipart/alternative; and the attachment as Content-Disposition: inline;.
    • The contents of the file will be in the body but not in the body text and not attached to the incident.

  3. Export the report to Excel.
  4. Enter an email address in the dialogue and click Mail it.
  5. When the report is emailed open the attached Excel file.
    • There is a missing record between sheets 1 and 2.

PRB561416 Import Sets Transform Maps could run business rules when they were set to not do so.
  1. Set up a web service import.
  2. Set two business rules to run before insert with current.description set to += "\n This is a business rule that is running on incident or task";.
  3. Assign one business rule to incident and the other to task.
  4. Change the text depending on the table the business rule is assigned to.
    • Use the attached transform script.
  5. Load SoapUI and use the WSDL for your web service import set.
  6. Perform an insert on the task table.
    • With appropriate logs you will see the business rules fire even if set not to.
if (action == 'Insert') {
  target.sys_class_name = "Incident";
}

PRB559606 SLAs SLA Percent Elapsed was calculated incorrectly.
  1. Create an SLA definition with duration of 10 minutes.
  2. Create task that satisfies start condition of this SLA.
  3. Let SLA run 1 minute.
    • The elapsed percentage is 10%.
  4. Pause the SLA for one hour.
  5. Resume the SLA.
    • The elapsed percentage for the SLA will be 87%.
PRB561883 SLAs The flag has_breached was not set when SLA elapsed time was greater than or equal to 100%.
  1. Ensure an SLA Definition exists with a small duration for testing purposes.
  2. Create a record that meets the SLA conditions.
  3. Ensure task_sla is created and sys_trigger record created for that task_sla.
  4. Wait for the incident to reach the planned end time.
  5. Ensure that sys_trigger record ran.
  6. See that the task_sla in question is successfully recalculated with a percentage >= 100%, but the has_breached flag remains false.
  7. Update the parent task record such that the SLA is completed, the has_breached will be appropriately set to true.
PRB563289 SOAP Malformed "variables" element in SOAP responses when using displayvariables=true.
  1. Navigate to the service catalog and order business cards.
  2. Drill into the request item for the business cards and obtain the sys_id for the sc_req_item record.
  3. Use SOAP UI to retrieve the sc_req_item record with a 'get' or 'getRecords" request. Specify "displayvariables=true" on the SOAP URL.
  4. Observe that the variables element has incorrect hierarchical XML structure.
PRB563955 SOAP The wrong HTTP status could was returned by SOAPMessage, SOAPEnvelope, and SOAPRequest script includes if use_ecc was set to true.
PRB068711 SSH SSH probe threads, transport threads, and Discovery probe status would hang against network devices.
PRB560995 Surveys Survey Wizards did not update the relevant Task Survey record with the identity of the survey taker.
  1. Create a survey wizard.
  2. Set up the survey to be triggered by a task.
  3. Trigger the survey through this task.
  4. Fill out the survey.
    • Note the missing updates in Survey Instances and Task Surveys.
PRB562638 Timeline Duration calculation could be incorrect when dragging the end date from a timeline.
PRB563795 Transform Maps Updates to sys_script_include may cause exceptions in running transform maps.
  1. Create a transform table called u_test_cache.
  2. Give the table a field called u_input.
  3. Map u_input to the Short description field on incidents. Ensure this mapping is not set to coalesce.
  4. Execute loop.sh.
  5. Run the attached script as a background script while loop.sh is running.
for (var i = 0; i < 1000; i++){
gs.cacheFlush("syscache_script_include")
gs.sleep(10);
}
PRB561215 UI The None option in a slushbucket was populating the Information Fields for Available Items section below the slushbucket.
  1. Log on to any instance as a user.
  2. Open an incident and change the view from the default.
  3. On the Affected CI related list click the edit button.
  4. In the slushbucket search box enter a string that returns no results.
  5. Select the None item.

  1. Open an incident form.
  2. Click on a magnifying glass to display a popup window.
  3. Close the popup window by clicking the close button.
    • Firefox will crash or throw an error.

PRB562430 UI The user_image field showed escaped HTML in the related list view (sys_ref_list) when the glide.ui.escape_text property was set to true.
  1. Verify the property glide.ui.escape_text is set to true.
  2. Open an incident form.
  3. Click the magnifying glass next to the Assigned To field.
  4. At the top of sys_ref_list, right-click the header and choose Personalize > List Layout.
  5. Add the photo column.
  6. Click Save.
    • Note the escaped HTML displaying in the photo column.
  7. Set glide.ui.escape_text to false and repeat the above.
    • The photo is displayed correctly.

PRB561986 Web Services Some SOAP properties were ignored.
  1. Set glide.web_service.hierarchical to true.
  2. Post a hierarchical SOAP envelope.
    • Note only the top level record is inserted.
  3. Add hierarchical=true to the endpoint URL.
    • Note all the records are inherited.
PRB560251 Web Services Soap message fault was displaying full stack trace.
  1. Access the application with any valid user and configure a web proxy tool to catch the request.
  2. Navigate to the URL https://<instance name>/ecc_queue.do?SOAP''.
  3. Modify the GET parameter by POST.
  4. In the body content of the request add the attached statement.
  5. Click Continue.
    The stack trace appears in the XML due to an inappropriate parser validation.

PRB562564 Web Services Scripted web services that return a DOM element result generated an exception in java.io.FileOutputStream.writeBytes(Native Method) when responses were larger than 256K.
  1. Create a scripted web service which returns a large DOM element as its response. The serialized length of the DOM element must exceed 256K.
  2. The scripted web service must return the element response.soapResponseElement.
  3. Turn on debug for SOAPProcessor by setting glide.processor.debug.SOAPProcessor to true.
  4. Invoke the web service from the SOAP UI.
PRB562616 Workflow Pending Approvals could be improperly changed to the No Longer Required state.
  1. Create a workflow associated with sc_request_item.
  2. Ensure the workflow has an approval, either user or group.
  3. Assign a group or multiple individuals to approve.
  4. Associate the workflow with a service catalog item.
  5. Request the item through the service catalog.
  6. In My Approvals modify the filter to show all approvals.
  7. Verify that all the users assigned to the workflow have an approval and it's in the request state.
  8. Approve the request using an assigned approver.
  9. Verify that the workflow resumes and finishes.
  10. Check the approval status using a different user.
    • The pending approval could display as No Longer Required.
PRB562088 Workflow Workflow timers did not use a schedule when the This workflow's schedule option was specified.
  1. Checkout the Service Catalog Request workflow.
  2. Select a schedule to use on the workflow.
  3. Add a Timer activity before the condition activity. Give it a duration of 10 hours and set the Schedule based on field to This workflow's schedule.
  4. Order a catalog item to start this workflow.
    • Note that the scheduled job will have a Next Action of 10 hours in the future and does not take the schedule into account.
PRB561566 Zing Text Search There was a potential indexing error in the System Error Log if a record was inserted and then updated in the same second.
  1. Add an after insert business rule for Incident that inserts data in the short description and performs and update.
    • This will insert and update in the same second.
    • The update may get processed first resulting in duplicated entries.
PRB563087 Zing Text Search Possible text indexing error was displayed in the log if a word could not be indexed for any reason.
  1. Perform a global search on an incident number without using the 3 letter prefix (such as PRB).
  2. Open the ticket.
    • A severe error occurs.
PRB563064 Zing Text Search Global Text Search did not produce results for a table that had a size class value larger than 5000 in the table dictionary entry and there was a condition in the tables Search Table[ts_table].
