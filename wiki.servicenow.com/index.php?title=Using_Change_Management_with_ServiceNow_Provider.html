Using Change Management with ServiceNow Provider

1 Overview

By default, the ServiceNow system only allows changes to ServiceNow provider configurations as part of a controlled change management process. With change management, users cannot directly edit ServiceNow records such as configuration templates or the relationships between CIs and configuration templates. For all changes to ServiceNow resources, the requesting user's manager must approve the request. If the user record does not have a value in the Manager field, the request is approved automatically.

When a user requests a change, each change management approver and the requesting user receive a notification. Additionally, the Assigned to field on the change request is populated with the requesting user. This user receives a notification when the request is approved or rejected.

2 Modifying ServiceNow Provider Nodes

Change management prevents ServiceNow provider users from directly editing the Configuration automation or Configuration Template fields on the CI form. Additionally, the Create New Configuration Change related link appears on the CI form for ServiceNow provider node CIs. To modify a ServiceNow provider node with change management enabled:

  1. Navigate to Configuration > Servers > Linux.
  2. Select a Linux CI record that is assigned to the logged-in user.
  3. Click the Configuration Automation related list to view the current configuration automation settings.
  4. Click the Create New Configuration Change related link.
    The Create New Configuration Change related link appears for any CI with a Fully Qualified Domain Name, even if the CI does not have a provider set. The expectation is that discovery onboarding will populate the provider automatically. You can use Create New Configuration Change to assign a provider, including ServiceNow.
    The Configuration Automation Change form appears.
    If the CI belongs to an existing node group, a warning message appears listing the node groups that contain the CI. Click OK to proceed.
  5. Edit the fields as described in the table.
  6. On the Requested Configurations list, click New and click OK on the pop-up window.
    Initiating a change request for modifying a server configuration

  7. Select a configuration template and give it a Configuration name (for example, the name used for the configuration template) and the Order to run (lowest first). The order is automatically incremented, so you can create the requested configurations in the order that you want without manually modifying this field. For ServiceNow provider only: You can request more than one configuration.
  8. Save the form.
    Attributes are populated. Make any required changes and then click Update to return to the configuration change.
    Initiating a change request for modifying a server configuration

  9. The Requested Configurations list is populated with values. Click the Request Change related link and then click OK on the pop-up window.
    If the CI belongs to an existing node group, a warning message appears listing the node groups that contain the CI. Click OK to proceed.
    The system generates a change request and opens the page. The Description field shows the differences between the currently assigned configuration and the requested configuration.
    If there is an active change associated to that CI, the Cancel Change related link replaces Create New Configuration Change on the CI form. Only one active change request per CI is allowed at any time.
    After the requesting user's Manager approves the request, the Proceed with Change and Cancel Change related links appear. Cancel Change does not appear if the change request is for a node group.
    Initiating a change request for modifying a server configuration

  10. Navigate to the relevant CI record and click the Proceed with Change related link. The link is available only when the change has been approved.
    While the changes are being pushed, you can view the result by clicking the Configuration Record and checking the Result column on the Requested Configuration list.
    When the configuration process is complete, the user receives email that indicates success or failure of the request. The requested configurations appear in the Applied Configurations list on the CI.
    Requesting a change to a node

2.1 Fields on the Configuration Automation Change form

For ServiceNow, only the Provider is editable. All other fields are populated by the system.

Field Description
Phase The phase of the configuration automation change: Draft, Requested, Approved, Rejected, In Progress, Completed, Canceled, or Error. Always starts as Draft.
Configuration Item The CI affected by the change. Populated based on the CI page that you were on
Fully qualified domain name The fully qualified domain name that the ServiceNow provider uses to access this CI. Populated by Discovery.
Provider The configuration automation provider managing this CI.
Change request The change request number. You can view a Diff message in the Description field that shows the differences between the requested and applied configurations. Starts blank and is controlled by the Change Manager.
Requested CI Node attributes Node attributes included in this change request. This should be Requested Configuration and the page is where the attributes now reside. You can request multiple configuration templates for the ServiceNow provider. (You can request only one configuration template for Chef and Puppet).

3 Modifying Configuration Templates

The Checkout Draft related link on the ServiceNow Configuration Template form creates an editable draft copy of the record. Before a user can commit changes to a configuration template, the user's manager must approve the change request.

When making a change to a node with one or more associated configuration workflows and attributes, ServiceNow duplicates the related records, such as attributes, and associates the duplicates with the new configuration template draft record.

To update a configuration template:

  1. Navigate to Configuration Automation > Configuration Templates.
  2. Select a published configuration template.
  3. Click Checkout Draft.
    This creates an editable copy of the current record as a draft and opens the form for that record.
  4. Edit the draft record, as appropriate for the change.
  5. Click Publish.
    • This related link creates a change request record. A draft can have only one active change request at a time.
    • After the requesting user's Manager approves the request, the Proceed with Change related link appears on the ServiceNow Configuration Template Draft form.

After the user's manager approves the request:

  1. Navigate to the relevant configuration template draft record.
  2. Click Proceed with Change to push the changes to the original configuration template record and delete the draft record.
A configuration template draft that replaces a configuration template. Clicking Publish creates a change request.


4 Requesting Changes Using the Service Catalog

A user can request a change to a ServiceNow node through the service catalog. Changes to nodes through the service catalog require the same change requests and approvals as changes made through the CI form.

  1. Navigate to Self-Service > Service Catalog.
  2. Click Software.
  3. Click Modify computer configuration.
  4. Select a Computer to modify.
    Only computers Assigned to the requesting user can be selected.
    Based on the selected computer, the Configuration automation provider and Management server are updated. The values are used to filter which node definitions can be applied.
  5. Select the Node definition to apply to the selected computer.
  6. Click Order Now.
ServiceNow Service Catalog


When the request is approved, a Proceed with change related link is displayed on the computer's CI form.

5 Disabling Change Management

Disabling change management allows ServiceNow users to make and save changed on the ServiceNow configuration template form and the Configuration Automation section of the CI form for ServiceNow nodes.

An administrator can disable change management with ServiceNow:

  1. Navigate to Configuration Automation > Properties.
  2. Click Enable change management to control any updates to configuration templates (or node definitions) and Enable change management to control any updates to CI to enable or disable change management.
