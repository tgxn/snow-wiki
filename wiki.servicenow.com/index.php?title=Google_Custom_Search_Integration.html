Google Custom Search Integration

1 Overview

Custom Search Integration enables you to add Google Search to your instance and customize the search engine for your users. Activating Custom Search Integration replaces the default search, providing Google Search functionality. You can configure your Google Search engine to include or exclude sites in your search results. For more information on the Google Search Engine, see Google Site Search.

2 Menus and Modules

Under the Custom Search Integration menu:

  • Pages: defines the search pages available on the instance and their attributes.
  • Search History: provides a search history for all searches on the instance.
  • Robots.txt Definitions: definitions for the robots.txt file for the instance
  • Script Includes: the script includes applicable to custom search on the instance.
  • Properties: properties that Google search administrators can control, including the Google Custom search key, search name, and whether the search results open in a new page.

3 Setting Up Google Enterprise Search

To configure Google search on your ServiceNow instance, you must first go to the Google site and obtain a Search Engine ID and set up the Google's Enterprise Search and then register the ID with your instance. To set up Google Enterprise Search:

  1. Go to Google Site Search and sign up for Google Enterprise Search. You must sign up for a pay version of Google Search. The free version is not supported. Your organization must be compliant with the Google Enterprise Search licensing terms.
  2. Set up your Google Enterprise Search by going to the Custom Search page while logged into your Enterprise Account. In the Custom Search definition you can define the sites to search and the sites to exclude. Click Search engine ID to retrieve your Google Custom Search Key, which you will need to configure the search on your ServiceNow instance.
  3. Add the URLs you want your users to be able to search. To add a website, click Add under Sites to search.
  4. To exclude sites, click Advanced.
    Setting up Google Enterprise Search

  5. In your ServiceNow instance, go to Custom Search Integration > Properties, enter the Google Custom Search key that you created on the Google website, and click Save.
    Configuring you Google search engine ID

4 Defining Search Pages

Users can access the custom search either by navigating to https://<your_instance_name>/gc_search_widget.do to directly access the Google search page, or administrators can define custom search pages for an instance. To define a custom search page:

  1. Navigate to System UI > UI Pages, create a custom UI page, and embed the gc_search_widget UI page in the new search page.
  2. Navigate to Custom Search Integration > Pages.
  3. Click New.
  4. Fill in the fields, as appropriate (see table).
  5. Click Submit.
Field Description
URL The URL of the search page.
Title The title to display at the top of the search page.
URL Domain The domain of the search page.
Weight The weight assigned to the page.
Average user weighting The average user weighting for the page.
User rating count The number of ratings assigned to the page by users.
Exclude The sites to be excluded from searches on this page.
Hit count The total number of hits for this page.

5 Configuring the Custom Search

A user with admin privileges can configure search properties and prevent robots from searching your instance.

5.1 Configuring Search Properties

The custom search properties allow you to specify the Google custom search key, the name of the page that appears on the search form, and set whether to open search results in a new page. To configure the search properties:

  1. Navigate to 'Custom Search Integration > Properties.
  2. Edit the property (see table).
  3. Click Save.
Property
Google Custom Search key [gc_search.key] The Google custom search key from the Google site.
  • Type: String
  • Default value: None
Name of your search to appear on form [gc_search.name]
  • Type: String
  • Default value: None
Determines whether search results open in a new page [gc_search.new_page] Determines whether search results open in a new page.
  • Type: boolean
  • Default value: Yes

5.2 Blocking Robots From Your Site

Your ServiceNow instance is configured by default to disable robots from crawling your site. However, if you choose to use the Google Search, your site will be crawled by Google unless you specifically configure it not to do so.

The Robots.txt Definition module allows a user with the administrator role to define the contents of the robots.txt file associated with the instance. The syntax used is the same as for a standard robots.txt file. For more information on how to format the robots.txt file, see Robots.txt Specifications.

To define a robots.txt file:

  1. Go to Custom Search Integration > Robots.txt Definitions.
    The currently defined robots files are displayed.
    Current robots.txt files

  2. Click New.
  3. Enter the contents of the robots.txt file in the Text field.
    If you want this file to be active, select the Active field. There can only be one robots.txt file active at any time. Setting a file to active automatically sets all other files to inactive. Web crawlers will recognize the contents of the active robots.txt file and honor the robots exclusion protocol.
    Defining a robots.txt file

  4. Click Submit.

5.3 Retaining Only the Web Crawling Blocker

If only want to configure web crawlers using the robots.txt and not deploy Google search on your instance, you can request the Remove Google Custom Search update set from ServiceNow Support. This update set removes the Google Search functionality but leaves the web crawler configuration capability.

6 Viewing the Search History

The search history shows all of the searches made on the instance in chronological order. To view the search history, go to Custom Search Integration > Search History. To delete an entry, click on the query name and click Delete.

Field Description
Query The text of the search query.
User The user who executed the search.
Search history

7 Activating the Plugin

This integration requires activation of the Google Custom Search Integration plugin.

Click the plus to expand instructions for requesting a plugin.
  1. Navigate to [HI].
  2. Click Service Catalog.
  3. Click Request Plugin Activation.
    • [Required] In Target Instance, select the instance on which to activate the plugin.
    • [Required] In Plugin Name, enter the name of the plugin to activate.
    • [Optional] In Date and time would you like the plugin to be enabled?, specify a date and time at least 12 hours in the future. Leave this field empty if you want the plugin activated as soon as possible.
    Note: Plugins are generally activated during business hours in the Pacific time zone, but can be scheduled for a different time with advance notice.
    • [Optional] In Reason/Comments, add any information that would be helpful for the ServiceNow technical support engineer activating the plugin.
  4. Click Submit.
