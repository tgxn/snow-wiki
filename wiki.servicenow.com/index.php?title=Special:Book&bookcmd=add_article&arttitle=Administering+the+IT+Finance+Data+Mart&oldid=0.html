1 Overview

The Data Mart application stores the information that the Financial Management application uses to allocate expenses to specific accounts and segments in the general ledger. Users with the datamart_admin role can access and modify records in the Data Mart application. Users with the financial_mgmt_admin, financial_analyst, and financial_mgmt_user roles have read access to records in the data mart application.

The Data Mart application is activated with the Financial Management application. For information on what items are installed with the Data Mart application, see Installed with Financial Management.

2 Data Mart Structure

The data mart stores dimension tables and dimension definitions.

Note: In addition, the data mart also stores facts and fact definitions, which are not used in the Financial Management application.


2.1 Dimension Tables

A dimension table holds the records, like the specific projects, business units, and so on, that the application allocates expenses to. The dimension table is a copy of a transactional table, which is a standard table in ServiceNow like the Project [pm_project] table or Business Unit [business_unit] table. When the data mart copies the records from the transactional table to the dimension table, only the transactional table's display values are copied.

For example, by default the application provides the Project dimension table, which is based on the Project transaction table. The records in the Project dimension table copy the values for the Name column from the dimension table. The other columns, such as Number and Short description, are not copied. This is because the Name column is the default display value of the Project table.

Warning: Do not delete dimension tables. Because allocations refer to dimension tables, the allocation engine encounters errors if the dimension table is missing.

2.1.1 Dimension Tables and Segments

The Financial Management application creates a segment based on a dimension table. A segment is the financial term for a dimension. The specific records in the dimension table are called accounts in financial terminology.

The Financial Management application allocates expenses to dimensioned records rather than records in the transactional tables because transactional records change frequently, and might even be deleted. When the data mart administrator creates a dimension table, the records from the transactional table are copied once, and only the data mart administrator can make changes. If, for example, a specific project name changes, the data mart administrator can repopulate the dimension table and the dimensioned record of the project is updated. When the financial manager or financial analyst allocates expenses, they see the project names listed in the dimension table.

Data mart administrators create new dimensions when a new segment must be added to the IT chart of accounts.

2.2 Dimension Definition

A dimension definition is a record that has a reference to the dimension table and to the transactional table, and also has a a condition builder. You can use the a condition builder to restrict the records that are copied from the transactional table to the dimension table.

4 Setting up Dimension Definitions

  1. Navigate to IT Data Mart > Dimensions. The list of dimension definitions appears.
  2. Click New to create a new definition.
  3. Fill out the fields on the form (see table).
  4. Right-click the form header and select Save.
    Additional fields and related links appear after you save the record.
Creating a new dimension for applications

Field Description
Name The name for the dimension definition.
Table The transactional table on which the dimension table is based. Select a table from the system.
Merge dimensions Check box to copy the records from the Table to the Dimension table.
  • If you select this check box, a new dimension table is not created.
  • If you do not select this check box, a new dimension table is created based on the selected Table value.
Dimension table The dimension table into which the records from the transactional table are merged. This field is available when Merge dimensions is selected.
Dimension condition The condition to limit the records that are included the dimension table. The number of records that matches the filter appears in the link above the condition builder. Click the link to view a list of the records.
User created Check box that is selected if the dimension was created by a user. The check box is cleared for default dimensions.
Related Links
Populate dimension table Populates the dimension table with values from the transactional table.
Show dimension table Displays the records added to the dimension table.

5 Merging Dimensions

You can merge the records from two transactional tables into an existing dimension table. Then you can perform allocations and generate reports based on the segment which corresponds to the dimension table.

For example, you might have records in both a custom project table named My Company Projects [u_my_company_projects] and the Project application's Project [pm_project] table that you want to add to the Project dimension table. This allows you to see all the records in the Project segment without switching between two segments to see all your projects. To accomplish this, you create a dimension definition for My Company Projects and select the Merge option and the Project dimension table. When you create the dimension definition, the records from the My Company Project table are populated into the My Company Projects dimension. You then go to the Project dimension definition and populate that dimension table again. The records from the My Company Project table are then populated to the Project dimension table as well.

To merge dimensions:

  1. Create a new dimension definition.
  2. Select the Merge dimensions check box.
  3. Select the existing Dimension table, into which the records from the transactional table can be merged.
  4. Right-click the form and select Save.
  5. Navigate to the dimension definition that you are merging records to.
  6. Click Populate dimension table to populate the table.
    The list of records in the dimension table appears.
  7. You can group the list by the Original table column to see the records that came from either table.

6 Populating Dimension Tables

If you create a table that is used as a segment, or if you activate another application with tables that are used as segments, you must populate the corresponding dimension table if you want to allocate expenses to records in the segment. You can also repopulate a dimension table if the display name of the table used by the segment changes. If you do not populate dimension tables manually, the Populate IT Data Mart dimension tables scheduled job automatically populates dimension tables once every 24 hours.

Note: Users with the datamart_admin role can populate dimension tables even if they do not have read access to records in the transaction table.


To populate dimension tables:

  1. Navigate to IT Data Mart > Dimensions.
  2. Select the dimension definition for the table you want to populate.
  3. Use the condition builder under Dimension condition to filter the records from the transactional table.
  4. Under Related Links, click Populate dimension table.
    The application creates a record in the dimension table for each record in the transactional table.

7 Viewing Dimension Runs

Each time you populate a dimension table, the application creates a record in the Dimension Last Run [dm_dimension_last_run] table to track how many records were created or updated in the dimension table.

  1. To view dimension runs, navigate to IT Data Mart > Last Run.
  2. Click the dimension table name in the Table column.
    The Dimension Last Run form opens (see table for field descriptions).
A dimension run for the Project dimension.

Field Description
Added records Number of records added to the dimension table.
Real table Transactional table from which the records are taken.
Table Dimension table into which the data is populated.
Last run Date and time the application populated the dimension table.

8 Viewing Records in the Dimension Table

You can view records in dimension tables. Normally, you do not need to make changes to the field values.

  1. Navigate to IT Data Mart > Dimensions.
  2. Select the dimension definition for the table you want to populate.
  3. Under Related Links, click Show dimension table (see table for field descriptions).
Form fields for a dimension.

Field Description
Name The value of the transactional field in the dimension table definition.
Current record If the dimension record is the most up to date.
Original ID A reference to the record in the original table.
Original table The transactional table that the dimension table is based on. The tables in this choice list are transactional tables that are already dimensioned.
Valid until The date that this dimension expires. A dimension expires when a more current version of the record is transferred from the original table.
