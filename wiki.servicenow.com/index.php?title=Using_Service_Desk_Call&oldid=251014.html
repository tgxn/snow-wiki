1 Overview

ITIL users, users with the itil role, can quickly capture basic information within a call record while in contact with the customer. After the information has been gathered, they can decide how to classify that call record, for example as an incident.

ITIL users can use service desk call to:

  • Create a new call record.
  • Transfer a call record to an incident, problem, change, or service request.
  • View call record information.

2 Creating a New Call

  1. Navigate to Service Desk > Calls > New Call.
  2. Fill in the fields, as appropriate (see table).
  3. Click Submit.
    If the call type is Incident, Problem, Change, or Request, this transfers the call, creating another record of the appropriate type.
Field Description
Number An auto-generated identifying number for the call record.
Caller The person who contacted the service desk. This is mandatory if the call type is Incident, Problem, Change, or Request.
Company The company where the call originated.
Call type A field that specifies the status of the call.
  • Select Incident, Problem, Change or Request to transfer the call.
  • Select Hang Up, Wrong Number, Status Call, or General Inquiry to save the call record without transferring.
Opened The date and time when the call record was opened.
Opened by The ITIL user who communicated with the customer.
Contact type The type of communication used to create the call: email, phone, walk-in or self-service.
Short description A short description of the call.
Description Additional information about the call that might be helpful for others to know. This field is visible to customer.

2.1 Listing Tasks by Same Caller or Company

Submitting a call creates two related lists: Tasks by Same Caller and Tasks by Same Company:

These list all active incidents, problems, changes and service catalog requests for that caller or company. This can be useful to check for duplicate calls.

The Tasks by Same Caller list shows all active records where the value in the Caller field on the Call form matches the value in:

  • The Caller field on the Incident form.
  • The Opened by field on the Problem form.
  • The Requested by field on the Change Request form.
  • The Requested for field on the service catalog Request form.

The Tasks by Same Company list shows all active records where the value in the Company field on the Call form matches the value in:

  • The company of the caller on the Incident form.
  • The company of the user that opened the problem on the Problem form.
  • The company of the user that requested the change on the Change form.
  • The company of the user the request was made for on the service catalog Request form.

3 Transferring a Call

Use the details gathered during the customer contact to decide what type of record should be created, if any. Saving the call with a call type of Incident, Problem, Change, or Request saves the call record and also creates a new record of the type selected. This new record is populated with information taken from the call record.

For example, the service desk receives a phone call from a customer reporting email issues with his laptop. As soon as the phone call starts, an ITIL user working at the service desk creates a call record and enters basic information into that record during the course of the phone call. At the end of the phone call, the ITIL user decides this call should be logged as an incident, sets the Call type to Incident, and saves the call record. This creates a new incident record with details taken from the original call record. The new incident can then be processed with standard incident management processes.

After a call is transferred, the number of the new record appears in the Transferred to field on the Call form and the Calls list. A link to the new record appears above the Calls list, providing easy access.

All fields on the Call form then become read-only.

3.1 Transferring a Call to an Incident, Problem or Change

  1. In the Call form, set Call type to Incident, Problem or Change.
  2. Click Submit to transfer the call.

A record of the appropriate type is created, with the following information copied from the call:

  • Short description
  • Description
  • Company
  • Contact Type
  • Opened By
  • Caller
  • Location (Incident only, populated with the caller's location)
  • Requested by (Change only, populated with the caller's name)

3.2 Transferring a Call to a Service Catalog Request

You can transfer a call to a service catalog request and enter the relevant details for the requested catalog item as part of a single process.

  1. In the Call form, set Call Type to Request.
    A Request item field appears on the form.
  2. Select the catalog item to request.
    Note: Content items, record producers and order guides are not supported.
  3. Click Submit.
    The order form for that catalog item appears.
  4. Select any other options for that catalog item, as required.
  5. Order the item.
    The service catalog request is created, with the Requested For and Description fields in the Call form copied into the Description field. The Description field content is also placed in the Special instructions field on the catalog Request form (you may need to configure the form layout to see this field).

4 Viewing Calls

You can view call records in several modules.

  • Service Desk > My Calls lists all calls that you created.
  • Service Desk > My Open Calls lists all calls you created that have transferred records, such as incidents, that are still open.
  • Service Desk > Calls > All Open Calls lists all calls created in the last month that have transferred records that are still open.

The Time Spent column shows the time between creating and saving the call record.
