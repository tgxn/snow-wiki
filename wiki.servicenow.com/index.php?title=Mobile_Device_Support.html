1 Overview

ServiceNow supports a subset of the standard functionality on smartphones and similar devices, allowing mobile users to remotely access their ServiceNow instances to perform common tasks.

ServiceNow supports two smartphone interfaces:

  • The smartphone interface, available starting with the Dublin release.
  • The legacy mobile UI.

The smartphone interface is active by default on new instances starting with the Dublin release. For existing instances that are upgraded to Dublin or later, administrators can activate the Smartphone Interface.

2 Supported Devices

The following devices, operating systems, and browsers are supported by the smartphone interface.

  • Apple® iPhone® or iPod®: Safari is supported for the following devices.
    • Fuji or later: All models of iPhone or iPod running iOS7 and above.
    • Eureka or earlier: All models of iPhone or iPod running iOS6 and above.
  • Android™: Android version 4.0/Ice Cream Sandwich and above. Use the latest available version of Chrome to access the ServiceNow smartphone interface on Android devices. Native browsers and older versions of Chrome support major ServiceNow functionality, but have some known issues.
  • BlackBerry®: All Blackberry devices running Blackberry 10 and above.

For supported tablet devices, see Tablet Support.

Note: The smartphone interface should not be used on desktop browsers except for testing purposes.


3 Features of the Smartphone Interface

  • A property called glide.ui.m_agents defines which devices are treated as mobile. By default, this list is: iPod, BlackBerry, Android, and iPhone.
  • UI policies, UI actions, client scripts, and related lists are supported.
  • The device home screen can be configured to display tags, favorites, and recently-accessed lists and documents.
  • Online help is available in the base system. It can be modified and you can create new help screens. The help can also be localized for multiple language support.
  • Journal streams allow you to sequentially view comments and work notes associated with a group of records, such as incidents.

  • Device home screen

  • Online help

  • Journal stream

3.1 Unsupported Features

The smartphone interface supports a limited subset of functionality that enables users to perform common tasks. The following capabilities are not currently supported in mobile browsers, though they may work to varying degrees.

  • Switching to the standard browser interface from the smartphone interface
  • HTML fields
  • CODE tags to render HTML in Journal Fields
  • Live Feed
  • Collaboration
  • Chat
  • Field styles
  • Visual task boards
  • Timeline visualizations
  • Assessments, surveys, or legacy surveys
  • Automatic creation of time cards
  • Embedded lists
  • List UI actions
  • Interceptors
  • Mobile service catalog features
    • order guides
    • wizards
    • content items
  • These variable types:
    • Macro With Label
    • UI Page
    • List Collector
    • HTML
    • Macro
    • Label
    • Break
  • Data lookup rules
  • Advanced reference qualifiers
  • Script include calls in reference qualifiers
  • Custom auto-complete scripts

3.2 Using the Smartphone Interface on a Tablet

By default, the system provides tablets with their own mobile web UI. If you want a tablet to run the smartphone interface instead of the tablet interface, you must make the following changes.

  1. Remove the device from the list of tablet mobile agents in the glide.ui.tablets_agents property. For example, remove the ipad value.
  2. Add the device to the list of smartphone mobile agents in the glide.ui.m_agents property. For example, add the ipad value.

4 Menus and Modules

Use these modules to access common features for configuring smartphone interface applications. These modules are available only to administrators.

Under the System Mobile UI menu:
  • Navigator Apps: Add or remove applications from the mobile application navigator. See Defining Smartphone Application Menus and Modules.
  • Mobile Help: Maintain existing online help, create new online help screens, and even localize the online help into multiple languages. See Modifying and Creating Online Help.
  • Table Titles: Define unique titles for all tables viewed in the smartphone interface. See Defining Table Titles.
  • UI Actions - Mobile: Create UI actions for the smartphone interface. See Adding UI Actions on the Smartphone Interface.
  • Recent Selections: View the recent selections made from reference fields.
  • Home Page Sections: Define and maintain the sections that appear on the home page by default. See Defining Default Home Page Favorites.
  • Home Page Favorites: Define and maintain the default favorites. See Defining Default Home Page Favorites.

5 Activating the Plugin

The smartphone interface is active by default on new instances starting with the Dublin release. For instances upgrading from a previous version, administrators must activate the Mobile UI plugin to access the feature.

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.

6 Legacy Mobile UI

This section describes the devices, browsers, and features supported in versions prior to the Dublin Release. For more information, see Configuring the Legacy Mobile UI.

6.1 Supported Devices and Browsers

The following devices, operating systems, and browsers are supported by the legacy mobile UI in versions prior to the Dublin release.

Apple® iPhone®: All models of iPhone.

Apple® iPod®: All models of iPod touch connected by WiFi.

Android™: All versions of Google Android.

BlackBerry®: All BlackBerry devices running version 4.6.0 and above.

Opera™ Mini: All versions of Opera Mini.

Windows Mobile: Version 6.5.x.

Windows Phone: Version 7.x.

6.2 Features of the Legacy Mobile UI

  • Phone number type fields can be dialed directly.
  • A property called glide.ui.mobile_agents defines which devices are treated as mobile. By default this list is: iPod, Windows CE, BlackBerry, Android, iPhone, Opera Mini.
  • Dependent choice lists for mobile clients. For example, if you place both a category and subcategory on your mobile incident form, and then change the category field, the subcategory field will automatically update its set of choices.
  • iPhone multi-line text inputs expand when edited.
  • HTML field content can be read on mobile devices. This is especially useful for viewing the knowledge base on a mobile device.
  • Tags and splits on mobile lists
  • Reference fields that have a value in them are rendered as links. Users can click the field to view the referenced record.
  • Advanced mobile devices—iPhone, BlackBerry OS 4.6 or higher, and Android phones—provide a calendar widget to select values for date/time fields, similar to the browser UI.

  • Click-through reference fields

  • Mobile Date/Time Picker

  • Mobile Date Picker

The legacy mobile UI supports a limited subset of ServiceNow functionality that enables users to perform common tasks. The following capabilities are not currently supported in mobile browsers, though they may work to varying degrees.

  • Client scripts and UI policy
  • Mandatory fields
  • BSM Map or other graphical displays
  • List UI actions (form UI actions are supported), but client-side UI actions are not supported
  • Right-click functionality (for example, context menus to customize forms or lists)
  • Reporting
  • Service catalog and record producers
  • Knowledge base editing
  • CMDB relationships formatter
  • List (glide_list) field types, such as watch lists
  • Reference field autocomplete

7 Enhancements

7.1 Fuji

  • Users can view and open form attachments in the smartphone interface.

7.2 Eureka

  • Map pages are supported in the smartphone interface.
  • Related lists are supported in the smartphone interface.
  • UI actions for redirecting to a URL are supported in the smartphone interface.

7.3 Dublin

  • For existing instances that upgrade to Dublin, administrators can activate the Mobile UI plugin, which disables all application menus or modules that are defined in the legacy mobile UI for mobile devices.
  • For instances that use the smartphone interface, but have not activated the Mobile UI plugin, administrators define application menus and modules for mobile devices in a separate table from the menus and modules for the standard browser interface.
  • For instances that do not use the smartphone interface, administrators can define application menus and modules that are available both through the standard browser interface and on mobile devices.
  • Administrators can modify the online help, available in the base system, and create new help screens. The help screens can also be localized to support multiple languages.
  • Administrators can add UI actions for use on mobile devices.
  • UI policies can be defined for the desktop UI, smartphone interface, or both.
  • Users can customize the items displayed on their own device home screen. The screen can include any combination of favorites, tags, and recently-accessed lists and documents.
  • Users can view journal streams, which sequentially display comments and work notes from a group of records, such as incidents.
  • Users can edit the knowledge base from mobile devices.
  • Users can access the service catalog from mobile devices.
  • The UI Type field has been added to the Client Script form to accommodate client scripts defined for the desktop UI, mobile UI, or both UIs.
