1 Overview

Zing is the text indexing and search engine that performs all searches in ServiceNow. Administrators and users with the ts_admin role can configure the following options for Zing:

  • Control match relevance by field.
  • Configure global stop words and stop words for each index.
  • Configure text indexes for tables.
  • Disable the junk filter for tables.
  • Configure properties for optimal performance.
  • Control Zing security.
  • Debug Zing.
  • Update type-ahead suggestions.
  • Configure "Did you mean?" suggestions.
  • Enable punctuation handling.
  • Review components that are installed by text search.

2 Controlling Match Relevance by Field

To improve search results, the search engine assigns to each potential match a numeric score that represents its relevancy to the query. Administrators can control the relative importance of a match for each field in a table with the ts_weight attribute. The default weight of a field is 1.

To set the relative weight of a field:

  1. Open the system dictionary entry for the field.
  2. In the Attributes field, enter ts_weight=<value>, where <value> is the relative weight of the field.
    For example, if the Title field has a ts_weight=50 and the Description field has a ts_weight=10, then a match in the title is weighted 5 times more relevant than a match in the description and 50 times more relevant than a field without the ts_weight attribute.
  3. Click Update.
 Note: To view the complete scoring system, see Document scoring.


The following fields have elevated scoring weights by default:

  • kb_knowledge.number = 50
  • kb_knowledge.short_description = 10
  • kb_knowledge.meta = 10
  • task.number = 50
  • task.short_description = 10

3 Converting Text Indexes to Improve Performance

To improve performance, a new text index structure is available (Calgary release). It is configured automatically for instances created after the Calgary release. For upgraded instances, ServiceNow automatically starts the conversion.

  • All text indexes are regenerated. This process may take a while to complete, and you may notice performance degradation or incomplete search results while the indexes are being regenerated.
  • Punctuation handling is automatically enabled, if necessary. This process creates additional default global stop words (for example, I've and I'm).
  • When the upgrade completes, the text_event.upgrade_calgary event is written to the system log.

4 Configuring Stop Words

Stop words are common words that are not indexed because they are not meaningful in search results. Articles, conjunctions, personal pronouns, and prepositions are examples of stop words that are not used in keyword searches.

4.1 Global Stop Words

To configure the stop words for all indexed tables:

  1. Navigate to System Definition > Text Index Stop Words.
  2. Add or remove stop words from the list.
  3. If a message appears at the top of the list, contact Technical Support to regenerate all indexes.
    You must regenerate indexes whenever words may be missing from an index. For example, if you delete, inactivate, or change an active global stop word, the word may be missing from the index. An after business rule checks these conditions and generates the notification message when index regeneration is necessary.
Notification to regenerate all indexes

4.2 Table-Specific Stop Words

To configure the stop words for a specific table:

  1. Navigate to System Definition > Text Indexes.
  2. Open the text index entry for the table.
  3. Add or remove stop words from the Index Stop Words related list.
    For each word, select the desired Stop mode (Not a Stop Word (inactivates the stop word), Neither Index nor Query, or Index but do not Query).
  4. If a message appears at the top of the form, click Regenerate Text Index and click OK.
    You must regenerate indexes whenever words may be missing from an index. For example, if you delete or change a stop word for which the mode was Neither Index nor Query, the word may be missing from the index. An after business rule checks these conditions and generates the notification message when index regeneration is necessary.
 Note: Text search uses the global list plus the table-specific list of stop words when indexing the table.


Table-specific stop words

4.3 Automatic Stop Words

To configure automatic stop words for a table:

  1. Navigate to System Definition > Text Indexes.
  2. Open the text index entry for the table.
  3. Select the Auto stop check box.
  4. Enter the maximum number of occurrences for a non-stop word in the Auto threshold field.
  5. Click Update.
    When the number of occurrences of a word exceeds the threshold, the word is automatically add as a stop word for the table with the Stop mode of Index but do not Query and a Comment to indicate that the stop word was generated automatically. Automatic stop words are compiled on a nightly basis by the TS Index Stats scheduled job.

5 Configuring Text Indexes for Tables

You can add, rebuild, or remove text indexes against any table to control what is and is not searchable. The search engine only returns results for tables that are indexed. Ensure no indexing job is running before making these changes.

Note:
  • Each option in a choice list is defined with both a label and a value, which can be different. The label appears in the choice list on forms and is indexed for text searches. However, the value is used for scripting and is not found by a text search.
  • A table is not automatically reindexed when data on a referenced table changes. For example, if a record in the knowledge base references a group and the group name changes, searches for the old group name will still yield results and searches for the new group name will not.


5.1 Indexing a Single Table

To add or rebuild a text index for a table:

  1. Navigate to System Definition > Dictionary.
  2. Open the dictionary entry for the table.
  3. Click the Generate Text Index related link and click OK.
    Alternately, you can regenerate a text index from the table record in the list of text indexes. See Table-Specific Stop Words.
 Note: This process may take a while to complete depending on table size and other factors. You may notice performance degradation or incomplete search results while the indexes are being regenerated. To estimate text indexing duration, you can view historical statistics.


Generate a single text index

5.2 Viewing Text Indexing Statistics and Status

To view text indexing statistics and status, navigate to System Definition > Text Indexes.

  • If text indexing is in progress, view the status of each index in the State field. Refresh the list to view updates. Tables are indexed one at a time.
  • To see how long it takes to index a specific table, view the Last indexing duration field for the table entry. Although each process varies based on activity and data, historical data can provide a good estimate.
  • View statistics (such as the number of indexed rows, terms, and queries in the past week) in the list or on the form for a specific table.
Text index status

Text index statistics


5.3 Removing an Index

To remove an index against a table:

  1. Navigate to System Definition > Dictionary.
  2. Open the dictionary entry for the table.
  3. Clear the Text index check box and click Update.
    The system no longer indexes or queries the table for text search results.

5.4 Removing an Index for a Specific Field

To remove an index against a specific field within an indexed table:

  1. Navigate to a form containing the field.
  2. Right-click the field label and select Configure Dictionary (Personalize Dictionary in versions prior to Fuji).
  3. Add no_text_index=true to the Attributes field. Separate multiple attributes with a comma (with no spaces).
  4. Click Update.
    The system no longer indexes or queries the field for text search results.

6 Disabling the Junk Filter

By default, Zing does not index or search for 2-digit numbers and single character words (unless they are Chinese or Japanese characters). You can turn off this filter for a table. For example, to have the ability to search for error 92 in the knowledge base, turn off junk filtering for kb_knowledge.

  1. Navigate to System Definition > Dictionary.
  2. Open the dictionary entry for the table.
  3. In the Attributes field, enter text_index_filter_junk=false.
  4. Right-click the header and select Save.
  5. Click the Generate Text Index related link and click OK.
 Note:
  • To have the desired effect, you may also need to reconfigure the automatic stop words. For example, if a specific 2-digit number appears in all of your problem numbers (92-0001, 92-0002, and so on), the automatic stop word threshold is quickly exceeded and the word is no longer found in searches. To find that 2-digit number in the problem table, you must disable both junk filtering and automatic stop words.
  • Disabling the junk filter results in a larger table index. For optimal performance, do not apply this attribute unless it is required.


7 Properties

To configure Zing properties, navigate to System Properties > Text Search.

Name Description Details
Text Search Query Properties
glide.ir.query_method Query method for global text search. Run a query for each table, for each base class (parent table), or against a single index regardless of table (requires text index regeneration), or let GlideRecord handle the query (simple). With the Zing text search engine, the preferred value is "simple". To learn more, see Global Text Search Query Method.
glide.ts.query.parallelism Number of simultaneous processes (1 to 10) used to perform search tasks Default value is 5. Higher values do not provide additional benefits for mysql databases.
glide.ts.max_duration Length of time (in seconds, 10 to 600) that a search is allowed to run before it is canceled Default value is 60.
glide.ts.widget.top_search_tables Comma-separated list of table names which will have home page top searches widgets available for them Default value is 4.
Text Search Index Properties
glide.ts.index.parallelism Number of simultaneous processes (1 to 16) used to perform indexing tasks Default value is 8.
glide.ts.optimize.parallelism Number of simultaneous processes (1 to 16) used to perform index optimization tasks Default value is 2.
Text Search Database Properties
glide.ts.private_pool Use a separate database connection pool for text queries Default value is true.
glide.ts.oracle_hint Use query hints for text searches against Oracle databases Default value is true.
Did You Mean Properties
glide.ts.dym.enable_spell_correct Suggest alternate search spellings for knowledge or global search Default value is false.
glide.ts.dym.enable_chain_suggest Suggest related searches for knowledge or global search Default value is false.


To configure additional Zing properties, open the sys_properties table. To learn more, see Adding a property.

Name Description Details
glide.ts.max_wildcard_expansion Maximum number of matches for a wildcard term that are allowed as unambiguous Default value is 500. Type is integer. If a wildcard matches more words than this threshold, a message appears asking the user to refine the search.

8 Security

8.1 User Roles

Zing installs the ts_admin user role. Administrators and users with the ts_admin role can configure Zing text search settings.

8.2 Zing Table Access Rules

Administrators and users with the ts_admin role have the following access to Zing system tables:

  • ts_index_name: view the tables for which the data is indexed.
  • ts_index_stop and ts_stop: configure these tables for stop words.

Extending or modifying other Zing system tables is not recommended.

8.3 Search Results Automatically Filtered by ACL Rules

Zing automatically filters search results using ACL Rules. Only users who meet the ACL requirements of the search results will actually see them.

For example, suppose you index the System Properties [sys_properties] table. When the ITIL User searches for a term in the system properties table, Zing returns no search results because the ITIL user does not meet the ACL rule requirements.

ITIL User cannot see search results from restricted table

When a system administrator searches for the same property, Zing returns search results from the System Properties tables because the administrator meets the ACL rule requirements.

System Administrator can see search results

9 Debugging Zing

Debugging messages create a log of search processes as they occur. These messages may help administrators configure optimal performance settings.

To view debugging messages, navigate to System Diagnostics > Session Debug > Debug Text Search. The search log messages appear at the bottom of the results page for the remainder of your session.

Debugging messages

To debug text indexing problems, you may want to view text_index events in the system log.

In rare cases, you may need to reset the text search caches after regenerating a text index. If text search fails for known words, diagnose and fix this problem:

  1. Enable text search debugging messages.
  2. In any table with search problems, search for the problem text.
  3. In the debug output, note the numbers used for the Stemmed terms.
  4. Enter ts_word.list in the navigation filter.
  5. Locate the stemmed terms and compare the numbers in the list to the debug output (step 3). If the numbers are different, the UI node's cache is stale.
  6. Navigate to System Definition > Text Indexes.
  7. Click the Reset Text Search Caches related link and click OK.

10 Updating Type-Ahead Suggestions

The knowledge base and global text searches provide suggestions as you type. These type-ahead suggestions are compiled on a nightly basis by a scheduled job. If you need to refresh this list sooner:

  1. Navigate to System Scheduler > Scheduled Jobs.
  2. Open TS Search Stats.
  3. Run the scheduled job.

11 Configuring "Did You Mean?" Suggestions

The knowledge base and global text searches can provide “Did you mean?” suggestions. Suggestions appear if a search does not return any results and an alternate spelling or similar recent search does. This feature is disabled by default.

To enable "Did you mean?" suggestions:

  1. Navigate to System Properties > Text Search.
  2. Select any of the following check boxes:
    • Suggest alternate search spellings for knowledge or global search (the glide.ts.dym.enable_spell_correct property).
    • Suggest related searches for knowledge or global search (the glide.ts.dym.enable_chain_suggest property); uses search chains, which generate suggestions by tracking occurrences of similar searches, in order, over time.
  3. Click Save.

12 Installed Components

12.1 Business Rules

The Text Search Property Change Rationally business rule ensures that valid values are entered for Zing text search properties.

  • Text Index Stop Reminder: warns the user of stop word changes that require the index to be rebuilt (table-specific). The warning is issued if record is deleted that had a stop mode "neither index nor query", if record's stop mode is updated to something else and was "neither index nor query", and if record's word is updated to something else and stop mode is "neither index nor query".
  • Stop Word Reminder: warns the user of stop word changes that require the index to be rebuilt (global). The warning is issued if record is deleted and it was active, if record is inactivated, and if record's word is changed and was active.

12.2 Scheduled Jobs

The following scheduled jobs are added:

  • TS Search Stats: compiles type-ahead suggestions each night. See Updating Type-Ahead Suggestions.
  • TS Index Stats: runs every night; collects statistics and performs maintenance for text search and indexing.
  • text index events process: runs every 30 seconds; collects statistics and performs maintenance for text search and indexing.
  • TS Search Summary: compiles top search statistics each hour. See Top Searches Homepage.
  • TS Chain Summary: compiles search chain statistics each hour.

12.3 UI Actions

The following UI actions are added:

  • Regenerate Text Index: displays the "Regenerate Text Index" link on Text Index forms.
 Warning: Regenerating all text indexes can cause severe performance degradation. Contact Technical Support for assistance with this function.

12.4 Homepages

This homepage displays the most popular searches over the past hour, day, week, or month. To learn more, see Top Searches Homepage.

13 Enhancements

13.1 Eureka

  • Zing search can index certain elements of HTML tags.

13.2 Dublin

  • Search results with exact word and phrase matches have higher relative importance than stemmed matches.

13.3 Calgary

  • To improve performance, the text index structure is revised. For upgraded instances, ServiceNow automatically converts existing text indexes to the new structure. New instances use the new index structure by default.
  • When text indexes are converted to the new index structure, the text index reaper is no longer necessary to maintain indexes.
  • To prevent performance and data corruption problems, you can no longer regenerate all text indexes. Administrators can regenerate the index for an individual table if necessary.
