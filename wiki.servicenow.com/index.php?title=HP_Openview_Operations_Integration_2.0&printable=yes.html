1 Overview

The HP Openview Operations integration allows ServiceNow to send and receive event data to an HP OpenView Operations server. By default, the integration transforms OpenView Operations (OVO) trouble tickets into incidents using Web Services to create inbound incidents. It also uses the OpenView opcmsg command to send outbound trouble tickets to HP OpenView.

There are two parts to the integration.

  • HP OVO to ServiceNow
  • ServiceNow to HP OVO

1.1 HP OVO to ServiceNow

  • Set up the HP OVO trouble ticket interface to run the Perl script TroubleTicket.sh (if running OVO on a Windows server, see HP OpenView Operations Integration Windows for the equivalent vb script).
  • TroubleTicket.sh makes a SOAP call to ServiceNow and inserts a Notification input record (imp_notification),
  • The import transform map creates a new incident or correlates on an existing incident and updates it. It also creates a Listener for the incident; when the incident is modified, HP OVO is notified.

1.2 ServiceNow to HP OVO

  • ServiceNow runs the HP OVO opcmsg command line when an incident record is updated or closed.
  • The event triggers the HPOV Updater script object.
  • ServiceNow sends the opcmsg command to a local MID Server.
  • The message uses the correlation_id field to correlate a ServiceNow incident to an HP OVO event ID.

1.3 Workflow

The HP OVO integration uses the following workflow to create incidents in ServiceNow from HP OVO events and update HP OVO events from incident changes.

HP OVO integration workflow

1.4 Parts of the Interface

The Integration - HP OVO application consists of these modules:

  • Properties - This module configures how ServiceNow updates the HP OVO server via MID Server.
  • Notification WS - This module configures the Notification Web Service to create an incident from the incoming trouble ticket.
  • HPOV Updater - This module controls how ServiceNow queues HP OVO update commands to the MID Server.

2 Integration Requirements

Enabling an integration between your ServiceNow instance and an HP OpenView Operations server requires these actions.

  • Set up HP OpenView Operations 8.X for trouble ticketing
    • Provide an HP OVO user account to send events back to the HP OVO server
  • Install Perl 5.8 or later on the HP OVO server
  • Install the following Perl modules on the HP OVO server
    • SOAP::Lite (prerequisites http://soaplite.com/prereqs.html)
    • XML::Writer
    • XML::Writer::String
    • Crypt::SSLeay
    • IO::Socket::SSL
  • Provide HTTPS POST access from the HP OVO server to the ServiceNow instance
  • Provide the MID Server with port level access from the HP OVO server to the ServiceNow instance

3 Integration Setup

These steps enable the transfer of events between an HP OVO server and a ServiceNow instance.

  1. Activate the Integration – HP OVO plugin.
  2. Install the Perl modules on the HP OVO server.
  3. Install a MID Server on a local system that can access the HP OVO server.
  4. Set up the HP OVO trouble ticket interface.
  5. Create a Perl script for the inbound notification.
  6. Set properties for outbound communications to the HP OVO server.

3.1 Step 1. Activating the Plugin

To use this feature, request the plugin (Eureka release and versions prior to Dublin).

Click the plus to expand instructions for requesting a plugin.
  1. Navigate to [HI].
  2. Click Service Catalog.
  3. Click Request Plugin Activation.
    • [Required] In Target Instance, select the instance on which to activate the plugin.
    • [Required] In Plugin Name, enter the name of the plugin to activate.
    • [Optional] In Date and time would you like the plugin to be enabled?, specify a date and time at least 12 hours in the future. Leave this field empty if you want the plugin activated as soon as possible.
    Note: Plugins are generally activated during business hours in the Pacific time zone, but can be scheduled for a different time with advance notice.
    • [Optional] In Reason/Comments, add any information that would be helpful for the ServiceNow technical support engineer activating the plugin.
  4. Click Submit.

Activate the Integration – HP OVO plugin (Dublin release only).

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.

3.2 Step 2. Installing the Perl Modules

To install Perl modules, use the following command line call for each module: 

perl -MCPAN -e 'install <module name>'

For example: 

perl -MCPAN -e 'install SOAP::Lite'

The command installs packages from Comprehensive Perl Archive Network (CPAN). Each command starts an interactive installer. Accept the suggested or default responses.

3.3 Step 3. Installing a MID Server

Install and configure a MID Server on the same network as the HP OVO server.

3.4 Step 4. Setting Up the HP OVO Trouble Ticket Interface

  1. Update the conditions in each HP OVO policy, selecting the check box for each alarm that should generate a trouble ticket.
  2. Navigate to Edit > Server > Trouble ticket in HP OVO. Select the check box for the triggering alarm and specify a script to execute. Modify the script as needed. As an example, see the ttns_mail.sh script in your OpenView installation.

(Optional) You may also create a manual action that executes the Perl script if you want to manually push alarms to the integration.

3.5 Step 5. Creating an Inbound Notification Record with Perl

The TroubleTicket.sh script creates a SOAP message containing information about the OpenView event. ServiceNow transforms the SOAP message into a standard web service import set table and the integration transform map creates an Incident record from the event.

Note: The incoming SOAP message must send a value of HPOV for the Type field. The integration uses this value to create an HPOVUpdater listener process. The HPOVUpdater listener process observes modifications to the associated incident and transmits updates back to HP OVO.


Here is a listing of the TroubleTicket.sh Perl script: 

#!/usr/bin/perl -w

#use SOAP::Lite ( +trace => all, maptype => {} );
use SOAP::Lite;
use XML::Writer;
use XML::Writer::String;


#
# Get parameters passed by OVO notification
#
my $OVMSG_ID              = $ARGV[0];
my $OVMSG_NODE_NAME       = $ARGV[1];
my $OVMSG_NODE_TYPE       = $ARGV[2];
my $OVMSG_DATE_CREATED    = $ARGV[3];
my $OVMSG_TIME_CREATED    = $ARGV[4];
my $OVMSG_DATE_RECEIVED   = $ARGV[5];
my $OVMSG_TIME_RECEIVED   = $ARGV[6];
my $OVMSG_APPLICATION     = $ARGV[7];
my $OVMSG_MSG_GROUP       = $ARGV[8];
my $OVMSG_OBJECT          = $ARGV[9];
my $OVMSG_SEVERITY        = $ARGV[10];
my $OVMSG_OPERATOR_LIST   = $ARGV[11];
my $OVMSG_MSG-TEXT        = $ARGV[12];
my $OVMSG_INSTRUCTION     = $ARGV[13];

hpovo_create();

sub hpovo_create {
  # this is the URL for the SOAP endpoint to enqueue an HP OVO notification
  # onto ServiceNow's imp_notification, to be processed by a map that produces
  # an Incident
  my $soap = SOAP::Lite->proxy("$SNC_HOST/imp_notification.do?SOAP");
  my $method = SOAP::Data->name('insert')->attr({xmlns => 'http://www.service-now.com/'});

  # imp_notification record mapping, triggers the map
  # the type field value of "HPOV" is important
  # for the map to identify the incoming message as coming from HPOVO 
  my @params = ( SOAP::Data->name(type => 'HPOV') );

  push(@params, SOAP::Data->name(source => "$OVMSG_NODE_NAME") );
  push(@params, SOAP::Data->name(uuid => "$OVMSG_ID ") );
  push(@params, SOAP::Data->name(severity => "$OVMSG_SEVERITY") );
  push(@params, SOAP::Data->name(message => "$OVMSG_OBJECT") );
  push(@params, SOAP::Data->name(comments => "$OVMSG_MSG-TEXT") );
  push(@params, SOAP::Data->name(category => "$OVMSG_APPLICATION") );
  push(@params, SOAP::Data->name(assignment_group => "$OVMSG_MSG_GROUP") );

  # call the service and set the result object
  my $result = $soap->call($method => @params);

  # print out any SOAP Faults
  if ($result->fault) {
    print "faultcode=" . $result->fault->{'faultcode'} . "\n";
    print "faultstring=" . $result->fault->{'faultstring'} . "\n";
    print "detail=" . $result->fault->{'detail'} . "\n";
  }

  # print out all fields in the response document as a result
  # of the incident creation
  if ($result->body && $result->body->{'insertResponse'}) {
    my %keyHash = %{ $result->body->{'insertResponse'} };
    foreach my $k (keys %keyHash) {
        print "name=$k   value=$keyHash{$k}\n";
    }
  }
}

# convenient sub for creating xml elements
sub write_element {
  my ($label,$value) = @_;
  $writer->startTag($label);
   if($value) {
        $writer->characters($value);
   }
   $writer->endTag($label);
}

# this sub instructs the SOAP client to supply login credentials for
# the ITIL user is required
sub SOAP::Transport::HTTP::Client::get_basic_credentials {
   return 'itil' => 'itil';
}

ServiceNow writes the results of the script, whether a fault or a success, to the standard out. With a success, the script returns the internal sys_id and display value (by default this is the incident number) of the new ticket. If you need this information, you should parse the response for those points of data.

3.5.1 Using a Proxy Server

If your company uses a proxy firewall for restricting HTTPS access, you can specify proxy settings within the TroubleTicket.sh script. See http://cookbook.soaplite.com/#specifying%20proxy for examples.

If your proxy server does not allow a direct HTTPS POST, add these lines to the end of the TroubleTicket.sh script: 

$ENV{PERL_NET_HTTPS_SSL_SOCKET_CLASS} = "Net::SSL";
$ENV{PERL_LWP_SSL_VERIFY_HOSTNAME} = 0;
$ENV{HTTPS_PROXY} = 'my.proxy.server:80';
$ENV{HTTPS_PROXY_USERNAME} = 'username';
$ENV{HTTPS_PROXY_PASSWORD} = 'password';

If your proxy supports a direct HTTPS POST, replace the existing line: 

my $soap = SOAP::Lite->proxy("$SNC_HOST/imp_notification.do?SOAP");

With one of these lines: 

my $soap = SOAP::Lite->proxy('$SNC_HOST/imp_notification.do?SOAP', 
                              proxy => ['http' => 'http://my.proxy.server/']);

 # -- OR --

 my $soap = SOAP::Lite->proxy('$SNC_HOST/imp_notification.do?SOAP', 
                              proxy => 'http://my.proxy.server/');

 # -- OR --

 my $soap = SOAP::Lite->proxy('$SNC_HOST/imp_notification.do?SOAP');
 $soap->transport->proxy('http' => 'http://my.proxy.server/');

To configure authenticating proxy servers, please see http://cookbook.soaplite.com/#accessing%20service%20with%20proxy%20authentication.

3.5.2 Install Modified Perl Script on HP OVO Server

After downloading and modifying the script, copy it to the OVO directory for scripts and programs that call external interfaces. By default, this is: 

/opt/OV/bin/OpC/extern_intf
Note: Scripts and programs in this directory are erased if you uninstall HP OVO.


Refer to the HP OpenView Operations Administrators Guide for more information about installing scripts and configuring OVO to invoke them.

OVO sends its own message parameters to the external interface. You may not use a command that requires additional parameters. The ServiceNow Perl script (TroubleTicket.sh) accepts these parameters exactly the way they are, and in the right order. The following is a table of the default parameters and their acceptable values.


Parameter Description and Example
1 Unique message number.

Example: c1c79228-ae12-71d6-1a8f-0f887ebe0000

2 Message node name.

Example: hpbbxyz3.bbn.hp.com

3 Node type. For a list of supported managed nodes, see the Add Node window in the OVO administrator GUI or the HP OVO Installation Guide for the Management Server.

Example: HP 9000 PA-RISC.

4 Date (mm/dd/yyyy) on which the message was received on the managed node in the time zone (system-specific TZ variable) of the management server.

Example: 08/02/2002

5 Time (hh:mm:ss) at which the message was received on the managed node. This time uses a 24-hour clock in the time zone (system-specific TZ variable) of the management server.

Example: 16:22:04

6 Date (mm/dd/yyyy) on which the message was received on the management server in the time zone (system-specific TZ variable) of the management server.

Example: 08/02/2002

7 Time (hh:mm:ss) at which the message was received on the management server. This time uses a 24-hour clock in the time zone (system-specific TZ variable) of the management server.

Example: 16:22:05

8 Application name.

Example: /bin/su(1) Switch User

9 Message group.

Example: Security

10 Object name.

Example: root

11 Message severity (unknown, normal, warning, minor, major or critical).

Example: normal

12 List of responsible OVO operators. Names are separated with one space.

Example: opc_op Bill John

13 Message text. Text is not enclosed in quotation marks ("").

Example: Succeeded switch user to root by charlie

14 Instructions (empty string if not available). The instructions are passed without quotation marks (""), backslashes (\), or other characters that might be interpreted by a UNIX shell.

Example: This is the instruction text for the appropriate message condition. It is available for the operator when a message matching this condition appears in the Message Browser.

15 Custom message attributes (empty string if not available). Multiple name=value pairs are separated with two semi-colons (;;).

Example: Customer=Hewlett-Packard;;Country=United States of America

3.6 Step 6. Setting Integration Properties

In ServiceNow, navigate to Integration - HP OVO > Properties and set these properties to configure the HP OVO event as well as the MID Server that runs HP OVO commands.

Property Description
The absolute path to the command line command to use that posts an event to the HP OpenView server. This command line is going to be executed by the MID Server configured for this integration. Provide the path on the HP OVO server to run the opcmsg command. The default value is /opt/OV/bin/OpC/opcmsg.
The MID Server that executes the HP Openview command line, defaults to the system MID Server which is installed on this instance of Service-now. Typically, this would be a MID Server installed behind the firewall. http://wiki.service-now.com/index.php?title=MID_Server_Plugin Provide the name of the MID Server that connects to the HP OVO server. The default value is localhost.
User for submitting the Openview message: Provide the HP OVO User account the integration uses to update trouble tickets on the HP OVO server. The default value is opc_adm.



4 Parsing the SOAP response

The following SOAP response illustrates inserting in a new incident. 

<SOAP-ENV:Envelope
 xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xmlns:xsd="http://www.w3.org/2001/XMLSchema"
 xmlns:SOAP-ENV="http://schemas.xmlsoap.org/soap/envelope/"
 xmlns:SOAP-ENC="http://schemas.xmlsoap.org/soap/encoding/"
 SOAP-ENV:encodingStyle="http://schemas.xmlsoap.org/soap/encoding/">
  <SOAP-ENV:Body>
    <insertResponse>
      <sys_id>8060c70b0a0a0b50000d0816b5196a87</sys_id>
      <table>incident</table>
      <display_name>number</display_name>
      <display_value>INC10116</display_value>
      <status>inserted</status>
    </insertResponse>
  </SOAP-ENV:Body>
</SOAP-ENV:Envelope>
  • sys_id - Lists the unique ID of the Incident record that was inserted/updated.
  • table - Lists the mapped target table that was modified.
  • display_name - Lists the display name field of the table. For example, the number for task.
  • display_value - Lists the value of the display name field. For Incidents, this is the incident number.
  • status - Lists the action as a result of the map (inserted/updated/error).

If a server exception occurs, the SOAP response contains a fault message. 

<SOAP-ENV:Envelope
 xmlns:SOAP-ENV="http://schemas.xmlsoap.org/soap/envelope/">
  <SOAP-ENV:Header></SOAP-ENV:Header>
    <SOAP-ENV:Body>
        <SOAP-ENV:Fault>
          <faultcode>SOAP-ENV:Server</faultcode>
          <faultstring>java.lang.Exception: This is a fault message</faultstring>
          <detail>Error completing SOAP request</detail>
      </SOAP-ENV:Fault>
    </SOAP-ENV:Body>
</SOAP-ENV:Envelope>

If an error is detected during the processing of the web service map, the SOAP response contains an error message. 

<SOAP-ENV:Envelope
 xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xmlns:xsd="http://www.w3.org/2001/XMLSchema"
 xmlns:SOAP-ENV="http://schemas.xmlsoap.org/soap/envelope/"
 xmlns:SOAP-ENC="http://schemas.xmlsoap.org/soap/encoding/"
 SOAP-ENV:encodingStyle="http://schemas.xmlsoap.org/soap/encoding/">
  <SOAP-ENV:Body>
    <insertResponse xmlns="http://www.service-now.com">
      <sys_id>error</sys_id>
      <status>error</status>
      <error_message>No web service automap found or no fields were mapped</error_message>
    </insertResponse>
  </SOAP-ENV:Body>
</SOAP-ENV:Envelope>
  • sys_id - Indicates that an error has occurred.
  • status - Indicates that an error has occurred.
  • error_message - A short description of the type of error that has occurred.

5 Notification Web Service Import Set

After the the HP OVO server posts the SOAP message, ServiceNow creates an inbound Notification import set row. To view the import set, navigate to Integration - HP OVO > Notification WS and click the Import Sets related link. The integration uses standard web service import sets. Like any web service, you can add fields or change the import set transform map.



6 HPOV Updater Script Object

The integration runs the HPOVUpdater script whenever ServiceNow updates an incident originally created from HP OVO. This script sends the update back to HP OVO as an HP OVO event. The onAfter transform script creates a listener for the HPOVUpdater script.


To include any additional fields in the integration:

  1. Navigate to Integration - HP OVO > HPOV Updater.
  2. Edit the script to include additional fields.
  3. Click Update.
gs.include("PrototypeServer");

var HPOVUpdater = Class.create();

HPOVUpdater.prototype = {
  initialize : function() {
    this.command = gs.getProperty("com.snc.integration.hpov_command", "/opt/OV/bin/OpC/opcmsg");
    this.midServer = gs.getProperty("com.snc.integration.hpov_mid_server", "localhost");
    this.opcUser = gs.getProperty("com.snc.integration.hpov_user", "opc_admin");
    this.commandStr = this.command + 
                      " -u " + this.opcUser + " ";
    this.arguments = "";
  },

  process : function() {
    if (operation == "insert")
      this.processInsert();

    if (operation == "update")
      this.processUpdate();

    if (operation == "delete")
      this.processDelete();
  },

  processUpdate : function() {
    if (!delta.contains("assignment_group") &&
       !delta.contains("assigned_to") &&
       !delta.contains("severity") &&
       !delta.contains("comments") &&
       !delta.contains("incident_state")) {
         gs.log("conditions unchanged for " + current.number + ", HPOV not updated");
         return; // dont update HPOV
    }

    // map ServiceNow fields to HPOV attributes
    this.addArgument("severity",current.severity);
    this.addArgument("id",current.correlation_id);
    this.addArgument("msg_group",current.assignment_group.getDisplayValue());
    this.addArgument("application",current.category);
    this.addArgument("operator_list",current.assigned_to.getDisplayValue());
    this.addArgument("msg_text", current.incident_state + " - " + current.comments.getJournalEntry(1));

    var egr = new GlideRecord("ecc_queue");
    egr.initialize();
    egr.agent = "mid.server." + this.midServer;
    egr.queue = "output";
    egr.topic = "Command";
    // build the HP OVO command
    egr.payload = "<parameters><parameter name='name' value='" + 
                  this.commandStr + 
                  this.arguments + 
                  "'/></parameters>";
    egr.insert();
  },

  processDelete : function() {
    // process a deleted incident
  },

  processInsert : function() {
    // process inserting an incident
  },

  addArgument : function(name, value) {
    this.arguments += " " + name + "=\"" + value + "\"";
  }
}
