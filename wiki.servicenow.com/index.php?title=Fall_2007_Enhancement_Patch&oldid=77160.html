Fall 2007 Enhancement Patch

Fall 2007 Enhancement Patch

  • 2007 portal

The following issues are addressed by targeted fixes in a first patch deployed on 30 Nov 2007 to customer instances running our Fall 2007 Enhancement.

  • Under certain circumstances, wrong version of customer update files may be installed during an upgrade to an instance running Update Sets.
  • Errors in generating queries involving the LIKE operator against sets (e.g. queries against the watch list field on incidents).
  • Error when trying to close incidents, problems, or sc_tasks if certain sys_dictionary attributes are missing.
  • Error when email is sent w/o a subject.
  • Behavior change for "missing" choice list entries. Missing entries now show up color coded but otherwise look similar to previous versions.
  • Non-admin users cannot apply templates either from menu or via right click -> apply template.
  • Templates which set journal fields (work_notes, comments, etc) did not set properly.
  • Performance problems caused by large, empty, sys_cluster_message table resolved (via compacting same)
  • NPE when Show Matching/Filter Out done on a null date/time
  • Incorrect results when Show Matching/Filter Out done on a non-null date/time
  • Formatting errors when typing a decimal without a leading zero into a decimal field (e.g. .75 -> NaN.75)
  • Aggregation type query issue related to company separation
  • Delivery Plans not properly generating approvers on approval tasks. Approval task would commence, but no approvers would be added.
  • Delivery Plans not properly auto-setting an assignment based on the execution plan record.
  • Delivery Plans not automatically advancing to next step upon approval of an approval task.
  • Focus event would "steal" focus and place it in first field on screen, even if typing had begun elsewhere
  • Images on Catalog watch list macro didn't match newly revised watch list
  • Customers with update sets, who customized items in their production system, and then subsequently installed at least one update set, might be running version of customization from most recent update set, rather than version that had been changed locally subsequent to update set installation.
  • When ordering a "bundle" via the catalog, variables on the bundle itself made it into the request items, but variables from the catalog items did not. Behavior was proper if items were ordered individually.
  • Property to delay assignment of number until insert didn't delay same and used a number as soon as you pressed "New".
  • NPE if gr.<field>.setDisplayValue(null) was called under some circumstances.
  • Personalize Control erroneously appeared as an option on programmatic related lists (Audit Records, Attachments, etc).
  • Filters for dependent choice lists did not function.
  • MID server scripts caused ClassCastException under some circumstances.
  • BSM map popup didn't scroll properly.
  • Email Templates involving catalog variables are not sending properly.
  • Label headers in navigation frame did not take you to the Label record.
  • Fix for issue where system got confused with two records from different tables having same sys_id
  • Add back ScriptHelper.getDisplayValue() because some email templates still rely upon it
