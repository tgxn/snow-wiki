1 Overview

You can use stage sets to organize stages as a cohesive, named unit for reuse in any number of workflows. A stage set, which groups commonly used stages together, usually represents a process, such as the stages required to display the progress of a service catalog request. You can also assign a stage set as the default set for workflows on specific tables.

In an active context, workflow stages provide summary-level feedback about the progress of a workflow. Stage icons display the status of each activity as it is being driven by a workflow. The system updates a Stage field, defined in the workflow properties, to indicate the workflow's progress through the possible stage values.

Stages sets are available starting with the Dublin release. For more details on stages, see Workflow Stages.

2 Creating a Stage Set

To create a stage set, you have the following options:

  • Create a new stage set record and add stage set entries manually.
  • Import a choice list into a workflow.
  • Export the stage set from an existing workflow.

2.1 Creating a Stage Set Record

A stage set record contains pre-defined stage sets for reuse in workflows. In an active context, workflow stages provide summary-level feedback about the progress of a workflow. Stage icons display the status of each activity as it is being driven by a workflow. The system updates a Stage field, defined in the workflow properties, to indicate the workflow's progress through the possible stage values.

You create a stage set record only when you want additional stages that are not available by default on the table.

  1. Navigate to Workflow > Stage Sets.
  2. Click New.
  3. Enter a Name that indicates the purpose of the stage set.
    For example, you can create a Requested Item stage set to hold the stages commonly used by inventory tracking workflows or service catalog fulfillment workflows.
  4. Click Submit.
  5. Open the new stage set record.
  6. In the Stage Set Entries related list, click New. Each stage set entry represents a Stage for a workflow activity that uses this stage set.
  7. Enter a Name that indicates the stage name to appear in workflow fields.
  8. Enter a unique Value to use when referencing the stage, such as in a script.
  9. Enter a unique Order number for the stage. Enter a low Order number for a stage that should appear early in the workflow. Enter a high Order number for a stage that should appear later in a workflow.
  10. Complete the form, and then click Submit.
  11. In the Stage Set Entries related list, ensure that each entry has a unique Order value. This number can be updated at any time to reorganize stages.

The stage set can be added to a workflow or assigned as the default set for workflows that are created for a specific table.

2.2 Importing a Choice List into a Workflow

You can create and reuse a choice list of stages for workflows. By importing a choice list, you can assign stages to activities on the workflow.

  1. If there is no choice list with changes, do the following:
    • Add a custom field of Type Workflow to the table.
    • Configure the custom field to use a choice list.
    • If you are creating a new field, set the Choice List type to Display without --None-- and create the create the choices for the newly created field.
  2. Navigate to Workflow > Workflow Editor.
  3. Open and checkout the workflow that corresponds to the table that contains the newly created stage field.
  4. Click the menu icon (gear icon in releases prior to Fuji) and select Properties.
  5. Click the Stages tab.
  6. In the Related Links section, click Import Stages from Choice List.
  7. To confirm that you want to import the choice list, click OK.

2.3 Exporting a Stage Set from a Workflow

You can use the stages defined in an existing workflow to create a stage set. By exporting a stage set, you can reuse those stages without manually duplicating the stage set entries.

  1. Navigate to Workflow > Workflow Editor.
  2. Open and check out a workflow.
  3. Click the menu icon (gear icon in releases prior to Fuji) and select Edit Stages.
  4. Click the Export to Stage Set related link.
  5. Enter a unique Name for the new stage set.
  6. Click OK.

The stage set can be added to a workflow or assigned as the default set for workflows that are created for a specific table.

3 Adding Stage Sets to a Workflow

You can add any number of stage sets to an existing workflow. By importing a stage set, you can reuse those stages without manually duplicating the stage set entries.When multiple stage sets have stage set entries with the same Value, the stage appears on the workflow only once.

  1. If you want additional stages that are not available by default on the table, create a new stage set.
  2. Navigate to Workflow > Workflow Editor.
  3. Open and check out a workflow.
  4. Click the menu icon (gear icon in releases prior to Fuji) and select Edit Stages.
  5. Click Import from Stage Set.
  6. Select a stage set.
  7. Click OK.
  8. Import additional stage sets or edit the stages as needed for the workflow.

After you add all necessary stage sets to the workflow, you can add the stage set entries to activities in the workflow.

4 Assigning Default Stage Sets for a Table

You can assign a stage set as a default set to any number of tables. The stages in the assigned stage set pre-fill the workflow version stages when you create a new workflow for the associated table.

  1. Navigate to Workflow > Default Stages (by table).
  2. Click New.
  3. Select a Table.
  4. From the Set list, select the stage set that you want to assign to the selected table.
  5. Click Submit.

4.1 Upgrading Stages to Stage Sets

When you upgrade from a version prior to Dublin, ServiceNow automatically creates a default stage set for each table that had defined default stages. The Name of these stage sets is the display name of the table they affect. Each stage set contains all the stages that were associated to that table.
