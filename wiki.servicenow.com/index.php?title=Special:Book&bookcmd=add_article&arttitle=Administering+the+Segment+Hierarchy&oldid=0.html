1 Overview

All of the accounts in the IT chart of accounts belong to segments, which are structured in a hierarchy. The hierarchy makes it possible for you to roll up expenses from lower-level accounts and segments to higher-level accounts and segments. For example, assume you use the default hierarchy, which has the business service segment under the business unit segment. This enables you to have the expenses that are allocated to a business service, like email, also apply to the business unit that handles the email service, which would commonly be IT.

Note: The workbench creates all the rules and methods automatically when you use the workbench to modify the hierarchy. Using the workbench is the preferred method of modifying the hierarchy, rather than using the Segment Relationship form.


2 Default Segment Hierarchy

By default, the segments in the IT chart of accounts are in this hierarchy:

  • Root level: Business Unit
  • Level 2: Business Service
  • Level 3: Project and IT Shared Service
The default segment hierarchy

You can add a segment to the second level and lower levels but not to the top, or root, level. The hierarchy can only have one root segment. Each segment in the hierarchy can have only one parent segment. Each parent segment can have one or more child segments.

Note: The hierarchy only defines relationships between the segments. You can still allocate expenses to accounts in a segment that is not in the hierarchy.


To modify the segment hierarchy, you can use the workbench, or create relationship records between a parent and child segment. See Reviewing the Data Definition for instructions on modifying the hierarchy in the workbench. See Creating Segment Relationships for instructions on using the Segment Relationship form. Also see Rollups for Allocations for instructions on how to create rollup rules that define how expenses are assigned to both child and parent segments.

3 Segment Relationship Records

A segment relationship record associates one segment as the parent and one as the child. You can create or modify segment relationship records, which then modifies the segment hierarchy.

Note: If you want to modify the root segment or modify the relationships between several segments, consider using the workbench.


4 Creating Segment Relationships

  1. Navigate to IT Finance > Chart of Accounts > Segment Hierarchy.
    The list of all segments in the hierarchy appear with their parent segment.
  2. Click New to create a new parent-child relationship.
  3. Fill out the fields on the form (see table).
  4. Click Save.
    The list of segments appears.
  5. Click the Generate Rollups related links.
    The segment hierarchy is refreshed, and the application creates new relationships with the appropriate rule, method, and metric. Any relationships that you delete also have the associated metric, rule, and method deleted. If the deleted relationships already have rules that are associated with existing cost allocations, the rules are set to inactive rather than deleted.
The Segment Relationship form

Field Description
Root segment Select this check box to place this segment at the top of the hierarchy, which means that the segment cannot have a parent segment.
Parent segment Select the parent segment in the relationship. You actually select the associated dimension for the segment. This field is only available when Root segment is not selected.
Child segment Select the child segment in the relationship.
