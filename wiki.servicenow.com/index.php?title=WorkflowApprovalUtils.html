3 Method Detail

3.1 isTask(tableName)

Tests whether a table is the task table or derived from the task table.

3.1.1 Input Fields

Parameters:

  • tableName - The name of the table to test.

3.1.2 Output Fields

Returns: (Boolean) true, if the input table name is a task table or derivative from the task table.

3.2 cancelAll(task, comment)

Cancels all approvals for a task.

3.2.1 Input Fields

Parameters:

  • task - GlideRecord of a record derived from task.
  • comment - String comment to apply to cancelled approvals.

3.2.2 Output Fields

Returns: void

3.3 reset(task, comment)

Resets all approvals by deleting all of them along with any workflow associated with the task so that the workflows will be rerun and recalculated.

3.3.1 Input Fields

Parameters:

  • task - GlideRecord of a record derived from task.
  • comment - String comment to apply to the history logs.

3.3.2 Output Fields

Returns: void

3.4 createUserApprovals(groupApproval)

Creates individual user approvals for the input group approval. After calling this method, every user in the group approval is inserted in table sysapproval_approver.

3.4.1 Input Fields

Parameters:

  • groupApproval - GlideRecord on table sysapproval_group.

3.4.2 Output Fields

Returns: void

3.5 getMembersOfGroup(groupID)

Gets the list of user sys_ids for each member of a group.

3.5.1 Input Fields

Parameters:

  • groupID - The sys_id of a group (table sys_user_group).

3.5.2 Output Fields

Returns: Array of sys_ids of users (table sys_user).

3.6 getGroupApprovalsByApprovalManager(taskId, amId)

Gets an object of group approvals that were created by all of the children of an approval manager activity. Use this to determine if we are about to create a duplicate and to determine if we are reusing an approval that we previously created. Used in Activity "Approval Coordinator".

3.6.1 Input Fields

Parameters:

  • taskId - sys_id of current (the task derived GlideRecord that launched this workflow).
  • amId - sys_id of an approval manager activity. This is a sys_id on table wf_activity.

3.6.2 Output Fields

Returns: - Return an object that has: 

      key -> sys_id for the group approval
      value -> { sys_id: sys_id of the approval record
                 state: current approval state
                 group_id: sys_id of the group on the approval record
                 activity_id: sys_id of the activity that created the approval record
               }

3.7 getUserApprovalsByApprovalManager(taskId, amId)

Gets an object of user approvals that were created by all of the children of an approval manager activity. Use this to determine if you are about to create a duplicate and to determine if you are reusing an approval that was previously created.

3.7.1 Input Fields

Parameters:

  • taskId - sys_id of the current task (the task derived GlideRecord that launched this workflow).
  • amId - sys_id of an approval manager activity. This is a sys_id on table wf_activity.

3.7.2 Output Fields

Returns: - An object that has: 

      key -> sys_id for the user approval
      value -> { sys_id: sys_id of the approval record
                 state: current approval state
                 group_id: sys_id of the user on the approval record
                 activity_id: sys_id of the activity that created the approval record
               }

3.13 setGroupApprovalsByTask(taskID, approvalState, comment, currentStates)

Sets all group approvals of a parent task to the specified state.

3.13.1 Input Fields

Parameters:

  • taskID - sys_id of the task to set all approvals.
  • approvalState - The new state to set all the group approvals to.
  • comment - Optional string comment to apply to the approval state transition.
  • currentStates - Optional Array of strings. List of current states to apply the new state to. If absent then all states are affected.

3.13.2 Output Fields

Returns: void

3.14 setPendingGroupApprovalsByTask(taskID, approvalState, comment)

Sets all group approvals that are pending to the specified state. This is the same as calling:

Returns: void

3.23 addIdsToApprovalList(approvals, o)

Adds a list of ids to the approval list. This sanitizes the input list and evaluates JEXL statements. Used in activity "Approvals - User" and "Approval - Group".

3.23.1 Input Fields

Parameters:

  • o - Comma-delimited list or array of comma-delimited lists of ids. This can contain JEXL script such as ${some script}.

3.23.2 Output Fields

  • approvals - New ids are added to this object.

Returns: No return value except the approvals argument.

3.24 addUsersAndGroupsToApprovalList(users, groups, o)

Analyzes a list of ids, sanitizes them, and adds them to either the approval users list or approval groups list. This also evaluates JEXL statements.

3.24.1 Input Fields

Parameters:

  • o - Comma-delimited list or array of comma-delimited lists of ids. This can contain JEXL script such as ${some script}.

3.24.2 Output Fields

  • users - Sanitized users from o are added to this list of Ids.
  • groups - Sanitized groups from o are added to this list of Ids.

Returns: No return value except the method arguments above.

3.25 createGroupApprovers(groups, order, state)

3.25.1 Input Fields

Parameters:

  • groups - Array of sys_ids of the groups.
  • order - Not used.
  • state - The state value to apply to the new approval. Defaults to "not requested".

3.25.2 Output Fields

Returns: void
