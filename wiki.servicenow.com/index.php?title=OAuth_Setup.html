1 Overview

You can enable an OAuth 2.0 integration to authorize access to ServiceNow applications on behalf of a user, if you have the security_admin role. You can also configure settings for new endpoints and for third-party OAuth providers.

2 Setting the OAuth Property

By default, OAuth is active on new and upgraded instances. If it is not, you can activate the plugin and add the following property to generate OAuth 2.0 tokens to registered applications. See Adding a Property for instructions. The property details are:

  • Name: com.snc.platform.security.oauth.is.active
  • Type: true | false
  • Default value: false

Set the property to true to use OAuth 2.0.

3 Registering an Application on ServiceNow

Each client application must register with the ServiceNow instance to participate in OAuth authorization. You can create new endpoints for external clients and for third-party OAuth providers.

3.1 Creating Endpoints for External Clients

  1. Navigate to System OAuth > Application Registry.
  2. Click New.
  3. On the interceptor page, click Create an OAuth API endpoint for external clients.
  4. Fill out the fields, as appropriate (see table).
  5. Click Submit.
    The record is saved in the Application Registries [oauth_entity] table.
  6. After you create the endpoint, you must configure applications to create an HTTP POST requesting an OAuth token. The application must also be able to parse the JSON response to use the provided access and refresh tokens. See Generating OAuth Tokens.
The Application Registry form for a new endpoint

Field Description
Name A unique name identifying the application you are requiring OAuth access for.
Client ID [Read-Only] The automatically-generated unique ID of the application. The instance uses the client ID when requesting an access token.
Client Secret [Required] The shared secret string that both the instance and the application use to authorize communications with one another. The instance uses the client secret when requesting an access token. The instance automatically generates a client secret if you leave this field blank.
Logo URL The URL containing an image to use as the application logo.
Active A check box for indicating that the instance can authorize access to the application. Only active applications can request access tokens.
Refresh Token Lifespan The number of seconds a refresh token is valid. The instance uses this lifespan value when requesting a refresh token. By default refresh tokens expire in 100 days (8640000 seconds).
Access Token Lifespan The number of seconds an access token is valid. The instance uses this lifespan value when requesting an access token. By default access tokens expire in 30 minutes (1800 seconds).
Comments Any additional information you want to associate with this application.


3.2 Using a Third-Party OAuth Provider

  1. Navigate to System OAuth > Application Registry.
  2. Click New.
  3. On the interceptor page, click Connect to a third party OAuth Provider.
  4. Fill out the fields, as appropriate (see table).
  5. Click Submit.
    The record is saved in the Application Registries [oauth_entity] table.
    See the client API reference for more information on using the methods available for OAuth.


The Application Registry form for third-party OAuth providers

Field Description
Name A unique name identifying the data source you are using OAuth to access.
Client ID The unique ID of the protected resource.
Client Secret [Required] The shared secret string the instance and the resource server use to authorize communications with one another. Enter a string.
OAuth API Script The script used to customize the request and response to the third-party OAuth provider. The script name must have the prefix OAuth. By default, the OAuthUtil script include is available for use. See the client API reference for more information.
Logo URL The URL containing an image to use as the application logo.
Active A check box for indicating that the third-party provider can be accessed. Only active records can have valid authorization grants to access the protected resource.
Token URL The location of the token endpoint that the instance uses to retrieve access and refresh tokens.
Redirect URL The application endpoint that receives the authorization grant.
Token Revocation URL The location of the endpoint that the instance uses to revoke the token
Comments Any additional information you want to associate with this application.
