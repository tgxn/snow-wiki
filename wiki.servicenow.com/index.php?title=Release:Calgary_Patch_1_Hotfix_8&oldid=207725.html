Text Search PRB584653 When a Microsoft Excel file is attached to a record in a table and text search indexes the attachment, formulas on rows that reference other rows within the Excel file may cause the instance to run out of memory.

To enable this fix, create the glide property glide.ts.index.poi.use_event_extractor and set it to true. This will cause the indexer to use a document parser that uses less memory. It will also skip the metadata and some calculated cells' content during indexing.

Web Services PRB583293 Unable to insert asset_tag and serial_number into alm_hardware through SOAP.

  1. Use a SOAP UI to access the alm_hardware WSDL from an instance. For example, https://demo.service-now.com/alm_hardware.do?WSDL
  2. Pass a SOAP envelope to the alm_hardware service with values for the serial_number and asset_tag fields, but no value for the quantity field. For example:
    <soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/"; xmlns:alm="http://www.service-now.com/alm_hardware">;
    <soapenv:Header/>
    <soapenv:Body>
    <alm:insert>
    <alm:asset_tag>This is my asset tag.</alm:asset_tag>
    <alm:install_status>11</alm:install_status>
    <alm:location>25ab96850a0a0bb300d52a586a55468c</alm:location>
    <alm:serial_number>This is my serial number.</alm:serial_number>
    <alm:comments>INT2179253</alm:comments>
    </alm:insert>
    </soapenv:Body>
    </soapenv:Envelope>
  3. Login to the instance and review the Hardware [alm_hardware] table. The newly created record does not contain the asset_tag and serial_number attributes.
  4. Add the quantity field as the first field in the SOAP envelope. For example:
    <soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/"; xmlns:alm="http://www.service-now.com/alm_hardware">;
    <soapenv:Header/>
    <soapenv:Body>
    <alm:insert>
    <alm:quantity>1</alm:quantity>
    <alm:asset_tag>This is my asset tag.</alm:asset_tag>
    <alm:install_status>11</alm:install_status>
    <alm:location>25ab96850a0a0bb300d52a586a55468c</alm:location>
    <alm:serial_number>This is my serial number.</alm:serial_number>
    <alm:comments>INT2179253</alm:comments>
    </alm:insert>
    </soapenv:Body>
    </soapenv:Envelope>
  5. Resend the SOAP envelope. The alm_hardware record now contains values for the asset_tag and serial_number attributes.

Calgary Release


4 Overview

The Calgary release provides enhancements that accelerate IT responsiveness and innovation and drive enterprise visibility and standardization.

  • Enhancements: view new features and notable changes.
  • Fixes: see fixes included in the initial Calgary release.
  • Available Versions: browse fixes in all versions within this family.
  • Submit Feedback: provide feedback on documentation.

You can also download all Calgary release notes pages as a PDF.

Warning:
  • Customers running Berlin, Berlin Patch 1, Berlin Patch 2, or any hotfixes to these versions must contact ServiceNow Technical Support before upgrading to the Calgary release. This does not apply to customers whose instances are being upgraded to Calgary Patch 4 or higher.
  • Customers upgrading to Calgary may encounter issues with Outlook emails. For details, see KB0529478 in the ServiceNow Knowledge Base.
  • Customers upgrading to Calgary may encounter an issue where "guest" users cannot create incidents. This can be remedied by unlocking the guest user or by setting glide.pop3.process_locked_out to true. Be aware that setting the glide property to true enables users from untrusted domains to trigger inbound actions. See Allowing Locked Out Users to Trigger Inbound Email Actions for details.

5 Enhancements

6 Applications

Data Certification

Ensure that CMDB data is accurate. Define criteria, create filters, and use schedules to drive data validation. Supports notifications and escalations.

Procurement

Helps you manage requests and purchase orders. Features include creating purchase orders and purchase order lines, creating receiving slips and receiving slip lines, and sourcing requests using transfer orders and purchase orders.

Contract Management

New features include relating assets to contracts, terms and conditions, adjusting contracts, generating expense lines for contracts, logging contract history, and using rate cards with contracts.

Work Management

Provides skills and location driven work assignment. Offers seamless process flow from incident and other tasks.

7 Platform

Tablet Interface

New iPad interface supports nearly full-product functionality with a user-friendly interface.

PDF Page Header Footer Templates

Allows users with the report_admin role to create header and footer templates. Allows reporting users to apply header and footer templates to exported PDFs.

Field Comparison

Allows users to evaluate equality between field values using the condition builder.

Zing Text Search

Improves performance and replaces the text index reaper with a new text index structure.

8 Development, Integrations, & Email

Application Creation

Streamlines custom application creation by making it easier to create tables, linking code changes to applications, and publishing finished applications to update sets.

Table Administration

Simplifies and consolidates table creation and editing into one convenient location and uses a standardized user interface.

Schema Map

Improved representation and navigation of tables and relationship types. Ability to view table columns, navigate to table data and dictionary entries, and print the schema map.

Scripting APIs

Eliminates the need to make Java Packages calls from your scripts. Includes Packages Call Removal Tool to migrate custom code.

9 Automation

New application enables the provisioning and management of private (VMware) and public (Amazon EC2) virtual machines in a single application that is fully integrated with ServiceNow.

Workflow Validation

New validation tool enables workflow designers to run validation tests on their models prior to publication and to detect potential problems in update sets containing separate workflow elements.

Microsoft Hyper-V Discovery

Discovery now creates CMDB records for the Hyper-V server, including the host machine, running virtual machines, stored templates, and component relationships.

To mitigate performance issues during large discoveries, resource throttling controls the interval between Discovery sensor processing jobs so that no two sensors are processed simultaneously.
