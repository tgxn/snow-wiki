Fall 2010 Notable Changes

From ServiceNow Wiki

The following are notable changes you may encounter with our Fall 2010 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before. For additional changes, see Fall 2010 Fixes.

1 Changes for Fall 2010

1.1 SOAP strict security

SOAP strict security is now enabled by default in new instances.

1.2 MID Server role

A MID Server role has been added to ensure that the MID Server has access to tables protected by ACLs. This role is necessary if SOAP strict security is enabled.

1.3 Choice List can be focused on when form loads

When a form loads, the first writable field on the form is focused. Now, choice lists are considered writable fields for this purpose.

1.4 SOAP Web Services Performance

The SOAP Web Services execution speed has been optimized.

1.5 Catalog Cart Macro Changes

The servicecatalog_cart_template UI Macro (displays the catalog cart in edit mode) has been altered. Customers who have customized this macro will not receive the new version and may need to revert, then reapply their changes.

1.6 Correct Checking of Survey Names

An invalid survey link opens a blank survey rather than all survey questions. In previous versions, an incorrectly specified survey link (e.g., survey_take.do?sysparm_survey=survey_does_not_exist) opens a survey with all survey questions in the system.

1.7 JDBC Drivers for Unsupported Data Sources

A new JDBC driver is available that provides connection to databases not supported in the out-of-box ServiceNow system.

1.8 Session Object Unavailable in Sandboxed Scripts

The Session object, gs.getSession(), returns null when called within the sandbox (query condition evaluation or AJAXEvaluation). To test for interactivity, replace a call to gs.getSession().isInteractive() with gs.isInteractive().

1.9 Ext JavaScript Library removed

The Ext JavaScript library has been removed and is no longer available. Customers who wish to still use it may provide the functions they need in a custom Global UI Script.


2 Changes for Spring 2010 Stable 3

2.1 Updated Client Side Libraries

The version of the Prototype javascript library used by the product has been updated to version 1.7. The product now includes with the version 1.4.2 of the JQuery client side library.

2.2 Percent Complete field type

A new field type called percent_complete displays a progress bar in lists that tracks task or project completion rates. Add display attributes to this field to compare the actual percentage completion of a task with a predicted target value.

2.3 Removed AJAX Calls

All AJAX calls (some of which were synchronous) previously run when the navpage framework was loaded have been removed. This change improves the rendering speed of the framework.

2.4 New fields added to planned_task

Two new fields, Remaining duration and Remaining effort, have been added to the planned_task table.

2.5 Property to deactivate legacy SLA engine

A new property, com.snc.sla.run_old_sla_engine, deactivates the legacy SLA engine if the new SLA plugin is installed, preventing both engines from running and conflicting. The property can be viewed in the sys_properties list.

2.6 Changes to SLA Plugin Calculations

Three changes have been made to SLA Plugin calculations:

  1. The multiple business rules that power the SLA plugin have been consolidated into a single business rule, Process SLAs, which is now asynchronous. The SLA will no longer need to be calculated fully after the user submits the form; the user will be returned to the form immediately and the calculation will occur in the background. This means that the SLA may not appear immediately after the task is inserted; the form may need to refreshed.
  2. A single calculation script now calculates all of the duration fields in the SLA, including a new field named Business Pause Duration.
  3. The scheduled job that processes the SLAs every two minutes has been broken up into 6 smaller scheduled jobs, which will calculate more frequently as the SLA approaches breached, reducing the load on the system.

2.7 Increased use of Asynchronous AJAX

Throughout the system, many synchronous AJAX calls have been converted to asynchronous AJAX calls. This improves the end user experience because the browser no longer has to wait for the AJAX call to complete before the user can continue working.

2.8 Category field added to UI Page (sys_ui_page) table

A Category field to UI Pages, such as the one that exists for UI Macros, has been added to track the functional part of the system to which a UI Page belongs. This allows for easy grouping of all Service Catalog pages, CMS pages, Knowledge pages, etc.

2.9 New operators for Discovery Classification

Two new negative operators (does not equal and does not start with) have been added to Discovery Classification criteria to make classification conditions more flexible.

2.10 ESX added as an OS choice

ESX has been added as an operating system choice for computers.

2.11 New Mobile modules default to mobile view

If a new Mobile module is created, the view name will default to mobile on the sys_app_module record. This is done with the default value field on the sys_app_module.view_name Dictionary record.

2.12 Property specifying SNMP probe use of public

A new Discovery property, mid.snmp.enable_auto_public, controls whether SNMP probes automatically use the community string public. The default value is true.

2.13 Default value of a field can be overridden

A field's default value in the Dictionary can now be overridden with Dictionary Overrides.

2.14 CMDB schema change for hardware classes

Added an intermediate hardware class to the CMDB schema for various types of hardware. For example: cmdb_ci > cmdb_ci_computer is changed to: cmdb_ci > cmdb_ci_hardware > cmdb_ci_computer. The affected classes are:

  • cmdb_ci_computer
  • cmdb_ci_printer
  • cmdb_ci_netgear
  • cmdb_ci_ups
  • cmdb_ci_pdu.

2.15 addJoinQuery(joinTable, [primaryField], [joinTableField]) method added to GlideRecord

A new method, addJoinQuery(), has been added to GlideRecord. This method makes it possible to query records via relationships in other tables. For example, find all the users that are in the database group (users via sys_user_grmember table). Or find all problems that have an assigned incident (problems via the incident.problem_id relationship).

2.16 Help the Help Desk now requires soap_ecc role

Help the Help Desk now requires a particular ACL to access the instance through a script: soap_ecc.


3 Changes for Spring 2010 Stable 2

3.1 Using ACLs to Control Workflow Publishing

Access Control Rules (ACL's) can now restrict a user's rights to check out and publish workflows. This capability provides for users who can edit some workflows and not others (e.g., users who can edit workflows within their own department, but not within other departments).

3.2 Mutual Authentication for Outbound SOAP Requests

Web Services now supports mutual authentication for outbound SOAP requests by extending protocol support.

3.3 Variable Support for JSON and SOAP get/getRecords

Web Services now returns all the variables associated with a record for JSON and SOAP get or getRecords functions.

3.4 Fixed Queries in List v2 Breadcrumbs

A new breadcrumb option for modules enables an administrator to control the base view of a record list and prevent users from executing queries backwards beyond a configured point.

3.5 New Related Lists in CI records

Records for manageable configuration items, such as computers and network gear, now contain three new Related Lists that display the IP addresses, network adapters, and DNS names associated with that CI.

3.6 Data URLs for Report Images

Instances now use data URLs for report images, reducing the amount of transactions between the client and the server when rendering reports.

3.7 Cannot impersonate inactive or locked out users

Inactive and locked out users are no longer candidates for impersonation.


3.8 Email Client cannot send to inactive or locked users

Inactive and locked out users are no longer included in the email client address autocomplete.


3.9 Email Client no longer saves the record

Launching the Email Client no longer saves the parent record in the background. Values from the record are now populated as needed into the email without saving. Additionally, the email client icon is no longer available on new records (records that have not yet been inserted). An admin can configure an instance to include the icon on new records by creating a new true/false property: "glide.ui.email_client.pop_on_new_records" and setting its value to true.


3.10 Workflow Version security

Checking out and publishing workflows now requires Create access to the wf_workflow_version table instead of Write access.


3.11 Discovery cancelled if range too large

A discovery will be cancelled if its range is too large, as defined by the "glide.discovery.max_range_size" property. Default value is 100,000.


3.12 Enhanced CMDB modeling of IP addresses and DNS names, and the new model is used by Discovery

The CMDB now associates any number of IP addresses with CIs, any number of DNS names with IP addresses, and can represent either IPv4 or IPv6 addresses. Discovery now discovers and records all IP addresses and DNS names for each CI.

3.13 New database index on question_answer table

A new index has been added to the question_answer table, to match the query most often done to retrieve records.


3.14 Reports can be stacked by Boolean fields

Boolean fields are now available as options when selecting a Stack column for reports.


3.15 Basic Auth can now log "Last login time"

By default, Basic Auth does not log the user's last login time. This can now be accomplished by setting the glide.basicauth.update_last_login_time property to true.

3.16 Table Rotation enabled for new instances

The Table Rotation plugin is automatically enabled for newly created instances.



4 Changes for Spring 2010 Stable 1

4.1 Performance Improvements for Project Management

Enhancements to Project Management provide significant performance improvements in the speed with which the Gantt Chart is rendered. These improvements are most noticeable for large projects.

4.2 Reduced parsing on Oracle Instances

Memory improvements on the client side cache of JDBC prepared statements have allowed us to expand the per connection cache prepared statement cache from 5 -> 50. If you have already set this property manually, the upgrade will not overwrite your values, but you change this property under System Properties -> Oracle DB Options.

4.3 Tooltips for Project Management Gantt Charts

The rendering of tooltips in Gantt Charts has been changed to fade in and fade out.

4.4 OLAP Analysis has been Deprecated

The OLAP Analysis tool has been deprecated. The OLAP module has been removed, but Existing OLAP reports will continue to function through the Berlin release.

4.5 Discovery installed software performance improvement

Performance when updating installed software has been improved during Discovery and when Help the Help Desk runs.

4.6 List/Form layout for License and Software Package tables

Improved some of the list and form layouts for License and Software Package related tables.

4.7 Specify cluster node for LDAP Server Listener

A system_id field has been added to the ldap_server_config table to specify which cluster node the LDAP Listener should run on in a multi-node configuration.

4.8 Empty Knowledge Base search no longer submitted

A Knowledge Base search with no search text will no longer submit the search. Previously the search would return all articles in an instance.

4.9 UI Action to insert UI Policy with Actions

A new UI Action has been added to the UI Policy form to copy a UI Policy with its Actions, similar to the "Insert ACL with Roles" UI Action on the ACL form.

4.10 LDAP starting search directory can be empty

It is now possible for the LDAP starting search directory to be unspecified/empty when setting up an LDAP integration.

4.11 Disable number formatting with a field attribute

Automatic number formatting (e.g., 2500 being formatted as 2,500) can be disabled on a per-field basis by adding an attribute to the field's Dictionary entry: format=none

4.12 "Lock User Session" UI Action

A new "Lock User Session" UI Action has been added to the v_user_session table, letting an admin easily terminate a user's session. The user must reauthenticate at the next attempted transaction.

4.13 Form Link UI Actions for mobile client

"Form link" UI Actions are now rendered on the mobile client

4.14 Field Styles on PDF Export lists

Color and background color Field Styles (sys_ui_style) are now used on PDF exported lists.

4.15 More searchable record numbers

If a field is "number" or "u_number", the following get text indexed separately for ease of searching, and any of them can be searched for:

  1. the actual value (e.g., "INC0014343")
  2. the digits (e.g., "0014343")
  3. the unpadded digits (e.g., "14343")

To enable searching for records last modified before this release, regenerate the text index for the applicable table.

4.16 CurrencyConverter Script Include change

A misspelled method name, getReferenceCurencyCode(), was corrected to be getReferenceCurrencyCode(). This method is called from the CurrencyConverter Script Include, so any customer who has customized that Script Include will need to manually make the change to call the correct method.

4.17 Removed BSM Map -> Saved Maps module

The unused BSM Map -> Saved Maps module has been removed. If it does not get removed from an existing instance, it may be safely deleted.

4.18 Previous is available before delete

The "previous" variable, with the current record's previous values, is now available in "before delete" Business Rules.

4.19 Global Text Search performance improved

Global Text Search result performance has been improved.

4.20 IP Address table

A new CMDB table has been created to hold IP addresses, cmdb_ci_ip_address. It extends cmdb_ci.

4.21 no_truncate for Short Description

The no_truncate attribute has been added to short_description fields to prevent their values from being truncated in lists. To reverse this change, Personalize Dictionary and remove the no_truncate attribute.

4.22 Paging controls on Knowledge Base list

Paging controls have been added to the kb_list UI Page. Previously, all records in a Topic or Category were displayed in the list, which could lead to a performance issue.

4.23 Related list on Computer form

A new defined related list (System Definition --> Relationships) has been added to the cmdb_ci_computer form to show all the IP addresses associated with a computer.

4.24 Email Client icon checks mandatory

Clicking the Email Client icon on a record now enforces mandatory fields, since it saves the record.

4.25 Large Database View performance improvement

Performance has been improved when processing large database views.

4.26 Email property to append time zone

A new property, glide.email.append.timezone (System Properties -> Email), has been added. If true, the instance's system time zone is appended to any dates or date/times in outbound emails (e.g., 2010-07-02 04:01:14 PST)

4.27 Knowledge search result sort order is a user preference

The Knowledge Base system property that controlled sort order of Knowledge search results now sets a system preference. Individual users can override this system preference with a new dropdown in the Knowledge search results page header. A user's preference is remembered for subsequent Knowledge searches. Default sort options are Relevancy and View Count.

4.28 Knowledge search operator property

A new property, glide.knowman.search.operator (System Properties -> Knowledge), can be used to specify the default operator used for Knowledge searches from the Knowledge Base or using the blue book icon (next to Short description on the Incident form, for example). Default is an AND/OR query, which means that first a search is done for results containing ALL search results. If no matches are found, then a search is done for results containing ANY of the search results.

The other option for this property is an OR query, which only does one search and simply results all results containing ANY of the search results.

4.29 DNS name support for CMDB

Added tables, defined related lists (System Definition -> Relationships), and other UI to support DNS names in the manner requested by many customers. CIs, NICs, IP addresses, and DNS names are now modeled as follows: 

CI <= NIC <= IP address <=> DNS name

The defined related lists allow one to view a related list of all IPs and DNS names belonging to a CI.

4.30 New Discovery probe for finding serial number on Solaris machines

A new probe that finds serial numbers on Solaris machines is now available. To use this probe you must install the command line tool SNEEP, which can be found on the SUN website at http://wikis.sun.com/display/sneep/Home.
