1 Control who can see emails in Activity Formatter

A new property, glide.ui.activity.email_roles, specifies who can see emails in the activity formatter. End users may not be allowed to see emails, for example, in case they contain restricted work notes or other sensitive content. If this property does not exist or is blank, all users can see emails in the activity formatter if Sent/Received Emails are selected for display.

2 Improved attachment form

The attachment form has been simplified and made more intuitive.

3 Read only HTML rendered

If an HTML field is read only, rendered HTML is now displayed instead of the HTML markup.

4 Toggle Groups and Filters icon added for Firefox

The "Toggle Groups and Filters" icon has been added for Firefox. This icon hides and displays the "Group by", filters, and the rows per page controls at the top of lists.

5 New security properties

Two new security properties have been added to System Properties --> Security. One specifies whether [code] tags allow embedding HTML code in text fields, the other provides control over whether other websites can refer to URLs in the instance.

  • Allow support for embedding HTML code by using the [code] tag
  • Allow other websites to refer to URLs in your instance (Cross Site Requests)

6 SOAP requests impersonate the System user instead of Guest

When a SOAP request comes in and defaults to the Guest user, the System user is impersonated instead and the session is not interactive.

7 Display field for sys_email

The Subject field is now the display field for sys_email records. Previously references to sys_user would show the sys_id of the record, because it had no display field specified.

8 Email Watermark table exposed to admins

A module has been added, System Mailboxes --> Watermarks, to expose the email watermark table (sys_watermark) table to admins. An email watermark links an email to its source record (Incident, Problem, etc.) so replies can be properly associated.

9 LDAP Distinguished Name comparisons are case sensitive

Previously, LDAP distinguished name (DN) comparisons were case insensitive, like all database compares. They are now case sensitive.

10 IE and Firefox label wrapping consistent

IE and Firefox label wrapping is now consistent. Previously, IE would wrap long field labels, and Firefox would not. They are now consistent, and wrapping can be specified with the following property in System Properties --> CSS.

  • Form label white-space value. "normal" wraps long text in form labels, "nowrap" prevents wrapping.

11 last_login_device and ldap_server fields not text indexed

To improve text indexing performance on instances with heavy SOAP or LDAP update activity, the last_login_device and ldap_server fields are no longer text indexed. They can still be searched using a filter if necessary.

12 Faster UI Policy loading

Forms with a large number of UI Policies will load faster.

13 Email field change cmn_notif_device

The email field on the cmn_notif_device table (Notification Device) has been lengthened to 100 characters to be consistent with the email field on the sys_user table.

14 Homepage layout now a reference

Homepage layout was a text field, now it is a reference to a UI Macro to prevent invalid values.

15 View-specific UI Policy

UI Policies can now be made view-specific, like Client Scripts. Global and View fields have been added to the UI Policy table, and can be added to the form.

16 Session Timeout message displayed

If a user's session times out, they will see a message to this effect as they are being redirected to the login page.

17 User roles flagged as inherited

User roles are flagged as inherited unless explicitly added to user. All roles inherited from groups or contained in other roles will now be flagged as inherited.

18 Dictionary information context menu option

Form fields now have a context menu option for admins that displays a UI Page popup with relevant dictionary information about the field. This can be viewed without having to leave the original form.

19 Date/time picker saves when user clicks out

Clicking outside of the date/time picker after making a change will now save the change, not cancel it. This is consistent with list editing.

20 Global text search exact task number match

If a global text search query is an exact match for a task number, the task record will be returned instead of the normal search results. If this happens, an infoMessage provides a link to the full search results. This behavior can be disabled in the "Search tips and preferences" dropdown, and the system default can be changed in the ts.match system user preference in User Administration --> User Preferences.

21 Text index engine upgraded

We now use Lucene 2.4.1 for text indexing. Numerous performance improvements have also been made.

22 System UI --> Portal Pages module removed

Portal pages can still be accessed by navigating to Homepage Admin > Pages.

23 Portal Section has a new Summary field

A summary field is now included on a portal section record when looking at a Homepage (sys_portal_page) record. An admin who brings up a Homepage record (Homepage Admin --> Pages) can now see what is on the homepage by looking at the portal sections in the related list at the bottom instead of having to open each one to read the title.
