1 Overview

 Note: If the Knowledge Base Internationalization Plugin is installed and the instance is still using Knowledge Management v1, upgrading to Spring 2010 Release Notes will automatically install the Knowledge Management with KCS plugin.


The Knowledge Management Internationalization Plugin allows the creation of knowledge base articles in multiple languages. If an internationalization plugin (such as the Internationalization - German Plugin) is already installed, then the Knowledge Base Internationalization Plugin will add a choice-list field on the Knowledge Base Article form. This will allow an administrator to create multiple language versions of the same page, and to filter so that only articles in a user's language will appear in the Knowledge Base.

Note that this method creates different knowledge base articles for different languages. To translate the same article into different languages, use Translated Fields.

2 Included Changes

2.1 Database Table Structure

The only change to the database structure is to add a choice-list field on the Knowledge Base Article form. This field will automatically populate upon installation with already installed languages, from the Internationalization Plugins. If none are installed, the only choice will be "English."

3 Dependancies

The Knowledge Management Internationalization Plugin does not require any other plugins. It is, however, of limited use unless at least one Internationalization Plugin is installed, such as Internationalization - German.

4 Getting Started

4.1 Installing the Plugin

The Knowledge Management Internationalization Plugin can be installed by making a request to ServiceNow Technical Support using the Request Plugin Activation module from /hi.

4.2 Filtering the Knowledge Management Search by Language

Out-of-box, the Knowledge Management Internationalization Plugin allows pages to be added in different languages. When a user searches for a given term, however, the search will apply to all languages. To filter the Knowledge Base search based on the user's language, create a before business rule on the kb_knowledge table using the following script: 

if (!gs.hasRole("itil") && gs.getSession().isInteractive()) {
   var p = current.addQuery("language", gs.getUser().getLanguage() );
   gs.print("Language query for language:" + p);
}
