1 Overview

ServiceNow offers the ability to configure and provision VMware vCenter or Amazon EC2 virtual machines (VM) directly from the ServiceNow interface starting with the Calgary release. Use this guide to manually provision a VMware virtual machine. To provision an Amazon EC2 instance, see the EC2 Quick Start guide.

To provision a VMware virtual machine through ServiceNow:

  • Satisfy all prerequisites
  • Configure ServiceNow to provision virtual machines
  • Provision a virtual machine


This guide provides an condensed set of instructions to help you provision virtual machines. For more detailed information regarding cloud provisioning, ServiceNow offers these additional resources:

  • Cloud provisioning documentation
  • VMware provisioning video tutorial
  • Automatic provisioning video tutorial

2 Prerequisites

2.1 ESX Server

To provision VMware virtual machines, you must have access to a VMware ESX Server with vCenter installed. You need the IP address and administrator credentials for this server. Contact your vCenter administrator for more information.

2.2 Create vCenter Templates

Create the Windows and Linux templates within vCenter or use existing templates. ServiceNow uses these templates to create virtual machines. Refer to VMware product documentation for vCenter and the ESX Server for details about creating templates.

2.3 Activate the Plugin

VMware virtual provisioning requires Orchestration, which is available as a separate subscription from the rest of the ServiceNow platform.

View the procedure for determining whether your instance has Orchestration
  1. Navigate to Workflow > Activity Definitions.
  2. Locate the Run Probe activity.
    If this activity is present on your instance, Orchestration is installed. If this activity is not present, contact your ServiceNow account representative.

2.4 Create MID User

If your instance does not have a MID Server user, navigate to User Administration > Users as an administrator and create a MID User record. Give this user the mid_server role.

View the procedure for creating a MID server user
  1. Click New.
  2. In the User ID field enter MID User.
  3. Click Update.
  4. Reopen the new user record.
  5. In the Roles related list, click Edit.
  6. Move mid_server to the Roles list..
  7. Click Save.

2.5 Install the MID Server

Install a MID Server on the same network as the ESX Server. This guide provides instructions for setting up a MID Server on a Microsoft Windows using a basic configuration. For more advanced options, or instructions for installing a MID Server on Linux, see MID Server Installation.

View the procedure for installing a basic MID server on Windows

2.5.1 Download the MID Server software.

  1. Navigate to MID Server > Downloads.
  2. Select Windows (x86-32).
  3. Create a new MID Server directory on the Windows machine.
  4. Extract the downloaded archive to the MID Server directory.


2.5.2 Configure the MID Server to communicate with your ServiceNow instance.

  1. Navigate to the /agent directory within the MID Server directory.
  2. Edit the config.xml file using a text editor such as WordPad:
    1. Find the element <parameter name="url" value="https://YOUR_INSTANCE.service-now.com" /> and change the value to the URL of your instance.
    2. Enter the user credentials if basic authentication is enabled (default behavior) in the mid.instance.username/password elements. Use the credentials for the MID User.
    3. Find the element <parameter name="name" value="YOUR_MIDSERVER_NAME_GOES_HERE" /> and change the value to the name of your MID Server.
  3. Save the config.xml file.
  4. Run the start.bat file.

2.5.3 Install the MID Server as a Windows service.

For Windows XP, Windows 2000 Server, or Windows Server 2003:

  1. Open the \agent folder in the directory you created for the MID Server installation files. For example, the path might be C:\ServiceNow\MID Server1\agent.
  2. Double-click the start.bat file to install the Windows service.

For Vista, Windows 2008 Server, or Windows 7:

  1. Click the Start button.
  2. In the search box, enter command prompt or cmd.exe.
  3. In the result list, right-click Command Prompt or cmd.exe, and then click Run as administrator.
    This enables the MID Server to be installed with administrative rights under any Windows User Account Control (UAC) setting.
  4. In the command prompt, navigate to \agent in the directory you created for the MID Server files. For example, the path might be C:\ServiceNow\MID Server1\agent.
  5. Run start.bat.

2.5.4 Modify the MID Server credentials to run with elevated privileges

  1. Open the Windows Services console.
  2. Double-click on the ServiceNow <MID Server name> service.
  3. In the Properties dialog box, select the Log On tab.
  4. Set Log on as privileges with Domain User or Local Admin credentials.
  5. In the General tab, set Startup type to Automatic.
  6. Click OK.
  7. Restart the ServiceNow <MID Server name> service and make sure that the ServiceNow\<MID Server name>\agent\logs\agent0.log does not have error messages.

2.5.5 Confirm connectivity to your ServiceNow instance.

  1. On the instance this MID Server is connected to, navigate to MID Server > Servers. If Discovery is installed, navigate to Discovery > MID Servers.
    All MID Servers connected to this instance are listed.
  2. Make sure that the Status of the MID Server you just installed is Up.


2.5.6 Configure the MID Server to access your ESX Server.

  1. Navigate to Orchestration > MID Server Properties.
  2. Set the Default MID Server to use for Orchestration Activities to the new MID Server.
  3. Navigate to MID Server > IP Ranges.
  4. Click New.
  5. Enter a Name for the IP range. Enter the IP Range that contains your ESX Server.
  6. Click Submit.
  7. Return to the list of MID Servers.
  8. Select the MID Server you just installed.
  9. In the IP Ranges related list, select the new IP range.

2.6 Create Cloud Provisioning Users

Create these users and assign them to the designated groups:

User ID Group
Cloud User Virtual Provisioning Cloud Users
Cloud Admin Virtual Provisioning Cloud Administrators
Cloud Operator VMware Operators
VMware Approver VMware Approvers
View the procedure for creating new users
  1. Navigate to Users and Groups > Users.
  2. Click New.
  3. Enter a User ID for the user. Refer to the table for all required users.
  4. Click Submit.
  5. Open the new user record.
  6. In the Groups related list, click Edit.
  7. Select the group for that user. Refer to the table for group assignments by user.
  8. Click Save.

3 Set up VMware Provisioning

A cloud administrator can configure ServiceNow to provision VMware virtual machines using the vCenter application.

3.1 Discover vCenter

Navigate to VMware > vCenter Instances and discover vCenter and the templates it contains. Several templates are available by default with cloud provisioning. Discovering vCenter makes the templates available for configuration as catalog items. To create additional virtual machine templates, contact your vCenter administrator. See Gathering vCenter Data Without Running Discovery for more information.

View the procedure for discovering vCenter
  1. Click New.
  2. Enter the URL or IP address of your vCenter instance in the URL field.
  3. Fill in the form, as needed.
  4. Right-click the form header and select Save.
  5. Click Discover vCenter Details.
  6. Wait for the system to discover the detailed information. If your user record has an Email address, ServiceNow sends an email notification when the discovery finishes.
Discover vCenter

3.2 Link Catalog Offerings to Templates

Navigate to VMware > Configuration > Templates and link a catalog offering to a template record. Catalog offerings are operating systems that are available for the requested virtual machine. ServiceNow provides a Windows and Linux catalog offering by default. Assigning a catalog offering to a template links the operating system selection that a user makes in the service catalog with the proper template. For more information, see VMware Templates and Offerings.

View the procedure for linking a catalog offering to a template
  1. Select a virtual machine template record.
  2. Enter or select an available offering in the Catalog offering field.
  3. Click Update.
Link an offering to a template

3.3 Create Catalog Items from Templates

Navigate to VMware > Configuration >Templates and create a catalog item from a template record. Users can request the completed catalog item from the service catalog.

View the procedure for creating a catalog item from a template
  1. Select a template.
  2. Click Create Catalog Item.
  3. In the Task automation field, select Manual.
  4. Complete the catalog item form, as appropriate.
  5. Save the catalog item record.
Create a catalog item from a template

4 Provision a Virtual Machine

After configuring the VMware offerings, a cloud user can request a virtual machine from the service catalog. If using manual approvals, a VMware approver must approve the request. After the catalog request is approved, ServiceNow creates a new catalog task. A cloud operator can complete the catalog task to provision the virtual machine.

4.1 Request a Virtual Machine

As a cloud user, avigate to Service Catalog > Catalog and order a virtual machine.

View the procedure for requesting a virtual machine from the service catalog
  1. Select the newly created VMware catalog item.
  2. Enter a Business purpose for this virtual machine.
  3. Click Order Now.
Request a virtual machine

4.2 Approve the Request

As a VMware approver, navigate to Service Desk > My Approvals and approve the virtual machine request.

View the procedure for approving the virtual machine request
  1. Select the approval record for the virtual machine request item.
  2. Click Approve.
Approve the requestâ

4.3 Complete the Provision Task

As a cloud operator, navigate to Service Desk > My Groups Work and complete the catalog task to provision the virtual machine.

View the procedure for provisioning a virtual machine from a catalog task
  1. Select the open catalog task record for the virtual machine request.
  2. Complete all mandatory fields as appropriate for the virtual machine. For more information about this form, see Provisioning Virtual Servers.
  3. Select No from the Guest customization choice list.
  4. Click Close Task.
