The following are notable changes you may encounter with our Fall 2010 Stable 3 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before. For additional changes, see Fall 2010 Stable 3 Fixes.

1 Changes for Fall 2010 Stable 3

1.1 New Quick Filters for Date Time Fields

Show Before and Show After quick filters to easily filter on date/date-time fields

1.2 Time Worked

New property com.snc.time_worked.update_task_timer to enable updating of the task timer value based on changes to the task time worked records. This is accomplished through the Update task timer business rule.

1.3 Time Cards

  • View the list of time worked records by adding a related list to the time card form
  • New property com.snc.time_card.start_day to define the first day of time card
  • New module My Time Cards -> Current to view and manage all of your time cards for the current week

1.4 Planned Task Notification Events

All planned task type leverage a single business rule, planned task global events to fire notification events for insert, updated, commented, assigned, inactive. Includes events for field service, project v2 and release v2 tables

1.5 Project Portfolio Management

  • New Phase field added to Project and Project Portfolio tables with default choice options of Initiating, Planning, Executing, Monitoring/Controlling, Closing
  • New Project Manager field added to Project and Project Portfolio tables
  • New project report Active Project Unassigned Tasks 30 Days
  • New project report Active Projects by Manager
  • Plugin now includes the Process Flow Formatter Plugin. The default formatter can be added to the project portfolio form to display a graphical representation of the project stage
  • New related lists available on Project form to display Tasks Unassigned and Task without Assignment Groups

1.6 Cost Management

  • CI Cost test mode - By adding a property glide.cost_mgmt.test_mode with a value of true, enables actions for testing CI expense generation. Two new UI Actions TEST: Process (all dates) and TEST: Delete All Related Expenses will be available to admins on Distribution Cost and CI Rate Cards forms. They allow for bulk generation and deleting of expense lines for the cost source. These are for testing only and should not be used in a production environment.

1.7 Skills Management

The skills module is now only available for user_admin and admin roles

1.8 Plugin activation displays dependencies

When activating plugins the list of dependent plugins and their install status are displayed.

1.9 Runbook Automation VMware Support

Runbook Automation VMware Support now provisions VMware requests using Workflows instead of Execution Plans.

1.10 First Day of Week in Date/Time Picker

A new property, glide.ui.date_picker.first_day_of_week, can be created to specify the first (leftmost) day of the week for the date and date/time picker (1=Sunday, 2=Monday...). The example shown uses a value of 3 (Tuesday).

1.11 XML Output Sorts Fields

Fields in XML output for serialized GlideRecords (e.g., Payload in the sys_update_xml table) are now sorted alphabetically to make it easier to find fields when viewing the XML.

1.12 Field Style Preview in Field Styles List

When viewing the Field Styles list in System UI, the Style field is automatically styled with itself for an easy preview of the style.

1.13 Workflow Field Type Behavior in Lists

Workflow fields in a list (e.g., kb_knowledge.workflow) now behave differently. The expander next to a single item now expands/contracts just for that row instead of for all subsequent rows. A new list context menu option provides the ability to expand/collapse all.

1.14 Application Secondary Sort on Title

Two applications with the same order now subsorted by Title. Previously only unordered applications were subsorted by Title.

1.15 Disable Choice List Options (removed in Winter 2011 release due to browser incompatibility)

Two new GlideForm methods, disableOption(field, value) and enableOption(field, value), similar to addOption and removeOption, allow disabling/enabling of choice list options. Example: g_form.disableOption("category", "software")

1.16 Dashboard Gauge Change Link Role Change

On legacy dashboards, the report_admin role (vs. admin) is required for the "change" link of a gauge.

1.17 Clicking Text Selects Checkboxes and Radio Options

To select checkboxes and radio buttons in Service Catalog, it is now possible to click on the question text to make the selection. This functionality does not apply to field labels.

1.18 Approval Summarizer for Requested Item Table

An approval summarizer UI Macro has been created for the sc_req_item (Requested Item) table. If an instance already has a summarizer with the same name, the out-of-box one is renamed so as not to conflict.

1.19 Knowledge Base Performance Improvements

Performance has been improved for Knowledge Bases with a large number (thousands) of articles and large search result sets.

1.20 TableUtils.isValidField()

A new method, isValidField(fieldName), has been added to the TableUtils Script Include. It returns true if the fieldName parameter is a valid field in the table.

1.21 sys_ui_macro.name Field Length Increased to 100

The sys_ui_macro.name field length has been increased from 40 to 100.

1.22 Service Portfolio Display Templates Moved to Database

The hardcoded XML templates for Service Portfolio's rendering of Business Services have been converted to UI Macros in the database, where they can be customized if necessary.

1.23 Viewing Record History

The History form context menu option has been enhanced to provide two viewing modes. History -> Calendar provides the standard history view. History -> List navigates to the new History Set for the record.

1.24 Duration Field List Display

A new max_unit Dictionary attribute for glide_duration fields is available, specifying the largest time unit used when displaying the duration in a list. e.g., if max_unit=minutes, a duration of 3 hours 5 minutes 15 seconds will display as 185 minutes 15 seconds. If max_unit=hours, a duration of 4 days 2 hours 10 minutes will display as 98 hours 10 minutes. This attribute does not affect the display of duration fields in the form.

1.25 Service Portfolio Availability Gauge Enhancement

For Service Portfolio availability gauges, the "Did not meet commitment" tooltip for a success/failure indicator has been augmented with the amount of downtime experienced. e.g., "Did not meet commitment (4 hours 13 minutes downtime)"

1.26 GlideElement Method Name Typo Corrected

The GlideElement method name getDynamiCreationcScript() had been corrected to getDymanicCreationScript()

1.27 Knowledge Base Search Results Information

In Knowledge Base search results, published and modified dates are now displayed for each article.

1.28 Additional Knowledge Base Search Result Sorting Option

"Last Modified" (sys_updated_on) has been added as a third option for sorting Knowledge Base search results. Users can select their sorting preference in the upper right corner of the search results page.

1.29 Windows 7 Phone Uses Mobile UI

Windows Phone 7 devices will get the mobile UI.

1.30 List v2 UI Action Condition Checks

List v2 now performs the same condition checks for UI Actions that are performed in List V1. If a UI Action has a name that starts with sysverb_, then the action can be run without selecting any records in the list - If the UI Action has a condition that contains 'current', the condition is not evaluated until after the action is clicked (by making an AJAX call to the server) - If the UI Action does not contain 'current' in the condition, the condition is evaluated on the server when building the list, so this controls the visibility of the action.

1.31 task_rel_type Record sys_id Changed from 1 to T1

The task_rel_type record with a sys_id of 1 has had its sys_id changed to T1 to eliminate potential conflict with a cmdb_rel_type record with the same sys_id. All references to the old sys_id have been changed to reference the new sys_id with a fix script run during upgrade.

1.32 Send Multiple Reports in a Scheduled Report Email

New ability to send multiple reports in the same Scheduled Report email. Added "Include with" reference field to Scheduled Report form to specify another Scheduled Report. When the referenced Scheduled Report is executed, the current report is included in the email. "Omit if No Records" checkbox is honored per report. Conditional and Condition Script fields are only evaluated for the report being scheduled, not for any included reports. Also added "On Demand" run type so a Scheduled Report can be set to not send on its own, but still be included in another Scheduled Report's email. Added "Included in Email" related list to Scheduled Report form

1.33 Survey Response Answer field length increase

The length of the survey_response.answer field has been increased from 40 to 4000 to match the survey_response.response field. This is to prevent truncation of data when copying from the Response to the Answer field. As a result, the field can no longer be grouped by in a list.
