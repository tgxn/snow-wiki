Event Management

1 Overview

The Event Processing Statistics module displays statistics on event management performance. These statistics, such as average and maximum processing time for events, help event management users monitor and troubleshoot event management efficiency.

The Event Processing Statistics module is a type of homepage. For details about editing gauges on homepages, see Adding Existing Gauges to a Homepage.

Users with the evt_mgmt_admin and evt_mgmt_user roles can view and customize the Event Processing Statistics module.

2 Viewing Event Processing Statistics

The Event Management > Event Processing Statistics module allows you to track processing statistics (see table) for each batch run of events. All time calculations are based on Coordinated Universal Time (UTC).

Event Processing Statistics

Field Description
Start time The process start time for the event processing batch.
Number of events The number of events processed in the batch.
Elapsed time (ms) The elapsed time for the batch.
Avg processing time (s) The average processing time for all of the events in the batch.
Max processing time (s) The processing time for the slowest event in the batch.
Avg insertion time (s) The average time interval between when events are reported by the monitoring tool and when they are created in the Event [em_event] table for all of the events in the batch.
Max insertion time (s) The insertion time for the slowest event in the batch.
