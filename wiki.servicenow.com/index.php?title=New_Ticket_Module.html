1 Overview

New Ticket is a Service Desk module that enables users to collect information and create a generic task record before identifying the issue as an incident, problem or change request. The New Ticket module is installed with the Domain Support - MSP Extensions plugin. Any custom fields added to the Ticket form are not retained once you select the task type, unless the same fields exist on the target table.

2 Using the New Ticket Module

  1. Navigate to Service Desk > New Ticket.
  2. Click New.


  3. Fill in the fields using the descriptions from the table.
  4. Click Submit.
Field Description
Number An identifying number for the ticket. Numbers are generated automatically when the ticket is created.
Task type A field which specifies the type of task. Leave the type as Ticket until you know if the task should be an Incident, Problem, or Change Request. The task Class is changed when you select a type other than Ticket and update the record.
Company The company where the ticket originated.
Task for Common field that is distributed to the correct equivalent field when a Task type (Incident, Problem, or Change Request) is selected.
Configuration item The configuration item associated with the ticket.
Priority A choice list which signals how high a priority the ticket should be for the assignee. The value is calculated based on Impact and Urgency, by the business rule calculatePriority.
Opened The date and time when the ticket record is opened by a user for the first time.
State A choice list for status of the ticket:
  • Pending
  • Open
  • Work in Progress
  • Closed Complete
  • Closed Incomplete
  • Closed Skipped
Assignment group A group assigned to assess the ticket.
Assigned To Designates a user to complete the ticket.
Watch list A list of users who will receive email notifications when the ticket is updated. For more information, see Configuring Watch Lists.

Note: The watch list only receives notifications if email notifications are defined.

Short Description A short description of the ticket.
Additional comments Additional information about the ticket that might be helpful for others to know. This is a customer-visible field.
Work notes Information about how the ticket is being resolved.
