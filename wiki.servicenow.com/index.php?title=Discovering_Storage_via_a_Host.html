1 Overview

Discovery can find information about direct-attached storage (DAS), network-attached storage (NAS) and storage area network (SAN) storage units that connect to a host via a local I/O port or host bus adapter (HBA). Discovery collects and creates CIs in the CMDB for the following information, starting with the Fuji release:

  • File systems (local and NAS).
  • Disks (both SAN disks and DAS drives).
  • Fibre Channel (FC) HBAs and ports.
  • Linux Volume Manager (LVM) volumes. LVM volume data resides in the Storage Pool [cmdb_ci_storage_pool] table.

2 Requirements for Discovery

Discovering storage via a host requires the MID Server to have the necessary credentials to the host server.

2.1 Windows

  • The following Windows host configurations are supported:
    • Windows Server 2008, DAS or NAS with FC.
  • For Windows 2008 servers that attach to storage via FC, the Microsoft fcinfo.exe tool is required.
  • The host has Windows Management Instrumentation (WMI).
  • The MID Server has PowerShell enabled.

2.2 UNIX or Linux

  • The following UNIX or Linux host configurations are supported:
    • CentOS, DAS, NAS, or SAN with FC
    • Ubuntu Server, DAS, NAS, or SAN
  • The device has SSH credentials with root or sudo access

3 Probes and Sensors

  • KVM - Storage Pools: identifies storage attached to KVM virtual machines.
  • Linux - Storage: identifies storage attached to systems running the Linux operating system.
  • Solaris - Storage: identifies storage attached to systems running the Solaris operating system.
  • Windows - Storage 2008: identifies storage attached to systems running Windows 2000, 2003, or 2008.
    • Windows - Storage 2008 - PS: identifies storage attached to systems running Windows 2000, 2003, or 2008, using the PowerShell.
    • Windows - Storage 2008 - WMI: identifies storage attached to systems running Windows 2000, 2003, and 2008, using WMI Runner.
  • Windows - Storage 2012: identifies storage attached to systems running Windows 2012 and later.
  • VMWare - vCenter: collects information about ESX servers, including datastore information.

3.1 Configuring Discovery for Windows Hosts

To configure Discovery for Windows hosts:

  1. For Windows 2008 servers that attach to storage via FC. For servers that are using newer operating systems, similar functionality is present with the operating system; this step is not required:
    a.) Download and install the Microsoft Fibre Channel Information Tool from http://www.microsoft.com. Search the website for Microsoft Fibre Channel Information Tool.
    b.) Set the environment path for the fcinfo executable and run the 'fcinfo.exe' file.
  2. For all Windows hosts including Windows 2008, add Windows credentials to the Discovery Credentials table.
  3. On the ServiceNow instance, create a Discovery Schedule for each host IP address.
  4. (Optional) Create a behavior that uses a functionality definition with a wbem port probe to make the initial port-scanning phase (Shazzam) more efficient.
  5. Run network discovery.
 Note: If the host also connects to a NAS or SAN storage array, set up the SMI-S Provider and CIM credentials.


3.2 Configuring Discovery for UNIX or Linux Hosts

To configure Discovery for UNIX or Linux hosts:

  1. (Optional) On the host, assign elevated privileges to the account that Discovery will use.
  2. On the ServiceNow instance, add credentials to the Discovery Credentials table.
  3. Create a Discovery Schedule for each host IP address.
  4. (Optional) Create a behavior that uses a functionality definition with a wbem port probe to make the initial port-scanning phase (Shazzam) more efficient.
  5. Run network discovery.
 Note: If the host also connects to a NAS or SAN storage array, set up the SMI-S Provider and CIM credentials.


4 Data Collected

Discovery uses the following tables and probes to gather information about storage units that connect to a host via a local I/O port or Host Bus Adapter (HBA).

Table Label and Name Probes
Disk [cmdb_ci_disk]
  • KVM
  • Linux
  • Solaris
  • Windows 2008 and 2012
Disk Partition [cmdb_ci_disk_partition]
  • KVM
  • Linux
  • Solaris
  • Windows 2008 and 2012
Fibre Channel Disk [cmdb_ci_fc_disk]
  • Linux
  • Windows 2008
Fibre Channel Port [cmdb_ci_fc_port]
  • Linux
  • Windows 2008
File System [cmdb_ci_file_system]
  • KVM
  • Linux
  • Solaris
  • Windows 2008 and 2012
NAS File System [cmdb_ci_nas_file_system]
  • KVM
  • Linux
  • Solaris
  • Windows 2008 and 2012
Storage HBA [cmdb_ci_storage_hba]
  • Linux
  • Windows 2008
Storage Pool [cmdb_ci_storage_pool] Linux
Storage Pool Member [cmdb_ci_storage_pool_member] Linux
VMware vCenter Datastore [cmdb_ci_vcenter_datastore] VMware vCenter

5 Storage References

Discovery maps the following relationships for storage units that connect to a host via a local I/O port or Host Bus Adapter (HBA).

Table Name.Column Relationship Target Data Element
cmdb_ci_storage_device.computer refers to the cmdb_ci_computer
cmdb_ci_storage_device.provided_by refers to the cmdb_ci_fc_port (for FC only)
cmdb_ci_disk_partition.disk is a partition of cmdb_ci_disk
cmdb_ci_storage_hba.computer is the cmdb_ci_computer
cmdb_ci_fc_port.controller contains the cmdb_ci_storage_hba
cmdb_ci_fc_port.computer is the same cmdb_ci_computer as cmdb_ci_storage_hba.computer
cmdb_ci_storage_volume.computer is the cmdb_ci_computer
cmdb_ci_storage_volume.provided_by is the cmdb_ci_storage_pool or cmdb_ci_storage_pool or cmdb_ci_storage_device (providing storage)
cmdb_ci_storage_pool.hosted_by is the cmdb_ci_computer
cmdb_ci_computer the pool is on cmdb_ci_storage_pool.container
cmdb_ci_storage_pool.container is the cmdb_ci_storage_pool or cmdb_ci_storage_pool_member containing the pool (if the pool is present)
cmdb_ci_storage_pool_member.pool is the cmdb_ci_storage_pool
cmdb_ci_storge_pool_member.storage is the cmdb_ci_storage_pool, cmdb_ci_disk_partition or cmdb_ci_storage_device providing storage
