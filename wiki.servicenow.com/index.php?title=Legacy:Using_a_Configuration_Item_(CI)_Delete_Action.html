 Functionality described here is obsolete since the Winter 2010 release and has been replaced by Discovery Identifiers.
Warning: Do not install this plugin. It interferes with the current functionality.

1 Overview

Delete actions do exactly what you'd think: they delete CIs – either the CI just discovered (the “source”) or the CI that's a duplicate of the source (the “target”). You specify which CI to delete when you create the delete action.

2 Delete Action Form

When you create or modify a delete action, you'll see a form like the one below:



The fields on this form:

  • What to delete: This can be either Source or Target, to delete to the source CI or the target CI, respectively.  If you delete the source CI, the target CI becomes the source CI (for the purposes of any subsequent actions in this consilidator, and for the purpose of re-running the finders).  This little trick ensures that if you are merging source-to-target, then deleting the source, the finders will still search for duplicates of the remaining CI.
  • Order: An integer value that controls the order in which consolidator actions run.  Actions with smaller order values run earlier, actions with larger order values run later.
  • Active: If checked, this delete action will run when the consolidator runs.  If not checked, the delete action will not run.
  • Description:; This is here for your benefit. The delete action makes no use of the description.
