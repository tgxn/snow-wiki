1 Spring 2010 Stable 3 Fixes


1.1 PRB080092 (GlideRecord): Undefined value in GlideRecord queries

GlideRecord now handles a query on an undefined value or a nonexistent column by throwing an error instead of returning all records.


1.2 PRB427616 (List v2): Copy URL from breadcrumb

The right-click > Copy URL function for breadcrumbs in List v2 now generates the correct URL.


1.3 PRB419318 (Templates): onChange client scripts affected by templates

A new argument isTemplate has been added to onChange client scripts to indicate if the call is being made based on the template being applied. It is used as follows: 

function onChange(control, oldValue, newValue, isLoading, isTemplate) { 
   if (isTemplate)
     alert("Field change is due to template being applied, not user action");
 }


1.4 PRB373595 (Web Services): SOAP messages with special characters

SOAP messages containing special characters now use the correct UTF-8 character in the generated XML.


1.5 PRB069565 (SLA): SLA calculations of percentage

SLA business rules use a new SLA Calculator script include with improved logic for setting the percentage field.


1.6 PRB049586 (SLA): SLA - update calculations scheduled job

To improve performance, SLA - update calculations scheduled job has been split into 6 scheduled jobs.


1.7 PRB402679 (SLA): SLA calculations of elapsed time

SLA business rules and scheduled jobs use a new SLA Calculator script include with improved logic for calculating elapsed time outside of business hours as defined by Schedule Entries.


1.8 PRB308266 (Service Catalog): Checkout rendering in IE6

For users of IE6, order guide tabs no longer blink during checkout.


1.9 PRB422148 (Templates): References with derived fields

When a template contains a reference with derived fields and the reference value is set on a record with no data in the derived fields, the record is no longer updated with data from the form before the template is applied.


1.10 PRB317772 (List v2): Print button in database view

In List v2, the Print button in database view now works properly.


1.11 PRB420571 (List v2): CAB Report

The CAB Report now renders properly in List v2.


1.12 PRB360299 (List v2): Personalizing a form

You can now personalize a form for a table that is listed in a sys_m2m record for a many-to-many table that does not exist.


1.13 PRB418003 (List v2): Parent variable in List Control scripts

In List v2, the parent global variable is now correctly populated in List Control scripts.


1.14 PRB420790 (Reports): Pivot tables with long dot-walked fields

Pivot table reports now support dot-walked field names up to 255 characters long.


1.15 PRB381656 (Service Catalog): Catalog UI Policies on multiple variable sets

When using multiple variable sets, the Catalog UI Policies for each set are now properly applied.


1.16 PRB419575 (Mobile): Form views for related lists

When using a mobile client, records on related lists now open in the mobile form view.


1.17 PRB419270, PRB419555 (Filters): Personal and Group filters

Personal and Group list filters are now created properly, instead of as Global filters.


1.18 PRB381947 (i18n): Global Text Search label

The Global Text Search label now resets to the proper i18n language string when a user clicks out of the input field.


1.19 PRB197237 (UI): Related lists in the sys_popup view of a form

Since related lists are not rendered in the sys_popup view of a form, this configuration option is no longer available.


1.20 PRB415518 (UI): Reference field pop-up

Clicking a reference while the reference pop-up is visible no longer generates a JavaScript error when the reference is to the same table (the Parent field on the Group form).


1.21 PRB408566 (Reports): sys_id in list reports

The sys_id field is no longer added automatically to list reports.


1.22 PRB406859 (Reports): Grouped list with ampersand in label

In a grouped list report, a group with an ampersand in the label can now be expanded.


1.23 PRB375233 (UI): Welcome page content

Welcome page content now appears properly when the Guest user record is deleted or has a different language preference than sys_home records. English language is now used when no value is specified for a user's language preference.


1.24 PRB407678 (Discovery): Network printer with invalid MAC address

A network printer with an invalid MAC address string no longer generates an error.


1.25 PRB408612 (Discovery): Process classifiers that match Any

Discovery Process Classifiers that use Any as the match criterion no longer create multiple records.


1.26 PRB404186 (UI): Attachments exceeding maximum size

An error message now appears when users attempt to upload an attachment that exceeds the maximum file size.


1.27 PRB408519 (Workflow): Export to Excel

Workflow stages now export properly to Excel.


1.28 PRB408645 (Login): Logging remote IP address events

Remote IP address is now logged as Parm2 for logout and login.failed events.


1.29 PRB392491 (Knowledge Base): Search operator property

The Knowledge Base search operator property (Knowledge Base > Properties) is now properly applied for direct knowledge searches.


1.30 PRB294376 (UI): HTML field links in IE7

In IE7, HTML field links are now displayed in the correct style, instead of plain text.


1.31 PRB405199 (UI): Long labels in PDF exports and List v2

Long labels (long_label field attribute) are now displayed correctly in PDF exports or in List v2.


1.32 PRB395769 (Service Catalog): Special Instructions and Delivery Address on wizard checkouts

When using a wizard to order an item, Special Instructions and Delivery Address now populate the resulting request.


1.33 PRB395398 (Schedules): Monthly, day of the week schedule entries

A schedule entry that repeats Monthly on the selected Day of the week now correctly completes the transaction.


1.34 PRB401134 (Mobile): List field labels

Labels for mobile list fields now appear as configured. To configure labels, modify the Mobile list view and use the Add With Label option.


1.35 PRB387202 (Workflow): Prevent deletion of active workflows

Workflow versions that are used by active workflow contexts can no longer be deleted.


1.36 PRB394029 (Database Views): Reports on database views generate links

Reports on a database view now generate links to records in the first field of the table.


1.37 PRB397089 (Workflow): Prevent editing of published workflows

Published workflows can no longer be edited.


1.38 PRB391247 (i18n): Translated HTML fields

Translated HTML fields now function properly.


1.39 PRB396496 (Roles): Business Rule queries on sys_user_role

Custom before query Business Rules on sys_user_role no longer prevent the system from reading roles, which can create duplicates.


1.40 PRB387518 (Domain): Catalog client scripts execute by domain

The property that determines which catalog client scripts should execute based on domain now functions properly.


1.41 PRB395800 (Scripting): gs.addYears method

The gs.addYears() method now returns correct results for both DATE/TIME and DATE values.


1.42 PRB297474 (Service Catalog): UI Policy with containers

Catalog UI Policy now works properly with containers on the catalog item or variable set.


1.43 PRB361300 (Reports): Editing Once or Periodic scheduled reports

Editing a scheduled report of run_type Once or Periodic no longer immediately sends the report.


1.44 PRB391575 (Roles): Single quotes in role names

Single quote characters in role names no longer generate a client-side JavaScript error.


1.45 PRB363993 (Schedules): All=day schedule entries

Time zones are no longer applied to all-day schedule entries.


1.46 PRB042857 (UI): Mandatory field alert

The mandatory field alert now lists labels, instead of field names, and the required field flashes.


1.47 PRB281120 (UI): Context menus with export or attachments functions

Context menus now work properly after exporting a list or form or viewing an attachment.


1.48 PRB391519 (Notifications): Approval request email notification

The default Approval Request email notification now includes an approval table, instead of an incident table.


1.49 PRB067249 (Service Catalog): Editing wizard panel variables

The order of wizard panel variables is now maintained after editing the wizard.


1.50 PRB363644 (Service Catalog): Hide Label variables with client scripts and UI policies

Catalog client scripts and UI policies can now hide a Label catalog variable.


1.51 PRB359811 (Searching): Tilde in text search

Tilde (~) character functions now work properly in text searches.


1.52 PRB385844 (Service Catalog): Category names with apostrophes

Service Catalog Category names can now have an apostrophes.


1.53 PRB388944 (UI): Field hints show field name

Field hints no longer show the field name.


1.54 PRB375889 (Database): Rotation tables as related list options

Rotation tables are no longer included as related list options.


1.55 PRB375285 (Oracle): Create unique index on field

You can now create a unique index on a field for which an index already exists.


1.56 PRB372810 (Choice Lists): Prevent trailing white space

Trailing white space is no longer allowed in a choice list value.


1.57 PRB253657 (Service Catalog): Variables in order guides

Service catalog item variables now display correctly when users do not visit all items in an order guide.


1.58 PRB042364 (UI): Page navigation arrows in list view

The list navigation (blue) arrows now function properly.


2 Spring 2010 Stable 2 Update 08-13-10


2.1 PRB392607 (UI): Dialog windows with automatic resizing

Windows with automatic resizing enabled now render the full dialog instead of a small square graphic.


2.2 PRB396905 (UI): List optimization

List optimization now prevents additional queries when rendering lists with dot walked fields.


3 Spring 2010 Stable 2 Update 08-23-10


3.1 PRB415897 (Domain): Instance unavailable

Instances remain available during periods of intense cache flush activity because a conflict between domains and database views is resolved.


3.2 PRB407365 (UI): On-Call calendar/showQuickForm UI Action

The showQuickForm UI Action now works properly. In the On-Call calendar, double-clicking an individual now opens the Add Item dialog.
