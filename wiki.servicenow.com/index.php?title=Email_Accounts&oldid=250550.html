1 Overview

The Email Accounts plugin enables an administrator to add email accounts (outgoing and incoming) with a simple form instead of configuring accounts through email properties (System Properties > Email). Supported accounts are SMTP, POP3, and IMAP. For instructions on creating and sending custom emails when system events occur, see the tasks in Email Notifications.

2 How Email Accounts Work

When the Email Accounts plugin is activated, a script runs that converts existing account information from email properties into account records in the new Email Account [sys_email_account] table. This script creates an outgoing SMTP account and an incoming POP3 account in the new table. After the Email Accounts plugin is activated, the existing email properties for account configuration are not available. The properties removed by the Email Accounts plugin include Enable email sending (SMTP) and Enable email receiving (POP3). After the Email Accounts plugin is activated, email processing is controlled by the system properties glide.email.read.active and glide.email.smtp.active. If the Email - OAuth support for IMAP and SMTP plugin is activated, two additional sample accounts are created that demonstrate how to connect to GoogleMail via IMAP and SMTP using Open Authentication (OAuth). OAuth is supported but not required for IMAP configurations.

2.1 Limitations

  • There can be only one outgoing SMTP Server active in the system for sending email messages.
  • The same account record cannot be used for sending and receiving emails.

3 Additional Email Accounts

This plugin is designed to be able to process emails that are sent to multiple mailboxes. Each instance comes with one mailbox, which resides on the ServiceNow email architecture (for example, @service-now.com). Any additional mailboxes you want to have checked by the ServiceNow application will be outside of ServiceNow (for example, @yourdomain.com, @gmail.com). Your internal IT or email team should be able to create the additional mailboxes on your corporate internal email architecture. There may also be network changes necessary to allow the ServiceNow application to connect to your internal email architecture on the necessary POP3 or IMAP ports.

4 Configuring ServiceNow to use the Email Accounts

Navigate to System Mailboxes > Administration > Email Accounts and click New. Complete and submit the form using the following fields.

Field Description
Name Name to identify this email account.
Type Mail server type. The choices are:
  • Email Reader
    • POP3
    • IMAP
  • Email Sender
    • SMTP (only one active account permitted)
Authentication Type of authentication used for the email account to connect to the email server. The choices are Password or OAUTH (when the Email - OAuth support for IMAP and SMTP plugin is activated).
Server Remote server to which this account connects.
Port Connection TCP port.
Session Properties Comma separated list of session properties to use. For example, a session property might be mail.debug=true,smtp.debug=true.
Email User Label Used as a display value when sending outgoing emails.
User name User name in the format of an email address. This can be used as the reply-to email address if you are not using the Authentication option.
Password Password when Authentication type is Password.

Note: You may need to increase the size of this field to accommodate longer passwords. By default, this field has a size of 40.

Enable SSL Enables Secure Socket Layers (SSL) when connecting to an email server.
Enable TLS Enables Transport Layer Security (TLS) when connecting to an email server.
Active Determines if this email account is active.
Enable Debug Output Enables debug output when connecting to an email server.

5 Testing

After saving the email account, click Test SMTP connection to test the configuration.


6 Activating Email Accounts

An administrator can activate the Email Accounts plugin.

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.

7 Enhancements

7.1 Dublin

  • ServiceNow provides diagnostic information about each active email account from the email properties page.
  • Administrators can activate the Email Accounts plugin without contacting ServiceNow technical support.
