1 Overview

Report sources are pre-defined data sets for creating reports. Use them for reports containing the same conditions, so you do not have to define the conditions more than once. You can also use report sources to implement the same definitions across your organization. A report source always consists of a table and a number of conditions. When you create a new report, you can either use a report source or select a table and then define the conditions. Some examples of report sources are open incidents, closed problems, and so on. Report sources are available starting with the Fuji release.

Users with the report_admin role can create report sources.

Reporting: How to Use Report Sources

3 Creating a Report Source

  1. Navigate to Reports > Administration > Report Sources.
  2. Click New.
  3. Fill in the fields, as appropriate (see table).
  4. Click Submit.

The Report Source form looks like this:

Example of the Report Source form.

Field Description
Name The name of the report source. For example, Open incidents.
Table The table on which the report source is based. For example, Incident [incident].
Description A more detailed description of what the report source does and its purpose.
Filter Conditions for the specific table records to include in the report source. For example, to include open incidents, select [State] [is] [Active] for the Incident table.
Note: If the report source is used for a report that also includes OR conditions, records are only included in the report if they match the conditions in both the report source and the report.

After you create report sources, you can use them when creating a new report.

If you update the conditions in a report source, these conditions are automatically propagated to all reports based on that report source.

To view any reports based on a report source, click the Reports using this report source related link in the report source record.

While a report source is used by active reports, you cannot delete it.
