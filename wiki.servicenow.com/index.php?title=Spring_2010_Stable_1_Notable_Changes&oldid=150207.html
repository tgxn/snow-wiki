The following are notable changes you may encounter with our Spring 2010 Stable 1 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before.

1 Changes for Spring 2010 Stable 1

1.1 Performance Improvements for Project Management

Enhancements to Project Management provide significant performance improvements in the speed with which the Gantt Chart is rendered. These improvements are most noticeable for large projects.

1.2 Reduced parsing on Oracle Instances

Memory improvements on the client side cache of JDBC prepared statements have allowed us to expand the per connection cache prepared statement cache from 5 -> 50. If you have already set this property manually, the upgrade will not overwrite your values, but you change this property under System Properties -> Oracle DB Options.

1.3 Tooltips for Project Management Gantt Charts

The rendering of tooltips in Gantt Charts has been changed to fade in and fade out.

1.4 OLAP Analysis has been Deprecated

The OLAP Analysis tool has been deprecated. The OLAP module has been removed, but Existing OLAP reports will continue to function through the Berlin release.

1.5 Discovery installed software performance improvement

Performance when updating installed software has been improved during Discovery and when Help the Help Desk runs.

1.6 List/Form layout for License and Software Package tables

Improved some of the list and form layouts for License and Software Package related tables.

1.7 Specify cluster node for LDAP Server Listener

A system_id field has been added to the ldap_server_config table to specify which cluster node the LDAP Listener should run on in a multi-node configuration.

1.8 Empty Knowledge Base search no longer submitted

A Knowledge Base search with no search text will no longer submit the search. Previously the search would return all articles in an instance.

1.9 UI Action to insert UI Policy with Actions

A new UI Action has been added to the UI Policy form to copy a UI Policy with its Actions, similar to the "Insert ACL with Roles" UI Action on the ACL form.

1.10 LDAP starting search directory can be empty

It is now possible for the LDAP starting search directory to be unspecified/empty when setting up an LDAP integration.

1.11 Disable number formatting with a field attribute

Automatic number formatting (e.g., 2500 being formatted as 2,500) can be disabled on a per-field basis by adding an attribute to the field's Dictionary entry: format=none

1.12 "Lock User Session" UI Action

A new "Lock User Session" UI Action has been added to the v_user_session table, letting an admin easily terminate a user's session. The user must reauthenticate at the next attempted transaction.

1.13 Form Link UI Actions for mobile client

"Form link" UI Actions are now rendered on the mobile client

1.14 Field Styles on PDF Export lists

Color and background color Field Styles (sys_ui_style) are now used on PDF exported lists.

1.15 More searchable record numbers

If a field is "number" or "u_number", the following get text indexed separately for ease of searching, and any of them can be searched for:

  1. the actual value (e.g., "INC0014343")
  2. the digits (e.g., "0014343")
  3. the unpadded digits (e.g., "14343")

To enable searching for records last modified before this release, regenerate the text index for the applicable table.

1.16 CurrencyConverter Script Include change

A misspelled method name, getReferenceCurencyCode(), was corrected to be getReferenceCurrencyCode(). This method is called from the CurrencyConverter Script Include, so any customer who has customized that Script Include will need to manually make the change to call the correct method.

1.17 Removed BSM Map -> Saved Maps module

The unused BSM Map -> Saved Maps module has been removed. If it does not get removed from an existing instance, it may be safely deleted.

1.18 Previous is available before delete

The "previous" variable, with the current record's previous values, is now available in "before delete" Business Rules.

1.19 Global Text Search performance improved

Global Text Search result performance has been improved.

1.20 IP Address table

A new CMDB table has been created to hold IP addresses, cmdb_ci_ip_address. It extends cmdb_ci.

1.21 no_truncate for Short Description

The no_truncate attribute has been added to short_description fields to prevent their values from being truncated in lists. To reverse this change, Personalize Dictionary and remove the no_truncate attribute.

1.22 Paging controls on Knowledge Base list

Paging controls have been added to the kb_list UI Page. Previously, all records in a Topic or Category were displayed in the list, which could lead to a performance issue.

1.23 Related list on Computer form

A new defined related list (System Definition --> Relationships) has been added to the cmdb_ci_computer form to show all the IP addresses associated with a computer.

1.24 Email Client icon checks mandatory

Clicking the Email Client icon on a record now enforces mandatory fields, since it saves the record.

1.25 Large Database View performance improvement

Performance has been improved when processing large database views.

1.26 Email property to append time zone

A new property, glide.email.append.timezone (System Properties -> Email), has been added. If true, the instance's system time zone is appended to any dates or date/times in outbound emails (e.g., 2010-07-02 04:01:14 PST)

1.27 Knowledge search result sort order is a user preference

The Knowledge Base system property that controlled sort order of Knowledge search results now sets a system preference. Individual users can override this system preference with a new dropdown in the Knowledge search results page header. A user's preference is remembered for subsequent Knowledge searches. Default sort options are Relevancy and View Count.

1.28 Knowledge search operator property

A new property, glide.knowman.search.operator (System Properties -> Knowledge), can be used to specify the default operator used for Knowledge searches from the Knowledge Base or using the blue book icon (next to Short description on the Incident form, for example). Default is an AND/OR query, which means that first a search is done for results containing ALL search results. If no matches are found, then a search is done for results containing ANY of the search results.

The other option for this property is an OR query, which only does one search and simply results all results containing ANY of the search results.

1.29 DNS name support for CMDB

Added tables, defined related lists (System Definition -> Relationships), and other UI to support DNS names in the manner requested by many customers. CIs, NICs, IP addresses, and DNS names are now modeled as follows: 

CI <= NIC <= IP address <=> DNS name

The defined related lists allow one to view a related list of all IPs and DNS names belonging to a CI.

1.30 New Discovery probe for finding serial number on Solaris machines

A new probe that finds serial numbers on Solaris machines is now available. To use this probe you must install the command line tool SNEEP, which can be found on the SUN website at http://wikis.sun.com/display/sneep/Home.
