1 Overview

This page contains instructions for uploading Microsoft Project plans to the Project Management application. Tasks from a project plan are imported into the Project Task table (pm_project_task) as children of an pre-existing Project. This procedure uses a Visual Basic macro to assign task numbers to each task prior to upload so that parent-child relationships can be preserved in the upload process.

2 Set Number as the Display field for the Project table

When you install the Project Management Plugin, the Display field for the Project table is Name. For uploading projects it is recommended that the Display field be set to Number so that the project can be consistently referenced by its number in the upload file.

  • Go to System Definition > Dictionary
  • Find the pm_project table
  • Select the column name
  • If Display is checked, remove the check and click Update

3 Create a Project

Create a new project and make note of its Number. The remaining instructions on this page will enable you to upload an existing Microsoft Project plan as tasks beneath the newly created project.

4 Reserve a block of Project Task Numbers

Project task numbers will be assigned from within Microsoft Project and included in the upload file. This allows MS project to specify parent-child relationships between tasks. A secondary benefit is that it makes it possible to coalesce on the task number when making subsequent updates to the same project.

To prevent SNC from assigning to a new task a number that has already been assigned by Microsoft Project, we reserve a block of task numbers for use by Microsoft Project. In this example, we will reserve numbers 10001 through 10999. Thus, we will set the next available number in SNC to 11000.

  • Go to System Definition > Number Maintenance
  • Select table prn_project_task
  • Set the Number field to 11000 (or an appropriate value) and click Update

5 Using VB to compute user defined fields

Several user defined columns in Microsoft Project are used to store values required to import the plan into Servce-now.com. The following table shows the mapping between ServiceNow and Microsoft Project fields used by the VB macro.

ServiceNow field Microsoft Project field
Project (task.project.number) Text4
Number (task.number) Text5
Parent (task.parent.number) Text6
Status (task.status) Text7
Order (task.order) Number7

A Visual Basic macro is used to compute these columns. To create the macro

  • Select Tools > Macro > Visual Basic Editor.
  • Select Insert > Module
  • Copy and Paste the subroutine (below) into the module Window
  • Edit the three Const declarations. In our example, sncTaskNumBase is set to 10000 since this number will be added to the MS Project assigned ID.
  • Close the Visual Basic Editor to return to MS Project
  • Select Tools > Macro > Macros
  • Select SetServiceNowFields and click Run
Sub SetServiceNowFields()
    ' This subroutine computes the following fields for import into ServiceNow
    '
    ' MS Project    ServiceNow
    ' ----------    -----------
    ' Text4         task.project (project number)
    ' Text5         task.number
    ' Text6         task.parent
    ' Text7         task.status
    ' Number7       task.order
    '
    ' You MUST EDIT the next three lines before running this macro
    Const sncProject = "PRJxxxxx"
    Const sncTaskNumPrefix = "PRJTASK"
    Const sncTaskNumBase = 10000
    Dim t As task
    Dim pred As task
    Dim order As Integer
    For Each t In ActiveProject.Tasks
        t.SetField pjTaskText4, sncProject
        t.SetField pjTaskText7, "Pending"
        t.SetField pjTaskText5, sncTaskNumPrefix & Format(sncTaskNumBase + t.ID, "00000")
        If t.OutlineLevel > 1 Then
            t.SetField pjTaskText6, t.OutlineParent.Text5
        Else
            t.SetField pjTaskText6, sncProject
        End If
        order = 0
        t.SetField pjTaskMarked, "No"
        For Each pred In t.PredecessorTasks
            If pred.OutlineParent <> t.OutlineParent Then
                ' Only sibling dependencies can be imported into SNC.
                ' Flag this row as a problem and ignore the dependency.
                t.SetField pjTaskMarked, "Yes"
            Else
                If pred.Number7 > order Then
                    order = pred.Number7
                End If
            End If
        Next pred
        t.SetField pjTaskNumber7, order + 10
    Next t
End Sub

6 Importing to ServiceNow

To import your project file, it must be saved in Excel (XLS) or CSV format. Some versions of Project use an XLS format which in incompatible with SNC. If you have problems, try using CSV, or bring the file into Excel and File > Save As with a type of Microsoft Excel Office Workbook.

Refer to the Import Sets portal for general information about importing data.

6.1 Transform Map

Specify Project Task as the transform map target table. Be sure to Coalesce on the number field. The following table contains recommendations for the transform map.

Target field Microsoft Project field Coalesce Optional/Required
number Text5 true required
project Text4 false required
parent Text6 false required
name Name false recommended
status Text7 false recommended
planned_start_date Start false recommended
planned_end_date Finish false recommended
order Number7 false recommended

6.2 Duration

For Duration, the easiest method is include Start Date and Finish Date in the import file, and then use javascript to compute the duration as the difference. Check Run Script on the import map, and include the following statement in the Script box. 

target.planned_duration = gs.dateDiff(source.u_planned_start_date, source.u_planned_finish_date);

7 Notes regarding Dependencies

The Project Management plugin uses a simple technique for managing dependencies. All tasks with the same order start at the same time, and all of them must finish before any higher ordered tasks can start.

8 Alternate techniques for numbering tasks

8.1 UniqueID

If you intend to maintain your plan in MS Project, and then upload changes to SNC, you may may wish to base your task numbers on UniqueID rather than ID. UniqueID is like ID in that it is a sequentially assigned number. However, UniqueID does not change when rows are inserted in the plan or when tasks are moved around. To base task numbers on UniqueID, use the following VB statement. 

t.SetField pjTaskText5, sncTaskNumPrefix & Format(sncTaskNumBase + t.UniqueID, "00000")

8.2 Task number based on project number

Another technique is to prefix the task number with the project number rather than using "PRJTASK" as a prefix. An advantage of this is that it eliminates the need to reserve a block of numbers for each project. In this case, the task number would be set using the following VB statement. 

t.SetField pjTaskText5, sncProject & "." & Format(t.UniqueID, "0000")

Assuming the sncProject is "PRJ10001", the first task would be "PRJ10001.0001", the second task would be "PRJ10001.0002", and so forth.
