1 Fall 2010 Stable 2 Fixes

1.1 PRB559145 (UI Actions): "Insert with Actions" UI Action did not work on Catalog UI Policy form

A new UI Action has been created that works as expected for the Catalog UI Policy form to copy the UI Policy with actions.

1.2 PRB559033 (UI Pages): Some Legacy UI Pages referenced removed Ext JavaScript library

These UI Pages were no longer necessary and have been removed.

1.3 PRB559065 (Reports): List Report export to PDF was missing grouping

List Reports with a grouping that are exported to PDF now retain the correct grouping.

1.4 PRB559035 (Domain): Domain Picker could improperly set the "Guest" user's domain if used after a session timeout

Domain is now set only for the session and not the user record if the user is Guest.

1.5 PRB559074 (Discovery): Solaris disc sizes were not properly recorded

Solaris disc sizes are now properly recorded.

1.6 PRB544973 (Email): Potential failure to send email notification despite Weight being set to 0

Emails with a weight of 0 are always sent.

1.7 PRB559052 (Text Search): Records might show up twice in search results

The same record will no longer show up twice in search results.

1.8 PRB559092 (Reports): Bar chart grid might not display all the results when the report is ordered and not stacked

Bar chart grid now displays all results when the report is ordered and not stacked.

1.9 PRB559075 (Workflow): Workflow input variables could be created with the u_ prefix

Workflow input variables are now prefixed with u_, designating them as customer-created variables.

1.10 PRB559141 (UI): Forms did not render and an error was generated when a defined Relationship used a many to many table

Defined Relationships using a many to many table no longer break forms.

1.11 PRB043192 (Reports): Scheduled reports did not support the Display Grid details below a bar chart

Scheduled reports have been enhanced to support bar chart grids.

1.12 PRB559122 (List Editing): List editing a reference field to null did not get audited

List editing a reference field to null now gets audited.

1.13 PRB559073 (List v2): Goto search in List v2 did not add criteria to filter when expanded

Goto search in List v2 now adds criteria to filter.

1.14 PRB559039 (i18n): Special character problem with choice list dependency

Special characters in the value of a choice list option no longer break another choice list dependent it.

1.15 PRB559159 (Navigation): Goto search did not work properly with a single digit integer, converting the query to a STARTSWITH

Goto search using a single digit integer now works as expected.

1.16 PRB559160 (List v2): An "after date" condition might get lost when the filter was expanded (introduced Fall 2010 Stable 1)

"After date" conditions no longer get lost when expanding the filter in List v2.

1.17 PRB559088 (List Editing): Password fields could not properly be list edited

Password fields can now be properly list edited.

1.18 PRB559191 (UI): List v1 did not show the correct many-to-many relationship label

List v1 now shows the correct many-to-many relationship label.

1.19 PRB559232 (SSO): List v2 did not work properly with SSO with reverse proxies

List v2 now works properly with SSO with reverse proxies.

1.20 PRB446646 (Rota): On-call coverage did not let user select which Roster to provide coverage for

Users can now select which Roster to provide coverage for, if a Rota has more than one Roster.

1.21 PRB559121 (List v2): sysparm_fixed_query might not be passed along to subsequent actions, like right-click Bar Charting

sysparm_fixed_query now is passed along as appropriate.

1.22 PRB550328 (List v2): Overloaded choices (e.g., those with different labels for the same value for task and incident) did not render properly

List v2 now supports overloaded choices as expected.

1.23 PRB551231 (Discovery): Error when discovering 64-bit Windows installed software

64-bit Windows installed software is now properly discovered without error.

1.24 PRB559044 (UI Macro): Advanced glide_list functionality missing from lightweight macro_watch_list UI Macro (Enhancement)

"Add me", "Delete", and the reference list selector (magnifying glass) all now work as expected when using the macro_watch_list UI Macro in a Macro variable.

1.25 PRB559193 (Reports): Calendar report radio buttons gave the message, "You do not have access to view the requested report" (introduced Fall 2010 Stable 1)

Calendar reports once again work as expected if the user has roles and visibility to see them.

1.26 PRB559195 (Forms): Form view could not be changed if default UI Action to save the form was removed

Changing the view of a form is now independent of any UI Actions on the form.

1.27 PRB049000 (Workflow): The "SNC - Run Parent Workflows" business rule threw an error "Get for non-existent record" when the parent had already been deleted

The "SNC - Run Parent Workflows" business rule no longer throws an error when the parent has been deleted.

1.28 PRB559260 (Service Catalog): The "Include none" option for a multiple choice variable did not correctly default to None, causing a JavaScript error in IE that prevented the catalog item from saving

"Include none" now works as expected, and items are created.

1.29 PRB559183 (Log File): Log File Browser no longer generated out put if filter level of "All" was selected (introduced Fall 2010)

Log File Browsers once again works as expected.

1.30 PRB559197 (Workflow): Variables might not show up on tasks created for a context if the context's workflow was subsequently edited

Variables now show as expected if their workflow gets edited.

1.31 PRB559162 (Workflow): Variables might not show up on tasks created for a context if the context's workflow was subsequently updated via Update Set

Variables now show as expected if their workflow gets updated via Update Set.

1.32 PRB559173 (Chrome/Safari): A bug in the Chrome and Safari browsers could result in a timer field incrementing faster than expected

Code has been put in place to work around the WebKit browser bug (http://code.google.com/p/chromium/issues/detail?id=29644), and the timer field now increments as expected.

1.33 PRB043780 (Import Sets): Import the data through a MID Server using JDBC could modify international characters in a way that was not desired

Data is now written and read through the MID Server's cache file symmetrically.

1.34 PRB559051 (List v2): In List v2, IE7 users might be able to run List Choice UI Actions when the conditions should not allow it

In List v2, IE7 users can no longer run List Choice UI Actions when the conditions do not allow it.

1.35 PRB041653 (On-call Rotation): Multiple On-call Email Notifications could fire for one day

New OnCallRemindersNG Script Include implemented to provide modified logic for firing On-call email notifications.

1.36 PRB559200 (On-call Rotation): On-Call Reminders could send sending multiple emails to the same person for the same Coverage entry

On-Call Reminders are now prevented from sending multiple emails to the same person for the same Coverage entry.

1.37 PRB559187 (Global Text Search): Record links from a Global Text Search were broken if the glide.ui.text_search.view property was blanked out (introduced Fall 2010 Stable 1)

Global Text Search record links are now built correctly if the glide.ui.text_search.view property is empty.

1.38 PRB425845 (Import Sets): The "Transform history" link might not link to the correct import set after running a transform

The "Transform history" link is now built properly.

1.39 PRB501194 (On-call Rotation): Users could improperly edit in the monthly view

Users are now prevented from improper editing in the monthly view of the On-call Rotation calendar. New ACLs put in place. rota_admin role required to modify rotas. rota_manager role delegated for a specific group required to modify that group's rotas and associated data. Group manager can also modify that group's rotas and associated data.

1.40 PRB559241 (Knowledge): Knowledge search might not work properly with the "And then OR query" search operator (introduced Fall 2010 Stable 1)

Knowledge search now works as expected with the "And then OR query" search operator.

1.41 PRB421040 (On-call Rotation): When changing a Weekly 24x7 on-call rota to Daily, the rota did not repeat

When changing a Weekly 24x7 on-call rota to Daily, the rota now repeats as expected.

1.42 PRB252114 (On-call Rotation): Email Notifications not being sent on time for on-call rotation

The fix for PRB041653 also fixed this Problem.

1.43 PRB559265 (Homepage): Hovering over homepage icon as ESS user could cause an empty context menu to appear

Hovering over homepage icon as ESS user no longer causes an empty context menu to appear.

1.44 PRB559031 (CMDB): BSM Map Error: No option for map_ciview

The cause of this error has been removed.

1.45 PRB283171 (Web Services): Hierarchical web services did not work properly on tables with a circular reference

Hierarchical Web Services inserts are now limited by default to a scope of 3 to prevent an infinite loop. This is controlled by a property: glide.wsdl.maximum_hierarchy

1.46 PRB559248 (UI): Link text in List v1 lists did not show ServiceNow context menu (Introduced Fall 2010 Stable 1)

Links in List v1 lists once again show ServiceNow context menus.

1.47 PRB559117 (List v2): In List v2, List Choice UI Actions could be improperly executed against records

In List v2, List Choice UI Actions are now only executed against the correct selected records, based on their conditions.

1.48 PRB559114: Non-admin users could not run a report.

Users without admin or report_admin rights can now get report data when they press the Run Report button on an existing report.
