Overview

This page lists enhancements and fixes that have been included in ServiceWatch since version 3.0.

  • Version 3.6 Enhancements: view enhancements and notable changes included in version 3.6.
  • Version 3.1 Enhancements: view enhancements and notable changes included in version 3.1.
  • Fixes: view fixes that were included after version 3.0.
  • Submit Feedback: provide feedback on documentation.

Version 3.6 Enhancements

List Format Dashboard

This dashboard supports business level key performance indicators (KPIs) to display the status of your business services around the world. KPIs are automatically displayed in the new List format Dashboard that combines the Business Service status, Business Service user status, and essential KPIs that are selected by the end user as critical parameters that represent service health.

The gear icon () includes CI level monitors. The user icon () excludes CI level monitors. The color of these icons indicates the worst severity level of any monitor associated with that business service or group.

The Standalone List Format has an HTML5 screen that is compatible with a wide range of handheld devices (such as iPads, iPhones, Android tablets, and laptops). This format enables the user to monitor the system at a glance from anywhere via the web.

Note: In the figure above, the Monitoring panel is displayed. The Menu bar is not displayed because its options require Flash and most handheld devices do not support Flash.


Wizard to Plan Changes

This feature replaces the Physical Search. The new Dependencies screen provides a wizard to plan changes. You can view dependencies from any CI in any context and plan changes more accurately.

First, search and find relevant CIs using a search string. Then, you can select a CI and refine your search to a feature of the CI. For instance, if the CI is a server and you want to determine the impact of a network Port for network and storage devices, File systems data for hosts, Volume data for storage devices, and Inclusion data for CIs that are parents of other CI types.

On the Define Change panel, you can specify your changes and schedule when the changes are made. You can create a new scheduled change for a displayed CI or add the CI to an existing schedule change.

To display a table of all scheduled changes, click Scheduled Changes.

You can edit a scheduled change by clicking its edit icon () or double-clicking its row in this table.

In the History preview strip and History panel of other screens, the numbered call-out circles become green for CI removal or attribute changes that occur during a scheduled change interval. Topology changes are displayed in the usual way.

The Severity level for events that occur during a scheduled change interval is downgraded to Warning and the background color of that event’s row in the Severity column becomes grey.

If a system uses a CMDB that is integrated with ServiceWatch and you have populated the ServiceWatch CI’s ID as an attribute of the CI, a new utility lets you populate the Define Change panel from your change management system.

Systems Health Feature

The new System Health option replaces the Discovery Messages option. In addition to displaying discovery messages, it gives information about Server Status, Collectors, 3rd Party Connectors, Monitoring, Network Discovery, and Service Discovery.

The WiFi icon () is green if all Collectors are active. If it is red, there is a problem with a component such as a VM connector or monitoring tool.

To display the System Health screen, click either the WiFi icon () or tool icon (), and then select the System Health option in the Settings menu. By default, the Server Status panel is displayed. The color of the icon in each System Health tab indicates the worst severity of any system element associated with that tab.

Importing CI Attributes

In the Import/Export dialog box, you can now import CI attributes from a CSV file.

The system examines each line of the input file and locates CIs based on that line’s key attributes. If any key attributes are missing, the system updates all of the CIs of the specified CI type on the specified host.

This feature’s output includes error messages if the file contains an invalid format or if any line could not be processed for any reason. However, failure to process a line will not abort the update process. The output includes statistics on the number of CIs that were updated and the number of update attempts that failed.

To import CI attributes, click the tool icon () > Import / Export > Import CI Attributes from File. In the window that appears, select the CSV file to be imported. The import runs automatically when you select the input file. When the update process completes, the following message displays:

Import finished. To view the results of the import, click .

Active Directory Support of Multiple Domains

Active Directory support was extended and modified to support multiple domains. The Active Directory Users panel was added to facilitate the definition of domain connections and the mapping of ServiceWatch roles to Active Directory users. The list of domains is in the ad.properties file in the ServiceWatch\server\conf\props directory.

Enhanced Password Security for SaaS

SaaS password security was enhanced to enable customers to define extended password policies.

Easy Audit Log Extraction for SaaS

A button to easily extract security audit log data in SaaS environments has been added.

Version 3.1 Enhancements

Integration with Third-Party Tools

The integration with HP uCMDB and ServiceNow CMDB has been updated to provide customized mapping of CI types, attributes, and relations. This simplifies customization of the integration to support customer created CIs and attributes in the CMDB.

There is a new integration to support BMC applications with Atrium CMDB.

Revised Views for KPIs

A new KPI view was added to the Tile and Bubble dashboards. This view enables viewing and analysis of large numbers of KPIs. KPIs can now be sorted and filtered based on KPI name, type, severity, and entity monitored. The new KPI view contains a simple chart that facilitates viewing and comparing KPIs. Specific KPIs can be selected for a detailed comparison that synchronizes their vertical and horizontal scales and displays them on the same graph.

Severity Levels in the Infrastructure or User Perspective

New functionality was added to identify events from the viewpoint of how the user sees the service. A toggle was added to the Tile and Bubble dashboards to enable them to display the status of business services from either the new “user perspective” or the previous “system perspective.” Clicking the user icon () displays Severity levels from a User perspective based on monitors at the business service level. Clicking the user icon () again changes the display back to the Infrastructure perspective.

Exporting a Topology Map

The Export Map icon enables you to export the currently displayed topology as a PDF file.

Aggregate Monitors

A new type of monitor was added that uses an algorithmic function to create KPIs that represent a combination of other KPIs. For example, this enables the creation of group level KPIs that show the maximum response time of a group of business services.

Synthetic Monitors

The Business Service Synthetic monitor was enhanced to store and display recording screen images to help the user understand what actions the monitor is performing.

Chapter 12: Reports

The Excel-based report generator was enhanced so that user-created reports can be uploaded to the server to be shared with other users and become standard reports at a customer installation. In addition, these user-defined reports can be scheduled to run at predefined times and sent via email to the appropriate recipient.

SNMPv3 Support

SNMPv3 provides cryptographic security and supports remote configuration of SNMP entities.

Fixes

This is a list of fixes that were included after version 3.0. If you must know about a particular problem that is not listed here, contact the ServiceNow customer support team.

Discovery

Bugs that affected the fidelity of the discovery and rediscovery processes were resolved.

History Retention

Improvements were implemented to enable longer history retention periods.

Monitor Scheduling

The monitor scheduling algorithm was refined to prevent monitor hanging when multiple monitors are scheduled to start at the same time.

Scalability

Scalability was enhanced in a number of significant areas including history retention, response times, collector utilization, and monitoring techniques.

Storage

The storage analysis algorithm was rewritten to support very large storage arrays and eliminate open bugs in this area. Path calculation problems were resolved for very complex storage arrays.

User Interface response time

Response times to the user interface were significantly improved.
