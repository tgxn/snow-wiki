3 Fixed in This Release

To sort the table in ascending or descending order, click the sorting icon in the column heading.

Some fix descriptions provide steps to reproduce. Click the plus (+) icon to view a procedure that you can use to test the fixes on your instance.


EA Release Problem Description
EA 8 PRB588584 The # members to notify field in the Notification Rule for the Primary roster is ignored.
Click the plus for steps to reproduce
  1. Activate the com.snc.on_call_rotation plugin.
  2. Open System Definition > Business Rules and activate the On-call Rotation Notify business rule.
  3. Open On-call Rotation > Create New Rota. Fill out the form using the specifications below, and press Submit.
    Group: Service Desk
    Rota name: Service Desk Rota
    Rotation start date: use yesterday's date
    Schedule type: 24 x 7
  4. Open On-call Rotation > My Groups Rotas, remove the Group filter specification, and open the Service Desk Rota record.
  5. In the Notification Rules related list, click into the one record referenced there.
  6. Update this record by specifying "2" for the "# members to notify" field, and Save the record.
  7. Create a new incident record, making sure that the State is "New" and the Assignment Group is "Service Desk". Submit the record.
  8. Open the Rotation Escalation table (cmn_rota_escalation), and there should be a record there that starts with "Rota-" and ends with the number given to the previously created incident.
    The "<notifications>" node of the XML document contains three child nodes instead of two.
EA 8 PRB594774 Workflows fail to run on instances that do not use English as the system language.
Click the plus for steps to reproduce
  1. Activate the I18N: Brazilian Portuguese plugin.
  2. Navigate to System Properties > System Localization.
  3. Set the Default language for the system to pb.
  4. Set the Locale code to use for localization to pb:BZ.
  5. Reload the page.
  6. Create a new workflow with no activities between Begin and End.
  7. Run the workflow.
  8. Open the context for the new workflow.
    Note the workflow never completes and indicates a NPE in the log.
EA 8 PRB594337 A high availability node could become unresponsive due to a deadlock.
EA 8 PRB591408 A database pool can be retired while there is an active transaction running, leading to an indefinitely open connection to the database.
EA 8 PRB594672 A database connection used when cloning an instance can be closed unexpectedly during periods of no activity.
EA 8 PRB594262 Unloading a new portal page to add it to an update set creates an update set entry named sys_portal_null instead of sys_portal_<portal sys_id>.
Click the plus for steps to reproduce
  1. Create a new update set and select it as the current update set.
  2. Navigate to Homepage Admin > Pages.
  3. Right-click a homepage record.
  4. Select Unload Portal Page.
    The banner message displays sys_portal_null.
  5. Create an additional portal page and add it to the update set.
    No new update set entry is created.
EA 8 PRB593741 Project Task child nodes without a defined Level value can cause a loop leading to stuck transactions.
EA 7 PRB593423 SQL exceptions are generated by long duplicate column names in sibling tables being aliased differently.
Click the plus for steps to reproduce
  1. Choose two sibling tables in a flattened hierarchy, such as incident and change_request.
  2. Go to the list view on incident, personalize, and add a string field with length greater to or equal to 30. For example u_offsite_special_instructions.
  3. Verify the column was added correctly by populating/reading data in the list view.
  4. Go to the list view on change_request, personalize, and add a string field with the same name. For example u_offsite_special_instructions.
    You should see an error in your logs adding the field, and it will not be available on the list view to choose from. The column was not added.

Click the plus for steps to reproduce

  2. Have the subflow complete.
    You will see the error in the log.
EA 7 PRB593830 Created dates for Certification Instance records are modified to the upgrade time after upgrading.

EA 7 PRB593880 NullPointerException is generated from REST and SOAP activity.
Click the plus for steps to reproduce
  1. Set up an instance with the Orchestration plugin disabled.
  2. Set the MID server up to point to the instance.
  3. Create REST message.
  4. System Web Services -> REST Message.
  5. New REST Message:
    Name: test
    Check "Use basic auth"
    Basic auth user ID: admin
    Basic auth user password: admin
    Click save
  6. Click "get" function in "REST Message Functions".
  7. Enter the MID server you setup in "Use MID server".
  8. Click Test in Related link to verify message is set up correctly. Verify HTTP status is 200.
  9. Create a new test workflow and add REST Message activity.
  10. Config as followings in REST Message activity:
    REST Message: test
    REST Message Function: get
    Sensor Script: workflow.scratchpad.result = activity.output;
  11. Run the test workflow and the activity hangs.

    From = Incident
    M2M from field = u_incident

    To = incident
    M2M to field = u_incident_parent
    M2M to label = Incidents

  5. For those incidents, populate the new field with information.
  6. Go to the u_m2m_incidents_incidents table and add a filter "> Show Related Fields, Incident (u_incident), new field is same as..."

EA 6 PRB592770 Update set records that have sys_portal_page are appending to the role instead of replacing the roles that are contained in the update set.
Click the plus for steps to reproduce
  1. Locate a portal page inside sys_portal_page for your source that already has a populated role for both read and write.
    The target record should have populated write / read roles.
  2. Enable the desired update set to capture the modification.
  3. Add another role on top of the current roles for both read and write in the sys_portal_page.
  4. Save the record.
  5. In the list view for sys_portal_page, right-click and unload the modified record.
  6. Complete the update set.
  7. Go to the update set that was recorded and export to XML.
  8. Go to the target instance and retrieve update set.
  9. Import XML to the retrieve update set.
  10. Preview and commit.
  11. Go to the sys_portal_page.
    The modifications performed on the source are appended to the current roles on the target instead of replacing them.

EA 6 PRB591588 After subsequent upgrades, the value of the glide.sys.domain.partitioning property changes from False to True.
Click the plus for steps to reproduce

  2. Upgrade to Calgary Patch 2 Hot Fix 5.

  3. Then upgrade to Calgary Patch 2 Hot Fix 7.
    Both the domain plugins are remain active, but the glide.sys.domain.partitioning property changes to True.

EA 5 PRB591774 Scripts in the Catalog UI Policy fields Execute if true and Execute if false may fail to run when the glide.ui.escape_all_script property is set to true.
View the steps to reproduce
  1. Create a Catalog UI Policy with run_scripts set to true, and a simple alert in the Execute if true script. For example, use the Blackberry as the item.
  2. Set the UI Policy to run on load. You do not need to add any conditions.
  3. Open the Blackberry item in the service catalog.
    Notice that the alert you created in step 1 appears correctly.
  4. Set the glide.ui.escape_all_script property value to true.
  5. Open the Blackberry in the service catalog again
    An Uncaught SyntaxError: Unexpected token ILLEGAL error appears.
    Using the browser's developer tools, you can see that the script is escaped, causing the error.
