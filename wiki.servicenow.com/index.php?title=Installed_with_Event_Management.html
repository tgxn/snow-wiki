1 Overview

Activating event management adds or modifies tables, user roles, script includes, and other components.

Demo data is available with Event Management.

2 Tables

Table Name Description
Alert [em_alert] Collects the alerts to be managed by event management.
Alert Rule [em_alert_rule] Maps the alert fields to the Incident [incident] table.
Alert Services [em_map_alert_to_service] Maps alerts to affected services.
Alert Template [em_alert_template] Contains alert templates. This table extends the Template [sys_template] table with table as Alert [em_alert].
Connector Definition [em_connector_definition] Specifies settings to gather events from external event sources. Available starting with the Fuji release.
Connector Instance [em_connector_instance] Specifies connection details for external event sources. Available starting with the Fuji release.
Event [em_event] Collects the events received by event management.
Event Filter [em_event_filter] Stores the defined event filters.
Event Match Rule [em_match_rule] Updates events for alert processing. Available starting with the Fuji release.
Event Mapping Rule [em_mapping_rule] Updates event fields for alert processing. Available starting with the Fuji release.
Event Processing Statistics [em_event_stats] Collects statistics on event management performance.
Event Type [em_event_type] Maintains the event types.
Incident Template [em_incident_template] Contains incident templates. This table extends the Template [sys_template] table with table as Incident [incident].
Registered Nodes [em_registered_nodes] Maintains the registered nodes data.
Threshold Rule [em_threshold_rule] Specifies alert threshold rules. Available starting with the Fuji release.

3 User Roles

Role Contains Roles Description
Event Management Integrator [evt_mgmt_integration] rest_service Have create access to the Event [em_event] and Registered Nodes [em_registered_nodes] tables, which allow external sources to send events and register nodes using the REST API.
Event Management User [evt_mgmt_user] itil Have read access to all event management features and have write access to alerts to manage the alert life cycle. They also have the ITIL role to be able to manage the alert-incident life cycle.
Event Management Administrator [evt_mgmt_admin] evt_mgmt_user
template_editor_global 		Have read and write access to all event management features.

4 Script Includes

Script Name Description
ConnectorUtil Connector handler. Available starting with the Fuji release.
EvtMgmtIncidentHandler Creates an incident for an alert based on the incident template defined in the alert rule.

5 Business Rules

Business Rule Table Description
Add message key if missing Alert [em_alert] Constructs a message key from the Source, Node, Type, and Resource field values.
After insert (async) Alert [em_alert] Updates the parent field of an alert, creates mapping between alerts and CMDB services, and automatically creates incidents based on the alert rules.
After update (async) Alert [em_alert] Creates incidents automatically based on the alert rules.
Apply overwrite rule and validate Alert [em_alert] Applies and validates overwrite rules.
Close associated incident Alert [em_alert] Closes the incident associated with an alert as defined by the evt_mgmt.alert_closes_incident property.
Delete related Threshold Staging records Alert [em_alert] Removes closed alerts from the Threshold Staging [em_threshold_staging] table.
Name and pattern cannot be empty Event Type [em_event_type] Verifies that the Name and Pattern fields have values.
Reopen associated closed incident Alert [em_alert] Reopens the incident associated with an alert as defined by the evt_mgmt.alert_reopens_incident property.
Update Instance Parameters Connector Instance [em_connector_instance] Refreshes Connector Instance parameters in memory before connecting. Available starting with the Fuji release.
Validate Inputs Connector Definition [em_connector_definition] Validates entries in the Name and Schedule fields. Available starting with the Fuji release.
Validate Inputs Event Match Rule [em_match_rule] Validates entries in the Name field. Available starting with the Fuji release.
Validate Inputs Event Mapping Rule [em_mapping_rule] Validates entries in the Name, From field, and Field to fields. Available starting with the Fuji release.
Verify overwrite template table Alert Rule [em_alert_rule] Verifies the overwrite template of the alert rule is defined on the Alert table.
Verify template is on Incident Alert Rule [em_alert_rule] Verifies the incident template of the alert rule is defined on the Incident table.

6 Probes

Probe Description
Third Party Connector Probe Pulls events from external event sources. Available starting with the Fuji release.
