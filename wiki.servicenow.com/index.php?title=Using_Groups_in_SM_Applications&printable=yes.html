Using Groups in SM Applications

1 Overview

Requests in Service Management (SM) applications can use four different types of groups, each with an application-specific identifier, as follows:

Application Dispatch Group Type Qualification Group Type Vendor Group Type Work Group Type
Facilities Service Management fm_dispatch N/A facilities_vendor facilities
Legal Service Management legal_dispatch N/A legal_vendor legal
Work Management wm_dispatch wm_qualification wm_vendor wm_work

Before using an SM application in a production environment, create at least one group of each type to cover all locations. Creating the application-specific group types ensures that when, for example, you are working in work management, only groups defined for work management are displayed. For more information, see Creating Groups.

Note: If you create a custom application using the SM application designer, you should create at least one group of each type and prefix each one with the name of the application. For example, the group type for an application called Support would be support_dispatch, support_qualification, etc.


2 Qualification Groups

Used in Work Management only, qualification groups are sets of users who can provide technical information regarding the work to be performed to fulfill an order or request, or their associated tasks. If one qualification group covers the location specified on an order, the group name is automatically added to the Qualification group field on the order when its state changes to Awaiting Qualification. If no qualification group or more than one qualification group covers the specified location, the Qualification group field is not automatically populated; information can be added manually. If no qualification group is assigned, the order or request can remain without a qualification group and proceed through the application life cycle. However, it is good practice to set up location coverage and qualification groups.

Note: The Qualification group field is hidden in the application's request form when the application is configured for automatic qualification.


3 Dispatch Groups

A dispatch group is a set of users who can select an agent to complete a task. Dispatch groups are filtered based on location coverage. If only one dispatch group covers the location specified on a task, that group is added to the Dispatch Group field on the task record automatically. If multiple dispatch groups cover the specified location, use the lookup list to select a dispatch group. If no dispatch group is assigned, the order or request task cannot progress to the Pending Dispatch state.

4 Assignment Groups

An assignment group is a set of agents, filtered by location, from which an individual agent should be selected to complete an order or request task. If a dispatch group has been identified and the dispatch group only covers one assignment group, that assignment group is added to the Assignment Group field on the task record automatically. If the identified dispatch group covers multiple assignment groups, use the Assignment Group field lookup list to select an assignment group. If the Assignment Group field is empty, the system searches for assignment groups covering the territory that includes the task's location.

5 Vendor Groups

A vendor group is a set of vendors, filtered by location, from which an individual vendor should be selected to complete an order or request task. If a dispatch group has been identified and the dispatch group only covers one vendor group, that vendor group is added to the Vendor Group field on the task record automatically. If the identified dispatch group covers multiple vendor groups, use the Vendor Group field lookup list to select a vendor group. If the Vendor Group field is empty, the system searches for vendor groups covering the territory that includes the task's location.

Note: When a vendor group is created, be sure to assign a manager to the group who has the vendor manager role for assigning and controlling vendor tasks. Also be sure to add vendors to the group.


6 Group Usage Example

The following example illustrates how locations, qualification groups, dispatch groups, and assignment groups can be created and used together for work management. The best approach is to create locations first, then create the different types of groups and associate locations to the groups.

Company ZYX has a group of work management agents that cover four locations:

  • San Diego
  • Los Angeles
  • San Jose
  • San Francisco




There are two qualification groups that qualify work orders and work order tasks:

  • Southern California
  • Northern California

The Southern California qualification group covers the San Diego and Los Angeles locations. The Northern California qualification group covers the San Jose and San Francisco locations.




There are four dispatch groups that select agents to complete work order tasks:

  • San Diego
  • Los Angeles
  • San Jose
  • San Francisco

Each dispatch group covers their corresponding location.



There are 4 assignment groups of agents that complete work order tasks:

  • San Diego
  • Los Angeles
  • San Jose
  • San Francisco

Each assignment group covers their corresponding location.



Using the four locations and the different groups, work management tasks can follow a logical flow based on location. The work is qualified, dispatched, and then performed by people who are as close to the work site as possible for maximum efficiency. For example, a work order with San Francisco specified in the Location field is created. A member of the Northern California qualification group qualifies the work order by adding detailed information. A member of the San Francisco dispatch group selects an agent from the San Francisco assignment group to perform the tasks.
