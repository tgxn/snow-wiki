1 Overview

The ServiceNow plug-in for the ManageIQ Enterprise Virtualization Management Suite™ (EVM) creates a bidirectional integration between a ServiceNow and ManageIQ EVM instance which allows request, configuration, and incident automation between the two instances. This integration enables administrators to manage their virtualized servers, desktops, and private clouds from within ServiceNow. The plugin enables automated virtual machine request fulfillment from the ServiceNow Service Catalog and creates and maintains configuration items (CI) in the ServiceNow CMDB. ServiceNow creates incidents based on EVM policy-based enforcement and alerting in the virtual environment.

Note: The following integration is not part of the supported platform. The integration may require additional customization, typically performed by an experienced administrator or consultant, prior to operating correctly.


2 Installed with this Plugin

The Integration - ManageIQ EVM plugin installs a set of new Web Services, business rules, and properties into ServiceNow instances.

2.1 Web Services

Field Description
VMTemplate Invoked by EVM to synchronize automatically the VM templates that are available for ordering in the Service Catalog. The templates will be inactive by default and will require activation to make them available for presentation in the catalog.
Notification Invoked by EVM to open up incidents in ServiceNow automatically.

2.2 Business Rules

Field Description
EVM Provisioning Update Updates progress on provisioning requests in real-time.
EVM Create VM in Catalog Creates Catalog Item for virtual desktops and servers.
EVM - VM Request Processes request to EVM for provisioning virtual desktops and servers.
EVM - Create CMDB Computer CI Processes request from EVM to create CI in CMDB.
EVM Create Incident Processes request from EVM to create incident.
EVM Synch VM Template Processes request from EVM to synchronize VM templates in ServiceNow.

2.3 Properties

Field Description
Userid for EVM Server The EVM user name to be used for processing requests.
EVM Server password The EVM password.
The catalog item category for EVM template items The default category in the Self-Service catalog in which items such as virtual servers and desktops will be presented to users for ordering.
EVM created incident will have this value for its category The default category under which incidents will be opened in ServiceNow.
EVM created incident will have this value for its subcategory The default subcategory under which incidents will be opened in ServiceNow.


3 Enabling the Integration

Request the Integration – ManageIQ EVM plugin (com.snc.integration.manageiq_evm) from ServiceNow Technical Support, activate the plugin, and then go to http://www.manageiq.com/integration/ServiceNow to enable the ServiceNow plug-in with your EVM infrastructure.

4 ServiceNow Setup

Navigate to Integration – ManageIQ EVM > Properties to configure the integration. Enter the address of the EVM Server and a user account for EVM with the appropriate level of access. See Properties on this page for details.


5 Activating ManageIQ EVM

Activate the EVM Automate integration with ServiceNow in the EVM Server to turn on the appropriate integration points and enable the Web Services.
