1 Overview

Expense allocation refers to the assignment of expenses to accounts in the IT chart of accounts. You can allocate expenses by applying rules to groomed expenses in the general ledger. The methods and metrics in the rules specify how the expenses are allocated. See Administering Cost Allocations for more information on how rules, methods, and metrics work.

Note: You can also allocate expenses in the workbench. Using the workbench is the preferred method of allocating expenses, rather than using lists and forms.


2 Reversions and Reallocations

After a groomed expense in the general ledger is allocated, it cannot be allocated again unless the original allocation is reverted. You should revert allocations if changes to allocation rules, methods, or metrics are made and the general ledger expenses must be reallocated according to the modifications. In addition, if you must make changes to the IT chart of accounts, all expenses need to be reverted first.

3 Previewing Allocations from the Groomed General Ledger

Before you perform an allocation, you can see which methods would run on the expense and how the expenses would be broken down. When you preview an allocation, no actual allocation line records are created.

  1. Navigate to IT Finance > General Ledger > Groomed Expenses.
  2. Open an expense that is not allocated.
  3. Click the Preview Allocation related link to open the allocation viewer.
  4. If you do not want to use the allocation viewer, you can also click Preview Expense Allocation.
    The projected allocations are listed at the top of the page. The first row shows the rule that applies to the expense. The following rows show the projected expense allocation to each segment.
An allocation preview.

 Note: If the expense does not match any allocation rules, verify that your rules are configured correctly and are active.


4 Allocating Expenses from the Groomed General Ledger

From the groomed general ledger, you can allocate all expenses at once or allocate a single expense.

  1. Navigate to IT Finance > General Ledger > Groomed Expenses.
  2. To allocate all expenses in the general ledger that are not yet allocated:
    1. Click Allocate Expenses.
    2. Select a fiscal period to allocate the expenses that belong to that period.
    3. Click OK.
  3. To allocate a single general ledger expense:
    1. Click the number of an unallocated expense to open it.
    2. Click Allocate Expense. See Viewing Expenses in the General Ledger for a description of the fields on the Groomed General Ledger Data form.
The system allocates the expenses, creates allocation lines, and changes the State field on the general ledger record to Allocated. The values of all allocation lines created from the allocation add up to the value of the general ledger expense. If no rules apply to the general ledger expense, no allocation lines are created and the state of the general ledger record remains Unallocated.
 Note: The system rounds allocation lines to two decimal places. An allocation totaling $100.495 is rounded up to $100.50. An allocation totaling $100.494 is rounded down to $100.49. If an allocation is rounded down to $0.00, the system does not create an allocation line.


5 Allocating Expenses for a Rule

You can also allocate expenses that are matched by an allocation rule.

  1. Navigate to IT Finance > Cost Model > Allocation Rules.
  2. Click an allocation rule number to open it.
  3. Click Process Allocation. This UI action appears only when the rule is active and when there are matching expenses that the rule can process.
    After an allocation is processed, you can also revert the allocation if necessary.

6 Reverting Allocations

You can revert allocations at any time to delete the allocation lines. Revert allocations if an allocation is incorrect or if you want to edit the segments in the IT chart of accounts.

To revert allocations associated with a general ledger expense:

  1. Navigate to IT Finance > General Ledger > Groomed Expenses.
  2. Click the general ledger expense that has already been allocated.
  3. Click Revert Allocations.
    The system deletes all unlocked allocation lines associated with this general ledger expense and changes the State field on the general ledger record to Unallocated.

To revert allocations associated with an allocation rule:

  1. Navigate to IT Finance > Cost Model > Allocation Rules.
  2. Click an active rule that has already been used when a general ledger expense was processed.
  3. Click Revert Allocations.

You can also revert allocations on the workbench.

7 Viewing Allocation Lines

After you complete an allocation, you can view the allocation lines that the application created.

  1. Open the allocation line that you want to modify through either of the following methods:
    • To access an allocation through the general ledger that created it:
    1. Navigate to IT Finance > General Ledger > Groomed Expenses.
    2. Open the expense that generated the allocation you are looking for.
    3. In the Cost Allocations related list, click the allocation number.
    • To access an allocation from a list of all allocations:
    1. Navigate to IT Finance > Cost Model > Allocation Lines.
    2. Click the allocation number.
  2. Verify or change any of the editable fields on the form (see table).
  3. Click Update.
The Cost Allocation form

Field Description
Number Auto-generated identification number for the for the allocation.
Amount The amount of this allocation.
Fiscal period The fiscal period this expense belongs in.
GL Entry The groomed expense from which this allocation line was derived.
Allocation rule The allocation rule that met the criteria to create this expense line.
Allocation method The allocation rule's method that was used to create this expense line.
Primary segment The segment in the IT chart of accounts that this allocated expense is assigned to.
State The state of the allocation. After an allocation is generated, the state is Processed. If you lock the allocation, the state changes to Locked.
Final If the allocation line was processed by a rule that is marked final or if no subsequent rule can process this allocation. Reports are run on final allocation lines.
Fixed cost If the cost is a fixed cost.
Capital cost If the cost is a capital cost.
Unallocated If this expense is not allocated to any segment.
Bucket The bucket that the allocation belongs to. This bucket is taken from the expense during allocation.
Sub-bucket The sub-bucket that the allocation belongs to. The sub-bucket is also taken from the expense during allocation.
[Dimensions] The segments in the IT chart of accounts. The segments fields that contain values are the segments that you specified in the allocation methods that processed the allocation. If more than one method processes an allocation, all the allocated segments specified in all the methods contain a value. To make a change, click the lookup icon next for each dimension and select the relevant record.
