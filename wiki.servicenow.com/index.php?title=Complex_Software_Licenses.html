1 Overview

The Complex Software License module relates usage, infrastructure, and other complex licenses to a single record to control license usage. This is useful for tracking compliance for a software package that is licensed for both named users and CIs, or for multiple departments in multiple locations throughout an organization. This functionality requires the Software License Management plugin.

Note: A significantly improved and updated application named Software Asset Management was included in the Berlin release. For information about upgrading from Software License Management to Software Asset Management, see Upgrading to Software Asset Management.


2 Compliance Filter

The compliance filter for complex licenses uses the following locality fields to define entitled users or CIs.

  • Entitled company: All users or configuration items (CI) at all locations of this company, in all departments are entitled to use this software package.
  • Entitled location: CIs and users who are assigned to this company location in any department are entitled to use this software package.
  • Entitled department: Only the users or CIs in this department at this company location are entitled to use this software package.

The complex license displays the license count totals from related usage and infrastructure licenses that are using the same software package. The complex license provides links to the related licenses.

Possible compliance levels are:

  • Out of compliance: More licenses are being consumed than were purchased. There are more users or CIs using this license than the license allows, and some users or CIs might not be entitled to use this software package.
  • In compliance: All the related licenses are, themselves, compliant.

3 License Types

The out-of-box system provides three software license types (or None). These types are not used as filters, but can be added to conditions to drive compliance to suit an organization's processes.

  • Cluster: Cluster licenses check compliance based on total license count from all related licenses and does not permit compliance checking in the related licenses. Use cluster licenses to manage a software package that contains licenses with different expiration dates or license keys.
  • Enterprise Software License: Use this type for all related infrastructure and usage licenses that calculate compliance based on license count.
  • Enterprise Software Subscription: Use this type for all related infrastructure and usage licenses that calculate compliance based on a subscription model.

You can add types to this list as you would add selections to any choice list in the platform. To use a license type in compliance checking, ensure that it is included in the Condition Check Definitions for the compliance-level field in the slm_license_complex table.

4 Compliance Types

The available compliance types in the out-of-box system are:

  • License count: Checks compliance on license count only. Use this type for Cluster licenses that roll up usage statistics from infrastructure licenses and for Enterprise Software Licenses.
  • Unlimited: Use this type for licenses that do not require compliance checking, such as a software package with no limit on the number of licensed users or CIs permitted.

5 Compliance Checking

Software License Management provides a license checking UI Action to recalculate compliance levels, and expiration levels. Click SLM Complex License Checker after updating fields in the form to recalculate these measurements. Only users with the slm_admin role can use the License Checker. In the complex license, the license checker calculates whether it is compliant with the immediate children only, based on two factors: if the child record is internally In compliance, and if the child record is compliant with the parent record. The license checker in a complex license cannot check compliance within other records. Before running the license checker in a complex license, ensure that all the related licenses are compliant from the bottom of the hierarchy upward.



5.1 License Compliance Checker

To check all the licenses in the system for compliance at once, select Software License Management > License Compliance Checker. The platform does not display a prompt, but calculates the compliance of each record for each license type immediately. The progress is displayed above the list.



6 Creating Complex Software Licenses

The following example shows how complex and infrastructure software licenses might be used to calculate compliance for Oracle licenses used by different departments in a multi-national corporation. The complex license at the top of the diagram shows the license counts for the entire corporation. Three related complex licenses for each of the international offices (Entitled Locations) account for their licenses. Attached to each of the complex location licenses are infrastructure licenses for each entitled department, showing the actual computers on which the Oracle software package is licensed to run. Compliance is calculated from the bottom of the diagram upwards. Each department license must be compliant before the complex license from its home office can be compliant. All the international offices' licenses must then be compliant with the corporate license.

Note: Software License Management supports the addition of usage licenses to this structure as well. Both types of licenses have the same compliance requirements when used with complex parent licenses.




Note: When creating any type of license for a software package, ensure that no overlapping license coverage already exists for the same locality. This can result in seemingly inaccurate license counts in competing records.


In this example, we start with a Software License Contract, but this is not necessary. You can create complex licenses without relating them to contracts.

  1. Create a new Software License Contract for the software package.
  2. In the Software License Bases Related List, click New.
    A Wizard appears for selecting a new license base.
  3. Click Complex Software License.
  4. Complete the form, being sure to include the following elements:
    • Entitled Company: This is the level of the entitlement filter for the top of the hierarchy.
    • Expiration date: This is used to calculate the expiration level and does not affect compliance.
    • License type: Select the Enterprise Software License type for all licenses that roll up license count from compliance checks in related licenses.
    • Compliance type:
      • License count: Check compliance on license count only. Use this type for all cluster licenses.
      • Unlimited: Used for unlimited use of this software package by agreement with the vendor.
    • Parent: Select the complex license that is the parent to this license. This establishes the upward hierarchy for calculating compliance.
    • Software package: This is not necessary for the corporate license, which rolls up the license counts from software packages specified in usage and infrastructure licenses.
  5. Click Submit.
    The new complex license for the ACME Corporation appears in the Software License Bases Related List in the Software License Contract.


  6. Select the complex license for the company from the Related List.
  7. Click New in the Related Licenses Related List and select Complex Software License in the Wizard.
  8. Create complex licenses for the USA office location.
    • Entitled Company and Entitled Location: These are the levels of the entitlement filter for this complex license. For this example, enter Acme Corporation and Acme USA.
    • Expiration date: This is used to calculate the expiration level and does not affect compliance.
    • License type: Select the Enterprise Software License type for all licenses that roll up license count from compliance checks in related licenses.
    • Compliance type: Select License count, which is used in all condition checks to calculate compliance. Select Unlimited only if you are entitled to unlimited use of this software package by agreement with the vendor.
    • Software package: This is not necessary at this level. This complex license has related infrastructure license records that calculate compliance for different software packages.
  9. Click Submit.
  10. Repeat the above procedure and create complex licenses for the other office locations (Acme Europe and Acme Asia).
    The licenses for these localities must appear in the Related List in the complex company license. Ensure that the record number of the company license appears in the Parent field of each related location license. All license counts start at zero because the infrastructure licenses, where compliance is calculated, have not been created yet.


  11. Select the complex license for the USA office from the Related List.
  12. Click New in the Related Licenses Related List.
  13. In the Wizard, select Infrastructure Software License.
  14. Create an infrastructure license for the IT department.
    a. Add company, location, and department locality filters in this record.
    b. Select a Base CI type to limit the scope of the license. In this case, we select Computer [cmdb_ci_computer].
    Remember that this type allows the software package to be licensed on all servers and workstations (UNIX, Windows, AIX, etc.). In this example we select CPU count as the Limiting CI attribute, which grants one license to each CPU on the installation computers. If you do not select an attribute, a single license is counted for each entitled computer.
    c. Enter the name of the software package. In our example, we use Oracle 11i.
    d. Enter an entitled license count in the Entitled field.
    In our example the license count is set high to accommodate computers with multiple CPUs.
  15. Right-click in the header bar and select Save from the context menu.
  16. Run the license checker to calculate compliance for this license.
    The resulting count is rolled up in the parent complex license.


  17. Using the procedure above, create infrastructure licenses for the other localities, using the same software package.
  18. Make sure the record number of the parent complex license appears in the Parent field of each infrastructure license.
    This relationship must be established for all the license counts to roll up.
  19. Check each infrastructure license for compliance.
  20. When the infrastructure licenses are all compliant, run the compliance checkers in each of the complex location licenses.


  21. When all the complex location licenses are in compliance, return to the company license and run the license checker.
    If the related location licenses are compliant, the parent company license is also compliant.
