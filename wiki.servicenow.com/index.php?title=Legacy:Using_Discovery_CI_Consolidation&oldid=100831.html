1 Overview

Discovery CI Consolidation (DCIC) uses Duplicate CI Consolidators (or “consolidators” for short) to take some action to resolve a situation where two CIs are duplicates of one another. A consolidator can do anything at all, so long as the result is that the two CIs identified as duplicates are no longer duplicates when the consolidator is finished.

The simplest way to make the duplicate issue go away is to delete one of the duplicates – but that runs the risk of losing some useful information, so usually you'll want something a little more sophisticated.

Probably the consolidator you'll use the most is one that merges information from the just-discovered CI (known as the “source” into the duplicate CI (known as the “target”), then deletes the source. This leaves you with one CI (so no duplicate), and that CI will have all the information from both of the originals.

However, there are situations in which you might want to do something slightly more subtle. For example, perhaps some imported CIs look like duplicates, but are actually computers that are not yet in production. You wouldn't want to merge these CIs with duplicate production CIs, and you wouldn't want to delete either of them – so instead, your consolidator could change some information (perhaps stuffing a value into a custom field to indicated that the CI is pre-production) so that your finders would no longer see these CIs as duplicates.

2 How Consolidators Work

Consolidators have an ordered list of actions associated with them. When a consolidator is invoked by a finder, these actions are run in order (assuming they are active).  There can be any number of actions, but usually there are just a few.

CI Consolidation has three kinds of actions built in: merge, delete, and script. The merge action combines the data (including, if desired, the related lists) of two CIs.  This action is itself very configurable. The delete action is simple: it deletes either the source or the target CI.  The script action is anything you want it to be – it can modify data, filter which fields are merged, cancel the consolidator, and probably a host of things we never thought of.

3 Creating or Modifying Consolidators

You can see a list of all your system's consolidators by clicking Duplicate CI Consolidators. If you click on the name a consolidator in the list, you'll get a form similar to this example:


The name and description fields are there for your benefit; CI Consolidation makes no use of them.  The CI Consolidator Actions list is where all the action is. The actions in the example are sorted by order, so they're seen in the order that they will run.  You can see that only the first and last actions are active; the script action will not run.
