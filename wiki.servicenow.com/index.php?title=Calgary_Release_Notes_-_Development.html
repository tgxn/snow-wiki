Calgary Release Notes - Development

Development, Email & Integrations Enhancements

The Calgary release provides features for extending your ServiceNow instance, building custom applications, and taking advantage of advanced functionality.


Click to sort the features list:

Application Creation

Streamlines custom application creation by making it easier to create tables, linking code changes to applications, and publishing finished applications to update sets.


Automatic for all instances.

Table Administration

Simplifies and consolidates table creation and editing into one convenient location and uses a standardized user interface.


Automatic for all instances.

Schema Map

Improved representation and navigation of tables and relationship types. Ability to view table columns, navigate to table data and dictionary entries, and print the schema map.


Automatic for all instances.

Scripting APIs

Eliminates the need to make Java Packages calls from your scripts. Includes Packages Call Removal Tool to migrate custom code.


Activate the Packages Call Removal Tool plugin.

Update Sets

Ability to change certain fields and still receive updates to others, enhancements to work with application file records, and usability improvements for navigating, previewing, and committing.


Automatic for all instances.

System Dictionary

Redesigned the form to improve usability, streamlined the list of available field types, and added the ability to modify the type and length of a field.


Automatic for all instances.

Application Menus

New category options are available for Application Menus (formerly called Applications).


Automatic for all instances.

GlideForm

A new method to remove messages that were previously added with addErrorMessage() and addInfoMessage().


Automatic for all instances.

Context Menus

New JavaScript variable is available for passing the value of the Order field to a dynamic actions script.


Automatic for all instances.

Slow Query Logs

New system diagnostics tool for tracking slow queries provides better insight into how queries affect platform performance.


Automatic for all instances.

Non-Interactive Sessions

Creates a security distinction between interactive and non-interactive users.


Activate the Non-Interactive Sessions plugin.

Web Service Import Sets

New system property controls how the instance inserts records from web service calls into an import set table.


Automatic for all instances.

Web Service Consumer Support

The SOAP message module supports WSDL version 1.2.


Automatic for all instances.

Inbound Email Actions

Inbound email actions now create new users from the sender's full email address. A new system property allows ServiceNow to process inbound email from locked out users. The gs.createUser() method is deprecated.


Automatic for new instances.

Email Notifications

The email.setFieldsInCurrent() method is deprecated.


Automatic for new instances.

Email Client

A new system property displays additional columns in autocomplete. A new ACL rule controls access to and visibility of the email client.


Automatic for new instances.

Configuring Email

New system properties specify the email address to use as the reply to address and the maximum number of recipients for a single notification. The default SMTP server changed to relay.


Automatic for new instances.

Online Schema Changes

Modifying indexes or columns on a MySQL database table, such as when committing an update set, no longer locks that table from writes. A new table attribute controls this behavior.


Automatic for all instances.

<g:ui_reference/> Jelly Tag Enhancement

The <g:ui_reference/> Jelly tag has been enhanced to allow you to specify a reference qualifier.


Automatic for all instances.
