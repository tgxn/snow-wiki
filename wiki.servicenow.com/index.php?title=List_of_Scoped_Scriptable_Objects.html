Scripting in Scoped Applications

1 Overview

The following is a complete list of objects and methods that are allowed in server-side scripts running in a scoped application. The APIs are grouped by their namespace. ServiceNow core APIs do not need a namespace qualifier.

The scoped API is available starting with the Fuji release.

Note: Scripts in the global scope and scripts created prior to the Fuji release can access any method in the comprehensive system API, including GlideRecord, GlideSystem, and GlideElement.


2 Core APIs

These APIs do not need a namespace qualifier.

Object Description
GlideAggregate Provides data access APIs that are an extension of GlideRecord and provides database aggregation (COUNT, SUM, MIN, MAX, AVG) queries.
GlideDate Provides APIs to deal with date calculations and working with date fields..
GlideDateTime Provides APIs to deal with date time calculations and working with date/time fields.
GlideDuration Provides APIs to deal with time duration calculations and working with duration fields.
GlideElement Provides APIs for dealing with fields and their values. GlideElement methods are available for the fields of the current GlideRecord.
GlideQueryCondition Provides APIs to build complex queries by adding additional ADD or OR conditions to the current condition.
GlideRecord Provides data access APIs to retrieve, update, and delete records from a table.
GlideRecordSecure Performs GlideRecord functions and also enforces ACLs.
GlideSchedule Provides APIs to perform operations on GlideSchedule objects, such as adding new schedule segments to a schedule, determine if a datetime is within the schedule, and setting the schedule timezone.
GlideScopedEvaluator Provides APIs to evaluate scripts from a GlideRecord field.
GlideSystem Provides APIs to access system information.
GlideTableHierarchy Provides APIs for handling information about table relationships.
GlideTime Provides APIs to deal with time calculations and working with time fields.
GlideUser Provides APIs to find information about the current user including their roles.

2.1 Core XML APIs

Use these APIs to process an XML document. These APIs do not need a namespace qualifier.

Object Description
XMLDocument2 Represents a XML document with functions to access and modify the XML document.
XMLNode Represents a XML element with functions to access and modify the XML element.
XMLNodeIterator Iterates through elements of a XML Document.

3 Integration APIs - Namespace: sn_ws

Use these APIs to send a message to a web service provider. You must specify the namespace qualifier sn_ws when using these methods.

Object Description
RESTMessageV2 Sends REST messages.
RESTResponseV2 Returns data in the REST message response.
SOAPMessageV2 Sends a SOAP message.
SOAPResponseV2 Returns data in the SOAP message response.
