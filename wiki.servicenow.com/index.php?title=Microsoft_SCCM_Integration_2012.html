1 Overview

The Microsoft System Center Configuration Manager (SCCM) 2012 integration is a one direction import of SCCM data into ServiceNow's Configuration Management Database (CMDB). Scheduled imports bring relevant data from SCCM tables into the CMDB through the SQL database and map them to the ServiceNow instance. Beginning with this version, either a full data import or an incremental import can be performed. The import is achieved using a JDBC connection via the MID Server. This functionality requires the Integration - Microsoft SCCM plugin. This integration is available starting with the Dublin release.

3 How the SCCM 2012 Integration Works

3.1 Determine the Identity of the imported Configuration Item

Version 2012 Microsoft SCCM Integration
  1. A periodic schedule in the SQL database imports the Microsoft SCCM data into the SCCM Computer Identity [imp_sccm_computer_identity] table. This is configurable from within a ServiceNow instance. It does a JOIN of various database views from within SCCM 2012. Here's a list of tables that it imports from.
    • v_GS_COMPUTER_SYSTEM
    • v_GS_SYSTEM
    • v_GS_OPERATING_SYSTEM
    • v_GS_SYSTEM_ENCLOSURE
    • v_GS_WORKSTATION_STATUS
    • v_GS_PC_BIOS
    • v_GS_COMPUTER_SYSTEM_PRODUCT
    • v_GS_BASEBOARD
  2. A schedule in the ServiceNow Microsoft SCCM application imports these tables into the Computer Identity table. This must be the first table to be imported because it is important for it to determine the identity of a given imported configuration item (CI). In other words, based on the name, serial number, model and etc, the transform logic will create a new CI if it doesn't already exist in the CMDB and update if the CI already is in the CMDB.

3.2 Data Imported by ServiceNow

After the Computer Identity data source has been imported, the ServiceNow scheduled import continues with the following information from the SCCM data source and its relevant objects:

  • Operating System: ram, os_version, os_service_pack
  • Processor: cpu_core_thread, cpu_speed, os_address_width, cpu_manufacturer, cpu_core_count, cpu_name, cpu_type, cpu_count
  • Disk: disk_type, short_description, manufacturer, device_id, last_discovered, name, computer, disk_space
  • Network: dhcp_enabled, mac_address, name, cmdb_ci, netmask, last_discovered, ip_address, default_gateway
  • Software: Software packages installed (Microsoft Office, Adobe Photoshop, iTunes, etc.)


4 Setting Up SCCM 2012

4.1 Task 1: Connect to the SQL Server Database

Establish connection from within the ServiceNow Microsoft SCCM application to the SQL Server database in which Microsoft SCCM stores its data objects. This is the database in which you create the SCCM Computer Identity [imp_sccm_computer_Identity] data source table. Navigate to Integration - Microsoft SCCM 2012 > Setup and configure connection to the SQL Server database. Make sure to select a MID Server. Click the Test data source connections UI Action to make sure you can connect to this database before continuing.

Connecting to the SQL Server Database‎

4.2 Task 2: Configuring the Transform Map

To configure a transform map (Incremental Import, SCCM 2012 Software - Full Import, or Incremental Import (SAM enabled)), navigate to Integration - Microsoft SCCM 2012 > Web Services > Software and select the transform map from the Web Service Transform Maps list.

SCCM 2012 Transform Maps

4.3 Task 3: Schedule an Import

After data has been imported from SCCM into the SCCM Computer Identity [imp_sccm_computer_identity] table in the SQL Server database, schedule an import from that table into the ServiceNow instance using the default transform map provided in the plugin.

4.3.1 Creating Compatible Data Sources

If you want to access your SQL Server databases using Windows domain authenticated accounts, you must first create a JDBC data source that uses Integrated Authentication for Windows. See SQL Server Integrated Authentication for Windows.

4.3.2 Creating an Import Schedule

  1. Navigate to Integration - Microsoft SCCM 2012 > Scheduled Import.
  2. Select the SCCM System import from the list.
    SCCM 2012 Import Schedule‎

  3. Select an import type (Day, Week, etc.) and when the import should run.
    The selection in the Run type field presents different time options.
  4. Select the Conditional check box to display a field in which to define an import condition.
  5. Select Execute pre-import script or Execute post-import script check box to enter a script that runs before or after the import.
    For example, you can create log entries for when the import starts and stops by using scripts in these fields.
  6. Click Update to schedule the import or click Execute Now to perform the import immediately.
    After ServiceNow imports data from the SCCM Computer Identity [imp_sccm_computer_identity] table, the import automatically runs the remaining import sets.

5 Activating the Plugin

The Integration - Microsoft SCCM 2012 plugin can be activated by an administrator and also activates the Integration - JDBC plugin.

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.

6 Enhancements
