Fall 2010 Stable 1 Notable Changes

From ServiceNow Wiki

The following are notable changes you may encounter with our Fall 2010 Stable 1 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before. For additional changes, see Fall 2010 Stable 1 Fixes.

1 Changes for Fall 2010 Stable 1

1.1 Performance Enhancement - Global Search

Global search performance across large numbers of records has been improved by decreasing the number of lucene queries. This will be noticeable to any users who search globally with search terms which are not task numbers.

1.2 Performance Enhancement - Database Limited Activities

The database's memory efficiency has been improved. This affects the UI minimally, but will significantly improve throughput on database limited activities (e.g. Discovery processes).

1.3 Performance Enhancement - SOAP Web Services

The cost of creating a controller has been reduced. This does not affect the UI, but will increase performance in high volume SOAP environments.

1.4 Discovery Credentials Order

Discovery Credentials can be assigned an order value in the Credentials Form, which forces Discovery to try all the available credentials in a configured sequence.

1.5 sys_report Roles and visibility are enforced wherever a report is viewed

Before this release, users might be allowed to see reports where they did not have the proper Roles as specified on the sys_report record if that report was placed on their homepage. This is no longer allowed. Users must have the required roles to view a report. Visibility is also enforced. A report must either be Global, visible to that user, or visible to one of the user's groups in order for a user to view it in either the Reports portal or a homepage.

1.6 Conditional and Condition fields added to Scheduled Jobs

A new Boolean field, Conditional, has been added to Scheduled Jobs. If checked, a new Condition script field can be used to script the condition under which the job will run. If the last line of the script evaluates to true, the job executes. If the last line of the script evaluates to false, the job will not execute.

1.7 "Omit if no records" field added to Scheduled Reports

A new Boolean field, "Omit if no records", has been added to Scheduled Reports. If checked, and if the Conditional field is also checked, the referenced Report's filter will be evaluated to see if any records are returned. If none are, the report will not be emailed.

1.8 onChange Client Script Error Messages

onChange Client Scripts that generate a JavaScript error now display a field message with specific information about the bad script if the user viewing the form has the client_script_admin role, and a generic error message otherwise. Before, JavaScript errors caused by Client Scripts could cause subsequent scripts to not execute on a form. Now the errors are safely caught, allowing other scripts to still run.
