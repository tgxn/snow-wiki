1 Overview

To use Chef configuration automation, set up a user account and optionally set whether to enable change management to control any updates to node definitions. The procedures described in this article require the admin role.

2 Prerequisites

Chef version 12 and earlier are supported.

Before you can use ServiceNow as a Chef external node classifier, an administrator must:

  • Install the Chef server software: The Chef server communicates with Chef nodes and the ServiceNow instance. For more information on how to install the server software, see the Chef documentation.
  • Install a MID Server: The MID server must be configured to access both the Chef server and ServiceNow instance. For information on installing a MID Server, see MID Server Installation.

3 Setting Up Key Stores

The ServiceNow instance requires an X.509 certificate to communicate with Chef resources. To set up a key store:

3.1 Generate a Private Key and Save to a .pem File

Setting Up a Key Store

3.2 Generate a pkcs12 File from the .pem File

Enter the following command:

openssl pkcs12 -export -nocerts -name <alias> -inkey <file from chef.pem> -out <output file>

For example:

openssl pkcs12 -export -nocerts -name mycert -inkey chef_user.pem -out chef_user.pkcs12

3.3 Create a Key Store

  1. Navigate to System Definition > Certificates (X.509 Certificate) and create a new record.
  2. Set Type to PKCS12 Key Store and then press Enter.
  3. Enter the Key store password that you used when you generated the certificate.
  4. Attach the pkcs12 file that you just generated.
Note: The chef_user.pkcs12 file should be attached in the X.509 Certificate record and not in the user record.

4 Setting Up a User Account

A user account contains the access information required to communicate with Chef resources. Before the ServiceNow system can communicate with Chef, an administrator must define a credential record for a user on a Chef server. The user account on the Chef server must have rights to run Chef and view all Chef files. See the official Chef documentation for more information about setting up users on the Chef server.

  1. Navigate to Chef > Setup > User Accounts.
  2. Click New.
  3. Enter the Chef user information (see table).
  4. To validate the key store alias, click the Validate Key store alias related link.
  5. Click Submit.
Setting Up a Chef User Account

Field Description
Name An alias for the Chef user, to be used on the ServiceNow instance.
User The Chef user name with access to the Chef resource.
Key store The X.509 certificate required by the ServiceNow instance to communicate with Chef resources. Click the reference lookup icon and select the certificate.
Key store alias Enter the alias that you used when you created the .pkcs12 file.

5 Configuring Properties for Configuration Automation

  1. Navigate to Configuration Automation > Properties.
  2. Edit the properties (see table).
  3. Click Save.
Property Description
[glide.config_auto.node_definition.use_change] Enable change management to control any updates to node definitions
  • Type: Yes|No
  • Default value: No
[glide.config_auto.ci_assignment_use_change] Enables change management to control any assignments of definitions to CIs.
  • Type: Yes|No
  • Default value: No
