1 Fixes for the Spring 2010 Stable 2 release

1.1 PRB291977: Reports

1.1.1 Example: Clicking "Execute Now" on a "Run once" Scheduled Report could result in duplicate emails being sent

Resolution: Duplicate emails are no longer sent.

1.2 PRB043344: Reports

1.2.1 Example: The "Stacked field" on a bar chart report could not be set to "-- None --" after it had a different value

Resolution: The Stacked field will now preserve the -- None -- value when the report is saved.

1.3 PRB302308: UI

1.3.1 Example: List Control did not honor domain. e.g., a List Control for the global domain was not applied to users in other domains

Resolution: List Control now works as expected with domains. A List Control for a domain will apply to subdomains.

1.4 PRB327322: UI

1.4.1 Example: In List v2, Filter Out/Show Matching did not work for currency fields with a value of zero

Resolution: Filter Out/Show Matching now work properly for currency fields with a value of zero in List v2.

1.5 PRB328826: UI

1.5.1 Example: Filters using JavaScript functions (e.g., Group = javascript:getMyGroups()) only displayed the first display value in the filter breadcrumb (e.g., "Network" instead of "Network, Desktop, Software"

Resolution: Filters using JavaScript functions now display the correct list of display values in the breadcrumb.

1.6 PRB330279: Exports

1.6.1 Example: Related lists were not supported in the PDF Detailed export of a list

Resolution: A new property has been added, "glide.export.pdf.list.related_list", defaulting to false. If true, related lists will be included in the PDF Detailed export of a list.

1.7 PRB297476: i18n

1.7.1 Example: Several items on the "Password needs reset" page were not translated

Resolution: The whole "Password needs reset" page is now translated.

1.8 PRB268944: i18n

1.8.1 Example: The "Import Set Tables" message had an incorrect French translation

Resolution: The translation was fixed.

1.9 PRB331921: UI

1.9.1 Example: Many-to-many related list labels were not translated in List v2

Resolution: Many-to-many related list labels are now translated in List v2

1.10 PRB332387: Discovery

1.10.1 Example: The DNS Domain field did not populate properly

Resolution: The DNS Domain field now populates properly. The "HostnameJS" Script Include was fixed to do this.

1.11 PRB332393: Discovery

1.11.1 Example: The glide.discovery.hostname.always_update property was no longer honored to update the hostname of a CI

Resolution: The DiscoveryIDSensor Script Include was fixed to honor the property.

1.12 PRB343213: UI

1.12.1 Example: Show Matching/Filter Out did not work properly for zero-value choice list fields (e.g., a Priority of "0 - Critical" with a value of zero)

Resolution: Show Matching/Filter Out now works properly for zero-value choice list fields

1.13 PRB345783: UI

1.13.1 Example: Duplicate roles could appear in the roles filter

Resolution: This could happen if a record with a user_roles field was deleted, then imported. The behind-the-scenes kb_knowledgeroles table that stores information about the roles selected in the user_roles field did not have its corresponding records deleted when the article was deleted. Now those records are cleared along with the parent record's deletion.

1.14 PRB346005: UI

1.14.1 Example: Domain fields were not initialized to empty on Search or Update forms, like reference fields are

Resolution: Domain fields are now initialized to empty on Search and Update forms.

1.15 PRB345919: i18n

1.15.1 Example: Survey names were not properly translated

Resolution: Survey names are now properly translated.

1.16 PRB320582: UI

1.16.1 Example: List editing was prevented for a dependent reference field if the user couldn't write to the field it was dependent on

Resolution: List editing is no longer prevented in this situation.

1.17 PRB314705: Reports

1.17.1 Example: In Oracle, a trend chart with a filter condition spanning the beginning of a year (e.g., Date between 9-1-2009 and 2-28-2010) returned data in the incorrect trend bar

Resolution: The data is now reflected in the correct trend bar.

1.18 PRB304810: i18n

1.18.1 Example: Using a language other than English, editing a related list slushbucket to remove all entries returned to an empty record

Resolution: Added a check for the translated empty message "--None--" or "-- None --", and added translations for both messages.

1.19 PRB303849: Performance

1.19.1 Example: If a query is issued that returns a large enough number of records in MySQL, an instance could run out of memory

Resolution: MySQL now runs in streaming mode to prevent this problem.

1.20 PRB316222: Update Sets

1.20.1 Example: An admin could delete the Default or BaseLine update set

Resolution: An ACL on sys_update_set was put in place to prevent this.

1.21 PRB309282: Domain

1.21.1 Example: The Domain reference picker in the title bar showed an incorrect domain at login until the page was refreshed

Resolution: The Domain reference picker now shows the correct domain on login.

1.22 PRB315853: Update Sets

1.22.1 Example: Updates to Private properties (with the Private checkbox checked on the sys_properties record) were wrongly captured in update sets

Resolution: Private properties are no longer captured in update sets.

1.23 PRB252198: Service Catalog

1.23.1 Example: The Template field showed on the Catalog Item form for all item types

Resolution: A new client script on sc_cat_item now hides the Template field if the Item is neither a Template nor Record Producer

1.24 PRB331896: UI

1.24.1 Example: A photo field in a reference popup generated a JavaScript error

Resolution: Protection was added to avoid the JavaScript error in the sys_popup view

1.25 PRB330306: Email Client

1.25.1 Example: Special characters like quotes in the Subject did not display properly in the email client

Resolution: The Email Client subject is now properly escaped.

1.26 PRB350446: UI

1.26.1 Example: In List v2, list filters did not remain after a UI action was executed

Resolution: Stack parameters are now saved on a pop, so that they can be reapplied if the next push matches the pop. This ensures the filter is saved between list displays.

1.27 PRB359625: UI

1.27.1 Example: Spell check could add duplicate letters after clicking "resume editing" a second time

Resolution: The mod array is now reset each time spelling is checked to prevent this.

1.28 PRB319910: Database

1.28.1 Example: Change Phase and Release Phase "Order" field was not getting populated

Resolution: The order column in task was conflicting with the order fields in change_phase and release_phase tables. To resolve, values from the redundant columns are copied to the order column in task with a SQL statement, and the redundant columns are dropped.

1.29 PRB296323: Exports

1.29.1 Example: Exports that the user chose to have emailed would not honor the user's domain.

Resolution: Emailed exports now honor the requesting user's domain.

1.30 PRB361323: CMDB

1.30.1 Example: Newly expanded BSM nodes were colored the same as selected nodes, making it impossible to tell what was actually selected

Resolution: Coloring had been changed to make it clear what is selected and what is not in the BSM Map.

1.31 PRB361400: UI

1.31.1 Example: In List v2, any value in the glide.ui.list_mechanic.roles property prevented all non-admin users from personalizing lists with the gear icon

Resolution: The property is now correctly honored in List v2.

1.32 PRB362518: UI

1.32.1 Example: For grouped lists, the [sysparm_total_records] variable was wrongly displayed in place of the number of records in each group

Resolution: The variable now correctly displays the value and not the name of the variable.

1.33 PRB363710: Reports

1.33.1 Example: List report columns might not be correct in a list report modified and run again by a domain user

Resolution: List report columns now accurately represent what was requested when run by a domain user.

1.34 PRB042905, PRB043948: Email Client

1.34.1 Example: Re-loading a form on which the email client was used caused the email client to launch again

Resolution: The email client no longer requires the form to be saved, eliminating the cause of this problem.

1.35 PRB286235: UI

1.35.1 Example: Field messages (error, info, warning) set on a specific field might automatically scroll the form to an undesired location

Resolution:
  1. A new property overrides whether automatic message scrolling will occur at all (glide.ui.scroll_to_message_field).
  2. Scrolling occurs automatically when the field label or the message is offscreen, and scrolls the form to the field label.
  3. g_form.setFieldMsg() now takes an optional true/false parameter that controls whether scrolling occurs when the system default behavior needs to be overridden on a case-by-case basis. e.g., g_form.showFieldMsg("short_description", "Please enter", "error", false);

1.36 PRB362126: Reports

1.36.1 Example: The type-ahead filter atop the Reports portal might wrongly remove the first section's header

Resolution: The filter no longer removes the first section's header inappropriately.

1.37 PRB365329: UI

1.37.1 Example: Column header names showed as "null" when viewing any list in printer-friendly version

Resolution: Column headers are now rendered properly.

1.38 PRB365273: Metrics

1.38.1 Example: The out-of-box Metric Definition MTRC0000007 improperly queried on the type field vs. the id.type field

Resolution: The query has been changed in that metric definition to query the correct field.

1.39 PRB365443: UI

1.39.1 Example: The Day in Week field type might list days out of order

Resolution: The days are now ordered by value and not label.

1.40 PRB290610: UI

1.40.1 Example: "Insert web link" toolbar button in HTML editor might not actually insert a link in IE8

Resolution: Improved IE link functionality when an empty paragraph is selected

1.41 PRB366599: UI

1.41.1 Example: Clicking the "Update Set" link in the update set picker might not work properly in IE8

Resolution: Removed the association of the label in the anchor tag from the select to fix this issue

1.42 PRB369897: Reports

1.42.1 Example: Expiring Asset Contracts count gauge had incorrect query conditions that did not render properly in a filter

Resolution: The conditions have been fixed.

1.43 PRB370400: Update Sets

1.43.1 Example: A previously skipped update in an update set (skipped for legitimate reasons) would wrongly be skipped in all future update sets

Resolution: The "Commit Update Set" UI Action was fixed to prevent this problem.

1.44 PRB370265: UI

1.44.1 Example: Typing NULL into a reference field on a multiple update form did not properly clear the field for updated records

Resolution: Typing NULL into a reference field on a multiple update form now properly clears the field for updated records

1.45 PRB268926: Workflow

1.45.1 Example: An extremely large workflow might not be properly captured in an update set

Resolution: The extremely large workflow might create full sequences that are too long. Now if there are more than 5000 activities listed in the full or expected sequences, the list of sequences is not generated. The sequences are only used to optimize the Stage widget display, so giving up on the list only impacts which stages are displayed in that widget.

1.46 PRB372026: Exports

1.46.1 Example: When exporting reports to CSV, durations longer than 9,999,999 seconds display in scientific notation in Excel

Resolution: Durations now export as numeric regardless of length.

1.47 PRB373586: Service Catalog

1.47.1 Example: When a Requested Item was created via script, the variables weren't available

Resolution: Variables are once more available when creating catalog items through script.

1.48 PRB314795: UI

1.48.1 Example: Using IE6 for whatever reason, when there was a large number of input elements on a form, typing in a journal field could result both in a delay between keystrokes and in high CPU utilization

Resolution: Code has been modified so this issue no longer presents.

1.49 PRB328508: Discovery

1.49.1 Example: 32-bit software was not discovered on a 64-bit Windows OS

Resolution: Support has been added to the "Process Installed Software" discovery sensor for discovering 32-bit software running on 64-bit Windows operating systems.

1.50 PRB369778: UI

1.50.1 = Example: Editing related list contents with the slushbucket UI might not properly return a user to the form if the Tiny URL plugin was installed

Resolution: Editing related list contents with the slushbucket UI now properly returns a user to the form if the Tiny URL plugin is installed.

1.51 PRB375790: UI

1.51.1 Example: A "Go to" navigation on a partial record number might only return a single record even if multiple records matched the partial number

Resolution: An "ends with" query is done on a Number field for partial number Go to navigations.

1.52 PRB373796: Workflow

1.52.1 Example: Several workflow tables did not have out-of-box ACLs

Resolution: ACLs have been added for workflow tables lacking them.

1.53 PRB373518: Service Catalog

1.53.1 Example: Global Text Search did not honor the "no search" checkbox on a catalog item when determining result count

Resolution: Global Text Search now honors the "no search" checkbox on a catalog item when determining result count.

1.54 PRB149397: Import Sets

1.54.1 Example: If a scheduled import resulted in an error, subsequent dependent imports would still run

Resolution: A property can be created to cancel dependent scheduled imports if the parent import results in an error. Additional information can be found at this link.

1.55 PRB369299: Discovery

1.55.1 Example: While performing a networks discovery, some routers are not traversed

Resolution: Certain kinds of router configurations were not being properly analyzed to find attached networks. All router configurations should now be analyzed correctly.
