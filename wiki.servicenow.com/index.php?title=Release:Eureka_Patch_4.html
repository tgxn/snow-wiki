1 Overview

Asset Management PRB603982 The Enforce CI Verification setting for configuration items does not work when the model category gets changed on a model record that matches the CI's model category.
Click the plus for steps to reproduce

Without Discovery:

  1. Go to the cmdb_model_category table and find the record for Computer [cmdb_ci_computer].
    This record should have the asset class set to Hardware [alm_hardware].
  2. Open this record and set the Enforce CI verification field to true.
  3. Go to the cmdb_ci_computer table and create a new record with the following values:
    • Name: Any value (for example, Test Computer)
    • Model ID: Choose a model where the category is only "Computer" (for example, Asus G Series)
  4. Save the record.
    You should see in this record that there is no asset value.
  5. Go to the cmdb_hardware_product_model table and choose a model record that only has "Computer" for the model category. You can choose the same record as above or a different record.
    For this step and the previous step, make sure that the asset tracking strategy for the model record(s) you select is not set to "Don't create assets".
  6. Open this record.
  7. For the model categories, remove the "Computer" reference and save the record.
  8. Reopen this same record and add the "Computer" reference again.
  9. If you refresh the cmdb_ci_computer record, you should see that an asset record has now been created for this CI.

With Discovery:

  1. Prepare to scan a CI via Discovery for which you know the model and what class it will create.
    For example, if you scan an Apple computer, this will create a "cmdb_ci_computer" record with a model like "Apple MacBookPro10,1".
  2. Repeat steps 1-4 above but make sure the cmdb_model_category record where you enable the "Enforce CI verification" matches what will be created by the CI that you scan.
  3. Scan your CI.
  4. After scanning, check the following:
    • A new cmdb_hardware_product_model record should be created.
    • For the dummy CI you created, check to see if an asset record is now created for this CI.

  3. Depending on the table you link these mainflow and subflow records to, create two records that use this workflow/subflow.
    1. For the first record, go through the whole workflow process to call and run through the subflow and the mainflow. Both should be "Finished".
    2. For the second record, have the workflow call the subflow, but keep the subflow in a "executing" state.

    This is supposed to run an XML "fix_subflow_parent_activity"; however if you look at logs, you will likely see that this script throws an error and does not get run successfully.

    You should then expect that the mainflow will finish it's activity and proceed, but it appears that the mainflow will be stalled.
    In the subflow workflow logs, you will see this type of error:
    Subflow to parent flow signaling failed.

6 Before Upgrading to Eureka

Review this information before upgrading an instance to Eureka:

  • Customers who use Amazon EC2 or VMware for cloud provisioning must perform additional actions before upgrading to Eureka.
  • Customers upgrading to versions prior to Eureka Patch 2 may encounter issues with email notifications if the Subscription Based Notifications plugin is not active. For details and workaround information, see KB0539070 in the ServiceNow Knowledge Base.
  • For customers upgrading to versions prior to Eureka Patch 2, the UI14 user interface is not enabled automatically. For workaround information, see KB0539843 in the ServiceNow Knowledge Base.
  • For customers upgrading to versions prior to Eureka Patch 3 Hotfix 1, the TinyMCE HTML field editor is not enabled by default. This feature is required to use HTML fields in UI14. For details and workaround information, see KB0540618 in the ServiceNow Knowledge Base.
  • Customers upgrading to versions prior to Eureka Patch 3 Hotfix 1 may encounter issues with visual task boards if the Mobile UI plugin is not activated. For details and workaround information, see KB0541346 in the ServiceNow Knowledge Base.
