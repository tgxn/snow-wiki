Change Manager Script Include

Configuration Automation

1 Overview

The CfgAutoChangeManager script include provides methods for defining a configuration automation change manager. Implement the methods described here to customize a configuration automation provider or define a new provider. For information on how to use this script include, see Defining the Change Manager Script.

Note: A configuration template was called a node definition in earlier versions of the system.

1.1 Where To Use

Use this script include to define a custom configuration automation change manager.


2 Method Summary

To use this script include, implement the following methods.

Return Type Method Summary Description
boolean isAuthorized(GlideRecord record) Tests if the current user can create or update changes that are associated with the passed record.
boolean isChangeRequired() Tests if change management is required by checking the property for the configuration automation provider.
String getChangeId(GlideRecord current) Returns the change request ID associated with the passed current record. If change management is required, a change record must be associated with the resource to be changed.
void setChangeId(GlideRecord record, String changeId) Associates the passed change request ID with the passed record.
GlideRecord prepare(GlideRecord record) Prepares the upcoming changes. This method is called before the change execution.
void finalizeChange(GlideRecord record) Finishes the change process. This method is called before the change request is closed. Anything that needs to be done before the change request closes goes here.
void getWorkflowVariables(String workflowName, GlideRecord record) Returns the variables that the workflow consumes. Even if you do not need to use a workflow, implement this function and return an "undefined" value.
Object getPushWorkflowName() If a workflow is required by the provider, returns the master workflow name. Even if you do not need to use a workflow, implement this function and return an "undefined" value.
void cleanupAfterCancelChange(GlideRecord record) Cleans up any resource or data when a change is canceled. This method is called after the change request is canceled.

3 Method Detail

3.1 isAuthorized(GlideRecord record)

Determines if the user is authorized to make changes.

3.1.1 Input Fields

Parameters:

  • record - (GlideRecord) The current record.

3.1.2 Output Fields

Returns: (boolean) true if the user is authorized to make changes.


3.1.3 Example

isAuthorized : function(current) {
    return gs.hasRole('chef_admin');
}

3.2 isChangeRequired()

Determines if change management is required.

3.2.1 Input Fields

Parameters:

  • None

3.2.2 Output Fields

Returns: (boolean) true if change management is required; otherwise, false.


3.2.3 Example

isChangeRequired: function()  {
    return gs.getProperty('glide.config_auto.chef.node_definition.use_change') == 'true';
}

3.3 getChangeId(GlideRecord current)

Returns the change ID associated with the passed current record. If change management is required, a change record must be associated with the resource to be changed.

3.3.1 Input Fields

Parameters:

  • current - (GlideRecord) The current record.

3.3.2 Output Fields

Returns: (String) the Change ID.

3.3.3 Example

getChangeId : function(current) {
    	
    if(JSUtil.nil(current))
        return null;
    	
    if(!current.instanceOf('node_definition')) {
        return null;
    }
		
    return current['change'];
}


3.4 setChangeId(GlideRecord record, String changeId)

Associates the passed change request ID with the passed record.

3.4.1 Input Fields

Parameters:

  • record (GlideRecord) The current record.
  • changeId (String) The Change Request ID.

3.4.2 Output Fields

Returns: void

3.4.3 Example

setChangeId : function(current, changeId) {
    	
    if(JSUtil.nil(current))
        return ;
    	
    current['change'] = changeId;
    current.update();
}


3.5 prepare(GlideRecord record)

Prepares the changes. This method is called before the execution of the changes.

3.5.1 Input Fields

Parameters:

  • record - (GlideRecord) The current configuration template draft record.

3.5.2 Output Fields

Returns: (GlideRecord) the current configuration template draft record.

3.5.3 Example

prepare : function(current) {

    var change = current.cfg_auto_change;
    	
    if(JSUtil.notNil(change)) {		
        var phaseList = new ChoiceList('cfg_auto_change','phase');
        var gr = new GlideRecord('cfg_auto_change');
        gr.get(change.sys_id);
        gr.setValue('phase',phaseList.getValue(this.IN_PROGRESS));
        gr.update();
    }
    return current;
}


3.6 finalizeChange(GlideRecord record)

Finishes the change process. This method is called before the change request is closed. Anything that needs to be done before the change request closes goes here.

3.6.1 Input Fields

Parameters:

  • record - (GlideRecord) The current configuration template draft record.

3.6.2 Output Fields

Returns: void

3.6.3 Example

finalizeChange : function(current) {
    	
    var changeId = current.cfg_auto_change.sys_id;
		
    //Update current node definition from requested
    var changeGr = new GlideRecord('cfg_auto_change');
    changeGr.get(changeId);
        
    current.fqdn = changeGr.fqdn;
    current.cfg_auto_provider = changeGr.provider;
    current.cfg_auto_management_server = changeGr.management_server;
    current.node_definition = changeGr.node_definition;
    current.update();
		
    ...
}


3.7 getWorkflowVariables(String workflowName, GlideRecord record)

If a workflow is required by the provider, returns the variables that the workflow consumes. Even if you do not need to use a workflow, implement this function and return an "undefined" value.

3.7.1 Input Fields

Parameters:

  • workflowName - (String) The workflow name.
  • record - (GlideRecord) The current record.

3.7.2 Output Fields

Returns: (Object) the workflow variables.

3.7.3 Example

getWorkflowVariables : function(wfName, nodeDefGr) {
		
    var wfVars = {'u_node_def_id': nodeDefGr.getValue('sys_id')};
    return wfName === 'Deploy Chef Change Master' ? wfVars : {};

}


3.8 getPushWorkflowName()

If a workflow is required by the provider, returns the master workflow name. Even if you do not need to use a workflow, implement this function and return an "undefined" value.

3.8.1 Input Fields

Parameters:

  • None

3.8.2 Output Fields

Returns: (String) The workflow name.

3.8.3 Example

getPushWorkflowName : function() {
    return 'Deploy Chef Change Master';
}


3.9 cleanupAfterCancelChange(GlideRecord record)

Cleans up any resource or data if a change is canceled. This method is called after the change request is canceled.

3.9.1 Input Fields

Parameters:

  • record - (GlideRecord) The configuration template draft record.

3.9.2 Output Fields

Returns: void

3.9.3 Example

cleanupAfterCancelChange : function(nodeDefDraftGr) {

    new Workflow().cancel(nodeDefDraftGr); // cancel the flows against our extended table
    nodeDefDraftGr.change = '';
    nodeDefDraftGr.update();
}
