The following issues were fixed after the Winter 2009 release in the Winter 2009 release patch, available since January 16th, 2009, on an as-needed basis:

  • PRB42693 -- SLA percentage timers do not work properly in Winter release
  • PRB42689 -- Email text sometimes is not stripped of HTML tags
  • PRB42679 -- Reference pop-up view broken for multiple-section forms
  • PRB42676 -- Legacy glide.ui.tabsall property breaks forms after Winter release (patch removes the property)
  • PRB42668 -- text_index events may not get processed if instance has a custom date format
  • PRB42665 -- Grouping on a derived (dot-walked) field may fail if the derived name exceeds 30 characters
  • PRB42664 -- Reporting issues introduced with the Winter release
  • PRB42662 -- Form resolution logic errors (wrong view being displayed under some circumstances)
  • PRB42660 -- Inbound emails being ignored if they were forwarded through the outbound email address
  • PRB42655 -- After Winter release, import of a file looks like it is still running when it is not
  • PRB42654 -- Behavior change with multiple form sections
  • PRB42652 -- Mobile browser properties added to out-of-box properties
  • PRB42591 -- Special characters not properly displayed in Welcome Page Content HTML text area

The following issues were fixed in the Winter 2009 release:

  • Fixes for the Summer 2008 Stable 1 build:
    • PRB41944 -- Reference lookup forcing a second selection of the same result
    • PRB41940 -- Data Source QUERY type can't be changed once created
    • PRB41938 -- Mapping assistant deletes existing mapping
    • PRB41936 -- Change Import set table to not go through SYS_AUDIT_DELETE
    • PRB41931 -- "Group by" uses database values instead of display values e.g., group by active shows labels of 0 and 1
    • PRB41929 -- HTML fields not honoring 'mandatory' flag with WYSIWYG editor
    • PRB41923 -- Filters may not create correct results if running on Oracle
    • PRB41922 -- Variables created via record producers were not available in script if within a container
    • PRB41920 -- Order Guide options not kept without clicking a tab first
    • PRB41918 -- Installation Exit objects are cached and not refreshed when modified
    • PRB41914 -- If you click fast enough on "order now" in the catalog cart, you can end up ordering > 1 thing
    • PRB41909 -- Label change sometimes doesn't take immediate effect
    • PRB41899 -- Having a reference qualifier on a variable causes the lookup list navigation to not work
    • PRB41898 -- On IE cart always renders empty on the main catalog page. Cart renders correctly if you drill into an item.
    • PRB41884 -- onSubmit catalog client scripts wouldn’t function properly if they contained escapable characters e.g. & <>
    • PRB41880 -- Using script to set an integer field to a non integer value e.g. 4.5 results in SQL errors on MSSQL.
    • PRB41868 -- Minimize icon in top right of screen doesn’t function properly on Chrome or safari
    • PRB41865 -- Graphical workflow editor requires you exit and reenter before functioning on Chrome.
    • PRB41863 -- Under some circumstances, importing “display” values into reference columns could result in improper values being set in target records during an update.
    • PRB41862 -- BSM map “hangs” on Chrome.
    • PRB41861 -- Default price on items (in cart) wrong if you revisited an item within an order guide.
    • PRB41859 -- HTML type fields not property indexed for IR purposes
    • PRB41858 -- Variables inside of variable sets not available inside of script.
    • PRB41855 -- Cancel button on impersonate button doesn’t work correctly under Chrome
    • PRB41854 -- Misaligned column headers under Chrome
    • PRB41852 -- Incorrect columns displayed on list type reports under some circumstances if using domain separation.
    • PRB41848 -- If using Firefox, a multiple update form with an HTML widget on it would erroneously update the HTML field to empty, even if the HTML field on the update was left unchanged. Corrected behavior is that an empty HTML field (like an empty anything else) on an update form means “do not change this column”.
    • PRB41838 -- Update sets are only bringing over the first 250 entries during the pull
    • PRB41834 -- Wizards not launching correctly from subcategory view
    • PRB41826 -- If using “enter key submits form” option, and your cursor is on a button when you hit enter, system was erroneously submitting first button on form, rather than the button you were over.
    • PRB41821 -- Request Item workflow widget displays more steps than there are execution steps.
    • PRB41820 -- If you had dirty form support enabled, then hitting the green back button from within a dirty form and then canceling navigation would leave you in a state where you could not save the form at all.
    • PRB41811 -- Display errors on IE when displaying a dashboard in “printer friendly” mode.
    • PRB41783 -- Inbound email would stop being collected if you had specified a non-default email retention policy in the pop3 reader job.
    • PRB41779 -- Has_current variable not being set properly for advanced reference qualifier scripts if you used the magnifying glass to bring up a reference list (as opposed to using the type-ahead capability)
    • PRB41735 -- if a spreadsheet contained an invalid formula or link import would abort. New code simply ignores that cell.
    • PRB41708 -- Oracle DATE type being incorrectly mapped to a ServiceNow DATE when, in fact, it should have mapped to a DateTime since Oracle dates contain time information as well.


  • Fixes for the Summer 2008 Stable 2 build:
    • PRB41947 -- Expand/collapse widget doesn't work properly if the section contains the CI relationships editor and you are using IE
    • PRB42029 -- Searching users for roles=itil and roles!=itil returned the same results
    • PRB42023 -- If a filter contained the @ symbol, the condition widget truncated that part of the condition when you dropped down the filter UI
    • PRB40268 -- Issuing a GOTO query against a CLOB field (by using the goto box atop a list) created invalid SQL on Oracle and MSSQL server. Query was changed from a >= query to a startswith operation for consistency's sake
    • PRB42129 -- Roles were not being honored properly if KB articles were accessed via sys_id=url pattern.
    • PRB42128 -- If running in a localization like french with non default grouping or decimal separators, the list editor would save the wrong numbers under some circumstances. Form based editing worked correctly regardless.
    • PRB42141 -- A reference type column created via the UI (personalize form -> add new reference) wasn't getting a matching index on the database side.
    • PRB42063 -- The global localization property was not being ignored, forcing customers who wanted to localize to set a locale on every user record.
    • PRB42076 -- Delivery address field not being rendered properly on two step checkout form if the customer was using IE. Secondary issue was rendering syntactically correct but not useful information if user had, say, a city but not a state filled in on their user record.
    • PRB42115 -- Using UI policy to hide/reveal an HTML type field would result in javascript errors.
    • PRB41914 -- If you clicked repeatedly (and rapidly) on the order now button on a catalog item, it was possible to order more than one at the same time when you really just wanted to order one and were very, very fast at clicking the mouse.
    • PRB42162 -- If your name had a quote in it (like Bob O'Neil) then hitting the "add myself" button on the watch list resulted in a javascript error.
    • PRB42187 -- Removing the last relationship of a given category from a CI in stable 1 would result in an orphan relationship with no child, rather than removing the relationship entirely.
    • PRB42130 -- Bar chart type reports render improperly if grouped and filtered on the same reference field.
    • PRB42154 -- A UI action on a child table (like incident) which shared a name with a UI action on a parent table (like task) should have overridden it. It was not doing so.
    • PRB42153 -- gs.hasRoleInGroup() didn’t return correct results if group named was mixed case.
    • PRB42026 -- NPE when rendering a request item if the item in question contained variables with variable categories on them.
    • PRB42138 -- Aggregations functions on dates, date/times, and durations didn't produce proper results.
    • PRB42136 -- A catalog item which was deleted did not properly remove itself from users carts, potentially allowing the user to order an invalid item if they checked out with a cart generated prior to the deletion.
    • PRB42135 -- If a list contains non western characters (Chinese, Cyrillic, Arabic, etc), Export to Excel would put garbled text in those cells.
    • PRB42114 -- Cannot hide or reveal a user image type field via client script
    • PRB42048 -- Import sets failed to coalesce properly if coalesce field was a reference field and a non default display column was chosen
    • PRB40244 -- Import set row statistics wrong if a row insert/update is canceled via script
    • PRB41567 -- Javascript error when using the new button on a glide list widget on IE
    • PRB42070 -- Pressing the new button on a list with a very large/complex filter condition could lead to a page not found error on IE
    • PRB42078 -- UI Policy on "dot walked" fields e.g. incident.caller_id.name not firing correctly
    • PRB42084 -- Potential XSS vulnerability when viewing the service catalog cart.
    • PRB42088 -- Chinese characters in certain messages cause evaluation errors during list generation
    • PRB42040 -- Numeric scale type variables showed every entry as having a label of "0"


  • Fixes for the Summer 2008 Stable 3 build:
    • PRB41921 -- Clicking the New button on a reference list (to create a new item) created the item successfully, but did not put the new item into your reference field if a wizard intercepted the creation request.
    • PRB41949 -- If you have > 1 grouped list report on a home page, they should expand and collapse independently. They were not.
    • PRB42133 -- Out of Box email notifications do not render well in the Lotus Notes email client.
    • PRB42176 -- Certain Out of Box queries against the ECC queue could take unreasonably long periods of time on an instance with very large numbers of ECC queue records.
    • PRB42180 -- HTML type fields could not be set to read-only via client script or UI policy
    • PRB42194 -- Add new section option while editing a form used the JavaScript "confirm" dialog which was triggering popup blockers. Changed it to use a glide dialog instead.
    • PRB42211 -- Cleaning up a large update set could take unreasonably long periods of time and impact the performance of the system as a whole. Newer algorithm is much more efficient.
    • PRB42214 -- (Stable 1 only) Users who could view a group report could always change it, even if their roles indicated they should be able to view it only.
    • PRB42216 -- If an application module had a role on it, that role wasn't being honored if the user had specified a "preferred" view for that type of record. Preference will only kick in if the view is blank now.
    • PRB42226 -- Queries which specified an invalid operator such as gr.addQuery('firstname', 'badger', 'pat') would produce SQL errors. New code will not generate a SQL error, but neither will that particular term return any records.
    • PRB42247 -- Lookup select type questions did not display the reference qualifier field due to OOB UI policy.
    • PRB42251 -- If you updated a duration field and hit the "Enter" key to store the results (as opposed to tab or clicking away), the duration was not saved.
    • PRB42255 -- All UI policy was running on form load, even if you unchecked the "on load" checkbox.
    • PRB42258 -- In an instance with domain processing enabled, breadcrumbs on top of lists or related lists contained domain information and hence excessively long URLs. (Stable 2 only)
    • PRB42259 -- OOB security rights allowed an ITIL user to approve things for which he was not the approver. Out of box rights have been changed to prevent this.
    • PRB42261 -- If you changed the class of a CI, its relationships were being lost.
    • PRB42277 -- Fix JavaScript errors if you try to use a glide_list widget to reference a "tree" type list (like Locations).
    • PRB42278 -- If a report is public, so should its chart be.
    • PRB42316 -- A query against the "roles" field on the group table would return no records if contextual security was installed.
    • PRB42330 -- Special characters (umlauts, accents, etc) in variable answers were not correctly saved if you used the "order now" button to order a catalog item.
    • PRB42333 -- If a field is not viewable to a user because of security, its label was nonetheless showing up on PDF output (albeit without any data).
    • PRB42334 -- UI actions of type "client" did not emit their script properly if they were list options (the ones that show up in the select at the bottom of the list).
    • PRB42341 -- ESS view of request summary screen broken in Stable 2.
    • PRB42344 -- It was possible to create a recursive loop of catalog category parents which would cause a stack overflow while trying to view that category.
    • PRB42348 -- An "unsendable" email with an ambiguous error code could potentially clog the outbound email queue for up to 24 hours, preventing other mail from going out. Fix was two part: 1) handle more error states properly; 2) in the event of an ambiguous error state, don't let any one record clog the queue.
    • PRB42372 -- Low level database errors coming out of progress workers (Import Set loads, for example) could potentially consume and not release a database connection, leading to instance performance problems and an eventual lockup.
    • PRB42381 -- Incorrect results if running a trend type query against an Oracle database


  • Additional fixes for the Winter 2009 release
    • PRB42392 -- Personalize dictionary from a workflow variable corrupts sys_dictionary entry for that variable
    • PRB42393 -- History tag doesn't correctly display the first audit entry for a record
    • PRB42397 -- If using company separation, it was possible to impersonate a user and then not be able to impersonate yourself again
    • PRB42398 -- Visible on summary checkbox not being honored on catalog items
    • PRB42408 -- I18N: Browse button on attachment form not translating properly
    • PRB42430 -- View rules not being properly honored in Summer 2008 Stable 2 for extended tables e.g. change_request
    • PRB42436 -- Add pricing to labels checkbox not being honored within the catalog. Pricing was always being added to labels as soon as you changed a price sensitive field
    • PRB42437 -- Translated fields not being indexed properly for full text search
    • PRB42450 -- Gauges on home pages not honoring size properly if size was changed after gauge put on home page
    • PRB42476 -- Published list reports always used the default view; should have used the view from the report specification
    • PRB42489 -- Single ticks (the ' character) not being output correctly on the welcome page
    • PRB42501 -- Language picker does not always reflect current session language
    • PRB42516 -- Extremely large table cleanup operations can generated excessive redo logs and degrade system performance significantly
    • PRB42515 -- Using the back button from the choose options panel on an order guide puts a phantom item in your chart
    • PRB42550 -- List mechanic was available on reference popup forms. It should not have been
    • PRB42561 -- Mandatory variables inside of a container were not showing up as mandatory once an item was ordered
    • PRB42574 -- Changing a list aggregate (sum, average, max, min) didn't cause an entry to be written into an update set
    • PRB42614 -- Full text indexing was not always indexing all fields on a record
