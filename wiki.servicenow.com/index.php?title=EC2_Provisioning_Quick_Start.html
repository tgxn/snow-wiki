1 Overview

ServiceNow offers the ability to configure and provision VMware vCenter or Amazon EC2 virtual machines (VM) from the ServiceNow interface starting with the Calgary Release. Use this guide to manually provision an EC2 virtual machine. To provision an VMware vCenter instance, see the VMware Quick Start guide.

To provision an EC2 virtual machine through ServiceNow:

  • Satisfy all prerequisites
  • Configure ServiceNow to provision virtual machines
  • Provision a virtual machine


This guide provides an condensed set of instructions to help you provision virtual machines. To help you use cloud provisioning, ServiceNow offers these additional resources:

  • Cloud provisioning documentation
  • Amazon EC2 provisioning video tutorial
  • Automatic provisioning video tutorial

2 Prerequisites

2.1 Access an Amazon EC2 Account

To provision EC2 virtual machines, you must have access to an Amazon EC2 account. Contact your EC2 account administrator for more information.

2.2 Obtain Essential EC2 Files

Before configuring ServiceNow to provision EC2 virtual machines, ensure you have the following items. Contact your EC2 account administrator to obtain the certificate and private key.

  • EC2 Certificate file: Amazon provides one or more X.509 certificates for all EC2 accounts.
  • EC2 Private key file: Amazon provides a private key as a .pem file when you first create an EC2 account.

2.3 Generate a PKCS12 Key File

Use the certificate file and the private key file to generate a PKCS12 key file.

View the procedure for generating PKCS12 key file

Generate the PKCS12 key store file using a tool such as OpenSSL. The following command creates the PKCS12 key store file using OpenSSL on Linux. The key store alias can be any string. Enter this command on one line. Save the key store alias for later use. 

openssl pkcs12 -export -name <KEY STORE ALIAS> -out <KEY STORE FILE NAME> -in <AMAZON CERTIFICATE NAME> -inkey <AMAZON PRIVATE KEY>

An example of a command line might be: 

openssl pkcs12 -export -name snc_aws -out snc_aws.pkcs12 -in aws_auto_x509_cert.pem -inkey aws_auto_private_key_.pem


2.4 Activate the Plugin

EC2 virtual provisioning requires the Orchestration EC2 Support plugin.

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.

2.5 Create User Records

As an administrator, create these users and assign them to the designated groups:

Username Group
Cloud User Virtual Provisioning Cloud Users
Cloud Admin Virtual Provisioning Cloud Administrators
Cloud Operator EC2 Operators
EC2 Approver EC2 Approvers
View the procedure for creating users
  1. Navigate to Users and Groups > Users.
  2. Click New.
  3. Enter a Username for the user. Refer to the table for all required users.
  4. Click Submit.
  5. Open the new user record.
  6. In the Groups related list, click Edit.
  7. Select the group for that user. Refer to the table for group assignments by user.
  8. Click Save.

3 Set up EC2 Provisioning

A cloud administrator can set up ServiceNow to provision Amazon EC2 virtual machines.

3.1 Create a PEM Certificate Record

Navigate to Amazon EC2 > Configuration > Certificates and create a new PEM certificate record using the certificate provided by Amazon.

View the procedure for creating a PEM certificate record
  1. Click New.
  2. Enter a unique Name for the certificate.
  3. In the Format choice list, select PEM.
  4. Enter the PEM Certificate provided by Amazon. See Enter the Amazon Certificate for more information.
    Make sure the certificate begins and ends with:
    -----BEGIN CERTIFICATE-----
    -----END CERTIFICATE-----
  5. Click Submit.
Create a PEM certificate record


3.2 Create PKCS12 Key Store Certificate Record

Navigate to Amazon EC2 > Configuration > Certificates and create a new certificate record, then attach the PKCS12 key store file to it. For more details, see Uploading a Certificate.

View the procedure for attaching the PKCS12 key store file
  1. Click New.
  2. Enter a unique Name and set the Type to PKCS12 Key Store.
  3. Enter the Key store password.
  4. Click the attachment icon () and attach the PKCS12 file.
  5. Click Submit.
Attach the PKCS12 key file


3.3 Create an Amazon EC2 Account Record

Navigate to Amazon EC2 > Configuration > Account and create an Amazon EC2 account record using the newly created certificate records.

View the procedure for creating an Amazon EC2 account record
  1. Click New.
  2. In the EC2 Account form:
    • Enter a unique Name for the account.
    • Enter the Amazon EC2 Account number.
    • Select the Amazon EC2 PEM Certificate and PKCS12 Key store records created in the previous steps.
    • Enter the Key store alias used to generate the PKCS12 key store file.
  3. Click Submit.
Create an EC2 account record


3.4 Update the EC2 Account Key Pairs and Images

After creating an EC2 account record and associating the certificate and PKCS12 key store file to this account, update the key pairs for the account.

View the procedure for updating a key pair for an EC2 account record
  1. Open the account you just created.
  2. Under Related Links, click Update Key Pairs.
    After a brief pause, ServiceNow populates the EC2 Key Pairs related list. Key pairs consist of a private key and a fingerprint. To launch an EC2 instance in a region and access it via SSH, you use the private key from the key pair for that region.
  3. In the EC2 Region Settings related list, click New.
  4. In the Region Settings form, create a Name for the region, assign that region a Key pair, and select the Amazon Region associated with the key pair.
    Use the private key portion of this key pair to log into any virtual machine created with this key pair.
  5. Click Submit.
  6. Click the Update Images related link to populate the EC2 Images related list with images provided by Amazon.
Update the key pairs


3.5 Create EC2 Service Catalog Items

After successfully updating your Amazon EC2 account record and populating the list of available images, create a catalog item from an approved image. A cloud administrator can approve images and create service catalog items from approved images.

3.5.1 Approve EC2 Images

Navigate to Amazon EC2 > Available Images and approve the image.

View the procedure for approving an EC2 image
  1. Select the image record you want to approve.
  2. Click Approve.
EC2 image record

3.5.2 Create Catalog Items from Approved Images

Navigate to Amazon EC2 > Approved Images and create a new catalog item from an approved image.

View the procedure for creating a catalog item from an image
  1. Select an approved image record.
  2. Click the Create Catalog Item related link.
  3. In the Task automation choice field, select Manual.
  4. [Optional] Modify the EC2 Catalog Item record.
  5. Click Update.
Create an EC2 catalog item

4 Provision a Virtual Machine

After configuring the EC2 offerings, a cloud user can request a virtual machine from the service catalog. Since manual approvals are required, an EC2 approver can approve the request. After the catalog request is approved, ServiceNow creates a new catalog task. A cloud operator can complete the catalog task to provision the virtual machine.

4.1 Request a Virtual Machine

As a cloud user, navigate to Service Catalog > Catalog and place an order for the virtual machine.

View the procedure for requesting a virtual machine from the service catalog
  1. Select the newly created EC2 catalog item.
  2. Enter a Business purpose for this virtual machine.
  3. Click Order Now.
Request a virtual machine‎

4.2 Approve the Request

As an EC2 approver, navigate to Service Desk > My Approvals and approve the virtual machine request.

View the procedure for approving a virtual machine provisioning request
  1. Select the approval record for the virtual machine request item.
  2. Click Approve.
Approve the request‎‎

4.3 Complete the Provision Task

As a cloud operator, navigate to Service Desk > My Groups Work and complete the catalog task to provision the virtual machine.

View the procedure for provisioning a virtual machine from a catalog task
  1. Open the catalog task record for the EC2 request.
  2. Select the Amazon EC2 Account for provisioning this virtual machine.
  3. In Region settings, select an Amazon EC2 datacenter for the instance.
  4. If multiple Image choices are available, select one.
  5. Click Close Task.
Complete the catalog task‎
