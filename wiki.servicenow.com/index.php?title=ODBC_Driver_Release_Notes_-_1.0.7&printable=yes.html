1 Overview

This release of the ODBC driver provides the following fixes and is recommended for all versions of the ServiceNow platform. The previous version of the ODBC driver was 1.0.5.

2 Fixed in this Release

The following issues were fixed in this version of the ODBC driver since the last release.


Number Details
PRB593368 When property glide.web_service.hierarchical is set to true, it causes the sys_user WSDL to increase in size dramatically, and causes the ODBC driver to run out of memory.
PRB582470

PRB584916

iSQL(ODBC) when doing an evaluation between 2 points in time, the 2nd value is in a different timezone than the first.
Click the plus for steps to reproduce
  1. Connect with iSQL to an instance with a different system time zone than the ODBC client.
  2. Pull sys_created_on from any record.
  3. Run a query with 2 values in the evaluation against the time using >=TIME1 AND <TIME2.
    Notice that TIME1 and TIME2 are in different time zones. TIME1 seems to be the ODBC client time, and TIME2 seems to be the instance system time zone.
PRB585402 When retrieving the length of table columns using ODBC, the length is off by 6 characters.
Click the plus for steps to reproduce

Retrieve the length of table columns using this criteria: SELECT column_name, oa_length FROM system.oa_columns WHERE table_name = 'sys_audit';

The following results show that an additional 6 characters are added to the column lengths.
documentkey 38
fieldname 46
internal_checkpoint 46
newvalue 4006
oldvalue 4006
reason 1006
record_checkpoint 46
sys_created_by 46
sys_created_on 6
sys_id 38
tablename 46
user 46
PRB590432

PRB597621

The ODBC driver maps the glide_date dictionary type to a VARCHAR instead of DATE.
Click the plus for steps to reproduce

ODBC version: ServiceNow-ODBC-64-bit_2012-11-09-1513 Instance version: Build tag: glide-berlin-05-06-2012__patch7-02-14-2013

  1. Connect your ODBC driver to your instance.
  2. Run the following command in iSQL:
    SELECT * FROM OA_COLUMNS WHERE table_name = 'alm_asset'
    The expected results are that the field is mapped to a DATE.
    The actual results are that the field is mapped to VARCHAR.
    SCHEMA OAUSER alm_asset purchase_date -9 VARCHAR 46 46 NULL 6 1 NULL NULL NULL 1 0 NULL
PRB584992 When a character such as the en dash (Unicode U+2013 / Windows-1252 0x96 ) is found in a field, an ISQL query returns a value encoded as Unicode.
PRB599400

PRB593156

Using ISQL after connecting using ODBC causes the following error to appear:

SLF4J: Failed to load class "org.slf4j.impl.StaticLoggerBinder". SLF4J: Defaulting to no-operation (NOP) logger implementation. SLF4J: See http://www.slf4j.org/codes.html#StaticLoggerBinder for further details.

Null Pointer Exception Attempting to retrieve the ODBC driver version number causes a NullPointerException.
Null Pointer Exception Attempting to run a query that does not contain a column name while using ODBC causes a NullPointerException.
