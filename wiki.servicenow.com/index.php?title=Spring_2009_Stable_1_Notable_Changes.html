1 Associating a task with a survey

A task reference field has been added to the survey_instance table. When you build the survey URL you can add &sysparm_survey_task=task_sys_id to populate this field. You will then be able to report on survey responses based on task assignment, categories, or any other task related information. Survey functionality is now more field customizable using the SurveyProcessor Script Include and UI Pages for survey_take.do and survey_thanks.do.

2 Font Standardization

System fonts have been standardized throughout the instance to facilitate personal customization. Fonts in some areas that previously had hardcoded font sizes or font families may appear different due to this standardization.

3 Ongoing feedback on large catalog requests

When a catalog request is copied for multiple additional users, a progress indicator will provide updates as the catalog request is copied. At completion, the progress indicator will present options to return to the catalog request (and copy it again if desired), return to the catalog home page, or view the newly created requests.

4 Manager not auto-mapped in LDAP

When importing users from LDAP, the manager field is no longer automatically mapped when using the Mapping Assist capability. Manager information is handled in a separate part of the LDAP import, so the automatic mapping was unnecessary.

5 Plugins for newly created instances

View Definition and Metrics plugins are now installed automatically for newly created instances.

6 Refresh homepages

All users can use the "Refresh" functionality on homepages.

7 PDF output contains aggregations

PDF exports now include "Personalize Calculations" aggregations.

8 Cascade delete for SLAs

SLAs are now cascade deleted when their associated tasks are deleted.

9 Application and Module UI Actions restricted

Activate and Deactivate UI Actions for sys_app_application and sys_app_module tables are restricted to the admin role.

10 Mandatory derived field behavior

Previous behavior of mandatory derived fields was restored. They cannot be cleared out, but if they start out empty, they can remain empty.

11 Unavailable attachment message

If a user is not logged in, but tries to view an attachment, they now get an informative message.

12 Image picker UI cleanup

The image picker UI (e.g., when selecting an icon for a module) was improved.

13 AJAX scripts converted to Server Script Includes

Many AJAX Scripts (in System Definition) were converted to Server Script Includes.

14 UI Pages can be secured with ACLs

A new ACL type of "ui_page" was added. An ACL can be created with a Type of "ui_page", an Operation of "read", and a Name value equal to the name of a UI Page, to restrict access to a specific UI Page. The "governor" UI Page has been restricted to admin users.

15 Length of chart URLs reduced

Chart URL length has been dramatically reduced to accommodate IE's "get" string length restriction.

16 New ACLs in Security Jump Start plugin

New ACLs have been added to the Security Jump Start plugin to restrict system tables out-of-box.

17 Script input for admins

Admins can now use the script input in the sys.scripts.do page, and access uploaded Customer Scripts from that page.

18 Manager can see groups' Search Groups

If a user is a group manager (but not a member of the group), that group's Search Groups are now available.

19 Knowledge searches tracked

Knowledge Base searches are now tracked in the ts_query_kb table.

20 Custom charts can be emailed

Custom charts can now be emailed as scheduled reports.

21 All text search results tracked

All text search queries from all tables are tracked in the text_search table.

22 Brazilian-Portuguese i18n plugin

The Brazilian-Portuguese language plugin is not currently admin-installable.

23 Report button titles

Descriptive tooltips have been added to Report buttons.

24 Preview HTML Body UI Action

A new UI Action, "Preview HTML Body", has been added to the sys_email table.

25 Language picker added to login page

A language picker has been added to the login page so users can select a language before logging in.

26 View Article UI Action

A new UI Action, "View Article", has been added to the kb_knowledge table.

27 M2M table between Knowledge and Task

A new table, m2m_kb_task, has been added to relate articles to tasks in a many-to-many relationship.

28 Default filter on Event Log and Mailbox modules

A default filter has been added to Event Log and Mailbox modules to show only records created today.

29 System Localization Properties module

A new module, System Localization, has been added to the System Properties application.

30 CSS property for navigation frame width

A new property, css.nav.menu.width, can be created to specify the width of the navigation frame. Default value is 18% if the property doesn't exist in an instance.

31 Conditions and Roles fields added to Knowledge Add-ons

Knowledge Add-ons are used to add additional search targets and Knowledge Base links. They now have Conditions and Roles fields to flexibly restrict when and for whom they appear.

32 Import Sets "Run Transform" can run multiple maps in order

When running a transform on an import set, multiple transform maps can now be run in order. A slushbucket is shown where multiple maps can be specified and ordered.

33 Domain hierarchy loop detection

If a domain hierarchy loop is inadvertently set up (e.g., domain A is the parent of domain B which is the parent of domain A), the loop will be detected when building the domain display value.

34 Emails marked processed when processing skipped

When skipping email processing because the condition for the inbound email action doesn't match, the email is now marked as processed to prevent confusion.

35 max_length field mandatory on Dictionary form

The max_length field is now mandatory for sys_dictionary records.
