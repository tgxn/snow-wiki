2 Tables

Live feed adds or modifies the following tables.

Table Description
Link attachments [live_link] Stores links and attachments that are added to messages (read access for administrator and live_feed_admin).
Live Favorite [live_favorite] Stores the favorite conversations, user groups, and messages.
Live Feed Searches [live_feed_search] Stores live feed text searches. Access at Social IT > Feed Administration > Search Log (administrator, text_search_admin, and live_feed_admin).
Live Follows [live_follow] Maintains users' follow preferences for user feeds.
Live Group Profile [live_group_profile] Stores conversation properties, including name, description, and public status. Also stored are user group properties, such as name, description, and public status.
Live Group Member [live_group_member] Maintains the member lists for conversation and user group.
Live Mention [live_mention] Stores the profiles mentioned in a message. References sys_user table.
Live Message Tag [live_message_tag] Stores messages associated with user-created tags (read access for administrator and live_feed_admin). Appears as a related list on Live Tag records.
Live Messages [live_message] Stores messages. References sys_user table. Access at Social IT > Feed Administration > Messages (administrator and live_feed_admin).
Live Poll [live_poll] Stores a poll question.
Live Poll Cast [live_poll_cast] Stores profiles of users who voted for a particular option in a poll.
Live Poll Options [live_poll_option] Stores poll options.
Live Profiles [live_profile] Stores user live feed profiles. References sys_user table. Access at Social IT > Feed Administration > Profiles (administrator and live_feed_admin).
Live Table Notification [live_table_notification] Generates automatic table notifications, such as when a user reads live feed for the first time and when a high priority incident is opened. Access at Social IT > Feed Administration > Table Notifications (administrator, chat_admin, and live_feed_admin).
Live Tag [live_tag] Maintains user-created tags. Access at Social IT > Feed Administration > Tags (administrator and live_feed_admin).
Live Tag Follows [live_tag_follow] Maintains users' follow preferences for tags.
Messages Liked by [live_message_like] Maintains like ratings for posts. Access at Social IT > Feed Administration > Likes (administrator, live_feed_admin, and chat_admin).

3 Plugins

Live feed activates the following plugins.

Plugin Name Plugin ID Description
Live Feed com.glideapp.live_feed_v2 Activates Live Feed 2.0.0.

4 Properties

To access live feed properties, navigate to Social IT > Feed Administration > Properties.

These plugins install system properties used by live feed:

Name Description
glide.live_feed.max_popular_tags Indicates the maximum number of popular tags displayed.
  • Type: integer
  • Default value: 36
  • Location: System Properties [sys_properties] table
glide.live_feed.max_recent_tags Indicates the number of recent tags to show in the tag display.
  • Type: integer
  • Default value: 36
  • Location: System Properties [sys_properties] table
glide.live_feed.my_feed_enabled Enables user access to My Feed capabilities.
  • Type: true | false
  • Default value: false
  • Location: System Properties [sys_properties] table
glide.live_feed.company_feed_exclude_groups Prevents messages that belong to a group from appearing on the Company Feed (Berlin release)

To configure this property: navigate to the Social IT > Feed Administration > Properties page and modify the Do not show messages that belong to a group on the Company Feed option.

To configure this property (Berlin release): modify glide.live_feed.company_feed_exclude_groups on the System Properties [sys_properties] table and set it to true or false.

  • Type: true | false
  • Default value: true
  • Location: System Properties [sys_properties] table
glide.live_feed.message_poll_interval Indicates how frequently live feed is updated with new messages.

To configure this property: navigate to Social IT > Feed Administration > Properties. Enter the polling value in the Live message poll interval time in ms. field.

To configure this property (Berlin release): add the property glide.live_feed.message_poll_interval (it is not on the sys_properties table by default), and enter a value in milliseconds.

If you do not add the property, the default value poll interval time is 15000 (15 seconds). To minimize performance impact, the minimum value is 2000 (2 seconds).

  • Type: integer
  • Default value: 15000
  • Location: System Properties [sys_properties] table
glide.ui.show_live_feed_activity Displays two tabs on a record form's activity formatter: Live Feed, which allows users to collaborate on the document feed, and Activity, which shows a summary of all activity for the record. This property is not enabled by default.
  • Type: true | false
  • Default value: false
  • Location: System Properties [sys_properties] table
glide.live_feed.auto_join_document_group Enables automatic membership to a document conversation whenever either of the following conditions are met (starting with the Fuji release):
  • The associated document is visited and the user clicks Show Live Feed.
  • The glide.ui.show_live_feed_activity property is set to true.

This property is not enabled by default. If this property is not set to true, a user becomes a member of a document conversation only after the user posts a message to the conversation or explicitly joins the conversation.

  • Type: true | false
  • Default value: false
  • Location: System Properties [sys_properties] table

5 User Roles

Live feed adds the following user roles, starting with the Eureka release.

Role Description
live_feed_admin Can manage live feed functions.
chat_admin Can manage chat functions (if the Chat plugin is activated)

6 Security Rules (ACL) on Existing Tables

Live feed includes the following security rules on existing tables.

Table Description
sys_attachment Allows users to read and write live feed and live profile attachments.

7 Script Includes

Live feed adds the following script includes.

Name Description
LiveMsgUtil Provides helper functions for working with live_message records.
LiveFeedFilter Queries filters for working with live_feed records.
LiveFeedUtil Provides helper functions for working with live_feed records.

8 Script Actions

Live feed adds the following script actions.

Name Description
Live Feed Update Generates system messages when an update occurs on a table that is relevant to live feed.

9 Business Rules

Live feed adds the following business rules.

Name Table Description
Feed Group Creator Becomes Admin Live Group Profile [live_group_profile] Designates the group creator as the group administrator.
live feed events Task [task] Runs on task insert, update, and delete. Triggers event associated with the Live Feed Update Script action that processes Live Table Notifications to auto-generate live feed messages.
Live feed member update events Live Group Member [live_group_member] Generates a notification event when member state changes (invited, accepted, declined, left, rejected, request, request_accepted).
Live feed integration Journal Entry [sys_journal_field] Writes journal comments to the live feed if there is a group for this record.
Live feed new member events Live Group Member [live_group_member] Generates a notification event when new members are added.
live feed profile events Live Profile [live_profile] Runs on live_profile insert/update/delete, triggers event associated with the Live Feed Update script action that processes Live Table Notifications to auto-generate live feed messages.
Live Feed message events Live Feed Message [live_message] Runs on live_message, notification event trigger for new live messages.
Live message like events Message Liked by [live_message_like] Runs on live_message_like, notification event trigger for new like records.
LiveFeed Group Member Visibility 2.0 Live Group Member [live_group_member] Ensures users can only see the members list for public groups and groups they belong to.
LiveFeed Group Profile Validation Live Group Profile [live_group_profile] Ensures that a public group is visible.
LiveFeed Group Profile Visibility 2.0 Live Group Profile [live_group_profile] Ensures that the list of all groups only displays public groups, private groups that are visible, and groups the user belongs to.
LiveFeed Membership Changes Live Group Member [live_group_member] Enforces that only the group administrator and users with live_feed_admin role can manage membership for a group.
LiveFeed Single Group Membership Live Group Member [live_group_member] Ensures that a user is not added multiple times to the same group.
Live Message Likes Live Message Like [live_message_like] Updates the number of likes for a message.
LiveFeed Join Group Check Live Group Member [live_group_member] Ensures that users can not automatically join private visible groups.
Update Follow/Follower Counts Live Follow [live_follow] Updates the following/followers counts.
Live Feed Group Assessable Record [asmt_assessable_record] Creates/Deletes a live feed group for an assessable record
Live Feed Message Visibility Live Feed Message [live_message] Ensures user's access to live feed messages

live_feed.feed_member.accepted Triggers an email notification when a user accepts an invitation to join a feed.
live_feed.feed_member.request Triggers an email notification to administrators when a user requests to join a feed.
live_feed.feed_member.declined Triggers an email notification when an administrator has declined a user's request to join a feed.
live_feed.feed_member.removed Triggers an email notification when an administrator has removed a user from a feed.
live_feed.feed_member.invited Triggers an email notification when an administrator has invited a user to join a feed.
live_feed.feed_member.rejected Triggers an email notification to a user when the user's request to join a feed has been rejected by an administrator.
live_message.mentioned Triggers an email notification when a user is mentioned in a message.
live_message_like.inserted Triggers an email notification when another user likes a message that a user posts.
live_message.replied Triggers an email notification to all users in a thread when a user posts a reply.
live_message.inserted Triggers an email notification to subscribed users when a new message is posted.
live_message.group_inserted Triggers an email notification to subscribed users when a new message is posted to a feed.
live_feed.team_member.invited Triggers an email notification when a user is invited to join a team.
live_feed.team_member.accepted Triggers an email notification when an user accepted to join a team.
live_feed.team_member.declined Triggers an email notification when a user declines an invitation to join a team.
live_feed.team_member.request Triggers an email notification to administrators when a user requests to join a team.
live_feed.team_member.request_accepted Triggers an email notification when an administrator has accepted a user's request to join a team.
live_feed.team_member.rejected Triggers an email notification to a user when the user's request to join a team is rejected.
live_feed.team_member.left Triggers an email notification when a user leaves a team.
live_feed.team_member.removed Triggers an email notification when a user has been removed from a team.
live_message.group_replied Triggers an email notification to subscribed users when a reply to a message in a feed has been posted.
live_feed.update Triggers the Live Feed Update script action.

Live Feed Team Member Accepted Sends an email to administrators when a user accepts an invitation to join a team.

Live Feed Team Member Invited Sends an email when a user is invited to join a team.
Live Feed Team Member Left Sends an email when a user leaves a team.

13 Scripted Web Services

This following scripted web service is installed with live feed.

Scripted Web Service Description
PostToLiveFeed Performs the following functions:
  • Defines the user or profile to use; otherwise, the message posts from the authenticated soap user.
  • Defines the group (using the groupid); otherwise the message posts to the company feed.
  • Includes a link in the post.

13.1 Sample SOAP envelope

<SOAP-ENV:Envelope xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:SOAP-ENC="http://schemas.xmlsoap.org/soap/encoding/" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:m="http://www.service-now.com" xmlns:SOAP-ENV="http://schemas.xmlsoap.org/soap/envelope/" xmlns:tns="http://www.service-now.com/PostToLiveFeed"  SOAP-ENV:encodingStyle="http://schemas.xmlsoap.org/soap/encoding/">
  <SOAP-ENV:Body>
    <m:execute>
      <message xsi:type="xsd:string">Post this message to the Live Feed</message>
      <!-- optional sys_id of a sys_user record to use to get a live profile -->
      <user xsi:type="xsd:string"></user>
      <!-- optional sys_id of a live profile to post as -->
      <profile xsi:type="xsd:string"></profile>
      <!-- optional sys_id of a live group to post the message in-->
      <group xsi:type="xsd:string"></group>
      <!-- optional URL of a link to include in the message-->
      <linkurl xsi:type="xsd:string"></linkurl>
      <!-- optional name for the linkurl parameter-->
      <linkname xsi:type="xsd:string"></linkname>
    </m:execute>
  </SOAP-ENV:Body>
</SOAP-ENV:Envelope>

14 User Preferences

The following user preferences are available in live feed.

User Preference Description
live_tag.feed.order Tracks user's choice of tag display sorting.
live_message.feed.last_display Tracks last feed user has viewed. The last viewed feed is displayed the next time the user opens live feed. The first time a user opens live feed, the Company Feed is displayed.
