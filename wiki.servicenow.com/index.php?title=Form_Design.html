1 Overview

Administrators or users with the personalize_form role can use the form design feature to quickly create new or change existing form views. Form design is an alternative to configuring forms that combines several configuration options into one tool.

Form design is available by default for new instances starting with the Eureka release and requires UI15 or UI14. Administrators who upgrade from a previous version must activate the feature before it can be used.

Getting Started with Form Design

3 Parts of the Form Design Interface

The form design interface is divided into the following sections:

  • Page header
  • Field navigator
  • Form layout
Form Design Interface

3.1 Page Header

The page header displays the current table and view being designed. Each form is tied to a unique table and view combination. For example, there can only be one form for the incident table's default view. Select a different view to modify different form layouts for a table.

Form Design Page Header

3.2 Field Navigator

The field navigator allows you to manage fields on the form. It contains the following elements:

  • Filter: Allows text search on the currently selected tab.
  • Fields: Displays a list of existing fields you can add to the current form.
  • Field Types: Displays a list of field types you can add to the current form. Adding a field type to the form layout creates new fields in the selected table when the form layout is saved.
  • Field Properties: Displays relevant dictionary entry attributes for the currently selected field in the layout.
Form Design Field Navigator

3.3 Form Layout

The form layout displays a separate cell for each field, section, annotation, UI element, and related list that is currently on the form. Each cell has an identifying label and a handle icon to aid in selecting and moving it. The cell's location in the form layout represents its relative position on the form. Only fields visible on the form layout are displayed on a form.

Form Design Form Layout

4 Using Form Design

You can use form design to make the following changes:

  • Select what fields to display on a form.
  • Add a new field to a form and the associated table.
  • Add a new section to a form.
  • Add annotations to a form.
  • Add formatters to a form.
 Note: To add a field or annotation, the table must be in the same scope as the application you have selected in the application picker or must allow applications in another scope to add fields on it (starting with the Fuji release).


To change the content of a form:

  1. Navigate to the form you want to change.
  2. Right-click the form header and select Configure > Form Design (Personalize > Form Design in versions prior to Fuji) .
    The form design interface opens in a new browser tab or window.
  3. [Optional] Select the table and form view you want to change. By default, form design displays the form layout for the currently selected table and view.
  4. Make your form changes.
  5. Click Save.

4.1 Selecting Which Fields to Display

Move fields to and from the form workspace to display them on the form.

Note: Use the filter to quickly find fields in the field navigator.


To change the fields displayed on a form:

  1. From the list of fields in the Field Navigator, select the field you want to display from the list of available fields.
  2. Drag the field to the form layout.
  3. Drag the field to the location you want it to be on the form.
  4. To remove a field from the form layout, select the field and click the Remove this field (X) button beside the field label.
  5. Click Save.

4.2 Adding a New Field

Adding a field creates a dictionary entry for the new column in the current table. You may want to create a custom table and then use the form designer to add fields to the form.

Note: Use the filter to quickly find field types in the field navigator.


To add a new field to the form and its associated table:

  1. Click the Field Types tab.
  2. Select the cell for the field type you want to create.
  3. Drag the field type to the location you want it to be on the form.
  4. Enter a label for the field.
  5. [Optional] Click the Field Properties tab to change the default values for the field type.
  6. Click Save.

4.3 Adding a New Section

Move sections to and from the form layout to display or remove them on the form. Sections can have a one or two column layout.

Note: By default, the first section on a form is a read-only section displaying the table's label. You cannot remove this section.


To change form sections:

  1. Select an existing section.
  2. Click the Add a new section (+) button beside the section label to add a new section.
  3. Enter a label for the section.
  4. Drag and drop sections to reorder them.
    Note: The label for the first section on the form designer is displayed as the form title.
  5. To delete a section, click the Delete this section (X) button beside the section label.
  6. Click Save.

4.4 Adding Annotations

Use annotations to provide instructions or additional information about fields on a form.

  1. Click the Field Types tab.
  2. Select the Annotations field type.
  3. Drag the annotation to the appropriate location on the form.
  4. Select the annotation in the form layout.
  5. In the Field Properties tab, select the Annotation Type. See Annotating Forms for more information.
  6. In the Field Properties tab, enter a value for Annotation Text. You can use HTML tags to format the annotation text.
  7. Click Save.

4.5 Adding Formatters

Use formatters to add information from macros or previously scripted UI elements to a form.

  1. Click the Fields tab.
  2. In the filter, enter the string Format.
  3. Select the formatter you want to add to the form.
  4. Drag the formatter to the location you want it to be on the form.
  5. Click Save.

5 Activating Form Design

Form design is available by default for new instances starting with the Eureka release. Administrators who upgrade from a previous version must activate the Form Designer plugin (com.glide.ui.ng.fd) to access the feature. Prior to the Fuji release, the plugin was called NG form designer.

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.
