1 Overview

The Usage Software Licenses module tracks compliance and expiration levels for software packages used by named users. The platform evaluates license usage with a compliance filter based on Company, Location, and Department values. Compliance can be evaluated for each level of an organization, from the narrow focus of users in a single department to membership in an international parent organization with multiple locations. License compliance is calculated for each locality (Company, Location, or Department). Expiration dates are tracked separately in the same form, and have no effect on the compliance calculation. This functionality requires the Software License Management plugin.

Note: The License Bundle (from Software Asset Management Extensions) and Software License Contracts (from Software License Management) options have been deprecated. A significantly improved and updated application named Software Asset Management was included in the Berlin release. For information about upgrading from Software License Management to Software Asset Management, see Upgrading to Software Asset Management.


2 Compliance Filter

The compliance filter for usage licenses uses the following fields to define entitled users. These field values can be used independently or together to calculate compliance. Compliance at all levels is calculated by how many licenses are purchased for the locality compared with the number of actual users, regardless of whether or not they are in the locality.

  • Entitled company: All users at all locations of this company, in all departments are entitled to use this software package, unless other, restricting localities are defined.
  • Entitled location: All users at this location are entitled to use this software package, unless a limiting department is defined.
  • Entitled department: Only the users in this department are entitled to use this software package.

The license form can display information about all named users who are using this software package. The license displays the number of total users and the number of users who are out of the locality.

Possible compliance levels are:

  • Out of compliance: More licenses are being consumed than were purchased. There are more users using this license than the license allows, and some users might not be entitled to use this software package.
  • Unused: The licenses for this software package are currently unused.
  • Reconciliation required: This level is available only for versions prior to Spring 2011 Preview 1. Users who are not entitled to use this software are consuming licenses. Licenses that require reconciliation are considered out of compliance. Reconciliation requires action to ensure that unentitled users are not using the software. This might involve uninstalling software or increasing license counts to match actual user counts.
  • Nearly out of compliance: For a software package to be at this compliance level, more than 95% of the licenses for this software package must be in use by entitled users. Licenses at this level are considered to be In compliance.
  • In compliance: This software package has unused licenses. All users using a license are entitled to use this software package.

3 Checking for Compliance

Software License Management provides a license checking UI Action to recalculate compliance levels, expiration levels, and data reconciliation. Click SLM Usage License Checker after updating fields in the form to recalculate any of these measurements. Usage licenses that are related to a complex license must be compliant before the parent license can be compliant. Only users with the slm_admin role can use the License Checker.



3.1 License Compliance Checker

To check all the licenses in the system for compliance at once, select Software License Management > License Compliance Checker. The platform does not display a prompt, but calculates the compliance of each record for each license type immediately. The progress is displayed above the list.



4 Usage Licenses

To create a new usage license, navigate to Software License Management > Usage Software Licenses and click New, or create a software license contract first as described here:

  1. Create a new Software License Contracts for the software package.
  2. In the Software License Bases Related List, click New.
  3. In the Wizard, click Usage Software License.
    A blank usage license opens.
  4. Complete the form as follows:
    • Entitled localities: Enter a company, location, or department by itself or in any combination as the locality entitlement filter for this software package. Localities are used to calculate the compliance level for a software package by defining the specific location of licensed users.
    • Expiration date: Select the expiration date of this software package. The license checker updates the expiration level, but does not use this calculation to establish license compliance.
    • Parent: If this usage license is directly related to a complex license, it has a parent license record. Select the complex license from the list in the Parent field to establish the upward hierarchy for calculating compliance.
    • License type: Select Named users.
    • Software package: Select the software package to which this license applies. A single software package can be used in multiple usage licenses.
  5. Save the record.
  6. Click Edit in the Used by Related List.
    Select the named users entitled to use this software package.
  7. Click SLM Infrastructure License Checker to calculate the number of licenses used within the locality and the expiration level, and then a second time to display the compliance level.
  8. If you change the values in any of the filters, run the license checker again to recalculate the levels.
    Warning: To remove users from a license, click the Edit button in the Related List and move the user in the slushbucket. Deleting a user directly from a list with the Delete action permanently deletes the user record from the database.
  9. To see an example of a usage license, see below.

4.1 Calculating Compliance

ACME Corporation has a number of regional offices. Each office has three common departments: Sales, IT, and Development. ACME has purchased the Adobe Creative Suite for users in Sales and wants to check compliance for this software package. They define locality filters for Acme Corporation and the IT department, without specifying a location and add the users they know are using the software package. The license is In compliance if the number of licenses used does not exceed the number of licenses entitled for the entire company. When they run the license checker, they discover that:

  • ACME Corporation is using all its Adobe Creative Suite licenses, and is compliant.
  • The licenses will expire within 60 days.
  • The users who are entitled to the licenses are in the Sales department at various ACME locations around the world.
  • One user is in the Development department and is not entitled to use the software. However, this does not affect the compliance calculation.
