Scheduling a CMDB Baseline Update

From ServiceNow Wiki
Scheduling a CMDB Baseline Update
Jump to: navigation, search
 Functionality described here requires the Baseline CMDB plugin.
Caution: The customization described here was developed for use in specific ServiceNow instances, and is not supported by ServiceNow Customer Support. This method is provided as-is and should be tested thoroughly before implementation. Post all questions and comments regarding this customization to our community forum.

1 Overview

Use this procedure to recreate your CMDB Baseline automatically at a configured interval.

2 Scheduling an Automatic Update

To update your Baseline automatically, you create a new scheduled job.

  1. Navigate to System Scheduler > Scheduled Jobs and click New to create a new schedule record.
  2. Give the job a descriptive name, such as Monthly Baseline.
  3. Select the desired trigger type, run time, and other parameters to specify when you'd like the job to run.
    For example, to configure the job to run at 04:00 on the first day of each month, you enter:
    • Trigger type: Day in Month
    • Run time: 04:00:00
    • Run day of month: 1
  4. Paste the following script into the Script field, and then click Submit.
    You may change "baselineName" to whatever you would like the baseline to be named.

var baselineName = "MonthlyBaseline";
var oldBaseline = new GlideRecord("cmdb_baseline"); 
oldBaseline.addQuery("name", baselineName); 
oldBaseline.query(); 
if(oldBaseline.next()) { 
 oldBaseline.deleteRecord(); 
} 
var newBaseline = new GlideRecord("cmdb_baseline"); 
newBaseline.initialize(); 
newBaseline.name = baselineName; 
newBaseline.table = "cmdb_ci"; 
newBaseline.insert();
