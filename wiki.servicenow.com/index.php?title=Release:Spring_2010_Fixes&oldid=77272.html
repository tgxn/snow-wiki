Release:Spring 2010 Fixes

Spring 2010 Fixes
Jump to: navigation, search
  • Fixes for the Spring 2010 release (includes fixes for Winter stable builds, also listed below):
    • PRB230276 -- Business Rules -- Closed incident sometimes not properly getting Active set to false by "task reopener" Business Rule
    • PRB251745 -- Business Rules -- When doing a multiple record update, business rules would not run after the first record if a rule turned workflow off
    • PRB224253 -- CMDB -- CI could improperly be related to itself in cmdb_rel_ci. This is now prevented with a business rule
    • PRB238591 -- Database -- Fields promoted from incident to task table were improperly recreated in incident table during an upgrade
    • PRB237558 -- Discovery -- The application mapping property did not properly disable mapping when set to false
    • PRB222058 -- Discovery -- Because a Windows OS domain name may be different between NBT and WMI, a property has been added to Discovery Properties to determine which method is used to populate the OS domain data
    • PRB201004 -- Discovery -- Multiline SSHCommand probes often did not succeed when source and destination were the same
    • PRB210714 -- Domain -- Domain picker list could include duplicates
    • PRB210874 -- Domain -- Domain picker performance issue
    • PRB043755 -- Domain -- With Domain Separation, Global Text Search result count indicated the wrong number of matches
    • PRB230289 -- Domain -- Using the Domain Picker redirected user to homepage instead of back to the same page
    • PRB202424 -- Email -- Email Client "Reply to" showed the user instead of the reply_to value. Change to have From show sys_email.user and Reply to show sys_email.reply_to
    • PRB224843 -- Email -- If there is a user specified for SMTP, SMTP auth is enabled. This can be disabled by setting the glide.smtp.auth property to false
    • PRB221553 -- Email -- Inactive user accounts improperly could be sent emails
    • PRB253810 -- Homepages -- List of homepages in the homepage icon context menu list and the Switch to Page list were not consistent
    • PRB135482 -- i18n -- question.help_tag was not a translated field
    • PRB043117 -- i18n -- Translated Name / Fields and Translated Text not translated (description of catalog item/help_tag of variable)
    • PRB213591 -- i18n -- Could not send special characters and accents using email client
    • PRB227869 -- Import Sets -- sys_data_source table was not update synched
    • PRB164842 -- Indexing -- Choice list value was indexed, so label could not be searched, and Integer type choice lists were not indexed
    • PRB240989 -- Indexing -- A new failure condition was needed for lucene_index Component Status in System Diagnostics to indicate an EOFException error
    • PRB205504 -- Knowledge -- valid_to date not consistently part of the search filter for Knowledge articles
    • PRB199642 -- Knowledge -- ts_query_kb table recorded incorrect number of search matches if user did not have rights to see all matches
    • PRB228339 -- Knowledge -- Apostrophe in a Knowledge category was not properly displayed
    • PRB138586 -- Labels -- Remove Label UI Action did not immediately remove label from the nav frame
    • PRB236713 -- LDAP -- System LDAP > Transform Maps module did not work properly since getLDAPSources was not Client Callable
    • PRB242282 -- On-call Rotation -- Roster Members were not limited to group members
    • PRB214735 -- Performance -- glide.pool.sweeper not running on instances caused memory use to gradually rise
    • PRB214481 -- Performance -- Obsolete database connection pool with stuck connection could consume memory
    • PRB217499 -- Query -- m2m Task Relations related list had an unnecessarily limiting hardcoded 'active=true' default query
    • PRB197087 -- Reports -- Control Chart might produce incorrect date range in x-axis
    • PRB212659 -- Reports -- is a operator did not work in report filters for derived sys_class_name fields
    • PRB217283 -- Reports -- Filter operand labels incorrect: Nextyear, Nextmonth, Nextweek
    • PRB209688 -- Reports -- Add to Homepage button in report UI might error on an ampersand in the report title
    • PRB213243 -- Reports -- Error in pivot table report when grouped on a derived (dot-walked) field
    • PRB199255 -- Reports -- Behavior when using Show Related Fields in the Report UI field picker was flawed
    • PRB242504 -- Reports -- A trend chart could fail when merging labels if a choice list had duplicate labels
    • PRB207006 -- Schedules -- Customers were wrongly permitted to define a circular reference with Schedules (e.g., A --> B --> A)
    • PRB230327 -- Security -- ACLs for the sys_template table referenced a non-existent role
    • PRB218046 -- Service Catalog -- Add New Item on Request Form improperly included Record Producers
    • PRB202114 -- Service Catalog -- Temporary item options were not properly removed after they were no longer needed
    • PRB259908 -- Service Catalog -- Price was not displayed on order confirmation page if item started with $0 and pricing implication was used
    • PRB223108 -- System -- System Update Sets Previewer and Collision Report generations were timing out - Progress Worker was added to fix
    • PRB215737 -- System -- java.lang.ArrayIndexOutOfBoundsException related to cache management (introduced Winter 2010 Stable 3)
    • PRB238522 -- System -- System connection became locked when connection pool was recycled at the same time as a properties cache flush
    • PRB161457 -- System -- update_synch attribute could be wrongly added to tables. Now a business rule prevents this.
    • PRB221601 -- UI -- User Administration -> Logged In Users module might not correctly render its list due to an error visible in the System Log
    • PRB199836 -- UI -- Activity formatter missing if sysparm_query used in URL
    • PRB148234 -- UI -- Changing font size in the UI did not change the font size of label items in the nav frame
    • PRB210845 -- UI -- Modules that opened a new window opened instead in the main frame when using arrow keys from the nav filter to activate them
    • PRB217835 -- UI -- g_form.hideFieldMsg() had no effect on multiple choice variables when "choice direction" was "down"
    • PRB221951 -- UI -- Customers were wrongly permitted to create a duplicate default view
    • PRB225782 -- UI -- Formatter in Schedule Entry form displayed "null"
    • PRB205817 -- UI -- Derived URL fields did not get a darker label
    • PRB213214 -- UI -- Dirty form bit was not reset when journal field change was undone
    • PRB230130 -- UI -- A single percent sign (%) in the the "Go to" input threw an error
    • PRB242326 -- UI -- It was possible to get invalid SQL generated for a list if a table extending task had a reference to another table extending task BUT had its own display value, and then a choice list element from the second table was added to the list
    • PRB221999 -- UI -- Wrong field hint was sometimes used with an extended label
    • PRB221986 -- UI -- Personalize Label on custom extended table label didn't get the correct label
    • PRB217291 -- UI -- Unable to add multiple "End-Splits" to a form after changing View when personalizing a form
    • PRB194839 -- Update Sets -- Creating a new update set and setting it to the current set did not update the Update Set picker
    • PRB208095 -- Workflow -- A workflow activity with a "Due date based on:" of "duration of date/time field" did not properly display the chosen field
