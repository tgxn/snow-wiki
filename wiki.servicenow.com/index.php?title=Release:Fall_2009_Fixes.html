Release:Fall 2009 Fixes

From ServiceNow Wiki
Home > Release Notes > Legacy Versions > Fall 2009 Fixes
Jump to: navigation, search
  • Fixes for the Fall 2009 release:
    • PRB43899 -- Client Scripts -- Duration field values were not properly saved when populated from an onChange() client script
    • PRB42096 -- Client Scripts -- Could not properly set both category and subcategory with a client script
    • PRB43966 -- CMDB -- Paging on a related list did not work properly in IE7 if the relationship formatter was also on the form
    • PRB43908 -- CMDB -- Defining CI Relationships did not work if the CI name contained "#"
    • PRB44502 -- CMDB -- Reclassifying a Configuration Item improperly modified the sys_created_on value
    • PRB41850 -- CMDB -- GetMicrosoftlicenses() Type Mismatch Error generated when running Help the Help Desk on Windows Vista
    • PRB42986 -- Contextual Security -- save_as_template ACLs did not properly prevent users from creating templates
    • PRB43858 -- Database -- Table Schema functionality reported a non-existing table due to a reference field referencing a nonexistent table
    • PRB43394 -- Database -- Deleting a reference field might fail if the database had an index on that field
    • PRB42962 -- Database -- GlideRecord.update() did not carry setWorkflow(false) when updating a base table record with a different sys_class_name
    • PRB51577 -- Database -- It was easy to give sys_class_name fields invalid values
    • PRB51042 -- Database Views -- Could not add multiple view tables of the same table in a single database view definition
    • PRB43982 -- Discovery -- W2K Pro workstations were improperly classified as servers
    • PRB43922 -- Discovery -- VMWare VM serial numbers were sometimes improperly updated
    • PRB50996 -- Domain -- Fix domain visibility issue in Oracle when number of domains a user was in more than half of the available domains and the number they were not in exceeded 1000
    • PRB43712 -- Email -- Improper spaces within email properties broke them
    • PRB43709 -- Homepages -- ESS users could see homepages listed in the Home icon dropdown that could not be viewed
    • PRB43941 -- Imports -- Starting multiple import set cleanups could affect performance
    • PRB43200 -- Imports -- CSV imports where the header row was not the first row did not work properly
    • PRB51083 -- Imports -- Import using JDBC Probe may not import all records
    • PRB43994 -- Notifications -- Deleting the Unsubscribe notification filter broke the Unsubscribe capability
    • PRB48574 -- Notifications -- Deleting a notification device did not properly delete its notification messages
    • PRB49555 -- Service Catalog -- Order Guide variables did not appear properly in Rule Base conditions (introduced in Spring 2009 Stable 3)
    • PRB45810 -- Service Catalog -- Before query rule on sysapproval_approver broke "any of" process guide approvals
    • PRB49304 -- SLA -- SLA "Time left" was sometimes incorrectly blank
    • PRB42885 -- Table Cleaner -- TableCleaner generated errors if table with extensions had no sys_class_name field
    • PRB45356 -- UI -- Exclude sys_report_summary table from generating infoMessages via the Get Number business rule
    • PRB44011 -- UI -- List calculations for choice fields showed the value, not the label
    • PRB44010 -- UI -- Adding an 'and' condition to a list collector filter generated a JavaScript error
    • PRB43990 -- UI -- When list editing a date/time field, using the ESC or Enter key to submit did not destroy the date/time picker
    • PRB55927 -- UI -- Restore proper behavior of glide.ui.clickthrough.popup property (introduced in Spring 2009 Stable 3)
    • PRB56591 -- UI -- Restore proper behavior of Interceptor modules (introduced in Spring 2009 Stable 3)
    • PRB53451 -- UI -- Tree view related lists gave a JavaScript error on some forms
    • PRB51810 -- UI -- timer fields did not have the same operators as glide_duration fields
    • PRB51478 -- UI -- Read only fields might not appear disabled (grayed out), even though they were
    • PRB51331 -- UI -- Green "back" button did not work properly after clicking My Profile module
    • PRB47376 -- UI -- Font size changer did not work if the css.base.font-size property had no value
    • PRB43951 -- UI -- Activity Formatter sometimes did not load in the correct expand/collapse state
    • PRB42608 -- UI -- Home icon rarely led to nested frames after an instance restart
    • PRB42179 -- UI -- Home icon rarely was missing after an instance restart
    • PRB45780 -- Web Services -- When elementFormDefault="unqualified" for WSDL, SOAP response incorrectly contained the default namespace
    • PRB49522 -- Workflow -- Emails for a single recipient generated from script based on a catalog variable did not send properly
    • PRB46124 -- Workflow -- Restarting a workflow with an approval coordinator did not re-create approvals
    • PRB43964 -- Workflow -- Different results obtained when using Save vs. Update when modifying an Activity's name
  • Fixes for the Spring 2009 Stable 3 build:
    • PRB44288 -- Activity -- Activity formatter showed expand (+) icon on load even though activities were expanded
    • PRB45081 -- Contextual Security -- The Admin overrides flag was not honored for a create rule which was really an overloaded write rule
    • PRB44272 -- Contextual Security -- Incorrect ACL operations checked when editing CI and task relationships
    • PRB43938 -- Contextual Security -- An ACL for a Boolean could be overridden by a UI Policy
    • PRB43986 -- Dictionary -- Deleting a reference field generated an error in MS SQL databases
    • PRB43587 -- Homepages -- Homepage refresh was not supported for all users
    • PRB43991 -- I18n -- Refresh interval values on homepages could not be translated
    • PRB42421 -- I18n -- Wrong language entry sometimes displayed when customizing Labels
    • PRB42110 -- I18n -- Some international characters in attachments did not display properly
    • PRB44306 -- Notifications -- Subscription Based Notifications set an email's Subject after the Body, overwriting a Subject defined in mail script
    • PRB43996 -- Notifications -- Notification Preferences page did not render properly
    • PRB43953 -- Notifications -- Advanced Conditions were not looked at for Subscription Based Notifications
    • PRB43825 -- Other -- Scripted Processors could not be made public
    • PRB43678 -- Other -- New records might be created incorrectly from a reference lookup window if the reference field had a reference qualifier
    • PRB43209 -- Other -- Table extension was not always added to the dictionary entry for customer updates, potentially resulting in an improperly created field
    • PRB44002 -- Reports -- Adding an order to a stacked bar chart generated an error
    • PRB43880 -- Reports -- Fields with an internal type of "timer" could not be aggregated in a report despite essentially being duration fields
    • PRB43874 -- Reports -- Published List reports incorrectly displayed different rowcounts depending on user rowcount preference
    • PRB43872 -- Reports -- Stacked, grouped bar chart with a grid did not display properly if group name had special characters
    • PRB43864 -- Reports -- Histogram reports sometimes generated an error
    • PRB43857 -- Script -- onChange client scripts were not supported for fields with a type of glide_list
    • PRB43804 -- Script -- onChange client scripts were not supported for large text fields
    • PRB43737 -- Script -- The GlideRecord update() method did correctly default to an insert() if the record was not found
    • PRB43870 -- Service Catalog -- Currency plugin did not allow for catalog items to not display a price
    • PRB43014 -- Service Catalog -- Custom icon for a sub-subcategory was not displayed properly
    • PRB46206 -- SSH Timeout -- SSHCommand probes could hang when running on Windows servers and probing certain kinds of CIs
    • PRB43910 -- Survey -- Response field was improperly set to empty string instead of NULL
    • PRB44001 -- UI -- List navigation did not work consistently in MS SQL databases
    • PRB43993 -- UI -- (Introduced Spring 2009 Stable 2) Dependent choice list fields did not appear when list editing their parent field
    • PRB43913 -- UI -- Search Screen modules did not support advanced reference qualifiers
    • PRB43884 -- UI -- Search Screen form generated an error if one of the fields had a fieldname ending with an underscore
    • PRB43853 -- UI -- A field with a type of glide_list sometimes incorrectly appeared as an available related list
    • PRB43810 -- UI -- Spell checker would sometimes hang when checking for misspelled words
    • PRB43801 -- UI -- Switching views when personalizing form did not correctly sort the Available slushbucket
    • PRB43739 -- UI -- If a glide_list field had duplicate entries, the duplicates appeared as sys_id values instead of their display values
    • PRB43275 -- UI -- Sort specified in a filter was not honored when the filter was an OR filter on dates
    • PRB42267 -- UI -- Derived fields more than one table away did not properly update on a form when the parent reference field value changed
    • PRB42918 -- UI Policy -- IE JavaScript error when creating new Survey question
    • PRB43983 -- Update Sets -- Update records might be associated with the wrong table if one tablename was contained within the other
    • PRB43950 -- Upgrade -- Group Approval number sequence was reset during an upgrade
    • PRB43893 -- Web Services -- Incorrect SOAP endpoint was returned in dynamically generated WSDL when instance is proxied
    • PRB43883 -- Web Services -- Modifying an existing web service to add a new field did not create/update the appropriate dictionary update XML
    • PRB43998 -- Workflow -- The Workflow Generate activity did not process child activities of the Approval Coordinator
    • PRB43927 -- Workflow -- Multiple resets triggered in rapid succession could overlap
    • PRB43833 -- Workflow -- Workflows did not support domain separation
  • Fixes for the Spring 2009 Stable 2 build:
    • PRB43795 -- Database Views -- Database View SQL queries were incorrect against MSSQL databases
    • PRB43800 -- Exports -- Basic Authentication did not work properly for remote XML requests, preventing access
    • PRB43313 -- Exports -- Export to PDF from a form incorrectly prompted for authentication
    • PRB43162 -- Import Sets -- During import set creation from a JDBC data source, null Booleans were improperly saved as false
    • PRB43829 -- Knowledge -- Attachments on a kb_submission record were not copied to kb_knowledge record
    • PRB43738 -- Localization -- Firefox's language setting would sometimes interfere with Service Catalog pricing displays
    • PRB43325 -- LDAP -- Specifying an LDAP attribute as binary for an LDAP import when the attribute was not defined as binary in LDAP resulted in an error
    • PRB43813 -- Localization -- Language plugin scripts were unable to parse large sys_choice update files
    • PRB43799 -- Mobile -- In the mobile client, reference fields displayed as dropdowns were not updating
    • PRB43583 -- Navigation -- Unauthenticated users clicking a link in email lost redirect after authenticating
    • PRB41523 -- Other -- Modification count did not get reset when a record was inserted as a copy of another
    • PRB43442 -- Other -- With both an Advanced Reference Qualifier and the Tree Picker active for a reference field, current record values could not be accessed in the reference qualifier
    • PRB43773 -- Performance -- IE6 leaked memory when the Impersonate feature was used
    • PRB43768 -- Reports -- Counts could be incorrect in charts when grouping on a field with a very short length, like the country field in sys_user
    • PRB43806 -- Reports -- A trend report with only one group did not have a legend or tooltip indicating the group
    • PRB43811 -- Service Catalog -- If user's language was other than English, and catalog item had a non-integer price, a catalog subcategory would be formatted improperly
    • PRB42159 -- Templates -- Templates could apply field values a user did not have access to apply. Now ACLs are considered when applying a template
    • PRB43212 -- UI -- Multiple conditions fields on a form caused an error if the top one filtered on a derived field
    • PRB43746 -- UI -- Activity formatter rendered too slowly in IE if a very large number of audits were displayed
    • PRB43819 -- UI -- A showQuickForm UI Action that called a Business Rule action did not result in a update
    • PRB43754 -- UI -- UI Policy for a mandatory field would not be applied correctly if the field value started null when an existing record was loaded in a form
    • PRB41431 -- UI -- Images uploaded to the /updates folder were not displayed when inserting an image in the HTML image UI
    • PRB43798 -- UI -- Fields of type due_date were not correctly blank on Search or Multiple Update forms
    • PRB43673 -- UI -- Images uploaded with uppercase file extensions were not available to the HTML image picker
    • PRB42042 -- UI -- Condition fields did not honor g_form.setReadonly
    • PRB43784 -- UI -- Arguments field on Module form not properly displayed for all module types requiring it
    • PRB43679 -- UI -- Applications with nothing but HTML link modules were not displayed
    • PRB43757 -- UI -- In Safari and Chrome, personalizing a list with the cog icon returned to an empty form
    • PRB43760 -- Upgrade -- Field labels might rarely be overwritten during an upgrade
    • PRB43766 -- Workflow -- Rolling back a workflow with multiple paths would remove open tasks that were not in the rollback loop
  • Fixes for the Spring 2009 Stable 1 build:
    • PRB43686 -- CMDB -- Reclassifying a CI with a currency field generated an error
    • PRB43522 -- Domain -- Global domain displayed as empty on a form
    • PRB43607 -- Domain -- Setting a domain field in script using current.domain_field = "Domain display" no longer worked when it once did
    • PRB43627 -- Email -- When receiving two inline images in an email from Entourage or Thunderbird, only one attachment would be created
    • PRB43318 -- Email -- VCalendar emails were getting two copies of the body text included
    • PRB43308 -- Email -- Multiple notification devices led to duplicates in the email body
    • PRB43307 -- Email -- text/plain emails came out blank
    • PRB41896 -- Email -- Email field on sys_user form did not sync with user's notification device
    • PRB43625 -- Exports -- When exporting an incident record to XML from a task list, attachments were not included
    • PRB43293 -- I18N -- Knowledge base search popup did not properly display some international characters
    • PRB43389 -- I18N -- After switching user languages, catalog questions might not be translated properly
    • PRB43649 -- I18N -- Users with an invalid language specified on their user record did not see expected suggestions for a suggestion field
    • PRB43648 -- I18N -- Attachments with international characters in the filename sometimes appeared incorrectly incorrectly atop a form
    • PRB43633 -- I18N -- Brazilian Portuguese translations were not current
    • PRB43496 -- Import Sets -- Large numbers imported from Excel sometimes were incorrectly displayed in scientific notation
    • PRB43610 -- Indexing -- Text index failure was not handled; now a text index is regenerated upon detection of failure
    • PRB43558 -- Import Sets -- Web service import sets for date fields did not use the authenticated user's time zone
    • PRB43551 -- Plugins -- Company separation plugin did not properly flush cache after activation
    • PRB43594 -- Reports -- (Introduced Spring 2009) Scheduled reports failed without the new Custom Chart plugin
    • PRB43588 -- Reports -- (Introduced Spring 2009) Meter gauges did not properly display upper and lower limits
    • PRB43650 -- Schedules -- Holidays were improperly calculated when placed within "child" schedules with the SLA plugin
    • PRB43617 -- Script -- Journal fields were sometimes not reset when loading the next row in a script, resulting in multiple comments being added to a record
    • PRB43605 -- Script -- gs.eventQueue("eventname") no longer created an event with a table name of null when it once did
    • PRB42853 -- Service Catalog -- Variable Sets were not added to items when ordered from a bundle
    • PRB42563 -- Service Catalog -- Rule Base records with choice list conditions did not load properly in the form after being saved
    • PRB43661 -- Service Catalog -- Catalog categories and their definitions were sometimes cached improperly when using Domain Separation
    • PRB43653 -- Service Catalog -- Prices sometimes displayed improperly depending on language
    • PRB43288 -- SLA -- SLA percentage timer duration calculation was sometimes incorrect
    • PRB43148 -- SLA -- task_sla stage field choices could be modified
    • PRB43451 -- SMS -- SMS Provider Email Suffix had stopped working properly, and "Construct address manually" option on Notification Device did not correctly construct an address
    • PRB42994 -- UI -- Clicking an HTML field's label to focus on the field generated a JavaScript error in IE
    • PRB42899 -- UI -- Activity formatter title tags were not properly escaped
    • PRB43604 -- UI -- Incorrect reference field values no longer safely returned the user to the form to correct the value
    • PRB43603 -- UI -- Tree picker sometimes would not properly sort entries
    • PRB43593 -- UI -- Locations with no name would not render properly in the tree picker
    • PRB43570 -- UI -- User preference to expand/collapse Service Catalog categories on homepage was not being honored
    • PRB42595 -- UI -- Spell checking improperly set a field's modified flag
    • PRB41670 -- UI -- Welcome Page Content was not properly escaped
    • PRB41864 -- UI -- HTML fields generated an error in Safari and Chrome
    • PRB42032 -- UI -- IE6 has a bug preventing "is one of" filter conditions from rendering properly on load. Fix works around the IE bug.
    • PRB43630 -- Update Sets -- Update Set picker did not correctly set the selected update set
    • PRB43468 -- Web Services -- Journal elements were improperly sent with SOAP responses, contrary to the WSDL
    • PRB43641 -- Workflow -- When the last version of a workflow was deleted, the wf_workflow master was not properly deleted
    • PRB43612 -- Workflow -- Irrelevant stages were sometimes displayed, e.g. if a stage's tasks were all skipped and never visited or if a stage only had generate-on-skip tasks that were not generated
    • PRB43337 -- Workflow -- Calculating due date via script did not add the seconds to the start date
    • PRB43599 -- Workflow -- Rollback skipped over tasks that were rolled back, because they were not set to active
