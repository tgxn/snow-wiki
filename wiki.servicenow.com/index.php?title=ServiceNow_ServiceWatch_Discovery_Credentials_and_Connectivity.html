1 Credentials and commands necessary for ServiceWatch discovery

1.1 Operating Systems

You need to obtain root privileges to run some of the discovery commands listed below. You can do this by editing the sudoers file as shown in the following example:

Example of the sudoer file on Linux
qauser ALL=/bin/netstat, /bin/cat, /bin/ls, /usr/sbin/dmidecode, /usr/bin/stat, /usr/bin/find, /usr/bin/stat, /usr/sbin/lvs, /sbin/lspci, /sbin/fdisk
Example of updating the sudoer file on Linux
If a certain command exists in several places on the machine, ServiceWatch uses the following search path to find it. The sudoers file should be updated accordingly:

  • /usr/local/sbin
  • /usr/local/bin
  • /sbin:/bin
  • /usr/sbin
  • /usr/bin

1.1.1 AIX

Users

  • Non-root user and password allows running selected commands using the sudo utility.
  • Non-root user and password + root password - for running commands with “su”.
  • Root user and password

Commands


Process-related commands (locating processes and their arguments)

  • procwdx <pid>
  • ps -ef

Finding process listening on a port, outgoing connections (all fallbacks)

  • pfiles
  • netstat -Aan
  • netstat -rn -f inet

Storage related

  • lspv
  • lsvg
  • lspath
  • lscfg -vl
  • lsdev -l ${controller} -F'parent'
  • lsdev -Ccadapter -l fcs* | awk '{print \$1}'
  • lspath -l ${diskName} -F'parent connection'
  • lscfg -vp

Network related

  • entstat -d
  • /etc/ifconfig -a

Miscellaneous

  • cat
  • ls
  • ps
  • procwdx - get working directory of a process
  • rmsock - find process listening on a specific port
  • (optional) bootinfo - find the architecture of a server, for example if it is 32 or 64-bit system.
  • oslevel
  • getconf HARDWARE_BITMODE
  • lsattr -El sys0 -a systemid - serial number
  • uname -L
  • bootinfo –y

Sudo commands

  • cat
  • ls
  • ps
  • procwdx (get working directory of a process)
  • rmsock (find process listening on a specific port)
  • bootinfo (find the architecture of a server, i.e. 32/64 bit) – optional

1.1.4 Solaris

Users

  • Non-root user and password allows running selected commands using the sudo utility.
  • Non-root user and password + root password - for running commands with “su”.
  • Root user and password

Commands
Process related commands (locating processes and their arguments

  • inetadm | grep `grep -w " + port + "/tcp /etc/services - not mandatory
  • inetadm -l - not mandatory
  • pwdx <pid>
  • /usr/ucb/ps -aglxwww
  • netstat -rnv

Finding process listening on a port, outgoing connections (all fallbacks)

  • lsof -Pnl +M -i
  • lsof -p
  • dtrace -C -s - fall back to lsof
  • netstat -an

Storage-related

  • iostat -En
  • lofiadm
  • zpool status
  • mount -v
  • mount (no args)
  • /sbin/lspci
  • iscsiadm list initiator-node
  • zonecfg -z " + localZone + " info
  • vxdisk list
  • /usr/sbin/fcinfo hba-port
  • /usr/sbin/metastat

Network-related

  • zoneadm list
  • /sbin/ifconfig -a
  • ifconfig -a"
  • arp -an

Miscellaneous

  • stty raw
  • id
  • isainfo -v
  • uname -p
  • uname -v

Sudo commands

  • cat
  • ls
  • pwdx
  • pargs
  • zonecfg (in global zone only)
  • Either of:
    • dtrace
    • o lsof

For storage discovery:

  • fcinfo
  • isainfo (only if storage is needed)
  • iscsiadm list – only if SCSI is being used
  • vxdisk list– only if Veritas Volume Manager is installed
  • find /sys/class/scsi_host/ -name 'port_name' -print -exec cat {} \; – only if Emulex HBA is used
  • find /sys/block/ -name device -exec ls -l {} \; – only if Emulex HBA is used

1.1.5 Windows servers

Users

  • Local administrator, or
  • Domain administrator

WMI queries

  • Select Name,Started from Win32_Service where name like 'NeebulaPsExe%' and Started = 'False'
  • Select * from Win32_ComputerSystem
  • Select Caption from Win32_OperatingSystem
  • Select Version from Win32_OperatingSystem
  • Select SerialNumber from Win32_BIOS
  • Select AddressWidth from Win32_Processor
  • SELECT MACAddress, IPAddress, Description, MACAddress, Caption, IPSubnet, InterfaceIndex FROM Win32_NetworkAdapte
  • Select DeviceID, FileSystem, Name, VolumeSerialNumber, Size from Win32_LogicalDisk Where DriveType = 3
  • ASSOCIATORS OF {Win32_LogicalDisk.DeviceID='C:'} where AssocClass=Win32_LogicalDiskToPartition
  • Select DeviceID, Caption, Size, InterfaceType, Index, SCSIBus, SCSITargetId, SCSILogicalUnit from Win32_DiskDrive
  • Select SerialNumber from Win32_BIOS
  • ASSOCIATORS OF {Win32_DiskDrive.DeviceID='\\.\PHYSICALDRIVE0'} where AssocClass = Win32_DiskDriveToDiskPartition
  • Select Destination, InterfaceIndex, Mask, Metric1, Metric2, Metric3, Metric4, Metric5, NextHop, Protocol, Status,
  • Select * From Win32_Process where (ProcessId = ?)

Commands Process-related

  • get_process_info.exe – extract information on processes.

Cleaning temporary files created by ServiceWatch

  • clean_temp_files.vbs =- cleans temporary files created by ServiceWatch

Miscellaneous

  • cscript clean_temp_files.vbs
  • dir /B neebula_host_id_*
  • wlbs display | findstr ClusterIPAddress
  • nslookup -d 10.1.0.144 | findstr "Server: name"
  • "fcinfo.exe" /details
  • iscsicli SessionList
  • arp -a
  • netstat -a -n -o | findstr /R ":80"
  • get_process_info.exe <pid>

2 Firewall traversal guidelines

The collector(s) within Customer’s network need to be able to initiate communication to ServiceNow’s ServiceWatch network at address servicewatch.servicenow.com on port 443 (using HTTPS protocol). Optionally, the collector may use a proxy to communicate with the host server (if such proxy exists) using either basic or NTLM authentication.

2.1 Collector to host communication

2.1.1 Windows host

The ServiceWatch Collector communicates with hosts using WMI (Windows Management Instrumentation) which runs on top of RPC. The host may allocate ports in the range of 1024-65535 arbitrarily.

You can place an additional ServiceWatch Collector in the same zone as the host it needs to connect to.

If the ServiceWatch Collector is located in a different zone, enable firewall traversal as follows:

  1. Open the firewall for ports 135,1024-65535 or configure WMI to use a fixed port (http://support.microsoft.com/kb/897571/)
  2. Open one of the following ports:
    • 8585 (HTTP) - host-server connection over HTTP. The port number is configurable.
    • 445 (TCP) - for access to Administrative Shares using SMB.

2.1.2 Unix/Linux host

Open the following port to enable firewall traversal from a ServiceWatch Collector to a host.

  • 22 (TCP) – for ServiceWatch Collector-host connection over SSH. The port number is configurable.

2.2 Network devices

Open the following port to enable firewall traversal from ServiceWatch Collector to ServiceWatch Server:

  • 443 (HTTPS) – for enabling communication initiated by ServiceWatch Collector. This port number is configurable.

2.3 Collector to ServiceWatch Server Connection

Open one of the following ports to enable firewall traversal from ServiceWatch Collector to ServiceWatch Server to enable communication initiated by ServiceWatch Collector:

  • 443 (HTTPS) – for SaaS ServiceWatch deployments. This port number is configurable.
  • 8443 (HTTPS) - for on-premise ServiceWatch deployments. This port number is configurable.

2.4 Browser to ServiceWatch Server Connection

Open one of the following ports to enable access to the ServiceWatch interface:

  • 443 (HTTPS) – for SaaS ServiceWatch deployments. This port number is configurable.
  • 8080 (HTTP) – for on-premise ServiceWatch deployments. This port number is configurable.

2.5 ServiceWatch Server to 3rd party systems

Open the following ports on the ServiceWatch Server:

  • 443 (HTTPS) – to allow ServiceWatch Server to communicate with VMware vCenter.
  • 80 (HTTP) - to allow ServiceWatch Server to communicate Citrix XenCenter.
