June 2011 Preview 2 Notable Changes

June 2011 Preview 2 Notable Changes

The following are notable changes you may encounter with our June 2011 Preview 2 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before. For additional changes, see June 2011 Preview 2 Fixes.

1 Changes for June 2011 Preview 2

1.1 Chat Enhancements

The following enhancements have been added to Chat:

  • The buddy list (formerly My Stuff window) has been redesigned for better usability. Users can now change their profile picture and status and more easily view contact and room status.
  • Internationalization (i18n) support has been added for system messages and UI elements in the chat desktop.

1.2 Advanced Control Test Logic for Governance

New Control Test Condition Collections for the IT Governance, Risk and Compliance plugin allow more advanced logic in determining what records are in or out of compliance during data sampling.

1.3 Approvals in Workflows for Non-Task Tables

The Approval - User and Approval Action workflow activities are now available for non-task tables.

1.4 Approvals now can refer to any Record

Approvals are now able to refer to any record using the Approving document ID field.

1.5 Parent Breadcrumbs Formatter

A new Task Parent Breadcrumbs Formatter is available to show the parents of a record in breadcrumb format on the form. This is most useful in Project Management and Release Management, where tasks exist in multi-level hierarchies.

1.6 g_form.enableAttachments()

g_form.disableAttachments() has been available to disable attachments on a form. Now g_form.enableAttachments() will do the reverse.

1.7 Windows Mobile and Windows Phone supported versions

Added support for Windows Mobile 6.5 and Windows Phone 7

1.8 Variable types hidden for Survey questions

A new client script hides Container Start and Container End variable types for the survey_question_new form, since those types aren't supported for surveys.

1.9 glide_list display change in lists

When displaying a glide_list field in a list, a space has been added between a separating comma and the next display value. e.g., instead of "Beth Anglin,Don Goodliffe,David Loo", display "Beth Anglin, Don Goodliffe, David Loo".

1.10 List cell title attribute (tooltip) change

If the entire content of a field is displayed in a list cell, it is no longer duplicated in a tooltip of the cell. The tooltip is only included if it is different than what is already displayed in the cell (i.e., if it's been truncated).

1.11 List Edit for Workflow fields

Fields of type "workflow" can now be list edited.

1.12 Index for document_id fields

Fields of type "document_id" now get a database index by default, just like reference fields.

1.13 No sorting for glide_list

Fields of type "glide_list" can no longer be sorted, since sorting has never been supported for that field type and led to unexpected results.

1.14 No Record Producers for Wizard Catalog Order panels

Record Producers (just like Order Guides), are not supported when ordering via wizard Catalog Order panel, so they are now filtered out with a reference qualifier (just like Order Guides).

1.15 Knowledge article link to Affected Product

If a user can read the Affected Product record (cmdb_ci reference) for a knowledge article, a link is provided to it from the article.

1.17 Catalog Request clones populate delivery address differently

When a Catalog Request is cloned for other users in two-step checkout, the cloned requests have their Delivery Address pulled from the users' user records. Previously the original Request's delivery address was used for all.

1.18 Catalog variable removal via client script

Can now remove label/checkbox label/container variables from a catalog item and resulting task.

1.19 Removed support for Choice type 4 (pick list external)

With List v2, support has been removed for Dictionary choice type '4' (pick list external).

1.20 List v1 is no longer available

The boats have been burned. Long live List v2!

1.21 impersonator role

New "impersonator" role. A user with this role can impersonate other users. If the impersonator does not have the admin role, they are not allowed to impersonate someone who does. The impersonation dialog's reference field filters out admin users if the user doesn't have admin.

1.22 Friendlier NULL

In a grouped list, if the group value is "NULL", "(empty)" is shown instead. NULL can be scary.

1.23 Display field magic

If no display field is specified for a table, and a "name" field doesn't exist, now the "u_name" will be used as the display field if it exists before we punt and use sys_id.

1.24 Email Client active for Incidents

The email_client=true attribute has been added to the Incident table, enabling the email client for Incidents. Customers who do not wish to use the Email Client should set the attribute to false.

1.25 Legacy Certification app renamed to Record Certification

Since the legacy Certification application only allowed for certification of records (vs. elements of records), it has been renamed to Record Certification.

1.26 List DOM element and attribute changes

Significant changes have been made to how lists are built in HTML and JavaScript. Scripts that inadvisedly rely on specific DOM elements or attributes of a list may be thwarted with this release.

1.27 Pivot Table reports can be emailed

Pivot table reports can now be scheduled and emailed. Reference qualifier on the Report field on the Scheduled Report form has been modified to allow this.

1.28 Insert and 'Insert and Stay' UI Actions generate new number

Use of the Insert or 'Insert and Stay' UI Action will blank out the number field, ensuring a new number is generated for the new record.

1.29 Wikitext field attribute "dual_mode"

Wikitext field optional attribute "dual_mode=true" displays both editor and preview simultaneous when in edit mode. Refresh preview with refresh icon, similar to MediaWiki preview.

1.30 Performance improvements

MID Server throughput has been improved, including processing for Discovery and import sets. ODBC driver performance has been improved, preventing out-of-memory errors when a query produces an extremely large result set.
