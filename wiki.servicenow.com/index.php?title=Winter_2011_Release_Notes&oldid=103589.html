1 Overview

The Winter 2011 release includes the following enhancements and new functionality.

To download a manual detailing new functionality, visit: Winter 2011 Release.

This release is no longer recommended for upgrades. If you want to upgrade an instance, consider a current version. For the Winter 2011 release:

Build date: 02-04-2011_1258
Build tag: glide-winter2011-01-21-2011
 Note: See how to determine the version on your instance.

3 Description of Functionality

The Winter 2011 release contains the following new or improved functionality.


— Process Applications —

  • Project Management Improvements: Project Management now incorporates Skills Management and Task Resources. In addition, there is a new Project Management v2 Costing Demo Data available.
  • Cost Management Improvements: Cost Management now use Time Cards to calculate labor costs, add group rates to the group resources, and CI Rate Cards can use the Condition Builder to associate with conditions. In addition, there is new Cost Management Demo Data available.


— MID Servers —

  • Encrypting MID Server login credentials: Field encryption is now available in the MID Server config.xml file, enabling an administrator to encrypt the MID Server credentials that appear in clear text by default.
  • MID Server Heartbeat: Discovery now checks the MID Server for a response every 5 minutes, using a Heartbeat probe. The response triggers system events that can be used to notify users of a change in the MID Server's status.
  • MID Server Autofinder for Runbook: MID Servers can be associated with IP address ranges, enabling Runbook to select the correct MID Server to use for an Orchestration activity based on the IP address of the target machine.


— Platform Administration —

  • History Sets: History Sets provide a new table for tracking audit history for improved search and export.
  • Upgrade Monitor: A new user interface is available for monitoring the progress of upgrades.
  • Wizards enhancements:
    • Basic Panel Flow for Wizards: This feature allows wizards to moves through panels in order, without the need to define logic for each transition. The Basic panel flow field was added to the Wizard [expert] table and the order field was added to the Wizard Panel [expert_panel] table.
    • Role access control for Wizards: Administrators can now restrict user access to wizards by role.
    • UI Policy for Wizards: UI Policies can now be applied to Wizards.
    • Public Wizards: Administrators can now make wizards available to users that are not logged in (public).
    • Logged in users may now have more than one wizard in progress at a time. The expert field was added to the Wizard Instances [expert_instance] table.
  • New ACL Operation: A new Access Control Rule (ACL) operation (report_on) allows administrators to restrict who can create a report against a specific table. If a user does not have report_on access for a table, the table does not appear in the Table field when the user creates a new report.
  • Client Transaction Timings Plugin: The plugin is enhanced to track additional fields and provide more detailed information.
  • Group On-Call Rotation Plugin Enhancements: New roles allow easier delegation of Rota management, notifications and reminders now have more configuration options, and coverage is now required for all active periods.


— Additional Functionality —

  • Discovery for VMware vCenter: ServiceNow Discovery can now explore the VMware vCenter process running on a Windows machine and return information on managed ESX Servers, hosted virtual machines, and resource pools.
  • Monitoring Performance on Threads: Active threads are now monitored for system diagnostics, both within a Performance Chart and within the transaction log.
  • Inserting/Updating GlideRecord with References: When inserting or updating a record, these GlideRecord methods also insert or update related records. Any related records are inserted/updated first, and then the record itself is inserted/updated.
  • GlideForm getReference(callback) API: The getReference() method now accepts a second parameter, a callback function, for asynchronous use.
  • Display Business Rules: Business Rules can now be set to run onDisplay to populate a form's scratchpad, accessible from client scripts.
  • Form Scratchpad: Client Scripts can now access data populated in the scratchpad by Business Rules.
  • Aggregate Web Service plugin: This direct web service API function provides for querying a table using an aggregate function such as SUM, COUNT, MIN, MAX, AVG.
  • Email Accounts Plugin: This plugin provides a new interface for configuring POP3 and SMTP accounts and replaces Email Properties from previous versions for configuring email accounts.
  • Email - OAuth support for IMAP and SMTP plugin. This plugin enables an organization to configure secure IMAP and SMTP access to Gmail accounts through Google's OAuth support, using the new Email Accounts plugin.
  • Field decoration attribute. The field_decoration attribute is similar to ref_contribution but also works with non-reference fields.
