1 Overview

The SDLC integration with Project Portfolio Suite provides a solution for IT application development by integrating the SDLC application with project-based IT development. IT organizations typically work on multiple projects with shorter time schedules. The project planning and tracking is done using Waterfall but the development team prefers Agile or Scrum methodologies to carry out their work. It then becomes a challenge to track project status when there is an Agile development phase. This integration facilitates a combination of Waterfall and Agile methods for project management and development. The SDLC process is integrated with Project Portfolio Suite, starting with the Fuji release.

The SDLC integration with Project Portfolio Suite provides the following process flow:

  • Create a team, add team members, and set team capacity
  • Create sprints
  • Assign teams to projects from the project workbench
  • Maintain team and project backlogs
  • Perform team sprint planning
  • Create stories, including one-off stories that are not related to projects
  • Track progress in stories, phases, and projects

Multiple projects can be under development and teams can be assigned to multiple projects. Each project and each team has its own backlog. The project backlog includes all stories related to a project. The team backlog includes stories from multiple projects that have been assigned to the team. The team grooms this combined backlog, ranks the stories and assigns points, and adds stories to sprints based on priority.

To use the SDLC integration with Project Portfolio Suite, activate the Project Portfolio Suite plugin.

2 Creating a Team

Navigate to Project > Agile > Teams to view a list of current teams. Create a new team from this list or view the Team form for a current team and add team members.

  1. Navigate to Project > Agile > Teams.
  2. Click New.
  3. Fill in the fields, as appropriate (see table).
  4. Click Submit.
Team form

Field Description
Name Enter the name of the team.
Manager Select the manager of the team.
Points Enter the number of points assigned to the team for each sprint.
Scrum master Select the scrum master of the team.
Related Links
Create Sprints Access the Create Sprints for Team dialog box. Fill in the fields and then click OK to create multiple sprints for the current team. The new sprints are added to the Sprints related list.
Manage Stories Access the Manage Stories form. Use this form to manage the backlog across all projects for the current team.
Plan Sprints Access the Sprint Planning form. Use this form to display and groom stories based on project or sprint and add or remove stories from projects and sprints.
Velocity Chart Access a velocity chart for the team.
View Story Points by Sprints Access a chart which shows how story points in a project are allotted across sprints.

2.1 Adding Team Members

  1. Navigate to Project > Agile > Teams.
  2. Open the desired team.
  3. Click New in the Team Members related list.
  4. Add the name of the team member in the Name field.
  5. Select a Scrum Role for the team member.
  6. Click Submit.

2.2 Assigning Team Capacity

The team capacity is determined by the scrum master and is based on the historical velocity of the team for each sprint.

  1. Navigate to Project > Agile > Teams.
  2. Open the desired team.
  3. Determine the team capacity in points and enter this number in the Points field.
  4. Click Update.

3 Creating Sprints

Create multiple sprints from the Team form using the Create Sprints related link.

  1. Navigate to Project > Agile > Teams.
  2. Open the desired team.
  3. Click the Create Sprints related link to open the Create Sprints for Team dialog box.
  4. Fill in the following fields.
    • Name: the name of the sprint.
    • Starting Number: the number of the first sprint to be created.
    • Start date: the start date for the first sprint to be created.
    • Duration: the number of days in a sprint.
    • Number of sprints: the number of sprints to be created.
  5. Click OK.
    The new sprints are added to the Sprints related list on the Team form.

Create individual sprints from the Sprints related list.

  1. Navigate to Project > Agile > Teams.
  2. Open the desired team.
  3. Click New in the Sprints related list to open a new Sprint form.
  4. Fill in the following fields.
    • Short description: the name or a brief description of the sprint.
    • Planned start date: the start date for this sprint.
    • Planned end date: the end date for this sprint.
  5. Click Submit.

4 Assigning a Team to a Project

Assign a team to a project in the project workbench. A project can have one development phase and a development phase can have an assigned team. Create a development phase in the project timeline or edit an existing Agile phase and assign a team by selecting the team from the Team field. For more information, see Creating a Phase in the Timeline.

5 Creating Stories

Navigate to Project > Agile > Stories to view a list of current active stories. This list includes two types of stories: those associated with a project and one-off stories, which are not associated to either a product or a project. Create a new story from this list or view the Story form for a current story. Add scrum tasks from the Story form and view story and task progress boards.

Stories can be associated to projects but this is not mandatory, starting with the Fuji release. One-off stories can be created without a link to a product or a project; these stories are created with only a short description and can then be assigned to a project or a team.

  1. Navigate to Project > Agile > Stories.
  2. Click New.
  3. Fill in the fields, as appropriate (see table).
  4. Click Submit.
Story form

Field Description
Number The automatically assigned story number. For example, STRY0010013.
Project The associated project for this story.
Project phase The name of the Agile phase, if the story is associated with a phase in the project workbench.
Team The development team assigned to this story.
Sprint The sprint assignment for this story.
Points The number of points assigned to this story.
Priority The priority assigned to the story.
  • 1 - Critical
  • 2 - High
  • 3 - Moderate
  • 4 - Low
Type The type of story:
  • Development
  • Documentation
  • Spike
State The current state of the story:
  • Draft
  • Estimation
  • Ready
  • Work in progress
  • Awaiting info
  • Ready for testing
  • Testing
  • Complete
  • Cancelled
  • Testing failed
Assigned to The development team member assigned to this story.
Blocked An indicator that shows if the story is blocked and cannot move forward.
Short description A brief description of the story.
Acceptance criteria The criteria that the story must meet in order to be considered complete.
Description A more detailed description of the story.
Work notes Any additional notes or information to be saved with the story.
Related Links
Add Scrum Tasks Allows the user to add multiple scrum tasks to the story from the Add Scrum Tasks dialog box. Select the number of tasks to add for each task type: Analysis, Coding, Documentation, and Testing. Then click OK. The tasks are added to the Scrum Tasks related list.
Story Progress Board Displays a progress board with all of the stories in a sprint. For more information, see Story Progress Board.
Task Progress Board Displays a progress board with all of the tasks in a sprint. For more information, see Task Progress Board.

6 Managing Stories

Manage stories from the Team form using the Manage Stories related link or from the project workbench using the Manage Stories button in the workbench header. The Manage Stories form displays the backlog for the currently selected team. From this form, the team can filter the stories that are displayed, rank stories and assign points, and assign stories to sprints.

The team backlog includes two types of stories:

  • Project stories are associated to a specific project.
  • One-off stories are not associated with a project or a product, but are associated with a team.

The upper section of the Manage Stories form shows the backlog across all projects for the current team, including the total number of stories and points. The lower section of this form has two tabs that you can use to filter the stories displayed in the backlog.

  • In the first tab, select Project or One-off. If you select Project, you can make an additional selection from the second tab. If you select One-off, the second tab disappears because one-off stories are not associated with either a product or a project.
  • If you select Project in the first tab, select either All projects or a specific project in the second tab.

The stories displayed in the Backlog list are determined by the selections from these two tabs. After filtering and displaying the desired stories, drag and drop the stories in this list to rank them. The top of the list is the highest ranking. If necessary, open a story and assign points in the Points field.

  1. Navigate to Project > Agile > Teams.
  2. Open the desired team.
  3. Click the Manage Stories related link to open the Manage Stories form.
  4. Make a selection from the first tab, either Project or One-Off.
  5. If Project is selected in the first tab, select either All projects or a specific project from the second tab.
  6. Rank the stories in the Backlog list by dragging and dropping.
  7. Assign points to a story by opening the story and entering a number in the Points field.
  8. Click Update.
Manage Stories form

6.1 Creating a One-Off Story

From the Manage Stories form, you can create a one-off story and assign the story to a team. Creating a one-off story does not require a reference to a specific project.

  1. Navigate to Project > Agile > Teams.
  2. Open the desired team.
  3. Click the Manage Stories related link to open the Manage Stories form.
  4. In the lower section of the Manage Stories form, select One-Off from the first tab.
  5. Click New to open the Story form in One-Off view.
  6. Enter a Short description.
  7. If desired, select a Priority.
  8. Select a Team.
  9. Click Submit.

7 Planning Sprints

Use the Sprint Planning page to view and manage the backlog for a team across multiple projects, for individual projects, and for sprints within a project. Rank stories in the backlog and assign or remove stories from projects and sprints.

The Team field in the upper right corner displays the current team. The upper half of the Sprint Planning form shows the following information:

  • On the left side, the backlog for all current active projects, including total stories and points.
  • On the right side, the team details for the selected sprint, including the maximum point capacity and the points allocated. The capacity is also represented by the dial.

The lower half of the Sprint Planning form uses two columns to display the backlog for two separate projects or sprints. Use the tabs at the top of the columns to make the desired selections. Then use the arrow buttons between the two columns to move stories to the desired projects and sprints.

  1. Navigate to Project > Agile > Teams.
  2. Open the desired team.
  3. Click the Plan Sprints related link to open the Sprint Planning form.
  4. Select a team from the Team field.
    The Sprint Planning form displays the pending backlog for the selected team.
  5. In the left column, make a selection from the first tab: Project, Sprint, or One-Off.
  6. Depending on the selection in the first tab, make a selection from the second tab:
    • If Project, select All projects or a specific project.
    • If Sprint, select the sprint.
    The backlog in the left column is updated to display the stories based on the selections.
  7. In the right column, make a selection from the first tab: Project, Sprint, or One-Off.
  8. Depending on the selection in the first tab, make a selection from the second tab:
    • If Project, select All projects or a specific project.
    • If Sprint, select the sprint.
    The backlog in the right column is updated to display the stories based on the selections.
  9. If desired, rank the stories in the backlog lists by dragging and dropping.
  10. Enable the check box for a story and use the arrow buttons between the columns to move the story to the desired location.
  11. Use the information at the top of the form to track point capacity and points allocated.
Sprint Planning form

8 Tracking Progress

Stories are assigned to a project and are tied to an Agile development phase in the project workbench. As stories are completed, the story points and the story status get rolled up to the sprint and project phase. In turn, the phase gets rolled up to the project. In the project workbench timeline, the color in the phase bar increases to show progress and the Percentage complete field is updated.

9 Integration with Project Portfolio Suite

Activating the Project Portfolio Suite plugin adds the following SDLC modules to the Project Management menu:

  • Agile
    • Stories: displays the Stories list, which shows a list of current active stories and the associated projects. Add a new story or click a story to open the Story form. Add scrum tasks to a story.
    • Teams: displays the Teams list, which shows a list of current teams. Click a team to open the Team form. Create and plan sprints, manage stories, view velocity charts, and view story points by sprint from the Team form.
    • My Stories: displays a list of stories assigned to the current user.
    • Sprint Planning: displays the Sprint Planning form for the selected team. Use this form to see the team capacity and points allocated for each sprint. Also use this form to manage the project backlog and assign stories to sprints.

Activating this plugin also allows the following tasks from the project workbench:

  • Create an Agile phase for a project
  • Assign a team to an Agile phase within a project
  • Select an Agile phase to display the project-related stories in the list view
  • Use the Manage Stories button to open the Manage Stories form
