1 Summary

Available with this preview are High Security settings enabled by default on all new instances and enhancements to Zing text search.

The October2011 Preview 1 release became available July 8, 2011.

This release is no longer recommended for upgrades. If you want to upgrade an instance, consider a current version. For the October2011 Preview 1 release:

Build date: 07-08-2011_1436
Build tag: glide-october2011-preview1-07-07-2011
 Note: See how to determine the version on your instance.

2 Upgrade Recommendations

Production instances should run on major releases. Major releases go through the greatest amount of testing and are mostly widely used by the customer base. Customers may want to run a preview release on a production instance in limited cases, such as when functionality in the preview is critical to business processes or when the preview is installed during an implementation and an upgrade would constitute a major change before go-live.

3 New in This Preview

The October2011 Preview 1 release contains the following new or improved functionality.

Feature Description Activated
Discovery / Runbook
Discovery enhancements This release contains a number of minor updates, including new types of data discovered, new scripting functionality for probes, and new MID Server parameters. All these enhancements are described in a summary section as well as in appropriate sections in the documentation set. Request Discovery plugin
VMware probe A single probe replaces the five VMware probes used in previous versions to run each of the MID Server VMware script includes. This probe can execute any MID Server script include in the ECC queue name set by a VMware Runbook activity. Request Runbook plugin
Platform Administration
Content Management, Live Feed enhancement The ESS Portal now includes the Portal - Live page and Live Feed dynamic block. Use these components to provide access to Live feed from CMS pages. Activate Content Management plugin (automatic for new instances)
Global Text Search enhancements Global text search can now render results in parallel to improve performance. Administrators can configure this feature. Automatic for all instances
High Security High Security settings are now enabled by default on all new instances of ServiceNow. Role-based controls have been added to allow increased privilege. Automatic for new instances
Upload multiple image files Administrators and users with the image_admin role can now upload multiple image files as a .zip file. Automatic for all instances
Workflow "Rollback To" activity The Rollback To activity transitions directly to the activity specified by the transition line attachments and determines which activities to reset based on the actual workflow sequence of activities and not the execution order. Automatic for all instances
Zing text search enhancements Administrators and users with the ts_admin role now have improved controls for text indexing. Automatic for all instances
Process Applications
Gantt Chart enhancements

The Gantt Chart functionality has been improved to prevent dragging or dropping tasks in the following cases:

  • If the task has already started (has actual start date),
  • If the task has already ended (has actual end date),
  • If the task has a time constraint of ASAP, since the start date is calculated by the system based on predecessors.
 Automatic for all instances
ServiceNow Basics
Top Searches homepage The Top Searches homepage displays the most popular searches over the past hour, day, week, or month. The homepage includes widgets for specific tables and across the entire instance. Automatic for all instances
Wildcard searches Zing now supports wildcards in searches. Automatic for all instances

4 Other Notable Changes

This release includes the following minor enhancements or performance improvements.


Feature Description
Integrations
MID Server The mid_server role has been added to the Read ACL for the sys_data_source table.
LDAP Server The Advanced view of the LDAP Server form is no longer created. Advanced fields are now in an Advanced Options form section on the Default view.
Notifications
Inbound Email Actions A new reply Inbound Email Actions was added for the problem, problem_task, change_request, change_task, and sc_req_item tables (for instances created after June 3, 2011).
Email Notifications and Templates In several default Email Notifications and Templates, ${URI} has been replaced with ${URI_REF} where appropriate. Customized notifications will not get the change.
New email notifications Nineteen new email notifications were added for comment/worknote events and task assignment in the change_task, problem_task, problem, and change_request tables (applies to instances created after June 3, 2011).
Platform Administration
List record count This release adds List v2 support to display the total record count for grouped lists. An example would be to show 57 total Incidents just below the paging controls in the list. This is shown only when the list is grouped on a field.
Upgrades Customer updates (sys_update_xml records) with no Update Set value were being ignored during an upgrade, so customizations could get overwritten. Such updates are no longer ignored.
Filter labels In filters, the label Apply Filter was changed to Run and Save Filter label was changed to Save... for consistency.
High Security XML escaping behavior can be modified only by users with the elevated security_admin role.
Arranging fields using the slushbucket Moving a field from Available to Selected in the form/list slushbucket now places the new field just after the selected field instead of always at the bottom. This saves reduces the number of up clicks when creating a form.
Form Annotation Module Form Annotations are enabled by default for instances created after June 3, 2011. Two new tables were added: Form Annotations [sys_ui_annotation] and Form Annotation Types [sys_ui_annotation_type]. A new module called Form Annotation Types was added under System UI.
Enabling Form Annotation The form/list personalization slushbucket layout was modified to accommodate enabling of Form Annotations. This change eliminates the "Create new field" button, which was present when annotations were enabled previously, and moves field creation back to the main slushbucket form for everyone. This change moves annotation customization options to the right of the Selected side of the slushbucket. This update is designed to fit everything without scrollbars on a 1280px width screen.
List controls The New button was eliminated from the Roles Related List on the User and Group forms, and from the UI Action Visibility Related List on the UI Action form (available in instances created after June 3, 2011).
Update Sets A UI Policy was added to make sys_m2m fields mandatory.
User interface toolbar display The HTML field toolbar is now in a DIV rather than a TABLE. This allows the toolbar buttons to wrap instead of pushing the form out to the right.
User interface toolbar configuration HTML field toolbar buttons can now be configured with the glide.ui.html.toolbar property, found in System Properties > UI Properties with the following description: Configures the editing toolbar for HTML fields...
Application and Module tables show complete contents in lists Application and Module tables' Roles and Hints fields have been given the no_truncate attribute, so they shows their complete contents in a list.
Editing UI Actions UI Actions (buttons, links, etc.) now have an Edit UI Action context menu option that is available to users with the ui_action_admin or admin role.
Reference record pop-up Hovering over the diamond icon on a Glide List field now displays a pop-up of the selected referenced record, just like a Reference field pop-up.
Form templates lookup When applying form Templates, users now select a template from a lookup list that references the Templates table. Instances with a large number of Templates were experiencing a performance issue that rendered all their templates in the form context menu. This change includes a new version of the Read ACL for sys_template and the template query Business Rule to ensure that users can see applicable templates.
Process Applications - Change Management
Bulk CI changes CI Class and Proposed Changes fields are no longer added to the Change Request form when the Bulk CI Changes plugin is activated. They can be added by personalizing the form after plugin activation.
Affected CIs Related List The Applied and Applied date fields have been removed from the Affected CIs Related List (they are only relevant when working with Proposed Changes), and CI Class has been added. Customized related lists will not get the change.
Process Applications - Incident Management
Reopening incidents ESS users now see a Reopen Incident button on resolved Incidents. (available on instances created since June 3, 2011).
Inbound email actions Incident created from Inbound Email Actions now set the Contact type field of the resultant Incident to Email.
Record producer The Create a New Incident Record Producer has been modified to set the Contact type field of the resultant Incident to Self-service.
ESS user view The Watch list, Incident state, and Impact fields have been added to the ESS view of the Incident form, and the variable formatter has been removed. ESS users have write access to the Watch list and Impact fields (available in instances created after June 3, 2011).
Process Applications - Other
IT GRC plugin The new Applies to field (type document_id) was added to the grc_control table to allow for identification of the scope. The form and list layout are modified as a result. A property was also added to control tables available to the field.
Project Management Skills Related List Installing Project Management v2 no longer adds the Skills Related List to the User form.
Using an existing Project as a template Added a new feature to Project Management to use an existing Project as a template/model to create a new base project. This UI Action from a Project list or form makes a copy of the current Project and all Project Tasks and prompts for a new Project name and start date. All tasks are reset to Pending with no actual start/end date or percent complete values.
BSM Maps Deletion of a BSM Map View now deletes any related map filters.
Knowledge Management The Attach to Incident button on Knowledge Base articles has been changed to reflect the relevant table. For example, when searching the Knowledge Base from a Change Request, the button is labeled Attach to Change Request. If the table cannot be determined due to customer modification of Knowledge Base UI Pages or UI Macros, the button is labeled Attach to record.
Change and Problem form views This release provides revised versions of Change, Change Task, Problem and Problem Task form views to make the field layout more consistent with other forms (available in instances created after June 3, 2011).
Reporting
Pivot tables If a Pivot Table's aggregate is Average, then Average is used instead of Total for the final column/row label.
KPI Reports
  • Added grid to KPI - Number of Incidents per Month by Category line chart report, and filtered it on last 12 months.
  • Changed KPI - Number of Incidents by Category report to a monthly trend chart filtered on last 12 months.
  • Changed KPI - Incidents Assigned More than Once by Category report to a monthly trend chart filtered on last 12 months.
  • Changed KPI - Average Incident Resolution Time by Category report to a pivot table.
  • Changed KPI - Number of Incidents per Problem to a list report on Problem using the new Related Incidents field, filter on active Problems with more than one related incident (Instances created after June 3, 2011).
  • Changed KPI - Incident Resolution within SLA by Category to a bar chart stacked on SLA stage. Shows both achieved and breached, in green and red.
  • Changed KPI - Average Work Effort for Resolving Incidents by Category report to a bar chart.
  • Changed KPI - Number of Problems by State and CI report to a pivot table report, excluding where CI is empty, filtered on last 12 months.
  • Changed KPI - Average Problem Resolution Time report type to List to Trend Chart, showing monthly for last 12 months.
  • Changed KPI - Remotely Resolved Incidents report from a Bar Chart to a Trend Chart filtered on last 12 months.
List reports Fields from extension tables can now be added to List reports (for example, adding Caller ID to a report on task).
Scripting
GlideRecord query Performing a GlideRecord query() against a table with a field named query could generate an error. For these tables, a new _query() method can be used instead, for example, gr._query();. A similar method, _next(), has been added to address the same issue with tables that a field named next.
GlideSystem methods Added convenient GlideSystem methods for logging Warnings and Errors: gs.logWarning(message, [source]) and gs.logError(message, [source]).
GlideForm function Added a GlideForm function to return the decimal value of a field: g_form.getDecimalValue(fieldname)
Client Scripts warnings Two Async Delete Warning client scripts now use showFieldMsg() instead of addInfoMessage() when warning against using current in an async Business Rule.
Client Scripts field A new "Messages" multi-line field was added to Client Scripts. Strings entered here, one per line, are made available to the client script as localized messages without the need for an AJAX call, using getMessage(msg) syntax. This allows for the internationalization of messages displayed in alerts, for example, without a round trip to the server. As an example, if the Message field has the line Please populate the Reason field, then the client script could have the following line: alert(getMessage(Please populate the Reason field));, and the user would see it in their language, assuming an entry exists in the sys_ui_message table for their language with that key.
