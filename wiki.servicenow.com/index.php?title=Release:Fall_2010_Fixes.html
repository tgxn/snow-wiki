  • 6 Fixes for the Spring 2010 Stable 1 release

1 Fall 2010 Fixes

1.1 PRB482101 (Currency): Multiple update of currency/price field

A multiple record update of a form with a currency or price field no longer clears the value of that field.


1.2 PRB421098 (IE6): Choice list fields in tabbed view

Choice list fields are now visible in tabbed form view for IE6.


1.3 PRB197425 (SLA): Actual elapsed percentage calculation

Actual elapsed percentage is now calculated consistently, and all calculation logic has been moved to a single Script Include.


1.4 PRB428315 (IE6): IE6 crash using Order Guides

Order Guides now function correctly for IE6.


1.5 PRB429610 (UI): JavaScript error bringing up invalid record

Bringing up an invalid record (e.g., demo.service-now.com/incident.do?sys_id=badger) no longer generates a harmless JavaScript error in the addRenderEvent() function.


1.6 PRB372323 (List v2): Hierarchical list style

The third level of a hierarchical list in List v2 renders all list elements in the proper frame in IE8.


1.7 PRB422219 (UI): Time field improperly rendered after time format property change

If am/pm is added to the time format system property, time fields are no longer improperly rendered until next login.


1.8 PRB403197 (UI): Inconsistent reference popup view

The reference popup view for forms with multiple sections is now consistent and does not show an unexpected form section.


1.9 PRB427489 (Database): New button on Map View Configuration Types could lead to inability to save tasks

A complex series of steps following clicking of the New button on Map View Configuration Types no longer can result in a user succeeding in trying to make the Incident table extend itself.


1.10 PRB428204 (SLA): contract_sla table was not audited

The contract_sla table is now audited by default for new instances. Existing instances can set the "audit" flag on the dictionary entry for the table.


1.11 PRB294229 (Workflow): User approvals from Workflow not tracked in approval history

User approvals from Graphical Workflow activities can now be tracked in the approval history by setting the glide.workflow.user_approval_history property to true. Default value is false.


1.12 PRB390955 (List v2): Numbering of new records created in embedded lists

New records created in embedded lists will now be numbered property, making an AJAX call for each to get the next number.


1.13 PRB422622 (Reports): Monday could not be set as first day of week for trending reports

Monday can now be specified as the first day of the week for trending reports by setting the value of the glide.ui.date_format.first_day_of_week property to 2.


1.14 PRB408397 (Printing): Only first page of a record would print

All pages of a record will now print in tabbed form view with the new fixed form header.


1.15 PRB442801 (Single Sign-on): glide.authenticate.failed_requirement_redirect property not working consistently

The "failed authentication redirect" URL parameter now functions as expected for external authentication requests.


1.16 PRB445001 (Templates): glide_list not set properly in Template

glide_list fields can now be set properly with a Template. Note: Templates do not currently support the capability of a glide_list referencing sys_user to include a specific email address along with the sys_user references.


1.17 PRB446872 (Templates): Templates break with @ symbol

Templates no longer break if one field value has an @ ("at") symbol.


1.18 PRB442206 (Service Catalog): Wizard catalog ordering did not enforce mandatory variables

Ordering a catalog item using a Wizard now enforces mandatory variables.


1.19 PRB442857 (Currency): Default value ignored for Currency fields

Default values will now be honored for Currency fields, e.g., javascript:current.price='USD;500000';


1.20 PRB445415 (Email): Email Log related list on sys_email record used wrong query

The Email Log "Relationship" related list, defined in System Definition -> Relationships, now uses the correct query to show log entries for the current day based on the user's timezone.


1.21 PRB420901 (Reports): Large number of groups slowed rendering of Report UI

The select list of reports when "Visible to group" is selected when creating a report has been changed to a reference lookup to improve performance on instances with a large number of groups.


1.22 PRB419751 (CMS): Changes to CMS Page record were not captured in update set

Changes to fields on a CMS Page (content_page) record are now properly captured in update sets.


1.23 PRB453670 (UI): showFieldMsg autoscrolling error

Scrolling to an input element with showFieldMsg() now works as expected in the Service Catalog and Wizards.


1.24 PRB455543 (Client Script): setValue() in an onSubmit client script

setValue(field, value); once again works as expected in onSubmit client script (introduced Spring 2010 Stable 3)


1.25 PRB417473 (UI Actions): showQuickForm() in List v2

Using showQuickForm() in a UI Action now works as expected in List v2.


1.26 PRB456006 (Printing): Printing a grouped list report gave null headers

Printing a list report with a group by column once again shows the expected column headers.


1.27 PRB442053 (List v2): Approval with E-Signature

The Approval with E-Signature plugin now works properly with List v2.


1.28 PRB438294 (Reports): Scheduled reports an hour early

Scheduled reports now send on time, and Daylight Saving Time is no longer double-accounted for under some circumstances.


1.29 PRB444426 (List v2): Grouped list expand preference

List v2 now honors the user preference that remembers whether grouped lists should be expanded when rendered.


1.30 PRB456099 (Timeline Pages): IE JavaScript error in Timeline pages

A JavaScript error "object is undefined" has been identified and eliminated in IE8.


1.31 PRB459215 (Homepages): Read roles issue with Homepages

Read access to Homepages now requires both Read access via ACLs AND read roles as specified on the homepage record itself. Introduced Spring 2010 Stable 3, where only ACLs were looked at.


1.32 PRB426337 (Service Catalog): Mandatory field on List Collector variable

Enforcement of mandatory fields is improved for List Collector variables, and the mandatory property persists through selecting, deselecting, and switching references.

1.33 PRB482958 (Web Services): Error inserting hierarchical SOAP request

Inserting hierarchical SOAP requests no longer generates an error.


1.34 PRB442686 (Homepages): Write roles wrongly allowed read access

Write roles on a homepage no longer allow read access if the read roles do not.


1.35 PRB485392 (Multiple Update): Read-only reference field hindered multiple update

A read-only reference field on the multiple update form will no longer prevent the update from being done.


1.36 PRB485694 (SLA): SLA not cancelled if pause condition met

If a task changes so that it no longer meets the start conditions, while it does meet the pause conditions, the SLA will now be properly cancelled as expected.


1.37 PRB488756 (Domain): Duplicate choice list entries

Personalizing Choices on a Domain separated instance using an Oracle database no longer causes the selected choices to be duplicated.

1.38 PRB427874 (Database): Database deadlock related to MetaData

The potential source of a database deadlock related to MetaData (specifically in com.glide.script.MetaData.createTableMetaData) has been eliminated.


2 Spring 2010 Stable 3 Fixes


2.1 PRB080092 (GlideRecord): Undefined value in GlideRecord queries

GlideRecord now handles a query on an undefined value or a nonexistent column by throwing an error instead of returning all records.


2.2 PRB427616 (List v2): Copy URL from breadcrumb

The right-click > Copy URL function for breadcrumbs in List v2 now generates the correct URL.


2.3 PRB419318 (Templates): onChange client scripts affected by templates

A new argument isTemplate has been added to onChange client scripts to indicate if the call is being made based on the template being applied. It is used as follows: 

function onChange(control, oldValue, newValue, isLoading, isTemplate) { 
   if (isTemplate)
     alert("Field change is due to template being applied, not user action");
 }


2.4 PRB373595 (Web Services): SOAP messages with special characters

SOAP messages containing special characters now use the correct UTF-8 character in the generated XML.


2.5 PRB069565 (SLA): SLA calculations of percentage

SLA business rules use a new SLA Calculator script include with improved logic for setting the percentage field.


2.6 PRB049586 (SLA): SLA - update calculations scheduled job

To improve performance, SLA - update calculations scheduled job has been split into 6 scheduled jobs.


2.7 PRB402679 (SLA): SLA calculations of elapsed time

SLA business rules and scheduled jobs use a new SLA Calculator script include with improved logic for calculating elapsed time outside of business hours as defined by Schedule Entries.


2.8 PRB308266 (Service Catalog): Checkout rendering in IE6

For users of IE6, order guide tabs no longer blink during checkout.


2.9 PRB422148 (Templates): References with derived fields

When a template contains a reference with derived fields and the reference value is set on a record with no data in the derived fields, the record is no longer updated with data from the form before the template is applied.


2.10 PRB317772 (List v2): Print button in database view

In List v2, the Print button in database view now works properly.


2.11 PRB420571 (List v2): CAB Report

The CAB Report now renders properly in List v2.


2.12 PRB360299 (List v2): Personalizing a form

You can now personalize a form for a table that is listed in a sys_m2m record for a many-to-many table that does not exist.


2.13 PRB418003 (List v2): Parent variable in List Control scripts

In List v2, the parent global variable is now correctly populated in List Control scripts.


2.14 PRB420790 (Reports): Pivot tables with long dot-walked fields

Pivot table reports now support dot-walked field names up to 255 characters long.


2.15 PRB381656 (Service Catalog): Catalog UI Policies on multiple variable sets

When using multiple variable sets, the Catalog UI Policies for each set are now properly applied.


2.16 PRB419575 (Mobile): Form views for related lists

When using a mobile client, records on related lists now open in the mobile form view.


2.17 PRB419270, PRB419555 (Filters): Personal and Group filters

Personal and Group list filters are now created properly, instead of as Global filters.


2.18 PRB381947 (i18n): Global Text Search label

The Global Text Search label now resets to the proper i18n language string when a user clicks out of the input field.


2.19 PRB197237 (UI): Related lists in the sys_popup view of a form

Since related lists are not rendered in the sys_popup view of a form, this configuration option is no longer available.


2.20 PRB415518 (UI): Reference field pop-up

Clicking a reference while the reference pop-up is visible no longer generates a JavaScript error when the reference is to the same table (the Parent field on the Group form).


2.21 PRB408566 (Reports): sys_id in list reports

The sys_id field is no longer added automatically to list reports.


2.22 PRB406859 (Reports): Grouped list with ampersand in label

In a grouped list report, a group with an ampersand in the label can now be expanded.


2.23 PRB375233 (UI): Welcome page content

Welcome page content now appears properly when the Guest user record is deleted or has a different language preference than sys_home records. English language is now used when no value is specified for a user's language preference.


2.24 PRB407678 (Discovery): Network printer with invalid MAC address

A network printer with an invalid MAC address string no longer generates an error.


2.25 PRB408612 (Discovery): Process classifiers that match Any

Discovery Process Classifiers that use Any as the match criterion no longer create multiple records.


2.26 PRB404186 (UI): Attachments exceeding maximum size

An error message now appears when users attempt to upload an attachment that exceeds the maximum file size.


2.27 PRB408519 (Workflow): Export to Excel

Workflow stages now export properly to Excel.


2.28 PRB408645 (Login): Logging remote IP address events

Remote IP address is now logged as Parm2 for logout and login.failed events.


2.29 PRB392491 (Knowledge Base): Search operator property

The Knowledge Base search operator property (Knowledge Base > Properties) is now properly applied for direct knowledge searches.


2.30 PRB294376 (UI): HTML field links in IE7

In IE7, HTML field links are now displayed in the correct style, instead of plain text.


2.31 PRB405199 (UI): Long labels in PDF exports and List v2

5.1 PRB291977: Reports

5.1.1 Example: Clicking "Execute Now" on a "Run once" Scheduled Report could result in duplicate emails being sent

Resolution: Duplicate emails are no longer sent.

5.2 PRB043344: Reports

5.2.1 Example: The "Stacked field" on a bar chart report could not be set to "-- None --" after it had a different value

Resolution: The Stacked field will now preserve the -- None -- value when the report is saved.

5.3 PRB302308: UI

5.3.1 Example: List Control did not honor domain. e.g., a List Control for the global domain was not applied to users in other domains

Resolution: List Control now works as expected with domains. A List Control for a domain will apply to subdomains.

5.4 PRB327322: UI

5.4.1 Example: In List v2, Filter Out/Show Matching did not work for currency fields with a value of zero

Resolution: Filter Out/Show Matching now work properly for currency fields with a value of zero in List v2.

5.5 PRB328826: UI

5.5.1 Example: Filters using JavaScript functions (e.g., Group = javascript:getMyGroups()) only displayed the first display value in the filter breadcrumb (e.g., "Network" instead of "Network, Desktop, Software"

Resolution: Filters using JavaScript functions now display the correct list of display values in the breadcrumb.

5.6 PRB330279: Exports

5.6.1 Example: Related lists were not supported in the PDF Detailed export of a list

Resolution: A new property has been added, "glide.export.pdf.list.related_list", defaulting to false. If true, related lists will be included in the PDF Detailed export of a list.

5.7 PRB297476: i18n

5.7.1 Example: Several items on the "Password needs reset" page were not translated

Resolution: The whole "Password needs reset" page is now translated.

5.8 PRB268944: i18n

5.8.1 Example: The "Import Set Tables" message had an incorrect French translation

Resolution: The translation was fixed.

5.9 PRB331921: UI

5.9.1 Example: Many-to-many related list labels were not translated in List v2

Resolution: Many-to-many related list labels are now translated in List v2

5.10 PRB332387: Discovery

5.10.1 Example: The DNS Domain field did not populate properly

Resolution: The DNS Domain field now populates properly. The "HostnameJS" Script Include was fixed to do this.

5.11 PRB332393: Discovery

5.11.1 Example: The glide.discovery.hostname.always_update property was no longer honored to update the hostname of a CI

Resolution: The DiscoveryIDSensor Script Include was fixed to honor the property.

5.12 PRB343213: UI

5.12.1 Example: Show Matching/Filter Out did not work properly for zero-value choice list fields (e.g., a Priority of "0 - Critical" with a value of zero)

Resolution: Show Matching/Filter Out now works properly for zero-value choice list fields

5.13 PRB345783: UI

5.13.1 Example: Duplicate roles could appear in the roles filter

Resolution: This could happen if a record with a user_roles field was deleted, then imported. The behind-the-scenes kb_knowledgeroles table that stores information about the roles selected in the user_roles field did not have its corresponding records deleted when the article was deleted. Now those records are cleared along with the parent record's deletion.

5.14 PRB346005: UI

5.14.1 Example: Domain fields were not initialized to empty on Search or Update forms, like reference fields are

Resolution: Domain fields are now initialized to empty on Search and Update forms.

5.15 PRB345919: i18n

5.15.1 Example: Survey names were not properly translated

Resolution: Survey names are now properly translated.

5.16 PRB320582: UI

5.16.1 Example: List editing was prevented for a dependent reference field if the user couldn't write to the field it was dependent on

Resolution: List editing is no longer prevented in this situation.

5.17 PRB314705: Reports

5.17.1 Example: In Oracle, a trend chart with a filter condition spanning the beginning of a year (e.g., Date between 9-1-2009 and 2-28-2010) returned data in the incorrect trend bar

Resolution: The data is now reflected in the correct trend bar.

5.18 PRB304810: i18n

5.18.1 Example: Using a language other than English, editing a related list slushbucket to remove all entries returned to an empty record

Resolution: Added a check for the translated empty message "--None--" or "-- None --", and added translations for both messages.

5.19 PRB303849: Performance

5.19.1 Example: If a query is issued that returns a large enough number of records in MySQL, an instance could run out of memory

Resolution: MySQL now runs in streaming mode to prevent this problem.

5.20 PRB316222: Update Sets

5.20.1 Example: An admin could delete the Default or BaseLine update set

Resolution: An ACL on sys_update_set was put in place to prevent this.

5.21 PRB309282: Domain

5.21.1 Example: The Domain reference picker in the title bar showed an incorrect domain at login until the page was refreshed

Resolution: The Domain reference picker now shows the correct domain on login.

5.22 PRB315853: Update Sets

5.22.1 Example: Updates to Private properties (with the Private checkbox checked on the sys_properties record) were wrongly captured in update sets

Resolution: Private properties are no longer captured in update sets.

5.23 PRB252198: Service Catalog

5.23.1 Example: The Template field showed on the Catalog Item form for all item types

Resolution: A new client script on sc_cat_item now hides the Template field if the Item is neither a Template nor Record Producer

5.24 PRB331896: UI

5.24.1 Example: A photo field in a reference popup generated a JavaScript error

Resolution: Protection was added to avoid the JavaScript error in the sys_popup view

5.25 PRB330306: Email Client

5.25.1 Example: Special characters like quotes in the Subject did not display properly in the email client

Resolution: The Email Client subject is now properly escaped.

5.26 PRB350446: UI

5.26.1 Example: In List v2, list filters did not remain after a UI action was executed

Resolution: Stack parameters are now saved on a pop, so that they can be reapplied if the next push matches the pop. This ensures the filter is saved between list displays.

5.27 PRB359625: UI

5.27.1 Example: Spell check could add duplicate letters after clicking "resume editing" a second time

Resolution: The mod array is now reset each time spelling is checked to prevent this.

5.28.1 Example: Change Phase and Release Phase "Order" field was not getting populated

Resolution: The order column in task was conflicting with the order fields in change_phase and release_phase tables. To resolve, values from the redundant columns are copied to the order column in task with a SQL statement, and the redundant columns are dropped.

5.29 PRB296323: Exports

Resolution: Emailed exports now honor the requesting user's domain.

5.34 PRB042905, PRB043948: Email Client

Resolution: The email client no longer requires the form to be saved, eliminating the cause of this problem.

5.35 PRB286235: UI

5.35.1 Example: Field messages (error, info, warning) set on a specific field might automatically scroll the form to an undesired location

Resolution:
  1. A new property overrides whether automatic message scrolling will occur at all (glide.ui.scroll_to_message_field).
  2. Scrolling occurs automatically when the field label or the message is offscreen, and scrolls the form to the field label.
  3. g_form.setFieldMsg() now takes an optional true/false parameter that controls whether scrolling occurs when the system default behavior needs to be overridden on a case-by-case basis. e.g., g_form.showFieldMsg("short_description", "Please enter", "error", false);

5.36 PRB362126: Reports

5.36.1 Example: The type-ahead filter atop the Reports portal might wrongly remove the first section's header

Resolution: The filter no longer removes the first section's header inappropriately.

5.37 PRB365329: UI

5.37.1 Example: Column header names showed as "null" when viewing any list in printer-friendly version

Resolution: Column headers are now rendered properly.

5.38 PRB365273: Metrics

5.38.1 Example: The out-of-box Metric Definition MTRC0000007 improperly queried on the type field vs. the id.type field

Resolution: The query has been changed in that metric definition to query the correct field.

5.39 PRB365443: UI

5.39.1 Example: The Day in Week field type might list days out of order

Resolution: The days are now ordered by value and not label.

Resolution: Improved IE link functionality when an empty paragraph is selected

5.41 PRB366599: UI

Resolution: Removed the association of the label in the anchor tag from the select to fix this issue

5.42 PRB369897: Reports

5.42.1 Example: Expiring Asset Contracts count gauge had incorrect query conditions that did not render properly in a filter

Resolution: The conditions have been fixed.

5.43 PRB370400: Update Sets

5.43.1 Example: A previously skipped update in an update set (skipped for legitimate reasons) would wrongly be skipped in all future update sets

Resolution: The "Commit Update Set" UI Action was fixed to prevent this problem.

5.44 PRB370265: UI

5.44.1 Example: Typing NULL into a reference field on a multiple update form did not properly clear the field for updated records

Resolution: Typing NULL into a reference field on a multiple update form now properly clears the field for updated records

5.45 PRB268926: Workflow

5.45.1 Example: An extremely large workflow might not be properly captured in an update set

Resolution: The extremely large workflow might create full sequences that are too long. Now if there are more than 5000 activities listed in the full or expected sequences, the list of sequences is not generated. The sequences are only used to optimize the Stage widget display, so giving up on the list only impacts which stages are displayed in that widget.

5.46 PRB372026: Exports

5.46.1 Example: When exporting reports to CSV, durations longer than 9,999,999 seconds display in scientific notation in Excel

Resolution: Durations now export as numeric regardless of length.

5.47 PRB373586: Service Catalog

5.47.1 Example: When a Requested Item was created via script, the variables weren't available

Resolution: Variables are once more available when creating catalog items through script.

5.48 PRB314795: UI

5.48.1 Example: Using IE6 for whatever reason, when there was a large number of input elements on a form, typing in a journal field could result both in a delay between keystrokes and in high CPU utilization

Resolution: Code has been modified so this issue no longer presents.

5.49 PRB328508: Discovery

5.49.1 Example: 32-bit software was not discovered on a 64-bit Windows OS

Resolution: Support has been added to the "Process Installed Software" discovery sensor for discovering 32-bit software running on 64-bit Windows operating systems.

5.50 PRB369778: UI

5.50.1 = Example: Editing related list contents with the slushbucket UI might not properly return a user to the form if the Tiny URL plugin was installed

Resolution: Editing related list contents with the slushbucket UI now properly returns a user to the form if the Tiny URL plugin is installed.

5.51 PRB375790: UI

5.51.1 Example: A "Go to" navigation on a partial record number might only return a single record even if multiple records matched the partial number

Resolution: An "ends with" query is done on a Number field for partial number Go to navigations.

5.52 PRB373796: Workflow

5.52.1 Example: Several workflow tables did not have out-of-box ACLs

Resolution: ACLs have been added for workflow tables lacking them.

5.53 PRB373518: Service Catalog

5.53.1 Example: Global Text Search did not honor the "no search" checkbox on a catalog item when determining result count

Resolution: Global Text Search now honors the "no search" checkbox on a catalog item when determining result count.

5.54 PRB149397: Import Sets

5.54.1 Example: If a scheduled import resulted in an error, subsequent dependent imports would still run

Resolution: A property can be created to cancel dependent scheduled imports if the parent import results in an error. Additional information can be found at this link.

5.55 PRB369299: Discovery

5.55.1 Example: While performing a networks discovery, some routers are not traversed

Resolution: Certain kinds of router configurations were not being properly analyzed to find attached networks. All router configurations should now be analyzed correctly.
