1 Changes for Winter 2011

1.1 Published Report URLs

A published report's URL will be displayed at the top of the page when the report is run. This can be turned off with the new glide.ui.report.show_published_url property. A new business rule on sys_report will show the same information at the top of the sys_report form, controlled by the same property.

1.2 Navigation Add-ons for Searches

Navigation Add-ons of type "Search" will now honor Roles and Conditions specified on the Add-on. Previously only Link type Add-ons used these fields.

1.3 Unpublish Button for Reports

A new Unpublish button appears in the report UI if the report is currently public and the user is a report publisher.

1.4 Conditions Field Filter Operators

Conditions type fields now get the same filter operators as String fields.

1.5 "is one of" Query for String Fields

A new "is one of" query operator is available for string fields in a filter, using a text area. Values can be separated by tabs or newlines (one per line). Supports cut and paste from excel (tabs if cut cells are across, newlines if cut cells are down).

1.6 content_admin Role Added for CMS

A new role, content_admin, has been added, and CMS checks for that role instead of admin.

1.7 Paging of Knowledge Search Results

Users can now page through knowledge search results.

1.8 WSDL sysparm_view Support

WSDL now supports the sysparm_view parameter.

1.9 Extend Nav Filter Shortcults

A new global UI Script, navFilterExtension, can be customized to extent the nav filter shortcuts with additional custom shorts per instance. Instructions are commented out in the UI Script.

1.10 Edit Module Context Menu

Similar to the Edit Application context menu for application headers in the navigation frame, the image to the left of each Module can be right-clicked for an easy Edit Module link.

1.11 "personalize_choices" Security Operation

A new Security Operation, "personalize_choices", can be used to allow users to personalize choices on a per-table basis. New ACL added on all tables ("star" rule) for personalize_choices operation requiring personalize_choices role, but ACLs can be added/modified.

1.12 Wizard Redirect

Added new "redirect" field to expert (Wizard) table. Specifies the URL to which a user should be taken upon completion of the expert. Previously this could only be scripted on the final Panel of the Wizard.

1.13 Service Catalog and Wizard Images

Better looking Service Catalog and Wizard images for next/previous buttons, replacing the orange buttons.

1.14 Enhancements Module Removed From SCRUM

The Enhancements module has been removed from the SCRUM application. That's what Stories are for.

1.15 Email Importance Flag

Optional importance field to set the priority/importance of a message.



2 Changes for Fall 2010 Stable 3

2.1 New Quick Filters for Date Time Fields

Show Before and Show After quick filters to easily filter on date/date-time fields

2.2 Time Worked

New property com.snc.time_worked.update_task_timer to enable updating of the task timer value based on changes to the task time worked records. This is accomplished through the Update task timer business rule.

2.3 Time Cards

  • View the list of time worked records by adding a related list to the time card form
  • New property com.snc.time_card.start_day to define the first day of time card
  • New module My Time Cards -> Current to view and manage all of your time cards for the current week

2.4 Planned Task Notification Events

All planned task type leverage a single business rule, planned task global events to fire notification events for insert, updated, commented, assigned, inactive. Includes events for field service, project v2 and release v2 tables

2.5 Project Portfolio Management

  • New Phase field added to Project and Project Portfolio tables with default choice options of Initiating, Planning, Executing, Monitoring/Controlling, Closing
  • New Project Manager field added to Project and Project Portfolio tables
  • New project report Active Project Unassigned Tasks 30 Days
  • New project report Active Projects by Manager
  • Plugin now includes the Process Flow Formatter Plugin. The default formatter can be added to the project portfolio form to display a graphical representation of the project stage
  • New related lists available on Project form to display Tasks Unassigned and Task without Assignment Groups

2.6 Cost Management

  • CI Cost test mode - By adding a property glide.cost_mgmt.test_mode with a value of true, enables actions for testing CI expense generation. Two new UI Actions TEST: Process (all dates) and TEST: Delete All Related Expenses will be available to admins on Distribution Cost and CI Rate Cards forms. They allow for bulk generation and deleting of expense lines for the cost source. These are for testing only and should not be used in a production environment.

2.7 Skills Management

The skills module is now only available for user_admin and admin roles

2.8 Plugin activation displays dependencies

When activating plugins the list of dependent plugins and their install status are displayed.

2.9 Runbook Automation VMware Support

Runbook Automation VMware Support now provisions VMware requests using Workflows instead of Execution Plans.

2.10 First Day of Week in Date/Time Picker

A new property, glide.ui.date_picker.first_day_of_week, can be created to specify the first (leftmost) day of the week for the date and date/time picker (1=Sunday, 2=Monday...). The example shown uses a value of 3 (Tuesday).

2.11 XML Output Sorts Fields

Fields in XML output for serialized GlideRecords (e.g., Payload in the sys_update_xml table) are now sorted alphabetically to make it easier to find fields when viewing the XML.

2.12 Field Style Preview in Field Styles List

When viewing the Field Styles list in System UI, the Style field is automatically styled with itself for an easy preview of the style.

2.13 Workflow Field Type Behavior in Lists

Workflow fields in a list (e.g., kb_knowledge.workflow) now behave differently. The expander next to a single item now expands/contracts just for that row instead of for all subsequent rows. A new list context menu option provides the ability to expand/collapse all.

2.14 Application Secondary Sort on Title

Two applications with the same order now subsorted by Title. Previously only unordered applications were subsorted by Title.

2.15 Disable Choice List Options (removed in Winter 2011 release due to browser incompatibility)

Two new GlideForm methods, disableOption(field, value) and enableOption(field, value), similar to addOption and removeOption, allow disabling/enabling of choice list options. Example: g_form.disableOption("category", "software")

2.16 Dashboard Gauge Change Link Role Change

On legacy dashboards, the report_admin role (vs. admin) is required for the "change" link of a gauge.

2.17 Clicking Text Selects Checkboxes and Radio Options

To select checkboxes and radio buttons in Service Catalog, it is now possible to click on the question text to make the selection. This functionality does not apply to field labels.

2.18 Approval Summarizer for Requested Item Table

An approval summarizer UI Macro has been created for the sc_req_item (Requested Item) table. If an instance already has a summarizer with the same name, the out-of-box one is renamed so as not to conflict.

2.19 Knowledge Base Performance Improvements

Performance has been improved for Knowledge Bases with a large number (thousands) of articles and large search result sets.

2.20 TableUtils.isValidField()

A new method, isValidField(fieldName), has been added to the TableUtils Script Include. It returns true if the fieldName parameter is a valid field in the table.

2.21 sys_ui_macro.name Field Length Increased to 100

The sys_ui_macro.name field length has been increased from 40 to 100.

2.22 Service Portfolio Display Templates Moved to Database

The hardcoded XML templates for Service Portfolio's rendering of Business Services have been converted to UI Macros in the database, where they can be customized if necessary.

2.23 Viewing Record History

The History form context menu option has been enhanced to provide two viewing modes. History -> Calendar provides the standard history view. History -> List navigates to the new History Set for the record.

2.24 Duration Field List Display

A new max_unit Dictionary attribute for glide_duration fields is available, specifying the largest time unit used when displaying the duration in a list. e.g., if max_unit=minutes, a duration of 3 hours 5 minutes 15 seconds will display as 185 minutes 15 seconds. If max_unit=hours, a duration of 4 days 2 hours 10 minutes will display as 98 hours 10 minutes. This attribute does not affect the display of duration fields in the form.

2.25 Service Portfolio Availability Gauge Enhancement

For Service Portfolio availability gauges, the "Did not meet commitment" tooltip for a success/failure indicator has been augmented with the amount of downtime experienced. e.g., "Did not meet commitment (4 hours 13 minutes downtime)"

2.26 GlideElement Method Name Typo Corrected

The GlideElement method name getDynamiCreationcScript() had been corrected to getDymanicCreationScript()

2.27 Knowledge Base Search Results Information

In Knowledge Base search results, published and modified dates are now displayed for each article.

2.28 Additional Knowledge Base Search Result Sorting Option

"Last Modified" (sys_updated_on) has been added as a third option for sorting Knowledge Base search results. Users can select their sorting preference in the upper right corner of the search results page.

2.29 Windows 7 Phone Uses Mobile UI

Windows Phone 7 devices will get the mobile UI.

2.30 List v2 UI Action Condition Checks

List v2 now performs the same condition checks for UI Actions that are performed in List V1. If a UI Action has a name that starts with sysverb_, then the action can be run without selecting any records in the list - If the UI Action has a condition that contains 'current', the condition is not evaluated until after the action is clicked (by making an AJAX call to the server) - If the UI Action does not contain 'current' in the condition, the condition is evaluated on the server when building the list, so this controls the visibility of the action.

2.31 task_rel_type Record sys_id Changed from 1 to T1

The task_rel_type record with a sys_id of 1 has had its sys_id changed to T1 to eliminate potential conflict with a cmdb_rel_type record with the same sys_id. All references to the old sys_id have been changed to reference the new sys_id with a fix script run during upgrade.

2.32 Send Multiple Reports in a Scheduled Report Email

New ability to send multiple reports in the same Scheduled Report email. Added "Include with" reference field to Scheduled Report form to specify another Scheduled Report. When the referenced Scheduled Report is executed, the current report is included in the email. "Omit if No Records" checkbox is honored per report. Conditional and Condition Script fields are only evaluated for the report being scheduled, not for any included reports. Also added "On Demand" run type so a Scheduled Report can be set to not send on its own, but still be included in another Scheduled Report's email. Added "Included in Email" related list to Scheduled Report form

2.33 Survey Response Answer field length increase

The length of the survey_response.answer field has been increased from 40 to 4000 to match the survey_response.response field. This is to prevent truncation of data when copying from the Response to the Answer field. As a result, the field can no longer be grouped by in a list.



3 Changes for Fall 2010 Stable 2

3.1 Personalizing Lists

  • If a user personalizes a list, an indicator () appears in the upper left corner (requires List v2 plugin).
  • If a list administrator has personalized a list and then changes the default columns for the list view (right-click the header and select Personalize > List Layout), a message appears above the slushbucket:
    You have personalized this list with the gear: () You will not see changes made here until your personal list is reset to column defaults.

3.2 Personalizing Forms

A new user preference (owned_by_indicator.form) allows administrators to configure an indicator () that appears on the header of forms with customer updates. Clicking the customer updates indicator opens the update set records for the item.

3.3 New Content Management Navigation Menu Block and UI

Two changes have been made to Navigation Menu Blocks:

  • When creating a new Navigation Menu block, a new graphical interface has been included to preview the Navigation Menu blocks.
  • A new Navigation Menu called Tab Menu for Headers has been created to integrate the existing Tab Menu block into a header block.

For more information, see Creating a Navigation Menu Block.

3.4 Time Card Role

A new role called timecard_admin enables users to approve, modify, and delete the timecards of other users.

3.5 Performance improvements on Knowledge portal for large topics

Performance has been improved in the Knowledge portal for very large topics of which very few articles are viewable. The roles part of the query was pushed to the database to prefilter what a user can see. An index was added to kb_knowledge.topic. Optimizations were made to attachment queries.

3.6 Wrapping tabs

Form Section and Related List tabs now wrap to a new row rather than scroll horizontally when there are too many tabs to render in one row onscreen.

3.7 New Monthly Schedule Options

Two new options have been added when a Schedule Entry is set to repeat Monthly:

  • Last Day of Month (runs on the last day of each month, e.g., 31 March)
  • Last Week Day of Month (runs on the last week day of each month, e.g., if 31 March is a Sunday, runs on 29 March)

3.8 CSS Theme Support Plugin Published

The CSS Theme Support Plugin, available for some time, no longer requires Technical Support to install it. An admin can now install this plugin.

3.9 Scheduled Chart Reports can be sent in PDF with Grid Support

Scheduled Bar Chart reports could previously be sent only as .png images, with no grid support. Bar Chart reports, when scheduled to be emailed, now have two additional Output Type options: PDF and PDF-landscape. These both will include the chart's grid data if "Include grid" is checked on the report.

3.10 ArrayUtil Script Include additions

Four methods have been added to the ArrayUtil Script Include: diff, intersect, union, and unique. They can be used as follows to perform common tasks with Arrays, and each returns an Array: 

var a = ["a", "b"];
var b = ["a", "c"];
var c = ["a", "b", "a"];
gs.print(new ArrayUtil().diff(a, b)); // prints b
gs.print(new ArrayUtil().intersect(a, b)); // prints a
gs.print(new ArrayUtil().union(a, b)); // prints b,a,c
gs.print(new ArrayUtil().unique(c)); // prints b,a

3.11 Attachments Ordered by Creation Date

File attachments to a record are now displayed in the order in which they were added. Previously they were displayed alphabetically, making it difficult to easily determine which was the most recently added attachment.




4 Changes for Fall 2010 Stable 1

4.1 Performance Enhancement - Global Search

Global search performance across large numbers of records has been improved by decreasing the number of lucene queries. This will be noticeable to any users who search globally with search terms which are not task numbers.

4.2 Performance Enhancement - Database Limited Activities

The database's memory efficiency has been improved. This affects the UI minimally, but will significantly improve throughput on database limited activities (e.g. Discovery processes).

4.3 Performance Enhancement - SOAP Web Services

The cost of creating a controller has been reduced. This does not affect the UI, but will increase performance in high volume SOAP environments.

4.4 Discovery Credentials Order

Discovery Credentials can be assigned an order value in the Credentials Form, which forces Discovery to try all the available credentials in a configured sequence.

4.5 sys_report Roles and visibility are enforced wherever a report is viewed

Before this release, users might be allowed to see reports where they did not have the proper Roles as specified on the sys_report record if that report was placed on their homepage. This is no longer allowed. Users must have the required roles to view a report. Visibility is also enforced. A report must either be Global, visible to that user, or visible to one of the user's groups in order for a user to view it in either the Reports portal or a homepage.

4.6 Conditional and Condition fields added to Scheduled Jobs

A new Boolean field, Conditional, has been added to Scheduled Jobs. If checked, a new Condition script field can be used to script the condition under which the job will run. If the last line of the script evaluates to true, the job executes. If the last line of the script evaluates to false, the job will not execute.

4.7 "Omit if no records" field added to Scheduled Reports

A new Boolean field, "Omit if no records", has been added to Scheduled Reports. If checked, and if the Conditional field is also checked, the referenced Report's filter will be evaluated to see if any records are returned. If none are, the report will not be emailed.

4.8 onChange Client Script Error Messages

onChange Client Scripts that generate a JavaScript error now display a field message with specific information about the bad script if the user viewing the form has the client_script_admin role, and a generic error message otherwise. Before, JavaScript errors caused by Client Scripts could cause subsequent scripts to not execute on a form. Now the errors are safely caught, allowing other scripts to still run.
