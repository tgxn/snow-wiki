1 Overview

Activating the Facilities Service Management plugin adds or modifies several tables, user roles, and other components.

Demo data is available with Facilities Service Management.

2 Tables

The Facilities application adds the following tables.

Table Description
Facilities Request [facilities_request] Stores all requests for changes to facilities.
Facilities Request Flow [sf_facilities_request] Manages the state of requests from Draft through completion.
Facilities Request Model [cmdb_facreq_product_model] Stores all request models used to auto-populate fields for a request.
Facilities Request Task [facilities_request_task] Stores all request tasks for changes to facilities (starting with the Fuji release).
Facilities Request Task Flow [sf_facilities_request_task] Manages the state of request tasks from Draft through completion.
Facilities Request Task Model [cmdb_factask_product_model] Stores all request task models used to auto-populate fields for a request task.
Floor [fpv_floorplan] Stores all floor records for each building.
Room [fpv_element] Stores all room records for each floor.

3 Plugins

The Facilities application uses the following plugins:

Plugin Description
Facilities Service Management [com.snc.facilities.service_automation] Activates the Facilities application.
Facilities Service Management CMS Portal [com.snc.facilities_service_automation.cms] (Optional) Displays the Facilities Service Management SM application on the Service Management portal. Activation of this plugin on production instances may require a separate license. Contact ServiceNow for details.
Facilities Service Management Floor Plan Viewer [com.snc.facilities_service_automation.fpv] Activates the floor plan viewer for the Facilities application.
Service Management Core [com.snc.enterprise_service_management] Activates SM applications, such as field service management and facilities service management

4 Properties

The Facilities Service Management application uses the following properties:

Name Description
Properties for Facilities
facilities.management.default.start.time Default start time for all agents when no schedule is set. This value sets the start time for a day other than the current day, when no scheduled task exists or is continued from the previous day. This property uses a 24 hour clock.
  • Type: String
  • Default value: 08:00
  • Location: Facilities > Administration > Properties
  • Learn more: Default Start and End Times
facilities.management.default.end.time Default end time for all agents when no schedule is set, formatted in a 24 hour clock.
  • Type: String
  • Default value: 17:00
  • Location: Facilities > Administration > Properties
  • Learn more: Default Start and End Times
Dispatch Properties for Facilities
facilities.management.max.agents.processed Sets the maximum number of agents that will be processed by auto-dispatch at a time. The system has an absolute limit of 300 agents and sets the value at that level if you specify more than 300. The system cannot auto-dispatch a task for a dispatch group that contains more agents than the value configured.
  • Type: Integer
  • Default value: 100
  • Location: Facilities > Administration > Properties
  • Learn more: Processing Count
facilities.management.work.spacing Amount of time (in minutes) to add between the end of a task and the travel start of the next. An example of a valid time value is 10.
  • Type: Integer
  • Default value: 0
  • Location: Facilities > Administration > Properties
  • Learn more: Work Spacing

5 User Roles

The Facilities application uses the following roles.

Role Description Contains Roles
facilities_admin Facilities administrators create and modify all facilities requests, modify floor plans, and configure buildings, floors, and rooms. Administrators can also create tasks using the Clone Task feature.
  • catalog_admin
  • facilities_approver_user
  • facilities_dispatcher
  • facilities_staff
  • fpv_element_writer
  • fpv_floorplan_writer
  • knowledge_manager
  • skill_admin
  • skill_model_admin
  • template_admin
  • territory_admin
facilities_approver_user Can approve whether a facilities request can move forward.
  • approver_user
facilities_dispatcher Schedules and assigns the tasks to facilities staff. They can be searched (filtered by) the group they manage.
  • facilities_staff
  • inventory_user
  • skill_model_user
  • territory_user
facilities_read Role for members of the Facilities team who can read requests. none
facilities_staff Provides full access to the Facilities application and all modules. Provides the ability to create and modify facilities requests and access facilities reports. Facilities staff are typically the users who are assigned to facilities requests and update the request record accordingly.
  • document_management_user
  • facilities_read
  • fc_request_reader
  • fc_request_writer
  • fpv_element_reader
  • fpv_floorplan_reader
  • inventory_user
  • skill_user
  • territory_user
Do not assign the following roles to groups or users. These roles are automatically inherited by the Facilities staff and Facilities admin roles.
fc_request_reader Grants read access to facilities request records. none
fc_request_writer Grants create, read, write, and delete access to facilities request records.
  • fc_request_reader
fpv_floorplan_reader Grants read access to floor plans.
  • fc_request_reader
fpv_floorplan_writer Grants create, read, write, and delete access to facilities floor plans.
  • fpv_floorplan_readert
fpv_element_reader Grants read access to room records. none
fpv_element_writer Grants create, read, write, and delete access to room records. none

5.1 User Roles for the Facilities Service Management CMS Portal

The Facilities Service Management CMS Portal adds the following user roles, starting with the Fuji release.

Note: To use the Facilities Service Management CMS portal, you must first activate the Facilities Service Management CMS Portal plugin.


Role Description
fc_request_reader Provides access to the following link in the Facilities Service Management CMS Portal:
  • Knowledge
fc_request_writer Provides access to the following links in the Facilities Service Management CMS Portal:
  • Knowledge
  • Catalog
  • Status
fpv_element_writer Provides access to the following link in the Facilities Service Management CMS Portal:
  • Status

6 Email Notifications

The Facilities application uses the following email notification.

Notification Description
Facilities Request is assigned Sends an email message to the facilities staff member who is assigned to the facilities request.

The Facilities application uses the following catalogs.

Catalog Name Description
