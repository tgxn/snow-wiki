Spring 2010 Stable 2 Notable Changes

From ServiceNow Wiki
Home > Release Notes > Legacy Versions > Spring 2010 Stable 2 Notable Changes
Jump to: navigation, search

The following are notable changes you may encounter with our Spring 2010 Stable 2 release. Not necessarily problems, not necessarily major features, just things or behaviors you may notice being different than before. For additional changes, see Spring 2010 Stable 2 Fixes.

1 Changes for Spring 2010 Stable 2

1.1 Using ACLs to Control Workflow Publishing

Access Control Rules (ACL's) can now restrict a user's rights to check out and publish workflows. This capability provides for users who can edit some workflows and not others (e.g., users who can edit workflows within their own department, but not within other departments).

1.2 Mutual Authentication for Outbound SOAP Requests

Web Services now supports mutual authentication for outbound SOAP requests by extending protocol support.

1.3 Variable Support for JSON and SOAP get/getRecords

Web Services now returns all the variables associated with a record for JSON and SOAP get or getRecords functions.

1.4 Fixed Queries in List v2 Breadcrumbs

A new breadcrumb option for modules enables an administrator to control the base view of a record list and prevent users from executing queries backwards beyond a configured point.

1.5 New Related Lists in CI records

Records for manageable configuration items, such as computers and network gear, now contain three new Related Lists that display the IP addresses, network adapters, and DNS names associated with that CI.

1.6 Data URLs for Report Images

Instances now use data URLs for report images, reducing the amount of transactions between the client and the server when rendering reports.

1.7 Cannot impersonate inactive or locked out users

Inactive and locked out users are no longer candidates for impersonation.


1.8 Email Client cannot send to inactive or locked users

Inactive and locked out users are no longer included in the email client address autocomplete.


1.9 Email Client no longer saves the record

Launching the Email Client no longer saves the parent record in the background. Values from the record are now populated as needed into the email without saving. Additionally, the email client icon is no longer available on new records (records that have not yet been inserted). An admin can configure an instance to include the icon on new records by creating a new true/false property: "glide.ui.email_client.pop_on_new_records" and setting its value to true.


1.10 Workflow Version security

Checking out and publishing workflows now requires Create access to the wf_workflow_version table instead of Write access.


1.11 Discovery cancelled if range too large

A discovery will be cancelled if its range is too large, as defined by the "glide.discovery.max_range_size" property. Default value is 100,000.


1.12 Enhanced CMDB modeling of IP addresses and DNS names, and the new model is used by Discovery

The CMDB now associates any number of IP addresses with CIs, any number of DNS names with IP addresses, and can represent either IPv4 or IPv6 addresses. Discovery now discovers and records all IP addresses and DNS names for each CI.

1.13 New database index on question_answer table

A new index has been added to the question_answer table, to match the query most often done to retrieve records.


1.14 Reports can be stacked by Boolean fields

Boolean fields are now available as options when selecting a Stack column for reports.


1.15 Basic Auth can now log "Last login time"

By default, Basic Auth does not log the user's last login time. This can now be accomplished by setting the glide.basicauth.update_last_login_time property to true.

1.16 Table Rotation enabled for new instances

The Table Rotation plugin is automatically enabled for newly created instances.
