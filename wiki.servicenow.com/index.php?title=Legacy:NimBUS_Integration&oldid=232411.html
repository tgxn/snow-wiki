1 Overview

Bi-directional ServiceNow Incident integration with NimSoft's NimBUS alerts through the use of ECC Queue mapped. For each alarm that is assigned to the “servicenow” NimBUS user;

  1. A check is made to see if a ServiceNow incident has already been created.
  2. If not, an incident is raised/created in the defined ServiceNow instance.
  3. A note is added to the NimBUS alarm with the Incident ID.
  4. A level 2 alert is created indicating that an incident has been created for the alarm ID.
  5. A record of the Incident ID and Alarm ID is created in memory and a persistent store is updated.

For each Incident ID > Alarm ID record in memory;

  1. A check is made with ServiceNow instance for the current status
  2. If the status is “Closed”, then the associated NimBUS alarm (alarm ID) is closed
  3. A level 2 alert is created indicating that the <alarm ID> has been closed
  4. The Incident ID > Alarm ID record in memory is deleted and the persistent store updated.
 Note: The following integration is not part of the supported platform. The integration may require additional customization, typically performed by an experienced administrator or consultant, prior to operating correctly.


2 Requirements

  • The Integration - Nimsoft plugin (com.snc.integration.nimsoft) can be installed by making a request to ServiceNow Technical Support.
  • NimSoft NimBUS 3.50
  • NimBUS ServiceNow Gateway Probe (please refer to the NimBUS documentation for installation requirements)
  • Create a NimBUS user in ServiceNow
    • username: nimbus
    • password: nimbus
    • role: admin
  • Enable Basic Authorization for SOAP requests
    • System Properties > Web Service check the Require basic authorization for incoming SOAP request" checkbox

3 Activating the Plugin

Click the plus to expand instructions for requesting a plugin.
  1. Navigate to [HI].
  2. Click Service Catalog.
  3. Click Request Plugin Activation.
    • [Required] In Target Instance, select the instance on which to activate the plugin.
    • [Required] In Plugin Name, enter the name of the plugin to activate.
    • [Optional] In Date and time would you like the plugin to be enabled?, specify a date and time at least 12 hours in the future. Leave this field empty if you want the plugin activated as soon as possible.
    Note: Plugins are generally activated during business hours in the Pacific time zone, but can be scheduled for a different time with advance notice.
    • [Optional] In Reason/Comments, add any information that would be helpful for the ServiceNow technical support engineer activating the plugin.
  4. Click Submit.

4 Administration

4.1 Application / Modules

4.2 Maps

NimBUS Notification - Create Incidents maps the ECC Queue XML document into the target Incident.

4.3 Business Rules

The NimBUS Notification - Process Queue business rule on the ECC Queue table listens for incoming NimBUS XML and processes it using the map to produce a new Incident or update an existing one. Incidents are coalesced using the correlation_id field value.

4.4 Simulate NimBUS POST

Creates an inbound ECC Queue record that contains a simulated XML payload from NimBUS. Selecting this module will result in the creation of an Incident
