Winter 2009 Stable 1 Notable Changes

1 Format Phone Number Property

A property (glide.ui.format_phone) has been added which can be used to control the automatic formatting of 10-digit phone number fields in the U.S format of (999) 999-9999, for numbers that do not begin with "+".

If this property is ticked, include the "+" prefix to stop automatically formatting the phone number.

If the "+" prefix is not included, or the property is not ticked, the number is automatically formatted

To change the property, navigate to System Properties > UI Properties and look for the property with the description: Format the phone number fields if they do not begin with "+".

2 Security on a field is respected in the History

If a user does not have read-access to a field, that field's changes will be omitted when they view History of a record.

3 Customize view used for Global Text Search

In System Properties --> Global Text Search, there is a property to allow an admin to specify a different list/form view to be used for Global Text Search results. The default view is text_search.

4 Dependent Choice Fields Supported on Mobile Devices

Choice fields that depend on the value of another field are now supported on BlackBerry devices running 4.6 or better (such as the Bold and Storm) as well as iPhone.

5 Auto Pareto chart detection (focus field) has been removed

The capability caused confused consternation out of proportion to its usefulness.

6 g_form.setValue() now supports glide_list fields

The g_form.setValue() client script method now supports glide_list fields. See Set a field value in the Client Scripts article for additional detail.

7 Import Set Performance Improvements

The logic behind the transformation phase of an import set load has been revised for improved performance. In our test cases here, overall transformation time dropped by about 2/3 with the new logic. Individual customer results may vary, with the largest improvements being seen in transformations with extremely large numbers of reference or choice type fields.

8 Some client scripts converted to UI Policy

Many client scripts in the application were created before the advent of UI Policies. Client script on the following tables have been removed and UI Policies created in their stead:

  • expert_panel
  • sysauto
  • scheduled_import_set
  • scheduled_data_import
  • cmdb_ci_vpn

9 iPhone Multi-Line Text Inputs Expand When Edited

Multi-line text input fields expand to the size of their contents while being edited via iPhone to make the editing process easier.  When the "done" button is pressed to finish editing the field, it will return to the prior size.

10 Elimination of duplicate storage of journal fields

Prior to Winter 2009 Stable 1, the textual values of journal fields were stored twice. Once as rows in the sys_journal table, and additionally as a large text blob on the task table. The redundant copy on the task table has been removed.

11 SOAP changes with regards to journal fields

Prior to Winter 2009 Stable 1, a SOAP request against a table with journal fields (for example incident) would append all the available journal fields together (without formatting) and include that blob of text in the response.

Winter 2009 Stable 1 no longer includes journal fields in SOAP responses. To get the contents of a journal field, a second soap request should be made against the sys_journal table to pull the appropriate journal records back for the record in question.

12 Text index load reduction

Prior to Winter 2009 Stable 1, any change to a text-indexed table generated a text_index event that had to be processed to update the index for that record. This happened even if nothing changed that actually was part of the text index (like a Boolean or a date/time field).

Winter 2009 Stable 1 no longer creates the text_index event if nothing changed that the text index cares about, significantly reducing the text indexing workload for highly active tables.
