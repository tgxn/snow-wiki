1 Overview

A unique index on a table guarantees that the index key contains no duplicate values and therefore ensures that every row in the table is in some way unique. Specifying a unique index makes sense only when uniqueness is a essential for the data in a table.

A unique index on a field (or set of fields) prevents two records from existing in the table with the same value in the indexed field(s). ServiceNow uses several unique indexes by default to protect certain tables from duplicate record confusion.

2 Adding an Index

As an administrator, you can add an index starting with the Fuji release. For previous releases, you must contact Technical Support to request an index.

Note: You cannot add indexes to extended tables. Add indexes only to tables that are not extended, or to unique fields on child tables.


3 Unique Indexes

The following is a list of tables with a unique index, and the fields which are uniquely indexed (in the format TABLE - FIELD, FIELD):

  • cmdb_categories - name
  • content_block - name
  • expert - name
  • matcher - name
  • syntax_editor_macro - name
  • sys_app_category - name
  • sys_cluster_control - name
  • sys_dictionary - name, element
  • sys_lucene_block - file, block
  • sys_lucene_file - directory, name
  • sys_mutex - name
  • sys_properties - name
  • sys_script_validator - internal_type
  • sys_ui_macro - media_type, name
  • sys_ui_message - key, language
  • sys_ui_theme - name
  • sys_user_group - name
  • ts_group - name

For example, sys_dictionary - name, element means that no two sys_dictionary records can have the same combination of name and element values.
