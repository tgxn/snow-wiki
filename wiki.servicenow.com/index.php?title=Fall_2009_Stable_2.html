The Fall 2009 Stable 2 build was completed November 20, 2009, and includes the following enhancements and new functionality in addition to everything in Fall 2009 Stable 1. All content in a Stable build is included in the subsequent Release.

2 Description of Functionality

Our Fall 2009 Stable 2 build contains the following new or improved functionality:

  • SQL Server integrated authentication for Windows: You can now configure a JDBC data source to use the ID of the user configured for the Windows MID Server service for SQL Server authentication. The MID Server service on Windows must be installed with the same credentials that your SQL Server requires. A check box, called Use integrated authentication, was added to the Data Sources form (System Import Sets > Data Sources) to enable this feature.
  • Automatic record number re-padding: The record number padding configuration, in System Definition > Number Maintenance, now affects both existing and new records in a table. In versions prior to Fall 2009 Stable 2, changes to the number padding in a table appeared in new records only.
  • Script syntax error checking by line number: You can now search records for script syntax errors by line number. This is useful when ServiceNow logs a syntax error by line and column number. You can navigate to the record and enter the line number from the log entry. ServiceNow places the cursor at the site of the error.
  • Filter performance enhancement: The Fall 2009 Stable 2 release introduces significant performance improvements in the way UI filters are built. In prior releases, when a user activated a ServiceNow filter (the "+" icon at the top of a record list), multiple round trips to the Web server were required to build the entire filter widget. This caused noticeable delays, especially on high latency connections. The Fall 2009 Stable 2 release has reduced the number of round trips involved by a factor of about 4, leading to an overall improvement in filter performance. These improvements will be most noticeable to users with poor Internet connectivity, especially those users connecting to application servers from overseas networks.
  • Survey repeat interval override: You can now override the global survey property setting that defines how often a user can receive a survey of the same type. Configure surveys with different repeat intervals in the Survey Conditions form that installs with the Best Practice - Task Survey Management Plugin.
  • Display field messages on a form: You can now display an error on the form itself, directly below the field in which the error occurred. Use the methods showFieldMsg() and hideFieldMsg() to display or remove a field-level message.
  • Case sensitive coalesce field values: A new field called Coalesce case sensitive on the Field Map form (System Import Sets > Transform Maps) enables you to compare field values in import sets with the same values in existing records by case and create new records when they do not match.
