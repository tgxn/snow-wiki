1 Overview

Administrators should set up a facilities administrator group and one or more facilities staff groups and assign the appropriate roles and users to those groups. Then the facilities administrator can add every building and floor in your organization to the Facilities application. Each floor record can have an attached floor plan, which is a graphic that outlines the rooms on the floor. Facilities administrators can use the floor plan to create a record for each room so that facilities requests can be associated with specific rooms.

2 Creating Facilities Groups

Users with the admin or user_admin role can set up your groups and assign the necessary roles and users. The users in the group inherit the roles of the group, so you do not have to assign roles to each user separately.

A good practice is to create as many facilities staff groups as needed in your organization. For example, you can create a staff group for each geographic location or for each function, such as building maintenance or building security, assign the necessary users to those groups, and then assign the Facilities staff role to those groups. It is also a good practice to create one group for facilities administrators and assign the facilities admin role to this group only.

To create a facilities group:

  1. Navigate to User Administration > Groups.
  2. Click New.
  3. Fill in the fields. See Creating Groups for an explanation of each field.
  4. Click the lock icon beside the Type field. If the field is not visible, configure the form to add it.
    The Type field expands.
  5. Click the reference lookup icon (), and select the facilities type.
  6. Right-click the form header and select Save.
  7. Add the facilities_admin or facilities_staff role to the Roles related list.
  8. Add users to the Group Members related list.
  9. Click Update.

3 Set Up the Facilities Application

Users in the facilities administration group can set up the Facilities application by adding all the buildings, floors, and rooms in the organization.

3.1 Adding Buildings

  1. Navigate to Facilities > Administration > Buildings.
  2. Click New.
  3. Fill in the fields, as appropriate (see table).
  4. Click Submit.
The Building form.

Field Description
Name Provide a descriptive name for this building.
Location Select the location for this building. You must first define the locations in Organization Management. A good practice is to select a location that is defined at the address, not at the floor level. Floors are defined separately in the Facilities application.
Floors Enter the number of floors the building has.
Contact Select a contact person from the User [sys_user] table.
Notes Provide any additional notes to describe this location.

3.2 Adding Floors

  1. Navigate to Facilities > Administration > Floors.
  2. Click New.
  3. Fill in the fields, as appropriate (see table).
  4. Click Submit.
The Floor form.

Field Description
Name Enter a descriptive name for the floor.
Building Select the building that the floor is in.
Level Enter a character or a number that represents the level the floor is on. For example, enter G for garage or 3 for the third floor. This is a text field that takes any value.
Floor Plan Attach an image file of the floor plan. Supported file types are: .jpg, .png, .bmp, .gif, .jpeg, .ico.

3.3 Managing Floor Plans

Floor plans can be any type of line drawing, such as a CAD drawing or blueprint, that outlines areas on each floor like meeting rooms and employee cubicles. To use floor plans with the Facilities application, save the floor plan as one of the supported image file types, and add it to the floor record. If you have any trouble viewing the floor plan on the screen, you might need to resize the image or capture the floor plan in a different resolution.

Note: Creating incidents from the floor plan requires that the floor plan image fit within the browser window. The X, Y coordinates cannot be properly calculated if the scroll bars are visible. Press control + or control - to resize the map.


Facilities administrators can add room markers on the floor plan with a descriptive label, such as Executive Meeting Room, Lunch Room, and so on. These tagged locations do not actually have to be rooms. They can be any type of space or area that you want to identify, such as a stairwell or receptionist desk. After all the important rooms and areas on the plan are tagged, users can attach facilities requests to these tagged locations. This allows your facilities team to know exactly where they need to address a request.

3.3.1 Adding Rooms to a Floor Plan

  1. Navigate to Facilities > View Floor Plans.
  2. Select the correct building and floor from the choice lists to display the floor map you want to edit.
  3. To display the label of every location currently defined on the map, select the Show labels option.
  4. To search for a specific location, enter the identifying label in the Search box.
  5. Drag the room marker icon () to a point on the map.
    The Create New Facilities Room form appears.
  6. Fill in the fields, as appropriate (see table).
  7. Click Submit.
    The Facilities application creates a record in the Room [fpv_element] table.
The Create New Facilities Room form

Field Description
Name A descriptive name for the point on the map.
Building The building that the room record is associated with. Select another building if necessary.
Floor The floor that the room record is associated with. Select another floor if necessary.
Type The location type: Conference Room, Cubicle, Office.
Location The location where the building, floor, and room are located. Choose the appropriate location if it is already defined for your organization or create a location record.

Facilities administrators can edit or delete the room record by navigating to Facilities > Administration > Rooms and finding the appropriate record.
