Normalizing and Transforming Field Values

1 Overview

Records for such things as devices and companies are brought into the ServiceNow platform by manual entry, imports, and Discovery. Depending on how records are introduced into the database, field values that mean the same thing might appear in different forms or have a value that is not meaningful to an organization. Field normalization includes two features that are available for every field in the ServiceNow platform and are used to normalize or standardize specific values in the database.

  • Normalization forces the platform to convert different forms of the same field value to a single, accepted value automatically. By forcing a field to use a simple, recognizable description for multiple variations of the same thing, normalization can eliminate duplicate records and make searches easier. In addition to reconciling different forms of the same value in fields, normalization can be configured to adjust queries automatically to return normalized results.
  • Transformation enables an administrator to transform raw field input into standardized values that are more meaningful to an organization. An example of a standardized value might be to round RAM size in configuration items to a whole number, such as 4 GB instead of 3.92 GB. Transformations can be configured to adjust values in queries.

2 Administering Field Normalization

Field normalization functionality requires the Field Normalization plugin. An administrator must contact ServiceNow Technical Support to request activation of this plugin.

3 Normalization

When new records containing normalized fields are entered into the database, the platform automatically replaces any values for which a normal value is configured. An example of a company name that might be normalized is that of the computer manufacturer Dell. This name might appear in different records as Dell, Dell, Inc., Dell Computers, and Dell Incorporated. The administrator selects Dell Computers as the normal value and applies the change to all the records in the database.

4 Transformation

When new records containing transformed fields are entered into the database, the platform automatically converts any value for which a standardized value has been configured. Field transformations can be applied to fields of all types. Examples of this might be a name format that removes designations such as Jr. and II from names or a mode that rounds the amount of computer RAM to a useful interval (4112 MB to 4000 MB). Other general uses for transformations are to round integer fields, delete characters from strings, or add prefixes.

5 Testing

Field normalizations and transformations are created and tested by users with specific roles. During this testing phase, only new records opened by users with the designated testing role or records updated by those users are normalized or transformed. Records created or updated by all other users, including existing records in the database, are not normalized or transformed until testing is completed.

6 Raw Fields

An administrator might add a custom field to a form to show the original, or raw, value of a normalized or transformed field. This is a read-only field that might be called something like Raw CPU type or Original Name. 7 Scripting and Normalization/Transformation

Scripts that update records or insert records into the database (GlideRecord) are also normalized or transformed. For example, if a script to insert a CI record contains a CPU type of Xeon L3350, the script can be normalized to insert the CI with a CPU type of Xeon instead. Scripts that query the database for normalized field values return the normal or transformed value rather than the original (raw) value.

8 Normalized Queries

An administrator can configure normalization and transformation to apply to queries issued against normalized fields in lists. In the list, create a filter using the original (raw) value for the normalized field in the query condition.



The filtered list returns records with the normal or transformed value substituted for the raw value. However, the breadcrumbs for the filter display the original query conditions, including the raw value.


9 Identifying Normalized or Transformed Fields

Fields that are normalized or transformed in ServiceNow records might be marked with an icon that links to two different targets, depending on the user's role. Users with the normalizer role can click on the icon to access the configuration record for that field directly. When a user without the normalizer role clicks the link, a Wiki page appears that provides help on the form. An administrator can restrict the visibility of this icon in a property, based on user roles.
