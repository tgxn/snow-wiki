1 Overview

Workflow scripts provide an interface with the workflow engine.

1.1 Where To Use

Use these methods to manipulate workflows.

2 Method Summary

Method Summary Description
initialize() Method called by the Prototype JavaScript Framework during object construction.
startFlow(workflowId, current, operation, vars) Starts a specified workflow.
startFlowFromContextInsert(context, operation) Helper method for business rule "Auto start on context".
startFlowRetroactive(workflowId, retroactiveMSecs, current, operation, vars, withSchedule) Used by the "Start Workflow" business rule on table task_sla to start a workflow.
runFlows(record, operation) Runs all workflows for a given record in a given table and its descendant tables.
getReturnValue(workflowId) Gets the appropriate workflow return value for the input workflow ID.
getVersion(workflowId) Gets the appropriate workflow version for the input workflow ID.
getVersionFromName(workflowName) Gets the appropriate workflow version for the input workflow name.
getWorkflowFromName(workflowName) Gets the appropriate workflow for the input workflow name.
hasWorkflow(record) Determines if a specified record has an associated workflow.
fireEvent(eventRecord, eventName) Fires the named event on the input record.
fireEventById(eventRecordId, eventName) Fires the named event on the record specified by record ID.
broadcastEvent(contextId, eventName) Broadcasts the named event for the specified context ID.
cancel(record) Cancels any running workflows for the document.
cancelContext(context) Cancels the specific running workflow context.
getRunningFlows(record) Gets the running workflow contexts for a specified record.
getContexts(record) Gets the workflow contexts for a specified record whether it is running or not.
restartWorkflow(current, maintainStateFlag) Restarts the workflows associated with a GlideRecord.
deleteWorkflow(current) Deletes all workflows associated with a specified GlideRecord.
sortStages(versionID) Not implemented.
getEstimatedDeliveryTime(workflowID) Gets the estimated time for a workflow to complete.
getEstimatedDeliveryTimeFromWFVersion(wf_version) Gets the estimated time for a workflow to complete.

3 Method Detail

3.1 initialize()

Called by the Prototype JavaScript Framework during object construction. Do not call this method directly.

3.1.1 Input Fields

Parameters:

  • None

3.1.2 Output Fields

Returns: void

3.2 startFlow(workflowId, current, operation, vars)

Starts a specified workflow. See script include WorkflowScheduler and Business Rule "Start Workflow" on table sc_req_item for examples of use.

3.2.1 Input Fields

Parameters:

  • workflowId - The sys_id of the workflow to start. This sys_id refers to table wf_workflow.
  • current - GlideRecord of the record to use as current in this workflow. This is normally from the "Table" field of the workflow properties for this workflow.
  • operation - The operation being performed on current. Possible values: "insert", "update", "delete".
  • vars - Collection of variables to add to the workflow.

3.2.2 Output Fields

Returns: A GlideRecord on table wf_context on the inserted record for this newly created workflow context.

3.2.3 Example

////where current is a task record with a workflow context
var w = new Workflow();
var context = w.startFlow(id, current, current.operation(), getVars());

3.3 startFlowFromContextInsert(context, operation)

Helper method for business rule "Auto start on context".

3.3.1 Input Fields

Parameters:

  • context - GlideRecord on table wf_context of a new record (the "current" record in the business rule).
  • operation - Database operation being performed. One of "insert", "update", "delete".

3.3.2 Output Fields

Returns: void

3.3.3 Example

////where current is a task record with a workflow context
current.name = current.workflow_version.name;
current.started_by.setValue(gs.userID());

if (gs.nil(current.id)) {
    var gr = new GlideRecord('wf_workflow_execution');
    gr.name = current.name;
    gr.insert();

    current.table = 'wf_workflow_execution';
    current.id = gr.sys_id;
}

var wf = new Workflow();
wf.startFlowFromContextInsert(current, current.operation())

3.4 startFlowRetroactive(workflowId, retroactiveMSecs, current, operation, vars, withSchedule)

Used by business rule "Start Workflow" on table task_sla. This starts a workflow and the extra arguments to this method are used by activity "Timer" to pause the execution of the workflow for some duration.

3.4.1 Input Fields

Parameters:

  • workflowId - The sys_id of the workflow to start. This sys_id refers to table wf_workflow.
  • retroactiveMSecs - Delay in milliseconds used by Activity "Timer".
  • current - GlideRecord of the record to use as current in this workflow. This is normally from the "Table" field of the workflow properties for this workflow.
  • operation - The operation being performed on current. Possible values: "insert", "update", "delete".
  • vars - Collection of variables to add to the workflow.
  • withSchedule - Schedule used by Activity "Timer".

3.4.2 Output Fields

Returns: A GlideRecord on table wf_context on the inserted record for this newly created workflow context.

3.4.3 Example

// is this a retroactive start?
////where current is a task record with a workflow context
var msecs = new GlideDateTime().getNumericValue() - current.start_time.getGlideObject().getNumericValue();
   
// treat this as a retroactive workflow start if the SLA started more than 5 seconds ago
var w = new Workflow();
if (msecs <= 5000)
   w.startFlow(id, current, current.operation());
else
   w.startFlowRetroactive(id, msecs, current, current.operation());

// update the record in case the workflow changed some values
current.update();

3.5 runFlows(record, operation)

Runs all workflows for a given record in a given table and its descendant tables. Sample usage can be seen in the Script Includes "SNC - Run parent workflows", and "SNC - Run parent workflows (Approval)".

3.5.1 Input Fields

Parameters:

  • record - The record to run workflows against.
  • operation - Database operation. One of "insert", "update", "delete".

3.5.2 Output Fields

Returns: void

3.5.3 Example

var wkfw = new Workflow();
wkfw.runFlows('f2400ec10b0a3c1c00ca5bb5c6fae427','update');

3.6 getReturnValue(workflowId)

Gets the appropriate workflow return value for the input workflow ID. This is either the workflow checked out by the current user or the published workflow with the most recent date. This method is available starting with the Fuji release.

3.6.1 Input Fields

Parameters:

workflowId - The sys_id of the workflow (table wf_workflow)

3.6.2 Output Fields

Returns: The return value of the workflow as specified by the Return Value activity. Workflows without a Return Value activity return a null value.

3.6.3 Example

var wkfw = new Workflow();
wkfw.getReturnValue('context');

3.7 getVersion(workflowId)

Gets the appropriate workflow version for the input workflow ID. This is either the workflow checked out by the current user or the published workflow with the most recent date.

3.7.1 Input Fields

Parameters:

  • workflowId - The sys_id of the workflow (table wf_workflow)

3.7.2 Output Fields

Returns: GlideRecord on table wf_workflow_version and the current record the correct workflow version.

3.7.3 Example

var wkfw = new Workflow();
wkfw.getVersion('b99a866a4a3623120074c033e005418f');

3.8 getVersionFromName(workflowName)

Gets the appropriate workflow version for the input workflow name. See getVersion for more information

3.8.1 Input Fields

Parameters: The name of the workflow.

  • workflowName - The name of the workflow.

3.8.2 Output Fields

Returns: The sys_id of the appropriate workflow version in table wf_workflow_version

3.8.3 Example

var wkfw = new Workflow();
wkfw.getWorkflowVersionFromName('Emergency Change');

3.9 getWorkflowFromName(workflowName)

Gets the appropriate workflow for the input workflow name.

3.9.1 Input Fields

Parameters:

  • workflowName - The name of the workflow.

3.9.2 Output Fields

Returns: The sys_id of the workflow in table wf_workflow.

3.9.3 Example

var wflw = new Workflow();
gs.print(wflw.getWorkflowFromName('Emergency Change'));

Output: 

*** Script: b99a866a4a3623120074c033e005418f

3.10 hasWorkflow(record)

Determines if a specified record has any workflow contexts associated to it. This includes running and completed workflow contexts.

3.10.1 Input Fields

Parameters:

  • record - GlideRecord under scrutiny. This GlideRecord can be from any table.

3.10.2 Output Fields

Returns: True, if record has associated workflow; otherwise, returns False.

3.10.3 Example

var wkfw = new Workflow();
gs.print(wkfw.hasWorkflow('f2400ec10b0a3c1c00ca5bb5c6fae427'));

*** Script: false

3.11 fireEvent(eventRecord, eventName)

Fires the named event on the input record. Used in Activities "Approval Coordinator", "Timer", "Lock", and some others.

3.11.1 Input Fields

Parameters:

  • eventRecord - (GlideRecord) The event record.
  • eventName - The name of the event to send to the executing workflow.

3.11.2 Output Fields

Returns: void

3.11.3 Example

////where current is a task record with a workflow context
var w = new Workflow(); 
w.fireEvent(current, 'execute');

3.12 fireEventById(eventRecordId, eventName)

Fires the named event on the record specified by record ID. Used in Activities "Approval Coordinator", "Timer", "Lock", and some others.

3.12.1 Input Fields

Parameters:

  • eventRecordId - The sys_id of the glide record.
  • eventName - The name of the event to send to the executing workflow

3.12.2 Output Fields

Returns: void

3.12.3 Example

var wkfw = new Workflow();
wkfw.fireEventById('f2400ec10b0a3c1c00ca5bb5c6fae427','Timer');

3.13 broadcastEvent(contextId, eventName)

Broadcasts the named event for the specified context ID.

3.13.1 Input Fields

Parameters:

  • contextId - The context ID.
  • eventName - The name of the event.

3.13.2 Output Fields

Returns: void

3.13.3 Example

//where current is a task record with a workflow context
var wf = new Workflow().getRunningFlows(current);
while(wf.next()) {     
   new Workflow().broadcastEvent(wf.sys_id, 'resume');
}

3.14 cancel(record)

Cancels all running workflows on this record by broadcasting the "cancel" event to activities in all running workflows on this record.

3.14.1 Input Fields

Parameters:

  • record - GlideRecord on any table. All workflows running on this record will be cancelled.

3.14.2 Output Fields

Returns: void

3.14.3 Example

//get workflow helper
var workflow = new Workflow();
//cancel all the workflows, where current is a task record with a workflow context
workflow.cancel(current);
gs.addInfoMessage(gs.getMessage("Workflows for {0} have been cancelled", current.getDisplayValue()));

3.15 cancelContext(context)

Cancels this running context by broadcasting a "cancel" event to activities in this workflow context.

3.15.1 Input Fields

Parameters:

  • context - GlideRecord of the running context to cancel.

3.15.2 Output Fields

Returns: void

3.15.3 Example

// If a workflow has started for this item, cancel it, where current is a task record with a workflow context
if ((current.stage == 'Request Cancelled') && current.context && !current.context.nil()) {
   var w = new Workflow();
   var gr = new GlideRecord('wf_context');
   
if (gr.get(current.context)) 
   w.cancelContext(gr);
}

3.16 getRunningFlows(record)

Gets all the currently running workflow contexts for the input record. The input record is any record on any table for which the caller wants the running workflow contexts.

3.16.1 Input Fields

Parameters:

  • record - GlideRecord of the record for which the caller wants a list of all running workflows

3.16.2 Output Fields

Returns: GlideRecord on table wf_context and filtered for all executing workflow contexts.

3.16.3 Example

//where current is a task record with a workflow context
var wf = new Workflow().getRunningFlows(current);
while(wf.next()) {     
   new Workflow().broadcastEvent(wf.sys_id, 'pause');
}

3.17 getContexts(record)

Gets all workflow contexts for the input record. The input record is any record on any table for which the caller wants the running workflow contexts. See getRunningFlows.

3.17.1 Input Fields

Parameters:

  • record - GlideRecord for which the caller wants a list of all workflow contexts.

3.17.2 Output Fields

Returns: GlideRecord on table wf_context filtered for all workflow contexts for the input record (in any state, e.g. running, cancelled, finished).

3.17.3 Example

//where current is a task record with a workflow context
var wkfw = new Workflow();
gs.print(wkfw.getContexts(current).started);

3.18 restartWorkflow(current, maintainStateFlag)

Recalculates the approvals and tasks for a workflow by adding new approvals and tasks while not resetting current approvals and tasks. For example, use this script to add an additional company to a change request, without reseting current approvals for companies already in the workflow.

3.18.1 Input Fields

Parameters:

  • current - GlideRecord of the record this workflow is executing. This can by any record on any table.
  • maintainStateFlag - (Boolean) If True, maintains all approvals and tasks in their current state.

3.18.2 Output Fields

Returns: void

3.18.3 Example

//mark all existing approvals for the change as 'cancelled' and restart the workflow to create new approvals
//where current is a task record with a workflow context
new WorkflowApprovalUtils().cancelAll(current, comment);
new Workflow().restartWorkflow(current);

3.19 deleteWorkflow(current)

Deletes all the workflows on the record.

3.19.1 Input Fields

Parameters:

  • current - GlideRecord for which the caller wants to delete all workflows. This can be any record on any table.

3.19.2 Output Fields

Returns: void

3.19.3 Example

//where current is a task record with a workflow context
var wkfw = new Workflow();
wkfw.deleteWorkflow(current);

3.20 getEstimatedDeliveryTime(workflowID)

Gets the estimated time for a workflow to complete.

3.20.1 Input Fields

Parameters:

  • workflowID - Sys_id of the workflow (table wf_workflow) to get the estimated run time.

3.20.2 Output Fields

Returns: Display value from a GlideDuration (e.g., 3 days), or blank if unknown.

3.20.3 Example

var wkfw = new Workflow();
gs.print(wkfw.getEstimatedDeliveryTime('b99a866a4a3623120074c033e005418f'));

Output: 

*** Script: 2 Days

3.21 getEstimatedDeliveryTimeFromWFVersion(wf_version)

Get the estimated elapsed execution time for the workflow version.

3.21.1 Input Fields

Parameters:

  • wf_version - GlideRecord on table wf_workflow_version of a specific workflow version for which the caller wants the estimated during of executing.

3.21.2 Output Fields

Returns: Display value from a GlideDuration (e.g., 3 days), or blank if unknown.

3.21.3 Example

//where current is a task record with a workflow context
var wkfw = new Workflow();
var context = wkfw.getContexts(current);
gs.print(wkfw.getEstimatedDeliveryTimeFromWFVersion(context.wf_version));
