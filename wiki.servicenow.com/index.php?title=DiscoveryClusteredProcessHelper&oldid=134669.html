DiscoveryClusteredProcessHelper

1 Overview

Provides a mechanism for re-wiring classified process associations to their cluster rather than a cluster node.

1.1 Where To Use

Use this script include with a Discovery script when you need to classify processes.


2 Method Summary

Method Summary Description
process(deviceGR) If the current device is a member of a cluster, processes classified processes that can be clustered.
getSupportedProcs() Determines which classified processes support clustering.
getCluster() Gets the first valid cluster where the current CI is a node.
processClusteredProcs() Processes any existing classified processes associated with the cluster, and newly discovered classified processes from one of the cluster nodes.
getClassifiedProcesses(child) Gets all classified processes that have a "Runs on::Runs" relationship with the given child.
processExisting(classyProc) Rewrites the "@host" portion of an existing cluster process name if a node has overwritten it.
processNew(classyProc) Processes a new classified process.
removeNodeRelationship(process) Removes the relationship between the classified process and the cluster node.
isClusterResource(process) Determines if a specified process is a cluster resource.
getExistingProc(procName) Gets a classified process that already exists on the cluster.
layer7Connections(deviceGR) Builds up TCP half connections for clustered processes that are listening on * (enumerate over cluster IPs in addition to the CI IPs that are already there).
isListeningOnAll(pid) Determines if the specified process is listening on all TCP connections.


3 Method Detail

3.1 process(deviceGR)

If the current device is a member of a cluster, processes classified processes that can be clustered.

3.1.1 Input Fields

Parameters:

  • deviceGR - (GlideRecord) The record of the device to process.

3.1.2 Output Fields

Returns: void


3.2 getSupportedProcs()

Determines which classified processes support clustering, by creating a array of supported processes.

3.2.1 Input Fields

Parameters:

  • None

3.2.2 Output Fields

Returns: void


3.3 getCluster()

Gets the first valid cluster where the current CI is a node.

3.3.1 Input Fields

Parameters:

  • None

3.3.2 Output Fields

Returns: (GlideRecord) the first valid cluster where this CI is a node.


3.4 processClusteredProcs()

Processes any existing classified processes associated with the cluster, and newly discovered classified processes from one of the cluster nodes.

3.4.1 Input Fields

Parameters:

  • None

3.4.2 Output Fields

Returns: void


3.5 getClassifiedProcesses(child)

Gets all classified processes that have a "Runs on::Runs" relationship with the given child.

3.5.1 Input Fields

Parameters:

  • child

3.5.2 Output Fields

Returns: (GlideRecord) the classified processes.


3.6 processExisting(classyProc)

Rewrites the "@host" portion of an existing cluster process name if a node has overwritten it.

3.6.1 Input Fields

Parameters:

  • classyProc - The cluster process name.

3.6.2 Output Fields

Returns: void


3.7 processNew(classyProc)

Processes a new classified process. Note that this process may be merged with an existing classification process on the cluster.

3.7.1 Input Fields

Parameters:

  • classyProc - The classification process.

3.7.2 Output Fields

Returns: void


3.8 removeNodeRelationship(process)

3.8.2 Output Fields

Checks if the specified process is a cluster resource.

3.9.1 Input Fields

3.9.2 Output Fields

Gets a classified process that already exists on the cluster.

Builds up TCP half connections for clustered processes that are listening on * (enumerate over cluster IPs in addition to the CI IPs that are already there).
