1 Overview

A document feed is a live feed group that is associated with a record, such as an incident or change. Document feeds allow users to work on tasks and other records through the live feed interface. Users can post messages in live feed that are automatically maintained as comments or work notes on the record (if the record has these journal fields).

The benefits of using document feeds include:

  • Improving communication between users working on the same record.
  • Improving visibility into progress for end users.
  • Providing a single place to see updates on multiple records.
  • Saving conversation history in the record so that knowledge is not lost in an email chain.

This article helps administrators activate and configure the Live Feed Document plugin.

To use document feeds, activate the Live Feed Document plugin. See Using Live Feed to Work on Records to learn how to use this feature.

A document feed for an incident

1.1 How Document Feeds Work

A document feed is a live feed group associated to a record. ServiceNow automatically creates a document group for the feed when a user follows or shows a record on live feed. The system also creates a document group when a user creates a new record on a table that has live feed enabled (Calgary release).

The document feed group:

  • Is unlisted (does not appear when users view all groups on live feed).
  • Automatically approves membership for every user who can access the record.
  • Uses the record number as the group name.
  • Uses the record short description as a group description.
  • Maintains all messages posted to live feed on the record (if the record has a journal field for comments). If the record has a standard text field for comments, each live post overwrites the field value.
  • Maintains all messages posted on the record in live feed (if the record has a journal field for comments). When the group is created, existing messages are added to the document feed.

A user who participates in live feed becomes a member of the group.

2 Activating the Plugin

Administrators can enable document feeds in an instance by activating the Live Feed Document plugin, which activates the Live Feed plugin, if it is not active. The Live Feed Document plugin is active by default for new Calgary instances. For upgrades to Calgary, administrators must activate the plugin. You may want to review the installed components before activating the plugin.

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.

3 Installed Components

3.1 Business Rules

The following business rule is added:

  • Live Feed integration: writes journal comments to the live feed if there is a group for this record.

3.2 UI Actions

The following UI actions are added:

  • Follow on Live Feed (for lists): adds the user to the live feed group for this record. Appears as a new option on the context menu for task lists.
  • Follow on Live Feed (for forms): adds the user to the live feed group for this record. Appears as a new option on the context menu for task forms.
  • Show Live Feed (for lists): displays live feed for the document. Appears as a new option on the context menu for task lists.
  • Show Live Feed (for forms): displays live feed for the document. Appears as a new option on the context menu for task forms.

3.3 UI Pages and Widgets

The following UI page and widget are added:

  • render_gadget_live_feed_small (UI page): adds a small version of live feed for viewing document feeds from forms and on homepages.
  • Live Feed (widget): allows the user to add live feed on a homepage.

3.4 System Properties

The following property is added to the System Properties [sys_properties] table when you activate the Live Feed Document plugin for the Calgary release. This property is not enabled by default. Administrators must enable it by navigating to Social IT > Feed Administration > Properties.

  • Toggle the display of the live feed tab in the activity formatter (glide.ui.show_live_feed_activity): enables the Live Feed-Activity toggle on the activity formatter.

3.5 Bookmarks

The following bookmark is added (requires the UI11 plugin):

  • Live Feed: adds live feed in a flyout window.

4 Configuring Document Feeds

Configuring a table to support document feeds includes these procedures:

  • Add the Show Live Feed button () in the form header.
  • Add Follow on Live Feed and Show Live Feed as list and form UI actions.

By default, the Show Live Feed icon appears on the form header for the Incident, Change Request, and Problem tables and in the list and form context menus for all task tables.


To add the Show Live Feed button in the form header for a table:

  1. Navigate to System Definition > Tables.
  2. Open the table record.
  3. Select the Live feed check box.
  4. Click Update.

If the check box is already selected and the live feed functionality is not active for the table:

  1. Navigate to System Definition > Dictionary.
  2. Open the dictionary entry for the table.
  3. Add live_feed=true in the Attributes field.
  4. Click Update.

To add live feed UI actions on a table:

  1. Navigate to System Definition > UI Actions.
  2. Open one of the live feed UI actions (for example, the Follow on Live Feed list UI action).
  3. In the Table field, select the table name.
  4. Right-click the header and select Insert to create a copy of the UI action for the desired table.
  5. Repeat steps 1 – 4 for all live feed UI actions.

4.1 Disabling Document Feeds

To disable Live Feed functionality from the form of any table:

  1. Navigate to System Definition > Tables.
  2. Open the table record.
  3. Clear the Live feed check box.
  4. Click Update.

If the check box is already cleared and the live feed functionality still exists for the table:

  1. Navigate to System Definition > Dictionary.
  2. Open the dictionary entry for the table.
  3. Add live_feed=false in the Attributes field.
  4. Click Update.

5 Configuring Security

Document feeds honor the access control rules (ACLs) for the associated record. Users can only view messages on the document feed if they have access to the same information on the record. For example:

  • If an ACL allows a user to read and create comments on an incident, then the user can view and add messages posted as comments on the incident feed.
  • If an ACL restricts a user from reading work notes, then the user cannot view messages posted as work notes on the incident feed.
 Note: Access control rules are only checked when a user first accesses the document feed. After users view the feed, an administrator must remove them manually to change their access.


6 Enhancements

6.1 Dublin

  • Administrators can enable document feeds for a table from the table record.

6.2 Calgary

  • The Live Feed Document plugin is now enabled by default.
  • Users can view and interact with a document feed directly on a form's activity formatter if administrators enable the new document feed properties.
  • A new property is available to control document feeds on records (but is not enabled by default). The glide.ui.show_live_feed_activity property changes the record form's activity formatter to include two tabs: Live Feed, which allows users to collaborate on the document feed, and Activity, which shows a summary of all activity for the record. Access this property on the Social IT > Feed Administration > Properties page.
  • When a user creates a record on a table that uses document feeds, the live feed application automatically creates a document group at the same time and the creator becomes the administrator. Likewise, when a user modifies a record that has document feeds enabled, live feed automatically adds the user to the document group.
