1 Overview

MDSL Telecom Expense Management provides inventory and expense management for telecom goods and services. Integration between ServiceNow and the MDSL request and procurement system is a two-way web service integration. This integration currently supports service catalog requests for mobile devices but can be expanded to support any other telecom product or service.

  • The request process begins when a user orders a mobile device through the ServiceNow service catalog.
  • A service catalog workflow for mobile telecom devices triggers a SOAP message to MDSL.
  • ServiceNow passes relevant variables to MDSL using the SOAP message. Refer to the list of usable variables for more information. This SOAP message starts the MDSL request workflow.
  • The MDSL integration layer reads the SOAP message from ServiceNow and processes the information for the MDSL request workflow.
  • After the MDSL request workflow is complete, MDSL sends a success or failure acknowledgement to ServiceNow along with these device specific details to update the ServiceNow CMDB: device serial number, mobile number, contract number, and service plan.


2 Setting up an MDSL Integration for Service Catalog Requests

To integrate ServiceNow and MDSL, create an outbound SOAP message and a catalog request workflow to trigger the SOAP message.

2.1 Create a SOAP Message

  1. Navigate to System Web Services > SOAP Message.
  2. Click New.
  3. In the Name field enter MDSL_Start_Workflow_V2 or another descriptive name.
  4. Enter the SOAP message to be sent to MDSL in the WSDL XML field.
    See the SOAP message example for more information
  5. [Optional] Enter a Description for the SOAP Message.
  6. Do not modify other fields.
  7. Click Submit.


This example SOAP message outlines the variables that need to be sent from ServiceNow to MDSL. Variables are indicated with the format ${variable}. 

SOAP Envelope:
<SOAP-ENV:Envelope xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:SOAP-ENC="http://schemas.xmlsoap.org/soap/encoding/" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:SOAP-ENV="http://schemas.xmlsoap.org/soap/envelope/" xmlns:m="http://www.mdsl.com/ExtProcessToWorkflow"  SOAP-ENV:encodingStyle="http://schemas.xmlsoap.org/soap/encoding/">
  <SOAP-ENV:Body>
    <StartWorkFlow xmlns="http://www.mdsl.com/ExtProcessToWorkflow">
       <requestid xsi:type="xsd:string">${requestid}</requestid>
       <requester_hrid xsi:type="xsd:string">${requester_hrid}</requester_hrid>
      <enduser_hrid xsi:type="xsd:string">${enduser_hrid}</enduser_hrid>
      <authorizer_hrids xsi:type="m:ArrayOfAuthHridValues">
      <AuthHridValues xsi:type="m:AuthHridValues">
      	<authhrid xsi:type="xsd:string">${authhrid}</authhrid>
      </AuthHridValues>
      </authorizer_hrids>
      <reqstatus xsi:type="xsd:string">${reqstatus}</reqstatus>
      <!-- one out of {'New', 'Change', 'Cancel', 'Other'} -->
      <reqType>${reqType}</reqType>
      <requestVariables xsi:type="m:ArrayOfRequestVariablesArray">
      	<!-- Array[]   nillable-->
     	 <requestVariablesArray xsi:type="m:requestVariablesArray">
      		<key xsi:type="xsd:string">${key}</key>
      		<value xsi:type="xsd:string">${value}</value>
     	 </requestVariablesArray>
      </requestVariables>
      <!-- optional -->
      <Notes xsi:type="xsd:string">${notes}</Notes>
    </StartWorkFlow>
  </SOAP-ENV:Body>
</SOAP-ENV:Envelope>

2.1.1 Variables

MDSL uses specific variable names when retrieving data from ServiceNow. Additional variables not included in this table must be created in MDSL before they can be passed from ServiceNow.

Variable Definition Example
${requestid} The ID generated by ServiceNow when placing the service catalog request. RITM0010083
${requested_hrid} The ServiceNow User ID of the person who submitted the catalog request. HRID1234
${enduser_hrid} The ServiceNow User ID of the final recipient of the requested item. HRID4321
${authhrid} The ServiceNow User ID of the catalog request approver. HRID2341
${reqstatus} The approval status of the catalog request. 1
${reqType} The request type generated by ServiceNow (New, Change, Cancel, Other) New
${key} The type of device ordered. R_MOBILE
${value} A unique item identifier used by MDSL to update the ServiceNow CMDB. 290052243
${notes} Any optional notes that need to be passed to MDSL. Please confirm pricing.

2.2 Configure the Service Catalog

  1. Create a new service catalog workflow. See Defining a Service Catalog Workflow for more information.
  2. Create a Run Script activity in the workflow. See the example script for more information.
  3. Click on the Workflow Actions Menu () and click Publish.
  4. Navigate to Service Catalog > Maintain Items.
  5. Edit the catalog item record for a mobile device.
  6. Enter the new workflow in the Workflow reference field.
  7. Click Update.



This example workflow includes a Send to MDSL workflow activity following successful approval of the request. The workflow activity sets the required variables and sends the appropriate SOAP message to initiate the request within MDSL. 

var s = new SOAPMessage('MDSL_Start_WorkFlow_V2', 'StartWorkFlow');
s.setParameter('requestid', current.number);
s.setParameter('requester_hrid', 'MDSLSN');
s.setParameter('enduser_hrid', 'MDSLSN');
s.setParameter('authhrid', '10276926');
s.setParameter('reqstatus', '1');
s.setParameter('reqType', 'New');
s.setParameter('key', 'R_MOBILE');
s.setParameter('value', '290052243');
s.setParameter('notes', current.variables.requirements);
var response = s.post(true);

3 Setting up Access to the MDSL Web Portal Through ServiceNow

End users can check the status of a request through the MDSL web portal. Access to the MDSL web portal is set up by MDSL and passed to the client for stand-alone access or iFrame integration within ServiceNow through single sign-on (SSO).

3.1 SSO Integration

MDSL allows access to the MDSL web portal through SSO. SSO access can be configured with industry standard SAML/WS-Federation technologies or a custom-built simple SSO. The custom SSO solution requires the user to provide a specific parameter, including appropriate identification information, in the URL used to access the web portal. Refer to the table of usable methods for detailed information. Parameters passed to MDSL are indicated in bold. The MDSL system compares the paramater provided in the URL against existing user records. If the parameter matches a single user, MDSL allows access using those user credentials.

To set up iFrame access to the MDSL web portal in ServiceNow:

  1. Navigate to Content Management > iFrames.
  2. Click New.
  3. Enter a Name for the iFrame.
  4. Set Sizing to Fixed size.
  5. [Optional] Set the Height and Width fields to the desired size.
  6. [Optional] Enter a Frame name.
  7. Enter the MDSL web portal URL including the specific parameter in the URL field. See the table of usable methods for more information.
  8. Click Submit.
  9. Add the iFrame to a page.
MDSL returns the following message if the user is denied access:
SSO Succeeded but the current user is either not permissioned to access this application or the details provided didn't match a single user. See server event log for more details.

3.1.1 Usable Methods

Method Parameter URL
Email https://asp.mdsl.com/<Customer MDSL Instance>/Portal/Usage.aspx?SN-IDP-email=username@yourcompany.com
HR ID (employee number) https://asp.mdsl.com/<Customer MDSL Instance>/Portal/Usage.aspx?SN-IDP-hrid=HRIDDemo
UserID (Windows NT login) https://asp.mdsl.com/<Customer MDSL Instance>/Portal/Usage.aspx?SN-IDP-userid=windows/userID
