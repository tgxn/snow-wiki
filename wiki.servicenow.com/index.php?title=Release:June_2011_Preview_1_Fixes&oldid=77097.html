1 June 2011 Preview 1 Fixes

1.1 PRB559908 (Related Lists): Related Lists could not be personalized after a Relationship that was part of the form was deleted

Deleting a Relationship no longer prevents personalizing Related Lists on forms that had the Relationship on them.

1.2 PRB559829 (Service Catalog): Variable information could be disassocated from Requested Items (extremely rare)

A null sys_id in the cart_remove action will no longer result in the deletion of unrelated sc_item_option_mtom records

1.3 PRB559850 (Reports): Report grid might not label aggregation type properly

The aggregation type of a report (COUNT, SUM, AVERAGE) is now property reflected in the report grid

1.4 PRB559849 (Reports): Report grid showed duration fields in seconds

Durations are now formatted as expected in the report grid

1.5 PRB559722 (Reports): Report with a duration field filter might not calculate the duration correctly for durations greater than 24 hours

Reports filtering on a duration field are no longer dependent on the user's time format, fixing the issue

1.6 PRB559716 (i18n): Short description field in the Knowledge Base did not display translated values

The knowledge portal and the article view have been modified to show translated versions of short_description, topic, and category where appropriate

1.7 PRB559772 (Variables): Giving a Numeric Scale variable a ridiculously large maximum value could consume excess memory

Numeric Scale variables are now limited to a maximum value of 1000

1.8 PRB044269 (Labels): Assigning a record to a label from a form could result in an empty record form being displayed

The current form is now properly reloaded when assigning the record to a label

1.9 PRB559942 (Project): Project task end date calculation did not properly use the Schedule defined on the top task

Changing the Schedule on a project's top task now properly calculates the end date of its project tasks.

1.10 PRB559995, PRB560201 (List v2): New button omitted from reference popups

The New button is once again available as expected for the sys_ref_list view of a list.

1.11 PRB560060 (Mobile): Mobile Boolean fields might not save updated values

Mobile Boolean fields now save updated values as expected.

1.12 PRB560032 (History Sets): Admins were wrongly able to delete History Line records

A new ACL prevents deletion of History Line records.

1.13 PRB560072 (UI): Copy URL might not work in a multiple-proxy configuration

Thecase where 'x-forwarded-host' headers arrive with multiple values, separated by commas, is now handled. The first value is used, because it is the one that the first

reverse-proxy put there.

1.14 PRB560212 (List v2): Non-client "list link" UI Actions did not work in List v2

Non-client "list link" UI Actions now function in List v2 consistently with how they functioned in List v1. They execute against each checked record in the list.

1.15 PRB560083 (Project): Project tables did not have sufficient ACLs

New ACLs have been added to Project tables as follows:

  • pm_project (write) Project Manager and Project User roles
  • pm_project (delete) Project Manager role
  • pm_project_task (write) Project Managers and Project User roles
  • pm_project_task (delete) Project Managers role

1.16 PRB559691 (List v2): Reference field Autocomplete and lookup list might return different number of records

The reference qualifier is no longer ignored for a reference field dependent on another field that has a value and has "group" somewhere in the field name.

1.17 PRB560043 (Task Activity): Personalize Activity was available to all, then failed if user did not have right role

Personalize Activity now only appears if the user has the right roles to actually use it. Failure is prevented before it is attempted.

1.18 PRB560058 (Reports): Users might see unauthorized reports in a list

The existing Read ACL for sys_report in the Security Jump Start plugin now honors the sys_report.roles field in addition to other security constraints. Customers who have installed the Security Jump Start plugin and who have not modified the existing ACL will get this change.

1.19 PRB560053 (UI): A read-only multi-line dotwalked field did not refresh when the reference field's value was changed on the form

1.20 PRB560171 (UI): Expand/collapse on dotwalked workflow widget in a list always executed against the top row

1.21 PRB560179 (CMDB): An invalid entry could be created in cmdb_ci_rel when removing the last relationship entry from a type

The last cmdb_rel_ci record is now successfully deleted and does not get recreated with a null parent or child.

1.22 PRB560170 (Report): Only the first page of records was displayed for a published List report in List v2

All rows are now displayed.

1.23 PRB559365 (Workflow): The "Turnstyle" Workflow Activity Definition was misspelled

Changed to "Turnstile".

1.24 PRB043753 (Report): Trend info was not included in filter when View Report was clicked from the sys_report form

Trend info is now preserved when clicking that UI Action.

1.25 PRB559533 (Security): Write ACL might be used on insert instead of Create ACL with high security plugin installed

The proper ACL is now checked.

1.26 PRB559119 (History Sets): Issue when first and only audit entry is a relationship change

This situation is now handled as expected, with relationship changes properly audited in the absence of other audit entries.


1.27 PRB560055 (Export): Comments exported to PDF would lose text within "<" and ">" brackets

That text is now preserved when exporting journal fields to PDF.


1.28 PRB560061 (Reports): Control Chart reports would fail if no data was returned

Protection is now there for a Control Chart with no data returned.

1.29 PRB560123 (Report): A Bar Chart with an aggregation other than COUNT would fail if changed to a Pie Chart

Pie Charts now only try to use COUNT aggregations, which is the only one they support.

1.30 PRB560093 (Database Views): Database views with an array field would generate an error in Oracle

The error (ORA-00942: table or view does not exist) is no longer generated for database views containing an array field.

1.31 PRB560073 (On-call Rotation): A manually entered on-call coverage person would not get assigned incidents (Introduced Winter 2011)

Manually entered on-call coverage now works as expected.

1.32 PRB560044 (Service Catalog): The default "Copy" UI Action on sc_cat_item did not copy Variable Sets

1.33 PRB560125 (Query): Query on roles via a dotwalked field did not work

Query on roles via dotwalked field (e.g., a query on sys_user_grmember where user.roles = "admin") now works as expected.

1.34 PRB560068 (Update Sets): Removing the My Current Set field from Update Set form breaks UI Actions

UI Actions for the sys_update_set form now protect against the My Current Set field not being present.

1.35 PRB380136 (Email): Duplicate emails could be sent

When the Subscription Based Notifications plugin is not installed, the code to ignore duplicate emails (identical recipients/subject/body) once again works as expected. De-duplication is not currently supported with the Subscription Based Notifications plugin.

1.36 PRB560098 (Domain): Domain Numbering issue after clearing the domain table

After deleting all records in the domain table on an instance with the Domain Number Support plugin active, new domains can now be created.

1.37 PRB559900 (Reports): A star level (*) report_on ACL could prevent creation of new reports despite overriding ACLs on tables

report_on ACLs now work as expected with row-level ACLs overriding a *-level ACL.

1.38 PRB560040 (CMDB): "Affected CIs" related list slushbucket did not show available CIs after activiating "Best Practice - Bulk CI" plugin

Appropriate available CIs are now shown in the slushbucket of the Affected CIs related list on the Change Request form.

1.39 PRB560015, PRB560016 (Chat): Unhandled JavaScript errors could limit Chat's capabilities

The source of the error has been removed.

1.40 PRB560075 (Views): JavaScript error when changing form view from a form section header (Chrome)

The source of the error has been removed.

1.41 PRB559980 (UI): The slushbucket for a glide_list field did not have "current" available to its reference qualifier

The current record is now provided to a glide_list Add/Remove Multiple slushbucket.

1.42 PRB559730 (Chat): "Create or Join Chat Room" form link UI Action incorrectly appeared in the Mobile view of Incident form

That UI Action has been excluded from the Mobile view.

1.43 PRB559969 (List Editing): Duplicate comments were possible after list editing a Journal field and then another field

The source of the problem has been removed.

1.44 PRB559909 (Table Creation): Creating a new module in an existing application might not work when creating a table

Now if a new application is created, but the admin then chooses to create the module in an existing application, the existing application gets the module.

1.45 PRB559635 (UI): Blank form when saving a new record after an initial save was cancelled via Business Rule

The 2nd save attempt now completes as expected.

1.46 PRB559927 (MID Server): MID server bug - if the username/password for proxy was set, it was improperly also used as the instance username and password

The correct properties are now used for authentication.



1.46.1 PRB560050: A Null Pointer Exception occurred when the Encryption Support Plugin was first activated while sessions were active.

If context vectors were not initialized, a session could have an uninitialized vector when the plugin was loaded and then threw null pointer exception until that session ended.

1.46.2 PRB560028: Prevent potential database deadlock from occuring in rare circumstances.

This situation occurred when the database connection pool was stressed. It also happened when the database connection pool was full on the first lease retry attempt.

1.46.3 PRB560031: A module with no title generated JavaScript errors when a user typed in the navigation filter.

1.46.4 PRB560016: Errors were generated when users first ran Chat Desktop.

Social IT/Chat functionality was updated to correct errors in the scrolling capability within chat windows.

1.46.5 PRB560018: HTML that previously rendered properly in calendar reports quit working. This occurred when the opening and closing tags in the html were of different case (<div></DIV>).

Changes were made to disregard case in these HTML tags.

1.46.6 PRB560017: Issues arose with the ODBC Driver for instances that did not have the Database Views plugin installed.

A isValid() check was added when creating the sys_db_views, since some instances do not have this enabled.

1.46.7 PRB560013: An issue occured with the ODBC Driver for Database Views. If no tables were defined with Database Views, then an invalid schema was returned.

The platform was updated to check for valid tables.

1.46.8 PRB560010: When a customer had their own POP3 server set up, and the user name had an @ sign in it, the platform did not pass the whole user name when logging in to the POP3 server.

With this update, we no longer truncate POP3 email user names at the @ sign if using the glide.pop3.user property value. We only truncate when using the glide.email.user name which is shared with the SMTP sender.

1.46.9 PRB560004: The GlideForm.getReference return value changed when the reference field was blank, causing JavaScript errors in cases where a client script did not check the return value from getReference.

This patch ensures that the return value is the same as it was previously to ensure that client scripts continue to function without change.

1.46.10 PRB559997: Derived field labels did not work when the short_label attribute was specified. The following error was displayed on the Incident form: *Unknown field label for Caller.User ID

With this update, the short_labels field attribute once again behaves as expected.

1.46.11 PRB560073 (On-Call Rotation): A manually entered on-call coverage person would not get assigned incidents.

Manually entered on-call coverage now works as expected.

1.46.12 PRB560098 (Domain): Domain Numbering did not work properly after clearing the domain table.

After deleting all records in the domain table on an instance with the Domain Number Support plugin active, new domains can now be created.

1.46.13 PRB560114 (Notifications): The IMAP integration was displaying an error.

The email connections for IMAP were restored.

1.46.14 PRB559995 (List v2): The New button was missing from reference pop-ups.

The New button is now available as expected for the sys_ref_list view of a list.

1.46.15 PRB560243 (Web Services): Web Services import sets lost its embedded Fields list.

The embedded list was restored to the form.

1.46.16 PRB559989 (User Interface): Multi-line text fields saved in a template were concatenated into one line. On saving a template, all line endings were replaced by a comma.

Multi-lined text fields now display properly.

1.46.17 PRB559865 (Queries): Clicking the Insert button within a report that uses a very complex query can build a URL that is too large for Microsoft Internet Explorer.

The resulting URL from a complex query in a report is now an acceptable length for Internet Explorer.

1.46.18 PRB559900 (Reports): A star level (*) report_on ACL could prevent creation of new reports despite overriding ACLs on tables.

report_on ACLs now work as expected with row-level ACLs overriding a *-level ACL.

1.46.19 PRB043753 (Reports): Trend information was dropped from the filter when View Report was clicked from the sys_report form.

Trend information is now preserved when clicking the View Report UI Action.

1.46.20 PRB450829 (Reports): A user could view reports for other users or groups if they had sufficient roles to view the report and the correct URL

Security for reports created with viewing restrictions is now being enforced properly.

1.46.21 PRB560185 (Domains): Domain Separation causes the Database View to return unexpected results if there are more than one top level domain and LEFT JOIN is used.

When a query is run on a 'global domain, the expected data is returned.

1.46.22 PRB560368 (API): The list_id was missing from the edit list row function.

The list edit API was modified to expect an additional parameter when marking a row as deleted in an embedded list. This fix ensures that the additional parameter is now passed to the API to correct the mark row as deleted action.

1.46.23 PRB560374 (User Interface): Reference lists should not allow access to records through the pop-up window.

Drilling through to a record from a reference list pop-up is no longer allowed. The pop-up document icon was removed from the list.

1.46.24 PRB560088 (Notifications): Sent emails were not showing up in the activity logs of the records with which they were associated.

All emails associated with an incident are now appearing in the activity log of the appropriate record.

1.46.25 PRB560292 (Service Catalog): The pop-up description of a catalog item could be truncated by the page frame.

When users hover the cursor over a catalog item's icon, the entire description is visible.
