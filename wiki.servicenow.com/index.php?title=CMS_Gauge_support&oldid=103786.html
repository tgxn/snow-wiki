1 Overview

Gauges are a graphical way to display information from an instance. A gauge might, for example, show a bar chart breaking down all open incidents by category. Gauges are fully supported within the CMS system. What this means in practice is that A) you can put a gauge on a CMS page and B) you can control what will happen when a user clicks on a cell within that gauge.

2 Putting a Gauge on a CMS Page

  1. Bring up the CMS page in question in edit mode
  2. Click Add Content
  3. Select the gauge you want to add
  4. Place it on the page in the desired location

3 Controlling what happens on a click

In the normal (non cms) system, when you click on, for example, a bar in a bar chart or a wedge on a pie chart, you drill through and your current screen is replaced with a list of records meeting the chart's criteria. For example if you have a gauge of "incidents by category" and you click the bar labelled "hardware" you will drill through into a list of all incidents with a category=hardware.

Within the CMS system, there is a similar drill through mechanism at work, but since the CMS system uses multiple pages, you have to tell the system which page to use to display the drill through content.

There are two different models available for you in terms of the drill through. First, you can use an in-place target frame and direct all content there. Alternately you can specify an alternate page to use to display the drill-through content.

3.1 Using an in-place target

An in-place target is a named iframe on the same CMS page as the gauge. When the gauge is clicked the drill-through content appears within that iframe rather than changing out the CMS page. The gauge (and the rest of the page other than the target) remain in place and active.

To use an in-place target, put an iframe block on the same page as the gauge and name it gsft_main. All drill through content will automatically be directed there for you.

3.2 Using a gauge target page

A gauge target page is a separate CMS page that is used to display the drill-through content from a gauge. When a gauge is clicked, the current CMS page is replaced with the gauge target page and the drill-through content is rendered within that second page.

3.2.1 Setting a site default

On the site configuration page, you can select a "Gauge Target Page". This will be used as the drill-through target for any gauges in the system that do not have a more specific gauge page specified in their content type (see below).

3.2.2 Setting a content type specific page

On a content type page, you can specify a "Gauge Page". Any gauges for this type of content will then use that page for any drill through. Note that the gauge target on a particular content type overrides the default, site level, gauge target.

3.2.3 Characteristics of a gauge target page

A gauge target page is a normal CMS page, with one special requirement. Somewhere on that page, there must be a Gauge Target block. This is a market block telling the system where to output the drill through data.
