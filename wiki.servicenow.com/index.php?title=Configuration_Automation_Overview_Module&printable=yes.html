Configuration Automation

1 Overview

The Configuration Automation Overview module displays various configuration management reports that are used as gauges. These gauges allow you to track and manage different aspects of the configuration management process by displaying information on managed nodes, configuration templates, node groups and change requests.

Note: A configuration template was called a node definition in earlier versions of the system.


The Overview module is a type of homepage. For details about editing gauges on homepages, see Adding Existing Gauges to a Homepage.

2 Roles

Users with the config_auto_admin or config_auto_user role can view the overview page and refresh, add, delete, and rearrange gauges.

3 Using the Configuration Automation Overview Module

To use the Configuration Automation Overview module, navigate to Configuration Automation > Overview. You can click elements within the gauges to obtain more information or add and move gauges as needed.

The overview module displays the following types of gauges:

  • Pending Managed Nodes: number of managed nodes that are either in In Draft, Awaiting approval, or Ready to deploy stages.
  • Managed Nodes by Node Definition: chart displaying the number of managed nodes grouped by configuration template.
  • Managed Nodes by Department: chart displaying the number of managed nodes grouped by assigned department.
  • Managed Nodes by Operating System: chart displaying the number of managed nodes grouped by operating systems.
  • Managed Nodes by Owner: chart displaying the number of managed nodes grouped by the owner.
  • Deployed Nodes Daily: chart displaying the number of deployed node counts by day for the past 30 days (ServiceNow Provider only).
  • Deployed Nodes Weekly: chart displaying the number of deployed node counts by week for the past 6 months (ServiceNow Provider only).
  • Managed Nodes: a list of managed nodes grouped by the defined configuration management provider.
