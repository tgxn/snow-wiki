1 Introduction

 Note: This functionality is replaced by Service Catalog record producers.


An Interceptor can be used to gather upfront information from the user which can then be used to place data into a record that is being created. For example, ask the user a few questions to see what the category and subcategory should be for an incident the user is having.

There are two ways that data can be placed into an initial record (other than default values).

  • A query can be populated that contains the values. When a record is created, the system will look for a sysparm_query parameter on the URL. If one is found then the fields mentioned in the query will be used to initialize the record. In this case you would probably ask your users to provide some very specific data that you would use to construct the sysparm_query.
  • A template can be specified. In this case you would probably ask the user some general questions and then react to those answers to pick the template that you have already created. The template contains all the values you want the record to be initialized with. This is a preferred method since the Interceptor code you write does not have to know the format for a sysparm_query.

In either case you would construct your Interceptor to gather the answers from the user. At the end of the Interceptor you would have an Execute entry. The script in the Execute entry would look at the data provided by the user and build a URL which would initialize a record and present it to your user.
