1 Overview

Adding a custom field to the Netcool integration requires updating both the inbound and outbound workflows.

  • Add a custom ObjectServer field to inbound events.
  • Include a custom ObjectServer field in outbound events.

2 Adding a Custom ObjectServer Field to Inbound Events

By default, the integration only uses the ObjectServer fields available in a default installation of a Netcool/OMNIbus ObjectServer. To add a custom ObjectServer field to the inbound events, use one of these workflows.

  • If the integration uses a Socket Monitor to listen for inbound events, use the Socket Monitor workflow.
  • If the integration uses a direct web service to receive inbound events, use the direct web service workflow.

2.1 Socket Monitor Workflow

A field is represented by the following elements when a Socket Monitor processes inbound events.

Part of the Integration Field Definition Action Required
ObjectServer table Alert.Status U_MyCustomField Add field to ObjectServer Alert.Status table. See the Netcool ObjectServer documentation.
ECC queue payload U_MyCustomField None, the inbound Netcool event automatically contains the custom field.
Socket Monitor map = '@U_MyCustomField' Add field to Socket Monitor map. See Updating the Socket Writer Gateway Map.
Script output of business rule NetcoolEvents Sensor u_mycustomfield None, the business rule uses the Socket Writer Gateway map to update the custom field.
Name field in web service Netcool Alerts Status u_mycustomfield Add field to web service fields list. See Web Service Fields.
Import sets table imp_netcool_alerts_status u_mycustomfield None, this import sets table should already contain the custom field.
Transform map Netcool Alerts Status u_mycustomfield Add field as transform map source field. See Creating a Field Map.
Table incident mycustomfield Add field as transform map target field. See Creating a Field Map.

2.2 Direct Web Service Workflow

A field is represented by the following elements when a direct web service processes inbound events.

Part of the Integration Field definition Action Required
ObjectServer table Alert.Status U_MyCustomField Add field to Alert.Status table. See the Netcool ObjectServer documentation.
Name field of web service Netcool Alerts Status u_mycustomfield Add field to web service fields list. See Web Service Fields.
Import sets table imp_netcool_alerts_status u_mycustomfield None, the import sets table should already contain the custom field.
Transform map Netcool Alerts Status u_mycustomfield Add field as transform map source field. See Creating a Field Map.
Table incident u_mycustomfield Add field as transform map target field. See Creating a Field Map.

2.3 Inbound Example

This example illustrates adding a custom ObjectServer field, u_MyCustomField, to the integration. The contents of this field will reside in the Incident table. This example assumes that the integration uses a Socket Monitor to receive inbound events and that there is no existing field in the incident table suitable to receive inbound data.

  1. Add the custom column to the alerts.status table on the ObjectServer. For example, U_MyCustomField.
    Note: You must add a U_ prefix to the beginning of the ObjectServer custom field because a ServiceNow business rule adds this prefix on all user-created web service fields.
  2. Add the custom field to the ServiceNow table you want to receive the event data. For example, add the field MyCustomField [incident.u_mycustomfield] to the incident table.
    1. Navigate to Incident > All.
    2. Click the Number link from an incident.
    3. Right-click the form header and select the appropriate option for your version:
      • Fuji or later: Configure > Form Layout
      • Eureka or earlier: Personalize > Form Layout
    4. In the Name field of the Create new field section, enter a field name. For example, MyCustomField.
      A business rule automatically converts this name into a column name that has all lowercase letters, replaces spaces with underscores, and appends the "u_" prefix to the front of the name. In this example, the final column name becomes u_mycustomfield.
    5. Click Add.

    6. Click Save.

  3. Edit the Socket Monitor Map field to add the custom field to the map.
    1. Navigate to Integration - Netcool/OmniBus > Socket Monitors.
    2. Click the Port link for a Socket Monitor.
    3. Select the Start socket writer checkbox.
    4. Scroll to the bottom of the Map field, type a comma after the last map entry and add a new line. For example: 
      " = '@URL',
      Note: The previous field mapping needs to end with a comma "," character.
    5. On the new line, type the map entry for the custom field. For example: 
      " = '@U_MyCustomField'
      Note: The integration uses the name of this field to match against a field in the import sets table. Since the integration adds the u_ prefix to any custom field in the web service field list, the corresponding field in the Socket Monitor map must also begin with a U_ prefix. The case of the prefix in the Socket Monitor map does not matter as the NetcoolEvents Sensor business rule automatically converts the field name to lower case.
    6. Click Update.

  4. Add the custom field to the list of web service fields in the Netcool Alerts Status web service record. For example, add the Netcool field label MyCustomField.
    1. Navigate to Integration - Netcool/OmniBus > Netcool Alerts Status.
    2. Scroll to the bottom of the Web Service Fields section, double-click the Insert a new row... text.
    3. Enter the name of the web service label. For example MyCustomField.
    4. Click the green check mark. The integration automatically converts the label to a lower case field name. For example, u_mycustomfield.
      Note: The system automatically creates the web service field name based on the label you type. It appends the prefix u_ to the beginning of the label and converts the string to all lowercase. Do not include a u_ prefix in the label name or the web service field name will start with a duplicate u_ prefix.

    5. Click Update.
  5. Add the custom field to the Netcool Alerts Status transformation map. For example:
    1. Navigate to Integration - Netcool/OmniBus > Transform Map.
    2. From the Field Maps section, click New.
    3. From Source field, select the label of the custom field you added to the web service field list. For example, MyCustomField.
    4. From the Target field, select the field where you want the integration to store the matching event data. For example MyCustomField.

    5. Click Submit.

3 Including a Custom ObjectServer Field in Outbound Events

By default, the integration only updates the ObjectServer fields listed in the Updater Field Map module. This module displays a list of scripts that transform incident field values into appropriate values for the ObjectServer.

The baseline mappings are:

Netcool Alerts Status Field Mapped value
Acknowledged If the incident's current state is Resolved or Closed, set the incident state to 1, otherwise set it to 0.
Agent Set the value to the word "ServiceNow - " plus the incident number. For example, "ServiceNow - INC10056."
URL Set the ServiceNow URL to open the associated incident.
Severity Maps the incident's Severity field values to their equivalent values in the Netcool Severity field.

3.1 Outbound Events Workflow

To include a custom ObjectServer field in outbound events, use this workflow:

Part of the Integration Field definition Action Required
Table incident u_mycustomfield Add the custom field to an inbound event workflow so the integration can first receive any initial data from the ObjectServer for the custom field.
Listener u_mycustomfield None, the listener automatically identifies changes to Incident records.
Script output of Updater Field Map U_MyCustomField Create Netcool Field Map record for field for outbound communication to the ObjectServer.
ObjectServer table Alert.Status U_MyCustomField Add the custom field to the ObjectServer's inbound event workflow.

3.2 Outbound Example

This example illustrates adding a custom ObjectServer field u_MyCustomField to outbound events.

  1. Add the custom field to the inbound events so the integration can receive data from the ObjectServer for the custom field.
  2. Add the custom field to the Updater Field Map.
    1. Click Integration - Netcool/OmniBus > Updater Field Map.
    2. Click New.
    3. In Field, enter the name of the custom ObjectServer field. For example, U_MyCustomField.
    4. Select the field type.
    5. In the Value Script field, enter a script to gather the field value you want to send to the ObjectServer (see sample script).
    6. Click Submit.

3.2.1 Sample Script

The following sample script retrieves the current value of the custom field. 

// Get the value of the u_mycustomfield field in the incident table.
current.u_mycustomfield;
