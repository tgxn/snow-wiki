1 Overview

The ServiceNow smartphone interface supports many of the features found in the standard browser interface, including:

  • Lists
  • Forms
  • Navigation pane
  • Filtering

There are several UI differences that affect how users perform important actions, including:

  • Tags and Favorites on the home page
  • Memory of recent selections, lists, and documents
  • Journal streams
 Note: The smartphone interface uses familiar, industry-standard techniques for performing most actions. The documentation points out any differences.


For a list of supported devices, see Smartphone Interface.

For videos that demonstrate navigating through and customizing the smartphone interface, see Example Videos.

2 Configuring Smartphones

Configuration procedures vary by device. Use the information in this section to optimize mobile ServiceNow functionality on your device.

2.1 Apple iPhone and Android Phones

There are no special configurations needed for the iPhone or Android phones. They just work. However, it is worth noting that multi-line text input fields can be scrolled by placing two fingers on the touch screen within the borders of the field and moving up or down.

2.1.1 ServiceNow Icon on the Apple Home Screen

Administrators can add the icon for launching ServiceNow to the iPhone home screen from within the standard browser interface. Administrators can replace the ServiceNow icon with any 57x57 pixel icon, which is the maximum icon size that Apple does not automatically resize.


Note: If a user downloads the ServiceNow app, the icon appears automatically.


To replace the icon:

  1. Navigate to System Properties > My Company and open the company record.
  2. Configure the form to add the Apple icon field.
  3. Upload the new icon.

2.2 Apple iPod touch

The ServiceNow smartphone interface supports the iPod touch running iOS6 and above, and connected by WiFi. No special setup procedures are necessary.

2.3 BlackBerry

Blackberry OS version 4.6.0 or later is required. On the BlackBerry, navigate to Settings > Options > About to find the version.

In the BlackBerry browser application, bring up the menu and select Options > Browser Configuration. Make sure the following options are set:

  • Support JavaScript
  • Use Background Images
  • Browser Identification: BlackBerry

Depending on the size of the BlackBerry screen and personal preference, users can also navigate to the General Properties browser submenu and set the default font size and minimum font size to a smaller size. Smaller text displays more content, but may be difficult for some users to read.

3 Accessing a ServiceNow Instance

There are two methods for accessing a ServiceNow instance in the smartphone interface.

  • Accessing an instance from the browser.
  • Accessing an instance from the mobile app (starting with the Eureka release).

3.1 Accessing an Instance from the Browser

On a supported smartphone, access the standard URL of an instance using a supported browser. The ServiceNow instance automatically detects the device and redirects to the smartphone interface by appending $m.do to the end of the URL. Accessing the smartphone interface does not prevent the user from accessing the tablet or desktop interfaces.


4 Working with the Smartphone Interface

4.1 Using the Application Navigator

To open the application navigator, tap the icon on the top-left side of the interface.


Scroll the navigator up or down using one or two fingers. To expand an application, tap the application name. To hide the navigator, click the application navigator icon again.

4.2 Customizing the Home Page

The Home page displays the following types of information by default.

  • Favorites
  • Recent Lists
  • Tags
  • Recent Documents

To change the types of information on the Home page, tap the gear icon . The Tools screen opens.


For each section, you can tap the Show section switch to toggle that section on and off. Each section also contains options for limiting the number of items shown and enabling or disabling horizontal scrolling, where applicable.

4.2.1 Managing Favorites

You can add any list, record, or module to the Favorites section of the home page. The items you add to favorites appear as tags. Use the favorites for quick access to the items that you use most often.

To add an item to the favorites section:

  1. Navigate to the screen that you want to add as a favorite.
  2. Tap and hold the home icon ().
  3. Give the favorite a name, assign its tag a color and icon, and tap Save.


To change or remove a favorite:

  1. On the Home page, tap the Favorites section heading.
    The tags for your favorites are displayed.
  2. Tap and hold a tag until the Save Favorite to Home Page dialog box opens.
  3. Change the name, color, or icon as needed, and tap Save.
  4. To remove the favorite from the Home page, tap the trash can icon ().

4.2.2 Viewing Recent Lists

The Recent Lists section of the Home page shows the lists and records you have viewed recently. With a single tap, you can return to a recently viewed list of Priority 1 incidents or a user record, for example. Whenever you view a list or make a selection, the system adds it to Recent Lists and records when you viewed it. To see the details of a list or selection, simply tap its row.

4.2.3 Defining Tags

To help group and organize records, you can create tags and add them to specific records. For example, you could create a tag called High Priority ABC and assign it to critical incidents submitted by an important customer. Tags can be defined with an icon and color for easy identification, and tags that you create can be viewed by other users in your organization. With a single tap, you or another user can then locate all critical incidents logged by ABC Corporation. You can add as many tags as needed to any record.

4.2.3.1 Creating a New Tag

  1. On the Home page, tap the Tag section heading. You can also tap My Tags on the application navigator.
    Any previously defined tags are displayed. The image shows the number 1 in the High Priority tag, indicating that one record has been assigned this tag.

  2. Tap in the Create or search tags field and enter the new name. As you type, the new tag appears below the field.

  3. When you finish, tap Done.

  4. Select a color and icon for the new tag, and tap Save.

4.2.3.2 Assigning a Tag to a Record

  1. Navigate to a record that you want to assign a tag to, such as an incident record.

  2. Tap the More icon in the lower right-hand corner of the screen and select Tags.
  3. In the Tags screen, tap the tag you want to assign to this record. Notice that a check mark appears on the tag.

  4. If needed, you can click the tag again to deselect it. You can also select multiple tags for the record.
  5. When you finish, navigate away from the screen. Your tag selections are automatically saved.
 Note: You may also see green and red numbers on tags, as shown below. A green number indicates the number of records associated with that tag. A red number indicates the number of records associated with that tag that you have not viewed (that is, they were created or updated by other users).

4.2.3.3 Accessing Records That Have Tags

The Tags section on the Home page shows the tags you have defined and each tag shows the number of records assigned to it.

  1. Tap the tag to display records that have been assigned that tag.
    The screen shows the records with that tag. Note that they are not necessarily the same kind of records. In the example below, the first record is an approval and the second is an incident.

  2. You can sort the records by tapping the buttons at the top of the screen.

4.2.3.4 Removing Tags from Records

  1. Navigate to the record from which you want to remove a tag.
  2. Tap inside the record.
  3. Tap the More icon and select Tags.
  4. Tap the tag you want to remove from the record. The check mark disappears, indicating that the tag has been removed.

4.2.4 Viewing Recently Accessed Records

The Recent Documents section of the Home page provides an at-a-glance summary of recently accessed records. You can quickly access these records without tapping through the application navigator. To view details of a recently viewed record, tap the record.

4.3 Viewing and Filtering Lists

You can open lists from the Recent Lists section of the home page or from a module in the application navigator. Edit the filter conditions to control which records appear in a list.

  1. Open a list.
  2. Tap the filter.

    The filter editor opens. It shows the current filter conditions and lists the other available conditions that can be added to the filter.

  3. To delete a condition, tap the adjacent X.
  4. To add a condition, tap it.
    The screen presents options based on the condition selected. For example, it shows a text field if you tap Short Description. It shows list of operators if you tap a conditional field, such as Category, and you are presented with a list of available operators. You can select, for example: [Category] [is not] [Software].
  5. When you finish defining the filter, tap Run.

4.3.1 Viewing Journal Streams

A journal stream is a real-time, sequential display of journal field entries, such as comments and work notes, from a group of records. You can, for example, view a stream of incident work notes. The stream automatically aggregates all of the journal field entries from incidents and displays them, starting with the most recently updated record.

  1. Navigate to a list of records that contain journal fields.
  2. Tap the Stream icon. If no journal entries have been created for the type of records you are viewing, the Stream icon is not displayed.


    Comments and work notes for the incidents are shown, starting with the most recent record.

4.4 Viewing Forms

Many common forms have a Mobile view. The smartphone interface opens forms in this view by default. If a Mobile view is not defined for a form, the smartphone interface opens the Default view of the form. These views can be modified in the desktop interface.

On the form, tap a field to edit the value. Tap the arrow icon by a reference field to preview the record. You cannot edit fields in a record preview. Client scripts and UI policies do not work in the preview.

If the form contains attachments, they appear below all the fields (starting with the Fuji release). Tap an attachment name to open the attachment. If you are using the ServiceNow app, the attachment opens in Safari.

4.4.1 Viewing Comments and Work Notes

To view comments and work notes on a form, tap the Comments tab. Work notes are highlighted yellow.

Enter text, then tap the Comment or Work Note button to post.

Note: UI policies and client scripts related to comments and work notes do not work as expected in the smartphone interface. Use data policies or business rules instead.


4.4.2 Viewing Related Lists

Related lists that were created for records in the desktop interface are visible in the smartphone interface (starting with the Eureka release). To view a related list on a form, tap the Related List tab.

4.5 Adding ServiceNow to the Home Screen

You can add ServiceNow to the Home screen on an iPhone:

  1. Navigate to a page that you want to access as a shortcut, such as a list of open incidents.
  2. Tap the sharing icon at the bottom of the Safari browser.
  3. Tap the ServiceNow Add to Home Screen icon.
  4. Enter a descriptive name for the page and tap Add. The shortcut is saved to your device.
