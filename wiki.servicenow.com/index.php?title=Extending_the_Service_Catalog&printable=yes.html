Extending the Service Catalog

From ServiceNow Wiki
Home > Deliver > IT Services > Service Catalog Management > Extending the Service Catalog
Jump to: navigation, search


Overview

The standard service catalog and request fulfillment setup can be extended with a variety of features, to provide more flexible design and more powerful features for service catalog offerings.

Extended Catalog Item Functions

The basic service catalog item functions can be extended, with functions including:

  • Order guides to group multiple catalog items in one request.
  • Record producers: giving alternative ways of adding information such as Incidents via the service catalog.
  • Content Items: catalog items which provide information instead of goods or services.

Extended Variable Usage

The basic service catalog variable functions can be extended, via functions including:

  • Variable sets: group sets of variables together, for easy re-use.
  • Scripting: scripts reference any variables used by a request item, from any table.
  • Reporting options: administrators can create a report with variables and request items or tasks.


Service Catalog Configuration

The service catalog can be configured to provide more advanced functions, including:

  • Service Catalog Access Controls: defining access to the catalog.
  • UI Policies: to define custom process flows for tasks and catalog items.
  • Renderers allow administrators to define how categories are displayed on the service catalog homepage.
  • Data Lookup: to configure data lookup and record matching for the service catalog.

Using Properties

Service catalog properties are available, to configure:

  • Category appearance: how categories are displayed to users. For example, display subcategories in a panel or as a list in the category view.
  • Homepage content types and search and navigation elements. For example, enable or disable service catalog searching for all users.
  • Catalog item behavior: how items, item class names and item categories appear to users. For example, define the number of catalog items or categories to preview in a section.

Configuring the Checkout Process

The basic service catalog checkout can be configured and extended, with functions including:

  • Enabling a Two-Step Checkout allowing review of ordered items before purchase.
  • Enabling Bulk Requests: enabling the same service catalog request to be made, for multiple users.
  • Flexible Checkout and Delivery Forms offer more control over content within template-driven forms.

Using Service Catalog Scripting

Some scripting functions are available, specifically for service catalog:

  • Client-side scripting: allows catalog designers to use the same functionality available on other forms.
  • The Service Catalog Script API makes catalog ordering easier from inside business rules.
  • Scriptable Assignment of Execution Plans allows selection of an execution plan at the time of ordering, based on scripted conditions.
