Personalizing Mobile Lists

1 Overview

In the legacy mobile UI, mobile lists can be customized using views such that the fields that are most appropriate for your instance will be displayed. Typically these views are named Mobile.

2 Field Labels

You can show field labels when appropriate in views named Mobile. For example, if you are displaying one or more date fields on a list it might not be clear exactly which date is displayed. The label can be used to identify the field.

To add a field label, switch to the Mobile view and configure the list. Create a view named Mobile if you do not already have one for this list. A new button Add with label appears above the normal Add button in the slushbucket. Select the field which you want to add with a label and click the Add with label button. Note that the field is listed with an asterisk before its name, which indicates it will be displayed with a label.

3 Splits

You can use splits to display more than one field on a line. Normally, due to the screen size limitations of mobile devices, only one field is displayed per line. However, sometimes it is useful to show more than one field per line. A good example might be showing the incident number and category on the first line.

To add a split, switch to the Mobile view and configure the list. Create a view named Mobile if you do not already have one for this list. At the bottom of the Available column there are entries for |- begin split -| and |- end split -|. Add both the begin and end split and position them so that the fields to appear together appear between the begin and end splits.

If you would like to include multiple splits, that is, group multiple lines, turn on the multiple split property.
