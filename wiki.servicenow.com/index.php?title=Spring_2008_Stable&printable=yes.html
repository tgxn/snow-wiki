1 Diagnostics Page

The diagnostics page (<instance>/system_diagnostic_page.do) is a UI Page designed to provide a simple overview of useful information about a running instance and any cluster nodes. You can control when things turn red on the page with the Diagnostics Properties. It is another step towards providing useful troubleshooting information to customers.


For more information, see the Diagnostics article.

2 Transaction Data

2.1 Log

You can see transaction information by navigating to System Logs > Transaction Log. For each completed transaction, the following information is available (times are in milliseconds):

  • Date/time, User ID, IP address and URL of the transaction
  • Total response time
  • Network time (network transmission time, both from and to the user)
  • SQL time (time spent executing SQL commands)
  • SQL count (number of SQL commands executed)
  • Business rule time (time spent processing business rules)
  • Business rule count (number of business rules executed)

2.2 Form

Transaction response data at the bottom of lists/forms now contains the following:

  • Total transaction time, broken down into
    • Network time
    • Browser time
    • Server time

3 System Mailboxes (Email Logs)

Sent and received email can be seen in the System Mailboxes application, where you can view the Inbox, Outbox, and Sent mailboxes. Each provides a filtered view of emails to let you see only what you need to at any given time, including list fields applicable to each. From the left navigation pane, select System Mailboxes > Status to see your current POP and SMTP status.

The System Mailbox is hosted by ServiceNow, who have sole access to the mailboxes. By default, once the ServiceNow application has pulled an email message, it will be deleted from the mail server and stored in the application on the sys_email table.

3.1 Mail Delivery

  • Inbound emails: All inbound mail is placed into the Inbox until it is processed. Once cleared, it moves to Received (if processed) or Junk (if it is ignored by your Inbound Email Actions). If the ServiceNow system is restarted for any reason (such as during a system upgrade), all inbound mail waits on the external mail server until ServiceNow can request delivery.
  • Outbound emails: All outbound mail is placed into the Outbox until it is processed. Once cleared, it is moved to Sent (if sent) or Skipped (not sent, as in the case of no valid recipients). If the ServiceNow system is restarted for any reason (such as during a system upgrade), all outbound mail waits in the ServiceNow database until ServiceNow comes online, and the scheduler looks for mail to deliver.

4 LDAP Enhancements

 Functionality described here requires the System Import Sets plugin.

4.1 LDAP Servers

You can define one or more LDAP servers by navigating to System LDAP > LDAP Servers. An inactive example is provided. Once you've provided the necessary information for a server, you can test the connection and browse the LDAP tree right from the server record, using related links at the bottom of the form.


If you set target.ldap_server in your LDAP transform map (see the out-of-box "Example LDAP User Import" transform map by navigating to System Import Sets > Transform Maps for an example), you'll know what LDAP server created or updated a given record. Here is the line in the out-of-box transform map that does this: 

target.ldap_server = source.sys_import_set.data_source.ldap_target.server;

4.2 LDAP OU Definitions (for users, groups, etc.)

Once you've set up an LDAP server, you need to define what records you want from it. Navigate to System LDAP > LDAP Servers, and open an existing server record to set up its OU Definitions in a related list. See the out-of-box "Example LDAP Server" for examples.

4.3 Data Sources

Data sources are used to create an import set from the data pulled from your LDAP server, so that data can be processed if necessary prior to being mapped onto a production table.

4.4 Transform Maps

Transform maps take the data in your import sets (already imported from your LDAP servers) and map it to tables instance (users, groups, etc.). Two LDAP transform maps are provided out-of-box and can be modified as needed:

  • Example LDAP User Import
  • Example LDAP Group Import

4.5 LDAP Log

Navigate to System LDAP > LDAP Log to see all available log messages relevant to an instance's LDAP integration. These messages may include:

  • Connectivity errors
  • Authentication errors
  • Timeout warnings

4.6 LDAP Certificates

Upload your X.509 certificate for secure LDAP connection by navigating to System LDAP > X.509 Certificates. The following formats are accepted:

  • DER
  • PEM (a base-64 encoded DER certificate, enclosed between "-----BEGIN CERTIFICATE-----" and "-----END CERTIFICATE-----"

To upload the certificate, create a new record and then attach the certificate to the record as a file attachment. Information about the certificate will be extracted and displayed in the record, so you can see the following information:

  • Valid from date
  • Expiration date
  • Issuer
  • Subject of the certificate
  • (PEM only) the base-64 encoded string from the certificate

5 Excel Export Threshold

Excel exports are intended for relatively small exports (less than 500,000 cells), while CSV can handle larger exports.


If you export to Excel and the resultant spreadsheet has more than 500,000 cells (by default), the export process stops and you are given the Excel file at that point. In the bottom row there will be the following message: Export stopped due to excessive size. Use CSV for a complete export:


The Excel export cell threshold is customizable using the glide.excel.max_cells integer property. Note: Increasing this threshold may cause a memory issue in your instance. The threshold is set at an appropriate level to prevent resource issues.

6 UTF-7 encoding support for incoming emails

Some email clients, (e.g., Hotmail) still send in UTF-7 encoded format. ServiceNow can process these inbound emails.

Note: UTF-7 is no longer supported as of the Eureka release.


7 Resource use enhancements

Significant improvements have been made to reduce resource use in circumstances which previously might have impacted availability.

8 Module icons

Default module icons have been given a facelift.

9 Advanced reference qualifiers for Service Catalog variables

(since 15 July 2008)

Service Catalog reference variables can use advanced reference qualifiers. Using advanced reference qualifiers, you can use variable values on the ordered item to generate the reference qualifier query string for a reference variable on the same item. For example, you could create a "Location" reference variable with a reference qualifier that restricts available locations to ones associated with groups the Requested For users is in.


In the Javascript function called by the advanced reference qualifier, "current" is in scope, which is the sc_cart_item record for the item being ordered. So you have access to information available through the current sc_cart_item record. For example:

  • Variable information for the ordered item is available using "current.variables.variablename"
  • The Requested For individual is current.cart.requested_for


Following is an example script of a function that defines an advanced reference qualifier for a sys_user reference variable, that filters the available sys_user records to those where Name contains the value of a variable named "textmatch" on the same item: 

 function getBlackberryUsers() {
   var answer = ' ';
   var includes = current.variables.textmatch;
   var usr = new GlideRecord('sys_user');
   usr.addQuery('name','CONTAINS',includes);
   usr.query();
   while (usr.next()) {
     if (answer.length > 0) {
       answer += (',' + usr.sys_id);
     } else {
       answer = usr.sys_id;
     }
   }
   return 'sys_idIN' + answer;
 }

See Reference Qualifiers for additional information on reference qualifiers.

10 Oracle performance improvements

We have changed our Oracle indexing approach, improving performance significantly. The end result is approximately 40% faster in general (40% more transactions can be done in the same timeframe), and some transactions are up to 300% faster. At the same time, average CPU load is lower.

11 Map Pages

You can define a map page that can graphically display locations of things like companies, locations, critical incidents, customers, etc. Navigate to System UI > Map Pages to see out-of-box map pages.

12 Export resource use enhancements

(since 25 July 2008) The export code no longer queues any exports in memory buffers. It now provides the underlying export generator (Excel, CSV, PDF, XML) with an output stream. Those generators then wite directly into that stream. The stream, in turn, writes through into the attachment table in realtime. This gives the various export generators the opportunity to avoid memory buffering their complete output, reducing the amount of memory necessary for the export in the case of a large export.


XML exports now go through the same background generation mechanism as other exports, allowing for reduced resource use in the case of a large export.

13 GlideElementCompressed type

(since 25 July 2008) Large, redundant strings (e.g., those stored in sys_db_cache, lucene, jrobin tables) are now stored as a compressed, base64encoded string to use less database storage. Data stored in this manner cannot be reported against easily, so this storage type is only used where this is not a hindrance.

15 Restrict homepage "Add content" using roles

A new property, glide.home.add_content_role, is a comma-separated list of roles that can add content to homepages. This property is found by navigating to System Properties > Homepage. If the property is blank, all users with a role (any role) can add content to their homepage. If the property is used, users without one of the comma-separated roles will not be able to:

  • Add content to a homepage
  • Create a new homepage (since they would not be able to add content to it)
  • Remove gauges from an existing homepage (since they would not be able to add them back)

16 Remove the email box from a glide_list (like "Watch list")

By default, glide_list fields referencing sys_user, like the out-of-box Watch list field, have an email text box where you can add an email to the watch list. To remove this email box, personalize the field and add the "no_email" attribute.

17 New dictionary table attributes for high activity tables

Two new table attributes, no_attachments and no_audit_delete, can be used for high activity tables to reduce system workload on those tables. Documented in the Dictionary Attributes article.

18 CMDB Relationship UI

The CMDB Relationship UI, where you can add/remove CMDB relationships, has been improved to make it more intuitive and easy to use. The idea is that no documentation should be necessary. From a CMDB item with eh CI Relations formatter on the form, click the green + sign to add new CMDB relationships for your CI.
