1 Overview

The SLA Contract Add-on extends the existing SLA functionality by introducing a Contract as the master document that houses all appropriate data needed to drive your task's SLA processing . This new functionality allows businesses with numerous SLA, CI, location, user, group, and contract combinations to better manage their systems.

2 Contracts

A new table of SLA Contracts is provided through the Service Level Management > Service Contracts module. These Contracts group together SLAs that relate to a single vendor or customer, as well as the CIs, locations, groups, users, and child contracts that are related to the contract:


3 Getting Started with the SLA Contract Add-on Plugin

The following steps are required to get started with the SLA Contract Add-on Plugin:

  • Activating the Plugin
  • Defining the Service Contract
  • Adding Related Items
  • Adding the Contract Field to Forms

3.1 Activating the Plugin

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.

3.2 Defining the Service Contract

The first step in using the SLA Contract Add-on Plugin is to define master contracts. For information on how, see Defining the Service Contract.

3.3 Adding Related Items

Once the service contracts are defined, associate the related items, including the SLAs that the contract uses and the users, groups, locations, and CIs that the contract apply to. For more information, see Adding Related Items.

3.4 Adding the Contract Field to Task Forms

Finally, depending on how the contract is being used, the Contract field can be added to any task form by configuring the form. As the Caller, Assignment Group, Location, and Configuration item fields are populated, the Contract field filters appropriately, helping the support desk to associate the ticket to the correct Contract.

4 Using the SLA Contract Add-on Plugin

When a task is created, the Contract field will filter based on the Caller, Location, Configuration Item, and Assignment Group fields. Once the 'Contract' is selected and the incident is submitted, the system will restrict its SLA search to only those associated with the appropriate contract.

5 Using Contract SLAs by Default

The property com.snc.sla.contract.tables contains a comma-separated list of tables that will use Contract SLAs by default. The default value is fsm_service_order.

6 Disabling the SLA Contract Add-on Plugin

To disable the SLA Contract Add-on Plugin, navigate to sys_properties.list, locate the property com.snc.sla.contract.tables, and blank out the Value field. This removes any table from using the SLA contract add-on. To reverse this, restore the comma-separated list of tables to the Value field.
