Financial Management


1 Overview

The following components are installed with Financial Management:

  • Tables
  • Plugins
  • User Roles
  • UI Policies
  • Script Includes
  • Scheduled Jobs
  • Client Scripts
  • Business Rules

Demo data is available with Financial Management. The demo data provides sample chart of accounts information, allocations, and expenses.

2 Tables

Financial Management adds the following tables.

Table Table Content
Bucket

[itfm_bucket]

Buckets used in the allocation workbench to group similar expenses.
Business Unit

[business_unit]

All business units.
COA Segment

[itfm_coa_segment]

Segments defined in the General Ledger Chart of Accounts.
Cost Allocation

[itfm_cost_allocation]

All allocation lines that are processed from allocations.
Cost Allocation Group

[itfm_cost_allocation_group]

Groups that can be associated with allocations.
Cost Allocation Method

[itfm_cost_allocation_method]

Cost allocation methods that are used by allocation rules.
Cost Allocation Metric

[itfm_cost_allocation_metric]

Metrics that weighted and custom allocation methods use.
Cost Allocation Preview

[itfm_cost_allocation_preview]

Allocations lines that are created when using the allocation previewer.
Cost Allocation Rule

[itfm_cost_allocation_rule]

Cost allocations rules that are used to break down expenses during allocation.
Filter Condition

[itfm_filter_condition]

Filters used by allocation rules.
General Ledger Cleansed Data

[itfm_gl_data_cleansed]

Expenses in the general ledger that is cleansed.
General Ledger Staged Data

[itfm_gl_data_staged]

Imported expenses that have not been cleansed or groomed.
Groomed General Ledger Data

[itfm_gl_data_groomed]

Expenses in the general ledger that have been groomed.
IT Shared Service

[itfm_shared_service]

Services shared across the IT infrastructure.
IT Shared Service Type

[itfm_shared_service_type]

Categories that are associated with IT shared services.
ITCOA Definition

[itfm_itcoa_definition]

Segments defined in the IT Chart of Accounts.
ITFM Bucket Allocation

[itfm_bucket_allocation]

Allocations of the expenses in buckets to specific accounts in the IT chart of accounts.
ITFM Bucket Allocation Account

[itfm_bucket_allocation_account]

Amounts of the allocations of the expenses in buckets to specific accounts in the IT chart of accounts.
ITFM Bucket Rule Map

[itfm_bucket_rule_map]

Buckets that cause the creation of allocation rules in the workbench.
ITFM Budget

[itfm_budget]

Budgets used in the CIO dashboard.
ITFM COA Relations

[itfm_coa_relations]

Relationships between segments established by rollups in the Allocation Setup stage of the workbench.
ITFM Grooming Condition

[itfm_grooming_condition]

Conditions for grooming data.
ITFM Split Bucket

[itfm_split_bucket]

Sub-buckets that belong to buckets.
Segment Relationship

[itfm_itcoa_hierarchy]

Relationships between segments in the segment hierarchy as defined in the Data Definition stage of the workbench.

The application also uses this table created by the Fiscal Calendar.

Table Table Content
Fiscal period

[fiscal_period]

Fiscal calendars and the periods used by fiscal calendars.

The application also uses these tables created by the Data Mart.

Table Table Content
DataMart dimension

[dm_dimension]

Dimension records copied from transactional tables for each segment in the IT chart of accounts.
DataMart fact

[dm_fact]

Fact records copied from transactional tables for each segment in the IT chart of accounts. This table is not used in the Fuji release.
Dimension Last Run

[dm_dimension_last_run]

The number of records created or updated in a dimension table when it is repopulated by the data mart administrator.
Dimension Table Definition

[dm_dimension_table_def]

Dimension definitions that relate a dimension table to a transaction table.
Fact Definition

[dm_fact_table_def]

Not used in the Fuji release.
Fact Dimension Definition

[dm_fact_field_def]

Not used in the Fuji release.

In addition to these data mart tables, the application also creates tables for the nine default segments in the IT chart of accounts and for buckets, which are used to group expenses. See Default Fact and Dimension Definitions for a list of these tables.

3 Plugins

Financial Management activates the following plugins.

Plugin Name Plugin ID Description
Project Portfolio Suite com.snc.project_portfolio_suite The Project Portfolio Suite (PPS) plugin activates an integrated set of applications for project portfolio management and IT software development.
IT Data Mart com.snc.it_data_mart The data mart used to dimension transactional tables.
Fiscal Calendar com.snc.fiscal_calendar Fiscal calendars used by the Financial Management application.

4 User Roles

Financial Management adds or uses the following user roles.

Role Description Contains Roles
financial_mgmt_admin Has access to all Financial Management application menus and modules.
  • financial_mgmt_user
  • financial_analyst
financial_mgmt_user Has read-only access to general ledger and allocation records.
  • none
financial_analyst Has access to the workbench to allocate expenses, and can also modify rules, methods, and metrics. Can also create, edit, and delete finance reports.
  • financial_mgmt_user
datamart_admin Has access to all Data Mart application menus and modules.
  • none

5 UI Policies

Financial Management adds the following UI policies.

Name Table Description
Hide order field and remove mandatory Cost Allocation Method

[itfm_cost_allocation_method]

Hides the Order field and the Mandatory condition when the allocation method's amount type is a percentage.
Hide primary segment Cost Allocation Method

[itfm_cost_allocation_method]

Hides the Primary segment field on the Cost Allocation Method form if an allocation metric is selected or if the Unallocated field is selected.
Hide segments if unallocated Cost Allocation Method

[itfm_cost_allocation_method]

Hides the segments from the IT chart of accounts on the Cost Allocation Method form if the Unallocated field is selected.
Hide Unallocated checkbox when a metric is selected Cost Allocation Method

[itfm_cost_allocation_method]

Hides the Unallocated field on the Cost Allocation Method form if an allocation metric is selected.
Hide weight reference and aggregate fields Cost Allocation Metric

[itfm_cost_allocation_metric]

Hides the reference and aggregate fields when the metric is not weighted.
Make filter condition table field read-only Cost Allocation Rule

[itfm_cost_allocation_rule]

Changes the Table field to read-only so the user cannot change it to another table.
Show aggregate field Cost Allocation Metric

[itfm_cost_allocation_metric]

Displays the aggregate field options when the user selects any aggregate other than Count.
Show fields for cycle properties Cost Allocation Metric

[itfm_cost_allocation_metric]

Shows additional fields when the Enforce relationship option is selected.
Show fields for direct reference "from" relationships Cost Allocation Metric

[itfm_cost_allocation_metric]

Displays the fields on the weighted metric form to configure allocations from records in a selected table.
Show fields for direct reference "to" relationships Cost Allocation Metric

[itfm_cost_allocation_metric]

Displays the fields on the weighted metric form to configure allocations to records in a selected table.
Show fields for intermediate table relationship Cost Allocation Metric

[itfm_cost_allocation_metric]

Displays the fields on the weighted metric form to configure allocations based on an intermediate table.
Show parent segment Segment Relationship

[itfm_itcoa_hierarchy]

Hides the Parent segment field when the segment is the root in the hierarchy.
Show weighted fields Cost Allocation Metric

[itfm_cost_allocation_metric]

Shows fields that are specific to the weighted metric if the metric type is Weighted.


The Data Mart application adds the following UI policies.

Name Table Description
Hide fact fields Dimension Table Definition

[dm_dimension_table_def]

Hides the fact fields, which are not used by the Financial Management application.
Hide field type Dimension Table Definition

[dm_dimension_table_def]

Hides the type of field when the Table field has a value.
Hide table and merge fields for system generated dimensions Dimension Table Definition

[dm_dimension_table_def]

Hides the Table field and Merge dimensions option for dimensions that the system generates automatically.
Show dimension table name when Merge dimensions is selected Dimension Table Definition

[dm_dimension_table_def]

Shows the Dimension table field when the user selects the Merge dimensions option.

6 Script Includes

Financial Management adds the following script includes.

Name Description
FinancialMethodUtils Provides functions for creating allocation methods. Customers can modify this class to add allocation methods.
FinancialMgmt Provides additional functions that the application uses for the chart of accounts, the general ledger, and so on.
FinancialMgmtAJAX Contains AJAX APIs.
itfm_AbstractWrapper Wraps GlideRecord objects related to the application.
itfm_BaseFactory Provides application access to the layered architecture.
ITFM_CIO_Dashboard Provides application interface with the CIO Dashboard.
itfm_Constants Provides application access to the layered architecture.
itfm_Factory Provides a customizable class providing the correct script wrapper type. Customers can modify this class to add wrapper implementations.
itfm_GLDataStaged Provides access to the General Ledger Staged Data table for the general ledger, where financial data is imported.
itfm_GLDataStagedAjax Contains AJAX APIs for the General Ledger Staged Data table.
ITFM_RulesBridge Provides methods that the application uses to manipulate allocation rules and the buckets, segments, and accounts associated with them.
ITFM_RulesBridge_MetricBuilder Returns the segment in the IT chart of accounts that is required by other scripts.
ITFM_ToolBox Provides methods that the application uses to get data from elements of the cost model.
ITFM_Workbench Provides APIs for access to the workbench.
ITFM_Workbench_DataDefinition Provides APIs for access to the segment hierarchy and the advanced actions on the Data Definition page of the workbench.
ITFM_ExportChangeRequest Exports Financial Management application records to XML.

7 Client Scripts

Financial Management adds the following client scripts.

Name Table Description
Autofill group from metric Cost Allocation Method [itfm_cost_allocation_method] Populates the Allocation group field when the allocation metric is set.
Clear metric on group change Cost Allocation Method [itfm_cost_allocation_method] Clears the Allocation metric field on the allocation method form when the allocation group changes.
Clear subbucket on bucket change Cost Allocation Rule [itfm_cost_allocation_rule] Clears the Sub bucket field on the Cost Allocation Rule form when the value for the Bucket field is changed.
Extend Business Service cost query Business Service [cmdb_ci_service] Determines if the business service allocations can be shown on a report. Shows an error message if the allocations cannot be shown.
Hide empty segments Cost Allocation [itfm_cost_allocation] and

Cost Allocation Preview [itfm_cost_allocation_preview]

Hides the segments that do not have any data.
Increment method order Cost Allocation Method [cost_allocation_method] Increments the value for the Order field by 10 when a new method is created.
No UI Grooming conditions modification ITFM Grooming Condition [itfm_grooming_condition] Prevents the modification of grooming conditions
personalized script Company [core_company],

Portfolio [pm_portfolio], and Project [pm_project]

Determines if the company, portfolio, or project allocations can be shown on a report. Shows an error message if the allocations cannot be shown.
Show only valid segments Cost Allocation Rule [itfm_cost_allocation_rule] Shows the valid segments on the method form.
Show/hide bucket related list Bucket [itfm_bucket] Controls the appearance of the sub-bucket related lists for buckets.
Update allocate to reference fields Cost Allocation Metric [itfm_cost_allocation_metric] Updates the choices available for the Reference field choice list field to the values in the table selected for the Allocate to field.
Update intermediary table fields Cost Allocation Metric [itfm_cost_allocation_metric] Updates the allocation fields for a metric based on the tables selected for the Intermediate table.
Update weight reference field Cost Allocation Metric [itfm_cost_allocation_metric] Updates the choices available for the Reference field choice list field to the values in the table selected for the Weight table field.
Weighted initialization Cost Allocation Metric [itfm_cost_allocation_metric] Checks if the allocation metric is a weighted metric.

8 Business Rules

Financial Management adds the following business rules.

Name Table Description
Cascade delete conditions on a bucket Bucket [itfm_bucket] Verifies if a bucket is used for an allocation line.
Check max rule order Cost Allocation Rule [itfm_cost_allocation_rule] Verifies that a no two cost allocation rules have the same value in the Order field.
Check method order is unique Cost Allocation Method [itfm_cost_allocation_method] Prevents methods from having the same order number.
Check Overlap When Update Fiscal Period Fiscal period [fiscal_period] Verifies that fiscal periods do not overlap.
Check percentages Cost Allocation Rule [itfm_cost_allocation_rule] Checks if the sum of the amounts for manually divided cost allocation methods that use percentage calculations is equal to 100.
Check rule order is unique Cost Allocation Rule [itfm_cost_allocation_rule] Verifies that a no two cost allocation rules have the same value in the Order field.
Check sub bucket Cost Allocation Rule [itfm_cost_allocation_rule] Verifies that a sub bucket has a parent.
Check sum of percentages Cost Allocation Method [itfm_cost_allocation_method] Verifies that percentage-type methods sum up to 100 percent within each rule.
Child should be excluded if parent is Bucket [itfm_bucket] Excludes a sub-bucket from the cost model if the parent bucket is excluded.
Clear segments when unallocated is true Cost Allocation Method [itfm_cost_allocation_method] Hides the Primary segment and dimension fields on the Cost Allocation Method form when the value of the Unallocated field is true.
Delete associated workbench rule data Cost Allocation Rule [itfm_cost_allocation_rule] Deletes the rules created in the workbench when buckets in the ITFM Bucket Allocation Rule Map are deleted.
Delete cleansed row on update of staged data General Ledger Staged Data [itfm_gl_data_staged] Deletes records in the General Ledger Cleansed Data table when the corresponding record is deleted in the General Ledger Staged Data table.
Forbid adding conds of the same priority ITFM Grooming Condition [itfm_grooming_condition] Prevents grooming conditions from having the same priority.
grooming_priority_auto_populate ITFM Grooming Condition [itfm_grooming_condition] Increments the priority of grooming conditions by 10.
Increment rule order Cost Allocation Rule [itfm_cost_allocation_rule] Increments the order value of new cost allocations rules by 10.
itfm_prevent_empty_advanced_condition ITFM Grooming Condition [itfm_grooming_condition] Verifies that advanced bucket conditions have valid values.
ITFM Bucket Naming Rule Bucket [itfm_bucket] Verifies that bucket contains a valid name.
Prevent deletion of non-leaf segments Segment Relationship [itfm_itcoa_hierarchy] Prevents the deletion of segments that have child segments under them in the hierarchy.
Prevent having parent and children Bucket [itfm_bucket] Prevents a parent bucket from having a value in the parent field.
Prevents adding children to grooming con Bucket [itfm_bucket] Prevents adding a sub-bucket to a bucket that already has a grooming condition.
Process root node Segment Relationship [itfm_itcoa_hierarchy] Sets a segment as the root node, which is the only segment allowed at the top of the hierarchy.
Process the Child segment Segment Relationship [itfm_itcoa_hierarchy] Sets a segment as a child segment of other segments.
Process the Parent segment Segment Relationship [itfm_itcoa_hierarchy] Sets a segment as a parent segment of other segments.
Sync between staged and cleansed tables General Ledger Staged Data [itfm_gl_data_staged] Synchronizes the records in the General Ledger Cleansed Data table with the new and modified records in the General Ledger Staged Data table.
Sync Date_time and Schedule Fields Fiscal period [fiscal_period] Synchronizes the start and end dates on the Fiscal Period form with schedule fields on the same form, which are used for reporting.
Update Dictionary Entries ITCOA Definition [itfm_itcoa_definition] and Dynamic Filter Options [sys_filter_option_dynamic]. Updates segments upon changes.
Validate Percentage Cost Allocation Method [itfm_cost_allocation_method] Displays a message to the user that percentage values for each method must be between 0 and 100.
Verify primary segment has a value Cost Allocation Method [itfm_cost_allocation_method] Verifies that an Object ID matches at least one segment.

Populate IT Data Mart dimension tables Populates the dimension tables in the data mart every 24 hours.
