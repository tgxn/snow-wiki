1 Overview

ServiceNow record lists provide a wide range of controls for sorting, filtering, moving, and editing data. List controls enable a user to manipulate and view data and records from within a list without having to open individual records. Information about lists include the administrative functions for controlling the presentation, management of lists and the list controls available to end users for searching and managing the data they are permitted to view.

2 Administering Lists

Administrators control the data presented to end users in a list, the controls that are available, and the behavior of those controls. See the following topics for information about the administrative functions for managing lists.

  • Configuring Lists: Customize the columns in any list to show only the desired information to all users. Add or remove fields (columns) from a list or change the order in which the fields appear in the list. Hide controls and define access conditions by role to existing controls.
  • Controlling the Sort Sequence Used to Display Lists: These controls determine how lists are sorted when they are presented to users and if user-specified sort orders are remembered by the system.
  • Context Menus: Control the content and behavior of list context menus for all users.
  • Administering the List Editor: The List Editor enables users to modify data directly in the list view without having to open a report. This privilege can be restricted by an administrator.
  • Administering Personal Lists: Administrators manage the controls end users can access for personalizing their own layout for any list view.
  • Restricting Filters and Breadcrumbs: Administrators can limit access to parts of a table by restricting active links in breadcrumbs or by suppressing breadcrumbs and filters altogether for specific roles.
  • Hierarchical Lists: Display Related Lists directly from a record list instead of having to drill into the form.
  • Embedded Lists: Embed lists directly into forms so that changes to the contents of the list are saved with the form.
  • Context Ranking: Context Ranking allows a user to sort a collection of records preferentially, rather than using any attributes of those records.

3 End User List Controls

See these pages for instructions about controlling and navigating lists and editing data from within a list. All these features are controlled by the administrative steps in the previous section.

  • Finding Information in Lists: This page shows users how to find information in a list by searching, sorting, and grouping the list.
  • Editing Lists: The list editor allows users to edit field values directly from a list without opening to a record.
  • Using Filters and Breadcrumbs: Learn to use the filters and breadcrumbs provided on a list to display records in a useful manner.
  • Creating Personal Lists: Users can create personal lists that provide alternate views of a list according to their individual preferences. Views created here are available only to the user who created them.
  • Using Lists: Learn about the context menu options available to end users in lists.
