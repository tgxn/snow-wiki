1 Overview

These updates fix the following issues after the Winter 2011 release.

This release is no longer recommended for upgrades. If you want to upgrade an instance, consider a current version. For the Winter 2011 Update release:

Build date: 03-10-2011_1622
Build tag: glide-winter2011-01-21-2011
 Note: See how to determine the version on your instance.



1.1 PRB560050: A Null Pointer Exception occurred when the Encryption Support Plugin was first activated while sessions were active.

If context vectors were not initialized, a session could have an uninitialized vector when the plugin was loaded and then threw null pointer exception until that session ended.

1.2 PRB560028: Prevent potential database deadlock from occuring in rare circumstances.

This situation occurred when the database connection pool was stressed. It also happened when the database connection pool was full on the first lease retry attempt.

1.3 PRB560031: A module with no title generated JavaScript errors when a user typed in the navigation filter.

1.4 PRB560016: Errors were generated when users first ran Chat Desktop.

Social IT/Chat functionality was updated to correct errors in the scrolling capability within chat windows.

1.5 PRB560018: HTML that previously rendered properly in calendar reports quit working. This occurred when the opening and closing tags in the html were of different case (<div></DIV>).

Changes were made to disregard case in these HTML tags.

1.6 PRB560017: Issues arose with the ODBC Driver for instances that did not have the Database Views plugin installed.

A isValid() check was added when creating the sys_db_views, since some instances do not have this enabled.

1.7 PRB560013: An issue occured with the ODBC Driver for Database Views. If no tables were defined with Database Views, then an invalid schema was returned.

The platform was updated to check for valid tables.

1.8 PRB560010: When a customer had their own POP3 server set up, and the user name had an @ sign in it, the platform did not pass the whole user name when logging in to the POP3 server.

With this update, we no longer truncate POP3 email user names at the @ sign if using the glide.pop3.user property value. We only truncate when using the glide.email.user name which is shared with the SMTP sender.

1.9 PRB560004: The GlideForm.getReference return value changed when the reference field was blank, causing JavaScript errors in cases where a client script did not check the return value from getReference.

This patch ensures that the return value is the same as it was previously to ensure that client scripts continue to function without change.

1.10 PRB559997: Derived field labels did not work when the short_label attribute was specified. The following error was displayed on the Incident form: *Unknown field label for Caller.User ID

With this update, the short_labels field attribute once again behaves as expected.

1.11 PRB560073 (On-Call Rotation): A manually entered on-call coverage person would not get assigned incidents.

Manually entered on-call coverage now works as expected.

1.12 PRB560098 (Domain): Domain Numbering did not work properly after clearing the domain table.

After deleting all records in the domain table on an instance with the Domain Number Support plugin active, new domains can now be created.

1.13 PRB560114 (Notifications): The IMAP integration was displaying an error.

The email connections for IMAP were restored.

1.14 PRB559995 (List v2): The New button was missing from reference pop-ups.

The New button is now available as expected for the sys_ref_list view of a list.

1.15 PRB560243 (Web Services): Web Services import sets lost its embedded Fields list.

The embedded list was restored to the form.

1.16 PRB559989 (User Interface): Multi-line text fields saved in a template were concatenated into one line. On saving a template, all line endings were replaced by a comma.

Multi-lined text fields now display properly.

1.17 PRB559865 (Queries): Clicking the Insert button within a report that uses a very complex query can build a URL that is too large for Microsoft Internet Explorer.

The resulting URL from a complex query in a report is now an acceptable length for Internet Explorer.

1.18 PRB559900 (Reports): A star level (*) report_on ACL could prevent creation of new reports despite overriding ACLs on tables.

report_on ACLs now work as expected with row-level ACLs overriding a *-level ACL.

1.19 PRB043753 (Reports): Trend information was dropped from the filter when View Report was clicked from the sys_report form.

Trend information is now preserved when clicking the View Report UI Action.

1.20 PRB450829 (Reports): A user could view reports for other users or groups if they had sufficient roles to view the report and the correct URL

Security for reports created with viewing restrictions is now being enforced properly.

1.21 PRB560185 (Domains): Domain Separation causes the Database View to return unexpected results if there are more than one top level domain and LEFT JOIN is used.

When a query is run on a 'global domain, the expected data is returned.

1.22 PRB560368 (API): The list_id was missing from the edit list row function.

The list edit API was modified to expect an additional parameter when marking a row as deleted in an embedded list. This fix ensures that the additional parameter is now passed to the API to correct the mark row as deleted action.

1.23 PRB560374 (User Interface): Reference lists should not allow access to records through the pop-up window.

Drilling through to a record from a reference list pop-up is no longer allowed. The pop-up document icon was removed from the list.

1.24 PRB560088 (Notifications): Sent emails were not showing up in the activity logs of the records with which they were associated.

All emails associated with an incident are now appearing in the activity log of the appropriate record.

1.25 PRB560292 (Service Catalog): The pop-up description of a catalog item could be truncated by the page frame.

When users hover the cursor over a catalog item's icon, the entire description is visible.
