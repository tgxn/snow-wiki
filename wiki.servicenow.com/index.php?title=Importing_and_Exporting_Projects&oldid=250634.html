1 Overview

You can manage projects using both Microsoft Project and the ServiceNow Project application. Users with the project manager role can export ServiceNow projects and project tasks to Microsoft Project where they can be managed separately and then imported back into ServiceNow as needed.

You can import projects created in Microsoft Project 2003, 2007, or 2010 into the Project Management application. If a project created in Microsoft Project 2013 with multiple tasks and relationships is imported into the Project Management application, an incorrect lag value may get inserted. As a workaround, you can first save the project created in Microsoft Project 2013 with the Microsoft Project 2000-2003 or the Microsoft Project 2007 file format and then import the saved project into the Project Management application.

2 Importing Projects

If you import a Microsoft project into ServiceNow as a new project, a new record is created in the Project [pm_project] table, and tasks associated with the project are added to the Project Task [pm_project_task] table. Only the fields that are common or mapped between the applications are imported. Imported projects are brought into ServiceNow with both Priority and Risk set to Low.

If you import a Microsoft project into an existing ServiceNow project, ServiceNow checks the Text10 field in the top-level Microsoft Project task. If the Text10 field contains a recognizable ServiceNow sys_id, the project was previously exported from ServiceNow. In this case, the values from the Microsoft project overwrite the values for the ServiceNow project. For information on the values that are overwritten during an import, see Project Field Mapping.

If the Text10 field does not contain a recognizable sys_id, the Microsoft project is imported as new tasks within the selected ServiceNow project.

When you import a project into ServiceNow, project dependencies, relationships, and constraints that are not supported by ServiceNow are converted as follows:

  • Relationships and dependencies: ServiceNow only supports finish-to-start task dependencies. For example, if Task 1 is a predecessor of Task 2, then Task 1 must finish before Task 2 can start. Microsoft Project tasks with other types of dependencies, such as finish-to-finish dependencies, are imported as standalone tasks without any dependencies.
  • Time constraints: ServiceNow Project supports Start ASAP and Start on specific date constraints. ServiceNow sets the time constraint for imported tasks to Start on specific date if their Microsoft Project time constraint is Must Start On. All other imported Microsoft Project time constraints are set to Start ASAP.
 Note: For the Dublin release, resources are not supported when importing a Microsoft Project. For the Eureka release, resources are supported when importing a Microsoft Project if those resources already exist in a Service Now instance. The resource name in Microsoft Project should map to the name of the user in the Service Now instance.


2.1 Calendars and Schedules

The following calendar elements from Microsoft Project are not imported into ServiceNow project management:

  • Project calendars
  • User calendars
  • Schedules

The imported project uses the default schedule of a Monday to Friday workday from 8 A.M. to 5 P.M. with an hour break for lunch, starting with the v3 application (Dublin release).

The v2 application imports the project with a 24-hour calendar, which displays task durations incorrectly. To solve this, create a calendar in ServiceNow that reflects the one used by the original project, then apply it to the imported project in the ServiceNow Project form. See Using Schedules for more information.

2.2 Importing Microsoft Project Files

To import a Microsoft Project file:

  1. Navigate to one of the following:
    • Project > Projects > Import (Fuj)
    • Project > Administration > Import Project (Eureka and prior versions)
  2. Under Choose project to import, click Browse and find the Microsoft Project file.
  3. Do one of the following:
    • To import the Microsoft project as a new project, proceed to the next step.
    • To import the Microsoft project as a subset of an active, existing project or task, click the reference lookup icon () under Choose an existing project to import into and select a project or task. Only active projects appear in the list.
    Alternatively, type the project name in the field under Choose an existing project to import into and select the project from the suggested list.
  4. Click Import.
The Microsoft Project import screen

If a Microsoft Project file import fails, do one of the following:

  • Open the original file in OpenProj, save it as a Project 2003 or Project 2007 file, and try to import it again.
  • Open the original file in Microsoft Project to verify that the project works, save the project under a different name, and try to import it again.

4 Exporting Projects to Microsoft Project

If you are using Microsoft Project to manage project activities, you can export a ServiceNow project to XML format and import it into Microsoft Project. This feature is available starting with the Eureka release.

Users with the project manager role can export a ServiceNow project using:

  • The Export Project module
  • The Project form

Project managers can also export project tasks using the Project Task form

4.1 Using the Export Project Module

ServiceNow projects must use either the Project Management Schedule or the Default MS Project schedule before they can be exported.

  1. Navigate to one of the following:
    • Project > Projects > Export (Fuji)
    • Project > Administration > Export Project (Eureka and prior versions)
  2. Select a project in the Project to export field.
  3. Click Export.
    The project is exported to a folder on your system.
  4. Open Microsoft Project to import the exported project files. Refer to Microsoft product documentation for instructions.
Export to Microsoft Project form

4.2 Using the Project Form

ServiceNow projects must use either the Project Management Schedule or the Default MS Project schedule before they can be exported.

  1. Navigate to Project > Projects > All
  2. Open the project.
  3. Right-click the form header and select Export to MS Project from the context menu.
    The project is exported to a folder on your system.
  4. Open Microsoft Project to import the exported project files. Refer to Microsoft product documentation for instructions.

4.3 Exporting Project Tasks

The task being exported must be associated with a ServiceNow project that uses either the Project Management Schedule or the Default MS Project schedule.

  1. Navigate to Projects > Project > All.
  2. Open the project.
  3. Scroll to the Project Tasks related list and click a task number to open the Project Task form.
  4. Right-click the form header and select Export Task to MS Project from the context menu.
    The task is exported to a folder on your system.
  5. Open Microsoft Project to import the exported project task files. Refer to Microsoft product documentation for instructions.
