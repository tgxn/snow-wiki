 The current general availability release for the Berlin family is Berlin Patch 8 Hotfix 1.


1 Overview

The Berlin Hotfix 6 release provides fixes for the Berlin release. This release is no longer recommended for upgrades. If you want to upgrade an instance, consider a current version. For more information about how to upgrade your instance, see Requesting an Upgrade.

For the Berlin Hotfix 6 release:

Build date: 12-11-2012_2104
Build tag: glide-berlin-05-06-2012__patch0-hotfix6-10-30-2012
 Note: See how to determine the version on your instance.


2 New in This Release

Problem Description
PRB579317 The approval summarizer would not work when a Currency type field was on the form.
Click the plus for steps to reproduce
  1. Navigate to Change > New.
  2. Personalize the form to add a Currency type field.
  3. Open any change request record that has an Approvers related list.
  4. On any record in the Approvers related list, click the State column.


3 Included from Previous Releases

Berlin Hotfix 5

The Berlin Hotfix 5 release provides fixes for the Berlin release.

Problem Description
PRB574901 Deferred workflows (workflows in which Run after bus. rules run is set to true) were experiencing delays of up to 1 minute before completing. A message similar to the following was displayed in the log: Mutex wf_context_547b8a01bf1320005d1008f7ff073973 acquired after spinning 121 times.
Click the plus for steps to reproduce
  1. Create a workflow to run against all active change requests.
  2. In the properties of the workflow, be sure to set "Run After Bus. Rules Run" to TRUE.
    This makes the workflow a "deferred" workflow.
  3. Create a Change Request.
    Observe the latency of at least 1 minute. The following log entry appears:
    Log entries (notice time delay of 1 minute):
    2012-07-16 11:29:33 (492) http-4 44AA0601BF1320005D1008F7FF0739AA Spin waiting for mutex wf_context_547b8a01bf1320005d1008f7ff073973 to be freed
    2012-07-16 11:30:34 (489) http-4 44AA0601BF1320005D1008F7FF0739AA Mutex wf_context_547b8a01bf1320005d1008f7ff073973 acquired after spinning 121 times
PRB575050 Autocomplete would not display results in the email client.
View the steps to reproduce
  1. Enable the email client for the Incident table.
  2. Navigate to Incident > Open.
  3. Select an existing incident.
  4. Click the email icon.
  5. Enter a user name in the To: field.
  6. The email client does not display autocomplete matches.
PRB574223 A recursion in the email reader could cause a StackOverflowError.
PRB573851 Canceling a running transaction could cause a streaming resultset error within the cancel transaction.

Berlin Release


4 Summary

These release notes describe enhancements in the Berlin release, which include:

  • Asset Management: track assets in coordination with the CMDB and manage stockroom inventory.
  • Software Asset Management: manage and audit software licenses more effectively.
  • SDLC - Scrum Process: improve your ability to manage agile software projects in multi-team and multi-product environments.
  • Data Archiving: improve system performance by archiving data that is no longer accessed frequently.


This release is no longer recommended for upgrades. If you want to upgrade an instance, consider a current version. For more information about how to upgrade your instance, see Requesting an Upgrade. For the Berlin release:

Build date: 07-02-2012_2029
Build tag: glide-berlin-05-06-2012
 Note: See how to determine the version on your instance.

5 Deliver

This release provides new and enhanced process applications for delivering business services.

Feature Description Activated
Asset Management Major enhancements help you track assets in coordination with the CMDB and manage stockroom inventory. Features include defining asset bundles, enhanced product definitions, detailed status tracking, and automated stock rules. Automatic for all instances.
Software Asset Management This new application helps you manage and audit software licenses more effectively. Features include entitlements, suite definitions, enterprise licenses, subscription licenses, upgrading/downgrading licenses, usage-based licenses, software counters, responsive calculations, and software discovery models for normalization. Activate the Software Asset Management plugin.
IBM PVU Process Pack This new process pack enables you to manage software that is licensed under the IBM Processor Value Units licensing model. Activate the Software Asset Management IBM PVU Process Pack.
SDLC Scrum Process Pack Major enhancements to the SDLC Scrum functionality improve your ability to manage agile software projects in multi-team and multi-product environments. Features include planning and progress boards, preferential ranking capabilities, burn down charts, and standard roles such as product owner and scrum master. Activate the SDLC Scrum Process Pack plugin.
Field Service Management The transfer order process is significantly enhanced to provide better support for both Asset Management and Field Service Management. Automatic for instances running Field Service Management.
New Ticket Module A new feature lets users collect information before identifying the issue as an incident, problem, or change request. Activate the Domain Support - MSP Extensions plugin.
Coaching Loops This new application allows an experienced coach to provide feedback to a coachee to improve performance on critical events in a business process. Features include automatically generated assessments, customizable coaching disciplines and specific opportunities, and user-specific performance tracking reports. Request the Coaching Loops plugin.

6 Administer

This release provides new and enhanced ways to configure and manage your ServiceNow instance.

6.1 Content Management

Feature Description Activated
CMS Administration Page A new, comprehensive administration page is added to the CMS application. The page provides samples, code examples, and links to documentation articles for site administrators and site designers. Automatic for all instances.
CMS Div-Based Layouts New div-based layouts are added to support standards, code simplification, and layouts that adapt to different device resolutions. A new theme named Administration Theme - Charcoal is included as an example of CSS-driven, div-based layouts. Automatic for all instances.
CMS Block Tags New block tags provide an option for inserting blocks within other blocks, frames, or layouts. Automatic for all instances.
CMS Translation Support CMS menu strings are moved to the Translated Field [translated_field] table, where relevant, to reduce duplicate string generation. CMS menu strings are also streamlined to improve consistency. Automatic for all instances.

6.2 Core Configuration

Feature Description Activated
HTML Editor

Administrators can configure HTML Fields to use TinyMCE as the HTML editor instead of the legacy (htmlArea) editor. This new editor provides better stability and more editing functions.

Automatic for new instances. For existing instances, activate the TinyMCE HTML Field Editor.
Syntax Editor A new syntax editor replaces the JavaScript editor and provides more editing functions, line numbers, and real-time error indicators. Automatic for new instances. For existing instances, activate the Syntax Editor.
Zing Text Search Improvements
  • Zing now handles special cases of punctuation in search terms. On existing instances, administrators must enable punctuation handling. It is enabled by default for new instances.
  • When debugging text search problems, administrators can diagnose and fix text search cache errors.
 Automatic for all instances.
Preferential Ranking of Related Lists Users can now sort a related list in preferential order by dragging and dropping records manually. Initially, users can rank related lists of stories in the SDLC - Scrum Process Pack. Administrators can create ranking controls for other related lists. Request the Context Ranking plugin.
Canceling Navigation Actions Users can now cancel an application navigator action by clicking another application navigator link. This allows the user to cancel a slow loading transaction without waiting for the previous transaction to complete. Administrators can control this behavior. Automatic for all instances.
Schedule Entry Repeat Schedule entries can now repeat for portions of the week, weekly, monthly, and yearly. There are also new default schedules, including a U.S. holidays schedule. Automatic for all instances.
Displaying Metrics as Timelines Users can display some metrics as timelines from a Task record. You can define what metrics are available as timelines from the Metric Definition record. Automatic for all instances
Range Calculator Scripts You can specify a script include that calculates range restrictions and processes parent updates in a timeline page. Automatic for all instances
Timeline Sub Items You can specify child spans for a timeline to create hierarchal relationships between spans. Automatic for all instances

6.3 Discovery

Feature Description Activated
External Credential Storage Repository Integration Discovery can now retrieve credentials from external credential repositories rather than storing those credentials internally in the Credentials [discovery_credentials] table. Request the Discovery External Credential Storage plugin.
CIM-based Storage Systems Discovery Discovery now finds storage systems based on the Common Information Model (CIM) and can query a CIM server for information about VMware ESX servers. Automatic for instances running Discovery.
Discovering Software with the Software Asset Management Table Schema Discovery accommodates the table schema in the new Software Asset Management application. Administrators must configure Discovery to store software installation data correctly. Automatic for instances running Discovery.
Application Profile Discovery Administrators can use a preconfigured XML file to supply Discovery with information about custom applications. This information enables Discovery to update the CMDB with application version information and relationships that it cannot otherwise detect. See the example and the instructions for customizing data. Automatic for instances running Discovery.
VMware vCenter Data Collected Discovery now establishes relationships for datacenters, clusters, datastores, folders, and networks when it discovers a vCenter service running as a process on a Windows host. Automatic for instances running Discovery.

6.4 Domain Separation

Feature Description Activated
Domain Separation for MSPs Managed Service Providers (MSPs) can now set up domain separation more easily. Features include automatic domain assignment by company and a visual map of the domain hierarchy. Activate the Domain Support - MSP Extensions plugin
Domain Paths The new domain path option makes database queries more efficient in instances with numerous domains. Activate the Domain Support plugin.
Domain-Specific Application and Module Names Delegated administration allows administrators to set domain-specific application and module names. Automatic for all instances running the Domain Support plugin.

6.5 Managing Data

Feature Description Activated
Data Archiving A new application helps you improve system performance by archiving data that is no longer accessed frequently. Activate the Data Archiving plugin.
Auto-Complete Administrators now have an improved UI and additional configuration options for auto-complete functionality. Automatic for all instances.
Data Lookup and Record Matching Support Administrators can now define rules that automatically set one or more field values when certain conditions are met. Automatic for all instances.
System Clone A new system clone application allows administrators to clone an instance over a sub-production instance without impacting system availability. Instance clones are scheduled within the ServiceNow application and do not require a clone request through HI. Automatic for all instances.

6.6 Platform Performance

Feature Description Activated
Transaction Quotas Maintain system performance and health by setting quotas that automatically cancel excessively long transactions. Automatic for new instances. For existing instances, activate the Transaction Quotas plugin.
Display Limits for Journal Fields Administrators can now control the amount of text that a journal field displays, eliminating the need to load the entire journal field into memory. Automatic for all instances.
Homepage Splash Page A splash page is now available to improve the user experience with homepages. Administrators can configure the homepage splash page. Automatic for new instances. For existing instances, activate the Homepage Splash Page plugin.
Homepage Debugging Administrators can now turn on homepage debugging to display load times on homepage widgets. Automatic for all instances.

6.7 Runbook Automation

Feature Description Activated
Assigning Credentials to Orchestration Activities Administrators can now assign individual credentials to any activity in an Orchestration workflow or assign different credentials to each occurrence of the same activity type in an Orchestration workflow. Automatic for instances running Orchestration Automation.
New VMware Orchestration Activities New activities are available for use in a VMware Orchestration. Automatic for instances Orchestration Automation.
Configuring Virtual Servers The service catalog offers new configuration options for virtual servers. Automatic for instances running Orchestration Automation.
Provisioning Virtual Servers The workflow controlling approval and provisioning of server requests is now more efficient. Automatic for instances running Orchestration Automation.

6.8 Service Administration

Feature Description Activated
Email Filters Administrators can now configure specific email filtering preferences, including the option to prevent ServiceNow from creating incident records from emails containing vCalendar attachments. Activate the Email Filters plugin.
Email Notifications Administrators can create email notifications more easily with the enhanced Email Notifications form. Automatic for all instances.
Document Feeds Users can now work on tasks and other records through live feed. Activate the Live Feed Document plugin.
Inbound Email Actions Users who are locked out of an instance or whose accounts are inactive cannot update incidents via email. ServiceNow does not process inbound emails from these users. Automatic for all instances.

6.9 Workflow

Feature Description Activated
Variables and Subflows Administrators can now add input variables to service catalog workflows with the new property, glide.workflow.enable_input_variables. This property is enabled on new instances and disabled on upgraded instances. Automatic for all instances.
REST Message Module A new REST Message module enables workflow administrators to run REST messages from a MID Server on the internal network. A REST message can be used by a REST workflow activity to retrieve or manage data on a REST endpoint. Automatic for all instances.
REST Message Activity The new REST Message activity executes a REST function (POST, PUT, GET, or DELETE) on an endpoint and can be executed with a MID Server. Automatic for all instances.
Workflow Error Handling Workflow error handling detects and logs syntax errors found in the advanced scripts of selected activities. The platform now marks activities in an error state with a red banner and provides error details in pop-up box when users hover the cursor over the activity. New functionality enables a workflow designer to add an error exit to certain activities. Automatic for all instances.

7 Integrate

This release provides enhanced ways to integrate ServiceNow with third-party technologies.

Feature Description Activated
Direct Web Services Administrators can now set the WSDL target namespace to www.service-now.com/<table name> with the new property, glide.wsdl.definition.use_unique_namespace. When this property is true a table's direct web service WSDL uses a targetNamespace value of http://www.service-now.com/<table name>. When the property is false or not present, all tables use the same targetNamespace value of http://www.service-now.com. This property is enabled on new instances and disabled on upgraded instances. Automatic for all instances.
SAML 2.0 Update 1 A new plugin improves integration security by requiring additional checks against the SAML response, support for a signed single logout request, and the ability to specify how an Identity Provider authenticates a SAML request. Available by request.
