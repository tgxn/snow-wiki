Localized Term Localized Definition English Term English Definition
заказчик В подключаемом плагине Sales Force Automation заказчики представляют компании, являющиеся существующими или потенциальными клиентами. account In the Sales Force Automation plugin, accounts represent companies that are sales prospects or customers.
список управления доступом Список управления доступом определяет с помощью диапазонов IP-адресов, к каким устройствам можно получить доступ с помощью Discovery. Списки управления доступом можно использовать, чтобы устанавливать или обходить ограничения доступа к определенным системам. ACL (Access Control List) An access control list defines via a range of IP address which devices can be accessed and scanned using Discovery. These can be used to establish or bypass access restrictions on certain systems.
действие В плагине Sales Force Automation действия отслеживают все события, связанные с продажами: сообщений электронной почты, телефонные звонки и встречи. Действия можно сопоставить с контактом, с заказчиками или с возможностями. activity In the Sales Force Automation plugin, activities track sales-related actions, such as email messages, phone calls, or meetings. Activities may be associated with a contact, account, or opportunity.
принадлежность Принадлежностью называется сопоставление компьютера или сетевого устройства со своими учетными данными в ServiceNow Discovery или Orchestration. После сопоставления устройства с учетными данными это отношение записывается в таблицу базы данных. Учетные данные используются для запроса устройства в ходе обнаружения и действий рабочих процессов. affinity Affinity is the association of a computer or network device to its credentials as established by ServiceNow Discovery or Orchestration. Once a device is matched with its credentials, the relationship is written to a database table, and those credentials are used to query the device in subsequent discoveries and Workflow activities
агент Агент - это имя внешней системы, записывающей сообщения в таблицу Queue [ecc_queue]. Например, имя агента для сервера MID имеет формат mid.server.xxx, где «xxx» - имя сервера MID. agent An agent is the name of an external system (integration) that writes messages to the Queue [ecc_queue] table. For example, the agent name for a MID Server is in the format mid.server.xxx, where \xxx\ is the name of a particular MID Server.
правило распределения Правила распределения указывают, каким образом обрабатывается каждая статья расходов для создания одной или нескольких записей распределения расходов. Условия правил определяются на основе данных исходных статей расхода. allocation rule Allocation rules define how to process each expense line to generate one or more expense allocation records. Rule conditions are defined based on data from the expense line source.
единица распределения Единица распределения отслеживает общий объем ресурсов бизнес-услуги и количество потребляемых единиц согласно записям взаимоотношений между конфигурационными единицами (КЕ) и центрами затрат. allocation unit An allocation unit tracks the total capacity of a business service as well as the number of units being consumed as defined by the CI Cost Center Relationship records
панель навигации Панель навигации расположена на левой стороне страницы и содержит список приложений и модулей. application navigator The application navigator is a navigation bar on the left side of the page that displays a list of applications and modules.
подтверждение Подтверждение - это задача в составе рабочего процесса, запрашивающая действие (подтверждение или отклонение) пользователя или группы перед продолжением выполнения рабочего процесса. approval An approval is a task within a workflow that requests an action (approval or denial) from a user or user group before the workflow continues.
правило утверждения Правила утверждения - это настроенные действия, необходимые для утверждения запроса службы или каталога. approval rule Approval rules are the configured steps required to approve a service or catalog request.
актив Активом называется любое оборудование, программный продукт или расходный материал в среде бизнеса. К активам поставщика услуг относится все, что может способствовать предоставлению услуги. asset An asset is any hardware, software, or consumable found in the business environment. Assets of a service provider include anything that could contribute to the delivery of a service.
класс активов Класс активов - это группа активов, хранящихся в одной и той же таблице базы данных (в таблице Asset [alm_asset] или в ее расширении). В ServiceNow классами активов по умолчанию являются: оборудование, программы и расходные материалы. Если классы по умолчанию не подходят для определенной группы активов, можно создать собственный класс активов. asset class An asset class is a group of assets all stored in the same database table (the Asset [alm_asset] table or a table that extends it). In ServiceNow, the default asset classes are hardware, software, and consumables. If the default classes are not appropriate for a specific group of assets, a custom asset class can be created.
резерв спринта Резерв спринта в гибкой среде разработки Scrum - это упорядоченный список описаний, над которыми члены группы соглашаются работать в режиме спринта. В процессе Scrum используется три списка нерешенных дел: продукты, релизы (выпуски) и спринт. backlog A backlog in the Scrum agile development environment is an ordered list of stories that team members agree will be worked on in a sprint. The Scrum process uses three backlogs: product, release, and sprint.
окно баннера Окно баннера находится в верхней части каждой страницы и содержит эмблему платформы и базовые элементы управления навигацией. banner frame The banner frame is the portion across the top of every page that contains the platform's logo and basic global navigation choices.
базовый класс Базовый класс - это расширенная таблица, которая не расширяет никакие другие таблицы. base class A base class is a table which is extended but does not extend any other tables.
базовое состояние Базовое состояние КЕ - это снимок конфигурационной единицы в ее базовом виде в системе. Все изменения КЕ после создания снимка отображаются в форме КЕ в виде изменений базовых атрибутов. При создании базового состояния записываются атрибуты КЕ и все отношения первого уровня. Записываются и отображаются все изменения базовой КЕ и всех связанных с ней КЕ. baseline A CI baseline is a snapshot of a configuration item as it appears in your system in its basic form. All the changes that you make to that configuration item since the snapshot was created are displayed on the CI form as basic attribute changes. When a baseline is created, the attributes of the CI are captured as well as all first level relationships for the CI. Any changes to the base CI or to any related CI will be captured and displayed.
набор обновлений базового состояния Набор обновлений базового состояния - это набор обновлений, в который записываются все изменения до установки плагина набора обновлений системы. baseline update set A baseline update set is an update set that captures all of the customizations before the System Update Sets Plugin was installed.
окно бездействия Окном бездействия называется запланированный интервал времени, в течение которого запрещены любые изменения. Окно бездействия используется плагином Change Management Collision Detector для обнаружения взаимопротиворечащих изменений. blackout window A blackout window is a schedule that defines a period of time in which no changes should be performed. Used by the Change Management Collision Detector Plugin to detect conflicting changes.
навигационная цепочка Навигационная цепочка - это последовательность фильтров, показанных в виде ссылок в верхней части списка записей. Для просмотра различных представлений нажмите на отдельные элементы цепочки. breadcrumbs Breadcrumbs are a series of filters displayed as links at the top of the record list. Click the individual elements of the path to see a different view.
комплексная модель Комплексная модель - это единая модель, состоящая из индивидуальных моделей. Например, ноутбук, принтер, клавиатуру и мышь можно объединить в единую комплексную модель. bundled model A bundled model is a single model comprised of individual models. For example, a laptop, printer, keyboard, and mouse can be combined in a single, bundled model.
диаграмма сгорания задач График процесса разработки используется Scrum-мастерами в гибкой среде разработки (agile) для сравнения планового и фактического выполнения спринта. burn down chart The burn down chart is used by Scrum masters in an agile development environment to view ideal sprint progress against actual sprint progress.
бизнес-правило Бизнес-правило - это фрагмент кода JavaScript, настроенный для запуска при вставке, обновлении или удалении записи, а также при запросе к таблице. Можно настроить бизнес-правило для запуска до или после выполнения действия с базой данных. В случае запроса бизнес-правило должно быть выполнено до действия с базой данных, чтобы возвращенные пользователю данные соответствовали его правам в системе (его пользовательской роли). Обычно бизнес-правило может выполнить сценарий после того, как пользователь обновит инцидент или повысит приоритет заявки на изменение. business rule A business rule is a piece of JavaScript configured to run when a record is inserted, updated, or deleted, or when a table is queried. A business rule can be set to run before or after the database action has occurred. In the case of a query, the business rule should run before the database operation, so that the data returned to the user is appropriate to his system privileges (user roles). A typical business rule might execute a script after a user updates an incident or escalates the priority of a change request.
бизнес-услуга Бизнес-услуга - это услуга (например, электронная почта или мгновенные сообщения), предоставляемая клиентам подразделением, которое опирается на ИТ-инфраструктуру, состоящую из конфигурационных единиц (КЕ). business service A business service is a service (such as email or electronic messaging) delivered to customers by business units that is supported by an IT infrastructure of configuration items (CI).
схема управления бизнес-услугами Схема управления бизнес-услугами - это база данных, состоящая из конфигурационных единиц (КЕ) и их взаимоотношений в рамках корпоративной сети. Business Service Management (BSM) map A BSM map is a graphical display of the configuration items (CI) within a system and their relationships to one another.
возможность Возможности сервера MID определяют определенные функции сервера MID в пределах диапазона IP-адресов. Функциональность по умолчанию позволяет администраторам выбирать определенные возможности (например, SSH, SNMP, PowerShell) для зондов, запускаемых каждым сервером MID. capability MID Server capabilities define the specific functions of a MID Server within an IP address range. The default functionality enables an administrator to select specific capabilities (for example, SSH, SNMP, PowerShell) for the probes launched by each MID Server.
позиция в каталоге Позиция в каталоге - это любой уникальный и доступный для заказа конечным пользователям элемент или услуга в каталоге услуг. Чтобы указать, каким образом данную позицию можно приобрести, как осуществляется доставка и т.п., эту позицию необходимо связать с планом выполнения каталога услуг. catalog item A catalog item is any unique item or service in the Service Catalog that can be ordered by the end user. To specify how a catalog item is procured, who works on it, or how it is delivered, the item must be tied to a Service Catalog execution plan.
изменение Под изменением понимается любое добавление, изменение или удаление, способное повлиять на работу ИТ-услуг. change A change is the addition, modification or removal of anything that could have an effect on IT services.
подключаемый модуль определения конфликтов в управлении изменениями Плагин определения конфликтов в управлении изменениями дает возможность обнаруживать случаи, когда запланированные изменения противоречат другим изменениям или расписанию. Change Management Collision Detector plugin The Change Management Collision Detector plugin provides the ability to detect whether planned changes conflict with other changes or have other scheduling issues.
действие чата Действия чата - дополнительные элементы, отображающиеся в меню окна чата (требуется подключаемый модуль «Чат»). chat action Chat actions are additional items that appear in the chat window menu (requires the Chat plugin).
очередь чата Очереди чата определяют расписание, персонал поддержки и системные сообщения для чата службы поддержки (требуется плагин «Чат»). chat queue Chat queues define the schedules, support staff, and system messages for help desk chat (requires the Chat plugin).
агент очереди чата Агенты очереди чата - это сотрудники службы поддержки, предоставляющие поддержку для очереди чата службы поддержки (требуется плагин «Чат»). Для поддержки определенной очереди пользователь должен быть членом назначенной группы для этой очереди. chat queue agent Chat queue agents are Service Desk staff who provide support for a help desk chat queue (requires the Chat plugin). To support a particular queue, a user must be a member of the assignment group for the queue.
КЕ (конфигурационная единица) КЕ (конфигурационная единица) - это любой компьютер, устройство или программа в базе данных управления конфигурациями (БДУК). Запись КЕ включает все необходимые данные: изготовитель, поставщик, размещение и т.п. CI (configuration item) A CI (configuration item) is any computer, device, or piece of software in the CMDB. A CI's record includes all relevant data, such as manufacturer, vendor, or location.
взаимоотношение центра затрат и КЕ Взаимоотношение центра затрат и КЕ - это запись, связывающая центр затрат с потребляемой бизнес-услугой и количеством использованных при этом единиц. Используется для вычисления количества выделенных единиц с целью обновления записи о выделенных единицах. CI cost center relationship A CI cost center relationship is a record associating a cost center to a business service that it consumes and the number of units it uses. This is used to calculate the number of allocated units to update the allocation unit record.
тарифная карточка КЕ Тарифная карточка КЕ - это набор повторяющихся затрат КЕ, связанных с набором КЕ. Тарифные карточки обычно создаются для определенной модели затрат. Например, может существовать определенный сервер определенной модели в московском центре обработки данных. Связанные с ним затраты могут отличаться от затрат такой же модели в центре обработки данных в Санкт-Петербурге. У каждой модели будет отдельная тарифная карточка. CI rate card A CI rate card represents a collection of recurring CI Costs associated with a collection of configuration items. Rate cards are usually designed around a certain cost model. An example might be a certain model server running in the New York data center. The costs associated could be different than the same server model running in the London data center. Each model would have a separate rate card.
класс Класс - это таблица, расширенная другой таблицей или расширяющая другую таблицу. class A class is a table that extends or is extended by another table.
дополнительная информация Дополнительная информация - это свойство, позволяющее пользователям открывать запись, на которую указывает ссылка, в текущем окне. Значки ссылок по умолчанию поддерживают открытие дополнительной информации. Также администраторы могут включить всплывающие значки, при использовании которых ссылки открываются в другом окне. clickthrough Clickthrough is a property that allows users to open a referenced record in the current window. Reference icons allow clickthrough by default. Administrators may also enable pop-up icons, which open a referenced record in another window.
сценарий клиента Сценарий клиента - это сценарий JavaScript, созданный для запуска в браузере клиента. ServiceNow поддерживает следующие типы сценариев:\n* onLoad() -- запускается при загрузке формы\n* onChange() -- запускает при изменении значения определенного виджета\n* onSubmit() -- запускается при отправке формы. client script A client script is a JavaScript you create to run in the client browser. ServiceNow supports the following script types:\n* onLoad() -- Runs when a form is loaded\n* onChange() -- Runs when a particular widget changes value\n* onSubmit() -- Runs when a form is submitted
клон Клон - это снимок рабочего экземпляра ServiceNow, используемый для обновления в тестовой среде и в среде разработки. clone A clone is a snapshot of your production ServiceNow instance that you use to refresh your test or development environments.
кластер Кластер - это экземпляр ServiceNow, состоящий из двух или более узлов, в том числе следующих: один или несколько интерфейсных узлов, к которым направляются все запросы пользователей из браузеров, а также рабочие узлы, обрабатывающие отдельные операции, например, зонды Discovery. Кластер может быть организован на одном физическом компьютере или состоять из нескольких физических компьютеров. Все узлы в составе кластера связаны с одной и той же базой данных и имеют одинаковый идентификатор экземпляра. cluster A cluster is a ServiceNow instance comprised of two or more nodes: one or more interface nodes, to which all user requests from the browser are directed, and worker nodes that handle discrete operations, such as Discovery probes. A cluster can exist on the same physical machine or on separate machines. In a cluster, all the nodes point to the same database and have the same instance ID.
база данных управления конфигурациями (БДУК) БДУК - это база данных, состоящая из конфигурационных единиц (КЕ) и их взаимоотношений в рамках корпоративной сети. CMDB (Configuration Management Database ) A CMDB is a database of the configuration items (CI) and their relationships within an enterprise's network.
слияние Слиянием называется метод, с помощью которого наборы импорта могут обновлять существующие записи в рабочей таблице назначения, вместо простой вставки новых записей. Если для какого-либо поля настроено слияние, приложение набора импорта пытается сравнить исходное значение с существующим значением в таблице назначения, которую следует обновить. Если значения не совпадают, создается новая запись. coalescence Coalescence is the method by which import sets may update existing records in a destination production table, rather than simply inserting new records. When coalescence is configured for a field, the import set application attempts to match the source value to the currently existing target value on a production table for updating. If the source value does not match, a new record is created.
коллизия Коллизия возникает, когда входящее обновление заказчика в наборе обновлений применяется к тому же объекту, что и более позднее локальное обновление заказчика. Подключаемый модуль наборов обновлений системы определяет, применяется ли входящее обновление. collision A collision is when an incoming customer update in an update set applies to the same object as a newer local customer update. The System Update Sets Preview determines whether or not the incoming customer update is applied.
конструктор условий Конструктор условий создает инструкцию условия с последовательностью созданных полей. Конструкторы условий используются во многих областях, например в фильтрах, для администрирования опросов и управления доступом. condition builder A condition builder constructs a condition statement with a series of contextually generated fields. Condition builders are used in many applications, such as filters, survey administration, and access control administration.
страница конфигурации Страница конфигурации - это модуль в составе системы управления содержимым, где устанавливаются глобальные значения по умолчанию для страниц содержимого и сайтов. configuration page The configuration page is the module within the Content Management System that sets the global defaults for content pages and sites.
конфликт Конфликт - это запись, создаваемая плагином определения конфликтов в управлении изменениями, когда запланированное изменение пересекается с другими изменениями или запланировано неправильно. Записи конфликтов содержат информацию о типе конфликта и о том, между какими элементами или расписаниями возник конфликт. conflict A conflict is a record generated by the Change Management Collision Detector Plugin when a scheduled change overlaps with other changes, or is otherwise improperly scheduled. Conflict records detail the type of conflict and what items or schedules were in conflict.
расходуемый актив Расходуемый актив - это актив, для которого отслеживается количество, и который может быть передан в количестве больше нуля. Некоторые данные не отслеживаются, например, все места, где может быть установлен расходуемый актив. К числу активов, которые часто обрабатываются как расходуемые, относятся карандаши, кресла и подставки для мониторов. consumable asset A consumable asset is an asset that is tracked qualitatively and can be transferred in quantities greater than zero. Some details are not tracked, such as all locations where the consumable is installed. Examples of assets that are often managed as consumables include pencils, chairs, and monitor stands.
контакт В плагине Sales Force Automation контакты обозначают контактные лица, занимающиеся вопросами продаж. Контакт связывается с заказчиком и назначается представителю отдела продаж для данного заказчика. contact In the Sales Force Automation plugin, contacts represent sales contacts at companies. A contact is linked to an account and hence is assigned to the sales representative for the account.
блок содержимого Блок содержимого - это элемент, который можно добавить на домашнюю страницу или страницу содержимого. Этот элемент содержит определенную форму содержимого: статический код HTML, отчет или список. content block A content block is an item that can be added to a homepage or a content page, and which contains a defined form of content such as static HTML, a report, or a list.
окно содержимого Окно содержимого - это главная область основного пользовательского интерфейса. В окне содержимого отображаются страницы, на которые переходит пользователь. content frame The content frame is the main area of the primary user interface. При использовании короткого опроса браузер отправляет запросы через фиксированные интервалы времени, заданные с помощью свойства glide.short_poll_delay. Например, можно открывать новый запрос через каждые 1000 миллисекунд, получать немедленный ответ (возможно, пустой) и закрывать подключение. short polling Polling is the method by which the browser gets information from the server to send instant messages in Chat. With short polling, the browser sends a request to the server in fixed intervals as defined by the glide.short_poll_delay property. For example, open a new request every 1000 milliseconds, receive an immediate (possibly empty) response, and close the connection.
страница входа на сайт Страница входа на сайт - это страница содержимого, которую пользователь видит при первом входе на сайт содержимого. site entry page A site entry page is a content page designed to be displayed to the user upon first arriving at a content site.
двухпанельное окно выбора Двухпанельное окно выбора - это интерфейс, состоящий из двух столбцов, с помощью которого можно выбирать несколько элементов из списка. slushbucket A slushbucket is a double column interface for choosing multiple selections from an available pool of items.
социальный сегмент ИТ Социальный сегмент ИТ - это использование технологий социальных сетей (например, мгновенных сообщений и микроблоков) для расширения возможностей ИТ. social IT Social IT is the use of social media technology (such as instant messaging and microblogging) to enhance IT operations.
режим редактирования электронной таблицы Режим редактирования таблицы позволяет пользователям изменять данные в представлении списка с помощью клавиатуры, аналогично вводу данных в электронную таблицу. Администраторы могут включить режим редактирования таблицы, установив значение true для свойства glide.ui.list_edit_grid. spreadsheet edit mode Spreadsheet edit mode, or grid editing, allows users to modify data in list view using keyboard navigation, similar to entering data in a spreadsheet. Administrators can enable spreadsheet edit mode for lists by setting the glide.ui.list_edit_grid property to true.
спринт В Scrum спринт - это короткий фиксированный набор циклов заданной длительности, предназначенный для решения быстро возникающих проблем и потребностей. sprint In Scrum, a sprint is a short, fixed schedule of cycles with adjustable scope, designed to address rapidly changing development needs.
статический HTML-блок Статический HTML-блок - это блок содержимого, отображающий код HTML на странице содержимого. static HTML block A static HTML block is a content block that renders HTML within a content page.
правило создания запасов Правило создания запасов - это заданные правила, работающие следующим образом: когда количество определенных активов на складе опускается до заданного числа, определенное количество активов перемещается с другого склада или заказывается у поставщика. stock rule Stock rules are defined criteria stating that when inventory of a particular asset in a stockroom reaches a specified threshold, a certain number should either be transferred from another stockroom or ordered from a vendor.
склад Склад - это место хранения активов. stockroom A stockroom is a place where assets are stored.
игнорируемое слово Игнорируемые слова - это слова, которые не индексируются, поскольку они нерелевантны для результатов поиска. В качестве игнорируемых слов для текстового поиска Zing можно настроить артикли, союзы, личные местоимения и предлоги. stop word Stop words are common words that are not indexed because they are not meaningful in search results. Articles, conjunctions, personal pronouns, and prepositions are examples that may be configured as stop words for Zing text search.
описание История - это краткая запись требований к продукту или бизнес-сценария клиента, используемая при гибкой разработке программ по методике Scrum. Как правило, описание приводится простым языком, чтобы читатель лучше понимал, как должна работать программа. story A story, or user story, is a brief statement of a product requirement or a customer business case that is used in the Scrum method of agile software development. Typically, stories are expressed in plain language to help the reader understand what the software should accomplish.
балл описания Очки за пользовательскую историю в Scrum - это произвольные измерения усилий (не обязательно времени), необходимых для выполнения истории по оценкам членов группы Scrum. story point Story points in Scrum are arbitrary measurements of the effort (not necessarily the time) required to complete a story, based on the estimates of Scrum team members.
Summary Set Таблица Summary Set [sys_report_summary] служит для временного хранения информации, необходимой для отображения графика. Эта таблица поддерживается плагином Custom Charts. Summary Set The Summary Set [sys_report_summary] table temporarily holds the necessary information for rendering a chart. This table is enabled by the Custom Charts plugin.
поддерживаемый релиз (выпуск) Все релизы (выпуски) ServiceNow поддерживаются исправлениями и обновлениями в течение двух циклов релиза (выпуска) компонентов. supported release All ServiceNow releases are supported with patches and hotfixes for two feature release cycles.
тег Теги - это слова, помеченные решеткой (#) в сообщениях Live Feed. Теги позволяют распределять сообщения на категории по ключевым словам или по темам для более эффективного поиска. tag Tags are words marked with a hash (#) symbol in Live Feed messages. Tags are a way to categorize messages by keyword or topic for improved search results.
задача Таблица задач Task [task] является одной из основных таблиц в базовой системе. В этой таблице содержится набор стандартных полей, используемых во всех дополнительных таблицах, таких как Incident и Problem. task Task [task] is one of the core tables provided in the base system. It provides a series of standard fields used on each of the tables that extend it, such as the Incident and Problem tables.
тема Темой называется группа таблиц стилей в системе управления содержимого. theme A theme is a grouping of style sheets in the Content Management System.
отслеживаемая настройка Отслеживаемые настройки - это настройки таблиц, записываемые в наборы обновлений. tracked customization Tracked customizations are customizations to tables which are recorded into update sets.
схема преобразования Схема преобразования сопоставляет поля из набора импорта с полями в одной таблице назначения. Одно поле набора импорта может быть сопоставлено с несколькими полями в таблице назначения. При создании схемы преобразования можно использовать функцию автоматического сопоставления для сравнения полей или сценарии с более сложной логикой. Созданную схему преобразования можно использовать многократно. К набору импорта можно применять несколько схем преобразования (каждую для определенной таблице) в указанном порядке. transform map A transform map matches fields from an import set to fields in a single destination table. A single import set field can be mapped to multiple fields in the destination table. When creating a transform map, the automap feature can be used to match fields, or scripting can be used for advanced logic. Once defined, a transform map can be reused, and multiple transform maps (each for a specific table) can be applied to an import set in the order you specify.
средство навигации по древовидной структуре Средство навигации по древовидной структуре предназначено для навигации по иерархическим спискам. tree picker A tree picker is a navigational tool for selecting from hierarchical lists.
контроль учетных записей Контроль учетных записей - это функция безопасности, ограничивающая права приложений, запускаемых пользователями, у которых нет прав администратора. Эта функция реализована в современных версиях операционной системы Microsoft Windows и может повлиять на работу подключаемого плагина «MID сервер». UAC (User Account Control) UAC is a security feature that limits the permissions of a particular software for non-administrative users. This feature is included in modern Microsoft Windows operating systems and can affect the behavior of the MID Server plugin.
действие пользовательского интерфейса Действие пользовательского интерфейса - это всплывающее меню, позволяющее создавать запись в таблице на основе информации из записи в другой таблице. Например, можно создать новую запись Change непосредственно из существующей записи Problem. UI action A UI action is a pop-up menu selection that enables you to create a record in a table based on the information from a record in another table. For example, you can create a new Change record directly from an existing Problem record.
макрос пользовательского интерфейса Макрос пользовательского интерфейса - это многократно используемый сценарий Jelly, который можно вызвать со страницы пользовательского интерфейса. UI macro A UI macro is a reusable Jelly script that can be called from a UI page.
набор обновлений Набор обновлений - это набор изменений, сделанных пользователем в своем экземпляре, пригодный для передачи на другой экземпляр. При изменениях в экземпляре это изменение автоматически добавляется в существующий набор обновлений. Одновременно может существовать только один текущий набор обновлений. update set An update set is a collection of customizations made by a customer to their instance that can be transferred to another instance. When an update is made to the instance, the change is added automatically to the current update set. There can be only one current update set at a time.
средство выбора набора обновлений Средство выбора набора обновлений - это раскрывающийся список в строке заголовка экземпляра, где можно выбрать набор обновлений, в который будут добавляться изменения, сделанные текущим пользователем. update set picker The update set picker is a drop-down choice list in the instance header bar that selects which update set customizations made by the current user will be grouped in.
источник обновлений Источник обновлений - это экземпляр ServiceNow, откуда текущий экземпляр может получать наборы обновлений. update source An update source is a ServiceNow instance that the current instance can pull update sets from.
суффикс URL-адреса Суффикс URL-адреса определяет URL-адрес страницы контента в системе управления содержимым. Сайты содержимого и страницы контента имеют собственные суффиксы URL-адресов, которые создаются в следующем формате: <путь к экземпляру> + /<суффикс сайта> + /<суффикс страницы> + .do URL suffix A URL suffix defines the URL of a content page within the Content Management System. Both Content Sites and Content Pages have their own URL suffixes, and are generated in the format: <path to your instance> + /<site suffix> + /<page suffix> + .do
сценарий проверки Сценарий проверки управляет форматированием данных, вводимых пользователем в поля, и отображает сообщения об ошибке при нарушениях синтаксиса. Например, можно создать сценарий, проверяющий правильность формата адресов электронной почты, вводимых пользователем. validation script A validation script controls the formatting of user input into data fields and displays an error message for syntax violations. For example, you can create a script that enforces the proper formatting of an email address entered by a user.
набор переменных Набор переменных - это модульный блок переменных, который может совместно использоваться элементами каталога. Достаточно задать набор переменных один раз, чтобы использовать его в разных местах. variable set A variable set is a modular unit of variables that can be shared between catalog items. Define a variable set once and use it in multiple places.
график скорости В Scrum график скорости показывает, сколько единиц работы команда может сделать за один спринт на протяжении определенного времени. Этот расчет помогает Scrum-мастеру составить прогноз о том, сколько нагрузки группа сможет обработать в ходе дальнейших спринтов. velocity chart In Scrum, a velocity chart shows how much effort (in points) a team can handle in one sprint, measured over time. This calculation helps the Scrum master predict how much load a team can handle in future sprints.
представление Представление - это сохраненная версия настроенной формы или списка. С помощью представлений пользователи могут быстро отображать одну и ту же форму или список различными способами. Администраторы могут создавать настраиваемые представления. view A view is a saved version of a personalized form or list. Views enable users to quickly display the same form or list in multiple ways. Administrators can create custom views.
действие рабочего процесса Действия рабочего процесса - это отдельные действия, выполняемые рабочим процессом. К их числу относится: выполнение сценариев, обработка записей, ожидание в течение заданного времени, запись события в журнал и пр. workflow activity Workflow activities are the individual actions the workflow performs as it runs. This can include (but is not limited to) performing scripts, manipulating records, waiting for a set period of time, or logging an event.
поле рабочего процесса Поле рабочего процесса в записи задачи хранит и отображает расположение записи задачи в рабочем процессе. workflow field A workflow field in a task record stores and displays the location of the task record in the workflow.
