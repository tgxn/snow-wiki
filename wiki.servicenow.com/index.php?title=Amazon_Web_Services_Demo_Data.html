1 Overview

The demo data available with Amazon Web Services provides samples of the data types to illustrate its use. Demo data should be loaded only in a development or test instance to prevent conflict with production data.

2 Group Hierarchy

Three user groups and four users are created as demo data. The group Infrastructure and the group Service are the children groups of group Cloud Operation. Users Adam Alba and Bob Brown are in Infrastructure group. User Cindy Cyrus is in Service group. User Dan Daniels belongs to Cloud Operation group. All of the users contain the cloud_admin role.

4 CloudFormation Service Catalog Item

Six CloudFormation Service Catalog Items are created from the demo templates. The demo service catalog item names follow the pattern of "CloudFormation Stack – " plus the name of CloudFormation Template. For example, one item name is CloudFormation Stack – AdwithPrivateSubnet. All of the demo catalog items use a demo account named AWS Account and US West (Oregon) Region.

In each catalog item, the user can specify input parameters to provision stacks. For example, user can specify what public and private subnets to use for the CloudFormation Stack – LAMPstack-VPC-PubPrivSubnet-RDS catalog item. Dynamic input validation client scripts are also available in the order form to help user validate input parameters before actual provisioning.

For the Application tag, Active Directory, Community, Retail Portal, Shopping Cart and PeopleSoft are created. For the Cost Center tag, Retail is created.
