1 Overview

The Workday integration provides you with a way to keep employee profile information in Human Resources Service Management in sync with their your organization's existing Workday solution. You need a Workday account with administrative privileges to configure the Workday integration.

2 How the Workday Integration Works

Workday is the primary source of all employee data. After you put your employee information in Workday and enable the integration, the instance uses SOAP web services to pull information from Workday on a regular basis and populate the HR Profile [hr_profile] and Job Profiles [hr_job_profile] tables in Human Resources Service Management, and the system Department [cmn_department] and Location [cmn_location] tables. The data taken from Workday is contained in the Worker object. By default, the integration pulls the following data from the Worker object:

  • Worker information, such as user name, first and last name, and email address
  • Location
  • Department
  • Job profile

The integration also supports automatic record updates based on future dates for events such as employee onboarding, offboarding, transfers, or promotions. For example, after you add information in Workday for an employee that comes on board in a week, the instance retrieves the employee data at the scheduled pull and creates the user and HR records but does not activate the records until the onboard date. Offboarding works in a similar manner. When you set a termination date for an employee in Workday, the system automatically inactivates the user and HR records, but one day after the termination date.

This integration is available starting with the Fuji release.

3 Menus and Modules

The Workday Integration module
 Activating the Workday integration adds the Integration - Workday menu with the following modules:
  • Import Schedule: Access the scheduled job that calls the workday_Data_Retriever script include for data retrieval. You can use the default import schedule, or create your own.
  • Import Sets Runs: Access the import sets that start with u_workday to show you the data you import into the workday or HR tables. You do not have to use import sets to import data, but you have an option to run an import set transform before actually writing user data to the HR tables.
  • Script Includes: Access scripts that are needed for data retrieval. You can customize some script includes to add additional fields to the import table.
  • Transform Errors: Open the log of all errors encountered during any import set operations.
  • Transform Maps: Access the maps for the relationships between the import table and the target table in ServiceNow. You can modify transform maps if you want to import additional information from Workday that is not imported by default.
  • Transform Runs: Access transform histories that the instance performed. Use these transform runs to see details about each data import.
  • Workday Webservice: Open the SOAP web service that you use to connect to Workday. You can make changes, if necessary, to the default web service settings, which contain the WSDL to connect to Workday.
  • Workday Properties: Configure properties to connect to Workday web services.
  • Integration Table: Open the dictionary entries for all fields on the integration tables, where all your imported Workday data is stored until it is pushed to the HR Profile table. You can modify the table if you want to import additional information from Workday that is not imported by default.
  • Script Logs: View the list of system log messages for information about Workday data import. Use these to troubleshoot any integration issues with data import.

4 Activating the Workday Integration

Human Resources: Workday Integration is available as a separate subscription. It requires the Human Resources Service Management: Employee Change plugin, which is automatically activated when you activate the Human Resources: Workday integration plugin.

To purchase a subscription, contact your ServiceNow account manager. After purchasing the subscription, activate the plugin within the production instance.

You can evaluate the feature on a sub-production instance without charge by activating it within the instance.

Click the plus to expand instructions for activating a plugin.

If you have the admin role, use the following steps to activate the plugin.

  1. Navigate to System Definition > Plugins.
  2. Right-click the plugin name on the list and select Activate/Upgrade.
    If the plugin depends on other plugins, these plugins are listed along with their activation status.
  3. [Optional] If available, select the Load demo data check box.
    Some plugins include demo data—sample records that are designed to illustrate plugin features for common use cases. Loading demo data is a good policy when you first activate the plugin on a development or test instance. You can load demo data after the plugin is activated by repeating this process and selecting the check box.
  4. Click Activate.
